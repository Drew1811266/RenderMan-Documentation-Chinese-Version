Engine Metrics

Source: https://rmanwiki-27.pixar.com/space/TRA/22184236/Engine+Metrics
Downloaded: 2026-06-15T03:49:12+00:00

"Engine Metrics" 标签页以图形方式提供若干 engine 健康状况和性能统计的近期历史。Tractor engine 会定期采样这些值，并发送给已订阅的 Dashboard 客户端。默认情况下，大约每 15 秒采集一次新样本，图表中显示约 10 分钟的历史。

这些图表旨在为管理员提供 engine 状态的快速概览，并帮助在出现性能问题时识别问题。由于图表基于从快速变化系统中采集的样本，并非所有显示值都会在每一刻都精确匹配系统状态。
 

等待中的任务：整个作业队列中等待执行的任务数。

活动任务：已经分派到 blade 且当前正在运行的任务数。
 

总 slot 数：整个 farm 上所有 blade 报告的 "slot" 计数总和。slot 是可分配容量的抽象单位。有些站点会为每台主机配置一个 slot，有些会为每个核心使用一个 slot，其他站点则会向 blade 添加自己的插件，以生成动态 slot 值。

活动 slot 数：正在运行的活动任务消耗的 slot 数。每个命令使用的数量在作业脚本中指定，并且可以是在给定 "at-least" 与 "at-most" 值之间的动态数量。
 

每秒请求数：tractor-engine 每秒接收的 HTTP 请求总数估计值。其中包括来自 blade、浏览器和外部脚本的所有请求。

每秒调度数：成功任务分配的速率。也就是每秒有多少来自 blade 的工作请求成功匹配作业队列中等待的任务，并交回 blade 执行。
 

接收积压：tractor-engine 已接收但尚未委派给具体 engine 子系统处理的入站 HTTP 请求的内部度量。

分配器积压：等待从队列分配任务的 blade 待处理请求数。其中一些请求可能已成功处理但不会导致任务启动，例如没有剩余的排队任务，或 blade 的 service key 不匹配任何等待命令的要求。

i/o 池积压：tractor 使用一组 "shipper" 线程来并发处理已连接 socket 上的 i/o 操作。此值是等待可用线程的高层 i/o 动作数量。

monitor 积压：等待处理的 UI 专用 HTTP 请求数。
 

engine 负载：tractor-engine 主机自身当前 "load average" 的归一化值。显示值是系统一分钟 loadavg，例如由 'uptime' 或 'top' 给出的值，但会再除以该主机的 CPU 核心数。因此，拥有 16 个核心且其中 8 个核心满负载的主机会报告名义 loadavg 为 "8.0"，而此图表会报告 "0.5"。其意图是表示主机 CPU 容量中正在使用的总体比例。此负载不一定完全来自 tractor-engine 活动。主机上的其他活动，例如文件服务或其他正在运行的应用程序，也会贡献负载。
 

用户连接：最近连接的客户端会话数。其中包括 Dashboard 会话，以及也使用 HTTP 请求协议接收作业状态更新或请求作业/blade 详情的其他客户端。由于持续 HTTP "session" 的 "end" 很难界定，engine 会认为某个连接仍然是 "current"，直到相当长的间隔内没有来自预期客户端的其他活动。具体来说，完全重新加载 tractor 浏览器窗口会暂时导致旧会话和新会话都被计为可能活动。
