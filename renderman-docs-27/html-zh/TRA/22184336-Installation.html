安装前，请选择 Tractor Engine 主机

选择一台主机来运行 Tractor Engine。该主机需要运行 Linux 或 macOS；支持的版本列在 RenderMan Products 下载页面上。Blade 主机可以是 Linux、macOS 或 Windows。

具有更快 CPU、更多核心和更快存储的主机能够更快完成调度，并处理更高吞吐量。

更大的磁盘往往很重要，因为可以保存更多已完成作业用于分析。

对于大型队列，普通商用硬盘已经足够；1 GB 约可存储 200,000 个任务。不过，如果作业会永久归档用于历史分析，则可能需要更大的硬盘。另外，默认配置下还需要磁盘空间来存储日志文件。

通常不建议在虚拟机上运行 tractor-engine，除非仔细分配 CPU、磁盘和网络资源，使其性能接近裸机。

为简单起见，Engine 主机不应运行 Web 服务器或其他使用端口 80 的服务器。Engine 可以配置为使用不同端口，但使用默认 80 会 极大 简化配置和 Web 浏览器 UI 访问。

在 Tractor Engine 主机上下载并运行安装程序

要下载 Tractor 安装程序，请登录 https://renderman.pixar.com/forum/ 并点击页面顶部蓝色的 Downloads 链接，然后从列表中选择 Tractor 条目查看可用下载。

在所有情况下都需要 完整安装程序 来安装 Tractor Engine，并且它也包含所有其他工具。

注意：对于其他角色，也有几个小得多的软件包，包含仅 Python 的 engine-client 组件，可以按需 单独下载。这些包括用于 tractor-blade 和脚本 API 的 Python wheel，适合通过 pip 在给定主机或容器的 Python 虚拟环境中安装其中一个或两个组件。还有一个 Python zipapp，包含连接到 Engine 的 Python 客户端集合。该 zipapp 可以在独立模式中使用，而不改变现有本地标准 Python。 有关这些 Python 子包的详细信息，请参阅 clients 下载归档中的 README。这些轻量级、仅 Python 的软件包对于其他非 Engine 主机类型是可选的，完整安装程序也可以用于这些主机。

完整安装程序（rpm、msi 或 pkg）必须以系统管理员权限运行（例如在 Linux 上使用 sudo）。安装目录为：

  • Linux 上为 /opt/pixar/Tractor-VERSION，

  • macOS 上为 /Appications/Pixar/Tractor-VERSION，

  • Windows 上为 C:\Program Files\Pixar\Tractor-VERSION。

本文档中会将该路径称为 INSTALL_DIR。

注意：

  • 完整安装程序包含 全部 Tractor 组件，但 只有一个子集 在每种机器类型上需要：Engine 主机、渲染节点和用户工作站。请参阅上面关于小得多的 Python-only 客户端下载的说明；某些情况下它们可能更合适。

  • 中央 tractor-engine 服务（唯一的一个）比任何其他组件都需要完整安装中的更多内容。

  • 可以把完整软件包和较小的 Python 软件包安装到文件服务器上，让许多主机和用户访问。但如上所述，每种机器类型上只需要其中一部分。

获取许可证。

在完整安装过程接近结束时，LicenseApp 程序会启动以获取许可证文件。许可证 只需要在 Engine 主机上使用，并且只在之前尚未安装许可证时需要。如果你已有许可证，或只是安装到 Blade 主机，那么可以直接 关闭许可证对话框，使用关闭按钮。如果确实需要下载许可证，则应输入 RenderMan 论坛登录名、密码以及购买时收到的序列号；随后应用程序会连接到 Pixar 并获取许可证。

如果在 Engine 安装后需要下载许可证文件，只需运行位于 INSTALL_DIR/bin 中的 LicenseApp 应用程序。

默认情况下，许可证文件 pixar.license 会下载到安装目录的父目录（/opt/pixar 或 /Applications/Pixar）。也可以通过设置 PIXAR_LICENSE_FILE 环境变量，或在 tractor.conf 中设置 LicenseLocation 来覆盖许可证位置。因为 Tractor 通常与浮动许可证服务器配合运行，pixar.license 文件将是存根或重定向许可证，只是指向许可证服务器机器。

Engine 管理可用的 Tractor 席位；总席位数表示农场中可并发运行的最大任务数量。Blade 启动不需要许可证，空闲时也不会消耗席位。Dashboard 不消耗许可证。

创建 tractor-engine 主机名别名

Dashboard、Blade 和命令行工具需要能够找到 Engine 才能与其通信。因此，创建名为 tractor-engine 的主机名别名很有用，该别名指向 Engine 主机的真实主机名，尤其是在生产环境中。所有这些工具默认都会尝试连接到 tractor-engine，因此使用该名称而不是应用覆盖选项，可以极大简化日常使用。Engine 可以选择使用简单的多播协议广播其网络地址，这适合某些 小型 安装。（更多信息请参阅 engine discovery.)

使用 tractor-engine 别名还允许通过切换别名来使用新的底层主机作为 Engine 主机，而不必在所有工具使用场景中更新覆盖选项。请咨询系统管理员，选择创建主机名别名 tractor-engine 以将连接重定向到所选 Engine 主机的最佳方法。这可能包括：

  • 添加 DNS CNAME 记录，将 tractor-engine 指向站点的 Engine 主机名，或

  • 向 Engine 主机名的 LDAP 条目添加 cn 条目，或

  • 向 /etc/hosts 添加主机名别名（不可扩展）

可以在用户/Blade 主机上使用几个命令行工具来验证它们是否能通过主机名别名到达 Engine 主机本身：
     host tractor-engine
  nslookup tractor-engine
  ping tractor-engine

Engine 运行后（见下文），Web 浏览器也可以通过浏览该主机名来测试 tractor-engine 别名。

tractor-blade、tractor-spool 和 tq 等命令行组件可以通过设置 TRACTOR_ENGINE 环境变量，或添加选项 --engine=HOSTNAME:PORT  来指向不同 Engine，以匹配你的 Engine 安装。配置文件中对 tractor-engine 的引用也可能需要更改。

安装 Blade 服务
每台将执行 Tractor 任务的机器（通常是渲染农场节点）都必须运行 tractor-blade Python 程序。具体内容请参阅 Setting Up Services。
