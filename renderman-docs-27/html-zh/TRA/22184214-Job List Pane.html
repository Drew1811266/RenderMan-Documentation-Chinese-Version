作业列表面板

Source: https://rmanwiki-27.pixar.com/space/TRA/22184214/Job+List+Pane
Downloaded: 2026-06-15T03:49:12+00:00

作业列表面板是用户与 Tractor 作业交互的主要界面。它可以查看农场上正在运行的作业，对队列顺序进行排序并重新调整优先级。你也可以选择作业，并把它们从活动作业队列中移除。被移除的作业会移动到归档目录，供其他软件访问。
 

选择操作

作业列表面板会响应多种点击和键盘命令：

  • 选择

    只需点击一个作业，它就会被选中。所选作业的任务图会显示在右侧的任务图面板中。

  • 多选操作

    在 Mac 上按住 Option 键，或在 Windows 上按住 Control 键点击，会把当前作业添加到选择列表。如果该作业已在列表中，则会将其移除。

    按住 Shift 键点击，会选中当前选择项与上一次选择项之间的所有作业。

    使用 Control 键配合 A 键会执行 全选操作 操作。注意，具体浏览器不同，可能需要使用并非该操作系统常规习惯的按键。例如，由于 Macintosh 浏览器使用 Apple-A，你需要改用 Control-A 或 Option-A。
     

  • 上下文菜单

    右键点击会打开上下文菜单，用于对所选作业执行操作。根据所选作业当前适用的动作，可能只显示部分菜单项。并非所有菜单项在所有情况下都可用。


     

  • 跳过所有出错任务（Skip All Error Tasks）

    因错误停止的任务（非零命令退出码）会被忽略，也就是强制标记为 "Done"，这样等待它们的其他任务就可以继续执行。通常当然应当理解并处理错误，但在测试期间，有时越过已知问题继续执行会很有价值。
     

  • 重试所有出错任务（Retry All Error Tasks）

    如果某个作业因错误而停止，此命令会让 Engine 重试出错的任务，以及其他因错误而被阻塞的任务。
     

  • 重试正在运行的任务（Retry Running Tasks）

    所选作业中当前正在运行的任务会被取消并重新排入队列。
     

  • 暂停新任务启动（Pause New Task Launches）

    通知 Engine 暂停所选作业。活动任务会继续运行直到完成，但不会启动任何 新的 任务。
     

  • 取消暂停并恢复任务启动（Unpause & Resume Task Launches）

    通知 Engine 取消所选作业的暂停状态。随着 Blade 可用，之前暂停的作业会再次成为可调度候选。
     

  • 重启作业

    通知 Engine 重新运行该作业，就像它刚刚重新提交到作业队列一样。重启发生前，所有正在运行的任务都会被取消。对作业属性的任何手动编辑都会重置为提交时的值。
     

  • 中断作业

    通知 Engine 暂停该作业，并终止它的所有活动任务。
     

  • 固定与取消固定作业（Pin & Unpin Job）

    通知 Engine：Dashboard 用户希望固定或取消固定所选作业。作业被固定后，无论选择哪个排序列，它都会显示在作业列表顶部。所选排序列实际上成为次级排序条件，作业的 pin/unpin 状态成为主要条件。固定作业不会影响其他用户 Dashboard 中的作业显示。
     

  • 锁定与解锁作业（Lock & Unlock Job）

    通知 Engine 锁定或解锁所选作业。作业被锁定后，只有锁定该作业的用户可以编辑其属性或对其执行操作。协调人员可以锁定作业，以防作业所有者在调试期间删除该作业。管理员可能希望锁定关键作业，确保它们不会被意外删除。
     

  • 归档作业

    通知 Engine 从队列中移除所选作业。当作业列表顶部的 archived 按钮被启用时，可以通过 Dashboard 访问历史作业信息，例如任务执行历史、命令统计信息和作业备注。
     

  • 删除作业

    通知 Engine 从队列中移除所选作业。历史作业信息会丢失。
     

  • 将作业提前到队首（Shuffle Job To Top）

    通知 Engine 将所选作业与同 priority、同 owner、且更早提交的作业轮换其 "place in line"，使所选作业比默认提交顺序更早运行。此轮换不会影响队列中不同 priority 或不同 owner 的其他作业的顺序。
     

  • 追踪作业

    向 Engine 请求一份报告，说明该作业是如何被纳入调度考量的。结果会显示在覆盖作业列表面板的专用面板中。tracer 输出面板可通过点击右上角的 'x' 图标关闭，也可以按 'Esc' 关闭。
     

键盘快捷键

有几个键盘快捷键可用于管理所选作业：

  • 删除或退格键（Delete or Backspace）

    终止所选作业的所有活动任务，并将这些作业从队列中移除。已删除的作业不能在 Dashboard 的作业列表中查看；不过可以在 Query Page 中通过在搜索菜单中选择 在搜索中包含归档（include archives in search） 来查看。

     

  • R

    重试所选作业中所有出错的任务。

  • Ctrl/Command-A 全选快捷键

    选择所有作业。
     

  • Return/Enter 确认键

    在对话框中执行 'yes'。
     

  • Esc

    在对话框中执行 'no'。关闭任务日志面板和 tracer 面板。
     

  • F5

    刷新作业列表。
     

  • L

    切换所选任务的日志视图。
     

  • A、B、D、E 快捷键

    在作业图中选择下一个 active、blocked、done 或 errored 任务。
     

  • 使用 Shift + A、B、D、E 组合的快捷键

    在作业图中选择上一个 active、blocked、done 或 errored 任务。
     

  • P

    选择作业图中当前所选任务的父任务。
     

  • C

    选择作业图中当前所选任务的第一个子任务。
     

  • S

    选择作业图中当前所选任务的下一个同级任务。
     

  • Shift + S 快捷键

    选择作业图中当前所选任务的上一个同级任务。
     

  • X

    切换作业图中所选节点的折叠和展开状态。
     

归档作业

如果系统已配置为归档作业，则作业上下文菜单会显示 归档作业 而不是 删除作业。要查看 archived 作业，请按作业列表顶部的 current/archived 按钮，使 archived 高亮。对 archived 作业唯一可执行的操作是 restore，它会取消归档该作业，并将其移回当前作业集合。

归档或删除作业可以帮助系统保持最佳性能，特别是对于拥有数十万个作业的站点。

作业被归档后，可以检查历史作业信息，例如任务执行历史、命令统计信息和作业备注。
 

其他界面提示

  • 进度条

    显示区第一列包含一个小型进度条，用于指示作业的完成进度。进度条会以绿色显示 active 作业，以红色显示 error 作业。等待执行的任务显示为灰色。
     

  • 所选作业

    作业被选中后，该作业的背景色会改变，以表示它已被选中。
     

  • 重新加载计时器

    首选项面板包含一个条目，用于定义自动重新加载间隔，默认设置为 600 秒（10 分钟）。系统会按照下拉选择器中的值定期自动刷新作业数据。使用筛选器查看作业列表时，自动刷新很有优势。筛选器由 Engine 应用，而自动重新加载能让作业列表更有效地保持同步。

    在大型站点，此查询返回的数据量可能很大，因此应谨慎选择重新加载间隔，以避免网络瓶颈。
