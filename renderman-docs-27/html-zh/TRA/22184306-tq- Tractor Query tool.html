tq: Tractor 查询工具

Source: https://rmanwiki-27.pixar.com/space/TRA/22184306/tq%3A+Tractor+Query+tool
Downloaded: 2026-06-15T03:49:12+00:00

tq 是一个强大的命令行工具，用于搜索并操作作业、任务、命令、调用记录和 blade。它允许广泛检查队列和渲染农场的当前状态，也可对已经运行过的命令进行历史分析。

tq 具有完整的内置帮助，因此无需离开命令行就能获得说明。要访问此帮助，只需输入 tq help。

更深入的示例请参阅 tq Cookbook。

 

简单示例

tq 把子命令名称作为第一个参数。在这个简单示例中，tq jobs 会列出系统中的所有作业。

% tq jobs

    jid user   title                     pri wait actv err done spooled
======= ====== ======================== ==== ==== ==== === ==== ===========
1305301 joe    RenderMan Monkey Left     400       840      160 05/30|14:26
1305303 joe    RenderMan Monkey Right    400  925   72   1    3 05/30|14:48
1305311 eve    Comp MV-CA 8_25            50         1          05/31|12:02
1305312 pablo  Interactive 4              20    5             3 05/31|12:03
1305313 btr    Interactive 5              20         1        7 05/31|12:07
1306042 mcgr   Interactive 6              20    8               06/04|16:47
...

类似地，也可以列出任务、命令、调用记录和 blade。

通常，用户只想查看条目的一个子集。命令行上剩余的非选项参数会组成一个 搜索子句，用于缩小搜索范围。例如，此命令会列出用户 mango 拥有且在 24 小时前提交的所有作业：

% tq jobs "owner=mango and spooltime < -1d"

简而言之，搜索子句是一个布尔表达式（这里使用 and），由实体属性（owner、spooltime）的比较（=、<）组成。为了减少命令行上的输入量，做了一些便利处理。例如，字面值 mango 没有放在引号中，而字符串 -1d 是表示“一天前”的简写形式。

有关搜索子句语法的更多细节可在 这里找到。

 

列

tq 对每个命令都有默认显示的属性列表。可以使用 --columns 或 -c 选项显示另一组属性。

例如，要显示作业 id、owner、title 和 metadata 列：

% tq jobs --columns jid,owner,title,metadata

还可以在列名之前加 + 或 -，相对于默认列集合添加或移除列。例如，下面的命令会移除 title 列，并添加 spoolhost 和 spoolfile 列：

% tq jobs -c -title,+spoolhost,+spoolfile

某些列有默认固定宽度；较宽的值会被替换为 ... 以适配列宽。其他列具有动态宽度，会设置为要显示的最宽值。可以使用 = 运算符指定列宽，其中 0 用于指定动态大小列。在此示例中，metadata 列的宽度为所显示值中的最宽值，现有 title 列固定为 30 个字符：

% tq jobs -c +metadata=0,=title=30

注意，在某些 shell（如 zsh）中，如果选项参数以等号开头，必须放在引号中。

% tq jobs -c "=title=30,+metadata=0"

 

排序

可以使用 --sortby 或 -s 选项对结果排序。可以指定多个属性来表示次级排序。条目默认按升序排列，除非属性前加上 - 以表示降序排列。

例如，要显示类似作业在 FIFO 队列中会被处理的顺序，可以按 priority 降序、spooltime 升序列出它们，如下所示：

% tq jobs --sortby -priority,spooltime

 

限制结果数量

结果集可以缩短，使其适用于“top”列表。例如，此命令会列出农场上活动任务数量最多的十个作业：

% tq jobs active -s -numactive --limit 10

此命令会列出磁盘空间最少的二十台主机。

% tq blades -s availdisk --limit 20

 

关联关系

在上述示例中，条目的属性用于搜索子句、列规范和排序。在所有这些情况下，也可以通过在属性前加实体名称来指定关联实体的属性。例如，下面的命令会显示 walt 拥有的活动任务。

% tq tasks state=active and Job.owner=walt

类似地，下面的命令会显示过去 24 小时内提交的作业中的所有错误任务，包括这些作业的 title 和 spooltime，并按作业 priority 排序。

% tq tasks "state=error and Job.spooltime > -24h" -c +Job.spooltime,Job.title -s Job.priority

因为 spooltime 和 priority 不是作业以外其他实体的属性，所以不需要实体前缀。下面是等价写法：

% tq tasks "state=error and spooltime > -24h" -c +spooltime,Job.title -s priority

操作

tq 不仅可以列出条目，也可以对它们执行操作。这个命令会显示我所有带错误的活动作业：

% tq jobs mine and active and error

可以按如下方式全部暂停：

% tq pause mine and active and error

这个命令会对磁盘空间最少的十个 blade 执行 nimby：

% tq nimby --sortby availdisk --limit 10

所有可能的命令可以用 tq help commands 列出。

某些操作有额外选项。例如，修改作业的 priority 时，使用 --prioriy 或 -p 选项指定 priority。下面的命令会把我所有作业的 priority 改为 500。

% tq chpri -p 500 mine

在对条目执行操作之前，会显示该条目，并提示用户。

% tq delete mine 

    jid user   title                pri blkd redy actv err done spooled
========== ====== =================== ==== ==== ==== ==== === ==== ===========
      1234 adamwg starting sleep test    1                      26 01/31|16:53
Delete this job? yes/no [y]:

这有助于避免一次输错命令造成大范围破坏。如果你 绝对 确定目标条目已正确识别，可以用 --yes 选项绕过提示；该选项出现在命令 之前。当把操作应用于数千个条目，或 tq 由另一个脚本或 cron 作业调用时，这很有用。

例如，这会向你的所有作业说再见。

% tq --yes delete mine

 

全局选项

有些选项被视为对所有命令都全局生效，因此出现在 command 之前。其中一个就是上面所示的 --yes。

tq help usage 会列出所有全局选项。

--user、password、--login 和 --logout 在需要认证时使用。

--engine 用于让 tq 指向不同于默认 engine，或不同于 TRACTOR_ENGINE 环境变量所定义 engine 的另一个 engine。

--force 是操作由其他用户拥有的作业时必需的，用于帮助防止意外损坏其他用户的作业。注意，用户仍需要 engine 授予适当权限，才能成功完成操作。

 

已归档作业

默认情况下，运行 tq 时只会搜索未删除作业的实体。不过，已删除作业的实体也可以被 包含，方法是使用 --archives 或 -a 全局选项。这使它适合查看历史调用记录数据。

下面的命令会查找过去一周在 blade c2001 上运行的所有调用记录，包括已删除作业的调用记录：

% tq --archives invocations "blade=c2001 and starttime > -1w"

如果你想要 只包含 已删除作业的结果，可以向搜索子句添加 deleted 别名。

% tq --archives invocations "blade=c2001 and starttime > -1w and deleted"

要使用 undelete 子命令，--archives 标志是必需的，因为只有已删除作业可以取消删除。

% tq -a undelete jid=1234

内置帮助

tq 有详细的内置帮助和有用示例，因此无需离开命令行就能获得更多信息。

% tq help

会显示帮助主题的顶层视图。

% tq help subcommands

会显示所有可用 tq 子命令的完整列表。

要获取特定命令的帮助，请用命令名称运行 tq help。例如：

% tq help jobs

会显示用法说明，以及示例和每个可用选项的详细说明。
