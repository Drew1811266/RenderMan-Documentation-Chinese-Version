tq: Tractor Query tool

Source: https://rmanwiki-27.pixar.com/space/TRA/22184306/tq%3A+Tractor+Query+tool
Downloaded: 2026-06-15T03:49:12+00:00

tq 是一个强大的 command line tool，用于 search 并 operate jobs、tasks、commands、invocations 和 blades。它允许对 queue 和 render farm 的当前状态进行广泛 inspection，也可对已经运行过的 commands 进行 historical analysis。

tq 具有 extensive built-in help，因此无需离开 command-line 就能获得帮助。要访问此 help，只需输入 tq help。

更深入的 examples 请参阅 tq Cookbook。

 

简单使用示例（A Simple Example）

tq 把 subcommand name 作为第一个 argument。在这个 simple example 中，tq jobs 会列出 system 中的所有 jobs。

% tq jobs

    jid user   title                     pri wait actv err done spooled
======= ====== ======================== ==== ==== ==== === ==== ===========
1305301 joe    RenderMan Monkey Left     400       840      160 05/30|14:26
1305303 joe    RenderMan Monkey Right    400  925   72   1    3 05/30|14:48
1305311 eve    Comp MV-CA 8_25            50         1          05/31|12:02
1305312 pablo  Interactive 4              20    5             3 05/31|12:03
1305313 btr    Interactive 5              20         1        7 05/31|12:07
1306042 mcgr   Interactive 6              20    8               06/04|16:47
...

类似地，也可以列出 tasks、commands、invocations 和 blades。

通常，user 只想查看 items 的一个 subset。Command line 上剩余的 non-option arguments 会组成一个 search clause，用于缩小 search 范围。例如，此 command 会列出 user mango 拥有且在 24 小时前 spooled 的所有 jobs：

% tq jobs "owner=mango and spooltime < -1d"

简而言之，search clause 是一个 boolean expression（这里使用 and），由 entity attributes（owner、spooltime）的 comparisons（=、<）组成。为了减少 command line 上的输入量，做了一些便利处理。例如，literal mango 没有放在 quotes 中，而 string -1d 是表示 one day ago 的 short form。

有关 search clause syntax 的更多细节可在 这里找到。

 

Columns（列）

tq 对每个 commands 都有默认显示的 attributes list。可以使用 --columns 或 -c option 显示另一组 attributes。

例如，要显示 job id、owner、title 和 metadata columns：

% tq jobs --columns jid,owner,title,metadata

还可以使用 + 或 - 加 column name，相对于默认 columns set 添加或移除 columns。例如，下面的 command 会移除 title column，并添加 spoolhost 和 spoolfile columns：

% tq jobs -c -title,+spoolhost,+spoolfile

某些 columns 有默认 fixed width；较宽的 values 会被替换为 ... 以适配 column。其他 columns 具有 dynamic width，会设置为要显示的最宽 value。可以使用 = operator 指定 column width，其中 0 用于指定 dynamically sized column。在此示例中，metadata column 的 width 为所显示值中的最宽值，现有 title column 固定为 30 characters：

% tq jobs -c +metadata=0,=title=30

注意，在某些 shells（如 zsh）中，如果 option arguments 以 equal sign 开头，必须放在 quotes 中。

% tq jobs -c "=title=30,+metadata=0"

 

Sorting（排序）

可以使用 --sortby 或 -s option 对 results 排序。可以指定多个 attributes 来表示 secondary sorting。Items 默认按 ascending order 排列，除非 attribute 前加上 - 以表示 descending order 降序排列。

例如，要显示类似 jobs 在 FIFO queue 中会被处理的顺序，可以按 priority descending、spooltime ascending order 列出它们，如下所示：

% tq jobs --sortby -priority,spooltime

 

Limiting（限制结果数量）

Result sets 可以缩短，使其适用于 top lists。例如，此 command 会列出 farm 上 active tasks 数量最多的十个 jobs：

% tq jobs active -s -numactive --limit 10

此 command 会列出 disk space 最少的二十台 hosts。

% tq blades -s availdisk --limit 20

 

关联关系说明（Affiliation）

在上述 examples 中，items 的 attributes 用于 search clauses、column specifications 和 sorting。在所有这些情况下，也可以通过在 attribute 前加 entity name 来指定 affiliated entities 的 attributes。例如，下面的 command 会显示 walt 拥有的 active tasks。

% tq tasks state=active and Job.owner=walt

类似地，下面的 command 会显示过去 24 小时内 spooled 的 jobs 中的所有 error tasks，包括其 jobs 的 titles 和 spooltimes，并按 job priority 排序。

% tq tasks "state=error and Job.spooltime > -24h" -c +Job.spooltime,Job.title -s Job.priority

因为 spooltime 和 priority 不是 job 以外其他 entity 的 attributes，所以不需要 entity prefix。下面是等价写法：

% tq tasks "state=error and spooltime > -24h" -c +spooltime,Job.title -s priority

Operations（操作）

tq 不仅可以 list items，也可以 operate on them。这个 command 会显示我所有带 errors 的 active jobs：

% tq jobs mine and active and error

可以按如下方式全部 pause：

% tq pause mine and active and error

这个 command 会对 disk space 最少的十个 blades 执行 nimby：

% tq nimby --sortby availdisk --limit 10

所有可能的 commands 可以用 tq help commands 列出。

某些 operations 有额外 options。例如，修改 job 的 priority 时，使用 --prioriy 或 -p option 指定 priority。下面的 command 会把我所有 jobs 的 priority 改为 500。

% tq chpri -p 500 mine

在对 item 执行 operation 之前，会显示该 item，并提示 user。

% tq delete mine 

    jid user   title                pri blkd redy actv err done spooled
========== ====== =================== ==== ==== ==== ==== === ==== ===========
      1234 adamwg starting sleep test    1                      26 01/31|16:53
Delete this job? yes/no [y]:

这有助于避免一次 mistyped command 造成大范围破坏。如果你 绝对 确定目标 items 已正确识别，可以用 --yes option 绕过 prompt；该 option 出现在 command 之前。当把 operation 应用于数千 items，或 tq 由另一个 script 或 cron job 调用时，这很有用。

例如，这会向你的所有 jobs 说再见。

% tq --yes delete mine

 

Global Options（全局选项）

有些 options 被视为对所有 commands 都 global，因此出现在 command 之前。其中一个就是上面所示的 --yes。

tq help usage 会列出所有 global options。

--user、password、--login 和 --logout 在需要 authentication 时使用。

--engine 用于让 tq 指向不同于默认 engine 或 TRACTOR_ENGINE environment variable 所定义 engine 的另一个 engine。

--force 是操作由其他 user 拥有的 jobs 时必需的，用于帮助防止意外损坏其他 users 的 jobs。注意，user 仍需要 engine 授予适当 permissions，才能成功完成 operation。

 

Archived Jobs（已归档作业）

默认情况下，运行 tq 时只会 search 非 deleted jobs 的 entities。不过，deleted jobs 的 entities 也可以被 包含，方法是使用 --archives 或 -a global option。这使它适合查看 historical invocation data。

下面的 command 会查找过去一周在 blade c2001 上运行的所有 invocations，包括 deleted jobs 的 invocations：

% tq --archives invocations "blade=c2001 and starttime > -1w"

如果你想要 只包含 deleted jobs 的 results，可以向 search clause 添加 deleted alias。

% tq --archives invocations "blade=c2001 and starttime > -1w and deleted"

要使用 undelete subcommand，--archives flag 是必需的，因为只有 deleted jobs 可以 undeleted。

% tq -a undelete jid=1234

Built-in Help（内置帮助）

tq 有详细的 built-in help 和有用 examples，因此无需离开 command line 就能获得更多 information。

% tq help

会显示 help topics 帮助主题的 top-level view 顶层视图。

% tq help subcommands

会显示完整的可能 tq subcommands 列表。

要获取特定 command 的 help，请用 command name 运行 tq help。例如：

% tq help jobs

会显示 usage statement，以及 examples 和每个可用 option 的 detailed description。
