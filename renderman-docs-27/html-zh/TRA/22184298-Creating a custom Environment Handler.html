Creating a custom Environment Handler

要跟随学习，Project X Handler 的完整代码清单可在 这里找到。

让我们逐段查看代码，了解这个处理器会做什么。

  • 首先，看一下处理器初始化：

    from TrEnvHandler import TrEnvHandler
import logging

class projxhandler(TrEnvHandler):
    def __init__(self, name, envkeydict, envkeys):
        self.logger = logging.getLogger("tractor-blade")
        self.logger.info("initializing projxhandler: %s" % (name))
        TrEnvHandler.__init__(self, name, envkeydict, envkeys)

    初始化非常直接。首先导入 TrEnvHandler（基类）和 logging 模块。

    __init__ 例程设置日志记录器，然后调用超类初始化。

  • 环境处理器的主要处理发生在两个方法中。

    updateEnvironment() 方法：

    def updateEnvironment(self, cmd, env, envkeys):
    self.logger.debug("projxhandler.updateEnvironment: %s" % repr(envkeys))
    self.initLocalVars()
    if envkeys and type(envkeys) == type([]):
        for envkey in envkeys:
            key,val = envkey.split("=")
            self.logger.debug("projxhandler.envkey: %s" % envkey)
            if key == "SCENE":
                self.scene = envkey
                self.environmentdict["SCENE"] = self.scene
            elif key == "SHOT":
                self.shot = envkey
                self.environmentdict["SHOT"] = self.shot

    return TrEnvHandler.updateEnvironment(self, cmd, env, envkeys)

def initLocalVars(self):
    self.scene = None
    self.shot = None

    updateEnvironment 方法会针对每个处理器调用一次，参数是待处理命令中的 envkeys 列表，这些 envkeys 与该处理器处理的 envkeys 匹配。在我们的示例中，如上方字典所示，当 envkey 键包含 “ProjectX”、“SHOT=somevalue” 或 “SCENE=somevalue” 时，会调用这个处理器。

    假设命令的 envkey 定义通常大致如下：

    RemoteCommand{ ...... } -envkey { ProjectX SHOT=27234-A SCENE=5125 }

    updateEnvironment() 方法的主要职责，是定义放入处理器自身环境中的值。它最后通常会调用基类的 updateEnvironment() 方法，将处理器环境与 blade 提供的启动命令环境合并。

    这个处理器中的代码会遍历与启动命令关联的所有 envkeys。然后设置一些内部变量，供后续使用，并在处理器的本地环境字典中设置多个环境变量。

    在上面的 Project X 处理器环境字典中，你会找到以下内容：

    "environment": {
  "CURRENT_SHOW": "Project X, Venture of Mystery",
  "CURRENT_SHOT": "$SHOT",
  "CURRENT_SCENE": "$SCENE",
  "RMANTREE": "/opt/pixar/RenderManProServer-15.2",
  "PATH": "$RMANTRE:/bin;$PATH"
},

    CURRENT_SHOT 和 CURRENT_SCENE 环境变量基于处理字典时预期存在的变量。因此，在处理器代码中，你会看到 self.environmentdict["SCENE"] 和 self.environmentdict["SHOT"] 在处理器的 updateEnvironment() 方法中定义。调用基类 updateEnvironment() 方法时，该字典会合并到启动命令的环境中，从而正确定义 CURRENT_SHOT 和 CURRENT_SCENE。

    我们还看到一个环境内部扁平化的示例：RMANTREE 被定义为指向该项目的特定 RPS 版本，并且 $RMANTREE/bin 被加载到 PATH 环境变量的开头。

  • 调用 updateEnvironment() 方法后，blade 接着调用处理器的 remapCmdArgs() 方法。此方法的目的是允许处理器在命令启动之前重写原始命令。

    重写方法如下：

    def remapCmdArgs(self, cmdinfo, launchenv, thisHost):
    self.logger.debug("projxhandler.remapCmdArgs: %s" % self.name)
    argv = TrEnvHandler.remapCmdArgs(self, cmdinfo, launchenv, thisHost)
    self.logger.info("scene: %s, shot:%s" %
        (self.scene, self.shot))

    # indicate command was launched by traactor
    launchenv["TRACTOR"] = "1"

    if argv[0] == "render" and "RMANTREE" in launchenv:
        argv[0] = os.path.join(launchenv["RMANTREE"],"bin","prman"))
        argv.[1:1] = ["-statsfile", "%s-%s" % (self.scene, self.shot))

    # on windows for add the Visual Studio default libs and includes
    p = platform.platform()
    if p.find("Windows") != -1:
        if launchenv.has_key("INCLUDE"):
            launchenv["INCLUDE"] += ";" + launchenv["VCINCLUDE"]
        else:
            launchenv["INCLUDE"] = launchenv["VCINCLUDE"]

        if launchenv.has_key("LIB"):
            launchenv["LIB"] += ";" + launchenv["VCLIB"]
        else:
            launchenv["LIB"] = launchenv["VCLIB"]

    return argv

    处理器通常做的第一件事，是调用基类 remapCmdArgs() 方法。基类处理所有软件都会使用的通用重映射。它处理 Alfred 脚本语言允许的标准重映射，例如 “%h, %H”。它还会查找任何命令行中的 $val。随后预期从启动环境中提供这个 $val。例如 $render 可能会被重映射为 prman。

    调用基类后，处理器可以提供所需的任何自定义重映射。在我们的示例处理器中，它会执行以下操作：

    • 它将环境变量 TRACTOR 设置为 1，让下游脚本知道这是一个由 Tractor 启动的作业。

    • 它检查要启动的应用程序是否为 render，并检查 RMANTREE 是否在环境中。如果是，它会把启动命令改为（实际上） $RMANTREE/bin/prman，并通过添加基于当前场景和镜头值的输出统计文件来扩展命令。

    • 随后这个处理器会检查当前 blade 的平台。如果确定是 Windows 平台，它会把 Visual Studio INCLUDE 和 LIBS 值添加到启动环境中。

    根据为命令提供的 envkeys 数量，可能会有多个环境处理器处理该命令。 default 处理器总是最先被调用，然后其他处理器会级联执行。当最后一个处理器处理完环境和启动命令后，blade 才会启动修改后的命令。
