要跟随学习，Project X Handler 的完整 code listing 可在 这里找到。

让我们逐段查看代码，了解这个 handler 会做什么。

  • 首先，看一下 handler initialization：

    from TrEnvHandler import TrEnvHandler
import logging

class projxhandler(TrEnvHandler):
    def __init__(self, name, envkeydict, envkeys):
        self.logger = logging.getLogger("tractor-blade")
        self.logger.info("initializing projxhandler: %s" % (name))
        TrEnvHandler.__init__(self, name, envkeydict, envkeys)

    initialization 非常直接。首先导入 TrEnvHandler（base class），以及 logging module。

    __init__ routine 设置 logger，然后调用 superclass initialization。

  • environment handler 中的主要处理发生在两个 methods 中。

    环境更新方法说明（updateEnvironment method()）：

    def updateEnvironment(self, cmd, env, envkeys):
    self.logger.debug("projxhandler.updateEnvironment: %s" % repr(envkeys))
    self.initLocalVars()
    if envkeys and type(envkeys) == type([]):
        for envkey in envkeys:
            key,val = envkey.split("=")
            self.logger.debug("projxhandler.envkey: %s" % envkey)
            if key == "SCENE":
                self.scene = envkey
                self.environmentdict["SCENE"] = self.scene
            elif key == "SHOT":
                self.shot = envkey
                self.environmentdict["SHOT"] = self.shot

    return TrEnvHandler.updateEnvironment(self, cmd, env, envkeys)

def initLocalVars(self):
    self.scene = None
    self.shot = None

    updateEnvironment method 对每个 handler 调用一次，参数是 pending command 中与该 handler 处理的 envkeys 匹配的 envkeys 列表。在我们的示例中，如上方 dictionary 所示，当 envkey key 包含 “ProjectX”、“SHOT=somevalue” 或 “SCENE=”somevalue” 时，会调用这个 handler。

    假设 command envkey definition 通常大致如下：

    RemoteCommand{ ...... } -envkey { ProjectX SHOT=27234-A SCENE=5125 }

    updateEnvironment() method 的主要职责，是定义放入 handler 自身 environment 中的 values。它最后通常会调用 base class 的 updateEnvironment() method，将 handler environment 与 blade 提供的 launching command environment 合并。

    这个 handler 中的 code 会遍历与 launching command 关联的所有 envkeys。然后它设置一些 internal variables，供后续使用，并在 handler 的 local environment dictionary 中设置多个 environment variables。

    在上面的 Project X handler environment dictionary 中，你会找到以下内容：

    "environment": {
  "CURRENT_SHOW": "Project X, Venture of Mystery",
  "CURRENT_SHOT": "$SHOT",
  "CURRENT_SCENE": "$SCENE",
  "RMANTREE": "/opt/pixar/RenderManProServer-15.2",
  "PATH": "$RMANTRE:/bin;$PATH"
},

    CURRENT_SHOT 和 CURRENT_SCENE environment variables 基于处理 dictionary 时预期存在的 variables。因此，在 handler code 中，你会看到 self.environmentdict["SCENE"] 和 self.environmentdict["SHOT"] 在 handler 的 updateEnvironment() method 中定义。当调用 base class updateEnvironment() method 时，该 dictionary 会合并到 launching command 的 environment 中，从而正确定义 CURRENT_SHOT 和 CURRENT_SCENE。

    我们还看到一个 environment 内部 flattening 的示例：RMANTREE 被定义为指向该 show 的特定 RPS version，并且 $RMANTREE/bin 被加载到 PATH environment variable 的开头。

  • 调用 updateEnvironment() method 后，blade 接着调用 handler 的 remapCmdArgs() method。此 method 的目的是允许 handler 在 command 启动之前重写原始 command。

    rewrite method 重写方法如下：

    def remapCmdArgs(self, cmdinfo, launchenv, thisHost):
    self.logger.debug("projxhandler.remapCmdArgs: %s" % self.name)
    argv = TrEnvHandler.remapCmdArgs(self, cmdinfo, launchenv, thisHost)
    self.logger.info("scene: %s, shot:%s" %
        (self.scene, self.shot))

    # indicate command was launched by traactor
    launchenv["TRACTOR"] = "1"

    if argv[0] == "render" and "RMANTREE" in launchenv:
        argv[0] = os.path.join(launchenv["RMANTREE"],"bin","prman"))
        argv.[1:1] = ["-statsfile", "%s-%s" % (self.scene, self.shot))

    # on windows for add the Visual Studio default libs and includes
    p = platform.platform()
    if p.find("Windows") != -1:
        if launchenv.has_key("INCLUDE"):
            launchenv["INCLUDE"] += ";" + launchenv["VCINCLUDE"]
        else:
            launchenv["INCLUDE"] = launchenv["VCINCLUDE"]

        if launchenv.has_key("LIB"):
            launchenv["LIB"] += ";" + launchenv["VCLIB"]
        else:
            launchenv["LIB"] = launchenv["VCLIB"]

    return argv

    handlers 通常做的第一件事，是调用 base class remapCmdArgs() method。base class 处理所有软件都会使用的通用 remapping。它处理 alfred scripting language 允许的标准 remapping，例如 “%h, %H”。它还会查找任何 command line 中的 $val。随后预期从 launch environment 中提供这个 $val。例如 $render 可能会被 remapped 为 prman。

    调用 base class 后，handler 可以提供它需要的任何 custom remapping。在我们的示例 handler 中，它执行以下操作：

    • 它将 environment variable TRACTOR 设置为 1，让 downstream scripts 知道这是一个由 tractor launched job。

    • 它检查要启动的 application 是否为 render，并检查 RMANTREE 是否在 environment 中。如果是，它会把 launch command 改为（实际上） $RMANTREE/bin/prman，并通过添加基于 current scene 和 shot values 的 output stats file 来扩展 command。

    • 随后这个 handler 会检查 current blade 的 platform。如果确定是 windows platform，它会把 Visual Studio INCLUDE 和 LIBS values 添加到 launch environment 中。

    根据为 command 提供的 envkeys 数量，可能会有多个 environment handlers 处理该 command。 default handler 总是最先被调用，然后其他 handlers 会级联执行。当最后一个 handler 处理完 environment 和 launch command 后，blade 才会启动修改后的 command。
