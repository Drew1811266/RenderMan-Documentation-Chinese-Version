Initial Configuration

Source: https://rmanwiki-27.pixar.com/space/TRA/22184326/Initial+Configuration
Downloaded: 2026-06-15T03:49:12+00:00

Tractor 的灵活性和强大能力来自它可被 customized（自定义）的能力。虽然有几十个 tuning parameters，也有编写 custom plug-ins 的机会，但 Tractor 被设计为无需编辑 configuration files 即可投入运行。

如果需要 customizations，则首先 必须创建一个目录 用于存放 configuration files，并且该目录需要可被 engine 的 process owner 读取。因为 engine 使用 /var/spool/tractor 作为默认 data directory，所以 /var/spool/tractor/config 会是 configuration directory 的合理选择。

Configuration files 采用 JSON 编写。

 

Key Parameters（关键参数）

在设置新系统时，administrators 应检查几个关键参数，以确保默认设置符合预期。以下所有参数都可以在你站点自己的 tractor.config 副本中覆盖。 NOTE：对大多数 studios 而言，默认值通常是正确的。

EngineOwner 指定 engine process 的特定 run-time owner。System services 会以 root 启动 tractor-engine，以便获取 port 80 等 protected ports；但随后该 process 会把 privilege level 降低到 EngineOwner。建议 studios 为此创建一个专门的 tractor user。

ListenerPort 指定 engine 监听 blade、dashboard 和其他 client connections 的 port。engine 是一种 web server，默认监听常规 HTTP port 80；这对用户和 site administration 来说是最简单的配置。通常只有在 port 80 上已经运行另一个 web server，或者 engine 不是以 root 启动、无法获取 80 时，才需要 alternate port number。

EngineDiscovery 指定 blades 和其他 client programs 应如何发现 engine。默认情况下，clients 会尝试连接 hostname tractor-engine，或者连接由 TRACTOR_ENGINE environment variable 或每个 tool 的 --engine=NAME command line parameter 给出的 alternate hostname。当该 hostname 无法解析时，会尝试 EngineDiscovery scheme。

LicenseLocation 指定 license file 的 path，或 license server name。此设置 通常不需要，因为 Tractor 会读取 Tractor installation 父目录中的 thepixar.license 文件，而该文件由所有 Pixar applications 共享。

MaxConcurrentDispatch 指定 engine process 将消耗的 Tractor license seats 最大数量。当多个 test engines 从同一个 license server 运行时，custom setting 可能很有用。使用默认值 zero 会使 tractor-engine 在 start-up 时获取 所有 可用 seats。

TractorDataDirectory 指定一个 EngineOwner 可写的位置，用于存储 job 和 user data。默认位置是 /var/spool/tractor。出于测试、file system permissions 限制，或指向更大/更快 storage volumes 的需要，可以选择其他位置。通常最好选择 engine host 上的某个本地位置。请确保 data directory 先被创建，并可由 EngineOwner 读写；如果 configuration directory 要作为其子目录（例如 /var/spool/tractor/config），这一步无论如何都是必要的。

 

Example（示例）

下面展示的是一个示例 tractor.config 文件，它将存储在站点决定的 configuration directory 中。在此示例中，站点设置了名为 tractor 的 user，使用其 home directory 存储 data，并监听较高的 port。这里使用 in-house license server，并且 engine 只允许 10 个 concurrent dispatches。

{
  "LicenseLocation": "my-license-server@9010",
  "EngineOwner": "tractor",
  "ListenerPort": 8000,
  "MaxConcurrentDispatch": 10,
  "TractorDataDirectory": "/ssd/tractor/data"
}
