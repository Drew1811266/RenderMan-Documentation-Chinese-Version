Initial Configuration

Source: https://rmanwiki-27.pixar.com/space/TRA/22184326/Initial+Configuration
Downloaded: 2026-06-15T03:49:12+00:00

Tractor 的灵活性和强大能力源于其可自定义性。虽然存在几十个调优参数，也可以编写自定义插件，但 Tractor 在设计上无需编辑配置文件即可投入运行。

如果需要自定义配置，则首先 必须创建一个目录 用于存放配置文件，并且该目录需要能被 Engine（调度引擎）的进程所有者读取。由于 Engine 使用 /var/spool/tractor 作为默认数据目录，因此 /var/spool/tractor/config 是配置目录的合理选择。

配置文件采用 JSON 编写。

 

关键参数

设置新系统时，管理员应检查几个关键参数，以确保默认设置符合预期。以下所有参数都可以在你站点自己的 tractor.config 副本中覆盖。 注意：对大多数工作室而言，默认值通常是正确的。

EngineOwner 指定 Engine 进程的特定运行时所有者。系统服务会以 root 身份启动 tractor-engine，以便获取端口 80 这类受保护端口；随后该进程会把权限级别降低到 EngineOwner。建议工作室为此创建一个专门的 tractor 用户。

ListenerPort 指定 Engine 监听 Blade、Dashboard 和其他客户端连接所用的端口。Engine 是一种 Web 服务器，默认监听常规 HTTP 端口 80；这对用户和站点管理来说是最简单的配置。通常只有当端口 80 上已经运行另一个 Web 服务器，或者 Engine 不是以 root 身份启动、无法获取 80 端口时，才需要使用其他端口号。

EngineDiscovery 指定 Blade 和其他客户端程序应如何发现 Engine。默认情况下，客户端会尝试连接主机名 tractor-engine，或者连接由 TRACTOR_ENGINE 环境变量或每个工具的 --engine=NAME 命令行参数指定的备用主机名。当该主机名无法解析时，会尝试 EngineDiscovery 方案。

LicenseLocation 指定许可证文件路径，或许可证服务器名称。此设置 通常不需要，因为 Tractor 会读取 Tractor 安装目录父目录中的 thepixar.license 文件，而该文件由所有 Pixar 应用程序共享。

MaxConcurrentDispatch 指定 Engine 进程将消耗的 Tractor 许可证席位最大数量。当多个测试 Engine 从同一许可证服务器运行时，自定义设置可能很有用。使用默认值 0 会使 tractor-engine 在启动时获取 所有 可用席位。

TractorDataDirectory 指定一个可由 EngineOwner 写入的位置，用于存储作业和用户数据。默认位置是 /var/spool/tractor。出于测试、文件系统权限限制，或指向容量更大/速度更快的存储卷的需要，可以选择其他位置。通常最好选择 Engine 主机上的某个本地位置。请确保数据目录先被创建，并可由 EngineOwner 读写；如果配置目录要作为其子目录（例如 /var/spool/tractor/config），这一步无论如何都是必要的。

 

示例

下面展示的是一个示例 tractor.config 文件，它将存储在站点决定的配置目录中。在此示例中，站点设置了名为 tractor 的用户，使用其主目录存储数据，并监听较高的端口。这里使用内部许可证服务器，并且 Engine 只允许 10 个并发分派。

{
  "LicenseLocation": "my-license-server@9010",
  "EngineOwner": "tractor",
  "ListenerPort": 8000,
  "MaxConcurrentDispatch": 10,
  "TractorDataDirectory": "/ssd/tractor/data"
}
