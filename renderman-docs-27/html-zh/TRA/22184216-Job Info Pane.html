作业信息窗格

Source: https://rmanwiki-27.pixar.com/space/TRA/22184216/Job+Info+Pane
Downloaded: 2026-06-15T03:49:12+00:00

作业信息窗格显示当前在 Job List 窗格中选中的作业专属信息。

蓝色的作业属性可编辑，可用于在作业提交后影响其任务调度。要编辑已有值的字段，只需点击并修改。按 Return 结束编辑，并可按设置提示确认该编辑。要编辑空字段，只需点击空白区域并输入新值。字段被编辑后，Dashboard 会先验证字段，再发送给 Engine。

注意，可以选择多个作业，编辑会应用到所有选中的作业。如果所有选中作业在某个字段上具有相同值，则显示该值；否则会显示 <various>。
 

可编辑字段

  • 调度层级（Tier） - 此字段是一个字符串，表示作业应在哪个调度层级下被调度。无效的层级名称会导致作业在默认层级下调度。
  • 优先级（Priority） - 此字段是一个浮点数，通常为正数。负优先级表示该作业应暂停。
  • 项目（Projects） - 此字段是一个逗号分隔的字符串列表，表示与该作业关联的项目。项目用于把 Adaptive Farm Allocation 划分成不同份额。更多信息请参见 Adaptive Farm Allocations。
  • Crew 成员列表字段 - crews 字段是一个逗号分隔的工作组名称列表。数据可以用逗号或空格分隔。工作组用于定义允许处理所选作业的用户与机器组合。crew 值应匹配 crews.config 文件中定义的 crew。每个 crew 定义一个用户列表。blade.config 文件中的 profile 可以选择为某个 crew 定义一个 Access 条目。
  • 最大活动数（Max Active） - 这是一个整数，表示该作业可同时拥有的最大活动任务数。
  • 限制标签字段 - limit tags 字段是一个逗号分隔的字符串列表，表示要关联到该作业每个命令的标签。例如，输入 "maya, shake" 会使该作业的每个活动命令同时递增 "maya" 和 "shake" 的限制计数器。
  • 环境键（Env Key） - keys 字段也是一个列表。该列表包含 envkeys，用于指定添加到启动链的环境处理器。
  • 服务键表达式（Service Key Expr） - 此服务键表达式是一个字符串，用于描述该作业每个命令所需的资源。更多信息可见 Job Scripting Operators。
  • 元数据（Metadata） - metadata 是一个字符串。此值不影响作业处理，只提供附加信息，并显示在作业信息窗格中。
  • 注释（Comment） - comment 字段是一个字符串。此值不影响作业处理，只提供附加信息，并显示在作业信息窗格中。
  • 前置作业字段 - 这是一个逗号分隔的作业 ID 列表，表示所选作业被考虑调度之前必须完成的作业。
  • 前置时间字段 - 这是一个格式为 MM/DD HH:MM:SS 的时间戳字符串，表示该作业可被考虑调度的时间点 之后。
     

作业备注

作业属性右侧的 Notes 框会显示所选作业的用户评论和系统事件。如果用户有权限向作业添加备注，则备注区域下方会出现 "click to add note"。可以拖动备注框的左下角来调整备注区域大小。

系统事件备注可用于展示作业历史，例如优先级何时被更改，或作业何时被重启。

用户生成的备注可作为作业所有者与调度管理员之间的沟通渠道，用于说明作业的重要性或状态，或交流调试建议。
