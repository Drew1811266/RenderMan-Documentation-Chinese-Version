这是 RenderMan 21 及更早版本的旧版文档。

简介

RenderMan Studio 的默认配置会通过 Tractor 分发工作。开箱即用时，用户可以进入 Render Options 窗口并进行设置，使任何不是本地内部渲染的渲染都使用 Tractor，以确保作业正确完成。当然，这也假定你的站点同样使用 Tractor 的默认开箱即用配置。

如果你碰巧认为自己很特别，并选择更改默认配置，也不用担心。RenderMan Studio 通过偏好设置和其他辅助设置提供了你需要的全部钩子，用于在 RenderMan Studio 工作站与 Tractor 工作分发系统之间维持协调配合。

Engine 位置和 Tractor 用户

RenderMan_for_Maya.ini 提供了两个关键偏好设置，用于确保 RMS 与 Tractor 的连接：

SetPref TractorEngine {tractor-engine:80} SetPref TractorUser {}

这些偏好设置由生成作业脚本并提交作业的 MEL 脚本使用。与所有偏好设置一样，它们可以在基于 aRMS_SCRIPT_PATHS 的 ini 覆盖文件中设置。

默认情况下，Tractor 会提交到一台名为 tractor-engine、端口为 80 的机器。由于各种原因，这可能不是你的设置，但你可以在站点专用 ini 中覆盖此偏好设置，例如：

SetPref TractorEngine { myserver:8080 }

格式为 engine:port，也就是 Engine 和端口。

tractor-spool.py（RMS 发行版中的 tractor.py）的默认行为是以提交该作业的用户身份提交。你可能希望作业以特定用户身份提交。第二个偏好设置可将作业锁定到特定用户，例如：

SetPref TractorUser { mwazowski }

EnvKeys

RenderMan_for_Maya_Pro.ini 文件现在有一个额外偏好设置，用于指定传递给 Tractor 的默认环境键。

SetPref DefaultEnvKey "rms-[GetPlatform rms_version]-maya-[GetPlatform rfm_mayaversion]"

此 envkey 由 Tractor envhandler处理；它会解析该键，并在 tractor-blade 上正确设置 $RMSTREE 和 $MAYA_LOCATION`。不过需要特别注意，RenderMan Studio 的 envhandler 会基于 "default" 安装位置设置环境，而该位置在不同站点可能不同。

DefaultEnvKey 偏好设置会为 RMS 中的 Render Options（包括 Current Frame 和 Batch）设置默认值。在此对话框中，用户可以完全更改发送给 Tractor 的环境设置，允许用户把其他 envkey 附加到默认值。注意，生成的 envkey 随后会像批量渲染对话框中的其他值一样保存到 Maya 偏好设置，并传播到未来的 Maya 场景。点击 Render Options 窗口中 Environment Key 的 Default 按钮会从 .ini 文件恢复 DefaultEnvKey。

tractor-blade 配置

如果你更改了 RenderMan Studio 的默认安装位置，就需要编辑 Blade 配置来更改安装位置，以便 Tractor 正确处理 RMS 作业。尤其是，有三个 shared.*.envkeys 文件会从 tractor-engine 发送到 tractor-blade。这些 envkey 文件包含环境字典。envkey "rms-*-maya-*" 的环境字典需要修改为指向正确的安装位置。

  • 重要提示

与 .ini 文件一样，我们建议用户不要编辑出厂配置文件。相反，应将这些文件复制到站点专用的特殊配置目录，并在那里编辑。然后使用 --configdir=/path/to/config/files 选项启动 tractor-engine。

额外的 envhandler

还有几个其他 envhandler 可用，其中最常用的可能是 prman envkey。 prman 键的格式为：

"prman-XX.X"

XX.X 指批量渲染要启动的 prman 版本（例如 '19.0'）。当此键添加到批量渲染对话框后，会调用 rmanhandler。它会在磁盘上定位目标 prman 版本，在启动环境中设置 RMANTREE，并将 $RMANTREE/bin 添加到启动环境路径。

如果 RMS 或 RPS 安装在非标准位置，则可以改用 rmantree 和 rmstree 处理器。

"rmantree=/path/to/my/rmantree/installation"

"rmstree=/path/to/my/rmstree/installation"

这些处理器会取出 rmantree 和 rmstree 的路径，并把它们作为 $RMANTREE 和 $RMSTREE 添加到环境中。它们还会把 $RMANTREE/bin 和 $RMSTREE/bin 附加到已启动命令的路径。
