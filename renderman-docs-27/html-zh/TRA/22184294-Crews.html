工作组

Source: https://rmanwiki-27.pixar.com/space/TRA/22184294/Crews
Downloaded: 2026-06-15T03:49:12+00:00

crews.config 文件定义用户对 Tractor 的访问权限，指定谁可以登录以及他们对系统各方面的控制级别。crews.config 文件通常会完整复制到 站点覆盖目录 中，然后在那里自定义。

定义

工作组只是一个具名用户列表：

"the_crew_name": ["john", "paul", "george", "ringo"],

保留工作组名称

除了任何站点定义的工作组之外，Tractor 还使用四个保留工作组名称作为特殊用途：ValidLogins、BannedLogins、Wranglers、Administrators。

ValidLogins 工作组会枚举所有允许登录 Tractor 的用户。这个工作组 必须 存在。请参阅下面关于元名称 @syslogins 和 @externlogins 的说明。

BannedLogins 工作组列出被明确拒绝登录的用户名。它主要在 ValidLogins 包含 @syslogins 或 @externlogins 这两个元名称之一时，用于提供例外。

元工作组名称

给定工作组的用户名列表可以包含以下任一特殊名称。@syslogins 表示引擎主机上本地有效的所有用户登录名（即可由 getpwnam(3) 函数解析的名称），通常也包括可由站点名称服务器（如 LDAP 或 NIS）解析的名称。

"ValidLogins": ["@syslogins"],

名称 @externlogins 会让引擎跳过对潜在 Dashboard 用户名的内部验证，而是把所有登录名直接传递给站点的外部 trSiteLoginValidator.py 脚本（或下面 SitePasswordValidator 指定的任意验证应用）。外部验证器可以接受或拒绝给定的名称+密码对。

"ValidLogins": ["@externlogins"],

注意，用户名中的通配符字符（例如 *）在搜索用户访问权限时不会展开，也不会被特殊处理。不过，用户名列表也可以包含其他工作组名称，这些名称会递归展开，以包含那些工作组中的用户。形成“引用循环”的工作组名称引用会被截断，但可能造成意外的成员关系。

工作组成员列表中任何第一个字符为减号的用户名，都会从该工作组定义中移除；子工作组名称也是如此。

"ourRestrictedCrew": ["ourBigCrew", "-knownMiscreants"],

名称列表中引用的工作组可以用前导 $ 标记，以便明确区分它们是工作组名称而不是用户名。这种方法也让 tractor-engine 在引用的工作组未知时能够输出更好的诊断消息；否则该名称只会被当作另一个用户名。

"theLuncheon": ["terry", "vandana", "$theGuysFromShipping"],

某个特定用户名可以出现在多个工作组中。

"Crews": {
    "ValidLogins": ["@syslogins"],
    "BannedLogins": [],

    "Wranglers": [],
    "Administrators": ["root"]
},

作业编辑权限

默认情况下，每个具有有效登录名的用户都会获得 “standard” 权限，即运行和修改自己作业的基本能力，以及查看队列和 blade 状态的能力。列在 Wranglers 工作组中的用户还可以修改一般作业队列，包括其他用户的作业状态。列在 Administrators 工作组中的用户可以通过 URL 接口更改站点范围策略。权限级别是累积的：Wranglers 拥有 “standard” 访问权限以及额外的 wrangler 权限。Administrators 拥有 standard、wrangler 和 admin 访问权限。

可选子字典 JobEditAccessPolicies 可用于进一步限制或扩展对作业属性（如作业优先级）的编辑访问权限。可以为每个可编辑属性指定不同访问列表；否则会应用名为 default 的条目设置。这些编辑列表中的特殊元工作组名称 @owner 会授予每个作业所有者在自己作业中编辑给定属性的权限。

编辑权限被组织为具名 策略，其中名为 defaultPolicy 的策略会默认应用。作业脚本可以使用 Job -editpolicy {NAME} 选项指定适用于它们的策略。工作室的作业创建管线通常会用它对来自特定项目或管理类作业的作业应用额外限制。如果作业明确请求的策略在此处缺失，那么引擎会应用 defaultPolicy 条目中的设置（如果存在），而不是在缺失策略定义之前拒绝所有编辑。Administrators 工作组中的用户始终对任何作业拥有编辑控制权。

可识别的细粒度编辑权限关键字包括：argv、afterjids、aftertime、bnotes、jnotes、comment、crews、cwd、delete、envkey、interrupt、jrestart、jshuffle、lock、metadata、pause、priority、retract、retry、runsecsbounds、service、skip、slotbounds、tags、title

"JobEditAccessPolicies": {
Note:  don't forget commas after each entry!

    "defaultPolicy": {
    "priority": ["Wranglers", "Administrators"],
    "tier":     ["Wranglers", "Administrators"],
        "default":  ["@owner", "Wranglers", "Administrators"]
    },

    "SomeCustomPolicyName": {
        "priority": ["Wranglers", "Administrators"],
        "tier":     ["Wranglers", "Administrators"],
        "default":  ["jedi", "Administrators"]
    }
},

SitePasswordValidator

SitePasswordValidator 指定用于验证从用户界面和脚本发送到引擎的密码的验证方案。使用空字符串 "" 可禁用密码检查，允许任何有效用户名作为客户端继续访问（即 ValidLogins 中授权的名称）。

SitePasswordValidator 的任何非空值都会要求密码，并指定将应用的验证类型。所有用户+密码验证都发生在 tractor-engine 主机上，因此这里的值都相对于引擎的环境、文件系统、操作系统等。

Tractor 引擎内置支持 Linux 和 Mac OS 提供的 “PAM” 认证系统。只有引擎需要这种支持，客户端可以位于任何平台。使用下面的字符串值 internal:PAM 可通过引擎主机上的 PAM 模块启用密码验证。

PAM 方案本身要求你选择适当的访问策略模块（见 /etc/pam.d 或 /etc/pam.conf）。Tractor 默认会尝试加载名为 “tractor” 的模块，你必须先用适合站点的策略定义该模块。或者，你也可以在这里指定不同的 PAM 模块名，只要它已经存在并支持你站点的需求。例如，要使用 su 认证规则，请使用下面的 internal:PAM:su。

Tractor-engine 会在你用密码通过认证后，向 Dashboard Web 浏览器发送会话 cookie。保存的会话信息后续可用于在刷新 Dashboard 页面时自动登录。可以通过更改 SitePasswordValidator 设置中的前缀来禁用这种自动登录行为，强制用户在 Dashboard 页面重新加载时重新输入密码；将 “internal:” 改为 “internal_nocookie:”，或为外部验证脚本添加前缀 “external_nocookie:”。

注意： PAM 方法要求 tractor-engine 在引擎主机本地向 PAM 接口交付明文密码。因此，它必须从 Dashboard 和其他客户端脚本接收可恢复的密码编码。内置传输编码能对窥探提供适度保护，但主要的密码传输保护预期来自你站点私有 LAN 以及接入该 LAN 的 VPN 连接的安全性。

Tractor 还支持一种更复杂的自定义替代方案：你提供自己的客户端单向密码哈希插件（根据需要使用 JavaScript 或 Python），并提供一个匹配的引擎端哈希验证器，在这里指定它的名称。更多细节请参阅模板示例中的注释和 Tractor 管理文档。

"SitePasswordValidator": ""
"SitePasswordValidator": "internal:PAM:tractor"
"SitePasswordValidator": "internal:PAM:su"
"SitePasswordValidator": "internal_nocookie:PAM:su"
"SitePasswordValidator": "python ${TractorConfigDirectory}/trSiteLoginValidator.py",
"SitePasswordValidator": "python ${TractorConfigDirectory}/trSiteLdapLoginValidator.py"
