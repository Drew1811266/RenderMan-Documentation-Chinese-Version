Job Author Python API

Source: https://rmanwiki-27.pixar.com/space/TRA/22184314/Job+Author+Python+API
Downloaded: 2026-06-15T03:49:12+00:00

tractor.api.author 是用于构建 Tractor jobs 的 Python API。

此 module 随 Tractor 附带的 Python interpreter rmanpy 一起安装。它也可以与其他 Python interpreters 一起使用，但需要 site-specific configuration 来定位或安装所需的 Tractor Python modules。

以下 examples 假定 query module 已按如下方式 import：

>>> import tractor.api.author as author

Overview（概览）

job 使用 Job、Task、Instance 和 Command classes 构建。随后可以使用 Job.spool() method 将其 spooled，或使用 Job.asTcl() method 显示。下面是一个 simple example：

>>> job = author.Job(title="a one-task render job", priority=100, service="PixarRender")
>>> job.newTask(title="A one-command render task", argv=["/usr/bin/prman", "file.rib"], service="pixarRender")
>>> print job.asTcl()
Job -title {a one-task render job} -priority {100.0} -service {PixarRender} -subtasks {
  Task {A one-command render task} -cmds {
    RemoteCmd {{/usr/bin/prman} {file.rib}} -service {pixarRender}
  }
}

提交作业流程（Spooling a Job）

要 spool job，请调用该 job 的 spool() method。new job 的 job id 将被返回。

>>> newJid = job.spool()

如果与 engine 通信出现问题，会 raise exception。如果 job 成功 spools，它会出现在 Dashboard 或 tq jobs 中。否则，engine 会记录 diagnostic information，说明该 job 为什么无法 spooled。

Spooling 可以选择以 blocking mode 执行：spool(block=True) 在 job submission 完全处理完之前不会返回。如果发生 error，会 raise exception。block=True option 有助于向 client 提供更多 diagnostic information。不过它需要更长时间才能完成，当还有许多其他 jobs 同时 being spooled 时尤其如此。因此，blocking 最适合用于 debug authoring scripts 或开发新 scripts。

owner keyword argument 可用于指定 job owner。默认 job owner 是 script 的 process owner。

>>> job.spool(owner="cher")

Setting Attributes（设置属性）

Attributes 可以使用 keyword arguments（如上所示）设置，也可以通过 attribute assignment 设置。上面的 job 也可以按如下方式指定：

>>> job = author.Job()
>>> job.title = "a one-task render job"
>>> job.priority = 100
>>> job.service = "PixarRender"
>>> job.newTask(title="A one-command render task", argv=["/usr/bin/prman", "file.rib"])

添加任务流程（Adding Tasks）

向 job 添加 task 可以像上面那样使用 Job.newTask() 完成，也可以分成独立步骤：先创建 task，再将其作为 job 的 child 添加。

>>> job = author.Job(title="a one-task render job", priority=100, service="PixarRender")
>>> task = author.Task(title="A one-command render task", argv=["/usr/bin/prman", "file.rib"])
>>> job.addChild(task)

如果 tasks 需要 serially 运行，使一个 task 完成后另一个 task 才开始，则先运行的 task 会声明为第二个 task 的 child。

>>> parent = author.Task(title="parent runs second", argv=["/usr/bin/command"])
>>> child = author.Task(title="child runs first", argv=["/usr/bin/command"])
>>> parent.addChild(child)

默认情况下，sibling tasks 允许 parallel 运行。

>>> parent = author.Task(title="comp", argv=["/usr/bin/comp", "fg.tif", "bg.tif"])
>>> child1 = author.Task(title="render fg", argv=["/usr/bin/prman", "fg.rib"])
>>> child2 = author.Task(title="render bg", argv=["/usr/bin/prman", "bg.rib"])
>>> parent.addChild(child1)
>>> parent.addChild(child2)

Serialsubtasks（串行子任务）

将 task 的 serialsubttasks attribute 设置为 1，会使其 children serially 运行。

>>> parent.serialsubtasks = 1

当 parent task 设置了 serialsubtasks 时，某个 child task 的整个 subtree 必须完全完成后，它的 sibling task 才允许运行。
 

添加命令流程（Adding Commands）

Tasks 通常包含一个或多个 commands。向 task 添加 command 可以使用 Task.newCommand()，也可以分成独立步骤：创建 new command，然后将其作为 task 的 child 添加。下面的 examples 会构建等价的 task。

>>> task = author.Task(title="render rib 1")
>>> command = author.Command(argv=["/usr/bin/prman", "1.rib"], service="pixarRender")
>>> task.addCommand(command)

>>> task = author.Task(title="render rib 1")
>>> command = task.newCommand(argv=["/usr/bin/prman", "1.rib"], service="pixarRender")

此外，还有一种 shortcut：创建 task 时，可以通过设置 argv attribute 实例化 command 并自动将其与 task 关联。此方法也适用于 author.newTask() method。

>>> task = author.Task(title="render rib 1", argv=["/usr/bin/prman", "1.rib"], service="pixarRender")

>>> task = otherTask.newTask(title="render rib 1", argv=["/usr/bin/prman", "1.rib"], service="pixarRender")

因为每个 command 都需要 service key，所以使用此 shortcut 时，API 会把 service attribute assignment 推送到 command 上。

如果需要设置 command 的其他 attributes，例如 envkey，则必须使用 author.Command() 或 author.Task.newCommand() 显式构建 commands，而不是使用此 shortcut。

task 可以没有 commands。例如，task 可以用来 group 一个 tasks subtree，即使该 task 本身没有 commands 要执行。
 

Multi-command Tasks（多命令任务）

当 task 有多个 commands 时，它们会 serially 执行。可以连续调用 author.Task.addCommand() 或 author.Task.newCommand() methods，将多个 commands 与一个 task 关联。

>>> task = author.Task(title="multi-command task", service="PixarRender")
>>> copyin = author.Command(argv=["scp", "remote:/path/file.rib", "/local/file.rib"])
>>> task.addCommand(copyin)
>>> render = author.Command(argv=["/usr/bin/prman", "/local/file.rib"])
>>> task.addCommand(render)
>>> copyout = author.Command(argv=["scp", "/local/file.tif", "remote:/path/file.tif"])
>>> task.addCommand(copyout)

>>> task = author.Task(title="multi-command task", service="PixarRender")
>>> task.newCommand(argv=["scp", "remote:/path/file.rib", "/local/file.rib"])
>>> task.newCommand(argv=["/usr/bin/prman", "/local/file.rib"])
>>> task.newCommand(argv=["scp", "/local/file.tif", "remote:/path/file.tif"])

异常处理说明（Exceptions）

向 element 的 attribute 赋值时，如果该值无效，会 raise exception。

>>> try:
>>>     job.atmost = "three"
>>> except author.AuthorError, err:
>>>     print "you can expect to see this message"

目录映射说明（Directory Mapping）

directory map 是在不同 operating systems 之间转换 file system paths 的一种方式。Directory maps 按如下方式指定：

>>> job.newDirMap(src="X:/", dst="//fileserver/projects", zone="UNC")
>>> job.newDirMap(src="X:/", dst="/fileserver/projects", zone="NFS")

关于 directory mapping 和此 example 的更多信息可在 这里找到。
 

Engine 连接设置说明（Engine Connectivity）

spool job 时，author module 会处理与 engine 的 connectivity。可以使用 setEngineClientParam() 更改 engine hostname、port、connected user、password 和 debug flag。

默认 target engine 根据 TRACTOR_ENGINE environment variable 设置；否则使用 tractor-engine:80。根据是否已配置 EngineDiscovery，也可能使用不同的 default engine。

默认 user 根据 USER environment variable 设置；否则为 root。

默认 password 根据 TRACTOR_PASSWORD environment variable 设置；否则不设置。

默认 debug flag 根据 TRACTOR_DEBUG environment variable 设置；否则为 False。

下面的 example 演示如何 override 某些 connection values。请注意，应在 spool 任何 jobs 之前调用 setEngineClientParam()，connection parameters 才会生效。

>>> author.setEngineClientParam(hostname="test-engine", port=8000, user="donovan", debug=True)

可以通过调用 closeEngineClient() 终止 session。这有助于减少 engine 维护的 sessions 数量。

>>> author.closeEngineClient()
