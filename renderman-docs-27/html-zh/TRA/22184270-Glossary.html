实体

Tractor 有 5 个关键实体，它们会在数据库中跟踪，并可在 Dashboard、tq 命令行工具和 Python API 中查看。

Job（作业）

A job 是提交给系统进行处理的逻辑工作体。它描述需要完成什么以及以什么顺序完成。它可以构建得任意大且复杂，从渲染单帧单层，到在一系列帧上渲染多层。它可以包含渲染、合成、管理以及任何能在主机上运行的其他 process。

Task（任务）

A task 是部分完成 job 的工作单元。task 可以与其他 task 并行运行，也可以串行运行。Tasks 及其操作顺序构成 job 的结构。

Command（命令）

A command 是在 blade上执行的单个程序。它可以是 render、notification script、administration command，或任何其他能在该主机上执行的程序。

Invocation（调用记录实体）

一个 invocation 是一组描述 command 在 blade 上执行情况的 metric 指标。当一个 command 多次运行时，它会有多个 invocation，用来显示它何时运行、在哪里运行，以及使用了多少内存和 CPU 资源。

Blade（执行节点）

A blade 是远程执行服务器，也就是一个运行 commands 并将 command 结果报告给 engine 的程序。

覆盖属性

Job 和 task 可以指定与 command 相关的 attribute value。这些 attribute 包括 service, tags, maxslots，以及 envkey。在 command 级别定义的 value 会覆盖 task 和 job 级别指定的 value；在 task 级别定义的 value 会覆盖 job 级别指定的 value。
