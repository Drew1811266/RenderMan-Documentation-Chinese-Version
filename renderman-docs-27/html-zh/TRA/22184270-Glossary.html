实体

Tractor 有 5 个关键实体，它们会在数据库中跟踪，并可在 Dashboard、tq 命令行工具和 Python API 中查看。

Job（作业）

作业是提交给系统处理的一组逻辑工作。它描述需要完成什么，以及按什么顺序完成。作业可以构建得任意大且复杂，从渲染单帧中的单个层，到在一系列帧上渲染多层都可以。它可以包含渲染、合成、管理任务，以及任何能在主机上运行的其他进程。

Task（任务）

任务是完成作业一部分工作的单元。任务可以与其他任务并行运行，也可以串行运行。任务及其执行顺序共同构成作业的结构。

Command（命令）

命令是在 Blade 上执行的单个程序。它可以是渲染命令、通知脚本、管理命令，或任何其他能在该主机上执行的程序。

Invocation（调用记录实体）

调用记录是一组指标，用于描述命令在 Blade 上的执行情况。当一个命令多次运行时，会生成多个调用记录，用来显示它何时运行、在哪里运行，以及使用了多少内存和 CPU 资源。

Blade（执行节点）

Blade 是远程执行服务器，即运行 命令 并将命令结果报告给 Engine 的程序。
 

属性
 

作业属性

nametypedescription
jid整数作业标识符，即作业的唯一数字 ID。
owner字符串提交该作业的用户。
spoolhost字符串提交该作业的主机。
spoolfile字符串已提交作业文件的路径。
spoolcwd字符串提交该作业的命令所在工作目录。
spooladdrip adress（IP 地址类型字段）提交该作业的主机 IP 地址。
title字符串作业的标题。
assignments编码字符串这些是全局作业变量赋值，定义在作业文件的 Assign 段中。
dirmapJSON 结构根据架构转换路径的映射。
tier字符串层级名称。层级是队列的有序分区。
priority浮点数优先级决定作业在队列中的位置。
crews字符串列表该作业的 crew 列表。这会限制该作业只能在配置为接受这些 crew 的 Blade 上运行。
projects字符串列表designation 标记列表，会影响共享限制中活动任务的计数方式。
tags字符串列表约束该作业所有命令的限制名称。限制会控制带有该限制名称的命令可并发执行的数量。
service字符串约束该作业所有命令的服务键表达式。服务键表达式是布尔表达式，用于把命令与具备对应运行能力或 designation 的 Blade 匹配。
envkey字符串列表环境键名称列表，用来建立该作业所有命令将运行于其中的环境。
editpolicy字符串影响哪些用户可以操作作业的策略名称。
minslots整数运行该作业中一个命令所需的最小槽位数。
maxslots整数运行该作业中一个命令所需的最大槽位数。
etalevel整数用于估算剩余完成时间的作业图层级（未使用）。
afterjids字符串必须在此作业启动前完成的作业 ID 列表。
maxactive整数该作业允许的最大并发活动命令数。
serialsubtasks布尔值布尔值，表示子任务是否要串行执行。
spooltime时间戳该作业被提交的时间。
pausetime时间戳作业被暂停的时间。
aftertime时间戳此时间之后，该作业才会被纳入调度考虑。
starttime时间戳作业首次拥有活动任务的时间。
stoptime时间戳该作业最近一次有任务完成的时间。
deletetime时间戳作业被删除的时间。
elapsedsecs整数该作业处于活动状态的总耗时秒数。
esttotalsecs整数该作业完成所需的预计任务总耗时秒数。
numtasks整数该作业中的任务数量。
numblocked整数该作业中 blocked 状态任务的数量。blocked 状态任务需要其他任务完成后，才能变为 ready，并随后变为 active。
numready整数此作业中 ready 状态任务的数量。ready 状态任务是一种等待中的任务，不需要任何其他任务完成即可变为 active。
numactive整数作业中 active 状态任务的数量总计。
numerror整数作业中 error 状态任务的数量总计。
numdone整数此作业中 done 状态任务的数量。
maxtid整数该作业所有任务中最高的任务 ID，包括 detached 任务。Engine 使用它管理从 expand 任务新创建的任务 ID 分配。detached 任务是指由 expand 任务生成、但由于作业后来被重启而不再纳入调度考虑的任务。
maxcid整数该作业所有命令中最高的命令 ID，包括 detached 任务中的命令。Engine 使用它管理从 expand 任务新创建的命令 ID 分配。
comment字符串用户定义的作业注释字符串。
metadata字符串作业的用户定义元数据。
pil整数队列位置：作业相对于相同优先级的其他作业的排序；初始设为 jid。
lastnoteid整数作业最近一条备注的 ID。

 

任务属性

nametypedescription
jid整数该任务所属作业的唯一标识符。
tid整数任务标识符，即作业内任务的唯一标识符。
title字符串任务的标题。
id字符串作业内该任务的唯一字符串 ID。
service字符串约束该任务所有命令的服务键表达式。服务键表达式是布尔表达式，用于把命令与具备对应运行能力或 designation 的 Blade 匹配。
minslots整数运行该任务中一个命令所需的最小槽位数。
maxslots整数运行该任务中一个命令所需的最大槽位数。
cids整数列表该任务的命令 ID 列表。
serialsubtasks布尔值布尔值，表示子任务是否要串行执行。
ptids整数列表父任务的任务 ID 列表。只有带有关联实例的任务才会有多个父任务。所有子任务必须成功完成，其父任务才能变为 active。
attached布尔值布尔值；如果为 false，表示该任务是某个已重试的 expand 任务的结果。
state字符串任务状态取值，可为 blocked、ready、active、error 或 done。
statetime时间戳任务变为当前状态的时间。
readytime时间戳任务变为 ready 的时间。
activetime时间戳任务变为 active 的时间。
currcid整数下一个将运行或当前正在运行的命令 ID。
haslog布尔值布尔值，表示任务是否有日志输出。
preview字符串列表预览命令的命令参数（argv）列表。
chaser字符串列表追踪命令的命令参数（argv）列表。
progress浮点数任务进度，以 0 到 100 之间的百分比值表示。当任务发出 ALF_PROGRESS 消息时会设置此值。
metadata字符串任务的用户定义元数据。
resumeblock布尔值布尔值，表示该任务是否会自动恢复任何可恢复的祖先任务。
retrycount整数如果作业一直运行到完成，该任务将经历的 pass 数计数器。如果任务曾在某个时间点变为 active，任务重试或作业重启会递增该值。

 

命令属性

nametypedescription
jid整数该命令所属作业的唯一标识符。
tid整数该命令所属任务的唯一标识符。
cid整数命令标识符，即作业内命令的唯一标识符。
argv字符串列表表示命令的命令参数列表。

runtype字符串命令的类型。可能的值包括 "normal" 和 "cleanup"。

service字符串约束该任务所有命令的服务键表达式。服务键表达式是布尔表达式，用于把命令与具备对应运行能力或 designation 的 Blade 匹配。
tags字符串列表约束该作业所有命令的限制名称。限制会控制带有该限制名称的命令可并发执行的数量。
id字符串作业内该任务的唯一字符串 ID。
refersto字符串另一个任务或命令的 ID。设置此值会让该命令在被引用任务或命令曾运行所在的同一 Blade 上运行。
minslots整数运行该命令所需的最小槽位数。
maxslots整数运行该命令所需的最大槽位数。
envkey字符串列表环境键名称列表，用来建立该命令将运行于其中的环境。
retryrcodes整数列表会触发该命令自动重试逻辑的返回码列表。
metadata字符串命令的用户定义元数据。
resumewhile字符串列表由 Blade 执行的命令参数列表，或特殊关键字列表；任一者都会决定该命令是否可恢复。
resumepin布尔值布尔值，表示命令恢复时是否应在同一主机上运行。
minrunsecs浮点数命令运行后不被视为 error 所需的最少秒数。
maxrunsecs浮点数命令在被终止之前允许运行的最大秒数。

tid整数该调用记录所属任务的唯一标识符。
cid整数该调用记录所属命令的唯一标识符。

starttime时间戳调用记录的开始时间。

pid整数调用记录的进程 ID。
rss浮点数进程 resident set size，单位为 GB。
mem浮点数进程的 memory usage，单位为 GB。
cpu浮点数进程的当前 CPU utilization。
elapsedapp浮点数进程已用用户时间，单位为秒。
elapsedsys浮点数进程已用系统时间，单位为秒。
elapsedreal浮点数进程已用挂钟时间，单位为秒。

resumecount整数一个按恢复 pass 对迭代排序的数字。此源句存在明显残缺；按上下文，它描述调用记录运行时与任务重试/恢复计数相关的值。
resumable布尔值如果命令可以恢复，则此布尔值为 true。
bladeidid调用记录对应命令正在运行或曾运行所在 Blade 的 bladeid。

覆盖属性

作业和任务可以指定与命令相关的属性值。这些属性包括 service、tags、maxslots 以及 envkey。在命令级别定义的值会覆盖任务和作业级别指定的值；在任务级别定义的值会覆盖作业级别指定的值。
