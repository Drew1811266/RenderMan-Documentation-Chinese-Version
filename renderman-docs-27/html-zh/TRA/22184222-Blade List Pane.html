Blade List 窗格

Source: https://rmanwiki-27.pixar.com/space/TRA/22184222/Blade+List+Pane
Downloaded: 2026-06-15T03:49:12+00:00

Blade List 窗格提供用于监控服务器及其活动的主要界面。与从 Engine 接收实时更新的 Job List 窗格不同，Blade 列表会以批量方式刷新，刷新可以手动触发，也可以由 Dashboard 管理的计时器触发。在工具栏中选择 Blade 列表图标即可触发手动刷新。

默认情况下，Blade 列表每 60 秒自动刷新一次。此值可在偏好设置窗格中更改。在大型站点，此查询返回的数据量可能很大，因此应谨慎选择重新加载间隔，以避免网络瓶颈。
 

特殊功能

Blade List 窗格会响应多种鼠标操作：

  • 选择（Select）

    点击任意行，会在 Blade List 窗格下方的 Blade Info 窗格中显示所选 Blade 的信息。
     

  • 多选操作

    Blade List 窗格也响应标准的多重点击控制。这通常与上下文相关菜单选择配合使用。
     

  • 上下文相关菜单

    右键点击（或 Control-click）会打开上下文相关菜单，用于对所选 Blade 执行操作。

  • Clear Earlier Blade Data（清除较早的 Blade 数据）

    通知 Engine 从内存中移除所选 Blade。这是清理曾经启动但已不再活动的 Blade 的好方法。如果清除的是活动 Blade，它在向 Engine 报到时会重新出现。
     

  • NIMBY 限制（On | Off）

    向所选 Blade 发送请求，设置其 NIMBY（Not-In-My-Backyard）模式。当 NIMBY 设置为 "on" 时，Blade 只接受从该主机（通常是用户桌面）提交出来的作业。NIMBY 为 "off" 时，任何匹配常规配置条件的作业都可被接受。注意，也可以通过发送到 Blade 的 HTTP 指令，把该模式限制到特定用户。
     

  • Assignment Tracer

    Assignment Tracer 提供一种机制，用于在 Blade 没有接收新任务时辅助调试。

    选择一个 Blade 并从上下文菜单中选择 "Assignment Tracer" 后，会打开追踪器对话框。在所选 Blade 向 Engine 请求作业之前，该对话框会保持空白。随后 Engine 会记录它如何遍历现有作业，以判断是否向该 Blade 分配工作。

键盘快捷键

Blade List 窗格中可使用以下键盘快捷键：

  • Control/Command-A 全选快捷键

    选择所有 Blade。
     

  • F5 刷新键

    刷新 Blade 列表。
