Blade List Pane

Source: https://rmanwiki-27.pixar.com/space/TRA/22184222/Blade+List+Pane
Downloaded: 2026-06-15T03:49:12+00:00

Blade List pane 提供用于监控 server 及其 activity 的主要界面。与从 Engine 接收实时 update 的 Job List pane 不同，blade list 会以批量方式刷新，刷新可以手动触发，也可以由 Dashboard 管理的 timer 触发。在 toolbar 中选择 blade list 图标即可触发手动刷新。

默认情况下，blade list 每 60 秒自动刷新一次。此值可在 preferences pane 中更改。在大型站点，此查询返回的数据量可能很大，因此应谨慎选择 reload interval，以避免网络瓶颈。
 

特殊功能

Blade List pane 会响应多种 mouse 操作：

  • 选择（Select）

    点击任意 row，会在 Blade List pane 下方的 Blade Info pane 中显示所选 blade 的信息。
     

  • 多选操作

    Blade List pane 也响应标准的 multi-click control。这通常与 context sensitive menu 选择配合使用。
     

  • 上下文菜单（Context-sensitive menu）

    右键点击（或 Control-click）会打开 context sensitive menu，用于对所选 blade 执行操作。

  • 清除较早的 Blade 数据（Clear Earlier Blade Data）

    通知 engine 从内存中移除所选 blade。这是清理曾经启动但已不再活动的 blade 的好方法。如果清除的是 active blade，它在向 engine check in 时会重新出现。
     

  • NIMBY 限制（On | Off）

    向所选 blade 发送请求，设置其 NIMBY（Not-In-My-Backyard）mode。当 NIMBY 设置为 "on" 时，blade 只接受从该 host（通常是用户桌面）spool 出来的 job。NIMBY 为 "off" 时，任何匹配常规 profile criteria 的 job 都可被接受。注意，也可以通过发送到 blade 的 http directive，把该 mode 限制到特定用户。
     

  • Assignment Tracer

    assignment tracer 提供一种机制，用于在 blade 没有接收新 task 时辅助 debug。

    选择一个 blade 并从 context menu 中选择 "Assignment Tracer" 后，会打开 tracer dialog。在所选 blade 向 engine 请求 job 之前，该 dialog 会保持空白。随后 engine 会记录它如何遍历现有 job，以判断是否向该 blade 分配 work。

键盘快捷键（Keyboard Shortcuts）

Blade List pane 中可使用以下 keyboard shortcut：

  • Control/Command-A 全选快捷键

    选择所有 blade。
     

  • F5 刷新键

    刷新 blade 列表。
