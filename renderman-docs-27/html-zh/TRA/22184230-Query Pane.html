Query Pane

Query pane 是用于搜索和显示 Tractor entity 的通用工具。
 

布局

header 的布局由 drop-down entity menu、refresh button、search clause text entry 和 bookmark drop-down menu 组成。pane 的其余部分用于显示匹配的 entity。
 

用法（Usage）

使用搜索页面通常包含三个步骤。

首先，如果所需 entity 尚未选中，则从 drop-down menu 中选择它。可选 entity 包括 job、task、command、invocation 和 blade。 在搜索中包含归档项 checkbutton 可以按需选中，以便同时显示匹配的已删除 job。

其次，从同一个 drop-down menu 中选择所需 attribute。鼠标 hover 到 attribute 上时会显示其描述。对于某些 entity，可以选择关联 entity 的 attribute。例如，搜索 task 时可以选择 job owner。可以通过 shift-select 来切换某个 entity 的所有 attribute。如果没有为要查询的 entity 选择任何 attribute，则会检索全部 attribute。

最后，输入 search clause。search clause 必须遵循 Tractor search clause syntax。如果没有输入 search clause，则返回所有 entity。

可以通过按 refresh button，或在 search clause text entry 中按 Enter 来执行搜索。

query 执行后，entity 和 attribute 会显示在 pane body 的 row 和 column 中。选择某个 attribute 的 column header，可按该 attribute 对 entity 排序。拖动 header 可以重新排列 column，拖动 header 之间的 divider 可以调整 column 大小。
 

书签

可以选择 bookmark icon 来显示 drop-down menu，用于创建新 bookmark 或执行已有 bookmark。bookmark 会保存 search string、entity、attribute、column order 和 column width。bookmark 有名称，并会出现在以相关 entity 命名的 section 下。

要添加 bookmark，请按带加号的 bookmark icon。随后会出现提示，要求输入 bookmark title，该 title 会用于在 menu 中表示此 bookmark。

要移除 bookmark，请按带减号的 bookmark icon。该 icon 会变为红色，表示选择某个 bookmark 会将其移除。
 

到 Job 和 Task 的超链接

双击某个 job 会在 job pane 中显示该 job 及其 graph。Dashboard 必须能看到 jid 这个 displayed attribute，才能推断要显示哪个 job。

双击 task、command 或 invocation，也会在 job pane 中显示该 job，并同时选中相关 task。Dashboard 必须能看到 jid 和 tid 这两个 displayed attribute，才能推断要显示哪个 job 和 task。
 

截断的结果集（Result Sets）

注意，为避免 Dashboard 和 engine 出现性能问题，任何搜索返回的 entity 数量都有一个站点可配置的上限，因为在大型数据集中，不受限制的 query result 可能包含数百万 row。
