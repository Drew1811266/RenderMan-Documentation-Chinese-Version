tractor-dbctl

Downloaded: 2026-06-15T03:49:12+00:00

这个 command line tool 由 engine 使用，用于自动启动和停止后端 PostgreSQL relational database server。Site admins 也可以使用它执行各种 database maintenance tasks。

 

Usage: tractor-dbctl [options]（用法）

Options（选项）：

Option（选项）Description（说明字段内容）
--start-for-engine

启动 postgresql server，并按 db.config 中的配置自动 build/upgrade schema（engine 使用）

--stop-for-engine

按 db.config 中的配置停止 postgresql server（engine 使用）

--start启动 postgresql 数据库 server
--stop停止 postgresql server；如果 engine 无法自动完成停止，或 postgresql server 是手动启动的，可以用它关闭 postgresql；engine 正在运行时不要执行此命令。
--status检查 postgresql server 的状态
--init初始化 postgresql data directory；server 必须未运行
--build构建 tractor database；server 必须正在运行
--destroy移除 postgres database directory；server 必须未运行；这会移除 所有 当前和历史 job 与 blade information。
--upgrade更新 database schema 数据库结构
--check-upgrade检查是否需要 upgrades
--no-auto-upgrade阻止 database schema 数据库结构自动 upgrade 升级
--purge-jobs移除所有 jobs
--vacuum重建未删除 jobs 的 tables 以节省空间
--reset-job-counter重置 job id counter，使 job ids（jids）从 1 开始；只能与 --purge-jobs 一起使用
--show-params显示存储在 database 中的 configuration parameters
--update-params将 config files 中定义的某些 configuration parameters 推送到 database
--logs-usage报告 postgresql log files 使用的 disk space
--purge-logs移除 postgresql 日志 log files 文件
--backup=filename将 backup 写入指定 file；server 必须未运行。
--restore=filename从指定 backup file 恢复 database；server 必须未运行
--config-dir=pathtractor config directory 的配置 path 路径
--quiet, --q除 errors 外，抑制所有 output
--verbose, -v调整 verbosity 详细程度级别
--debug, -d打印 debug messages 调试消息
--help, -h显示此 message
