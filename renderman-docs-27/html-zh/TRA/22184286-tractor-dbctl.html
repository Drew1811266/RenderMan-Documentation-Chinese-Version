tractor-dbctl

Downloaded: 2026-06-15T03:49:12+00:00

这个命令行工具由 Engine 使用，用于自动启动和停止后端 PostgreSQL 关系型数据库服务器。站点管理员也可以使用它执行各种数据库维护任务。

 

用法：tractor-dbctl [options]

选项：

选项说明
--start-for-engine

启动 PostgreSQL 服务器，并按 db.config 中的配置自动构建/升级 schema（Engine 使用）

--stop-for-engine

按 db.config 中的配置停止 PostgreSQL 服务器（Engine 使用）

--start启动 PostgreSQL 数据库服务器
--stop停止 PostgreSQL 服务器；如果 Engine 无法自动完成停止，或 PostgreSQL 服务器是手动启动的，可以用它关闭 PostgreSQL；Engine 正在运行时不要执行此命令。
--status检查 PostgreSQL 服务器的状态
--init初始化 PostgreSQL 数据目录；服务器必须未运行
--build构建 Tractor 数据库；服务器必须正在运行
--destroy移除 PostgreSQL 数据库目录；服务器必须未运行；这会移除 所有 当前和历史作业与 Blade 信息。
--upgrade更新数据库 schema（结构）
--check-upgrade检查是否需要升级
--no-auto-upgrade阻止数据库 schema（结构）自动升级
--purge-jobs移除所有作业
--vacuum重建未删除作业的表以节省空间
--reset-job-counter重置作业 ID 计数器，使作业 ID（jids）从 1 开始；只能与 --purge-jobs 一起使用
--show-params显示存储在数据库中的配置参数
--update-params将配置文件中定义的某些配置参数推送到数据库
--logs-usage报告 PostgreSQL 日志文件占用的磁盘空间
--purge-logs移除 PostgreSQL 日志文件
--backup=filename将备份写入指定文件；服务器必须未运行。
--restore=filename从指定备份文件恢复数据库；服务器必须未运行
--config-dir=pathTractor 配置目录的路径
--quiet, --q除错误外，抑制所有输出
--verbose, -v调整详细程度级别
--debug, -d打印调试消息
--help, -h显示此消息
