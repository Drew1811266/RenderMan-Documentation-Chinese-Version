Checkpoint-Resume and Incremental Processing

Source: https://rmanwiki-27.pixar.com/space/TRA/22184310/Checkpoint-Resume+and+Incremental+Processing
Downloaded: 2026-06-15T03:49:12+00:00

作业脚本、调度和 Dashboard 的最新扩展增加了与增量计算相关的新能力。Tractor 还支持通用的“从最后一个检查点重试”方案。这两项功能都与新的 RenderMan 19 RIS 检查点和增量渲染功能集成。
 

从最后一个局部结果重试

例如，Pixar RenderMan 渲染器 prman 在使用兼容输出格式时，多年来一直支持内置重试方案。

A basic render command:
   prman frame123.rib

如果该进程在渲染一小时后由于某种原因被终止，但期间生成了局部图像文件，那么通常可以从该点重新开始渲染。

Restart from the last image checkpoint:
   prman -recover 1 frame123.rib

初始渲染等同于使用 -recover 0，因为零表示渲染应从开头开始，并忽略（覆盖）任何预先存在的输出文件。

一个调用该基础渲染命令的简单 Tractor 作业脚本，可以用 Tractor 的 "%r" 运行时替换参数来编写；Tractor 会在命令启动时填入该参数。

Job -title {simple recover example} -subtasks {
     Task {Frame 123} -cmds {
         RemoteCmd {prman -recover %r frame123.rib} -service PixarRender
     }
}

在命令第一次启动时，或用户请求完全重启任务时，Tractor 会在参数列表中把 %r 替换为数字零；当用户请求从最后一个检查点恢复时，则替换为数字一。

因此，Tractor 对此功能的支持假定许多应用程序或其包装脚本的重启行为，可以通过某个接受简单 '0' 或 '1' 的命令行参数来控制。
 

增量检查点恢复迭代

引入新的 Tractor 作业脚本控制项：

-resumeblock
-resumewhile
-resumepin

目标是将一组任务（整个依赖子树）标记为一个组；当某些元素已经到达检查点但尚未完成时，该组会重新运行。

-resumeblock 选项放在组根部的 Task 上。-resumewhile 子句放在遵循 checkpoint-resume 语义的 RemoteCmd 上。 -resumepin 1 选项是带有 -resumewhile 的命令上的额外限定符，表示重启必须发生在该命令初始轮次所用的同一 blade 上。

注意：在 -resumeblock Task 指定的子树中，并非所有任务都会在每个轮次中重启。只有 -resumewhile 子句求值为 "true"（非零）的那些命令，才会在下一轮重试。其中一些命令可能会比块中的其他命令提前几个轮次完全完成。一旦某个命令的 resumewhile 子句求值为 "false"（零），它就不会再由此循环机制重启。还请注意， 任何 依赖于被恢复任务的任务，一直到恢复块的根部，也会被重试。由 -resumewhile 引起的重试在这方面的行为与手动任务重试相同：依赖的后继任务也会被重试。

替换模式 %r、%R 和 %q 会在 tractor-blade 启动每个命令之前填入：当任务全新开始时 %r 为 0；当用户或系统请求从检查点恢复时，%r 为大于零的整数；%R 是因 -resumewhile 构造而重启的命令的“循环计数”；当任务因其依赖的子任务进入最终质量运行而执行时，%q 的值为 1.0；如果某个子任务在当前 resumewhile 循环中只到达检查点，则 %q 小于 1.0。许多被启动的命令不支持这些区分；作业作者需要根据每个命令的情况适当使用这些可选参数。

Job -title "checkpoint-resume iteration example" -service PixarRender -subtasks {
    Task {the wait} -subtasks {
        Task {the loop} -resumeblock 1 -subtasks {
            Task {render f1} -cmds {
                RemoteCmd {
                    prman -Progress -recover %r -checkpoint 0,15s /net/rib/rings.rib
                } -resumewhile {testcheckpoint exr /tmp/t.exr} -resumepin 1
            }
            Task {script test1} -cmds {
                RemoteCmd {echo otherApp --pass %R} -resumewhile {exitcode 123}
            }
        } -cmds {
            RemoteCmd "echo looping"
        }
    } -cmds {
        RemoteCmd "echo loop done"
    }
}
