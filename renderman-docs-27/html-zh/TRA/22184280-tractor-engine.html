tractor-engine

Source: https://rmanwiki-27.pixar.com/space/TRA/22184280/tractor-engine
Downloaded: 2026-06-15T03:49:12+00:00

Engine 维护中央作业队列，并把任务分配给渲染农场上的服务器。它也是 Blade 上报状态信息的中央交换中心。Engine 还会交付 Tractor Dashboard 所需的网页内容。用法：tractor-engine [options]

选项：

选项说明
--version打印版本信息，然后退出。
-h, --help打印用法摘要，然后退出。
--configdir=path
或 -c path
指定包含 Tractor 配置文件的目录，默认值为 /var/spool/tractor/config。tractor-engine 也会检查环境变量 TRACTOR_CONFIG。
--log=filename指定用于记录 Engine 自身诊断信息的 filename。默认值为 stderr。
--verbose 或 -v将日志级别从 NOTICE 提高到 INFO。
--quiet   或 -q将日志级别从 NOTICE 降低到 SEVERE。
--debug将日志级别提高到 DEBUG。
--trace将日志级别提高到 TRACE。
--httpdebug记录传入的 HTTP 请求
--nshippers=n
或 -n nnn
设置为处理 I/O 而创建的线程数量（default=ncores）。
--maxdispatch=n
或 -D nnn
限制并发派发数量（以及许可证使用）。
--port=n指定 Engine 的传入连接端口（default=80）。
--port=addr:port将传入监听端口限制到 address 给出的网络接口，并使用端口号 port。默认情况下，Engine 会在所有接口的端口 80 上监听连接。示例用法：--port=8080 或 --port=127.0.0.1:9999
--supersede
允许新启动的 Engine 进程平稳接管仍在运行的先前 Engine。新 Engine 会向旧 Engine 发送 drain and exit 消息，然后等待旧 Engine 退出，再继续成为新的 Engine 服务。在 drain mode 中，旧 Engine 停止向发出请求的 Blade 分配新工作，并在所有之前启动的任务完成后退出。部分完成的作业会保留在队列中，并在新 Engine 接管后继续处理。
--paused使新重启的 Engine 立即进入 pause all dispatching 状态。因此，在管理员使用 Dashboard 的 Gears->Admin 标签页中的红色 play 按钮手动重新启用派发之前，不会有新的命令分配给 Blade。即使 Engine 以 --paused mode 重启，作业数据库中标记为 still-active 的任何现有命令仍会被新 Engine 视为 active。在这些情况下，仍在运行的 Blade 会与新 Engine 通信，以确认这些命令中哪些实际仍在运行。
--startupdelay=n  Engine 在启动时简单休眠 n 秒。
--daemonEngine 在系统服务模式中自动进入后台。
--pidfile=filename在 daemon mode 中，将最终 PID 写入 filename。
--sigdfl=1不要覆盖默认的 SEGV/BUS 处理器。
