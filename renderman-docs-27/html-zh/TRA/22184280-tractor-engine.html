tractor-engine

Source: https://rmanwiki-27.pixar.com/space/TRA/22184280/tractor-engine
Downloaded: 2026-06-15T03:49:12+00:00

Engine 维护中央 job queue，并把 tasks 分配给 farm 上的 servers。它也是 blades 上报 status information 的中央 clearinghouse。Engine 还会交付 Tractor Dashboard 所需的 web content。Usage: tractor-engine [options]（用法）

选项：

Option（选项）说明
--version打印版本信息，然后退出。
-h, --help打印用法摘要，然后退出。
--configdir=path
或 -c path
指定包含 Tractor configuration files 的目录，默认值为 /var/spool/tractor/config。Tractor-engine 也会检查环境变量 TRACTOR_CONFIG。
--log=filename指定用于记录 engine 自身 diagnostics 的 filename。默认值为 stderr。
--verbose 或 -v将 logging level 从 NOTICE 提高到 INFO。
--quiet   或 -q将 logging level 从 NOTICE 降低到 SEVERE。
--debug将日志 logging level 提高到 DEBUG。
--trace将日志 logging level 提高到 TRACE。
--httpdebug记录传入的 http requests
--nshippers=n
或 -n nnn
设置为处理 i/o 而创建的 thread 数量（default=ncores）。
--maxdispatch=n
或 -D nnn
限制并发 dispatches（以及 license use）。
--port=n指定 engine 的 inbound connection port（default=80）。
--port=addr:port将 inbound listener port 限制到由 address 给出的 network interface，并使用 port number port。默认情况下，engine 会在所有 interfaces 的 port 80 上监听 connections。示例用法：--port=8080 或 --port=127.0.0.1:9999
--supersede
允许新启动的 engine process 平稳接管仍在运行的先前 engine。新 engine 会向旧 engine 发送 drain and exit 消息，然后等待旧 engine 退出，再继续成为新的 engine service。在 drain mode 中，旧 engine 停止向发出请求的 blades 分配新工作，并在所有之前启动的 tasks 完成后退出。部分完成的 jobs 会保留在 queue 中，并在新 engine 接管后继续处理。
--paused使新重启的 engine 立即进入 pause all dispatching 状态。因此，在管理员使用 Dashboard 的 Gears->Admin 标签页中的红色 play 按钮手动重新启用 dispatching 之前，不会有新的 commands 分配给 blades。即使 engine 以 --paused mode 重启，job database 中标记为 still-active 的任何现有 commands 仍会被新 engine 视为 active。在这些情况下，仍在运行的 blades 会与新 engine 通信，以确认这些 commands 中哪些实际仍在运行。
--startupdelay=n  Engine 在 startup 时简单 sleep n 秒。
--daemonEngine 在 system service mode 中自动进入 background。
--pidfile=filename在 daemon mode 中，将最终 PID 写入 filename。
--sigdfl=1不要覆盖默认的 SEGV/BUS handlers。
