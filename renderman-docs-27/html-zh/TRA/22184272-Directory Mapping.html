Directory Mapping

Source: https://rmanwiki-27.pixar.com/space/TRA/22184272/Directory+Mapping
Downloaded: 2026-06-15T03:49:12+00:00

Directory Mapping（目录映射）是 Tractor 的一项功能，允许你指定一个系统上的 file system paths，使其对应到另一个 operating system 上的相同 asset paths。例如，如果 job script 在 Windows 系统上使用 UNC file paths 创建，而你希望该 job 在通过 NFS 访问文件的 Linux 或 macOS 系统上运行，就可以指定 directory mapping，将 UNC paths 改写为 NFS paths。Directory maps 也可用于把 Windows paths 中的 drive letter 替换为 fully qualified UNC path。

要指定 directory map，可以添加 Job 参数 -dirmaps，形式如下：

Job -dirmaps { maps }

其中 maps 是采用以下格式的 directory maps 列表：

{ { source } { destination } zone }

在此格式中，如果 job 正在指定 zone 的计算机上运行，file path 中的 source 会被 destination 替换。

例如，下面是 job script 中一些有效的 directory maps：

 Job -title {A Simple Job}
     -dirmaps {
         {{mayabatch} {maya} NFS}
         {{X:/} {//fileserver/projects/} UNC}
         {{X:/} {/fileserver/projects/} NFS}
     }
     -subtasks {
         Task -title {render a.rib} -cmds {
             RemoteCmd {%D(mayabatch) -batch -file {%D(X:/proj1/scenes/test.ma)} -proj {%D(X:/proj1)}} -service {RfMRender}
     }
 }

这个 -dirmap option 中指定了三个不同的 maps。第一个指定当 job 在 NFS zone 的计算机上运行时，mayabatch 应被替换为 maya。第二个指定当 job 在 UNC zone 的计算机上运行时，X:/ 应被替换为 //fileserver/projects/。Directory maps 只应用于 command 中 %D() 区域内的文本。因此，如果这个 job 在 UNC zone 的 Windows 计算机上运行，command 将按如下方式运行：

mayabatch -batch -file //fileserver/projects/proj1/scenes/test.ma -proj //fileserver/projects/proj1

如果该 job 在 NFS zone 的 Linux 或 macOS 计算机上运行，则会按如下方式运行：

maya -batch -file /fileserver/projects/proj1/scenes/test.ma -proj /fileserver/projects/proj1

默认情况下，在 Windows 上运行的 Tractor Blades 会使用 UNC zone，而 Linux 和 macOS blades 会使用 NFS zone。也可以通过在匹配的 blade profile

中指定 DirMapZone 值，为给定 hosts 显式设置 zone name。另请注意，directory maps 仅 在 %D() 中的文本以 directory map 的 source text 开头时才会应用。因此，如果一个 directory map 写作 { {x} {y} UNC }，那么 %D(zzzx) 中的文本不会改变，但 %D(xzzz) 中的文本会变为 yzzz。

Windows Service Considerations（Windows 服务注意事项）

Windows services 无法访问 mapped drives。因此，如果你的 Maya project 设置为类似 X:\projects\proj1 的 mapped drive 位置，那么你需要设置 directory mapping，使作为 Windows service 运行的 Tractor Blades 能访问该位置。如果你的 X: drive 映射到 \\fileserver\projects，那么应为 UNC zone 设置类似下面的 directory mapping：

SetPref WSDirMaps { {X:/ //fileserver/projects/ UNC } }

Note（注意）： Windows 有时会缓存 SMB mount credentials，也有时会清除之前已认证 credentials 的缓存。尤其是在较新的 Windows 版本（8.1+）中，当 blade 作为 service 运行时，UNC name 的 credential 需要把 Persistence 设置为 “enterprise”。如果你只是用 File Explorer 浏览到 share 并在那里输入密码，它不会被设置成这种值。设置 persistence 的步骤如下：

  • 打开 Control Panel->User Accounts and Family Safety->Manage Windows Credentials
  • 如果给定 share 的旧 credentials 的 persistence 尚不是 “enterprise”，则删除它们
  • 点击 “add a Windows credential”（添加 Windows 凭据）
  • 输入相应的 \\server\share、account name 和 password。

记得将 tractor-blade service 配置为以同一用户运行。* 进入 Control Panel->...->Services，并将 blade service 的 “Run As Account” 设置为你选择的 account。* 重启该 service
