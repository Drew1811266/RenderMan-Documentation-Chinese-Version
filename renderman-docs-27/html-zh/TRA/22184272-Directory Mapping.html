Directory Mapping

Source: https://rmanwiki-27.pixar.com/space/TRA/22184272/Directory+Mapping
Downloaded: 2026-06-15T03:49:12+00:00

目录映射是 Tractor 的一项功能，允许你指定一个系统上的文件系统路径，使其对应到另一个操作系统上的相同资产路径。例如，如果作业脚本是在 Windows 系统上使用 UNC 文件路径创建，而你希望该作业在通过 NFS 访问文件的 Linux 或 macOS 系统上运行，就可以指定目录映射，将 UNC 路径改写为 NFS 路径。目录映射也可用于把 Windows 路径中的驱动器盘符替换为完全限定的 UNC 路径。

要指定目录映射，可以添加 Job 参数 -dirmaps，形式如下：

Job -dirmaps { maps }

其中 maps 是采用以下格式的目录映射列表：

{ { source } { destination } zone }

在此格式中，如果作业正在指定区域的计算机上运行，文件路径中的 source 会被 destination 替换。

例如，下面是作业脚本中一些有效的目录映射：

 Job -title {A Simple Job}
     -dirmaps {
         {{mayabatch} {maya} NFS}
         {{X:/} {//fileserver/projects/} UNC}
         {{X:/} {/fileserver/projects/} NFS}
     }
     -subtasks {
         Task -title {render a.rib} -cmds {
             RemoteCmd {%D(mayabatch) -batch -file {%D(X:/proj1/scenes/test.ma)} -proj {%D(X:/proj1)}} -service {RfMRender}
     }
 }

这个 -dirmap 选项中指定了三个不同的映射。第一个指定当作业在 NFS 区域的计算机上运行时，mayabatch 应被替换为 maya。第二个指定当作业在 UNC 区域的计算机上运行时，X:/ 应被替换为 //fileserver/projects/。目录映射只应用于命令中 %D() 区域内的文本。因此，如果这个作业在 UNC 区域的 Windows 计算机上运行，命令将按如下方式运行：

mayabatch -batch -file //fileserver/projects/proj1/scenes/test.ma -proj //fileserver/projects/proj1

如果该作业在 NFS 区域的 Linux 或 macOS 计算机上运行，则会按如下方式运行：

maya -batch -file /fileserver/projects/proj1/scenes/test.ma -proj /fileserver/projects/proj1

默认情况下，在 Windows 上运行的 Tractor Blade 会使用 UNC 区域，而 Linux 和 macOS Blade 会使用 NFS 区域。也可以通过在匹配的 Blade 配置档

中指定 DirMapZone 值，为给定主机显式设置区域名称。另请注意，目录映射 仅 在 %D() 中的文本以目录映射的 source 文本开头时才会应用。因此，如果一个目录映射写作 { {x} {y} UNC }，那么 %D(zzzx) 中的文本不会改变，但 %D(xzzz) 中的文本会变为 yzzz。

Windows 服务注意事项

Windows 服务无法访问已映射驱动器。因此，如果你的 Maya 项目设置为类似 X:\projects\proj1 的映射驱动器位置，那么你需要设置目录映射，使作为 Windows 服务运行的 Tractor Blade 能访问该位置。如果你的 X: 驱动器映射到 \\fileserver\projects，那么应为 UNC 区域设置类似下面的目录映射：

SetPref WSDirMaps { {X:/ //fileserver/projects/ UNC } }

注意： Windows 有时会缓存 SMB 挂载凭据，也有时会清除之前已认证凭据的缓存。尤其是在较新的 Windows 版本（8.1+）中，当 Blade 作为服务运行时，UNC 名称的凭据需要把 Persistence 设置为 “enterprise”。如果你只是用 File Explorer 浏览到共享位置并在那里输入密码，它不会被设置成这种值。设置持久性的步骤如下：

  • 打开控制面板 -> 用户账户和家庭安全 -> 管理 Windows 凭据
  • 如果给定共享位置的旧凭据的持久性尚不是 “enterprise”，则删除它们
  • 点击“添加 Windows 凭据”
  • 输入相应的 \\server\share、账户名和密码。

记得将 tractor-blade 服务配置为以同一用户运行。* 进入控制面板 -> ... -> 服务，并将 Blade 服务的 “Run As Account（运行账户）” 设置为你选择的账户。* 重启该服务
