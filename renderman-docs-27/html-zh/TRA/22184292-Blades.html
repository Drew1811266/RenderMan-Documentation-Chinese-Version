Tractor Blade 这个术语既用于描述渲染农场中的物理主机，也用于描述 tractor-blade 远程执行服务。tractor-blade 服务程序是 Tractor 系统的“执行前端”，它实际启动作业中指定的命令，从而触发渲染、仿真或其他任何操作。

在给定主机上，只需要一个 tractor-blade 实例。它可以管理该主机上的多个并发 槽位，每个槽位运行不同命令。它还会跟踪各种 主机性能指标，例如整台机器的 CPU 负载和内存用量，并将它们报告给 tractor-engine。

由于一个渲染农场可能包含 许多 blade 主机，但机器类型或职责只有少数几种 变化形式，Tractor 会以 blade 主机配置档案 来定义 blade 行为；这些配置档案只需定义一次，然后可以应用到渲染农场中的许多 blade 实例。

Server Profiles（服务器配置档案）
对渲染主机进行分类。
Environment Handlers（环境处理器）
管理命令的执行环境。
Status Filter（状态过滤器）
监控并响应命令输出。
Dynamic Blade Capability（动态 Blade 能力配置）
针对特殊情况，对 blade 配置档案的 "Provides" 列表进行高级变体设置。
