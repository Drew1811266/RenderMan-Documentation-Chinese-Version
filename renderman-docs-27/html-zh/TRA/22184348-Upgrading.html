Upgrading

Source: https://rmanwiki-27.pixar.com/space/TRA/22184348/Upgrading
Downloaded: 2026-06-15T03:49:12+00:00

升级 Engine 就像用新版本重启它一样简单。除非另有说明，活动任务可以在重启期间继续运行。新的 Engine 与较旧的 Blade 兼容；只有在需要利用新的 Blade 功能时，才需要重启 Blade。

某些站点可能希望在升级前备份作业数据。这样做有两个好处：备份可以在升级期间发生意外问题（例如软件或硬件故障）时防止历史作业数据丢失；另外，如果需要降级，系统可以恢复到升级前状态。备份和降级会在下文讨论。

 

停机时间

执行升级所需时间主要受修改数据库架构所需操作影响。特定升级的文档会提示何时可能需要较长时间（例如超过一分钟）来执行升级。升级期间，活动任务可以继续运行；不过不会调度新工作，不能提交新作业，Dashboard、命令行和 API 与 Engine 的通信会阻塞。

 

备份

备份作业数据库前，请确保 Tractor Engine 和 PostgreSQL 数据库已停止。除非站点在其 db.config 文件中将 DBShutdown 设置为 False，或 Engine 难以与数据库服务器通信，否则 Engine 会自动关闭数据库服务器。 tractor-dbctl --status 可用于验证数据库服务器是否已停止。如果它仍在运行，可以用 tractor-dbctl --stop停止。

要备份作业数据库，请运行 tractor-dbctl --backup <path-to-backup-file>。请确保有足够空间存储备份文件。备份文件是 Engine 数据目录的 psql/ 子目录的压缩 tarball，因此只需要该目录所用空间的一小部分。

 

降级

降级时，如果现有数据库已经历架构变更，可能无法继续使用。在这种情况下，需要从新的作业数据库开始，或从备份恢复。如果必须保留升级后发生的作业提交和状态变更，请联系 Customer Support。

要执行降级，请按上文 备份所述停止现有 Engine 和数据库服务器。然后用以下命令销毁作业数据库： tractor-dbctl --destroy。

要从备份恢复作业数据库，请使用 tractor-dbctl --restore <path-to-backup-file> 并以适用版本的 Engine 启动；作业 ID 会从创建备份时最后使用的作业 ID 继续。备份时处于 queued 状态的作业会在系统恢复到备份后重新调度。备份之后提交的作业不会出现在恢复后的作业数据库中。

如果作业数据库没有从备份恢复，它就只是一个空作业数据库，作业 ID 从 1 开始。

注意，如果作业 ID 被重复使用，其日志文件输出会追加到之前同 ID 作业的日志文件中。因此，建议移除与将被重复使用的作业 ID 相关的日志文件。

作业 ID 序列可以从不同值恢复，以避免重复使用作业 ID。首先用以下命令启动数据库服务器： tractor-dbctl --start-for-engine 然后运行 tractor-dbctl --set-job-counter <#>.

概括来说，降级的步骤如下：

  1. 在 Dashboard 管理面板中暂停调度。
  2. 重试所有活动任务：tq --force --yes retry active
  3. 使用 Kill 命令终止 Engine 进程。
  4. 销毁作业数据库： tractor-dbctl --destroy
  5. 可选地恢复已备份的数据库： tractor-dbctl --restore <path-to-backup-file>
  6. 启动数据库服务器： tractor-dbctl --start-for-engine
  7. 用以下命令查看最后使用的作业 ID： tractor-dbctl --show-params 并移除作业 ID 更高的作业的任何日志文件。
  8. 可选地设置作业计数器： tractor-dbctl --set-job-counter <#>
  9. 启动适用的 Engine。

注意，这里描述的所有 tractor-dbctl 命令都需要使用 --configdir 选项指定站点配置目录。

 

多个 Engine

可以同时运行不同版本的 Tractor Engine，例如在部署到生产环境前测试新版本。多个 Engine 可以在同一主机上运行；不过它们必须具有不同的端口、配置目录、数据目录和数据库目录。即使 Engine 不在同一主机上运行，只要它们对上述任一目录使用相同的网络共享卷，就可能发生损坏。这里重要的配置设置是 tractor.config 中的 TractorDataDirectory 和 db.config 中的 DBDataDir。还需要管理不同的日志文件位置，以及让 Blade 指向合适的 Engine。
