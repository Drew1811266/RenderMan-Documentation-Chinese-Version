Upgrading

Source: https://rmanwiki-27.pixar.com/space/TRA/22184348/Upgrading
Downloaded: 2026-06-15T03:49:12+00:00

升级 engine 就像用新版本重启它一样简单。除非另有说明，active tasks 可以在 restart 期间继续运行。新的 engine 与较旧的 blades 兼容；只有在需要利用新的 blade functionality 时，才需要重启 blades。

某些站点可能希望在 upgrading 前备份 job data。这样做有两个好处：backup 可以在 upgrade 期间发生意外问题（例如 software 或 hardware failure）时防止 historical job data 丢失；另外，如果需要 downgrade，system 可以恢复到 pre-upgrade state。Backups 和 downgrades 会在下文讨论。

 

停机时间说明（Down Time）

执行 upgrade 所需时间主要受修改 database schema 所需 operations 影响。特定 upgrade 的 documentation 会提示何时可能需要较长时间（例如超过一分钟）来执行 upgrade。Upgrade 期间，active tasks 可以继续运行；不过不会 schedule 新 work，不能 submit 新 jobs，Dashboard、command-line 和 API 与 engine 的通信会 block。

 

备份操作说明（Backing Up）

备份 job database 前，请确保 tractor engine 和 postgresql database 已停止。除非站点在其 db.config 文件中将 DBShutdown 设置为 False，或 engine 难以与 database server 通信，否则 engine 会自动关闭 database server。 tractor-dbctl --status 可用于验证 database server 是否已停止。如果它仍在运行，可以用 tractor-dbctl --stop停止。

要备份 job database，请运行 tractor-dbctl --backup <path-to-backup-file>。请确保有足够空间存储 backup file。Backup file 是 engine data directory 的 psql/ subdirectory 的 compressed tarball，因此只需要该 directory 所用空间的一小部分。

 

降级操作说明（Downgrading）

Downgrading 时，如果现有 database 已经历 schema change，可能无法继续使用。在这种情况下，需要从新的 job database 开始，或从 backup restore。如果必须保留 upgrade 后发生的 job submissions 和 state changes，请联系 Customer Support。

要执行 downgrade，请按上文 Backing Up所述停止现有 engine 和 database server。然后用以下命令 destroy job database： tractor-dbctl --destroy。

要从 backup restore job database，请使用 tractor-dbctl --restore <path-to-backup-file> 并以适用版本的 engine 启动；job IDs 会从创建 backup 时最后使用的 job ID 继续。Backup 时处于 queued 状态的 jobs 会在 system 恢复到 backup 后重新 scheduled。Backup 之后提交的 jobs 不会出现在 restored job database 中。

如果 job database 没有从 backup restore，它就只是一个空 job database，job IDs 从 1 开始。

注意，如果 job ID 被 reused，其 log file output 会 append 到之前同 ID job 的 log files 中。因此，建议移除与将被 reused 的 job IDs 相关的 log files。

Job ID sequence 可以从不同 value 恢复，以避免 reusing job IDs。首先用以下命令启动 database server： tractor-dbctl --start-for-engine 然后运行 tractor-dbctl --set-job-counter <#>.

概括来说，downgrade 的步骤如下：

  1. 在 Dashboard admin panel 管理面板中暂停 dispatching 调度。
  2. Retry 重试所有 active tasks 活动任务：tq --force --yes retry active
  3. 使用 Kill 命令终止 engine 进程。
  4. Destroy job database： tractor-dbctl --destroy
  5. 可选地 restore 恢复 backed up database 备份数据库： tractor-dbctl --restore <path-to-backup-file>
  6. Start 启动 database server： tractor-dbctl --start-for-engine
  7. 用以下命令查看最后使用的 job ID： tractor-dbctl --show-params 并移除 job ID 更高的 jobs 的任何 log files。
  8. 可选地设置 job counter： tractor-dbctl --set-job-counter <#>
  9. Start 启动适用的 engine。

注意，这里描述的所有 tractor-dbctl commands 都需要使用 --configdir option 指定 site configuration directory。

 

多个 Engines 引擎说明（Multiple Engines）

可以同时运行不同版本的 Tractor engines，例如在部署到 production 前测试新版本。多个 engines 可以在同一 host 上运行；不过它们必须具有不同的 ports、configuration directories、data directories 和 database directories。即使 engines 不在同一 host 上运行，只要它们对上述任一 directories 使用相同的 network-sharable volumes，就可能发生 corruption。这里重要的 configuration settings 是 tractor.config 中的 TractorDataDirectory 和 db.config 中的 DBDataDir。还需要管理不同的 log files locations，以及让 blades 指向合适的 engine。
