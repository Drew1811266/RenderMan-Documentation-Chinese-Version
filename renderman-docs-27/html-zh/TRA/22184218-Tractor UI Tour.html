概览（Overview）

该 UI 允许用户查看并控制已提交到 Tractor queue 的 job。它还会提供已连接到 engine 的 tractor-blade 信息。

要启动 Dashboard session，请将 web browser 指向 http://tractor-engine/tractor/dashboard 或者，如果你选择了其他 hostname 或 port，则指向 http: //your_engine_host:your_engine_port/tractor/dashboard。


重要提示

虽然 Tractor Dashboard 的访问方式像一个简单网页，但它是一个完整应用程序，因此会消耗不可忽略的内存，尤其是在复杂 pipeline 中数据量可能很大时。

还值得注意的是，不同 browser 执行 javascript 的性能不同。在我们的测试中，Safari 和 Google Chrome 表现最好。Firefox 工作得很好，但略慢。运行 Tractor UI 时，建议你测试不同 browser。

 

登录面板（Login Panel）

如果这是你第一次连接到 Dashboard，可能需要登录。目前，只要是 engine 主机上的有效 username 都可以接受。登录后，Job List pane 会显示该用户的 job。浏览器中会存储一个 cookie，使你在后续 session 中无需再次登录即可连接。

 

工具栏（Toolbar）

Tractor Dashboard toolbar 横跨浏览器窗口顶部，包含以下区域。

  • 页面图标（Page Icons）

    有七个图标用于切换到 Dashboard 中的不同页面。从左到右分别是 "About Tractor"、"Job List"、"Blade List"、"Metrics"、"Administration" 和 "Documentation"。
     

  • 重新加载 Job 或 Blade List（Reloading Job or Blade List）

    job list 和 blade list 会按照用户 preferences 中定义的 interval 自动刷新；默认 interval 为 60 秒。此外，job list data 会通过 Engine 发送到 Dashboard 的消息进行实时更新。如果用户希望获取最新信息，可以重新选择 job list 和 blade list 图标，强制刷新相应列表。鉴于来自 engine 的 job update 具有实时性，手动刷新更可能对更新 blade list 有用；不过，拥塞网络或高负载 Engine 可能导致某些 update 未到达 Dashboard，因此手动刷新也会有帮助。
     

  • 消息区域

    像 "Dispatching Paused" 这类一般重要消息会显示在 toolbar 中间区域。
     

  • 过滤器下拉菜单（The Filter Pulldown）

    你可以定义 filter，以限制或筛选 job list 或 blade list 中显示的信息量。有关 filter 的更多信息可在 Filter Editor reference中找到。
     

  • 模式匹配器（Pattern Matcher）

    pattern matcher 是一个输入字段，用作 selector 来筛选 job list 或 blade list 中可见的项目。pattern match 字段会与 job list 中的 owner 和 title 属性比较，也会与 blade list 中的 host 和 profile 属性比较。注意，与 filter 不同，此操作在 browser 中执行，只限制显示的项目数量。filter 由 Engine 应用，会限制从 Engine 发送到 Dashboard 的 job 或 blade 数量。
     

  • 用户名（User Name）

    当前登录用户的名称显示在最右侧。点击 username 会弹出下拉菜单，可导航到 user preferences 或执行 logout。


主显示区

Dashboard 首次运行时会进入 Jobs 模式。点击顶部 toolbar 中看起来像小型 4-task job graph 的按钮，也可以激活此模式。在此模式中有四个主要 pane：

  • Job List Pane：左上 pane 是 Joblist pane。满足用户 type-in field 的所有 job 都显示在此 pane 中。点击任何 job，都会在另外两个 pane 中显示信息。

  • Job Info Pane：左下 panel 是 Job Info pane，包含来自 engine 的、描述所选 job 的信息。

  • Task Graph Pane：右上 panel 是 Task Graph pane。它可以把所选 job 显示为 graph 或 list。可通过点击 pane 顶部的相关图标来更改 display mode。

  • Task Info Pane：在 Task Graph pane 中选择某个 node 时，与该特定 task 及其当前 command 相关的信息会显示在 task info pane 中。该 pane 与 Job Info Pane 位于同一位置；当选择新的 job 后，显示会恢复为 job 相关信息。

点击 Blades 按钮（由三个 server 组成的堆叠图标表示）会使 Dashboard 进入 Blades 模式。在此模式中有四个 pane：

  • Blade List Pane：左上 pane 是 Blade List pane。所有 blade server 的列表和摘要信息会显示在这里。

  • Blade Activity Pane：右上 pane 是 Blade Activity pane。每个 blade 表示为一个方块，并根据其 status 着色。

  • Server Info Pane：在 Blade List pane 中选择特定 server 后，该 server 的信息会显示在左下 pane 中。

  • Assignment Tracer Pane：从 context menu 激活 Assignment Tracer 后，所选 blade 的 tracer output 会显示在右下 pane 中。

点击左上角的 Query 按钮（列表上方的放大镜）会使 Dashboard 进入 Query 模式。这是一个简单但强大的页面，可用于交互式搜索 tractor entity。详情请参见 Query Pane 页面。

点击 Graph 按钮会显示 system metrics 的图表，这些图表有助于了解 work queue 的大小、farm resource utilization 和 engine performance。它还会显示 donut chart，用于可视化 adaptive farm allocation 所管理资源的 utilization。

点击 Gears 按钮会显示 administration section，可在其中检查 engine log、触发 system reconfiguration、暂停 dispatching，并查看 limit count 和 user connectivity。

点击 Book 按钮会使 documentation 在新的 browser tab 或 window 中打开。

最后，点击 Tractor 按钮会显示系统的各种细节，例如 engine build、dispatching mode 和 license count。

 

界面提示

每个 pane 都有特定 UI 功能，相关页面已在上方链接中说明。Dashboard 主窗口提供以下功能：

  • 调整 Pane 大小（Pane Resizing）

    可以拖动 pane divider 来调整 main display pane 的大小。
     

  • 关闭 Pane（Pane Closing）

    点击 pane divider 的浅色中段，可以关闭或打开 pane。当 task graph pane 被关闭时，不会向 engine 请求用于渲染 graph 的数据，因为 graph 无论如何也不可见。选择非常大的 job 时，这有助于保持 Dashboard 响应。
     

  • 调整窗口大小（Window Resizing）

    如果 resize 整个 browser window，pane 会以相同 orientation/aspect-ratio resize 并适配新的 window。
     

  • 列排序

    点击 list view 中任意 header，会根据相关 attribute 对项目排序。再次点击同一 header 会反转排序。

    选择新的 sort column 后，之前的 sort column 会用于 secondary sorting。例如，在 Job List pane 中先选择 "Owner" header 再选择 "Priority" header，会使 job 按 priority 排序，同 priority 的 job 再按 owner 排序。
     

  • 调整列大小和重新排序（Column Resizing and Reordering）

    可以拖动 column header 之间的 separator 来调整 list view 中列的大小。也可以把 column header 拖到另一列旁边来重新排序。
     

  • 按 Z 滚动到所选项目（Z to Scroll Selected Item into View）

    在任何 view 中按 'Z' 键，都会将所选项目滚动到可见区域。当用户浏览大型 list 或 job graph 后难以找到所选项目时，这很有用。
