概览

该界面允许用户查看并控制已提交到 Tractor 队列的作业，也会提供已连接到 Engine 的 Tractor Blade（执行节点）信息。

要启动 Dashboard（仪表板）会话，请将 Web 浏览器指向 http://tractor-engine/tractor/dashboard 或者，如果你选择了其他主机名或端口，则指向 http: //your_engine_host:your_engine_port/tractor/dashboard。


重要提示

虽然 Tractor Dashboard 的访问方式像一个简单网页，但它是一个完整应用程序，因此会消耗不可忽略的内存，尤其是在复杂管线中处理大量数据时。

还值得注意的是，不同浏览器执行 JavaScript 的性能不同。在我们的测试中，Safari 和 Google Chrome 表现最好。Firefox 运行良好，但略慢。运行 Tractor UI 时，建议你测试不同浏览器。

 

登录面板

如果这是你第一次连接到 Dashboard，可能需要登录。目前，只要是 Engine 主机上的有效用户名都可以接受。登录后，Job List 面板会显示该用户的作业。浏览器中会存储一个 cookie，使你在后续会话中无需再次登录即可连接。

 

工具栏

Tractor Dashboard 工具栏横跨浏览器窗口顶部，包含以下区域。

  • 页面图标

    有七个图标用于切换到 Dashboard 中的不同页面。从左到右分别是 "About Tractor"、"Job List"、"Blade List"、"Metrics"、"Administration" 和 "Documentation"。
     

  • 重新加载作业列表或 Blade 列表

    作业列表和 Blade 列表会按照用户偏好设置中定义的间隔自动刷新；默认间隔为 60 秒。此外，作业列表数据会通过 Engine 发送到 Dashboard 的消息进行实时更新。如果用户希望获取最新信息，可以重新选择作业列表和 Blade 列表图标，强制刷新相应列表。鉴于来自 Engine 的作业更新具有实时性，手动刷新更可能对更新 Blade 列表有用；不过，拥塞网络或高负载 Engine 可能导致某些更新未到达 Dashboard，因此手动刷新也会有帮助。
     

  • 消息区域

    像 "Dispatching Paused" 这类一般重要消息会显示在工具栏中间区域。
     

  • 过滤器下拉菜单

    你可以定义过滤器，以限制或筛选作业列表或 Blade 列表中显示的信息量。有关过滤器的更多信息可在 Filter Editor 参考中找到。
     

  • 模式匹配器

    模式匹配器是一个输入字段，用作选择器来筛选作业列表或 Blade 列表中可见的项目。模式匹配字段会与作业列表中的 owner 和 title 属性比较，也会与 Blade 列表中的 host 和 profile 属性比较。注意，与过滤器不同，此操作在浏览器中执行，只限制显示的项目数量。过滤器由 Engine 应用，会限制从 Engine 发送到 Dashboard 的作业或 Blade 数量。
     

  • 用户名

    当前登录用户的名称显示在最右侧。点击用户名会弹出下拉菜单，可导航到用户偏好设置或执行注销。


主显示区

Dashboard 首次运行时会进入 Jobs 模式。点击顶部工具栏中看起来像小型四任务作业图的按钮，也可以激活此模式。在此模式中有四个主要面板：

  • Job List 面板：左上方面板是 Job List 面板。满足用户输入字段条件的所有作业都会显示在此面板中。点击任何作业，都会在另外两个面板中显示信息。

  • Job Info 面板：左下方面板是 Job Info 面板，包含来自 Engine 的、描述所选作业的信息。

  • Task Graph 面板：右上方面板是 Task Graph 面板。它可以把所选作业显示为图形或列表。可通过点击面板顶部的相关图标来更改显示模式。

  • Task Info 面板：在 Task Graph 面板中选择某个节点时，与该特定任务及其当前命令相关的信息会显示在任务信息面板中。该面板与 Job Info 面板位于同一位置；当选择新的作业后，显示会恢复为作业相关信息。

点击 Blades 按钮（由三个服务器组成的堆叠图标表示）会使 Dashboard 进入 Blades 模式。在此模式中有四个面板：

  • Blade List 面板：左上方面板是 Blade List 面板。所有 Blade 服务器的列表和摘要信息会显示在这里。

  • Blade Activity 面板：右上方面板是 Blade Activity 面板。每个 Blade 表示为一个方块，并根据其状态着色。

  • Server Info 面板：在 Blade List 面板中选择特定服务器后，该服务器的信息会显示在左下方面板中。

  • Assignment Tracer 面板：从上下文菜单激活 Assignment Tracer 后，所选 Blade 的追踪器输出会显示在右下方面板中。

点击左上角的 Query 按钮（列表上方的放大镜）会使 Dashboard 进入 Query 模式。这是一个简单但强大的页面，可用于交互式搜索 Tractor 实体。详情请参见 Query 面板 页面。

点击 Graph 按钮会显示系统指标图表，这些图表有助于了解工作队列大小、渲染农场资源利用率和 Engine 性能。它还会显示环形图，用于可视化自适应农场分配所管理资源的利用率。

点击 Gears 按钮会显示管理区域，可在其中检查 Engine 日志、触发系统重新配置、暂停调度，并查看限制计数和用户连接状态。

点击 Book 按钮会使文档在新的浏览器标签页或窗口中打开。

最后，点击 Tractor 按钮会显示系统的各种细节，例如 Engine 构建版本、调度模式和许可证数量。

 

界面提示

每个面板都有特定界面功能，相关页面已在上方链接中说明。Dashboard 主窗口提供以下功能：

  • 调整面板大小

    可以拖动面板分隔条来调整主显示面板的大小。
     

  • 关闭面板

    点击面板分隔条的浅色中段，可以关闭或打开面板。当 Task Graph 面板被关闭时，不会向 Engine 请求用于渲染图形的数据，因为图形无论如何也不可见。选择非常大的作业时，这有助于保持 Dashboard 响应。
     

  • 调整窗口大小

    如果调整整个浏览器窗口大小，面板会以相同方向和宽高比调整大小，并适配新的窗口。
     

  • 列排序

    点击列表视图中的任意表头，会根据相关属性对项目排序。再次点击同一表头会反转排序。

    选择新的排序列后，之前的排序列会用于二级排序。例如，在 Job List 面板中先选择 "Owner" 表头再选择 "Priority" 表头，会使作业按 priority 排序，同 priority 的作业再按 owner 排序。
     

  • 调整列大小和重新排序

    可以拖动列标题之间的分隔线来调整列表视图中列的大小。也可以把列标题拖到另一列旁边来重新排序。
     

  • 按 Z 滚动到所选项目

    在任何视图中按 'Z' 键，都会将所选项目滚动到可见区域。当用户浏览大型列表或作业图后难以找到所选项目时，这很有用。
