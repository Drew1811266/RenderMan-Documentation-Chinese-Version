数据量过大

Joblist 面板和 ServerList 面板都允许定义过滤器，以限制从 monitor 返回给 UI 的信息量。

可以创建并保存自定义过滤器定义供个人使用，管理员也可以创建通用过滤器，并把它们放在共享区域，供站点内所有人使用（见下文）。

这些过滤器很重要，因为 UI 是运行在 Web 浏览器中的 JavaScript 应用程序。必须限制从 monitor 发送到 UI 的数据量，以防止浏览器崩溃。

通常，joblist 或 bladelist 会在各自面板中显示条目数量。当达到数据限制，并且过滤正在限制记录总量时，计数后面会跟随 + 如上图所示。这表示当前查询产生的数据超过 UI 可显示的数据量。

由于 Joblist 面板会订阅事件，并且作业会不断添加到该面板或从该面板中移除，UI 会自行采取预防措施以避免数据溢出。

  • UI 会为作业列表实现一个滚动窗口。随着新作业进入 joblist，较旧的作业会被挤出列表，从而使显示的作业保持在最大列表大小。
  • 如果作业被删除，且作业列表现在低于阈值，则 UI 会自动向调度引擎请求作业数据，以确保始终显示尽可能多的作业。
     

编辑过滤器

Tractor 使用过滤器来限制从 monitor 发送到 UI 的数据量，以防止数据超限。Filter Lister 对话框会向用户提供可用过滤器列表，以及用于编辑现有过滤器的按钮。Filter Rule 编辑器允许你通过构建 tractor monitor 可理解的规则列表来创建或编辑过滤器。

选择下拉过滤器菜单时，会显示 Filter Lister 对话框（如上图所示）。如果当前存在正确类型（joblist 或 bladelist）的已定义过滤器，它们会显示出来。

操作过滤器时有几个选项：

  • New：打开 Filter Editor 对话框，用于构建新过滤器
  • Edit：打开 Filter Editor 对话框，用于编辑所选过滤器
  • Copy：在过滤器管理器中打开所选过滤器的副本
  • Delete：从 UI 中删除所选过滤器，并通知调度引擎从用户的过滤器目录中移除该过滤器。
  • Apply：将当前所选过滤器设置为 活动 过滤器。
  • None：移除当前所选过滤器。
  • Cancel：关闭对话框，不执行任何操作。

选择 New, Edit 或 Copy时会打开 Filter Editor 对话框。Filter Editor 允许你使用一组已定义的过滤器规则，针对当前面板（Jobs 或 Blades）动态构建新过滤器，或编辑现有过滤器。

  • Any/All Pulldown：在单个过滤器中使用多条规则时，此控件定义数据是满足任意一条规则即可包含，还是必须满足所有规则才能包含。
  • + Button：此按钮向当前过滤器添加一条新规则。
  • - Button：此按钮从过滤器中移除关联的规则。
  • First Column Pulldown：包含规则名称。更改此下拉菜单会改变规则定义的其余部分。选择所需规则，然后在其他条目中设置适当的值。注意，有些规则定义需要输入值。例如上方的Spool Time 和 Finish Time 规则都有用于持续时间的输入值（当前选择单位为分钟）。
  • Cancel：中止当前过滤器编辑会话，并返回主 UI。
  • Save：使用 Filter Name下定义的名称将当前规则集保存为过滤器。如果尚未指定名称，会显示警报，并取消保存。

定义任何过滤器后，它们会出现在作业面板或服务器面板的下拉菜单中。
 

站点级共享过滤器

管理员可以把有用的通用过滤器复制到 Tractor spool 区域中的共享区域；当用户重新加载 Dashboard 会话时，这些过滤器就会可用。此复制目前是手动操作。下面示例把用户 "yoda" 创建的名为 "NoDarkSide" 的过滤器复制到共享区域。此示例假定 Tractor spool 区域位于默认的 /var/spool/tractor；在你的站点上它可能位于其他位置。

  • cd /var/spool/tractor/users
  • mkdir -p _shared_/filters
  • cp yoda/filters/NoDarkSide.joblist _shared_/filters/
