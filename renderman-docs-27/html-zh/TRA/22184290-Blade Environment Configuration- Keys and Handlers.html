Blade Environment Configuration: Keys and Handlers

Source: https://rmanwiki-27.pixar.com/space/TRA/22184290/Blade+Environment+Configuration%3A+Keys+and+Handlers
Downloaded: 2026-06-15T03:49:12+00:00

概述

当 tractor-blade 启动一个新命令（例如一次渲染）时，它会为该子进程设置一组唯一的环境变量。定义这些环境变量及其值有几种机制。

默认做法是直接允许每个命令克隆 tractor-blade 启动时继承进来的环境。不过，这通常是专门化的 "daemon" 环境，因此并不太有用。

自定义环境配置最简单的做法，是在 blade.config 文件里的环境字典中添加名称-值对（见下文）。这通常已经完全足够，而且非常直接。有一个 "default" 字典，也可以有若干独立字典，对应作业中命令传入的 EnvKey 值。以这种方式指定的环境变量会作为新增项和覆盖项应用到传入的继承环境上。

Tractor envhandlers 是 Python 对象，可对 tractor-blade 启动的每个命令应用动态的、站点特定设置。它们最常用于根据传入的 EnvKey 或作业所有者配置一组特定环境变量。这些处理器也可以重写实际要调用的命令，这有助于处理某些使用非标准应用安装位置的机器，或其他特殊情况。


环境变量字典

可以把对继承外部环境的新增项和覆盖项添加到一个或多个 Python 字典中，Blade 会加载这些字典。它们可以加入 "default" EnvKey 定义，以影响所有命令；也可以加入某个特定 EnvKey 值的设置。同样，定义可以加入 ProfileDefaults，也可以专门加入单个 profile。

{
    "ProfileDefaults":
    {   "ProfileName": "default",
        ...
        "EnvKeys": [
            {
                "keys": ["default"],
                "environment": {
                    "SOME_VAR": "xyzzy",
                    "CURRENT_SHOW": "(not_set)"
                },
                "envhandler": "default"
            },
        ]
    },

    "BladeProfiles":
    [...]
}

这些环境字典要么定义在 blade.config 文件中，要么包含在一个外部文件里，并通过 blade.config 文件中的 @merge directive 引用。默认安装包含三个外部文件，它们提供默认环境字典设置。这些文件是 shared.linux.envkeys、shared.macosx.envkeys 和 shared.windows.envkeys。


环境处理器

默认环境处理器

Tractor 自带一组默认环境处理器。当传入的 envkey 与环境键字典中的 keys 定义匹配时，就会调用这些处理器。Tractor 脚本中的 envkey 参数实际上可以是由多个 envkey 条目组成的列表。在这种情况下，每个处理器会按顺序被调用，以修改启动环境。

每个默认处理器都有特定用途。

  • SiteModulesPath

    default - default 处理器会在每个命令上运行。它最先运行，环境键字典应该设置好预期的默认环境，而不是依赖已启动 Blade 的环境。

    按提供的默认行为，default 处理器只是把提交主机设置进启动环境，变量名为 REMOTEHOST。默认环境变量和路径应在这里设置。

  • rmshandler - rmshandler 会响应格式为 rms-*-maya-* 的传入键。* 表示会应用 glob 匹配。此处理器会匹配如下传入键：rms-3.0.0-maya-2011。

    此处理器会提取 RMS 版本和 MAYA 版本，并设置多个环境变量，包括 RMSTREE, MAYA_LOCATION, MAYA_MODULE_PATH。它还会把 RMSTREE/bin 和 MAYA_LOCATION/bin 加入路径。这些位置通过了解这些产品在特定平台上的默认安装位置来确定。处理器会在设置这些变量之前尝试验证 RMS 和 Maya 是否已安装。

  • mayahandler - mayahandler 会响应格式为 maya* 的传入键。* 表示会应用 glob 匹配。此处理器会匹配如下传入键：maya2011。

    mayahandler 提供与 rmshandler 类似的功能，但面向未安装 RMS、却执行 Maya 批渲染的 Maya 用户。此处理器会提取 MAYA 版本，并设置多个环境变量，包括 MAYA_LOCATION, MAYA_MODULE_PATH。它还会把 MAYA_LOCATION/bin 加入路径。Maya 位置通过了解 Maya 在特定平台上的默认安装位置来确定。处理器会在设置这些变量之前尝试验证 Maya 是否已安装。

  • rmanhandler - rmanhandler 会响应格式为 prman-* 的传入键。此键用于定义要使用的 prman 版本，并且经常与上面的 rmshandler 键一起使用。

    此处理器会匹配类似 prman-15.2 的传入键。匹配后，处理器会设置环境变量 RMANTREE，并把 RMANTREE/bin 加入路径。RMANTREE 的位置通过了解特定平台上的默认安装位置来确定。处理器实际上会在设置该 prman 版本的 RMANTREE 环境变量之前，尝试判断该 prman 版本是否已安装。

  • rmantreehandler - rmantreehandler 是一个特殊处理器，客户通常不会使用。此处理器响应格式为 rmantree=* 的键。它的工作方式类似上面的 rmanhandler，但该键会提供指向非标准 RMANTREE 安装的完整路径。例如，传入键可能类似：rmantree=/opt/software/pixar/RenderManProServer-15.2。键中定义的完整路径会被设置为 RMANTREE 变量，并且 RMANTREE/bin 会加入路径。

  • rmstreehandler - rmstreehandler 与上面的 rmantreehandler 类似，但它设计用于指向非标准 RMSTREE 位置。处理器响应格式为 rmstree=* 的传入键。典型用法可能是 rmstree=/opt/software/pixar/RenderManStudio-3.0.0-Maya-2011。此处理器会设置 RMSTREE 环境变量，并把 RMSTREE/bin 加入 path。

默认处理器源代码可用，位于 Blade 安装的 blade-modules 目录中的 TrEnvHandlers.py 文件。


添加自定义环境处理器

  • 为你的处理器选择一个名称 - 它应该能大致描述所处理 EnvKey 的类型。在下面的示例中，假设有一个名为 "ProjectX" 的传入 EnvKey，因此我们把处理器称为 "projxhandler"。

  • 选择一个用于保存 Python 模块的目录 - 这通常是中央文件服务器上的某个位置，所有 Blade 主机都可以访问。此位置将由 SiteModulesPath指定。

  • 把选定目录添加到 blade.config 中的 SiteModulesPath 指定。如果所有主机都使用相同路径挂载共享位置，就可以把它放在 ProfileDefaults 区段；否则按需给每个 profile 添加自定义变体（如下所示）。

  • 还要添加一个 EnvKeys 条目，用于你的新键 - 同时在字典中保留名为 "default" 的键。然后把你的模块名称添加为 envhandler 值。

    {
    "ProfileDefaults":
    {   "ProfileName": "default",
        ...
        "SiteModulesPath": "/net/yourserver/tractor_extensions",
        "EnvKeys": [
            {
                "keys": ["default"],
                "environment": {
                    "SOME_VAR": "xyzzy",
                    "CURRENT_SHOW": "(not_set)"
                },
                "envhandler": "default"
            },
            {
                "keys: ["ProjectX", "SHOT=*", "SCENE=*"],
                "environment": {
                    "CURRENT_SHOW": "Project X, Venture of Mystery",
                    "CURRENT_SHOT": "$SHOT",
                    "CURRENT_SCENE": "$SCENE",
                    "RMANTREE": "/opt/pixar/RenderManProServer-15.2",
                    "PATH": "$RMANTREE:/bin;$PATH"
                },
                "envhandler": "projxhandler"
            },
        ]
    },

    "BladeProfiles":
    [...]
}
