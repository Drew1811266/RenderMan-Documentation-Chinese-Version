Your First Job

安装 Tractor、customized 你的 configuration files，并启动 engine 和 blade services 后，就可以提交你的 first job。

 

打开 Dashboard 的操作步骤（Open a Dashboard）

Dashboard 是一种方便的方式，可在你喜欢的 browser 中轻松管理 jobs 和 blades。

转到 http://tractor-engine，或 http://HOSTNAME:PORT（如果你有 custom hostname 和 port），然后 login 到 Tractor。

如果无法访问 Dashboard，请检查 engine 实际上是否正在运行。engine log file 可以提供有用的 diagnostic information。

 

示例使用说明（About the Examples）

以下 examples 假定你的 current directory 是 INSTALL_DIR/bin。

TRACTOR_ENGINE environment variable 或 --engine flag 可用于把 command line programs 指向与默认值不同的 engine，或指向通过 engine discovery确定的 engine。

 

提交作业步骤（Submit a Job）

Jobs 可以提交到 Tractor queue，方式包括使用 tractor-spool script、RenderMan for Maya plug-in，或使用 author API（作业创作 API）提交到 Tractor queue。Job submission 有时也称为 spooling，这是类比 print spooling 或传统 batch job systems 的说法。

tractor-spool 最常把一个 Alfred-syntax job file（Alfred 语法作业文件） 的 filename（文件名）作为 argument（参数）。

% tractor-spool jobfile.alf

下面的 example job 可以放入一个 file 中，并用 tractor-spool 提交。它只会输出 command 运行所在的 environment。

Job -title {Environment Job} -subtasks { Task -title {Environment Task} -cmds { RemoteCmd {{printenv}} -service {pixarRender} } }

你可以使用 Dashboard 查看 task 的 output，也可以在 command line 中使用 tq。job id JOBID 应替换为 tractor-spool 输出的 job id。

tq log jid=JOBID and tid=1

-c option 会为剩余 arguments 指定的 single command 创建一个新的 tractor job。试运行这个 simple command 来测试 system 是否正常工作。job status 和 output 可以在 Dashboard 或 tq 中检查。

% tractor-spool -c printenv

 

删除作业步骤（Delete a Job）

可以在 Dashboard 中点击 job queue 中的 job entry，然后按 Delete 键 或 Backspace 退格键按键 来删除 jobs。

也可以在 command-line 中使用 spooling tool，并通过 --jdelete 指定 job id 来删除 jobs。

% tractor-spool --jdelete 1534

tq 也允许从 command-line 删除 job。

% tq delete jid=1534

Admins 可以在 Configuration section 中探索 advanced customization techniques。
