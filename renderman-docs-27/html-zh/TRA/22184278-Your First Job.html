Your First Job

安装 Tractor、自定义配置文件，并启动 Engine（调度引擎）和 Blade（执行节点）服务后，就可以提交你的第一个作业。

 

打开 Dashboard（仪表板）

Dashboard（仪表板）是一种便捷方式，可在你喜欢的浏览器中轻松管理作业和 Blade。

转到 http://tractor-engine，或 http://HOSTNAME:PORT（如果你有自定义主机名和端口），然后登录 Tractor。

如果无法访问 Dashboard，请检查 Engine 是否确实正在运行。Engine 日志文件可以提供有用的诊断信息。

 

关于示例

以下示例假定你的当前目录是 INSTALL_DIR/bin。

TRACTOR_ENGINE 环境变量或 --engine 标志可用于把命令行程序指向不同于默认值的 Engine，或指向通过 Engine discovery（Engine 发现）确定的 Engine。

 

提交作业

作业可以提交到 Tractor 队列，方式包括使用 tractor-spool 脚本、RenderMan for Maya 插件，或使用 author API（作业创作 API）提交到 Tractor 队列。作业提交有时也称为 spooling（提交），这是类比 print spooling（打印假脱机）或传统批处理作业系统的说法。

tractor-spool 最常把一个 Alfred 语法作业文件 的文件名作为参数。

% tractor-spool jobfile.alf

下面的示例作业可以放入一个文件中，并用 tractor-spool 提交。它只会输出命令运行所在的环境。

Job -title {Environment Job} -subtasks { Task -title {Environment Task} -cmds { RemoteCmd {{printenv}} -service {pixarRender} } }

你可以使用 Dashboard 查看任务输出，也可以在命令行中使用 tq。作业 ID JOBID 应替换为 tractor-spool 输出的作业 ID。

tq log jid=JOBID and tid=1

-c 选项会为剩余参数指定的单条命令创建一个新的 Tractor 作业。试运行这个简单命令来测试系统是否正常工作。作业状态和输出可以在 Dashboard 或 tq 中检查。

% tractor-spool -c printenv

 

删除作业

可以在 Dashboard 中点击作业队列中的作业条目，然后按 Delete 键 或 Backspace 键 来删除作业。

也可以在命令行中使用提交工具，并通过 --jdelete 指定作业 ID 来删除作业。

% tractor-spool --jdelete 1534

tq 也允许从命令行删除作业。

% tq delete jid=1534

 

进一步阅读

所有用户都会对 Dashboard（仪表板） 一节感兴趣，该节描述用于管理系统的图形界面。

开发人员可以在 Job Author Python API 一节中学习如何生成作业脚本。用于查询和管理作业与 Blade 状态的工具可以用 Query Python API创建。

Wranglers、管理员，以及其他熟悉命令行的用户，会喜欢命令行工具 tq的强大能力。

管理员可以在 Configuration 一节中探索高级自定义技术。
