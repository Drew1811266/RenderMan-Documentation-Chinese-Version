Query Python API

Source: https://rmanwiki-27.pixar.com/space/TRA/22184266/Query+Python+API
Downloaded: 2026-06-15T03:49:12+00:00

tractor.api.query 是一个 Python API，提供与 command line tool tq 相同的 query 和 control 功能。它随 Tractor 自带的 Python interpreter rmanpy 一起安装。

你也许可以把此 module 与自己的 interpreter 一起使用，但这需要 site-specific configuration，以便正确定位或安装 Tractor Python modules。

下面的 examples 假定 query module 已按如下方式 import：

import tractor.api.query as tq

List Functions（列表函数说明）

query module 提供用于检索 jobs、tasks、commands、invocations 和 blades 信息的 functions。它们的名称与 command line tool 中的 list commands 相同，即 jobs()、tasks()、commands()、invocations() 和 blades()。

这些 functions 的第一个 argument 是一个 search clause（搜索条件子句），用于缩小 search 范围。

在此 example 中，这个 list function 会取回一组 users 拥有、且在 8 天前以前 spool 的所有 jobs。list functions 的 return value 是一个 dictionaries list，每个 dictionary 表示一个 matched item。

>>> jobs = tq.jobs("owner in [freddie brian john roger] and spooltime < -8d")
>>> print jobs[0]

Search Clause（搜索条件子句）

search clause 是一个 boolean expression（上面的 example 使用 and），由 comparisons 和 operators（in 运算符、<）以及 entity attributes（owner、spooltime）组成。为减少输入量，这里做了一些简化。例如，freddie 这样的 string literals 不需要加 quotes，而 string -8d 是表示 "one day ago" 的 short form 简写形式。

关于 search clause syntax 的更多细节可在 这里找到。

注意，search 范围越宽，在定位、传输和存储 results 时消耗的 CPU、memory 和 bandwidth 就越多。这会影响 client、engine 和 backend database server。
 

Columns（列选择）

默认情况下，query functions 会检索给定 entity 的所有 attributes。不过，如果并不需要所有 attributes，这会浪费 client、engine 和 backend database 的 memory 与 CPU。可以使用 columns= keyword parameter 指定要检索的 attributes list。

例如，下面只取回 job id、owner、title 和 metadata columns：

>>> tq.jobs("active", columns=["jid", "owner", "title", "metadata"])

Sorting（排序）

可以使用 sortby= keyword parameter 对 results 排序。可以指定多个 attributes 以表示 secondary sorting。Items 默认按 ascending order 排列，除非 attribute 前面加上 - 来表示 descending order（降序排列）。

例如，为了显示相似 jobs 在 FIFO queue 中的处理顺序，可以按 priority descending、spooltime ascending 的顺序列出它们，如下所示：

>>> tq.jobs("active or ready", sortby=["-priority", "spooltime"])

虽然 item sorting 可以在 client side 完成，但如果 result set 已通过 limit= keyword parameter 截断，就无法这样做，因为 truncation 会在 server side 发生，早于 client 有机会对 results 排序。

请注意 sorting 可能会根据 query 的 complexity 和 scope，对 backend database server 产生影响。
 

Limiting（限制返回数量）

limit= keyword parameter 会设置 engine 返回 items 数量的 upper bound。

例如，下面的 statement 会取回 farm 上 active tasks 数量最多的十个 jobs：

>>> tq.jobs("active", sortby=["-numactive"], limit=10)

下面的 statement 会列出 disk space 最少的二十台 hosts。

>>> tq.blades("name like east", sortby=["availdisk"], limit=20)

请注意 把 limit 设置为 0 会对返回 records 数量施加 NO LIMIT（无限制）。

注意，如果未使用 limit= keyword parameter，会有一个生效的 system default upper bound。该 upper bound 为 2500，不过 admins 可以通过在 tractor.config 中设置 SiteMaxListReplyCount 覆盖它。
 

Affiliation（关联实体属性）

在上面的 examples 中，items 的 attributes 用于 search clauses、column specifications 和 sorting。也可以在所有这些情况下指定 affiliated entities 的 attributes，方法是在 attribute 前加上 entity name。例如，下面的 statement 会取回 walt 拥有的 active tasks。

>>> tq.tasks("state=active and Job.owner=walt")

类似地，下面的 command 会显示过去 24 小时内 spool 的 jobs 中所有 error tasks，包括其 jobs 的 titles 和 spooltimes，并按 job priority 排序。

>>> tq.tasks("state=error and Job.spooltime > -24h", columns=["Job.spooltime", "Job.title"], sortby=["Job.priority"])

因为 spooltime 和 priority 不是除 job 之外任何 entity 的 attributes，所以不需要 entity prefix。下面的写法等价：

>>> tq.tasks("state=error and spooltime > -24h", columns=["spooltime", "title"], sortby=["priority"])

Archives（归档历史数据）

archive= keyword parameter 可以设置为 True，以便搜索已删除 jobs 的 records（以及其关联的 tasks、commands 和 invocations）。

请注意 archives 可能比 live data set 大得多，因此 queries 在执行、传输和存入 memory 时可能会 非常 昂贵。

可以通过以下方式避免这种开销：

  • 使用明确指定的 search clauses，确保 result sets 较小，

  • 不要在 server 上对较大的 sets 排序，并且

  • 对可能很大的 result sets 使用 limit argument 且不进行 sorting。
     

Operations（操作函数）

query module 不仅有用于 fetch items 的 functions，还能够对这些 items 执行 operations。 务必小心 使用这些 operation functions，因为它们 非常 强大，并且可能 非常 非常 非常 具有破坏性。

operation 的第一个 argument 指定要操作哪些 items，可以是以下任意一种：

  • 一个 search clause 搜索条件子句，

  • query function 返回的 objects list 对象列表，

  • list function 返回的 list 中的单个 object，

  • 指定 operation 所需 attributes 的 dictionary，或

  • 指定 operation 所需 attributes 的 dictionaries list。

例如，下面所有写法都等价：

>>> tq.retry("jid=123 and tid=1")
>>> tq.retry(tq.tasks("jid=123 and tid=1"))
>>> tq.retry(tq.tasks("jid=123 and tid=1")[0])
>>> tq.retry({"jid": 123, "tid": 1})
>>> tq.retry([{"jid": 123, "tid": 1}])

最后两个 use cases 允许 authors 构造自己的 objects，用于标识要影响哪些 items。dictionary 中需要的 attributes 会根据被操作的 entity 而不同。Job operations 需要 jid；task operations 需要 jid 和 tid；command operations 需要 jid 和 cid；blade operations 需要 name。

下面的 statement 会 pause 所有带 errors 的 active jobs：

>>> tq.pause("active and error")

下面的 statement 会对 disk space 最少的十个 blades 执行 nimby：

>>> tq.nimby("up and not nimby", sortby=["availdisk"], limit=10)

某些 operations 有额外的 keyword arguments。例如，修改 job priority 时，会使用 prioriy= keyword parameter 指定 priority。下面的 statement 会把 george 的所有 jobs 的 priority 改为 500。

>>> tq.chpri("owner=george", priority=500)

Connectivity（连接管理设置）

Python module 会静默处理你的 engine connectivity，按 lazy 方式与 engine 建立 session。

可以使用 setEngineClientParam() 更改默认 engine hostname、port、connected user、password 和 debug boolean flag。默认 engine hostname 会是 TRACTOR_ENGINE environment variable 的值；如果未设置，则为 tractor-engine。默认 port 为 80。根据 EngineDiscovery 的配置方式，现有 engine 可能无需使用此 function 就会被自动发现。

默认 user 是 USER environment variable 的值；如果未设置，则为 root。默认 password 不设置。默认 debug flag 设置为 False。

默认 password 是 TRACTOR_PASSWORD environment variable 的值；否则不设置。

默认 debug flag 是 TRACTOR_DEBUG environment variable 的值；否则为 False。

此 example 会覆盖部分默认 connection values。务必在 之前 运行任何 list 或 operation functions 前完成此操作。

>>> tq.setEngineClientParam(hostname="test-engine", port=8000, user="donovan", debug=True)

如果你的 sites 使用 passwords，可以提示 user 输入 password，并按如下方式设置使用的值：

>>> the_password = getpass.getpass()  # the_password may be obtained by other means
>>> tq.setEngineClientParam(password=the_password)

可以通过 sessionFilename= parameter 把 session information 缓存在文件中，避免 user 每次运行 script 时都输入 password。file name 通常会按 application name、user name 和 host name 限定范围。needsPassword() 会被调用，以判断 cached session id 是否有效，或者是否需要提示 user 输入 password。下面是一个完整 example：

>>> import os
>>> import socket
>>> import getpass
>>> import tractor.api.query as tq
>>>
>>> home = os.path.expanduser("~")
>>> app = os.path.basename(__file__)
>>> user = getpass.getuser()
>>> host = socket.gethostname()
>>>
>>> # qualify the session filename by host if it will be stored in a location shared my multiple hosts
>>> sessionFilename = os.path.join(home, ".{app}.{user}.{host}.session".format(home=home, app=app, user=user, host=host))
>>>
>>> # set the session filename
>>> tq.setEngineClientParam(sessionFilename=sessionFilename)
>>>
>>> # check if the password needs to be obtained; this block will be skipped if the session filen contains a valid session id
>>> if tq.needsPassword():
>>>     password = getpass.getpass()
>>>     tq.setEngineClientParam(password=password)
>>>
>>> # the session file will be created when the first successful API call is made
>>> jobs = tq.jobs("error")
>>> print("There are {numjobs} jobs with errors.".format(numjobs=len(jobs)))

可以调用 closeEngineClient() 终止 session。这有助于减少 engine 维护的 sessions 数量。这会强制 session 不再有效，因此 app 下次运行时需要重新获取 password。

>>> tq.closeEngineClient()
