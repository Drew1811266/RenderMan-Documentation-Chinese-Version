查询 Python API

Source: https://rmanwiki-27.pixar.com/space/TRA/22184266/Query+Python+API
Downloaded: 2026-06-15T03:49:12+00:00

tractor.api.query 是一个 Python API，提供与命令行工具 tq 相同的查询和控制功能。它随 Tractor 自带的 Python 解释器 rmanpy 一起安装。

你也许可以把此模块与自己的解释器一起使用，但这需要站点特定配置，以便正确定位或安装 Tractor Python 模块。

下面的示例假定已按如下方式导入 查询模块：

import tractor.api.query as tq

列表函数

查询模块提供用于检索作业、任务、命令、调用记录和 Blade 信息的函数。它们的名称与命令行工具中的列表命令相同，即 jobs()、tasks()、commands()、invocations() 和 blades()。

这些函数的第一个参数是搜索子句，用于缩小搜索范围。

在此示例中，这个列表函数会取回一组用户拥有、且在 8 天以前提交的所有作业。列表函数的返回值是字典列表，每个字典表示一个匹配项。

>>> jobs = tq.jobs("owner in [freddie brian john roger] and spooltime < -8d")
>>> print jobs[0]
{u'comment': u'', u'maxslots': 0, u'numblocked': 0, u'spoolcwd': u'/home/freddie/unitard, u'numerror': 0, u'aftertime': None, u'elapsedsecs': 0.0, u'owner': u'paul', u'spoolhost': u'zanzibar, u'spooladdr': u'138.72.196.92', u'jid': 2680, u'service': u'', u'title': u'Somebody To Render', u'numactive': 0, u'editpolicy': u'', u'spooltime': u'2014-02-17 08:16:53-08', u'projects': [u'flash_gordon', u'hot_space'], u'crews': [u'queen'], u'maxcid': 2, u'metadata': u'', u'numready': 0, u'tags': [], u'afterjids': [], u'stoptime': u'2014-02-17 08:16:57.971-08', u'envkey': [], u'tier': u'', u'etalevel': 1, u'numtasks': 2, u'numdone': 2, u'maxtid': 2, u'assignments': u'', u'spoolfile': u'/home/freddie/champion.alf', u'esttotalsecs': 0.0, u'starttime': u'2014-02-17 08:16:53.73-08', u'pausetime': None, u'minslots': 0, u'deletetime': None, u'priority': 10.0}

搜索子句

搜索子句是一个布尔表达式（上面的示例使用 and），由比较、运算符（in、<）以及实体属性（owner、spooltime）组成。为减少输入量，这里做了一些简化。例如，freddie 这样的字符串字面量不需要加引号，而字符串 -8d 是表示“8 天前”的简写形式。

关于搜索子句语法的更多细节可在 这里找到。

注意，搜索范围越宽，在定位、传输和存储结果时消耗的 CPU、内存和带宽就越多。这会影响客户端、Engine 和后端数据库服务器。
 

列选择

默认情况下，查询函数会检索给定实体的所有属性。不过，如果并不需要所有属性，这会浪费客户端、Engine 和后端数据库的内存与 CPU。可以使用 columns= 关键字参数指定要检索的属性列表。

例如，下面只取回作业 ID、owner、title 和 metadata 列：

>>> tq.jobs("active", columns=["jid", "owner", "title", "metadata"])

排序

可以使用 sortby= 关键字参数对结果排序。可以指定多个属性以表示次级排序。条目默认按升序排列，除非属性前面加上 - 来表示降序。

例如，为了显示相似作业在 FIFO 队列中的处理顺序，可以按 priority 降序、spooltime 升序的顺序列出它们，如下所示：

>>> tq.jobs("active or ready", sortby=["-priority", "spooltime"])

虽然条目排序可以在客户端完成，但如果结果集已通过 limit= 关键字参数截断，就无法这样做，因为截断会在服务器端发生，早于客户端有机会对结果排序。

请注意，排序可能会根据查询的复杂度和范围，对后端数据库服务器产生影响。

限制返回数量

limit= 关键字参数会设置 Engine 返回条目数量的上限。

例如，下面的语句会取回渲染农场上活动任务数量最多的十个作业：

>>> tq.jobs("active", sortby=["-numactive"], limit=10)

下面的语句会列出磁盘空间最少的二十台主机。

>>> tq.blades("name like east", sortby=["availdisk"], limit=20)

请注意，把 limit 设置为 0 会对返回记录数量施加 NO LIMIT（无限制）。

注意，如果未使用 limit= 关键字参数，会有一个生效的系统默认上限。该上限为 2500，不过管理员可以通过在 tractor.config 中设置 SiteMaxListReplyCount 覆盖它。
 

关联实体属性

在上面的示例中，条目的属性用于搜索子句、列规格和排序。也可以在所有这些情况下指定关联实体的属性，方法是在属性前加上实体名称。例如，下面的语句会取回 walt 拥有的活动任务。

>>> tq.tasks("state=active and Job.owner=walt")

类似地，下面的命令会显示过去 24 小时内提交的作业中所有出错任务，包括其作业的标题和 spooltime，并按作业优先级排序。

>>> tq.tasks("state=error and Job.spooltime > -24h", columns=["Job.spooltime", "Job.title"], sortby=["Job.priority"])

因为 spooltime 和 priority 不是除作业之外任何实体的属性，所以不需要实体前缀。下面的写法等价：

>>> tq.tasks("state=error and spooltime > -24h", columns=["spooltime", "title"], sortby=["priority"])

归档历史数据

archive= 关键字参数可以设置为 True，以便搜索已删除作业的记录（以及其关联的任务、命令和调用记录）。

请注意，归档可能比实时数据集大得多，因此查询在执行、传输和存入内存时可能会 非常 昂贵。

可以通过以下方式避免这种开销：

  • 使用明确指定的搜索子句，确保结果集较小，

  • 不要在服务器上对较大的集合排序，并且

  • 对可能很大的结果集使用 limit 参数且不进行排序。
     

操作函数

查询模块不仅有用于取回条目的函数，还能够对这些条目执行操作。 务必小心 使用这些操作函数，因为它们 非常 强大，并且可能 非常 非常 非常 具有破坏性。

操作的第一个参数指定要操作哪些条目，可以是以下任意一种：

  • 一个搜索子句，

  • 查询函数返回的对象列表，

  • 列表函数返回的列表中的单个对象，

  • 指定操作所需属性的字典，或

  • 指定操作所需属性的字典列表。

例如，下面所有写法都等价：

>>> tq.retry("jid=123 and tid=1")
>>> tq.retry(tq.tasks("jid=123 and tid=1"))
>>> tq.retry(tq.tasks("jid=123 and tid=1")[0])
>>> tq.retry({"jid": 123, "tid": 1})
>>> tq.retry([{"jid": 123, "tid": 1}])

最后两个用例允许作者构造自己的对象，用于标识要影响哪些条目。字典中需要的属性会根据被操作的实体而不同。Job 操作需要 jid；task 操作需要 jid 和 tid；command 操作需要 jid 和 cid；blade 操作需要 name。

下面的语句会暂停所有带错误的活动作业：

>>> tq.pause("active and error")

下面的语句会对磁盘空间最少的十个 Blade 执行 nimby：

>>> tq.nimby("up and not nimby", sortby=["availdisk"], limit=10)

某些操作有额外的关键字参数。例如，修改作业优先级时，会使用 prioriy= 关键字参数指定 priority。下面的语句会把 george 的所有作业的优先级改为 500。

>>> tq.chpri("owner=george", priority=500)

连接管理

Python 模块会静默处理你的 Engine 连接，按惰性方式与 Engine 建立会话。

可以使用 setEngineClientParam() 更改默认 Engine 主机名、端口、已连接用户、密码和调试布尔标志。默认 Engine 主机名会是 TRACTOR_ENGINE 环境变量的值；如果未设置，则为 tractor-engine。默认端口为 80。根据 EngineDiscovery 的配置方式，现有 Engine 可能无需使用此函数就会被自动发现。

默认用户是 USER 环境变量的值；如果未设置，则为 root。默认密码不设置。默认调试标志设置为 False。

默认密码是 TRACTOR_PASSWORD 环境变量的值；否则不设置。

默认调试标志是 TRACTOR_DEBUG 环境变量的值；否则为 False。

此示例会覆盖部分默认连接值。务必在之前运行任何列表或操作函数前完成此操作。

>>> tq.setEngineClientParam(hostname="test-engine", port=8000, user="donovan", debug=True)

如果你的站点使用密码，可以提示用户输入密码，并按如下方式设置使用的值：

>>> the_password = getpass.getpass()  # the_password may be obtained by other means
>>> tq.setEngineClientParam(password=the_password)

可以通过 sessionFilename= 参数把会话信息缓存在文件中，避免用户每次运行脚本时都输入密码。文件名通常会按应用程序名称、用户名和主机名限定范围。needsPassword() 会被调用，以判断缓存的会话 ID 是否有效，或者是否需要提示用户输入密码。下面是一个完整示例：

>>> import os
>>> import socket
>>> import getpass
>>> import tractor.api.query as tq
>>>
>>> home = os.path.expanduser("~")
>>> app = os.path.basename(__file__)
>>> user = getpass.getuser()
>>> host = socket.gethostname()
>>>
>>> # qualify the session filename by host if it will be stored in a location shared my multiple hosts
>>> sessionFilename = os.path.join(home, ".{app}.{user}.{host}.session".format(home=home, app=app, user=user, host=host))
>>>
>>> # set the session filename
>>> tq.setEngineClientParam(sessionFilename=sessionFilename)
>>>
>>> # check if the password needs to be obtained; this block will be skipped if the session filen contains a valid session id
>>> if tq.needsPassword():
>>>     password = getpass.getpass()
>>>     tq.setEngineClientParam(password=password)
>>>
>>> # the session file will be created when the first successful API call is made
>>> jobs = tq.jobs("error")
>>> print("There are {numjobs} jobs with errors.".format(numjobs=len(jobs)))

可以调用 closeEngineClient() 终止会话。这有助于减少 Engine 维护的会话数量。这会强制会话不再有效，因此应用程序下次运行时需要重新获取密码。

>>> tq.closeEngineClient()
