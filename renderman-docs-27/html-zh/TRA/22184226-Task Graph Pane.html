Task Graph Pane

Source: https://rmanwiki-27.pixar.com/space/TRA/22184226/Task+Graph+Pane
Downloaded: 2026-06-15T03:49:12+00:00

Task Graph 面板以图形方式显示所选作业。作业树中的每个任务节点都表示为一个矩形。框会根据状态使用不同颜色：

绿色：活动

红色：错误

蓝色轮廓：已选中

浅灰色：完成

深灰色：等待中

进度条

如果应用程序以 RenderMan 格式或 Alfred 格式响应完成百分比，则会在活动的绿色条上绘制一条浅灰色条，用于表示正在运行命令的完成百分比。注意，对于包含多个命令的情况，此条可能会回到零，并循环多次。

日志输出指示器

如果正在运行的任务生成了日志输出，框左侧边缘附近会显示一个浅蓝色框。

预览图像指示器

如果任务定义包含预览或 chaser 命令，则任务框会在节点右侧边缘显示一个放大镜图标。

 
 

任务图特殊功能
 

选择

点击任何任务框，都会让 Task Info 面板显示所选任务的专属信息。该矩形会以浅蓝色轮廓绘制，表示它是当前选中的任务。
 

上下文菜单

在任务上右键点击或 Control-click，会在 Task Graph 窗口中靠近所选任务的位置弹出上下文相关菜单。此菜单可用于查看任务日志；在适用时，还会提供一个条目，允许在特定任务因错误退出后重启该任务。也可以跳过任务，使作业即使检测到错误也能继续执行。
 

日志

如果日志可用，双击会在覆盖 joblist 的新面板中显示任务日志，而不显示上下文菜单。
 

折叠作业图

父节点可以被折叠，以提供其下子节点的摘要视图。折叠后的节点用椭圆形表示，而不是方形节点；其颜色代表子节点的摘要。如果任何子节点处于错误状态，该节点会显示为红色；如果所有节点都已完成，则显示为灰色；如果节点处于活动状态，则显示为绿色。
