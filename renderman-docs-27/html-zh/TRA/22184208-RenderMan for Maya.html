简介（Introduction）

假设你的站点正好使用 Tractor 默认的、开箱即用的 configuration，那么 RenderMan for Maya（RfM）的默认 configuration 可以通过 Tractor 分发工作。如果你认为自己比较特殊，并且选择更改默认 configuration，也不用担心。RfM 通过 preferences 和其他相关设置提供所需的 hooks，让你的 RfM workstations 与 Tractor work dispatching system 保持协调配合。

RfM 自定义配置（RfM Custom Configuration）

RfM 的 Tractor configuration 通过 rfm.json 文件完成。对于 RfM 的 site configuration，我们建议不要修改随附的 rfm.json 文件。相反，站点应使用 environment variables RFM_SHOW_PATH 或 RFM_SITE_PATH， 来加载你的 custom overrides。更多信息请参阅 RenderMan for Maya 文档页面： Installing Custom Nodes。



Engine 位置和 Tractor 用户（Engine Location and Tractor Users）

rfm.json 文件在 "tractor_cfg" section 下提供三个 preferences，用于绑定 RfM 与 Tractor。默认值如下：


"tractor_cfg": {
    "engine": "tractor-engine", 
    "port": 80, 
    "user": ""
}


  • "engine" 是运行你站点 tractor engine 的机器 hostname。默认值为 "tractor-engine"。
  • "port" 是 tractor engine 正在监听的 port number。默认值为 80。
  • "user" 是提交 job 时使用的 username。默认值为 ""，表示以当前登录用户提交 job。

下面是一个完整 rfm.json 示例，它只覆盖 RfM 的 Tractor configuration settings，其余所有设置保持默认。

{
    "tractor_cfg": {
         "engine": "woody-engine", 
         "port": 8080, 
         "user": "andy"
   }
}



目录映射设置（Directory Mapping）

与设置 engine location 和 user 类似，directory mappings 也在 rfm.json 文件中指定。可以在 "dirmaps" section 下列出多个 directory mappings。默认值是一个通用 mapping：

    "dirmaps": {
        "myserver_to_z": {
            "to": "Z:/", 
            "from": "/myserver/", 
            "zone": "UNC"
        }
    },
  • "myserver_to_z" 是 dirmap 的通用描述名称。
  • "from" 是你想匹配并映射出去的 path 部分。
  • "to" 指定 "from" setting 中的 substring 应映射到什么位置。
  • "zone" 指定应使用此 dirmap 的 blade 所属 zone。

例如，使用这个 dirmapping 时，如果 RenderMan 和/或 Tractor 看到此 path，并且它运行在 "UNC" zone 中的 blade 上：

/myserver/path/to/maya/scenes/char_animation_01.ma

它会被转换为：

Z:/path/to/maya/scenes/char_animation_01.ma

下面是一个完整 rfm.json 示例，将前面的 configuration 与 custom directory mappings 结合起来。这里为 project 和 texture paths 提供从 Linux 到 Windows blade 的 mappings，也提供反向的 Windows 到 Linux mappings。

{
    "tractor_cfg": {
         "engine": "woody-engine", 
         "port": 8080, 
         "user": "andy"
   },
    "dirmaps": {
        "project_linux_to_windows": {
            "to": "Z:/", 
            "from": "/mnt/", 
            "zone": "UNC"
        },
        "textures_linux_to_windows": {
            "to": "Y:/", 
            "from": "/texture_server/", 
            "zone": "UNC"
        },
        "project_windows_to_linux": {
            "to": "/mnt/", 
            "from": "Z:/", 
            "zone": "NFS"
        },
        "textures_windows_to_linux": {
            "to": "/texture_server/", 
            "from": "Y:/", 
            "zone": "NFS"
        }
    }
}
