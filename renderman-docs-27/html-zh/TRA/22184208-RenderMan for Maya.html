简介

假设你的站点正好使用 Tractor 默认的开箱即用配置，那么 RenderMan for Maya（RfM）的默认配置即可通过 Tractor 分发工作。如果你认为自己比较特殊，并且选择更改默认配置，也不用担心。RfM 通过偏好设置和其他相关设置提供所需的钩子，让你的 RfM 工作站与 Tractor 工作分发系统保持协调配合。

RfM 自定义配置

RfM 的 Tractor 配置通过 rfm.json 文件完成。对于 RfM 的站点配置，我们建议不要修改随附的 rfm.json 文件。相反，站点应使用环境变量 RFM_SHOW_PATH 或 RFM_SITE_PATH， 来加载你的自定义覆盖配置。更多信息请参阅 RenderMan for Maya 文档页面： Installing Custom Nodes（安装自定义节点）。



引擎位置和 Tractor 用户

rfm.json 文件在 "tractor_cfg" 部分下提供三个偏好设置，用于绑定 RfM 与 Tractor。默认值如下：


"tractor_cfg": {
    "engine": "tractor-engine", 
    "port": 80, 
    "user": ""
}


  • "engine" 是运行你站点 tractor engine 的机器主机名。默认值为 "tractor-engine"。
  • "port" 是 tractor engine 正在监听的端口号。默认值为 80。
  • "user" 是提交作业时使用的用户名。默认值为 ""，表示以当前登录用户提交作业。

下面是一个完整 rfm.json 示例，它只覆盖 RfM 的 Tractor 配置设置，其余所有设置保持默认。

{
    "tractor_cfg": {
         "engine": "woody-engine", 
         "port": 8080, 
         "user": "andy"
   }
}



目录映射

与设置引擎位置和用户类似，目录映射也在 rfm.json 文件中指定。可以在 "dirmaps" 部分下列出多个目录映射。默认值是一个通用映射：

    "dirmaps": {
        "myserver_to_z": {
            "to": "Z:/", 
            "from": "/myserver/", 
            "zone": "UNC"
        }
    },
  • "myserver_to_z" 是 dirmap 的通用描述名称。
  • "from" 是你想匹配并映射出去的路径部分。
  • "to" 指定 "from" 设置中的子字符串应映射到什么位置。
  • "zone" 指定应使用此 dirmap 的 blade 所属区域。

例如，使用这个目录映射时，如果 RenderMan 和/或 Tractor 看到此路径，并且它运行在 "UNC" 区域中的 blade 上：

/myserver/path/to/maya/scenes/char_animation_01.ma

它会被转换为：

Z:/path/to/maya/scenes/char_animation_01.ma

下面是一个完整 rfm.json 示例，将前面的配置与自定义目录映射结合起来。这里为项目路径和纹理路径提供从 Linux 到 Windows blade 的映射，也提供反向的 Windows 到 Linux 映射。

{
    "tractor_cfg": {
         "engine": "woody-engine", 
         "port": 8080, 
         "user": "andy"
   },
    "dirmaps": {
        "project_linux_to_windows": {
            "to": "Z:/", 
            "from": "/mnt/", 
            "zone": "UNC"
        },
        "textures_linux_to_windows": {
            "to": "Y:/", 
            "from": "/texture_server/", 
            "zone": "UNC"
        },
        "project_windows_to_linux": {
            "to": "/mnt/", 
            "from": "Z:/", 
            "zone": "NFS"
        },
        "textures_windows_to_linux": {
            "to": "/texture_server/", 
            "from": "Y:/", 
            "zone": "NFS"
        }
    }
}
