Setting Up Services

Source: https://rmanwiki-27.pixar.com/space/TRA/22184328/Setting+Up+Services
Downloaded: 2026-06-15T03:49:12+00:00

将 tractor-engine 和 tractor-blade 设置为 system services 很有用，这样它们会在 boot time 自动启动。这也会简化从 command line 启动和停止它们。Instructions 因 OS 而异。

Linux - with systemd

所需 systemd files 位于 INSTALL_DIR/lib/SystemServices/systemd。

要把 engine 设置为 systemd service 系统服务：

  1. 安装 engine unit file 单元文件。
    sudo cp tractor-engine.service /usr/lib/systemd/system

  2. 安装 engine settings file。编辑此文件以自定义 Tractor installation directory、configuration directory，或 tractor-engine 的 command-line options。
    sudo cp tractor-engine /etc/sysconfig/tractor-engine 在 systemd 中启用 engine service 引擎服务：
    sudo systemctl enable tractor-engine

  3. Start/stop 启动或停止 engine：
    sudo systemctl start tractor-engine
    sudo systemctl stop tractor-engine

要把 blade 设置为 systemd service 系统服务：

  1. 安装 blade unit file 单元文件。
    sudo cp tractor-blade.service /usr/lib/systemd/system

  2. 安装 systemd override file。编辑此文件以 override blade process 的 owner。
    sudo mkdir /etc/systemd/system/tractor-blade.service.d && sudo cp 90-tractor-blade-overrides.conf /etc/systemd/system/tractor-blade.service.d

  3. 安装 blade settings file。编辑此文件以自定义 Tractor installation directory 或 tractor-blade 的 command-line options。
    sudo cp tractor-blade /etc/sysconfig/tractor-blade

  4. 在 systemd 中启用 blade service 服务：
    sudo systemctl enable tractor-blade

  5. Start/stop 启动或停止 blade：
    sudo systemctl start tractor-blade
    sudo systemctl stop tractor-blade

Engine 和 blade 的 systemd services 配置为使用 journald logging。例如，可使用以下命令查看今天的 engine log：

sudo journalctl --since today -u tractor-engine

或者 follow 一个 blade 的 log：

sudo journalctl -flu tractor-blade

有关管理 systemd services 的更多 information，请参阅 systemd documentation。

对于较旧的 Linux installs，请遵循下面这些 directions。

Linux - with sysVinit

所需 sysVinit files 位于 INSTALL_DIR/lib/SystemServices。

要把 engine 设置为 sysVinit service 系统服务：

  1. 安装 pixar settings file。编辑此文件以自定义 Tractor installation directory、configuration directory，或 tractor-engine 的 command-line options。如果此文件已存在，你需要把随附文件底部的 Tractor entries 手动添加到现有 /etc/sysconfig/pixar 文件中。

    sudo cp pixar /etc/sysconfig

  2. 安装 engine init.d file 初始化脚本文件。

    sudo cp tractor-engine /etc/init.d

  3. 启用 engine service，使其在 boot 时启动：

    sudo chkconfig tractor-engine on

  4. Start/stop 启动或停止 engine：

    sudo /etc/init.d/tractor-engine start

    sudo /etc/init.d/tractor-engine stop

要把 blade 设置为 sysVinit service 系统服务：

  1. 安装 pixar settings file。编辑此文件以自定义 Tractor installation directory、tractor-blade 的 command-line options，或 blade process 的 owner。如果此文件已存在，你需要把随附文件底部的 Tractor entries 手动添加到现有 /etc/sysconfig/pixar 文件中。

    sudo cp pixar /etc/sysconfig

  2. 安装 blade init.d file 初始化脚本文件。

    sudo cp tractor-blade /etc/init.d

  3. 启用 blade service 服务项目：

    sudo chkconfig tractor-blade on

  4. Start/stop 启动或停止 blade：

    sudo /etc/init.d/tractor-blade start

    sudo /etc/init.d/tractor-blade stop

Windows

Windows 上的 Tractor blade installer 会自动把 blade 安装并启动为 Windows service 系统服务项目。 该 service 也配置为每次 computer restarted 时自动启动 blade。你可以按以下 steps 查看 blade service entry：

  1. Start 启动 command prompt window：点击

    Windows 中的 Start button，然后选择

    All Programs -> Accessories -> Command Prompt

  2. 在 command prompt window 命令提示符窗口中输入：

    services.msc

    然后按 Enter key 打开 Windows Services Management Console。

  3. 在 Services window 中，向下滚动到 Pixar Tractor Blade Service service。

可以通过右键单击该 service 并选择 Start、Stop 或 Restart 来管理它。请注意，以前的 installations 可能留下多个 Pixar Tractor Blade Service entries。启动/停止或进行更改时，请确保选择最新版本。

建议修改 service 将以其身份运行的 user account。可通过右键单击 service 并选择 Properties 完成。在 Properties window 的 Log On tab 中，可以把 service 设置为以特定 user account 运行，而不是 Local System。Blade 运行的 commands 应以有权访问成功运行 commands 所需 remote drives 和 resources 的 user account 运行。

Note（注意） 如果安装的是同一版本 Tractor blade 的 updated build，建议先卸载 previous build。Uninstaller 会自动停止 existing service。如果 service 设置为以特定 user 运行（如建议所述），请记得在 Services Management Console 中重新输入该信息。

在 advanced installation 中，可能需要 blade service 的 command line parameters。Windows Service snap-in Properties dialog 中的 start parameters 对此没有用，因为在那里设置时，它们只适用于从该 dialog 手动启动 service 的单次运行。要添加 command line arguments，请改为编辑以下 path 下的 registry settings HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\  并修改已安装 blade 的 ImagePath entry，在 .exe path 后追加 command line options。可用 command line options 列在 Application - tractor-blade 页面上。关于谨慎修改 Windows Registry 的常规警告同样适用，因为它可能造成不可修复的 damage。

macOS

Engine 和 blade 的初始 .plist files 位于 INSTALL_DIR/lib/SystemServices。

  • com.pixar.tractor-engine.plist

  • com.pixar.tractor-blade.plist

编辑这些 files 以自定义 installation。

  1. 在你选择的 text editor 中打开相应 file，并调整 install location、launching user 和任何 program arguments。

  2. 按需将 files 保存到 /Library/LaunchDaemons

  3. 要启动 tractor-engine，请运行：

    sudo launchctl load -w /Library/LaunchDaemons/com.pixar.tractor-engine.plist

  4. 要启动 tractor-blade，请运行：

    sudo launchctl load -w /Library/LaunchDaemons/com.pixar.tractor-blade.plist

  5. 有时 .plist files 的 ownership 和 permission settings 可能在 editing process 中被错误覆盖。要确保这些 files 正确，请运行：

    sudo chown root /Library/LaunchDaemons/com.pixar.tractor*.plist

    sudo chmod 644  /Library/LaunchDaemons/com.pixar.tractor*.plist

Services 现在会在 reboot 时启动。

为 Blade Services 服务配置 Firewalls 防火墙

除了 anticipated jobs 可能需要的任何 network resources access 之外，blade service 尤其需要允许来自 engine host 的 access。在 Tractor system 中，大多数情况下 network requests 从 blade 发往 engine，并且大多数默认 OS installations 会允许 blade 的这种 outbound traffic。当然，engine host 必须配置为允许这些请求。不过，engine 会周期性地尝试专门连接 blades，以收集有关 blade health 的 information。如 Configuration 页面所列，这要求为 blade service 打开 port 9005。请查阅你所使用 firewall software（如果有）的 documentation，以打开该 port。
