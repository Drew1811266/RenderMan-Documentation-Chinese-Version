设置服务

Source: https://rmanwiki-27.pixar.com/space/TRA/22184328/Setting+Up+Services
Downloaded: 2026-06-15T03:49:12+00:00

将 tractor-engine 和 tractor-blade 设置为 系统服务 很有用，这样它们会在启动时自动启动。这也会简化从命令行启动和停止它们。具体说明因操作系统而异。

Linux - 使用 systemd

所需 systemd 文件位于 INSTALL_DIR/lib/SystemServices/systemd。

要把 engine 设置为 systemd 系统服务：

  1. 安装 Engine 单元文件。
    sudo cp tractor-engine.service /usr/lib/systemd/system

  2. 安装 Engine 设置文件。编辑此文件以自定义 Tractor 安装目录、配置目录，或 tractor-engine 的命令行选项。
    sudo cp tractor-engine /etc/sysconfig/tractor-engine 在 systemd 中启用 Engine 服务：
    sudo systemctl enable tractor-engine

  3. 启动或停止 Engine：
    sudo systemctl start tractor-engine
    sudo systemctl stop tractor-engine

要把 blade 设置为 systemd 系统服务：

  1. 安装 Blade 单元文件。
    sudo cp tractor-blade.service /usr/lib/systemd/system

  2. 安装 systemd 覆盖文件。编辑此文件以覆盖 Blade 进程的所有者。
    sudo mkdir /etc/systemd/system/tractor-blade.service.d && sudo cp 90-tractor-blade-overrides.conf /etc/systemd/system/tractor-blade.service.d

  3. 安装 Blade 设置文件。编辑此文件以自定义 Tractor 安装目录或 tractor-blade 的命令行选项。
    sudo cp tractor-blade /etc/sysconfig/tractor-blade

  4. 在 systemd 中启用 Blade 服务：
    sudo systemctl enable tractor-blade

  5. 启动或停止 Blade：
    sudo systemctl start tractor-blade
    sudo systemctl stop tractor-blade

Engine 和 Blade 的 systemd 服务配置为使用 journald 日志。例如，可使用以下命令查看今天的 Engine 日志：

sudo journalctl --since today -u tractor-engine

或者跟踪一个 Blade 的日志：

sudo journalctl -flu tractor-blade

有关管理 systemd 服务的更多信息，请参阅 systemd 文档。

对于较旧的 Linux 安装，请遵循下面这些说明。

Linux - 使用 sysVinit

所需 sysVinit 文件位于 INSTALL_DIR/lib/SystemServices。

要把 engine 设置为 sysVinit 系统服务：

  1. 安装 pixar 设置文件。编辑此文件以自定义 Tractor 安装目录、配置目录，或 tractor-engine 的命令行选项。如果此文件已存在，你需要把随附文件底部的 Tractor 条目手动添加到现有 /etc/sysconfig/pixar 文件中。

    sudo cp pixar /etc/sysconfig

  2. 安装 Engine init.d 文件。

    sudo cp tractor-engine /etc/init.d

  3. 启用 Engine 服务，使其在启动时运行：

    sudo chkconfig tractor-engine on

  4. 启动或停止 Engine：

    sudo /etc/init.d/tractor-engine start

    sudo /etc/init.d/tractor-engine stop

要把 blade 设置为 sysVinit 系统服务：

  1. 安装 pixar 设置文件。编辑此文件以自定义 Tractor 安装目录、tractor-blade 的命令行选项，或 Blade 进程的所有者。如果此文件已存在，你需要把随附文件底部的 Tractor 条目手动添加到现有 /etc/sysconfig/pixar 文件中。

    sudo cp pixar /etc/sysconfig

  2. 安装 Blade init.d 文件。

    sudo cp tractor-blade /etc/init.d

  3. 启用 Blade 服务：

    sudo chkconfig tractor-blade on

  4. 启动或停止 Blade：

    sudo /etc/init.d/tractor-blade start

    sudo /etc/init.d/tractor-blade stop

Windows

Windows 上的 Tractor Blade 安装程序会自动把 Blade 安装并启动为 Windows 服务。 该服务也配置为每次计算机重启时自动启动 Blade。你可以按以下步骤查看 Blade 服务条目：

  1. 启动命令提示符窗口：点击

    Windows 中的 Start 按钮，然后选择

    All Programs -> Accessories -> Command Prompt

  2. 在命令提示符窗口中输入：

    services.msc

    然后按 Enter 键打开 Windows Services Management Console。

  3. 在 Services 窗口中，向下滚动到 Pixar Tractor Blade Service 服务。

可以通过右键单击该服务并选择 Start、Stop 或 Restart 来管理它。请注意，以前的安装可能留下多个 Pixar Tractor Blade Service 条目。启动、停止或进行更改时，请确保选择最新版本。

建议修改服务运行所使用的用户账户。可通过右键单击服务并选择 Properties 完成。在 Properties 窗口的 Log On 标签页中，可以把服务设置为以特定用户账户运行，而不是 Local System。Blade 运行的命令应以有权访问成功运行这些命令所需远程驱动器和资源的用户账户运行。

注意 如果安装的是同一版本 Tractor Blade 的更新构建，建议先卸载之前的构建。卸载程序会自动停止现有服务。如果服务设置为以特定用户运行（如建议所述），请记得在 Services Management Console 中重新输入该信息。

在高级安装中，可能需要 Blade 服务的命令行参数。Windows Service 管理单元的 Properties 对话框中的 start parameters 对此没有用，因为在那里设置时，它们只适用于从该对话框手动启动服务的单次运行。要添加命令行参数，请改为编辑以下路径下的注册表设置 HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\  并修改已安装 Blade 的 ImagePath 条目，在 .exe 路径后追加命令行选项。可用命令行选项列在 Application - tractor-blade 页面上。关于谨慎修改 Windows Registry 的常规警告同样适用，因为它可能造成不可修复的损坏。

macOS

Engine 和 Blade 的初始 .plist 文件位于 INSTALL_DIR/lib/SystemServices。

  • com.pixar.tractor-engine.plist

  • com.pixar.tractor-blade.plist

编辑这些文件以自定义安装。

  1. 在你选择的文本编辑器中打开相应文件，并调整安装位置、启动用户和任何程序参数。

  2. 按需将文件保存到 /Library/LaunchDaemons

  3. 要启动 tractor-engine，请运行：

    sudo launchctl load -w /Library/LaunchDaemons/com.pixar.tractor-engine.plist

  4. 要启动 tractor-blade，请运行：

    sudo launchctl load -w /Library/LaunchDaemons/com.pixar.tractor-blade.plist

  5. 有时 .plist 文件的所有权和权限设置可能在编辑过程中被错误覆盖。要确保这些文件正确，请运行：

    sudo chown root /Library/LaunchDaemons/com.pixar.tractor*.plist

    sudo chmod 644  /Library/LaunchDaemons/com.pixar.tractor*.plist

服务现在会在重启时启动。

为 Blade 服务配置防火墙

除了预期作业可能需要访问的任何网络资源之外，Blade 服务尤其需要允许来自 Engine 主机的访问。在 Tractor 系统中，大多数情况下网络请求从 Blade 发往 Engine，并且大多数默认操作系统安装会允许 Blade 的这种出站流量。当然，Engine 主机必须配置为允许这些请求。不过，Engine 会周期性地尝试专门连接 Blade，以收集有关 Blade 健康状态的信息。如 Configuration 页面所列，这要求为 Blade 服务打开端口 9005。请查阅你所使用防火墙软件（如果有）的文档，以打开该端口。
