Blade Info Pane

Source: https://rmanwiki-27.pixar.com/space/TRA/22184224/Blade+Info+Pane
Downloaded: 2026-06-15T03:49:12+00:00

Blade Info pane 显示在 Blade List pane 中选中的 blade 的专属信息。

显示的信息由 engine Engine 和 blade 跟踪的 blade status 组成。如果 blade report 中没有信息，则与 blade 本身的连接可能失败，并可能表明 blade process 已停机或卡死。在这种情况下，你很可能会在 Blade List pane 中看到该 blade 的 "No Pulse" 消息。
 

环境

报告的 environment 是默认 environment。每个正在运行的 command 实际使用的 environment 还取决于 job definition 中指定的 envhandler。
 

丢失的命令

Engine 与 blade status information 的组合在 Commands 属性中尤其值得注意。这里会显示所选 blade 上每个 active command 的列表。

如果某个 command 正在 blade 上运行，但 Engine 不知道它，则该 entry 会显示为红色并标记为 "LOST"。这种情况可能发生在 Engine 试图终止 blade 的 active command，但 blade 无法接收 kill message，或无法终止该 command 的 process 时。

相反，如果 Engine 认为某个 blade 正在运行一个 command，而 blade 并不知道它，则该 command 的 PID 会显示为 "unknown"。如果 blade 无法启动分配给它的 command，或 engine 没有收到 blade 发来的 command 已完成消息，就可能出现这种情况。

注意，Engine 会定期对每个 blade 执行 validation test，以确保它正在运行预期的 command。
 

Blade Notes

blade 属性右侧的 Notes box 会显示所选 blade 的用户评论。如果用户有权限向 blade 添加 note，则 notes 区域下方会出现 "click to add note"。可以拖动 notes box 的左下角来调整 notes 区域大小。

用户生成的 note 可作为 wrangler 和 admin 之间的沟通渠道，用于标注 host 的 service history 和 configuration change。
