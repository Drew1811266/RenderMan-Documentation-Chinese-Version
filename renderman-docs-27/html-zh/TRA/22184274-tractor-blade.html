tractor-blade

Source: https://rmanwiki-27.pixar.com/space/TRA/22184274/tractor-blade
Downloaded: 2026-06-15T03:49:12+00:00

Tractor Blade 是运行在每台渲染农场主机上的远程执行服务器进程。每台主机上的 tractor-blade 进程会跟踪基本的系统健康指标以及先前启动过的 Tractor 任务。它会判断自身何时具备接收新工作的容量，然后从中央作业队列获取一个任务定义并启动指定命令。随后 Tractor Blade 会跟踪每个已启动应用程序的状态；可能有多个应用程序同时运行，并上报状态与结果。Tractor 系统要求每台计算服务器上都运行 tractor-blade，才能把工作分发到农场。用法：tractor-blade [options]

选项：

选项说明
--version显示程序版本信息，然后退出。
-h, --help显示用法摘要，然后退出。
--engine=HOST[:port]tractor-engine 运行所在的 hostname[:port]，默认值为 tractor-engine，通常是一个 DNS 别名；默认端口为 80。如果未给出此选项，Blade 还会查找名为 TRACTOR_ENGINE 的环境变量，用作 host:port 说明。如果显式设置和环境变量都不存在，Blade 会尝试通过本地名称服务器解析主机名 "tractor-engine"。如果该名称 未 注册，Blade 就会在 LAN 上发出 引擎发现 广播，以搜索可用的 Engine。
-L BPORT此 Blade 监听管理员查询的端口，默认值为 9005；使用 0（zero）可让操作系统选择一个空闲端口。Blade 会把给定端口注册到 Engine。注意，该端口必须能穿过 Blade 主机上可能存在的防火墙，因此在使用防火墙时，像 9005 这样的固定值比随机值更合适。可以使用 '--listen=addr:port' 选择指定的本地网络接口。
--hname=HOSTNAMEBlade 主机的备用名称，会注册到 Engine，也会发送给可能需要反向连接的对等端。默认值是 Engine 看到的 Blade 地址经反向查询得出的主机名。使用特殊形式 --hname=. 可让 Blade 只使用自身本地得出的主机名。
-v, --verbose记录 INFO 及以上日志级别。
--debug记录 DEBUG 及以上日志级别。
--trace记录 TRACE 及以上日志级别。
--warning记录 WARNING 及以上日志级别，这是默认值。
-q, --quiet只记录 CRITICAL 日志级别。
--log=LOGFILEtractor-blade 活动日志文件，达到 25MB 时自动轮转。
--logrotate=NNN指定日志轮转阈值，单位为 MB。
--logconfig=LOGCONFIGPython 的外部配置文件 logging-module，Blade 内部使用它记录自身状态。注意，此 Blade 启动的每个 Cmd 的输出会按照 CmdOutputLogging设置处理，该设置位于 blade.config 中，通常会写入一个网络共享。
--logenv记录 Blade 启动的每条命令的实际命令行和环境。
--cmdtee把每条已启动命令的输出复制到 Blade 自身的日志，同时也写入每个任务的日志；通常只在调试 Blade 动作时使用。
--zone=ZONE用于 'dirmap' 索引的 zone 名称。
--nimby=NAME限制此 Blade 将执行的作业；给出用户名表示只接受该用户的作业，或给出 '1' 表示接受从此 Blade 主机提交出来的任何作业。
--slots=SLOTS最大并发 slot 数；覆盖 blade.config 中通常的 Capacity 参数 MaxSlots；使用 '0' 可把 slot 数设置为系统 CPU 核心数。
--daemon（仅 Unix）让 Blade 进程脱离控制终端会话，并自动在后台运行，从而作为系统守护进程执行。
--supersede允许新启动的 Blade 进程平滑接管仍在运行的旧 Blade。如果同一端口上仍有旧 Blade，新 Blade 会向它发送 "drain and exit" 消息，然后等待旧 Blade 退出，再继续成为新的 Blade 服务。在 drain mode 下，旧 Blade 会停止从 Engine 请求新工作，并在它先前启动的所有任务完成后退出。
--profile=NAME指定 blade.config 中要使用的 Blade profile name，跳过通常的匹配条件；仅用于测试场景。
--env=ENVFILE应用程序覆盖环境设置的文件名（例如从 printenv 捕获）。注意：环境设置通常通过 blade.config 中具名的 EnvKeys 集合定义。
--pythonhome=PYTHONHOME为 Tractor 启动的 Python 程序设置 PYTHONHOME。
--ld_library_path=LDPATH为 Tractor 启动的程序设置 LD_LIBRARY_PATH。
--dyld_framework_path=PATH为 Tractor 启动的程序设置 LD_FRAMEWORK_PATH。
--no-sigint中断活动任务时，先用 SIGTERM 而不是 SIGINT 发起任务终止尝试（仅 Unix）。
--killdelay=SECS清扫活动子进程时，从 SIGINT 升级到 SIGTERM 再到 SIGKILL 之间等待的秒数。
--progresslimit=NSECS限制完成百分比更新的发送频率，两次发送之间至少等待 NSECS 秒。
--startupdelay=SECS服务启动之后、继续执行 Blade 启动动作之前的延迟。
--minsleep=MINSLEEP最小空闲休眠间隔，单位为秒。
--maxsleep=MAXSLEEP最大空闲休眠间隔，单位为秒。
