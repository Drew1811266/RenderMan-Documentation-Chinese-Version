tractor-blade

Source: https://rmanwiki-27.pixar.com/space/TRA/22184274/tractor-blade
Downloaded: 2026-06-15T03:49:12+00:00

Tractor Blade 是运行在每台 renderfarm 主机上的远程执行服务器进程。每台主机上的 tractor-blade 进程会跟踪基本的系统健康指标以及先前启动过的 Tractor tasks。它会判断自身何时具备接收新工作的 capacity，然后从中央 job queue 获取一个 task definition 并启动指定 commands。随后 Tractor Blade 会跟踪每个已启动 app 的状态；可能有多个 app 同时运行，并上报状态与结果。Tractor 系统要求每台 compute server 上都运行 tractor-blade，才能把工作分发到 farm。Usage: tractor-blade [options]

Options:

Option说明
--version显示程序版本信息，然后退出。
-h, --help显示 usage 摘要，然后退出。
--engine=HOST[:port]tractor-engine 运行所在的 hostname[:port]，默认值为 tractor-engine，通常是一个 DNS alias；默认 port 为 80。如果未给出此选项，blade 还会查找名为 TRACTOR_ENGINE 的 environment variable，用作 host:port 说明。如果显式设置和环境变量都不存在，blade 会尝试通过本地 nameserver 解析 hostname "tractor-engine"。如果该名称 未 注册，blade 就会在 LAN 上发出 engine discovery 广播，以搜索可用的 engine。
-L BPORT此 blade 监听 admin queries 的 port，默认值为 9005；使用 0（zero）可让 operating system 选择一个空闲 port。blade 会把给定 port 注册到 engine。注意，该 port 必须能穿过 blade host 上可能存在的 firewall，因此在使用 firewalls 时，像 9005 这样的固定值比随机值更合适。可以使用 '--listen=addr:port' 选择指定的本地 network interface。
--hname=HOSTNAMEblade host 的备用名称，会注册到 engine，也会发送给可能需要反向连接的 peers。默认值是 engine 看到的 blade 地址经 reverse look up 得出的 hostname。使用特殊形式 --hname=. 可让 blade 只使用自身本地得出的 hostname。
-v, --verbose记录 INFO 及以上 log level。
--debug记录 DEBUG 及以上 log level。
--trace记录 TRACE 及以上 log level。
--warning记录 WARNING 及以上 log level，这是默认值。
-q, --quiet只记录 CRITICAL log level。
--log=LOGFILEtractor-blade activity log file，达到 25MB 时自动轮转。
--logrotate=NNN指定 log-rotation threshold，单位为 megabytes。
--logconfig=LOGCONFIGPython 的外部配置文件 logging-module，blade 内部使用它记录自身状态。注意，此 blade 启动的每个 Cmd 的输出会按照 CmdOutputLogging设置处理，该设置位于 blade.config 中，通常会写入一个 network share。
--logenv记录 blade 启动的每条 command 的实际 command line 和 environment。
--cmdtee把每条已启动 command 的输出复制到 blade 自身的 log，同时也写入每个 task 的 logs；通常只在调试 blade actions 时使用。
--zone=ZONE用于 'dirmap' indexing 的 zone name。
--nimby=NAME限制此 blade 将执行的 jobs；给出 user name 表示只接受该用户的 jobs，或给出 '1' 表示接受从此 blade host spool 出来的任何 job。
--slots=SLOTS最大并发 slot 数；覆盖 blade.config 中通常的 Capacity parameter MaxSlots；使用 '0' 可把 slot count 设置为系统 CPU cores 的数量。
--daemon（仅 unix）让 blade process 脱离 controlling terminal session，并自动在后台运行，从而作为 system daemon 执行。
--supersede允许新启动的 blade process 平滑接管仍在运行的旧 blade。如果同一 port 上仍有旧 blade，新 blade 会向它发送 "drain and exit" 消息，然后等待旧 blade 退出，再继续成为新的 blade service。在 drain mode 下，旧 blade 会停止从 engine 请求新工作，并在它先前启动的所有 tasks 完成后退出。
--profile=NAME指定 blade.config 中要使用的 blade profile name，跳过通常的匹配条件；仅用于 testing scenarios。
--env=ENVFILE应用程序 override environment settings 的文件名（例如从 printenv 捕获）。注意：environment settings 通常通过 blade.config 中具名的 EnvKeys collections 定义。
--pythonhome=PYTHONHOME为 Tractor 启动的 Python programs 设置 PYTHONHOME。
--ld_library_path=LDPATH为 Tractor 启动的 programs 设置 LD_LIBRARY_PATH。
--dyld_framework_path=PATH为 Tractor 启动的 programs 设置 LD_FRAMEWORK_PATH。
--no-sigint中断 active tasks 时，先用 SIGTERM 而不是 SIGINT 发起 task termination attempts（仅 unix）。
--killdelay=SECS清扫 active subprocesses 时，从 SIGNINT 升级到 SIGTERM 再到 SIGKILL 之间等待的秒数。
--progresslimit=NSECS限制 percent-done updates 的发送频率，两次发送之间至少等待 NSECS seconds。
--startupdelay=SECSservice boot 之后、继续执行 blade startup actions 之前的 delay。
--minsleep=MINSLEEP最小 idle sleep interval，单位为 seconds。
--maxsleep=MAXSLEEP最大 idle sleep interval，单位为 seconds。
