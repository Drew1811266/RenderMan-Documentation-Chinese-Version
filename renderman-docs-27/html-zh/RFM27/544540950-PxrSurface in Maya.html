PxrSurface in Maya

Source: https://rmanwiki-27.pixar.com/space/RFM27/544540950/PxrSurface+in+Maya
Downloaded: 2026-06-15T03:49:12+00:00

在 Maya 中，可通过选择对象并点击按钮栏中的 PxrSurface 图标，将 PxrSurface BXDF 应用于该对象。也可以在 RenderMan Menu（RenderMan 菜单）的 Materials 标题下选择场景中的对象来应用 PxrSurface。

下图展示了可通过此 BXDF 修改的一小部分属性。

Globals 中描述的 Pattern Utility 参数不会显示在 Attribute Editor 中，因为 Maya 对数组的处理方式如此。你可以改为在 Hypershade 内的 Node Editor 中建立连接来使用它。示例包括使用 Maya 中的 PxrSurface 为每个材质实例创建 ID 通道，或将 PxrSurface 与另一个带纹理的 pattern 组合，创建用于合成的其他有用 AOV，例如噪声等。请注意，这样做可能会增加已指定材质的求值时间。


《光年》（Lightyear）© Disney/Pixar

在 RenderMan 中渲染表面依赖一种名为 PxrSurface 的可分层材质（见下方 Input Material）。 

这是 Pixar Animation Studios 长片动画中使用的同一套材质模型。因此，该材质有一些重要优势：

  • 单一材质即可处理所有外观，例如：

    • 木材

    • 皮肤

    • 塑料

    • 玻璃

    • 车漆

    • 以及更多更多！

  • 提供基于物理的预设，帮助你开始。

  • 无需牺牲灵活控件，也能得到物理可信的结果。

    • 艺术家可以有选择地“打破”能量守恒。

    • 艺术指导可以自然完成，无需变通方案。

    • 控件会方便地标记为 "Artistic" 和 "Physical"

  • Layering（分层）提供无限可能。

作为艺术家，能够为所有着色选择单一材质，可以消除猜测，并让 RenderMan 按需优化。我们会讨论所有不同设置，以及它们与制作特定材质的关系。请特别注意关于性能考量的说明，以及哪些组合对特定效果有用的提示。

下面会看到材质的每种效果或 波瓣 都被拆分到独立章节，以帮助用户定义希望学习的区域。不过，随着不同波瓣被启用和调整，材质的外观会发生变化。正是通过组合这些波瓣设置，你几乎可以创建任何想象中的材质。


  • Diffuse Parameters（漫反射参数）
  • Specular Parameters（镜面参数）
  • Clear Coat Parameters（清漆参数）
  • Glass Parameters（玻璃参数）
  • Iridescence Parameters（虹彩参数）
  • Fuzz Parameters（绒毛参数）
  • Subsurface Scattering Parameters（次表面散射参数）
  • Single Scatter Parameters（单次散射参数）
  • Glow Parameters（发光参数）



Input Material（输入材质）

连接到一个层 pattern，用于对 Bxdf 的参数进行分层。这让用户能够通过分层效果创建更复杂的外观。示例可能包括灰尘、划痕，甚至标签。

在下面的参数中，有些可以由 PxrLayer 覆盖；当它连接到 Input Material 或通过 PxrLayerMixer连接时即可生效。 PxrLayerSurface 的设计意图是只包含全局参数，从而更清楚地显示哪些参数无法在层中被覆盖。当你确定会使用分层时，我们建议使用此材质。这些设置的结果不会改变。

  • Bold Face（粗体） 参数是可分层的，可以按层覆盖。

  • Italicized（斜体） 参数无法分层或覆盖。这些参数会在所有层中作为全局设置生效。例如：选择 GGX 作为镜面模型意味着所有层的该参数都会使用 GGX。


Properties Parameters（属性参数）

本节提供整个材质的全局控制。例如，如果只在此处提供 Bump，则它会应用到上方所有波瓣。Presence 也是全局参数，会遮罩或“镂空”所有波瓣，就像在遮罩值为 0.0 的位置对象不存在一样。

Bump

除非被单个波瓣的 bump normal 覆盖，否则用于所有波瓣的 Normal/Bump。

Presence

在此连接遮罩，可将镂空 pattern 应用到对象。这对创建树叶和其他薄而复杂形状等镂空很有用。对于灰度值，当直接由相机看到时，它也可以作为不透明度用于半透明结果。

  • 对于非薄玻璃折射和 “thick” 风格对象，此项会被忽略；在这些对象中，颜色是穿过内部的衰减

Presence Cached

指定 Presence 是否缓存。启用缓存时，Presence 将按 per micropolygon 求值，而不是在关闭缓存时按 per shading point 求值。 

  • 由于 Presence 计算可能很昂贵，如果存在重型着色网络，例如纹理，则缓存有利于提升渲染速度，但会增加内存。

  • 因为缓存本质上存储在细分镶嵌级别，如果需要更精细的渲染细节，建议使用未缓存 Presence，因为它按 per shading point 求值。

  • 内存与速度之间存在权衡。如果 Presence 非常简单，例如常量值，缓存和未缓存可能几乎同样快，并能节省一些内存。请注意，如果 Presence 为常量 1.0（默认值），PxrSurface 会优化所有这些处理。

  • 启用 subsurface（次表面）或 single scattering（单次散射）时，为避免伪影（与 volume tracking 相关），PxrSurface 会在内部强制使用未缓存 Presence（忽略 Presence Cached 的参数设置）。 

  • 出于类似原因，对于非薄玻璃折射，PxrSurface 会忽略 Presence，无论缓存还是未缓存。

  • 该 integrator 可以控制不透明度不再是随机（有噪声）的最大深度。

Shadow Mode

阴影不透明度计算模式：

  • Shader and shadow color（Shader 和阴影颜色）

  • Shadow color only（仅阴影颜色）

Shadow Color

指定阴影颜色。此参数可用于出于艺术指导目的伪造阴影颜色。请注意，更改阴影颜色可能会影响性能和全局照明结果，因为它会被视为透明阴影（可理解为彩色玻璃），即使对象是不透明的。

Utility Pattern

指定用于输出的 pattern 节点。例如：可以将 PxrMatteID 连接到此参数以输出 ID AOV。也可以输出另一个 pattern，例如纹理。

Enable Manifold Exploration/Walk（启用流形探索/行走）

仅在使用 PxrUnified Integrator 且 Enable Manifold Walk（流形行走）选项设为 ON 时可用。对于要 捕获 caustic light paths（焦散光路）的对象，此设置应开启。例如镜面对象内部的漫反射表面。

Specular Curvature Clamp（镜面曲率钳制）

此控制项用于覆盖对应的积分器设置，即 Specular Curvature Filter 参数，该参数位于 PxrVCM 和 PxrUnified 积分器中。

User Color

指定要渲染到 userColor Lobe 的颜色或 pattern，它可以 作为 utility（实用输出）通过 LPE 输出。此连接或颜色不会渲染到 beauty 中，目的是输出对合成或后期调整有用的 AOV。
