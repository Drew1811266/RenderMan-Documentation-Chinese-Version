对象

本教程扩展了 Harsh Agrawal 在 Community Site version（社区网站版本） 中提出的说明，并聚焦于 look development（外观开发）和灯光。

该场景重现了摄影棚灯光以及有机对象，并加入景深以获得更好的真实感。我们会先拆解每个对象及其着色。

建模

创建资产时，理解哪些对象属于特定类别，并据此为它们建模相应的细节非常重要：

  • Hero assets（主角资产）- 例如主要角色，以及镜头或故事的焦点。

  • Set assets（布景资产）- 这些资产可能根据它们在场景中的位置采用不同细节级别。特写物体可能需要更多细节。

  • Props（道具）- 这些通常会尽量保持简单；你会惊讶于优秀的纹理处理能把低细节资产做得相当好看！

RenderMan 的设计是在可能时为动画角色和 hero objects 渲染 subdivision surfaces。这会减少可能的撕裂，并改善近距离看到的角色和有机对象外观。使用 displacement 时这也是一个好选择。多边形在位移后容易裂开，所以请转换为 Subdivision Surface。

为 Subdivision Surfaces 建模时，请记住以下事项，以获得想要的形状，同时不破坏性能：

  • 使用默认 Catmull-Clark 方案转换为 Subdivision Mesh 时，为避免过度 dicing，请只使用四边形。这意味着所有面都有 4 条边。避免三角形和 N 边形面。

  • 在 base mesh 中只建模你所需的最低限度细节，使细分后的结果表面具有正确形状，而不要更多。

  • 尽量减少 creased edges（折痕边），它们可能开销很高。

  • 利用更高的 micropolygonlength 设置。这会提供自动 level-of-detail（细节层级），数值越高越粗糙，但仍可能足以获得良好外观。合理的生产质量可以尝试 1.5 到 3 之间的设置。（请注意，如果过于粗糙，你会得到着色伪影，尤其是在边缘处，请慢慢降低该值。任何小于 1 的值都属于 非常高 质量。）

多边形对于任何具有硬法线的对象也很有用（许多非有机物体的 faceted faces，例如建筑、道具、布景件等）。你可以为近距离渲染软化这些法线，或添加更多建模细节使其更好看，但要注意，如果这样仍不足够，着色时可能出现轮廓伪影。尽管有这些注意事项，用于背景和道具对象的多边形通常能渲染得很好，并且性能也不错。这些对象通常不会变形，并且在场景中数量很多。把它们做成 subdivision surfaces 可能会显著增加内存占用，却只有很小的视觉收益。请在多边形上使用 bump mapping 而不是 displacement，或者为了 displacement 将它们转换为 subdivision surfaces。

布置 UVs 时，请避免 UVs 重叠，并留意带有翻转边的 UV shells，因为这可能在位移时导致伪影，或在渲染期间产生警告。

我们的纹理缓存机制可以处理充满高质量纹理细节和外观的场景，但我们需要让整套用于追踪的场景几何体驻留在内存中。只要谨慎选择建模方式，就能在场景中放入相当多内容，并一次渲染许多 GB 的纹理。

理解 PxrSurface 的基本设计。

在 RenderMan 中用于制作对象、道具和角色的主要材质是 PxrSurface 材质。这是一种可用于各种着色任务的单一材质（因此也是 "monolithic"，即整体式材质）。材质的每个部分都代表该材质的一个特定 "lobe"（波瓣）。

这些 lobes 的几个例子包括：

  • Diffuse（漫反射）

  • Specular（镜面反射）

  • Rough Specular（粗糙镜面反射）

  • Sub-Surface（次表面）

  • 等等。

它们各自都有一组控制项，用于定义外观，以及它们如何通过 gain 或 weighting 与其他 lobes 组合。

为什么称为 "lobe"？你可以根据材质的分布函数来可视化它如何反射或透射光线，这些通常以 lobe（在 2D 或 3D 中显示锐度、强度和方向的拉长形状）来表示。你可以在 Wikipedia上了解更多。


纹理贴图基础。

请看这张桌子：它使用纹理作为一个带木质颜色和光泽表面的基础材质。虽然它像抛光木材一样略有反射，但并不过于闪亮。参考图非常关键，可确保你达到艺术总监要求或你自己目标中的正确外观。起初这看起来很简单，但通过一些快速技巧和提示，再结合示例拆解后，会发现有许多因素需要考虑。

标准纹理命名、建议和使用思路：

  • 纹理应根据其目标效果和对象分配来恰当命名。例如：<object>_<effect>.jpg，或者 tableTop_specular.jpg。

    • 不要在名称中使用空格和特殊字符，例如：theCar&boat.jpg 或 "the Car and Boat.jpg"。

    • 文件路径中也不要使用空格和特殊字符。这些纹理可能无法加载。

    • Texture sequences（纹理序列）应有清晰的帧号命名约定。mySequence.0001.jpg 是正确形式，而不是 mySequence0001.jpg 或 mySequence.jpg.0001。

  • 大多数格式都可以使用，例如 .png、exr、.tif 和 .jpg，但若要渲染，必须将它们转换为 .tex format（.tex 格式）。在 Maya 中，这会在开始渲染时发生，并缓存到磁盘上以便重复使用，直到场景中发生更改或更新。包含大量纹理的场景在第一次渲染前可能需要一些时间来转换。

  • 有时会出现可互换的命名对；一致性对 pipeline 以及艺术家准确工作的能力很重要。

    • Diffuse 和 Albedo

    • Roughness 和 Glossiness（请注意，RenderMan 使用标准的 "Roughness" 约定，其中 0 = 镜面，1 = 粗糙或接近漫反射。）Roughness 也适用于 Diffuse 参数，因此艺术家有时会用 Glossiness 作为名称。

  • 纹理可以是 color（RGB）或 scalar（灰度），颜色通道可以连接到匹配参数来驱动它们。例如，如果有用，图像的 Red 通道可以用于驱动 scalar（float slider）。

  • Normal maps 通常比 bump maps（scalar 或 grayscale）更适合用于表现细节。

  • Linear color workflow（线性色彩工作流） 是 强烈建议采用的，最终渲染中可见其颜色的纹理应进行 linearize（典型的 sRGB gamma curve 应在渲染时移除）。用作数据的纹理，例如 bump、masks 和 displacement，则 不应 进行 linearize（线性化）。

  • Displacement maps 和其他 data textures 应尽可能使用最高精度，以避免纹理量化造成的伪影。通常这些会从建模应用程序导出为 floating point EXR 或 Tiff。有关 normal 和 scalar displacement，请参见 workflows（工作流）。

如果 Maya File Node 具备你需要的功能，或者你喜欢使用 Maya workflow 处理 texture sequences 等内容的简洁性，那么没有特别理由不使用它进行纹理处理。Maya file node 和 PxrTexture node 都会在 viewport 中显示纹理。不过，只有 Maya file node 会在 viewport 中显示 UDIM/tiled textures，并对 viewport 应用校正后的 gamma。


Wood Table（木桌）

这张桌子使用了几种不同的纹理贴图。其中一张用于 diffuse（有时称为 albedo）颜色，它控制木材的实际颜色变化和结果。这是对象颜色映射最基本的形式，独立于反射颜色和其他材质效果。

另一张是 specular map，用于定义表面上反射和高光的数量及位置。这张贴图可以定义反射颜色，也可以定义它在整个表面上的强度。除非你正在制作一个全新且完全未被触碰的产品渲染，否则这里的一些变化对真实感很重要。可以添加划痕和其他缺陷。请注意让这些效果与 bump map 等其他有用贴图匹配，使所有表面特征都能正确对应。你大概不会想要一个有划痕和磨损、却没有匹配 bump map 的表面。你可能会注意到这张贴图很暗，因为 PxrSurface specular color 对光照值很敏感，很容易迅速变亮。

roughness map 会在 specular map 的 roughness 结果中加入变化。真实世界中几乎没有物体拥有完美镜面般的表面，而表面更粗糙、带光泽饰面的物体，其外观也很少完全均匀。使用纹理贴图来控制这一点，有助于提供真实感，并在桌面上创建磨损和使用痕迹。它适用于指纹、污迹、油脂痕迹、摩擦和触碰造成的磨损，或制造材料中的自然变化等内容。

绘制 texture maps 时，你在绘制过程中可能只能获得有限反馈。这意味着你可能已有一张准备尝试的纹理，但还没有确定它是否正确产生了你想要的效果。它可能太暗或太亮；你的 roughness map 可能太粗糙，也可能不够粗糙。无穷无尽的变化可能迫使你在所选绘制程序之间来回切换，这会变得令人沮丧。

使用 Maya 的 IPR mode，你可以让纹理通过一些 utility nodes，并直接在渲染中进行数值调整。 

这些 pattern networks（模式网络） 可以在每次渲染处理纹理时被求值，或者你也可以 bake（烘焙） 结果，以提升复杂资产的整体效率。（你可能希望在获得客户最终批准前避免 baking（烘焙），以免反复往返 render farm。）某些 patterns 的求值成本高于其他 patterns，并可能开始影响场景渲染时间。Baking（烘焙）可以改善这一点，因为它只作为一张纹理求值一次并被缓存。你可以查看关于 Improving Performance（提升性能）的文章来了解更多效率相关内容。

虽然 Ptex 可以节省资产纹理准备时间，但其性能不如在已定义 UV 的对象上使用标准纹理。在规划项目时，应权衡这种成本和收益。如果有许多对象需要制作纹理，Ptex 可能总体上节省艺术家的时间。否则，你可能会发现使用 texture maps 和典型 UDIM（或 "tiled"）UV layout 能获得更好的渲染效率并简化流程。

牛奶瓶

现在我们来看 Milk and Bottle。

牛奶本身没有使用纹理，稍后会讲到；因此我们会先隔离瓶子，并讨论它如何真实地实现“脏瓶子”的外观。

玻璃有一些让它显得特殊的特性。也正因为如此，PxrSurface 材质有一整个分区专门用于重现玻璃及其最常见外观。若要理解不完美且脏污的玻璃，可以尝试拆解图像中正在发生的事情。不完美的玻璃表面会存在 roughness 变化。不同于水晶和少数其他表面，许多较旧且磨损的玻璃物体并不具有完美的反射或透射（折射）效果。玻璃表面还会增加指纹和污迹。你可以使用 masks 生成这些效果，并把它们添加到 diffuse lobe 中，以混入更多 dirt、paint 和 dust 等内容。

更容易分层实现这种效果有两个选项我们会讨论，另外还有第三个关于 trace depth 的注意事项，会在另一篇文档中介绍：

  • 使用 PxrLayerSurface 来分离并再组合玻璃和污垢的着色。

  • 正确的 UV layout，用于防止纹理在内侧和外侧重复。

  • 适当地控制 ray depth（射线深度），以获得正确的玻璃物理结果。


最好以体积方式对对象建模。它们应具有真实厚度，玻璃表面之间的空间会像真实世界中一样工作，并应按你预期的方式表现。除非对于树叶或纸张等对象确实合理，否则请避免使用二维模型。

渲染大多数带液体的玻璃容器时，理解 intersection priority（相交优先级） 有助于避免伪影和 "air gaps"（空气间隙）。这在技术上称为 dielectric interface（电介质界面）或 nested dielectrics（嵌套电介质）。



首先，我们会通过移除上层污垢/牛奶层来查看玻璃层。

在玻璃基础材质中，diffuse channel 有一些 dirt，gain 较低，为 0.2。低 gain 与较暗纹理结合，可以确保该效果不会过于不透明，否则瓶子会看起来不像玻璃。请注意，瓶子的内壁和外壁是分开映射的。指纹通常不应该同时出现在瓶子的外侧 和 内侧。

reflection gain 像托盘和桌子一样使用纹理，用于定义瓶子在哪些位置最具反射性，或者哪些位置磨损和污垢最少。

refraction color 用于对穿过玻璃的光线透射进行着色或衰减。在本例中，为了暗示更多不完美之处，将一张贴图连接到了这个 color input。白色区域表示这些区域的玻璃是透明的 且 无色。

牛奶

牛奶表面实际上很容易创建。PxrSurface 页面提供了一张有用的数据表，用于通过 Subsurface Scattering（次表面散射）创建常见物品。请使用正确的颜色值填充 Subsurface rollout 下的参数。

完成后，有 2 种简单方式可以控制你所需的结果：

  • Gain - 使用它以艺术方式控制该效果的权重。

  • Post Tint - 使用它来制作巧克力牛奶！

用户可能还需要调整另一个控制项，但通常只需调整一次，即 Mean Free Path Distance。此选项根据 scene units（场景单位）控制散射距离。把这个参数设置到合理数值很重要，以避免噪声等问题。

对于牛奶，进入液体的光线只会在介质中散射一定距离，然后被吸收。这意味着更真实的值应遵循这一理解。如果该值太低，光线散射不足，对象会显得更像实心。如果该值太大，光线会散射得过远，并引入噪声或伪影。在牛奶这个例子中，根据场景比例以及光线可能进入液体介质的距离，选择了 1.0。非常高的设置，例如 10 或 15，按 scene units（场景单位）测量时实际上已经超出了容器之外。这并不真实，因为这意味着散射不仅超出了液体本身，甚至超出了玻璃容器。虽然它可能看起来讨喜，但会更慢、引入噪声，并且对牛奶来说不真实。用户如果在忽略场景比例的情况下花太多时间调这个度量值，可能会因为试图重新平衡其他控制项来补偿而感到挫败。

对于表中未列出的材质，可以从表中最接近目标外观的项目开始，然后适当修改颜色和 gain。不过，Mean Free Path Distance 的建议仍然相同。

灯光

此场景使用了常见的 three-point lighting（三点布光）场景。其中有一个使用 PxrDomeLight的 fill light，以及一个使用 PxrRectLight的 rim light（轮廓光）和 key light（主光）。

除 Fill Light 外，其他灯光使用 emission focus（发射聚焦） 来创建更柔和的边缘和细微的聚光灯效果。

Fill Light（补光）

用于填充暗部和 shadows（阴影）的灯光提供基础照明，并称为 fill light。此场景使用了高动态范围图像（HDRI）。使用 PxrDomeLight 时，它需要一张 texture 作为输入，并且该 texture 应包含较宽的数值范围，其中许多值超过 1.0。这应该是 EXR、TIFF 或 HDR 图像。你可以从 Maxime Roz 免费找到一些示例，位置在 这里。

Rim Light（轮廓光）

Rim lighting 可用于定义形状或轮廓。名称 "rim" 来自它会勾勒对象的边缘或轮廓这一事实。

下方将它添加为 PxrRectLight；右侧则把它调整为较低的 specular amount（镜面反射量），使其更讨喜、更细微。在这张图像中，它只需要稍微改善轮廓，而不是过强地高亮轮廓。

Key Light（主光）

key light 是这张图像中的主要光源。它通常是图像中最亮的灯光，并帮助定义一天中的时间、光源来源以及空气质感（如果场景带有烟雾或尘埃）。

这里它被放在画面外，让人感觉像自然窗光。右侧同样降低了 specular amount，以柔化高光，并在与另外两个光源组合时生成最终图像。

Softbox（柔光箱）

该场景建模了一组墙面作为 softbox。Softbox 用于在场景中反弹并漫射光线。由于 RenderMan 使用 global illumination（全局照明），这个摄影技巧会像真实世界中一样工作。你可以通过改变 diffuse gain 和 color 来调整 softbox 的漫射量。
