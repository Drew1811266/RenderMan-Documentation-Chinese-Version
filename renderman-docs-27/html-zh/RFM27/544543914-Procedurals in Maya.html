Maya 中的程序化过程

Source: https://rmanwiki-27.pixar.com/space/RFM27/544543914/Procedurals+in+Maya
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan 程序化基本体是用户提供的子例程，可在渲染过程中按需调用以生成几何体。与原生 Maya 几何体相比，程序化过程的优势在于它们可以非常高效地生成复杂几何体。程序化过程可以从少量输入创建极高的几何复杂度，而且我们可以把处理延迟到渲染时。这可能让渲染出的场景比标准 Maya 表示方式所能表示的场景复杂得多。程序化过程也可以被视为内存管理单元。渲染器可以选择卸载与某个程序化过程关联的所有具体几何体，因为它知道如果后续仍有需要，这些几何体可以重新生成。

RenderManProcedural (RiProcedural)

RenderMan for Maya 随附通用的程序化节点类型 RenderManProcedural，可用于加载 RiProcedural 子例程（不是 RiProcedural2）。它的设计目标是快速、简单，不需要安装或自定义。为了提高灵活性，可以使用 Pre Shape Mel 属性在场景转换期间设置 string data 参数。

自定义节点类型扩展（RiProcedural 和 RiProcedural2）

或者，用户可以使用 .args 文件参数列表，为任意 Maya 节点指定到 DSO 程序化过程的自定义映射。这旨在让 RenderMan 与现有 Maya 插件无缝集成。

环境配置

可以在以下位置放置 config/mayaTranslation.json 扩展文件，让 RfM 识别自定义 .args 文件程序化过程： RFM_PLUGINS_PATH (22.2), RFM_SITE_PATH 或 RFM_SHOW_PATH。示例站点配置：

setenv RFM_SITE_PATH /tmp/simple_procedural

RFM_PLUGINS_PATH 专为程序化过程设计，并接收一个路径列表：

setenv RFM_PLUGINS_PATH "/tools/plugins/myHairPlugin:/tools/plugins/myCrowdPlugin:/tools/plugins/mySpherePlugin"

每个路径都应包含程序化过程，以及一个 config 目录，其中可以包含常规文件： rfm.json, extensions.json, menu.json, shelf.json，等等。

位于 RFM_PLUGINS_PATH 中的路径会被加入渲染时搜索路径，包括以下子路径：

Option "searchpath"
"string rixplugin" ["...:/tools/plugins/mySpherePlugin:/tools/plugins/mySpherePlugin/nodes:/tools/plugins/mySpherePlugin/nodes/args"]
"string shader" ["...:/tools/plugins/mySpherePlugin:/tools/plugins/mySpherePlugin/nodes:/tools/plugins/mySpherePlugin/nodes/osl"]
"string texture" ["...:/tools/plugins/mySpherePlugin"]
"string procedural" ["...:/tools/plugins/mySpherePlugin"]

节点类型映射

该目录应包含 config/mayaTranslation.json，用于声明转换器映射。在这个例子中，我们把自定义 Maya 节点类型 "MayaSphere" 映射到 RenderMan 程序化过程 "SphereProc"。示例 config/mayaTranslation.json：

{
    "nodeTable": [
        "MayaSphere", "SphereProc"
    ]
}

RfM 和 RenderMan 会在 RFM_SITE_PATH 与 RFM_SHOW_PATH 中搜索指定名称的程序化过程 DSO 和 args 文件，也就是 SphereProc.so 与 SphereProc.args。

程序化过程 args 文件

args 文件应将插件 typeTag 设置为 "procedural"（用于 RiProcedural）或 "procedural2"（用于 RiProcedural2）。还必须把特殊的 "string __dsoname" 参数设置为 DSO 的名称。其他程序化过程参数可以像着色参数一样声明。"match" 功能可用于转换参数名不同的 Maya 属性，例如：

<args format="1.0">
    <page name="Parameters" open="True">
        <param name="__dsoname" type="string" default="SphereProc.so" match="-none-"/>
        <param name="radius" type="float" default="1"/>
        <param name="material" match="rman_material" type="string" default=""/>
    </page>
    <typeTag>
        <tag value="procedural2"/>
    </typeTag>
    <rfmdata
        classification="RenderMan/procedural"/>
</args>

延迟材质绑定

材质可以使用这里描述的 __materialid 机制进行延迟绑定： /wiki/spaces/REN/pages/19859939。在 Maya 中，"string __materialid" 参数对应 Maya 着色组名称。用户可能需要在渲染设置的 Advanced 区域下启用 "Output All Shaders"，以便预先输出所有材质。

setAttr "rmanGlobals.outputAllShaders" 1;
