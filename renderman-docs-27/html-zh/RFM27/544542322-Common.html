Common（通用）

Source: https://rmanwiki-27.pixar.com/space/RFM27/544542322/Common
Downloaded: 2026-06-15T03:49:12+00:00

Common 标签页提供 Maya 中所有渲染器共享的设置。请注意，此配置会在将 RenderMan 设为渲染器后，或从 RenderMan 菜单打开设置后显示。

你会发现此类功能的大部分已经移到 Workspace Tab 以简化选择。下面说明的是我们不再支持的旧选项；其中许多已经从 RenderMan Common 标签页中移除。


Path

不适用；如果仍能看到此项，请将渲染器选择切换到其他渲染器，然后再切回 RenderMan 以刷新。

File Name

不适用；请使用 Workspace Tab 通过便捷 token 进行命名。

Image Size

显示分辨率（通常以像素和英寸表示）。

Color Management

RenderMan 不支持 Output Transform 设置。不过，"it" 中提供了视图 重映射 功能。

File Output

渲染图像文件名最多可由三个元素组成：文件名、帧编号和文件格式。

File Name Prefix

决定渲染图像的基础名称。

Image Format

不适用； AOVs 标签页 会设置显示驱动。

Compression
不适用；AOVs 标签页会在 Display Driver 中设置压缩。

Frame/Animation Ext

用于命名输出图像的格式。

Frame Padding

不适用；请使用 Workspace 标签页按文档页说明的 token 设置此项。

Framebuffer/Naming Scheme

不适用于 RfM；这些会在 AOVs 标签页中设置。

Use Custom Extension

不适用于 RfM；请使用上文提到的 Workspace 标签页。

Frame Range

Start Frame, End Frame

指定要渲染的第一帧和最后一帧。Frame/Animation Ext 必须设为包含 # 的选项才能启用这些参数，否则只会使用单帧。

By Frame

已渲染帧的输出名称之间的增量。只有当 Frame/Animation Ext 设为包含 # 的选项时，By Frame 才可用。

Renderable Cameras

Renderable Camera

从此列表中选择你的渲染摄影机。  如果场景只有一个可渲染摄影机（例如透视摄影机），无论在这个下拉列表中选择哪个摄影机，它都会成为可渲染摄影机。换句话说，新选择的摄影机会成为场景中的可渲染摄影机（默认透视摄影机会变为不可渲染）。

Alpha Channel (Mask)

确定已渲染图像是否包含遮罩通道。

Depth Channel (Z Depth)

确定已渲染图像是否包含深度通道。建议使用 AOVs 标签页指定 __depth 或 Z。

Image Size

Presets

根据行业标准预设选择输出图像的分辨率。

Maintain Width/Height Ratio

如果勾选此框，自定义分辨率会保持锁定的高度/宽度比例。禁用此参数后，可以为图像分辨率输入任意数值。

Width/Height

对于自定义分辨率，请在此处输入所需分辨率。

Size Units

选择合适的测量单位。

Resolution

设置图像的渲染分辨率。

Resolution Units

选择分辨率的定义单位，可以是每英寸像素数或每厘米像素数。

Device Aspect Ratio

目标显示设备（例如 "it" 或 Maya 的 Render View）的宽高比。

Pixel Aspect Ratio

显示设备上单个像素的宽高比，也就是你查看渲染图像时所用设备的像素纵横比。

Render Options

Enable Default Light

在渲染期间开启或关闭默认灯光。请注意，如果场景中有 RenderMan 灯光，我们不会渲染默认灯光。

Pre/Post MEL scripts

这些字段用于输入 MEL 脚本/命令，以便在指定时机运行。
