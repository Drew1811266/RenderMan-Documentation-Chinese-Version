高级设置

Source: https://rmanwiki-27.pixar.com/space/RFM27/544542672/Advanced
Downloaded: 2026-06-15T03:49:12+00:00

许多位于 Advanced 选项卡 中的设置你可能永远不需要编辑。不过，对于想要精细控制 RenderMan for Maya 的高级用户，这些选项都在这里。

Render Options

Output All Shaders: 通常我们只导出附加到对象的材质网络，但如果你正在为 procedural（程序化对象）着色，它可能没有以可导出的方式连接。此选项会无论如何导出所有网络。

Reentrant Procedural: 通常我们只导出附加到对象的材质网络，但如果你正在为程序化对象着色，它可能没有以可导出的方式连接。此选项会无论如何导出所有网络。

Output Holdout Matte: 使用 允许程序化对象在多个线程中同时运行，并显著加快包含程序化对象（如毛发）的渲染。你很可能应保持开启。 工作流时，将 Shadow AOV 输出到此指定缓冲区。

Enable Image Plane Filter: 使用 Pixar Image Plane Filter 渲染图像平面（image plane；Maya 使用实际几何平面）

Learn Light Results: 这是适用于多灯光场景的新选项。渲染器会细化灯光选择过程以改进采样。它尤其适合有许多被遮挡灯光的场景，可避免在阴影表面上浪费采样。此版本不是确定性的；如果同一场景重新渲染时没有充分收敛，可能看到细微差异。

Bucket Order: RenderMan 将输出图像细分为称为 bucket 的小矩形区域，并一次渲染一个 bucket。这允许任意时刻只有图像的一小部分保留在内存中。下拉列表允许用户选择 bucket 在 "it" 或 Render View 中显示的顺序。默认是 Spiral。请注意，只有部分 bucket 顺序支持 checkpointing 以及恢复相关机制。

RenderMan 将输出图像细分为称为渲染分块的小矩形区域，并一次渲染一个分块。这允许任意时刻只有图像的一小部分保留在内存中。下拉列表允许用户选择分块在 "it" 或 Render View 中显示的顺序。默认是 Spiral。请注意，只有部分分块顺序支持 指定 bucket 的大小。较大的 bucket 需要更多内存才能渲染。相反，较小的 bucket 对着色效率较低，但渲染期间会使用更少内存。如果场景使用大量内存，可以尝试将此字段降到 8 by 8，甚至 4 by 4 buckets，但你很可能不会更改此项。

Opacity Threshold: 指定分块的大小。较大的分块需要更多内存才能渲染。相反，较小的分块对着色效率较低，但渲染期间会使用更少内存。如果场景使用大量内存，可以尝试将此字段降到 8×8，甚至 4×4 分块，但你很可能不会更改此项。

Reference Frame: 每个作业 pass 一次，以及下面的 静态 对象会在这里指定的帧进行求值。

Adaptive Sampler

每个作业流程一次，以及下面的 默认使用改进的 Variance 采样算法。你可以使用 Contrast-V22 指标作为旧技术，以匹配之前的渲染结果。新技术改进了逐帧噪声的一致性。每种选择可能会显示额外选项。

    • Contrast-V22：基于样本之间对比度阈值的旧指标；会匹配最近旧版本的采样
      • Darkfalloff：用于防止图像中可能不重要的暗部区域被过度采样
    • Variance：改进的感知方差方案，应能提供逐帧一致的噪声
      • Exposure：用于提示渲染器基于此曝光区间进行采样（假定合成流程保持在该范围内，以免显露预期范围之外未采样的噪声）

Adapt All: 将采样质量标准（像素方差）应用到所有颜色/照明 AOV。这可能增加渲染时间，因为某些 AOV 可能比 Beauty 有更多噪声，并触发所有 AOV 产生更多采样。

Dicing

Micropolygon Lenth: 控制置换的细分率。默认设置 1 会将置换切分到 1 像素大小（从切分相机视角看，通常是渲染相机）。较小数值会产生更细的几何细节，但会创建更多几何体。

Watertight Dicing: 关闭切分后几何体中的针孔，或尝试关闭置换中的缝隙。如果实心对象的 alpha 中有孔洞，此项很有用。

Dicing Camera: 启用参考相机进行切分。可以全局使用，也可以通过将其作为按对象属性附加，限制到特定对象（例如带有置换且会随相机移动而“闪烁”的对象）。

Reference Camera: 在这里指定用作切分相机的相机。点击控件创建新相机，或在字段中右键选择现有相机。请注意，Reference Camera 需要不同于渲染相机。

Hair

Min Hair Width: 设置毛发/RiCurves 的最小宽度。更窄的毛发会被钳制到此值，并降低不透明度以补偿视觉权重差异。如果数值不太小，这可能改善抗锯齿质量。

Trace

Trace Auto Bias and Trace Bias: RenderMan 会自动计算追踪偏差，用于将光线原点从命中表面上偏移。不过，如果你希望关闭自动偏差，可以在这里手动设置偏差。按对象设置应关闭 Auto Bias。缓慢增大该值，直到瑕疵（自阴影、暗线等）消失。

Trace World Origin: 移动世界原点，以提高光线追踪的浮点精度。有时在渲染超大型场景时，相机会移动到距离原点很远的位置。根据场景单位，这可能引入精度问题；在这些情况下，导出到渲染器的值会非常大。可能的取值为：

  • "world"（默认）使用真实世界原点
  • "worldoffset" 使用用户提供的 trace:worldoffset 作为光线追踪世界原点（该控件会显示 XYZ 坐标字段）
  • "camera" 使用相机位置作为光线追踪世界原点；在有多个相机位置的大型场景中，你可能希望将其作为默认。

Open Shading Language

Enable Batched OSL: 通过受支持硬件上的向量化指令，为批处理着色点启用性能提升。（在至少支持 AVX 的机器上可用。此外，它可以利用支持 AVX-512 的机器上的 512 位宽数据路径。此项默认 开启。）

OSL Verbosity: 有效范围为 0 到 5，其中 5 输出最多消息。更多信息请参阅 OSL 文档。

OSL Statistics Level: 同上，有效范围为 0 到 5，其中 5 输出最多消息。

Deep Rendering

Quality: 控制合并样本的抽减率。设置为 1.0 表示不进行样本合并，并会生成巨大文件。默认值 0.75 映射到 Deepshadow Error 0.01；设为 0 时映射到 Deepshadow Error 0.16。请注意，这不是线性映射。

Crop Window

Enable: 使用裁剪窗口；off（默认）会根据相机设置渲染完整帧

Top Left/Bottom Right: 指定要渲染的裁剪窗口坐标（x 和 Y）。当进行许多调试或外观开发迭代且只关注特定区域时，这很有用。你也可以用它移动已渲染区域，将一个超大型渲染（可能用于打印）拼接在一起，并把不同区域发送到不同渲染节点。请注意，在 PxrVCM 渲染器中使用裁剪窗口可能会生成不同于完整帧的结果，因为作为优化，我们只为正在渲染的区域生成光子。

IES Profiles

Scene units to meters: 虽然场景会以“无单位”方式导出到渲染器，但 IES 文件会指定度量单位。使用此设置指定场景单位到米的换算系数。例如，如果 1 个单位 = 1 米，使用 1。如果 1 个单位 = 1 厘米，使用 100。

Ignore Watts: 忽略配置文件中的内部瓦特测量值

Cache Sizes

允许用户自定义各种资产的缓存大小，包括纹理、光线追踪几何体（特别是细分缓存）和不透明度（作为材质中 Presence 的函数）。这些值以 MB 为单位；当达到这里指定的限制时，渲染器会自动从缓存中卸载数据。这些值可能会基于机器可用资源而偏保守。如果渲染统计显示许多缓存未命中（重新加载和重新读取），且总体使用量低于可用内存，你可能希望增大这些值以改善性能，但要保持合理并处于内存预算内。

Statistics

Level: 启用此项可从渲染作业输出 诊断信息。可用于计时、调试，以及查看 RenderMan 正在做什么。数字越高，详细级别越高。请注意，这可能造成性能影响，因为渲染器需要查找这些字符串，累积起来会有开销。

XML File: 你可以为 "Old School" 和 XML 统计指定目标位置。

LPE 波瓣映射

此部分将指定波瓣映射到输出。你会在 光路表达式 文档中注意到，逐波瓣输出依赖这些映射正确，因此建议保持默认。你可以在自定义 C++ BxDF 中使用 User Lobes。

示例：你会注意到默认设置中 Diffuse 3 Lobe ID 是 Subsurface。对应 LPE 是：

color lpe:CD3[DS]*[<L.>O]

注意指定的 D3 波瓣。它与这里分配的 ID 匹配。

没有 D1、S1 或 U1 波瓣，因为未指定内容时它们会落入默认分支。User 2 波瓣是汇总的 Albedo 结果（纹理数据）。
