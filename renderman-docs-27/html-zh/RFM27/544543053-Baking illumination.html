Baking illumination

Source: https://rmanwiki-27.pixar.com/space/RFM27/544543053/Baking+illumination
Downloaded: 2026-06-15T03:49:12+00:00

Overview（概述）

此模式实际会 bake integrator 的 outputs。它主要用于 bake diffuse global illumination，但也可以 bake 任意 light path expression。

How does it work?（它如何工作？）

  • Scene objects 场景对象会逐个 baked。

  • integrator 会在每个 point 上以 single increment 运行（Max Samples = 1）。

  • 结果会 output 到一个或多个 AOVs，每个 AOV 只包含一个 display channel。

Settings（相关设置选项）

illumination baking settings 只会 override 你的 scene render globals 中的一部分 parameters。如果当前 scene 的 render 符合预期，那么这些就是 bake 所需的全部 settings。

Bake Mode（烘焙模式）

此 menu 的主要用途是在 rendering 前 validate 你的 setup，并在检测到 incorrect setup 时发出 warning。

2D Maps（2D 贴图）

  • 为每个 object 生成 1 张 image。

所有 baked objects 都必须具有用于 baking 的 valid uvs 有效 UVs

3D Point Cloud（3D 点云）

  • 根据 file naming format，生成一个或多个 point clouds。

  • 在 world space 中 bake

Baking Image File Format（烘焙图像文件格式）

通常，你会用 tokens 创建一个 pattern 来生成 named AOVs，但这里有一点不同。默认 file pattern 为：

<scene>_<user:bakingIdentifier>_<aov>.<ext>

user:bakingIdentifier 是一个 new attribute，可在 Attribute Editor 的 RenderMan section 中所有 bake-able shapes 上找到：

它的默认值为 <shapepath> 因此每个 object 都会获得唯一 file name，其中 "|" character 会被 "_" 替换；当你 bake 到 2D images 时，这正是所需行为。

如果想 bake 到单个 3D point cloud，应从 file name pattern 中移除 <user:bakeIdentifier>。

Bake to Maps (2D baking)（烘焙到贴图，2D 烘焙）

选择 global baking resolution。更大的 resolutions 会需要更长 render 时间。

Limitation（限制说明事项）

所有 objects 都会以相同 resolution baked。这是当前 implementation 的 limitation。

Bake to Point Cloud (3D baking)（烘焙到点云，3D 烘焙）

Point clouds 会在 world space 中 baked，其 density 取决于 shape 的 dicing density。在此为 scene 选择 global dicing density。

你可以通过将 Dicing Strategy 设置为 worlddistance 并输入不同的 Dicing Distance Length，按 shape override dicing density。较小 values 会创建更多 points。

Integrator（积分器）

选择一个 integrator 及其 settings。我们推荐 PxrPathTracer。

Limitation（限制说明事项）

某些 integrators（如 PxrVCM 和 PxrUnified） incompatible 不兼容状态，因为它们完全依赖 incremental rendering 来 accumulate samples，并随时间 resolve illumination。

Baking 会运行 integrator 每个 point 一次（Max Samples = 1），并且它必须设置为 在 single increment 中提供良好结果。

为获得最佳结果，需要充分增加 direct samples（light samples 和 BXDFs samples）以及 indirect samples 的数量，以得到 reasonably clean results。最佳 settings 高度依赖 scene，需要通过 experimentation 确定。

Limitation（限制说明事项）

Baking 是 not adaptive 非自适应采样模式（即 brute-force），因此无论 convergence 如何，都会计算所有 samples。

Illumination baking 可能非常慢：请确保你 选择最小必要 samples 数量，并记住 baked illumination 中少量 noise 几乎不可见。

请注意，bake view-dependent illumination（如 specular）没有意义；应始终尽量 bake diffuse、albedo 等 view-independent signals。

Limitation（限制说明事项）

目前无法在 baked 2D images 上运行 denoiser。

Default Ray Depth（默认射线深度）

照常可以设置 max diffuse depth 和 specular depth。这会在 baking 期间 override 常规 scene settings。

AOVs

AOV section 的工作方式与 render globals 中完全相同。

默认情况下，我们会 bake diffuse illumination，包括 direct 和 indirect。

如果需要 bake raw illumination，还需要同时 bake albedo，并将 illumination 除以 albedo（在 compositing package 或 shader 中），以获得 raw illumination（也称为 irradiance）。

Limitation（限制说明事项）

每个 AOV（也称为 Display）只能包含一个 display channel，最多 output 4 个 channels（RGBA）。

添加更多 displays 可 bake 更多 channels。

Bake 2D

任何 non-deep image format 都可用于 bake 到 images。推荐使用 OpenEXR，以避免 clamped results。

Bake 3D

请确保选择 d_pointcloud display driver 显示驱动来 bake point clouds。

如果忘记这样做，preflight checks 应会 warning 并 cancel bake render。


Launching a bake render（启动 bake render 渲染任务）

bake render 本质上是一个只有一项 setting 不同的 batch render。它使用 bake Hider，而不是 raytrace Hider。

要启动 bake render，只需选择 Bake Render: Illumination，而不是 RenderMan menu 中的 Batch Render。所有常规 batch render preferences 批量渲染首选项 都会应用到 bake render。
