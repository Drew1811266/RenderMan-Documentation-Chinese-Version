Baking illumination

Source: https://rmanwiki-27.pixar.com/space/RFM27/544543053/Baking+illumination
Downloaded: 2026-06-15T03:49:12+00:00

概述

此模式实际会烘焙积分器的输出。它主要用于烘焙漫反射全局照明，但也可以烘焙任意光路径表达式。

工作方式

  • 场景对象会逐个烘焙。

  • 积分器会在每个点上以单次增量运行（Max Samples = 1）。

  • 结果会输出到一个或多个 AOV，每个 AOV 只包含一个显示通道。

相关设置

照明烘焙设置只会覆盖场景渲染全局设置中的一部分参数。如果当前场景的渲染符合预期，那么这些就是烘焙所需的全部设置。

Bake Mode（烘焙模式）

此菜单的主要用途是在渲染前验证你的设置，并在检测到设置不正确时发出警告。

2D Maps（2D 贴图）

  • 为每个对象生成 1 张图像。

所有已烘焙对象都必须具有用于烘焙的有效 UV。

3D Point Cloud（3D 点云）

  • 根据文件命名格式，生成一个或多个点云。

  • 在世界空间中烘焙

烘焙图像文件格式

通常，你会用令牌创建一个模式来生成命名 AOV，但这里有一点不同。默认文件模式为：

<scene>_<user:bakingIdentifier>_<aov>.<ext>

user:bakingIdentifier 是一个新属性，可在 Attribute Editor 的 RenderMan 区段中所有可烘焙形状上找到：

它的默认值为 <shapepath> 因此每个对象都会获得唯一文件名，其中 "|" 字符会被 "_" 替换；当你烘焙到 2D 图像时，这正是所需行为。

如果想烘焙到单个 3D 点云，应从文件名模式中移除 <user:bakeIdentifier>。

烘焙到贴图（2D 烘焙）

选择全局烘焙分辨率。更大的分辨率会需要更长渲染时间。

限制

所有对象都会以相同分辨率烘焙。这是当前实现的限制。

烘焙到点云（3D 烘焙）

点云会在世界空间中烘焙，其密度取决于形状的细分密度。在此为场景选择全局细分密度。

你可以通过将 Dicing Strategy 设置为 worlddistance 并输入不同的 Dicing Distance Length，按形状覆盖细分密度。较小值会创建更多点。

Integrator（积分器）

选择一个积分器及其设置。我们推荐 PxrPathTracer。

限制

某些积分器（如 PxrVCM 和 PxrUnified） 不兼容，因为它们完全依赖增量渲染来累积样本，并随时间解析照明。

烘焙会运行积分器 每个点一次（Max Samples = 1），并且它必须设置为 在单次增量中提供良好结果。

为获得最佳结果，需要充分增加直接样本（灯光样本和 BXDF 样本）以及间接样本的数量，以得到足够干净的结果。最佳设置高度依赖场景，需要通过实验确定。

限制

Baking 是 非自适应采样模式（即暴力采样），因此无论收敛情况如何，都会计算所有样本。

照明烘焙可能非常慢：请确保你 选择最小必要样本数量，并记住已烘焙照明中的少量噪声几乎不可见。

请注意，烘焙与视角相关的照明（如镜面反射）没有意义；应始终尽量烘焙漫反射、albedo 等与视角无关的信号。

限制

目前无法在已烘焙的 2D 图像上运行降噪器。

Default Ray Depth（默认射线深度）

照常可以设置 max diffuse depth 和 specular depth。这会在烘焙期间覆盖常规场景设置。

AOVs

AOV 区段的工作方式与渲染全局设置中完全相同。

默认情况下，我们会烘焙漫反射照明，包括直接照明和间接照明。

如果需要烘焙原始照明，还需要同时烘焙 albedo，并将照明除以 albedo（在合成软件包或着色器中），以获得原始照明（也称为 irradiance）。

限制

每个 AOV（也称为 Display）只能包含一个显示通道，最多输出 4 个通道（RGBA）。

添加更多 display 可烘焙更多通道。

Bake 2D

任何非 deep 图像格式都可用于烘焙到图像。推荐使用 OpenEXR，以避免结果被钳制。

Bake 3D

请确保选择 d_pointcloud 显示驱动来烘焙点云。

如果忘记这样做，预检应会发出警告并取消烘焙渲染。


启动烘焙渲染

烘焙渲染本质上是一个只有一项设置不同的批渲染。它使用 bake Hider，而不是 raytrace Hider。

要启动烘焙渲染，只需选择 Bake Render: Illumination，而不是 RenderMan 菜单中的 Batch Render。所有常规 批渲染首选项 都会应用到烘焙渲染。
