Enabling Holdouts in Maya（在 Maya 中启用 Holdouts）

可以通过选择 holdout geometry，并点击 RenderMan shelf 中的 Holdout 按钮 来启用 RenderMan Holdouts。

这会产生两个效果。它会启用所选 shape 的 Hold-Out 属性：

同时还会启用 Advanced render settings 下的 Output Shadow AOV 选项：

可以在 Maya 中使用 image planes 图像平面对象 并启用 Image Plane As Sample Filter 高级渲染设置 来 visualize holdouts。此 setting 默认启用。

若要关闭 Holdout AOVs（阻止 output），可以如上所述在 "Output Shadow AOV" 中选择 "None"。


About Holdouts（关于 Holdouts）


VFX artists 经常需要将 CG objects composite 到 live action film plate 上。为了让观众相信 CG objects 真实存在并属于该 scene，需要使用 holdouts 来收集 CG objects 的 shadow 和 reflection，使其与 live action plate 中的 shadow 和 reflection 非常接近。

Geometry（几何体设置）

Holdout objects 用于代替 real world object，以捕捉来自 3D objects 的 light interactions。在 RenderMan 中，通过启用 trace holdout attribute 属性来指定 holdout objects。

Attribute "trace" "int holdout" [1]

Integrators and AOVs（积分器和 AOVs）

Holdouts 是在 integrator level 实现的 illumination feature。它们能够捕捉多种 integrator-specific illumination effects，例如 diffuse 和 specular reflections、transmissions 以及 caustics。支持 holdouts 的可用 integrator 包括： PxrPathTracer, PxrVCM，以及 PxrUnified。

对于 holdout geometry，LPE workflow 保持不变。

由于 holdouts workflow 涉及多个 AOVs，在 rendering 时启用 adapt all 全部自适应 可以降低 noise，但代价是速度较慢，因为它可能会根据 noise 更高的 AOVs 触发更多 samples。

Lights（灯光）

默认情况下，light sources 会被视为 holdouts，并且不会直接 illuminate 其他 holdout objects。这可以防止用于 illuminate non-holdout geometry 的 lights 与 plate image 上已经捕获的 illumination 之间产生 double contribution。

若要创建会直接 illuminate holdout objects 的 light source，请使用 "__illumholdout" 作为 light group suffix。这会告诉 integrator plug-ins，在确定 illumination 时将此 light 视为 non-holdout light source。

Light "PxrSphereLight" "spherelight" "string lightGroup" ["light__illumholdout"]

Alpha, Shadows, and Compositing（Alpha、阴影和合成）

Holdout contribution 可以使用 alpha 和 shadow pass 作为 masks，composite 到 film plate 上。shadow pass 使用 PxrShadowDisplayFilter plug-in 生成，并且可以选择写入自己的 AOV，或添加到 alpha 中。occluded 和 unoccluded input AOVs 应使用 holdouts LPE prefix 进行 rendering，使它们只包含 holdout geometry 的 shadowing term。

DisplayChannel "color shadow"
DisplayChannel "color occluded" "string source" ["color lpe:holdouts;C[DS]+[LO]"]
DisplayChannel "color unoccluded" "string source" ["color lpe:holdouts;unoccluded;C[DS]+[LO]"]

在 RenderMan 中，可以通过使用 PxrImageDisplayFilter plug-in，在 RenderMan 中 interactively preview compositing。它会使用 alpha channel 和 holdout shadow AOV 作为 mask，将 film plate 与 beauty channel composite 在一起。

Compositing Workflow（合成工作流）

compositing image 时，如果 shadow matte 是 beauty 的一部分，那么它就是一个简单的 over operation。这会根据 render settings 提供正确的 shadow density 和 look。

不过，这意味着调整 shadow density 会与你的 beauty alpha 绑定；你需要一个 mask，防止 shadow adjustments 破坏 CG object composite。Cryptomatte 可能可以作为你常用工具集的一部分处理这一点。

如果单独 output shadow matte，就可以单独调整 shadow density；不过可能会遇到 coverage issue。解决方法是在 comp 中将 beauty 和 shadow matte 的 alphas merge 到一起。使用 Nuke 的示例在 这里。
