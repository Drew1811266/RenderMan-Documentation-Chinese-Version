Batch Rendering in Maya

启动批渲染

批渲染可从 RenderMan Menu 菜单入口。也可以使用 Maya 的 Batch Render 菜单项。

停止批渲染

批渲染作业可以在 Local Queue 或 Tractor 中删除或暂停。右键点击列表中的作业可打开菜单。

Render Settings

Render Settings 窗口中的设置会用于批渲染。尤其是 Frame Range 和 Renderable Cameras 会在 Common 标签页下指定。对于预览渲染，则改用当前帧或活动摄像机。

批渲染排队

批渲染始终会提交到 LocalQueue（默认）或 Tractor。   批渲染排队首选项 可在 Maya Preferences 窗口中找到；选择 Batch Render 菜单项的 option box 时会打开该窗口。请注意，提交方式会决定是先导出到 RIB 文件再渲染，还是直接通过 Maya Batch 渲染。

输出位置

默认情况下，我们使用 Mayabatch 渲染已保存的 Maya 场景文件。Maya 会从内存渲染，类似于从 UI 发起的预览渲染，不进行中间 RIB 导出。

启动批渲染时，RIB 文件和纹理会存储在磁盘上，渲染器会读取这些内容。它们会存储在一个基于场景名称、并追加时间戳的目录中。首选项里的 Batch Context 设置用于指定这个时间戳，其变量名为 $JOBDATETIME。这可以避免多个并发作业尝试覆盖文件。

输出位置在 RenderMan for Maya 的 Workspace tab 中配置。

Local Queue

LocalQueue 是用于运行渲染作业的本地队列。它会读取  作业脚本文件  这些脚本由 RenderMan for Maya 生成，然后运行脚本中的命令，在本地计算机上渲染你的场景。

渲染作业加载到 LocalQueue 后，你应能在 Jobs 作业表格 中看到它。你可以右键点击作业，打开包含多种作业管理选项的菜单。选择作业后，它的命令会显示在 Job Commands 作业命令列表表格 中。

检查点保存

默认情况下， 检查点保存 会为批渲染启用。图像每五分钟更新到磁盘。检查点间隔可在 批渲染首选项中配置。Incremental mode 默认启用，位于 Render Settings 的 Sampling 标签页下。这是检查点保存正常工作所必需的。

从命令行进行 Maya 批渲染

你也可以使用额外标志：-rl（render layer）、-crop、-preRender、-postRender、-preLayer、-postLayer、-preFrame、-postFrame、-jobid

在命令行中使用以下命令：

Render -r renderman sceneFile

也可以只生成 RIB，而不随后渲染。

Render -r renderman -rib sceneFile

也可以运行以下命令查看完整选项列表：

Render -r renderman -h

如果收到类似以下警告，需要将 RenderMan for Maya 安装目录中的 rmanRenderer.xml 和 ribRenderer.xml 放到 Maya 能找到的位置。

警告消息示例：Cannot open renderer description file "rendermanRenderer.xml"

你可以从 RenderMan for Maya 安装目录复制这些文件，

eg. C:/Program Files/Pixar/RenderManForMaya-22.0/etc/rendermanRenderer.xml

放入 Maya 安装目录中 Maya 查找这些文件的目录，

eg. C:/Program Files/Autodesk/Maya2018/bin/rendererDesc/

或者，你可以设置名为 MAYA_RENDER_DESC_PATH 的 Maya 环境变量，以便找到 rmanRenderer.xml 和 ribRenderer.xml。

从命令行进行 Prman 渲染（高级）

执行 Maya 批量 渲染时，RenderMan for Maya 会生成 RIB 文件，然后针对这些 RIB 文件启动 prman（可执行程序）。不过，你可能已经有磁盘上的 RIB 文件，只想对它们运行 prman。

作业结构

RenderMan for Maya 生成的 RIB 文件位于 Maya 项目内。例如：

<maya_project>/renderman/myscene/rib/

RIB 文件会按每个帧组织到子目录中，如 0001、0002、0003。另有一个名为 "job" 的子目录，用于静态对象的缓存，以及不需要每帧执行的处理命令，例如转换纹理和清理。

下面是一个三帧作业通常会执行的命令示例。请注意，这里使用 -cwd 参数将 Maya 项目指定为 prman 运行时的当前工作目录。RIB 文件的项目相对路径作为命令的最后一个参数提供。RIB 文件默认包含项目相对路径。

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/job/job.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/0001/0001.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/0002/0002.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/0003/0003.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/job/post.rib

每个帧 RIB 文件（如 0001.rib）都会引用位于同一目录中的其他 RIB 文件，这些文件对应于该帧中处于活动状态的每个摄像机或渲染层。下面是帧目录中典型内容的示例：

renderman/test_0725120019/rib/0001:

0001.rib

perspShape_Final.0001.rib

perspShape_Final.0001.rlf

perspShape_Final.0001.xml

名为 0001.rib 的文件是该帧的 "driver" RIB 文件。它会引用该帧中每个通道（用于不同摄像机或渲染层）的 RIB 文件。

目录中有一些文件不是 RIB 文件。它们是什么？

用户制作的 RLF 文件（分配给 GPU archive）包含材质和绑定信息。材质会在渲染时由 RIF 注入到 RIB 文件中。

XML 文件会在渲染期间生成。它包含关于渲染的诊断信息。
