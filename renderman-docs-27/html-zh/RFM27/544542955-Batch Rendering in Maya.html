Batch Rendering in Maya

Downloaded: 2026-06-15T03:49:12+00:00

Starting a Batch Render（启动 Batch Render）

Batch renders 的启动位置是 RenderMan Menu 菜单入口。Maya 的 Batch Render menu entry 也可使用。

Stopping a Batch Render（停止 Batch Render）

Batch renders 可以在 Local Queue 或 Tractor 中 delete 或 pause。右键点击列表中的 job 可打开 menu。

Render Settings

Render Settings window 中的 settings 会用于 batch renders。尤其是 Frame Range 和 Renderable Cameras 会在 Common tab 下指定。对于 preview renders，则改用 current frame 或 active camera。

Batch Queuing（批量渲染队列）

Batch renders 总会 spool 到 LocalQueue（default）或 Tractor。   Batch queuing preferences（批量队列首选项） 可在 Maya Preferences window 中找到；选择 Batch Render menu item 的 option box 时会打开该 window。请注意，spool style 定义是先 export 到 RIB files 再 render，还是直接通过 Maya Batch render。

Output Locations（输出位置）

默认情况下，我们使用 Mayabatch 来 render 已保存的 Maya scene file。Maya 会从 memory 渲染，类似于从 UI 发起的 preview render，不进行 intermediate RIB export。

启动 batch render 时，RIB files 和 textures 会存储在 disk 上，renderer 会读取这些内容。它们会存储在一个基于 scene name、并追加 time stamp 的 directory 中。preferences 里的 Batch Context setting 用于指定这个 time stamp，其变量名为 $JOBDATETIME。这可以避免多个 simultaneous jobs 尝试 overwrite files。

Output locations 在 RenderMan for Maya 的 Workspace tab 中配置。

Local Queue

LocalQueue 用于运行 render jobs 的 local queue。它会读取  job scripts 作业脚本文件  这些 scripts 由 RenderMan for Maya 生成，然后运行 scripts 中的 commands，在 local computer 上 render 你的 scene。

render job 加载到 LocalQueue 后，你应能在 Jobs 作业表格 中看到它。你可以右键点击 job，打开包含多种 job 管理选项的 menu。选择 job 后，它的 commands 会显示在 Job Commands 作业命令列表表格 中。

Checkpointing（检查点保存功能）

默认情况下， checkpointing 检查点功能 会为 batch renders 启用。Images 每五分钟更新到 disk。checkpoint interval 可在 batch preferences 批量首选项中配置。Incremental mode 默认启用，位于 Render Settings 的 Sampling tab 下。这是 checkpointing 正常工作所必需的。

Maya Batch Rendering from the Command Line（从命令行进行 Maya Batch Rendering）

你也可以使用 additional flags：-rl（render layer）、-crop、-preRender、-postRender、-preLayer、-postLayer、-preFrame、-postFrame、-jobid

在 command line 中使用以下命令：

Render -r renderman sceneFile

也可以只生成 RIB，而不随后 rendering。

Render -r renderman -rib sceneFile

也可以运行以下命令查看完整 options list：

Render -r renderman -h

如果收到类似以下 warning，需要将 RenderMan for Maya installation 中的 rmanRenderer.xml 和 ribRenderer.xml 放到 Maya 能找到的位置。

警告消息示例：Cannot open renderer description file "rendermanRenderer.xml"

你可以从 RenderMan for Maya installation 复制这些 files，

eg. C:/Program Files/Pixar/RenderManForMaya-22.0/etc/rendermanRenderer.xml

放入 Maya installation 中 Maya 查找这些 files 的 directory，

eg. C:/Program Files/Autodesk/Maya2018/bin/rendererDesc/

或者，你可以设置名为 MAYA_RENDER_DESC_PATH 的 Maya environment variable，以便找到 rmanRenderer.xml 和 ribRenderer.xml。

Prman Rendering from the Command Line (Advanced)（从命令行进行 Prman 渲染，高级）

执行 maya batch render 时，RenderMan for Maya 会生成 RIB files，然后针对这些 rib files 启动 prman（executable）。不过，你可能已经有 disk 上的 RIB files，只想对它们运行 prman。

Job Structure（作业结构说明）

RenderMan for Maya 生成的 RIB files 位于 maya project 内。例如：

<maya_project>/renderman/myscene/rib/

RIB files 会按每个 frame 组织到 subdirectories 中，如 0001、0002、0003。另有一个名为 "job" 的 subdirectory，用于 static objects 的 caches，以及不需要每帧执行的 processing commands，例如 converting textures 和 cleanup。

下面是一个 three frame job 通常会执行的 commands 示例。请注意，这里使用 -cwd arg 将 maya project 指定为 prman 运行时的 current working directory。rib file 的 project relative path 作为 command 的最后一个 argument 提供。Rib files 默认包含 project relative paths。

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/job/job.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/0001/0001.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/0002/0002.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/0003/0003.rib

prman -t:0 -cwd C:/Users/user/Documents/maya/projects/default/ renderman/test_0725120019/rib/job/post.rib

每个 frame rib file（如 0001.rib）都会引用位于同一 directory 中的其他 rib files，这些 files 对应于该 frame 中 active 的每个 camera 或 render layer。下面是 frame directory 中典型内容的示例：

renderman/test_0725120019/rib/0001:

0001.rib

perspShape_Final.0001.rib

perspShape_Final.0001.rlf

perspShape_Final.0001.xml

名为 0001.rib 的 file 是该 frame 的 "driver" RIB file。它会引用该 frame 中每个 pass（用于不同 cameras 或 render layers）的 RIB files。

directory 中有一些 files 不是 RIB files。它们是什么？

用户制作的 RLF files（分配给 GPU archives）包含 material 和 binding information。Materials 会在 render time 由 RIF 注入到 RIB file 中。

XML file 会在 rendering 期间生成。它包含关于 render 的 diagnostic information。
