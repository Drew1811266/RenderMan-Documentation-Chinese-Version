使用 PxrOSL

Source: https://rmanwiki-27.pixar.com/space/RFM27/544541164/Using+PxrOSL
Downloaded: 2026-06-15T03:49:12+00:00

PxrOSL Pattern 节点是在 Maya 内部创建 OSL 图案并进行原型设计的有用方式。它非常适合试验各种选项和控件。

我们只支持 OSL 图案（用于着色的纹理），不支持闭包（材质）。

首次打开节点时，会看到上方所示的界面。如果展开 Source Code 卷展栏，会看到节点内置的实用示例。

可以在 Compile 和 Reload 按钮下看到源路径。

这里的命名非常重要。

  • shader sample_osl_pattern 是在 OSL Code 中定义的。

  • OSL Code 中定义的名称 必须匹配 .osl Source File Path 的名称

  • 文件路径必须手动填写，并且也必须与 OSL Code 名称匹配，只是使用编译后的 .oso 扩展名

如果名称不正确，会收到警告，并出现异常行为。

Compile - 将当前代码编译为用于渲染的 .oso 文件

  • 选择此选项时，它会询问是否覆盖你的文件，还是保存为编辑副本；编辑副本的名称中会追加 .edit

  • .oso 编译文件是渲染所必需的


Reload Source File - 可以从已保存/更新的 .osl 文件加载对文件的外部编辑

加号和减号图标用于展开和收缩编辑器区域。

只要提供正确的 .oso 文件，Parameters 卷展栏就会填充。

使用自定义 OSL 图案

首先，请查看 安装自定义节点。

定义 RFM_SHOW_PATH 环境变量，让它指向你自己的自定义配置目录。

在这个新位置创建 config 目录，并保存自定义 rfm.json 文件（可以复制原始文件后修改）。下面是一个简单版本：

{
    "$schema": "./rfmSchema.json",
    "resources": {
        "user": {
            "nodes": {
                "search_paths": ["/Users/Artist/Shading/OSL_pkg/shader/myshow"]
            }
        }
    }
}

你应具有以下结构：

RFM_SHOW_PATH
   |_ config
      |_ rfm.json

现在给你的 OSL 着色器添加一些元数据（下面使用一个简单示例）：

//
// AddFloat_1
//
shader AddFloat_1
[[
    string category = "math",
    string help = "Adds two float inputs",
    int rfm_nodeid = 1553001,
    string rfm_classification = "rendernode/RenderMan/pattern/Studio/math"
]]
(
    float input_A = .0,
    float input_B = .0,
    output float resultF = .0 [[ string widget="null" ]],
)
{
    resultF = input_A + input_B;
}

以下是 Help 数据的各个部分：

  • category - 我们将它归类为 "math" 节点类别

  • help - 提供给用户的有用数据，基本上就是描述。

  • rfm_nodeid - 是节点的唯一 ID；Autodesk 通常按区块分配这些 ID。这是避免冲突所必需的。

  • rfm_classification - 是 UI/Viewport 的分类，'shader' 对象必须使用 'rendernode/RenderMan/pattern/' 下的这个分类才能被处理
