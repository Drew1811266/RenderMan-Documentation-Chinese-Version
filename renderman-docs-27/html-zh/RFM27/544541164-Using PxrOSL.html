Using PxrOSL

Source: https://rmanwiki-27.pixar.com/space/RFM27/544541164/Using+PxrOSL
Downloaded: 2026-06-15T03:49:12+00:00

PxrOSL Pattern node 是一种有用方式，可在 Maya 内部创建 OSL patterns 并进行原型设计。它非常适合试验各种 options 和 controls。

我们只支持 OSL patterns（用于 shading 的 textures），不支持 closures（materials）。

首次打开 node 时，会看到上方所示的表示。如果展开 Source Code rollout，会看到 node 内置的实用示例。

可以在 Compile 和 Reload buttons 下看到 source path。

这里的命名非常重要。

  • shader sample_osl_pattern 在 OSL Code 中定义。

  • OSL Code 中定义的名称 必须匹配 .osl Source File Path 的名称

  • file path 必须手动填写，并且也必须与带有编译后 .oso extension 的 OSL Code 名称匹配

如果名称不正确，会收到 warning，并出现异常行为。

Compile - 将当前 code 编译为用于 rendering 的 .oso file

  • 选择此选项时，它会询问是否覆盖你的文件，还是保存为 edit copy；edit copy 的名称中会追加 .edit

  • .oso compiled file 是 rendering 所必需的


Reload Source File - 可以从已保存/更新的 .osl file 加载对文件的外部 edits

加号和减号图标用于展开和收缩 editor area。

只要提供正确的 .oso file，Parameters rollout 就会填充。

Using Custom OSL Patterns（使用 Custom OSL Patterns）

首先，请查看 Installing Custom Nodes（安装 Custom Nodes）。

定义 RFM_SHOW_PATH environment variable，让它指向你自己的 custom（自定义）configuration directory。

在这个新位置创建 config directory，并保存一个 custom（自定义）rfm.json 文件（可以复制原始文件后修改）。下面是一个简单版本：

{
    "$schema": "./rfmSchema.json",
    "resources": {
        "user": {
            "nodes": {
                "search_paths": ["/Users/Artist/Shading/OSL_pkg/shader/myshow"]
            }
        }
    }
}

你应具有以下结构：

RFM_SHOW_PATH
   |_ config
      |_ rfm.json

现在给你的 OSL shaders 添加一些 metadata（下面使用一个简单示例）：

//
// AddFloat_1
//
shader AddFloat_1
[[
    string category = "math",
    string help = "Adds two float inputs",
    int rfm_nodeid = 1553001,
    string rfm_classification = "rendernode/RenderMan/pattern/Studio/math"
]]
(
    float input_A = .0,
    float input_B = .0,
    output float resultF = .0 [[ string widget="null" ]],
)
{
    resultF = input_A + input_B;
}

以下是 Help data 的各个部分：

  • category - 我们将它归类为 "math" node category

  • help - 提供给用户的有用数据，基本上就是 description。

  • rfm_nodeid - 是 node 的唯一 ID；Autodesk 通常按区块分配这些 ID。这是避免冲突所必需的。

  • rfm_classification - 是 UI/Viewport 的 classification，'shader' objects 必须使用 'rendernode/RenderMan/pattern/' 下的这个 classification 才能被处理
