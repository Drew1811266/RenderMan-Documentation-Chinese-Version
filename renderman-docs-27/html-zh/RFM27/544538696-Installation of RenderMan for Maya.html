Installation of RenderMan for Maya

Source: https://rmanwiki-27.pixar.com/space/RFM27/544538696/Installation+of+RenderMan+for+Maya
Downloaded: 2026-06-15T03:49:12+00:00

有关支持的 Maya versions 列表，请参阅 Installation and Licensing（安装与许可）

RenderMan for Maya 使用 RenderMan Installer进行安装；它会管理 RenderMan Pro Server（实际 renderer）以及 RenderMan for Maya 和/或 RenderMan for Katana 的下载、安装和 licensing。

在 Windows 和 macOS 上，installer 会配置 RenderMan for Maya。安装后，你应该可以直接使用。Start Maya，RenderMan for Maya 应该会出现在 Plug-in Manager 的 plug-ins 列表中：RenderMan_for_Maya.py

在 Linux 上，需要进行一些 manual configuration（见下文）。

 在 Windows 和 macOS 上，你不需要手动配置 RenderMan for Maya plug-in。以下说明面向 advanced users。

Apple Silicon

想在 ARM Machines（M1、M2、Apple silicon）上加载 RfM 吗？

首先，请确保从 terminal 安装 Rosetta：

/usr/sbin/softwareupdate --install-rosetta

安装 Rosetta 后，只需 right click Maya app 并点击 “get info”，然后即可点击 "open with rosetta"。

你也可以从 shell 以 Rosetta mode 启动 Maya，只需在 terminal 中输入此 command：

arch -x86_64 /Applications/Autodesk/maya2024/Maya.app/Contents/MacOS/Maya


Rendering with Xgen（使用 Xgen 渲染）

请注意，users 可能需要为 Xgen 设置正确的 path variables 才能 render。

  • %MAYA_PATH%/bin

  • %MAYA_PATH%/lib

  • %MAYA_PATH%/plug-ins/xgen/bin

  • %MAYA_PATH%/plug-ins/xgen/lib

  • %RMANTREE%/lib;

其中 MAYA_PATH 是你的 Maya install location。

在 update/change versions 时，如果上述 settings 不再工作，你可能需要 reset/remake 你的 Maya Preferences。

对于下面的每个 OS，应将其添加到以下 system environment variables：

Operating System

Environment Variable

Windows

PATH

Linux

LD_LIBRARY_PATH

macOS

DYLD_LIBRARY_PATH

BiFrost 及其 data exports（包括 default Alembic attributes）预计无法正确工作，并且在此 version 中不受支持。

 Manual Configuration（手动配置）

  • RenderMan_for_Maya.module file 必须包含 RenderMan for Maya installation 的 path。

  • Maya 必须能在其搜索 modules 的某个 location 中找到 RenderMan_for_Maya.module file。

  • Maya 的 command line Render command 需要 Maya 能找到 rmanRenderer.xml。

在 Windows 和 macOS 上，installer 会处理这些事项；但如果由于某些原因需要调整，请继续阅读。

Install Location（安装位置）

以下是 RenderMan for Mayadefault install location 的 examples：

Windows

C:\Program Files\Pixar\RenderManForMaya-26.2

macOS

/Applications/Pixar/RenderManForMaya-26.2

Linux

/opt/pixar/RenderManForMaya-26.2

以下是 RenderMan Pro Serverdefault install location 的 examples：

Windows

C:\Program Files\Pixar\RenderManProServer-26.2

macOS

/Applications/Pixar/RenderManProServer-26.2

Linux

/opt/pixar/RenderManProServer-26.2

RenderMan for Maya 需要 RenderMan Pro Server，后者提供实际的 rendering engine。

RenderMan for Maya 会遵循名为 RFMTREE 和 RMANTREE的 environment variables，它们可用于指定 RenderMan for Maya 和 RenderMan Pro Server installations 的 location。如果未设置这些 variables，则会进行推断。

Module Location（模块位置）

以下示例展示了 RenderMan_for_Maya_23.0.module file 可以放置的位置，以便 Maya 找到它（请按需调整 versions）：

Windows

C:\Program Files\Common Files\Autodesk Shared\Modules\Maya\

~/Documents/maya/2018/modules

macOS

/Applications/Autodesk/maya2018/Maya.app/Contents/modules

~/Library/Preferences/Autodesk/maya/2018/modules

Linux

/usr/autodesk/maya2018/modules

~/maya/2018/modules

或者，你可以将 module file 放在 non-standard location，并设置 MAYA_MODULE_PATH environment variable。在 Maya 的 Script Editor 中运行 getenv MAYA_MODULE_PATH，可查看 Maya 检查 modules 的 standard locations。

Module Contents（模块内容）

module file 应包含指向 install location 的 path。以下是不同 platforms 上该 file contents 的 examples（请按需调整 versions）：

Windows

+ RenderMan_for_Maya 23.0 C:/Program Files/Pixar/RenderManForMaya-23.0

macOS

+ RenderMan_for_Maya 23.0 /Applications/Pixar/RenderManForMaya-23.0

DYLD_FALLBACK_LIBRARY_PATH+=/Applications/Autodesk/maya2017/plug-ins/xgen/lib

Linux

+ RenderMan_for_Maya 23.0 /opt/pixar/RenderManForMaya-23.0

注意，在 macOS 上，使用 XGen 的 batchrenders 可能需要设置 DYLD_FALLBACK_LIBRARY_PATH；这可以通过 module file 完成，也可以作为 environment variable 设置。

你可以设置以下 environment variable 来禁用创建额外 mod files：RFM_DO_NOT_CREATE_MODULE_FILE = 1

如果 RfM 在 default location 中找不到 mod files，这会阻止它创建更多 mod files。如果找不到任何 valid mod files，它会警告 user，并且 plugin 可能无法加载。

Render Command

要让 Maya 的 Render command 与 RenderMan 配合工作，需要将 $RFMTREE/etc/rmanRenderer.xml 复制或 symlink 到 Maya 的 rendererDesc directory 中。  

例如

sudo ln -s /opt/pixar/RenderManForMaya-23.0/etc/rmanRenderer.xml /usr/autodesk/maya2018/bin/rendererDesc/rmanRenderer.xml

或者，你可以设置 MAYA_RENDER_DESC_PATH environment variable，使其指向可找到 rmanRenderer.xml 的 directory。

 Troubleshooting（故障排查）

RenderMan for Maya 未列在 Maya 的 Plug-in Manager 中

Maya 没有找到 RenderMan_for_Maya.module file，或 RenderMan_for_Maya.module 中列出的 path 未指向有效的 RenderMan for Maya install。

  • 检查 module file 是否位于 Maya 的 Script Editor 中输入 "getenv MAYA_MODULE_PATH" 时返回的某个 location 中。

  • 检查 file 中的 path 是否看起来正确，并确认 install directory 实际存在。RenderMan_for_Maya plug-in 必须位于 install location 下的 plug-ins directory 中。注意，Maya 会去 installation 的 plug-ins sub-directory 中查找。module file 中列出的 path 实际上不应包含 plug-ins sub-directory。

RenderMan for Maya 无法加载

你会看到以下任一 errors：

// Error: line 1: The specified procedure could not be found.
(RenderMan_for_Maya) //

// Error: file: C:/Program Files/Pixar/RenderManForMaya-23.0-maya2018/plug-ins/RenderMan_for_Maya.py:

Unable to dynamically load : C:/Program Files/Pixar/RenderManForMaya-23.0/lib/2018/rfm.mll

The specified procedure could not be found.

此 error 的常见原因是尝试将 plug-in 加载到错误的 Maya version 中。请检查 error string 中的 version 是否与你正在运行的 Maya version 匹配。尤其是在安装了多个 Maya versions 时，RenderMan Installer 可能只下载了适用于最新 Maya version 的 plug-in。
