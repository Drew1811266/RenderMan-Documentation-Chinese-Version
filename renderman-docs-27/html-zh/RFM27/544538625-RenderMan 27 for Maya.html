RenderMan 27 for Maya

Source: https://rmanwiki-27.pixar.com/space/RFM27/544538625/RenderMan+27+for+Maya
Downloaded: 2026-06-15T03:49:12+00:00



RenderMan for Maya (RfM) 可通过以 Maya 为中心的工作流（包括 Maya 风格的 pattern 节点等），或结合其他工具，轻松使用 RenderMan 功能。

该插件还以新的方式连接 Maya 与 RenderMan Pro Server，无需编写 RIB 即可获得快速的交互式结果。RIB 输出功能会继续支持现有管线和渲染农场渲染。RfM 为 Maya 用户提供对 RenderMan 速度、能力和稳定性的无缝访问；他们既可以保持以 Maya 为中心的简单工作流，也可以利用插件的灵活性创建最适合自身需求的定制管线。


安装

RenderMan for Maya 需要同时安装 RenderMan Pro Server 和 RfM 插件才能正确运行。RenderMan 安装程序会自动完成这一点。


"it"

"it" 是一个可靠的帧缓冲/渲染视图窗口，提供完整浮点支持和强大灵活的目录功能，同时也是一个快速强大的图像处理工具，能够执行通常只有高端独立产品才具备的生产级图像调整与合成。得益于它与插件的集成以及全新的 Python 脚本接口，"it" 让用户能够把复杂的渲染后图像操作纳入自己的渲染管线。

RenderMan 功能

RenderMan for Maya 支持 Maya 原生不具备的高级渲染功能。由于 Maya 不直接支持这些高级功能，RfM 提供了自己的方法来创建和控制这些效果。这些方法通常很简单，只需把 RenderMan 专用属性附加并配置到几何体、灯光和/或材质上。Photo Realistic RenderMan (PRMan) 的设计目标是快速且易用，同时生成生产级渲染；全局照明开箱即用，交互式渲染可以为艺术家提供快速迭代。

真正的交互式渲染

无需启动新的渲染，就能完成更多工作。你的更改会立即从 Maya 渲染出来，不需要中间阶段，也不需要导出到 RIB。你可以更改参数、导入或更新资产、更新选项，甚至在查看结果更新的同时更改 LPE。

渲染到 Viewport 2.0

选择 RenderMan 作为渲染器，即可直接向 Maya 视口进行交互式渲染。无需重新启动渲染，就可以修改场景、编辑参数等。

改进的 AOV 与 LPE 交互式创建

用于创建 AOV 和自定义 LPE 的简化界面，是定义后期处理工作流所需输出的单一位置；这些输出会在实时渲染期间更新。

解析区域光

区域光是在优化采样预算并进而提升性能的同时，创建物理准确照明的强大方式。PRMan 的 区域光 支持 灯光滤镜，以及灯光链接、分组、按分组 AOV、IES 配置文件和门户光。

全局照明

有时也称为“间接照明”，RenderMan for Maya 支持全局照明效果，也就是从给定点对场景照明进行多次采样而产生的细腻漫反射着色效果。该技术可以得到高度真实的结果。Occlusion 只采样某个点的“覆盖率”（并且比间接照明更便宜），也会自动支持，或通过 PxrOcclusion 积分器支持。

体积渲染

RfM 包含 PRMan 一流的 体积 渲染能力，可使用 OpenVDB 5.0 创建体积效果，包括 Maya 的 Fluid Effects，并提供完整的 PRMan 支持：运动模糊、多次散射、非均匀介质等。

真正的曲面

使用 RenderMan 渲染 NURBS 和 细分 曲面时，曲面细分设置从来不是问题。由于 RenderMan 会渲染真正的曲面，NURBS 和细分曲面永远不会出现切面伪影。Maya 的曲面细分模式设置会被直接忽略，RenderMan 的优势会立即体现出来。

次表面散射

RenderMan for Maya 支持次表面散射，这是真实渲染半透明材质的重要效果，例如皮肤、肌肉、脂肪、水果、牛奶、大理石以及许多其他材质。次表面散射负责材质内部的颜色渗透，或阴影边界之间的光扩散等效果。RfM 通过 RenderMan 随附的材质支持路径追踪次表面散射。

二级输出

RenderMan for Maya 支持二级输出（AOV，即 Arbitrary Output Variables，任意输出变量），它们可以是着色器中包含的任意信息。用户还可以利用光路表达式（LPE）对输出进行强力控制。

电影级 RenderMan 着色器

RenderMan 随附 Pixar 在长片电影中使用的同一套 着色器。这让艺术家可以把 RenderMan 的全部能力带入自己的项目，并提供比 Maya 内置材质更高层级的控制。

Ri Procedurals（Ri 程序化几何）

作为基于 RIB 的管线基础，RenderMan for Maya 支持各种程序化几何。此外，RenderMan for Maya 还为程序化几何提供简单支持。


进入生产管线

RenderMan for Maya 是潜在复杂生产管线中的一环。为了把作品从脚本带到银幕，你可能还需要更多工具，这时 Tractor 和 RenderMan Pro Server (RPS) 就会登场。通过将 RfM 与 Tractor 和 RPS 配合部署，用户可以实现从桌面到渲染农场的无缝集成渲染工作流。

Tractor

Tractor 是一个系统，用于把排队作业中的任务分发到网络中的计算服务器。它包括中央排队机制、远程执行服务器、监控子系统和基于浏览器的用户界面。Tractor 被设计为 Alfred 的高性能替代方案，可用于任何工作分发场景，但尤其适合管理从 RenderMan for Maya 到 RenderMan Pro Server 的管线。

Tractor 作为单独的独立产品提供，并拥有自己的文档。

RenderMan Pro Server

RenderMan Pro Server 是高性能渲染管线的基础。它提供渲染器及其对应渲染工具的独立实现，并提供 API，让熟练用户能够扩展功能。作为核心技术，RenderMan Pro Server 是所有像素流出的源头。
