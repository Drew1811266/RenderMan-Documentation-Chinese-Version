Render（渲染）

Threads（线程）

Rendering Threads（渲染线程）

用于预览或 IPR 渲染的线程数量。值为 0 表示使用所有处理器。负值表示使用所有处理器减去指定数量。默认值 -2 可让用户界面在连续渲染期间保持响应。

Batch Threads（批量线程）

用于批量渲染的线程数量。

IPR Display（IPR 显示）

Render To（渲染到）

IPR 渲染使用的显示目标。选择 "It" 或 Render View。

Output AOVs（输出 AOVs）

Write out（写出）交互式会话期间的 AOVs/LPE。默认关闭此项以提升性能。

Prune Invisible Nodes（裁剪不可见节点）

在 IPR 期间，我们只裁剪模板化节点，以便其他对象可以交互式隐藏/取消隐藏。不过，为了节省转换时间，你也可以让 RfM 在 IPR 期间裁剪隐藏节点。

Batch Render（批量渲染）

在 LocalQueue 和 Tractor之间选择。下方有一个单独章节用于 Tractor 队列设置。

Spool To（派发到）

选择 LocalQueue 或 Tractor

Spool Style（派发方式）

立即从 Maya Batch 渲染，或在本地生成 RIB 文件以分发到渲染农场节点。

Frames Per Server（每台服务器帧数）

远程服务器上每个分块、每次按序处理多少帧。默认值为 5。

Checkpoint（检查点）

在批量渲染中使用检查点，选择写入检查点前的分钟数或迭代次数。你可以使用 Checkpointing Document（检查点文档）中 "How to Specify"（“如何指定”）一节所述的相同语法来控制此选项。你可以指定从单个渲染增量到秒、分钟、小时和天的任意单位；也可以指定检查点时间后的退出时间，或不写检查点、仅在给定时间退出。字段上的帮助文本弹窗会提供示例，例如： 5m,1h 表示每 5 分钟创建一个检查点，并在 1 小时后退出，同时写出最终结果。

Launch Denoiser（启动降噪器）

如果取消选择，降噪操作不会启动；你可以在图像写出后手动运行降噪器工具。请注意，渲染设置中的显示驱动必须设置为支持降噪。

Tractor

远程队列设置仅适用于 Tractor；对于 LocalQueue，所有设置都会在本地应用。

Start Paused（暂停启动）

将作业发送到 Tractor，但不要立即开始作业。需要手动或通过脚本 Start。

Priority（优先级）

此项会影响活动作业如何分配到远程服务器。它不会影响作业在调度队列中的位置。

Service（服务）

该作业的附加服务选择器

Env Keys（环境键）

Switch（切换）预设配置。名为 DefaultEnvKey 的 JSON 偏好设置指定此处显示的默认值。可以指定多个键，并且应使用空格分隔。

LocalQueue 和 Tractor 可识别这些环境键：

  • rfm-* （例如 rfm-22.0）
  • maya-*（例如 maya-2017）
  • rmantree=*（例如 rmantree=$RMANTREE）
  • rfmtree=*（例如 rfmtree=$RFMTREE）

After（之后）

在此日期和时间之后开始处理作业。例如 11 月 24 日下午 1:45 可写为：11/24 13:45

Crews（团队）

指定在确定远程服务器访问权限时要使用的团队列表。

Tier（层级）

请参阅 Scheduling Modes（调度模式）（Tractor 中）

Projects（项目）

请参阅 Limits Configuration（限制配置）（Tractor 中）

Comments（注释）

用户提供的附加作业注释

Metadata（元数据）

添加到作业中的数据，供你的流水线处理。

When Done（完成时）

作业完成时应执行的命令

Example（示例）: /usr/sbin/Mail -s 'Job done: %j' %u < %f（%j = job title，%t = task title，%u = user，%f = temporary status file）

When Error（出错时）

作业出错时应执行的命令

Example（示例）: /usr/sbin/Mail -s 'Job error: %j' %u < %f（%j = job title，%t = task title，%u = user，%f = temporary status file）

When Always（总是执行）

无论作业错误状态如何都应执行的命令

Example（示例）: /usr/sbin/Mail -s 'Job error: %j' %u < %f（%j = job title，%t = task title，%u = user，%f = temporary status file）


XPU

选择 XPU 要使用的设备

Texture Manager

Number of Processes（进程数量）

txmake 用来处理纹理文件的并行进程数量。默认值为 2。除非你的 CPU 配置有超过 8 个物理核心可用，否则建议保留默认值。

Fallback Texture Path（后备纹理路径）

默认情况下，我们会将 .tex 文件写入 Images project directory（Images 项目目录）。如果此操作失败（目录不存在或存在权限问题），则使用此处指定的文件夹。不要在路径文本中使用空格或变音符号。

Show Advanced Options（显示高级选项）

显示更多 .tex 转换选项

Texture Extensions（纹理扩展名）

可识别的纹理文件类型扩展名
