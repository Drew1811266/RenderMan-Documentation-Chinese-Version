Installing Custom Nodes

Source: https://rmanwiki-27.pixar.com/space/RFM27/544541146/Installing+Custom+Nodes
Downloaded: 2026-06-15T03:49:12+00:00

在生产环境中，通常希望创建并添加自己的 patterns 和 C++ plugins（materials、lights、patterns 等）到 Maya 的用户界面中，并在 RenderMan 加载时一同加载。这样 artists 就能在 Maya session 中访问这些 nodes。想要这样做的用户必须从 Autodesk 获取 Maya Node ID（这些 ID 通常按区块分配）。这让 Maya 能够分配唯一 ID 并识别 nodes。

由于我们不建议修改随附的 rfm.json 文件，可以使用系统环境变量， RFM_SHOW_PATH 或 RFM_SITE_PATH， 来引用你的 custom（自定义）版本。


RenderMan for Maya 会按以下顺序搜索，并按相同顺序进行 merge/override（合并/覆盖）。 注意 ：这个层级流程也适用于其他 config files：shelf.json、aovs.json、menu.json 等。

  1. rfm 会读取 RFMTREE/config/rfm.json。这是基础 configuration，绝不应由你修改；请制作一个 custom（自定义）版本。
  2. rfm 会查找 RFM_SITE_PATH 和 RFM_SHOW_PATH。它们既可以是 environment variables，也可以是主 rfm.json 中的值（优先使用 Environment Variable 方式）。
  3. rfm 会查找 RFM_SITE_PATH/config/rfm.json。如果找到，其内容会与 base config 合并。
  4. rfm 会查找 RFM_SHOW_PATH/config/rfm.json。如果找到，其内容会 merge/override base config 和 site config。
  5. rfm 会查找 ~/rfm/config/rfm.json。如果找到，其内容会 merge/override base config、site config 和 show config。

override.json 可用于覆盖标准和 custom（自定义）RfM nodes 的默认值。

要添加额外 nodes（patterns、bxdfs、integrators 等），应扩展 resources/user/nodes/search_paths。rfm 始终期望相同的目录结构：

BASE
  |_ config
     |_ rfm.json
     |_ shelf.json, etc    (optional)
  |_ nodes    (optional)
     |_ args    (optional)
     |_ osl    (optional)
  |_ icons    (optional)
  |_ logs    (optional)


OSL Shaders（OSL 着色器）

Patterns 的 Node IDs 嵌入在 metadata 中。下面是来自 PxrLayer OSL shader 的示例。

shader PxrLayer
   [[int rfm_nodeid=1053299, string 
rfm_classification="rendernode/RenderMan/pattern",        
    int rfc_nodeid=1037674, string rfc_description="Xpxrlayer",         
    string help = "An OSL pattern for layerable parameters for PxrSurface or "
            "input layer to PxrLayerMixer."
   ]]


C++ Plug-ins（C++ 插件）

在你的 custom（自定义）rfm.json 文件中，可以像下面的示例这样加载你的文件：

{
    "$schema": "./rfmSchema.json",
    "resources": {
        "user": {
            "nodes": {
                "node_exclusion_list": [
                    "PxrHair",
                    "PxrLMDiffuse",
                    "PxrLMGlass",
                    "PxrLMMetal",
                    "PxrLMPlastic",
                    "PxrLMSubsurface",
                    "PxrSkin",
                    "PxrValidateBxdf"
                ]
            }
        }
    },
    "dirmaps": {
        "user_to_z": {
            "from": "/User/name/maya",
            "to": "Z:/maya",
            "zone": "UNC"
        }
    }
}


这些项的 Node IDs 包含在 .args file 中。顺序无关紧要，但必须位于 tags 的第一层。下面的示例用于一个 light。

<rfmdata nodeid="1053261"
             classification="rendernode/RenderMan/shader/light"/>
