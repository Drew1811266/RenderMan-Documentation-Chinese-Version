OpenVDB Volumes in Maya

Source: https://rmanwiki-27.pixar.com/space/RFM27/544540066/OpenVDB+Volumes+in+Maya
Downloaded: 2026-06-15T03:49:12+00:00

OpenVDB 是用于 volumes（体积）的开源 hierarchical data structure（层级数据结构）。Renderman for Maya（RenderMan for Maya）支持 OpenVDB Maya plugin（OpenVDB Maya 插件）$RMSTREE/plug-ins/OpenVDB.so。


Maya shelf（Maya 工具架）中的 OpenVDB 图标

只需点击一次该图标，RfM 就会自动创建用于读取、可视化和渲染 OpenVDB volume data（OpenVDB 体积数据）的默认设置。

有关 OpenVDB 的更多信息，请参阅 OpenVDB FAQ（OpenVDB 常见问题）。


  • OpenVDB node（OpenVDB 节点）默认会把 indirect visibility（间接可见性）设为 "off"，这是因为 Maya 的旧默认设置会在 attribute editor（属性编辑器）的 Render Stats rollout 中，把导入对象设为对 reflections/refractions（反射/折射）不可见。请打开这些选项以获得正确渲染。


Read OpenVDB（读取 OpenVDB）的方法

在 Renderman for Maya（RenderMan for Maya）中，若要读取 OpenVDB 文件，只需点击 OpenVDB 图标。这会创建一个 OpenVDB Read（OpenVDB 读取），让用户可以通过用于选择 OpenVDB 文件的 file picker（文件选择器）来设置 VDB File Path（VDB 文件路径）。




Visualize OpenVDB（可视化 OpenVDB）的方法

加载 OpenVDB 文件后，会出现 OpenVDB Visualize（OpenVDB 可视化） 选项卡会出现在 Attribute Editor（属性编辑器）中。

OpenVDB Visualizer（OpenVDB 可视化器）会在 Maya 中显示 OpenVDB volume data（OpenVDB 体积数据），并提供选项来更改 OpenVDB data（OpenVDB 数据）如何输出到 RIB。

默认情况下，Renderman for Maya（RenderMan for Maya）会输出 RiVolume。如果勾选 OpenVDBVisualize node（OpenVDBVisualize 节点）中的 Surface checkbox（Surface 复选框），或者 OpenVDB 文件只包含 level set data（水平集数据），则会改为输出 RiBlobby。在这种情况下，用户会希望接入一个不同于默认 PxrVolume的 shader（着色器）。可以添加 Renderman for Maya（RenderMan for Maya）的 Custom Shading Group attribute（Custom Shading Group 属性）来连接 shading group（着色组）。


RIB Options（RIB 选项）

以下选项可用于更改 OpenVDB volume（OpenVDB 体积）的 data（数据）输出到 RIB 的方式。

VDB Grid Name（VDB 网格名称）

VDB grid（VDB 网格）是 voxel data（体素数据）的 sparse tree representation（稀疏树表示）。例如，float grid（浮点网格）会为每个 voxel（体素）存储一个 single-precision floating point value（单精度浮点值）。加载 OpenVDB 文件后，pulldown menu（下拉菜单）中会出现 grid names（网格名称）列表。选择你想采样的 grid name（网格名称）。

Surface（曲面）

这会创建 RiBlobby，而不是创建 RiVolume。请注意，开启此选项可能会减慢 render（渲染）。

Isovalue（等值）

此项设置 volume（体积）的 threshold（阈值）。

Visualizer Options（可视化器选项）

以下 attributes（属性）仅用于可视化。更改它们不会改变 volume（体积）的渲染。

Active Value Bounding Box（活动值包围盒）

Show/hide（显示/隐藏）active value（活动值）的 bounding box（包围盒）。

Internal Nodes（内部节点）

Show/Hide（显示/隐藏）。具有 dynamic branching factor（动态分支因子）的 root node（根节点）包含 fixed branching（固定分支）的 internal nodes（内部节点）。

Leaf Nodes（叶节点）

Show/Hide（显示/隐藏） 存储在 leaf nodes（叶节点）中的值对应于单个 voxels（体素）；所有其他值都对应于 "tiles"（区块）

Active Tiles（活动区块）

Show/Hide（显示/隐藏） 存储在类型为  RootNode  或  InternalNode  的节点中的值，对应于 index space（索引空间）中具有 constant value（常量值）和 active state（活动状态）、且尺寸为 2 的幂的区域。 grid（网格）中的每个值都有一个 binary state（二进制状态），称为其 "active state"（活动状态）。对此 on/off state（开/关状态）的解释取决于具体应用，但通常 active (on) value（活动/开启值）在某种意义上是“有意义的”，而 inactive (off) value（非活动/关闭值）则不那么有意义。

Active Voxels（活动体素）

Show/Hide（显示/隐藏） 同上，但适用于 voxels（体素）。 存储在类型为  LeafNode  的节点中的值（如果存在，这类节点具有固定深度）对应于单个 voxels（体素）。这些是 index space（索引空间）中最小的可寻址单位。

OpenVDB 着色网络（Shading Network）

OpenVDB Visualizer（OpenVDB 可视化器）图标会自动设置 volume shading group（体积着色组），并连接到相应的 openVDB visualizer（openVDB 可视化器）。
