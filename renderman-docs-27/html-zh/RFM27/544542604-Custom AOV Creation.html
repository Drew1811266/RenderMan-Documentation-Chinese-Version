Custom AOV Creation

Source: https://rmanwiki-27.pixar.com/space/RFM27/544542604/Custom+AOV+Creation
Downloaded: 2026-06-15T03:49:12+00:00

父级 页面说明了 AOVs render settings tab，包括 built-in options 以及如何选择所需内容。

不过，在许多 productions 中，你可能希望使用此 menu 编写自己的 LPE。下面说明这个流程。

请务必添加你自己的 custom AOVs 并正确命名， 先完成这一步，如果你打算之后从 presets 添加另一个 default 或其他 Display（Display title 右侧的蓝色 menu icon 包含三个 presets）；否则，如果你在 之前 或 过程中 创建 custom AOVs 时添加它们，命名可能会变得不正确，并且 settings 会被强制共享。

tl;dr：为避免问题，请在所有其他 displays 和 channels 都添加完成后，再从 Display menu 添加 default displays（如果需要）。

注意左侧是 Display Channels，这里简称为 "Channels"。右侧是 Display Drivers，这里称为 "Displays"。

每个 section 下方都有带 plus sign '+' 高亮的 icons，用于创建新的 Display Channel 或 Driver。

我们从 Channels 开始：

  1. 通过选择 plus sign 创建一个新 channel

2. 你可以通过 double-clicking 将此 channel 添加到右侧的 Displays，这会自动创建新的 Display Driver。（也可以使用右侧 plus sign 创建不带 Channel 的新 Display Driver。）选中新 driver 时，你可以使用 arrow-to-list button（在左侧 plus sign 旁边）将 Channel 移到你创建且高亮/选中的 Display Driver。

3. 双击名称可以重命名 Display Driver 和 channels。

谨慎命名可让 Denoiser 识别 channels 并对它们 denoise。你可以在 这里 的 Layered Filtering（For AOVs）下了解更多信息和可接受的 names。Denoise utility 会查找正确的 Channel name，以确保它使用正确 options 并相应地 filter。

最后的示例图显示了重命名，以及用于 render "cup" object 的 Direct Diffuse 的 LPE。（Cup object 已正确标记为名为 "cup" 的 LPE group。）

生成的 display drivers 会写入你未在此处指定的额外命名信息。 Workspace Tab 控制 render outputs 中包含的其他信息，包括 scene name、version、AOV name、camera name 等。

请记住，如果未选择 Display Driver 而是逐个添加，它们会创建多个 outputs，如左侧所示。另一种行为是先选择 Display Driver，再添加 Channels，以获得如右侧示例那样的单个（或有限数量）output。
