自定义 AOV 创建

Source: https://rmanwiki-27.pixar.com/space/RFM27/544542604/Custom+AOV+Creation
Downloaded: 2026-06-15T03:49:12+00:00

父级 页面说明了 父页面介绍了 AOVs 渲染设置选项卡，包括内置选项以及如何选择所需内容。

不过，在许多制作中，你可能希望使用此菜单编写自己的 LPE。下面说明这个流程。

请务必添加你自己的自定义 AOV 并正确命名， 先完成这一步，如果你打算之后从预设添加另一个 default 或其他 Display（Display 标题右侧的蓝色菜单图标包含三个预设）；否则，如果你在 之前 或 过程中 创建自定义 AOV 时添加它们，命名可能会变得不正确，并且设置会被强制共享。

tl;dr：为避免问题，请在所有其他 Display 和 Channel 都添加完成后，再从 Display 菜单添加默认 Display（如果需要）。

注意左侧是 Display Channels，这里简称为 "Channels"。右侧是 Display Drivers，这里称为 "Displays"。

每个部分下方都有带加号 '+' 高亮的图标，用于创建新的 Display Channel 或 Driver。

我们从 Channels 开始：

  1. 通过选择加号创建一个新 Channel

2. 你可以通过双击将此 Channel 添加到右侧的 Displays，这会自动创建新的 Display Driver。（也可以使用右侧加号创建不带 Channel 的新 Display Driver。）选中新 Driver 时，你可以使用“箭头到列表”按钮（在左侧加号旁边）将 Channel 移到你创建且高亮/选中的 Display Driver。

3. 双击名称可以重命名 Display Driver 和 Channel。

谨慎命名可让 Denoiser 识别这些通道并对它们降噪。你可以在 这里 的 Layered Filtering（For AOVs）下了解更多信息和可接受的名称。Denoise utility 会查找正确的 Channel 名称，以确保它使用正确选项并相应地过滤。

最后的示例图显示了重命名，以及用于渲染 "cup" 对象的 Direct Diffuse 的 LPE。（Cup 对象已正确标记为名为 "cup" 的 LPE 组。）

生成的 Display Driver 会写入你未在此处指定的额外命名信息。 Workspace Tab 控制渲染输出中包含的其他信息，包括场景名、版本、AOV 名称、相机名等。

请记住，如果未选择 Display Driver 而是逐个添加 Channels，它们会创建多个输出，如左侧所示。另一种做法是先选择 Display Driver，再添加 Channels，以获得如右侧示例那样的单个（或有限数量）输出。
