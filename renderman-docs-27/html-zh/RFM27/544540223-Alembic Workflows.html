Alembic Workflows

Downloaded: 2026-06-15T03:49:12+00:00

RenderMan for Maya 改进了 Alembic 渲染工作流，将现有功能整合到一起。

现在，在 RFM 中渲染 Alembic archives（Alembic 归档）有几种不同方式。我们会逐一介绍。

本文档使用的是 Alembic archive（Alembic 归档）： Rolling Teapot 模型的 Alembic archive，该模型来自 Brice Laville Saint Martin。

你可以在此处下载它： here（这里）

从 Maya 将几何体导出为 Alembic

首先，你需要导出几何体，以便 RenderMan 可以正确渲染它。

为简化流程，我们在 RFM 中添加了一些导出函数，用于保证正确结果；如果你需要自定义，也可以基于我们的工作 roll out（推出/实现）自己的流程。

如你所见，根据 archive 的渲染方式不同，可用选项也不同：

Export for the gpuCache node（为 gpuCache 节点导出）

为 gpuCache node（gpuCache 节点）导出时，Alembic archive 会直接发送给渲染器，而不会经过 RfM 处理。如果场景中包含已经平滑（使用 Smooth Mesh 控制）或带有 RenderMan subdivision surface attributes（细分曲面属性）的 polygon meshes（多边形网格），我们需要确保它们会作为 subdivision surfaces（细分曲面）而不是 poly meshes（多边形网格）导出。此菜单会在调用常规 Alembic 导出对话框之前包含一个预处理步骤。

我们使用 gpuCache 节点来引用、显示并渲染常规的高质量 Alembic archives（Alembic 归档）。

这不同于 Maya 的 GPU cache 工作流；后者会保存场景的简化版本，只是为了让大型场景保持更强的交互性。

为获得最佳结果，请始终使用我们的 Alembic Export 菜单。如果你尝试渲染通过 Cache > GPU Cache > Export... 菜单保存的 alembic 文件，我们无法保证结果。

Export for imported/referenced archives（为导入/引用的归档导出）

当 alembic node（Alembic 节点）被导入或引用到 Maya 场景中时，archive 的节点会重建为 Maya shapes（Maya 形状），并且必须携带其原始 RenderMan attributes（RenderMan 属性），才能按预期渲染。我们的导出菜单包含一个设置步骤，用于确保所有相关属性都会保存到 alembic cache 中，同时保存 uv sets 以及 subdivision surface creases and corners（细分曲面的折痕和角点）。

Rendering Imported or Referenced Caches（渲染导入或引用的缓存）

Alembic archives 可以在 Maya 中导入或引用。你可以为它们附加材质，它们会像任何其他 Maya geometry（Maya 几何体）一样渲染。

它们非常适合为 animated characters（动画角色）进行 lighting（布光）和 shading（着色），同时无需承担复杂 rig（绑定）求值的负担。但当 archive 包含大量节点和沉重几何体时，它们并不那么适合；这种情况下可以考虑改为渲染 GPU cache。

The easy way（简单方法）

右键点击 RenderMan shelf（RenderMan 工具架）中的 archive 图标，然后选择 "Import GPU Cache..."。文件选择器会打开，节点会被创建并设置为对 indirect rays（间接射线）可见。 

The hard way（手动方法）

Start（开始）时，先通过 GPU Cache 菜单导入你的 alembic 文件。如果你的 Maya 中未出现该菜单，请转到 Plug-in Manager（插件管理器） 窗口启用它。

对于 animated GPU caches（动画 GPU 缓存），请选择： Use Global Time（使用全局时间）

在此示例中，我导入了 Rolling Teapot 模型。请注意，整个模型现在是 Maya 中的单个节点，默认会由 RenderMan 渲染。

默认属性会出现在 RenderMan 部分中：

  • Render Alembic Cache（渲染 Alembic 缓存）

    • 必须启用才能渲染。

  • RLF File Name（RLF 文件名）

    • 将应用到 Alembic 文件上的 RenderMan Look File 的完整路径。你可以使用页面下方介绍的编辑器来创建并保存 RLF 文件。

  • Edit RLF File（编辑 RLF 文件）

    • 打开独立的 RLF editor（RLF 编辑器），用于修改 RLF File Name（RLF 文件名）中引用的文件。请参见 Assign materials（分配材质） 了解详情。

  • Edit objects（编辑对象）

    • 打开 shape editor（形状编辑器），用于修改 alembic 文件的内容。请参见 Geometric edits（几何编辑） 了解详情。

  • Inspect Alembic section（检查 Alembic 区段）

  • Read archive contents（读取归档内容）

    • 点击此按钮解析 archive 的内容，并在 node lists（节点列表）中显示其内容。

大型 alembic archives 可能需要许多秒才能解析。请耐心等待...

node list（节点列表）可能会错误地显示大量深度嵌套的节点。为绕过这个 Maya bug，默认情况下我们只展开前 3 层，并建议使用 filter field（过滤字段）浏览 archive。

解析后的数据会保存在节点上，因此可以快速显示而无需重新解析文件；但如果 archive 中有任何内容发生变化，你必须重新解析该 archive。

你可以通过在列表中双击，对层级结构中的任意位置进行“solo”（只显示所选项）。若要再次显示全部内容，请双击顶层节点 (ABC)。这不会影响最终渲染。

  • Filter（过滤）

    • node list（节点列表）的内容可以使用 glob expression（glob 表达式）进行过滤。

Batch Rendering（批渲染）

为提高效率，RenderMan for Maya 不会导出所有 shaders（着色器），只会导出附加到 geometry（几何体）上的 materials（材质）。这会在 batch rendering GPU Caches（批渲染 GPU 缓存）时导致问题。

要绕过此问题，请 Enable "Output All Shaders"（启用“输出所有着色器”） 在 RenderMan Render Settings 的 advanced menu（高级菜单）中启用。

Debugging（调试）

从 RFM 24.4 起，可以定义 RFM_RLF_DEBUG environment variable（环境变量，其值无关紧要；在启动 RFM 前设置），以便在 render-time（渲染时）获得 debugging output（调试输出）。
