Alembic Workflows

RenderMan for Maya 改进了 Alembic 支持 渲染工作流，将现有功能整合到一起。

现在，在 RfM 中渲染 Alembic 归档有几种不同方式。我们会逐一介绍。

本文档使用的是 Alembic 归档： Rolling Teapot 模型的 Alembic 归档，该模型来自 Brice Laville Saint Martin。

为 gpuCache 节点导出

为 gpuCache 节点导出时，Alembic 归档会直接发送给渲染器，而不会经过 RfM 处理。如果场景中包含已经平滑（使用 Smooth Mesh 控制）或带有 RenderMan 细分曲面属性的多边形网格，我们需要确保它们会作为细分曲面而不是多边形网格导出。此菜单会在调用常规 Alembic 导出对话框之前包含一个预处理步骤。

我们使用 gpuCache 节点来引用、显示并渲染常规的高质量 Alembic 归档。

这不同于 Maya 的 GPU cache 工作流；后者会保存场景的简化版本，只是为了让大型场景保持更强的交互性。

为获得最佳结果，请始终使用我们的 Alembic Export 菜单。如果你尝试渲染通过 Cache > GPU Cache > Export... 菜单保存的 Alembic 文件，我们无法保证结果。

为导入/引用的归档导出

当 Alembic 节点被导入或引用到 Maya 场景中时，归档的节点会重建为 Maya 形状，并且必须携带其原始 RenderMan 属性，才能按预期渲染。我们的导出菜单包含一个设置步骤，用于确保所有相关属性都会保存到 Alembic 缓存中，同时保存 UV 集以及细分曲面的折痕和角点。

渲染导入或引用的缓存

Alembic 归档可以在 Maya 中导入或引用。你可以为它们附加材质，它们会像任何其他 Maya 几何体一样渲染。

它们非常适合为动画角色进行布光和着色，同时无需承担复杂绑定求值的负担。但当归档包含大量节点和沉重几何体时，它们并不那么适合；这种情况下可以考虑改为渲染 GPU 缓存。

渲染 gpuCache 节点

创建新的 gpuCache 节点

简单方法

右键点击 RenderMan 工具架中的归档图标，然后选择 "Import GPU Cache..."。文件选择器会打开，节点会被创建并设置为对间接射线可见。 

手动方法

开始时，先通过 GPU Cache 菜单导入你的 Alembic 文件。如果你的 Maya 中未出现该菜单，请转到 插件管理器 窗口启用它。

对于动画 GPU 缓存，请选择： 使用全局时间

在此示例中，我导入了 Rolling Teapot 模型。请注意，整个模型现在是 Maya 中的单个节点，默认会由 RenderMan 渲染。

默认属性会出现在 RenderMan 部分中：

  • Render Alembic Cache（渲染 Alembic 缓存）

    • 必须启用才能渲染。

  • RLF File Name（RLF 文件名）

    • 将应用到 Alembic 文件上的 RenderMan Look File 的完整路径。你可以使用页面下方介绍的编辑器来创建并保存 RLF 文件。

  • Edit RLF File（编辑 RLF 文件）

    • 打开独立的 RLF 编辑器，用于修改 RLF File Name（RLF 文件名）中引用的文件。请参见 分配材质 了解详情。

  • 编辑对象

    • 打开形状编辑器，用于修改 Alembic 文件的内容。请参见 Geometric edits（几何编辑） 了解详情。

  • 检查 Alembic 区段

  • 读取归档内容

    • 点击此按钮解析归档内容，并在节点列表中显示。

大型 Alembic 归档可能需要许多秒才能解析。请耐心等待...

节点列表可能会错误地显示大量深度嵌套的节点。为绕过这个 Maya bug，默认情况下我们只展开前 3 层，并建议使用过滤字段浏览归档。

解析后的数据会保存在节点上，因此可以快速显示而无需重新解析文件；但如果归档中有任何内容发生变化，你必须重新解析该归档。

你可以通过在列表中双击，对层级结构中的任意位置进行“solo”（只显示所选项）。若要再次显示全部内容，请双击顶层节点 (ABC)。这不会影响最终渲染。

批渲染

为提高效率，RenderMan for Maya 不会导出所有着色器，只会导出附加到几何体上的材质。这会在批渲染 GPU Cache 时导致问题。

要绕过此问题，请 启用 "Output All Shaders" 在 RenderMan Render Settings 的高级菜单中启用。

调试

从 RFM 24.4 起，可以定义 RFM_RLF_DEBUG 环境变量（其值无关紧要；在启动 RfM 前设置），以便在渲染时获得调试输出。
