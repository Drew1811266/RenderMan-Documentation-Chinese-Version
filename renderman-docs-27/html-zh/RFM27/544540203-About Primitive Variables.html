About PrimVars for Maya（Maya 的 PrimVars 说明）

将数据附加到对象表面通常很有用。在 RenderMan 中，这是通过在声明 surface primitive（曲面图元）时把数据作为 token value pair（令牌-值对）包含进去实现的。数据一旦附加，就可以作为参数供分配给该 surface primitive 的 shaders（着色器）使用。以这种方式附加到 surface geometry（曲面几何体）上的参数称为 primitive variables（基本变量），for short（简称）primvars。RenderMan for Maya 同时提供简单而灵活的 primvars 访问方式。


在最简单的层面上，RfM 包含可通过 Maya Hypershade 访问的 Prim Var nodes（Prim Var 节点），可按需要附加到 Maya 节点。随后各种工具可以在 shading time（着色时）利用这些 primvars。这实际上会驱动参数，或向参数提供实现特殊效果所需的数据。一个例子是根据 mesh（网格）上的 vertex locations（顶点位置）提供颜色。

提供的节点支持 Color（颜色）、Float（1、2 和 3）、Normal（法线）、Point（点）以及 Vector（向量）类型的 primvars。

Under the Covers（底层机制）

RenderMan for Maya 会把 Maya attribute type（Maya 属性类型）与命名约定组合起来，从而推断 primitive variable（基本变量）请求：

rman?[?[?]]primvarname

必须提供一个特殊字符，用来提示 Ri primvar 类型。还可以提供额外的特殊字符，用于请求特定的 detail（细节级别）和 array representation（数组表示）。用户需要自行提供尺寸正确的 Maya attributes（Maya 属性）。

特殊名称有三种形式：

  1. rmanTprimvarName：其中 T 是以下之一：

    'F', 'C', 'V', 'N', 'P', 'H', 'M', 'S'

    它们会被解释为以下 RenderMan 类型：

    请注意，以下 RenderMan 类型名称需按原样使用和复制：float、color、vector、normal、point、hpoint、matrix、string
     

  2. rmanDTprimvarname：其中 T 如上，D 提供 detail（细节级别）提示：

    'c', 'u', 'f', 'v', 'x'

    表示标准 RenderMan details（细节级别）：

    请注意，以下标准 RenderMan detail 名称需按原样使用和复制：constant、uniform、facevarying、varying、vertex
     

  3. rmanaDTprimvarname：其中 T 和 D 如上，并且字母 a 表示请求给定类型和 detail（细节级别）的数组。
