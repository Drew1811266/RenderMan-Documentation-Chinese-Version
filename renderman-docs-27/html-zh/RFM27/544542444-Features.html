Features（功能）标签页可用于启用 RenderMan 渲染器的特殊功能，例如渲染器类型、OCIO、运动模糊等。



Renderer

定义场景使用的 RenderMan 渲染器。

  • RIS：使用 RIS 渲染场景。这是仅 CPU 渲染器。

  • XPU:  使用 XPU 渲染场景。这是 CPU + GPU 混合渲染器。

    • XPU Mode - CPU：使用 XPU 渲染，但只使用 CPU。

    • XPU Mode - GPU：使用 XPU 渲染，但只使用 GPU。


Motion Blur

RenderMan 提供了许多控件，用于创建快速而高效的运动模糊。 运动模糊。

运动模糊：  启用运动模糊会让移动对象沿其运动路径产生模糊，以防止时间走样和频闪。使用运动模糊时，应增加像素采样数量（Sampling 选项卡中的 max samples 参数），并预期渲染时间会更长。 

  • 3D：完整渲染且准确的运动模糊

  • 2D：会计算运动矢量，但运动模糊此时 不会 不会被渲染；在输出 dPdtime 和 dPdcameratime 等 AOV 后用于后期处理时很有用。

Camera Blur（摄影机模糊）：  当此功能被禁用时，渲染器在计算运动模糊时会忽略相机移动。如果你移动相机并希望场景中所有内容真实地产生模糊，则需要开启此功能；但应预期这可能是开销较高的操作，因为场景中的所有内容都会模糊，并产生必须收敛的噪声。

Shutter Angle（快门角度）：  Shutter Angle（快门角度） Shutter Angle（快门角度）只有在启用运动模糊时才有意义。它控制快门在一帧中打开多久。以角度为单位，设置为 360 表示快门在整个帧内保持打开。角度越大，模糊越强，可能会增加渲染时间，因为需要更多采样才能平滑模糊。

Shutter Opening（快门开启）：  默认情况下，快门实际上会瞬时打开和关闭。 Shutter Opening（快门开启） 设置由两个值组成： a 和 b。它们是快门间隔的分数，可用于调整快门打开和关闭所需时间。从 0 到 a 时，快门打开；从 b 到 1 时则关闭。  

Shutter Timing（快门时序）：  Shutter Timing（快门时序） 允许你控制快门打开时间应如何与帧对齐。它可以在帧开始时打开，或在帧之前打开以便以帧为中心，也可以在帧开始时关闭。


OpenColorIO

Configuration

  • $OCIO：使用系统环境变量定义的色彩空间

  • Off：  场景线性被定义为场景级色彩空间

  • ACES 1.2：使用 RenderMan 24 随附的 ACES 1.2 色彩空间

  • Custom：允许指定自定义色彩空间文件

Interactive Denoiser（交互式降噪器）

此交互式降噪器在 Maya 视口中启用，并提供 RenderMan Machine Learning Denoiser（机器学习降噪器） 的交互式版本。

Cheap First Pass（廉价首轮）：首次降噪遍次会使用更快（成本更低）但准确度较低的启发式方法。

Min Samples（最小采样）：降噪器首次运行前所需的最低平均采样数。

Interval：降噪运行之间的时间间隔，单位为秒。

Stylized Looks（风格化外观）

此复选框会启用以下 Stylized Looks（风格化外观） 工具集，包括向 beauty（美观图）通道添加所需的 LPE 和 AOV。


Filters

这些选项允许使用 Display or Sample Filters（显示滤镜或采样滤镜），例如 Stylized Looks、Cryptomatte 和其他效果。
