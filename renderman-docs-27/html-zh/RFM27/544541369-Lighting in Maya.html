灯光滤镜

在 Maya 中，可以通过 RenderMan Shelf、RenderMan Menu 或 Hypershade Window 创建 RenderMan 灯光。 

  • 不支持原生 Maya 灯光。请使用 "Pxr" 灯光。

  • 在 Maya 节点编辑器中，你可能会看到灯光上有一个可用的 "Light Data" 输出连接。尽管 Maya 将其显示为有效输出，但它并不是可连接的参数。

RenderMan 中的照明很灵活，对用户来说也非常直接。

RenderMan 内置了一组基础几何灯光。这些专用形状 - DomeLight、DistantLight、RectLight， 以及 SphereLight （尤其是后两者）- 与任意几何体（即附加灯光 Bxdf 的标准几何图元）相比，可以提供更优的采样行为。此外还有一个 PortalLight 用于改善通过窗户、天窗等开口间接照明的室内场景照明，以及一个 日光 模拟。

灯光滤镜 也可以修改和定向照明，提供更多控制与艺术选项。你可以找到真实世界中的可能性，例如 Gobo（遮光片） 以及 Cookie（投影片），也可以找到高级方式，用于从单个灯光链接并控制特定对象的 强度。 等结果。


  • 请注意，由于这些优化，球形灯光要求所有缩放变换都保持一致；例如 X、Y 和 Z 缩放必须完全相同，以免球形形状被扭曲成椭圆或其他形状。

  • 默认情况下，灯光在折射/透射中不可见。可以在灯光可见性设置中启用这一点。



请注意，照亮场景主要有三种方式：

  • 解析灯光 - 这是首选。这类灯光提供更好的内存占用、灵活性和速度。对于摄像机可见性，它们的可见性只是简单的开/关。

  • 网格灯光 - 这类灯光非常适合在照明中使用任意形状。由于它们是几何体，因此会消耗更多内存，复杂形状可能会增加噪声。它们可以像其他对象一样使用可见性：摄像机、阴影/透射和间接可见性。

  • 自发光表面 - 这类表面通常使用 constant Bxdf 或 "glow" 参数来间接照亮场景。这种方式效率较低，通常应避免使用；但在未来感对象和面板上用纹理映射制作“灯光”时可能会有帮助！


解析灯光类型

  • PxrDomeLight - 穹顶灯充当场景的环境灯，并且通常会映射高动态范围图像（HDRI）。

  • PxrRectLight - 此灯光常用于矩形区域灯。

  • PxrDistantLight - 此灯光非常适合作为太阳光或其他远距离光源使用，此时光线近乎平行并覆盖整个场景。

  • PxrDiskLight - 很适合作为区域灯使用，也可以与滤镜结合来制作聚光灯。

  • PxrSphereLight - 对怀念点光源的用户来说，此灯光可用于向所有方向照明。

  • PxrEnvDaylight - 一种日光模拟着色器，你可以提供地球上任意时间和位置，或者直接把阳光放在想要的位置。

  • PxrPortalLight - 对于间接照明室内等困难照明场景，此灯光可与 PxrDomeLight 一起使用，提高渲染质量和速度。

几何灯光类型

  • PxrMeshLight - 网格灯光使用一块几何体并将其转化为光源。这比使用 Bxdf 的 glow 或 emission 高效得多。它非常适合霓虹灯之类的对象！

实用灯光类型

  • PxrAOVLight - 这是一个工具，会被识别为灯光类型，但它 不会 照亮任何东西。它是一个有用工具，可通过灯光链接为对象快速创建遮罩。
