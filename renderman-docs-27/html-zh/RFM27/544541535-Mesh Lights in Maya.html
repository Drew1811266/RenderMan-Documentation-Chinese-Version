创建网格灯

在 RenderMan for Maya 中，任何一块几何体都可以成为光源。选择要发光的几何体，然后单击 RenderMan shelf 中的 Mesh Light 按钮。

下次渲染时，该对象会投射光照，并且自身具有恒定颜色；该颜色来自 PxrMeshLight 的 Color 和 Intensity。

PxrBlack

单击 Mesh Light 按钮时，会将名为 PxrBlack 的新材质指派给所选对象，并创建一个 PxrMeshLight，再将其连接到 PxrBlack 材质的着色组 中。PxrBlack 充当占位材质。它实际上没有任何参数，不过在 Attribute Editor 中会提供一个指向关联 PxrMeshLight 的链接。 

对于大多数用途，PxrBlack 是合适的材质；但你也可以尝试使用其他材质，例如 PxrVolume。最简单的设置方式是先使用 Mesh Light 按钮。然后选择 PxrBlack 的着色组，并在那里接入不同的 表面或体积材质。这样区域光就已经连接到着色组中。

PxrMeshLight

PxrMeshLight 的行为与 Maya 中其他灯光很相似。它们会显示在 Hypershade Window 的 Lights 选项卡下，并且可以在 Maya 的 Relationship Editor 中设置灯光链接。

PxrMeshLight 允许你创建任意形状，并将它们用作真正的光源。与发光几何体相比，这会提升性能，并为创建霓虹灯、未来感灯光面板，以及能照亮场景的变形形状提供很高的灵活性。

这些参数简单且易于调节，并且该灯光会遵循形状的渲染可见性，而不是灯光本身上的选择项。

虽然可以将 mesh light 用作一块变形几何体，但目前尚不支持变形运动模糊，可能会产生不正确的结果。

创建 mesh light 时，你可能会看到一个名为 PxrBlack 的材质与它相连。这个材质没有参数，其设计目的是防止网格再次从自身散射光照。这样做是为了提供直观行为，并不一定是物理准确的行为。


参数

基础


Color（颜色）

发出光照的颜色。


Intensity（强度）

线性缩放此灯光的贡献。灯光强度使用物理衰减计算。


Exposure（曝光）

以 2 的幂指定区域光的曝光。曝光增加 1 会使光源发出的能量加倍。值为 0 时光源处的强度为 1，-1 时为 0.5。现实世界 中的照明具有很高的能量，典型曝光通常是较低数值；而要获得等效的 Intensity，可能需要输入非常大的数字。熟悉摄影测量的艺术家也会觉得这种方式更顺手。


Texture Color（纹理颜色）

使用 pattern 节点将一张图像用作光源，例如 PxrTexture。最好使用高动态范围源。请注意，这会根据所用 HDRI 的内容影响颜色和阴影。如果必须使用 JPEG 图像等 8 位源，应先将其线性化（转换到线性色彩空间），再作为颜色使用。由于这确实是一个网格，你必须已经定义 UV 或适当的投影，才能正确放置纹理。


Enable Temperature（启用色温）

打开或关闭色温。


Temperature（色温）

允许用户以开尔文为单位选择灯光色温。与基础灯光颜色不同，这让用户能够基于标准色温测量轻松选择可信的灯光颜色。此控件会像放在灯光前方的滤镜或色胶（带色透明表面）一样工作。如果灯光是白色，你会得到所选色温。如果灯光已有颜色，色温会让它更冷（高于 6500K）或更暖（低于 6500K）。


在完全饱和的单色灯光上，此效果会不那么明显。默认值为 6500K，在大多数显示器上应非常接近白色（sRGB 和 Rec 709 使用的 D65 光源）。



细化


Specular Amount（镜面反射量）

发出的镜面反射光量。这是一个非物理控件。你可以使用  Specular Amount  为 0.0 的灯光，使其仅作为场景对象的漫反射光源，并避免添加高光。 


Diffuse Amount（漫反射量）

发出的漫反射光量。这是一个非物理控件。你可以使用  Diffuse Amount  为 0.0 的灯光，使其仅作为场景对象的高光光源。


Intensity Near Dist（强度近距离）

被照明点与灯光之间的近距离；在该距离内采样不会继续变亮。当灯光靠近表面时，这有助于避免热点和采样问题。这些值基于你的场景比例/距离，因此这里只是一个示例。


Specular Near Dist（镜面反射近距离）

带有镜面反射着色的被照明点与灯光之间的近距离；在该距离内采样不会继续变亮。当灯光靠近镜面反射表面时，这有助于避免热点和采样问题。 注意 此值不会应用于从相机看到的灯光。 下方示例中，右侧球体的距离依次为 0.0、5.0，最后是 10.0。这些值基于你的场景比例/距离，因此这里只是一个示例。


Diffuse Near Dist（漫反射近距离）

带有漫反射着色的被照明点与灯光之间的近距离；在该距离内采样不会继续变亮。当灯光靠近漫反射表面时，这有助于避免热点和采样问题。 注意 此值不会应用于从相机看到的灯光。 下方示例中，左侧球体的距离依次为 0.0、5.0，最后是 10.0。这些值基于你的场景比例/距离，因此这里只是一个示例。


高级


Area Normalize（面积归一化）

启用归一化时，随着你缩放光源，贡献到场景的光量不会改变。这样可以更容易地调整高光大小，而不改变场景照明。


Trace Light Paths（追踪灯光路径）

启用来自此灯光的光线追踪和光子追踪。此值会强制使用基于物理的灯光，并且作为副作用会禁用上面的 Shadows 控件。使用 PxrVCM 或 PxrUnified Integrators 时，用户可以用此功能有选择地决定哪些灯光发射光子。


Thin Shadow（薄阴影）

为此灯光启用薄阴影，并禁用折射焦散。如果启用了 Trace Light Paths，则会忽略此参数。这是一个非物理控件，会为透射对象创建“伪造”的彩色阴影，而无需为焦散生成光子。


Light Group（灯光组）

指定用于 light group LPE 的灯光组名称。这对于生成逐灯光 AOV，以便之后在合成中调整很有用。


Importance Multiplier（重要性倍增器）

用户不必设置显式的 Light Samples，而是可以通过更改此值来改变该灯光在内部被分配的采样数量。RenderMan 会在渲染时为场景中的所有灯光创建一组采样；更改此值会在各个灯光之间重新平衡采样。请注意，提高此值会使此灯光被选取 更多 采样，同时减少场景中其他灯光的采样。低于默认值则会使采样 减少，并为场景中其他灯光提供更多采样。
