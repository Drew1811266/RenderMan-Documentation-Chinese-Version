Geometric Settings

Source: https://rmanwiki-27.pixar.com/space/RFM27/544540160/Geometric+Settings
Downloaded: 2026-06-15T03:49:12+00:00

在 Maya 中指定 instance attributes（实例属性）时，它们可以像 Primvars（基本变量）这样命名，此时控制项位于 shape node（形状节点）上。也可以作为 transform node（变换节点）上的属性存在，并按每个实例变化。

示例命令：

rfm2.ui.user_attrs.user_attrs_add(data_attr, nodelist, pdict) -> add to one or more nodes

rfm2.ui.user_attrs.user_attrs_del(data_attr, nodelist, attrlist) -> delete from one or more nodes

rfm2.ui.user_attrs.user_attrs(node) -> list all user attributes

补充说明：

Emit Color Sets（发出颜色集） - 发出分配给对象的 color sets（颜色集）；在使用 procedural coloring（程序化着色）渲染 MASH 对象时尤其有用。

Prevent Polygon Cracking（防止多边形裂缝） - 通常预期你会对 subdivision surfaces（细分曲面）进行 displacement（置换）。不过，如果你需要对具有“hard” normals（硬法线）的 polygonal surface（多边形曲面）进行置换，此选项会尝试防止对象在置换后裂开。

Emit Face IDs（发出面 ID） - 这允许曲面使用 Ptex texturing（Ptex 纹理）。

Reference Object（参考对象） -  _WPref 和带 reference object（参考对象）的 place3d 应作为你正在动画化的对象的子级。实例若要让纹理以相同方式贴附，_WPref 是必需的。不过，在 IPR 中，如果你移动/平移对象但它没有发生形变，系统不会重新求值；如果只有 transform（变换） 发生变化，则 _WPref 不会更新。只有重启渲染，或以某种方式触发 mesh（网格）更新（例如微调 displacement bound），你才会看到正确的值。

为便于同 Instance Attributes（实例属性） 区分，这些也可能称为“prototype attributes”（原型属性），而 Instance Attributes 可以按实例变化。

int（整数）

Points（点）

Name（名称）

Type（类型）

Default（默认值）

Description（说明）

falloffpower

float（浮点）

0

用于 points（点）。如果未提供或设为 0，点会具有硬边缘。值为 1 是一个“合理”值，可模拟通常基于 cosine（余弦）的 falloff（衰减）；对大多数制作 volumetric particle effects（体积粒子效果）的人来说，这很可能是常用值。0 到 1 之间的值会让 falloff 更快，使点更快被侵蚀，也就是点的 “presence”（存在感）更弱。大于 1 的值（直到无穷大）会让 falloff 更慢，直至几乎不存在衰减。

volume:aggregate

string（字符串）

globalVolumeAggregate

体积所属的 volume aggregate（体积聚合）的名称。如果非空，只有 integrator（积分器）显式使用该命名 aggregate，或 PxrSurface 等 surface shader（曲面着色器）将该命名 aggregate 绑定到其 interior（内部）时，才会渲染该体积。

volume:aggregaterespectvisibility

int（整数）

empty（空）

如果设为 1，并且该体积属于某个 aggregate（聚合），则体积会遵守标准 visibility attributes（可见性属性）。如果设为 0，则会忽略 visibility attributes，只应用 integrator visibility settings（积分器可见性设置）。Value range（取值范围）：true/false。

volume:dsominmax

int（整数）

empty（空）

目前仅用于 aggregate volumes（聚合体积），并且只用于使用 ImplicitField DSO 的体积。如果设为 1，DSO 也许能够直接使用文件中存储的信息来计算体积内的最小值和最大值。这可能让渲染器避免预先对体积进行 shading（着色），从而大幅缩短 time to first pixel（首像素时间）。只有直接使用体积的 density signal（密度信号），或 density signal 只由 DSO 本身调制时，才能启用此项。对 density signal 的任何 shading 修改都要求关闭此参数。Value range（取值范围）：true/false。

volume:dsovelocity

int（整数）

0

仅用于使用 ImplicitField DSO 的 aggregate volumes（聚合体积）。如果设为 1，DSO 可以通过 BoxMotion 调用直接向渲染器提供 velocity bounds（速度边界）。当使用 Eulerian velocity（欧拉速度）时，这可以让渲染器避免预先对体积的 velocity data（速度数据）进行 shading（着色），从而显著改善渲染时间和 time to first pixel（首像素时间）。只有直接使用来自 DSO 的 velocity signal（速度信号）时，才能启用此项。Value range（取值范围）：true/false。

volume:fps

float（浮点）

1

volumetric velocity data（体积速度数据）的 frames per second（每秒帧数）。速度数据会除以该数量，以得出此帧的速度。

volume:shutteroffset

float（浮点）

1

用于解释 volumetric velocity（体积速度）的 shutter offset（快门偏移）。值为 1 时，会使用对象当前位置和对象下一帧位置作为 motion blur（运动模糊）的时间区间。值为 -1 时，会使用对象上一帧位置和当前位置作为时间。值为 0 时，会生成一个从上一帧中点开始、到下一帧中点结束的区间。

volume:velocityshuttercorrection

int（整数）

0

如果设为 1，用于 Eulerian motion blur（欧拉运动模糊）的 volumetric velocity data（体积速度数据）会被假定为相对于整帧，并会被校正以匹配当前 shutter interval（快门区间）。如果设为 0，则不进行校正。Value range（取值范围）：true/false。

Procedural（过程式）

Name（名称）

Type（类型）

Default（默认值）

Description（说明）

procedural:immediatesubdivide

int（整数）

0

由 RiProcedural 使用，用于强制 procedural（过程式）立即 subdivide（细分）（此行为通过 RiProcedural2 中的 __immediatesubdivide meta-parameter 支持）。此属性是“special”（特殊）的，因为它不会继承到 sub-procedural contexts（子过程式上下文）中，这意味着父级 immediate-subdivide procedurals 不会意外触发 immediate-subdivide 子级。值 0（默认值）表示禁用，1 表示启用，2 表示强制所有子级 procedurals 立即 crack（展开/裂分）。

procedural:reentrant

int（整数）

0

启用此属性会使随后出现的任何 procedural（过程式）不锁定 global mutex（全局互斥锁）。注意，这要求该 procedural 是 thread safe（线程安全）的。此属性默认禁用，也就是说所有 procedurals 都保持锁定。Value range（取值范围）：true/false。
