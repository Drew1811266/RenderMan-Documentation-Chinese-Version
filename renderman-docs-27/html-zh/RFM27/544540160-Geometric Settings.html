Geometric Settings

Source: https://rmanwiki-27.pixar.com/space/RFM27/544540160/Geometric+Settings
Downloaded: 2026-06-15T03:49:12+00:00

在 Maya 中指定实例属性时，它们可以像 Primvars（基本变量）这样命名，此时控制项位于形状节点上。也可以作为变换节点上的属性存在，并按每个实例变化。

示例命令：

rfm2.ui.user_attrs.user_attrs_add(data_attr, nodelist, pdict) -> add to one or more nodes

rfm2.ui.user_attrs.user_attrs_del(data_attr, nodelist, attrlist) -> delete from one or more nodes

rfm2.ui.user_attrs.user_attrs(node) -> list all user attributes

补充说明：

Emit Color Sets（发出颜色集） - 发出分配给对象的颜色集；在使用程序化着色渲染 MASH 对象时尤其有用。

Prevent Polygon Cracking（防止多边形裂缝） - 通常预期你会对 细分曲面进行置换。不过，如果你需要对具有“硬法线的多边形曲面进行置换，此选项会尝试防止对象在置换后裂开。

Emit Face IDs（发出面 ID） - 这允许曲面使用 Ptex 纹理。

Reference Object（参考对象） -  _WPref 和带参考对象的 place3d 应作为你正在动画化的对象的子级。实例若要让纹理以相同方式贴附，_WPref 是必需的。不过，在 IPR 中，如果你移动/平移对象但它没有发生形变，系统不会重新求值；如果只有 变换 发生变化，则 _WPref 不会更新。只有重启渲染，或以某种方式触发网格更新（例如微调置换边界），你才会看到正确的值。

为便于同 Instance Attributes（实例属性） 区分，这些也可能称为“原型属性”（原型属性），而 Instance Attributes 可以按实例变化。

falloffpower

float（浮点）

0

用于点。如果未提供或设为 0，点会具有硬边缘。值为 1 是一个“合理”值，可模拟通常基于余弦的衰减；对大多数制作体积粒子效果的人来说，这很可能是常用值。0 到 1 之间的值会让衰减更快，使点更快被侵蚀，也就是点的存在感更弱。大于 1 的值（直到无穷大）会让衰减更慢，直至几乎不存在衰减。

volume:aggregaterespectvisibility

int（整数）

空

如果设为 1，并且该体积属于某个聚合，则体积会遵守标准可见性属性。如果设为 0，则会忽略可见性属性，只应用积分器可见性设置。取值范围：true/false。

volume:dsominmax

int（整数）

空

目前仅用于聚合体积，并且只用于使用 ImplicitField DSO 的体积。如果设为 1，DSO 也许能够直接使用文件中存储的信息来计算体积内的最小值和最大值。这可能让渲染器避免预先对体积进行着色，从而大幅缩短首像素时间。只有直接使用体积的密度信号，或密度信号只由 DSO 本身调制时，才能启用此项。对密度信号的任何着色修改都要求关闭此参数。取值范围：true/false。

volume:dsovelocity

int（整数）

0

仅用于使用 ImplicitField DSO 的聚合体积。如果设为 1，DSO 可以通过 BoxMotion 调用直接向渲染器提供速度边界。当使用 Eulerian 欧拉速度时，这可以让渲染器避免预先对体积的速度数据进行着色，从而显著改善渲染时间和首像素时间。只有直接使用来自 DSO 的速度信号时，才能启用此项。取值范围：true/false。

volume:fps

float（浮点）

1

体积速度数据的每秒帧数。速度数据会除以该数量，以得出此帧的速度。

volume:shutteroffset

float（浮点）

1

用于解释体积速度的快门偏移。值为 1 时，会使用对象当前位置和对象下一帧位置作为运动模糊的时间区间。值为 -1 时，会使用对象上一帧位置和当前位置作为时间。值为 0 时，会生成一个从上一帧中点开始、到下一帧中点结束的区间。

volume:velocityshuttercorrection

int（整数）

0

如果设为 1，用于 Eulerian 欧拉运动模糊的体积速度数据会被假定为相对于整帧，并会被校正以匹配当前快门区间。如果设为 0，则不进行校正。取值范围：true/false。

过程式

名称

类型

procedural:immediatesubdivide

int（整数）

0

由 RiProcedural 使用，用于强制过程式立即细分（此行为通过 RiProcedural2 中的 __immediatesubdivide 元参数支持）。此属性是“特殊”的，因为它不会继承到子过程式上下文中，这意味着父级立即细分过程式不会意外触发立即细分子级。值 0（默认值）表示禁用，1 表示启用，2 表示强制所有子级过程式立即展开/裂分。

procedural:reentrant

int（整数）

0

启用此属性会使随后出现的任何过程式不锁定全局互斥锁。注意，这要求该过程式是线程安全的。此属性默认禁用，也就是说所有过程式都保持锁定。取值范围：true/false。
