AOVs

Source: https://rmanwiki-27.pixar.com/space/RFM27/544542501/AOVs
Downloaded: 2026-06-15T03:49:12+00:00

Passes 标签页的主要用途，是为渲染创建额外输出。这些输出也称为任意输出变量（Arbitrary Output Variables）（AOV）。  

也可以创建额外的 "passes"，它们更类似 Maya 的渲染层（Render Layers，也受支持）。每个渲染 Pass 都可以拥有自己的相机和渲染设置，用作全局渲染设置的覆盖项。

此界面可用于创建 Display Channel（显示通道，包含 AOV 选择和 LPE 来源）以及作为目标文件（TIFF、OpenEXR 等）的 Display Driver（显示驱动）。默认情况下，双击会把左侧菜单中的新 Display Channel 和 Driver 添加到右侧菜单。左侧菜单包含渲染常用的预设 AOV 和 LPE。它们分为以下类别：

  • Custom: 输入你自己的自定义 AOV 或 LPE

  • Standard: 这里包含大多数内置 AOV。

  • Diagnostic: 这里包含一部分 AOV，用于可视化像素采样、CPU Time 等诊断信息。

  • Lighting: 这是最常见的一组用于合成调整的LPE；direct diffuse、indirect specular 等输出都在这里。

  • Denoiser: 这些 AOV 由 Denoise 工具使用；选择 beauty Display Driver 并启用 Denoise 后，Batch Render 之后会对 Beauty 进行降噪。

  • Shadow: 这些是用于收集阴影的特殊 LPE

  • Mattes: 这些 Matte LPE 需配合 PxrMatteID

  • Integrator: 这些 AOV 内置在 Integrator 中，并且可能因积分器而异。

简短说明：双击左侧要渲染的 Channel。这样会自动创建一个同名的新显示驱动用于渲染。

如果你想要较短的列表，或希望这些通道输出到单个文件，请在使用 Channels 列表框左下角的箭头入列表按钮添加 Channel 时，确保右侧已选中一个 Display Driver。

必须在 Display Driver 中使用能够容纳这些通道的文件格式；我们建议几乎所有情况下都保留默认的 OpenEXR。如果不知道需要哪种文件类型，请保持默认设置。

第一次打开 AOV 标签页时，你会看到默认渲染 Pass 的输出。

Displays

Displays 标签页显示所选 Pass 的输出。它们位于右侧列表中。Final Pass 的默认输出如预期是 rgba，即典型图像的 颜色 (Ci) 和 alpha (a) 通道。你可以从左侧预设列表选择其他输出，并点击两个列表之间的箭头按钮添加。  自定义输出 也可以通过点击列表底部的 '+' 按钮添加。

每个输出都有自己的设置；在输出列表中选中条目后，这些设置会显示在列表下方。双击已有预设会将其添加到渲染列表中，首先列出 Display Driver (_directDiffuse)，其下方是 Display Channel（source，directDiffuse）。下面先说明 Display Driver 的选项。

Display Settings

Enable

允许此输出参与渲染

Display Type

选择 Display Driver 的输出文件格式，包括 Deep Data (DeepEXR)

OpenEXR

OpenEXR 是渲染最常用、最通用且功能强的文件格式。它是默认且推荐的文件类型。 Denoise 功能需要 OpenEXR 或 TIFF 类型。

As RGBA

按 RGBA 顺序排列通道

Storage

  • Tiled：图像会保存为分块格式的数据块

  • Planar：保存为扫描线格式

Type

  • Half Precision (16-half)

  • Float Precision (32-bit float)

通常 Half 对 颜色 AOV 和渲染 Pass 已经足够。对于 Z depth 或 P 等 Data AOV 和 Pass，建议使用 Float。OpenEXR 不支持更低的位深度。

Compression

  • ZIPS 是默认值，建议用于合成工作流，因为它按单条扫描线存储

  • ZIP 也有用，但在合成软件包中以 16 条扫描线为一组时会稍慢

  • 有关较少用选项的更多信息，可查看 Wiki 页面 中的 OpenEXR 说明

Compression Level

数据压缩程度；对于 DWAA 等有损方案，这可能会降低最终图像的质量

'it'

这用于渲染到 Image Tool 作为显示设备；没有其他选项。

DeepEXR

DeepEXR 是用于存储 deep data（深度数据）的常见格式，即逐采样数据，可用于按深度合成对象，例如 Volumes 中嵌套的对象。

Storage

  • Tiled：图像会保存为分块格式的数据块

  • Planar：保存为扫描线格式

Type

  • Half Precision (16-half)

  • Float Precision (32-bit float)

通常 Half 对 颜色 AOV 和渲染 Pass 已经足够。对于 Z depth 或 P 等 Data AOV 和 Pass，建议使用 Float。

Compression

我们在内部自动处理采样合并，以在大小和正确性之间取得最佳平衡；因此下面这些选项可用于控制输出帧的最终大小。

  • ZIPS 是默认值，建议用于合成工作流，因为它按单条扫描线存储

  • ZIP 也有用，但在合成软件包中以 16 条扫描线为一组时会稍慢

  • 有关较少用选项的更多信息，可查看 Wiki 页面 中的 OpenEXR 说明

PNG

PNG 是常见 Web 格式，包含用于透明度的 Alpha 通道

Quantize

此选项会尝试压缩配色方案以减小文件大小

Targa

Targa 是一种流行的 8 位（每通道）文件格式

Quantize

此选项会尝试压缩配色方案以减小文件大小

Texture

这会直接从渲染器渲染到 RenderMan .tex 格式。

用于此项的图像分辨率应具有 2 的幂尺寸，并且必须由用户设置。例如 512 x 512（其中 512 是 2 的幂且为正方形）。


Texture Format

  • Pixar：通常性能最好

  • OpenEXR：保存为 OpenEXR 格式

  • Tiff：保存为 TIFF 格式（注意此格式不能保存大于 4GB 的文件）

Bit Depth

设置输出文件的位深度；不适用于 OpenEXR

  • Byte：小文件

  • Short：小文件，比 Byte 包含更多数据

  • Float：文件较大，可保持完整精度

EXR Bit Depth

  • Half Precision (16-half)

  • Float Precision (32-bit float)

S and T Wrap Modes

  • black：适用于 UDIM 等基于贴片的工作流。

  • clamp：适用于非周期性纹理访问。

  • periodic：适用于周期性（重复）的纹理访问。

Compression

  • None：不执行压缩

  • Lossless：执行压缩，但不损失精度

  • Lossy：通过移除细节进行压缩并进一步减小大小

EXR Compression

我们在内部自动处理采样合并，以在大小和正确性之间取得最佳平衡；因此下面这些选项可用于控制输出帧的最终大小。

  • ZIPS 是默认值，建议用于合成工作流，因为它按单条扫描线存储

  • ZIP 也有用，但在合成软件包中以 16 条扫描线为一组时会稍慢

  • 有关较少用选项的更多信息，可查看 Wiki 页面 中的 OpenEXR 说明

EXR Compression Level

数据压缩程度；对于 DWAA 等有损方案，这可能会降低最终图像的质量

Tiff

Tiff 是常见格式，也可以存储浮点精度图像以及 Pass 数据

Format

  • uint8：8 位存储。文件较小，通常不适合后期合成工作。

  • uint16：16 位整数存储。文件较小，比 8 位对应格式有更好的还原效果。

  • float32：完整 32 位浮点图像。文件更大，但最适合后期合成。

Quantize

此选项会尝试压缩配色方案以减小文件大小

Compression

  • None：不执行压缩，文件保持完整大小且无损

  • LZW：TIFF 文件中非常常见的无损压缩。

  • Packbits (RLE)：快速无损压缩，压缩率低于 LZW

  • Deflate：常见无损压缩方案

  • Pixarlog：通常适合压缩 HDR 图像，同时不丢失高光数据

Resolution Unit

在 inches、centimeters 或 none 之间选择。这类似于告诉 Maya 你的单位分辨率。

Resolution

ImageWidth 和 ImageHeight 方向中每个 Resolution Unit（上方）的像素数。

Denoise

为所选 Display 启用降噪

Mode

  • Single Frame 用于静帧渲染

  • Cross-frame 用于动画或一系列帧

Remap

这在我们的滤波文档中有说明

  • Break Point：颜色保持不变的阈值下限，必须为非负值 (a)

  • Max Value：包含的最大值；为 0 时所有重映射关闭。它不能小于 Break Point 值。(b)

  • Smoothness：对 breakpoint 处二阶导数的调节（除非 a==b，否则函数是 C(1)，因此你无法指定一阶导数。如果此参数为 1，函数为 C(2)。较小的值会让它更平缓，较大的值会让它不那么平缓。如果大于约 2.5，曲线会有拐点，即在 breakpoint 右侧向上弯曲。将 c <= 0 设为无效值，且不建议将它设为 1 以外的任何值。默认值为 0。

Display Channels

Display Channel 是上方显示驱动使用的数据源。通常添加新 AOV 时，双击会添加一个新 AOV 以及一个显示驱动。这意味着你的 AOV 会被拆分到不同输出中。

如果希望它们写入单个文件（或更少文件），请在添加新 AOV 时先选中右侧的显示驱动，它会被添加到现有输出。下方左侧是新显示驱动的默认行为，右侧是添加显示通道时已有 driver 被选中的情况。你可以使用不同的显示驱动。

添加显示通道后，可以查看它的设置


Display Channel Settings

Enable

设置此通道是否可渲染。

Channel Type

数据使用的类型通常存储为支持浮点的格式，例如 OpenEXR 或 Float Tiff

  • Color

  • Float

  • Vector

  • Normal

  • Point 

Channel Source

这是 AOV 的来源，通常是在此处指定用于渲染的 LPE。

LPE Light Group

你可以在这里手动指定灯光组（light group），使其匹配已分配给灯光的灯光组；也可以使用下拉菜单，从灯光中已指定的现有灯光组中选择。这允许按灯光（组）输出 LPE。

如果在界面打开时向灯光添加了新的灯光组，可能需要关闭渲染设置和/或添加新的 LPE；这会强制它从场景重新填充组到菜单选项中。

Filter

你可以使用此部分来覆盖 指定滤波器。对于大多数颜色数据，继承的滤波器最好，这样可以匹配其他 AOV；但对于数据，你可能希望使用列出的 min、max、zmin、zmax 等滤波器之一，以避免跨对象边缘进行滤波。

Filter Width

这里可以更改此通道的 filter width（滤波宽度）。注意，使用不同宽度可能意味着它们不会与其他 AOV 对齐或匹配。

Statistics

这些选项并非专门面向艺术家；它们对流水线考量或渲染器外部的降噪应用可能有用，但并非必需使用。它们作为流水线实用工具暴露出来，因此你不必直接修改 RIB 文件。

  • Variance: 估计关联源通道中对像素有贡献的值的统计方差。

  • MSE: 类似 variance（方差），但会随着采样数增加而趋近于零。虽然它本身也有些噪声，但可以估计均方误差（mean-squared-error）相对于假设真值图像（ground-truth image）的大小。

  • Even: 将偶数编号采样渲染到此 AOV

  • Odd: 将奇数编号采样渲染到此 AOV
