AOVs

Downloaded: 2026-06-15T03:49:12+00:00

passes tab 的主要用途是为 render 创建 additional outputs。这些输出也称为 Arbitrary Output Variables (AOVs).  

也可以创建额外的 "passes"，它们更类似 Maya 的 render layers（也受支持）。每个 pass 都可以拥有自己的 camera 和 render settings，用作 global render settings 的 overrides。

此 UI 可用于创建 Display Channels（AOV selections 和 LPE sources）以及作为目标文件（TIFF、OpenEXR 等）的 Display Drivers。默认情况下，双击会把左侧 menu 中的新 Display Channel 和 Driver 添加到右侧 menu。左侧 menu 包含 rendering 常用的 preset AOVs 和 LPE。它们分为以下 categories：

  • Custom: 输入你自己的 custom AOV 或 LPE

  • Standard: 这里包含大多数 built-in AOVs。

  • Diagnostic: 这里包含一部分 AOVs，用于可视化 pixel samples 和 CPU Time 等。

  • Lighting: 这是最常见的一组用于 compositing 调整的 LPE；direct diffuse、indirect specular 等 outputs 都在这里。

  • Denoiser: 这些 AOVs 由 Denoise utility 使用；选择 beauty Display Driver 并启用 Denoise 后，Batch Render 之后会对 Beauty denoise。

  • Shadow: 这些是用于收集 shadows 的特殊 LPE

  • Mattes: 这些 Matte LPE 需配合 PxrMatteID

  • Integrator: 这些 AOVs 内置在 Integrator 中，并且可能因 integrator 而异。

简短说明：双击左侧要 render 的 Channels。这样会自动创建一个同名的新 display driver 进行 rendering。

如果你想要较短的列表，或希望这些 channels 输出到单个 file，请在使用 Channels box 左下角的 arrow-to-list button 添加 Channels 时，确保右侧已选中一个 Display Driver。

必须在 Display Driver 中使用能容纳这些 channels 的 file format；我们建议几乎 100% 的情况下都保留默认的 OpenEXR。如果不知道需要哪种 file types，请保持默认。

第一次打开 AOV tab 时，你会看到 default render pass 的 outputs。

Displays

Displays tab 显示所选 pass 的 outputs。它们位于右侧列表中。Final pass 的默认 output 如预期是 rgba，即典型图像的 color (Ci) 和 alpha (a) channels。你可以从左侧 preset 列表选择其他 outputs，并点击两个列表之间的 arrow button 添加。  Custom outputs 也可以通过点击列表底部的 '+' button 添加。

每个 output 都有自己的 settings；在 outputs 列表中选中 item 后，这些 settings 会显示在列表下方。双击已有 preset 会将其添加到 render 列表中，首先列出 Display Driver (_directDiffuse)，其下方是 Display Channel（source，directDiffuse）。下面先说明 Display Drivers 的选项。

Display Settings

Enable

允许此 output 进行 render

Display Type

选择 Display Driver 的 output file format，包括 Deep Data (DeepEXR)

OpenEXR

OpenEXR 是 render 最常用、通用且功能强的 file format。它是默认且推荐的 file type。 Denoise 功能需要 OpenEXR 或 TIFF types。

As RGBA

按 RGBA 顺序排列 Channels

Storage

  • Tiled：Image 会保存为 tile-formatted data blocks

  • Planar：保存为 scanline format

Type

  • Half Precision (16-half)

  • Float Precision (32-bit float)

通常 Half 对 color AOVs 和 passes 已经足够。对于 Z depth 或 P 等 Data AOVs 和 passes，建议使用 Float。OpenEXR 不支持更低 bit depths。

Compression

  • ZIPS 是默认值，建议用于 compositing workflow，因为它按单条 scanline 存储

  • ZIP 也有用，但在 compositing packages 中以 16 条 scanlines 为 bundle 时会稍慢

  • 有关较少用选项的更多信息，可查看 Wiki page 中的 OpenEXR 说明

Compression Level

数据压缩程度；对于 DWAA 等 lossy schemes，这可能会降低最终 images 的质量

'it'

这用于 render 到 Image Tool 作为 display device；没有其他选项。

DeepEXR

DeepEXR 是用于存储 deep data 的常见 format，即 per-sample data，可用于在 depth 上 composite objects，例如 Volumes 中嵌套的 objects

Storage

  • Tiled：Image 会保存为 tile-formatted data blocks

  • Planar：保存为 scanline format

Type

  • Half Precision (16-half)

  • Float Precision (32-bit float)

通常 Half 对 color AOVs 和 passes 已经足够。对于 Z depth 或 P 等 Data AOVs 和 passes，建议使用 Float。

Compression

我们在内部自动处理 samples merging，以在 size 和 correctness 之间取得最佳 balance；因此下面这些选项可用于控制 output frames 的最终 size。

  • ZIPS 是默认值，建议用于 compositing workflow，因为它按单条 scanline 存储

  • ZIP 也有用，但在 compositing packages 中以 16 条 scanlines 为 bundle 时会稍慢

  • 有关较少用选项的更多信息，可查看 Wiki page 中的 OpenEXR 说明

PNG

PNG 是常见 web format，包含用于 transparency 的 Alpha channel

Quantize

此选项会尝试压缩 color scheme 以缩小 file size

Targa

Targa 是一种流行的 8-bit（per channel）file format

Quantize

此选项会尝试压缩 color scheme 以缩小 file size

Texture

这会直接从 renderer render 到 RenderMan .tex format。

用于此项的 image resolution 预计应具有 power-of-two dimensions，并且必须由用户设置。例如 512 x 512（其中 512 是 power of two 且为 square）


Texture Format

  • Pixar：通常性能最好

  • OpenEXR：保存为 OpenEXR format

  • Tiff：保存为 TIFF format（注意此 format 不能保存大于 4GB 的 files）

Bit Depth

设置 output file 的 bit depth；不适用于 OpenEXR

  • Byte：小文件

  • Short：小文件，比 Byte 包含更多 data

  • Float：文件较大，可保持 full precision

EXR Bit Depth

  • Half Precision (16-half)

  • Float Precision (32-bit float)

S and T Wrap Modes

  • black：适用于 UDIM 等 tile-based workflows。

  • clamp：适用于 non-periodic texturing access。

  • periodic：适用于周期性的 periodic（repeating）texture access。

Compression

  • None：不执行 compression

  • Lossless：执行 compression，但不损失 precision

  • Lossy：通过移除 details 进行 compression 并进一步减小 size

  • ZIP 也有用，但在 compositing packages 中以 16 条 scanlines 为 bundle 时会稍慢

EXR Compression Level

数据压缩程度；对于 DWAA 等 lossy schemes，这可能会降低最终 images 的质量

Tiff

Tiff 是常见 format，也可以存储 floating point precision images 以及 pass data

Format

  • uint8：8-bit storage。file size 小，通常不适合 post compositing work。

  • uint16：16-bit integer storage。file size 小，比 8-bit counterpart 有更好的 reproduction。

  • float32：完整 32-bit float images。file size 更大，但最适合 post compositing。

Quantize

此选项会尝试压缩 color scheme 以缩小 file size

Compression

  • None：不执行 compression，file 为 full size 且 lossless

  • LZW：TIFF files 中非常常见的 lossless compression。

  • Packbits (RLE)：快速 lossless compression，compression 率低于 LZW

  • Deflate：常见 lossless compression scheme

  • Pixarlog：通常适合压缩 HDR images，同时不丢失 highlight data

Resolution Unit

在 inches、centimeters 或 none 之间选择。这类似于告诉 Maya 你的 per-unit resolution。

Resolution

ImageWidth 和 ImageHeight 方向中每个 Resolution Unit（上方）的 pixels 数。

Denoise

为所选 Display 启用 denoising

Mode

  • Single Frame 用于 still render 的单帧渲染

  • Cross-frame 用于 animation 或一系列 frames

Remap

这在我们的 Filtering documentation

  • Break Point：color 保持不变的阈值下限，必须为 non-negative (a)

  • Max Value：包含的 max value；为 0 时所有 remapping 关闭。它不能小于 Break Point value。(b)

  • Smoothness：对 breakpoint 处 second derivative 的 tweak（除非 a==b，否则 function 是 C(1)，因此你无法指定 first derivative。如果此 parameter 为 1，function 为 C(2)。较小 values 会让它更 flat，较大 values 会让它 less flat。如果大于 ~2.5，curve 会有 inflection point，即在 breakpoint 右侧向上弯曲。设置 c <= 0 是 illegal 的，且不建议将它设为除 1 以外的任何值。default value 为 0。

Display Channels

Display Channels 是上方 display drivers 使用的 sources。通常添加 new AOVs 时，双击会添加一个 new AOV 以及一个 display driver。这意味着你的 AOVs 会被拆分到不同 outputs。

如果希望它们写入单个 file（或更少 files），请在添加 new AOV 时先选中右侧的 display driver，它会被添加到现有 output。下方左侧是 new display driver 的默认行为，右侧是添加 display channel 时已有 driver 被选中的情况。你可以使用不同的 display driver。

添加 display channel 后，可以查看它的 settings


Display Channel Settings

Enable

设置此 channel 是否 renderable。

Channel Type

data 使用的 types 通常存储为支持 floating point 的 formats，例如 OpenEXR 或 Float Tiff

  • Color

  • Float

  • Vector

  • Normal

  • Point 

Channel Source

这是 AOV 的 source，通常是在此处指定用于 rendering 的 LPE。

LPE Light Group

你可以在这里手动指定 light group，使其匹配已分配给 lights 的 light groups；也可以使用 dropdown menu 从 lights 中已指定的现有 light groups 中选择。这允许 per-light (group) LPE output。

如果在 UI 打开时向 lights 添加了 new light groups，可能需要关闭 render settings 和/或添加 new LPE；这会强制它从场景重新填充 groups 到 menu choices。

Filter

你可以使用此 section 来 override specified filter。对于大多数 color data，继承的 filter 最好，这样可以匹配其他 AOVs；但对于 data，你可能希望使用列出的 min、max、zmin、zmax 等 filters 之一，以避免跨 object edges 进行 filtering。

Filter Width

这里可以更改此 channel 的 filter width。注意，使用不同 width 可能意味着它们不会与其他 AOVs 对齐或匹配。

Statistics

这些选项并非专门面向 artists；它们对 pipeline considerations 或 renderer 外部的 denoise applications 可能有用，并不是必需使用。它们作为 pipeline 的实用工具暴露出来，因此你不必直接修改 RIB files。

  • Variance: 估计 associated source channel 中 contributing to pixel 的 values 的 statistical variance。

  • MSE: 类似 variance，但会随着 samples 数增加而趋近于 zero。虽然它本身也有些 noisy，但可以为 mean-squared-error 相对于 hypothetical ground-truth image 的大小提供估计。

  • Even: 将 even numbered samples render 到此 AOV

  • Odd: 将 odd numbered samples render 到此 AOV
