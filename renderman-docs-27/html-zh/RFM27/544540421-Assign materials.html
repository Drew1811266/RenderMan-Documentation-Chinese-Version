着色器分配

gpuCache 在 Maya 中是单个形状，因此无法像平常一样分配材质。但我们可以使用 RenderMan 的 动态规则编辑器 在渲染时分配材质，但有以下限制：

  • 所有被引用的材质必须存在于场景中。

  • 不支持按 faceset 分配材质。

  • 任何会影响光线追踪核心的属性都会被渲染器忽略。这包括分组成员关系定义、选择性照明等。

  • 任何规则更改都需要重启 IPR 才会生效。

  • 这些限制将在后续版本中解除。

RLF 编辑器概览

打开动态规则编辑器，看看它的工作方式。

你可以点击工具架中的双尖括号图标，或点击 gpuCache 属性编辑器中的 Edit RLF File（编辑 RLF 文件） 按钮位于 gpuCache 的属性编辑器中。工具架按钮会编辑场景中的所有 Alembic，而每个节点上的按钮只会编辑与该节点关联的 RLF。

规则管理

左侧按钮负责所有规则和文件管理任务：

  • 添加新规则：创建一条新的空规则

  • 删除所选规则：点击规则编号选择一条或多条规则，然后按此按钮。

  • 复制所选规则：点击规则编号选择一条或多条规则，然后按此按钮。

  • 加载 RLF 文件：用磁盘上的文件替换编辑器内容。（仅场景编辑器）

  • 保存 RLF 文件：将编辑器内容保存到磁盘。

  • Auto-save changes（自动保存更改）：当图标为橙色时自动保存到磁盘。（仅节点编辑器）

规则顺序

规则会按照编辑器中的顺序求值。若要更改顺序，请点击规则编号并拖动到新位置。

规则控制

Enable/disable（启用/禁用）：当开关为绿色时，会对该规则求值。

Stop/continue（停止/继续）：当图标为叉号时，如果此规则匹配，规则求值会停止。向下箭头图标表示即使此规则匹配，也会继续求值后续规则。

材质：材质对应的着色组，它会引用曲面和置换着色器。

属性：插入或覆盖属性值。点击 新建/编辑 图标以打开属性对话框。

  • 点击 + 图标，并右键点击属性字段以选择一个受支持的属性。

  • 像平常一样选择一个或多个属性，然后点击 - 图标将其删除。

  • 具有固定可接受值列表的属性也可以通过右键点击来设置。

  • 保存窗口时，设置会自动保存。

Match type（匹配类型）：有 3 种匹配引擎：

  • exact：这是最快的方式，并且只匹配完整路径。

  • glob：基于 glob 匹配，使用以下标记：

    • *：任意数量的字符

    • ?：单个字符

    • **：层级中的任意深度

  • re： re2 正则表达式。这是最复杂、最精细，也可能最慢的匹配方法。

路径表达式：你可以双击字段来输入表达式，也可以从 Alembic 节点列表拖放。

若要拖放，必须先在 Alembic 节点列表中选择节点。

Notes（备注）：当事情变复杂时，可在此字段中写备注...

一条规则可以分配：

• 材质

• 属性

• 或两者都分配！

关于属性和节点类型

在 RenderMan 中，每个形状都是一个实例。这意味着附加到特定形状的属性会影响该形状的所有实例。

如果你想把属性分配给实例，请创建一个匹配该实例变换的表达式。这对于类似 maxspeculardepth, maxdiffusedepth, relativepixelvariance等经典属性尤其重要。

场景内着色

动态规则使用类似 shell 的 glob 表达式编写。更多信息请参见下方的 动态规则指南。

这是我们的起点：gpuCache 节点使用默认材质渲染：

为归档的一部分分配基础材质

我们准备了一些可供分配的基础材质。先把 hull_srf 材质分配给整个茶壶，并写下第一条规则：

  • 在节点列表中，选择我们要分配的层级的顶层节点：Rolling_Teapot。

  • 点击 添加规则（蓝色 + 图标）来创建新规则，或使用初始的空规则

  • 双击表达式字段进入编辑模式。此字段是 >_ 列。

  • 转到 Alembic 节点列表，并用鼠标中键将所选节点拖入表达式字段。

    • 请注意，拖放之前必须先在列表中选择一项。（你的窗口也必须 docked（停靠），不能在 Windows OS 上浮动；切换焦点会导致无法用鼠标中键拖动。）

  • 将表达式从 /Rolling_Teapot_Grp/Rolling_Teapot 编辑为下面这样：

    /**/Rolling_Teapot

    • ** 表示匹配任意嵌套深度（包括没有嵌套），而 * 只会匹配 / 分隔符之间的字符

    • 按 Enter 验证。

  • 右键点击 Payload（载荷） 字段（带滑块图标），然后选择 hull_srfSG。

  • 启动 IPR 进行测试。

在层级中匹配子字符串

现在我们要为茶壶上的铆钉着色。我们需要更复杂的表达式。

  • 复制第一条规则

    • 右键点击规则编号（本例中为 0），然后选择 复制规则 （位于 “-” 减号图标正下方的 复制图标）。

  • 扩展前一个表达式，以匹配任何包含单词 "rivet" 的对象。

    /**/*铆钉*/*

    • 第二个 ** 表示铆钉对象可以嵌套在茶壶下的任意层级

    • * 如果节点名称中出现 rivet，则 rivet * 会匹配。"rivet"、"Arivet" 和 "Brivet2" 都会匹配。

  • 在该模型上，还有一些螺丝可以使用相同材质。复制表达式，让它们也能匹配。

    /**/*screw*/*

规则执行顺序

我们来为茶壶的环分配材质。

使用相同工作流：复制规则，将 "rivet" 替换为 "ring" 并将 Payload 设置为 ring_srf。但这并不完全奏效，因为如你在节点列表中所见，有些形状名称中同时包含 "rivet" 和 "ring"，而环上的铆钉现在使用了错误的材质。

如果结果不正确，我们可以更改 规则执行顺序 （你的顺序可能与图中我的顺序不同）：如果先匹配 rivet，匹配引擎在找到匹配后会停止，随后只有仅包含 "ring" 的形状才会匹配第二条规则。

  • 将光标放在规则编号上，然后用鼠标左键拖动规则 1（rivets）到规则 0（ring）上方。

  • 渲染以检查结果。

当然，我们只是浅尝辄止，建议你阅读 Regular Expressions（正则表达式） 以了解更复杂的用法。

如果你已完成着色，请选择 disk/save RLF（RenderMan Look File）图标。导入资产时可以使用它，每次分配着色节点。这也可以成为你的资产发布方案的一部分。

自包含资产着色

如果你想导入已经完整着色的 资产，有 2 个选项：

  • 当 RfM 在与归档相同的位置找到同名 RLF 文件时，它会自动应用该文件。

    • 这在流程管线中并不总是很灵活，但对于归档资产可能很有用。

RLF 文件格式不包含材质描述，因此你必须确保：

  • 所有被引用的材质都存在于你的场景中。

  • Output All Shaders（输出所有着色器） 已在 Render Globals 的 Advanced（高级）部分中勾选。

  • 如果 gpuCache 节点的 RLF File Name（RLF 文件名） 字段包含指向 RLF 文件的路径，RfM 会将其应用到该 gpuCache 节点。

    • 这样就很容易用脚本进行 外观分配和更新。

请注意，你仍然可以在场景中使用动态规则编辑器覆盖自动 RLF 分配！

在这种情况下，场景 RLF 规则会先于归档的规则执行，因此有机会覆盖它们。

关于 RLF 导出的说明

当你在场景中引用多个 RLF 文件时，必须确保所有着色节点都有唯一名称。如果多次定义相同名称，渲染器会使用排在最前面的那个，你会得到意外结果。

你可以通过点击 RLF 编辑器中的  图标导出 RLF 文件，也可以编写一个简单的 Python 脚本，将场景当前的 RLF 数据保存到文件：
