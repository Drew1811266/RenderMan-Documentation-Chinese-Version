Overview（概览）

烘焙允许用户将图案网络预计算，或将其“烘焙”为 2D 纹理图像或 3D 点云。烘焙有几个优势：

  • 通过避免在运行时计算昂贵的图案网络来缩短渲染时间。这种节省可以跨许多帧累积。
  • 作为着色管线的一部分发布外观开发资产。
  • 无需使用 Pref，即可从参考位置锁定程序化图案信息
  • 在渲染器、实时视口和 3D 打印之间传递资产。

烘焙在 RenderMan 中通过直通图案插件公开，这些插件允许用户从同一场景同时渲染和烘焙。无需大量修改资产或场景，用户即可：

  • 使用之前已烘焙的结果（如果存在），或者
  • Read（读取）图案网络，而不是读取已烘焙结果。这可能因任何原因发生，但通常是因为已烘焙结果不存在，或用户希望忽略已烘焙结果以便调整图案网络。

烘焙本身通过使用 bake hider（烘焙 hider）启用。

Texture Baking（纹理烘焙）

2D 烘焙允许用户将图案信号烘焙到纹理图像。用户可以直接烘焙到 txmake (.tex)、TIFF (.tif) 和 OpenEXR (.exr) 输出。2D 烘焙要求用户提供格式良好的纹理流形，也就是说，用户已按需求正确创建并布局 UV。虽然最常见的纹理流形是 ST，但也支持任意流形。也支持 UDIM 等图集输出。

RenderMan 提供一个通用纹理烘焙图案： PxrBakeTexture

对于重叠的纹理流形，烘焙行为未定义。

Point Cloud Baking（点云烘焙）

RenderMan 提供一个通用点云烘焙图案： PxrBakePointCloud。这可用于生成 Ptex 图像。

Blender Setup（Blender 设置）

烘焙可以在 render properties（渲染属性） 中的 Baking panel（烘焙面板）下设置。

Bake（烘焙）

这会启动一个单帧烘焙渲染。如果想处理多帧，或想执行批量渲染，请使用 External Bake Render 按钮（参见 Blender 中的批量渲染) 

Bake Mode（烘焙模式）

在 Pattern（图案）、Illumination（照明）或 All（全部）之间选择。图案模式要求对象的材质着色器网络中某处存在 PxrBakeTexture 或 PxrBakePointCloud；照明模式会将每个 AOV 烘焙为 pointcloud (.ptc) 文件或 2D 纹理文件；全部模式会同时烘焙图案和照明。

Baking Illumination Mode（照明烘焙模式）

烘焙照明时，2D 会写入 2D 纹理，3D 会写入 3D 点云文件（.ptc）。

Density（密度）

执行 3D 烘焙时，此项决定点云的密度。数值越低，创建的点云文件越密集。

Resolution（分辨率）

如果是 2D 烘焙，此项决定要写出的 2D 纹理分辨率。

Filename Pattern（文件名模式）

执行照明烘焙时，此控制项指定应使用的文件名模式。Global 会把所有对象烘焙到一个文件中。Identifier 会为每个对象烘焙一个文件，并用对象名称作为文件名。Filename Attribute 会使用对象在 Baking panel 中 filename attribute（文件名属性）上设置的任何值（参见 object properties（对象属性） )

Primvar S/Primvar T

对于 2D 烘焙，可以通过设置 RenderMan 应用于 u 和 v 的 primitive variable（图元变量）名称，来使用不同的 UV 集。

Invert T

RenderMan 是否应反转 T 方向纹理坐标中的值。
