要启用遮挡保留对象，请先选择要设为遮挡保留对象的对象，并在 Object Properties（对象属性） 中，将 Hold-Out 设置为 Yes：

接下来，在 Render Properties（渲染属性） 中，将 Render Holdouts 改为 In Alpha 或 Separate AOV。

选择 Alpha 会把遮挡保留结果放入 beauty 图像的 Alpha 中；Separate AOV 会输出单独的图像。

关于遮挡保留对象


视觉特效艺术家经常需要把 CG 对象合成到实拍素材板上。为了让观众相信 CG 对象真实存在且属于该场景，会使用遮挡保留对象来收集 CG 对象的阴影和反射，使其与实拍素材板中的阴影和反射非常接近。

几何体

遮挡保留对象用于代替真实世界对象，以接收来自 3D 对象的光照交互。在 RenderMan 中，通过启用 trace holdout 属性来指定遮挡保留对象。

Attribute "trace" "int holdout" [1]

积分器与 AOV

遮挡保留是在积分器层级实现的照明功能。它们能够接收多种积分器特定照明效果，例如漫反射与镜面反射、透射以及焦散。支持遮挡保留的可用积分器包括： PxrPathTracer, PxrVCM，以及 PxrUnified。

对于遮挡保留几何体，LPE 工作流保持不变。

由于遮挡保留工作流涉及多个 AOV，启用 adapt all 进行渲染可以降低噪点，代价是速度变慢，因为它可能会根据噪点更高的 AOV 触发更多采样。

灯光

默认情况下，光源会被视为遮挡保留对象，并且不会直接照亮其他遮挡保留对象。这样可以避免用于照亮非遮挡保留几何体的灯光，与素材板图像上已捕获的照明之间产生双重贡献。

要创建一个会直接照亮遮挡保留对象的光源，请使用 "__illumholdout" 作为灯光组后缀。这会告诉积分器插件在确定照明时，将此灯光视为非遮挡保留光源。

Light "PxrSphereLight" "spherelight" "string lightGroup" ["light__illumholdout"]

Alpha、阴影与合成

遮挡保留贡献可以使用 Alpha 和阴影通道作为遮罩，合成到素材板上。阴影通道由 PxrShadowDisplayFilter 插件生成，并且可以选择写入自己的 AOV，或添加到 Alpha 中。occluded 和 unoccluded 输入 AOV 应使用 holdouts LPE 前缀进行渲染，使它们只包含遮挡保留几何体的阴影项。

DisplayChannel "color shadow"
DisplayChannel "color occluded" "string source" ["color lpe:holdouts;C[DS]+[LO]"]
DisplayChannel "color unoccluded" "string source" ["color lpe:holdouts;unoccluded;C[DS]+[LO]"]

可以在 RenderMan 中使用 PxrImageDisplayFilter 插件来交互式预览合成。它会使用 Alpha 通道和遮挡保留阴影 AOV 遮罩素材板，将素材板与 beauty 通道合成。

合成工作流

合成图像时，如果阴影遮罩是 beauty 的一部分，只需执行简单的 over 操作。这会根据渲染设置提供正确的阴影密度和外观。

不过，这意味着调整阴影密度会绑定到 beauty Alpha，因此需要一个遮罩来防止阴影调整破坏 CG 对象合成。Cryptomatte 可以作为常用工具集的一部分帮你处理这个问题。

如果单独输出阴影遮罩，就可以单独调整阴影密度；不过可能会遇到覆盖范围问题。解决方法是在合成中把 beauty 和阴影遮罩的 Alpha 合并在一起。一个使用 Nuke 的示例见 这里。
