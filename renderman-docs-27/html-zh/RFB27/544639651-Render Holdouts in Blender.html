要启用 holdouts（遮挡保留/遮罩对象），请先选择要设为 holdout 的对象，并在 object properties（对象属性） 中，将 Hold-Out 设置为 Yes：

接下来，在 render properties（渲染属性） 中，将 Render Holdouts 改为 In Alpha 或 Separate AOV。

选择 Alpha 会把 holdout 放入 beauty image（美图/主图）的 alpha 中；Separate AOV 会输出单独的图像。

About Holdouts（关于 Holdouts）


VFX artists（视觉特效艺术家）经常需要把 CG objects（CG 对象）合成到 live action film plate（实拍底片/素材板）上。为了让观众相信 CG objects 真实存在且属于该场景，会使用 holdouts 来收集 CG objects 的 shadow（阴影）和 reflection（反射），使其与 live action plate 中的 shadow 和 reflection 非常接近。

Geometry（几何体）

Holdout objects（holdout 对象）用于代替真实世界对象，以接收来自 3D objects（3D 对象）的光照交互。在 RenderMan 中，通过启用 trace holdout attribute（属性）来指定 holdout objects。

Attribute "trace" "int holdout" [1]

Integrators and AOVs（积分器与 AOV）

Holdouts 是在 integrator（积分器）层级实现的 illumination feature（照明功能）。它们能够接收多种 integrator-specific illumination effects（积分器特定照明效果），例如 diffuse 和 specular reflections（漫反射与镜面反射）、transmissions（透射）以及 caustics（焦散）。支持 holdouts 的可用 integrator 包括： PxrPathTracer, PxrVCM，以及 PxrUnified。

对于 holdout geometry（holdout 几何体），LPE workflow（LPE 工作流）保持不变。

由于 holdouts workflow（holdouts 工作流）涉及多个 AOVs，启用 adapt all 进行渲染可以降低 noise（噪点），代价是速度变慢，因为它可能会根据噪点更高的 AOVs 触发更多 samples（采样）。

Lights（灯光）

默认情况下，light sources（光源）会被视为 holdouts，并且不会直接照亮其他 holdout objects。这样可以避免用于照亮 non-holdout geometry（非 holdout 几何体）的灯光，与 plate image（底片图像）上已捕获的照明之间产生 double contribution（双重贡献）。

要创建一个会直接照亮 holdout objects 的 light source，请使用 "__illumholdout" 作为 light group suffix（灯光组后缀）。这会告诉 integrator plug-ins（积分器插件）在确定 illumination（照明）时，将此灯光视为 non-holdout light source（非 holdout 光源）。

Light "PxrSphereLight" "spherelight" "string lightGroup" ["light__illumholdout"]

Alpha, Shadows, and Compositing（Alpha、阴影与合成）

Holdout contribution（holdout 贡献）可以使用 alpha 和 shadow pass（阴影通道）作为 masks（遮罩），合成到 film plate（素材板）上。shadow pass 由 PxrShadowDisplayFilter plug-in（插件）生成，并且可以选择写入自己的 AOV，或添加到 alpha 中。occluded 和 unoccluded input AOVs（遮挡/未遮挡输入 AOV）应使用 holdouts LPE prefix（LPE 前缀）进行渲染，使它们只包含 holdout geometry（holdout 几何体）的 shadowing term（阴影项）。

DisplayChannel "color shadow"
DisplayChannel "color occluded" "string source" ["color lpe:holdouts;C[DS]+[LO]"]
DisplayChannel "color unoccluded" "string source" ["color lpe:holdouts;unoccluded;C[DS]+[LO]"]

可以在 RenderMan 中使用 PxrImageDisplayFilter plug-in（插件）来交互式预览 Compositing（合成）。它会使用 alpha channel（Alpha 通道）和 holdout shadow AOV（holdout 阴影 AOV）遮罩 plate，将 film plate（素材板）与 beauty channel（美图通道）合成。

Compositing Workflow（合成工作流）

合成图像时，如果 shadow matte（阴影遮罩）是 beauty（美图）的一部分，只需执行简单的 over operation（叠加操作）。这会根据 render settings（渲染设置）提供正确的 shadow density（阴影密度）和外观。

不过，这意味着调整 shadow density（阴影密度）会绑定到 beauty alpha（美图 Alpha），因此需要一个 mask（遮罩）来防止阴影调整破坏 CG object composite（CG 对象合成）。Cryptomatte 可以作为常用工具集的一部分帮你处理这个问题。

如果单独输出 shadow matte（阴影遮罩），就可以单独调整 shadow density（阴影密度）；不过可能会遇到 coverage issue（覆盖范围问题）。解决方法是在 comp（合成）中把 beauty（美图）和 shadow matte 的 alpha 合并在一起。一个使用 Nuke 的示例见 这里。
