有关获取支持的一般信息，请参阅 获取支持 页面。

复现问题/错误

报告问题或错误时，我们通常需要一种复现该问题的方法。这通常意味着我们需要查看你的文件/资产。

在 RfB 中，有一个方便的 Package Scene（打包场景）按钮，可用于将文件打包在一起，以便发送给 RenderMan 支持团队。

调试日志

对于崩溃问题，RfB 可以把渲染会话记录到调试 Python 文件中。要开始记录日志，请在 Render Properties（渲染属性）选项卡中启用 Debug Logging，如下图所示。

此选项必须在 之前 渲染开始前启用

日志会放在当前 .blend 文件所在的同一目录中（带有 .debug.py 文件扩展名）。请确保当前 .blend 文件所在文件夹可写。

日志文件可能占用大量磁盘空间，尤其是较重的场景。生成完日志文件后，应确保关闭此选项。

Package Scene（打包场景）

要打包场景，请在视口中按 'n' 键。这应会打开侧边菜单。

接下来，点击 RenderMan 选项卡。

在 Utilities 下，点击 Package Scene 按钮。这会打开文件浏览器。

选择一个新的目录或已有的空目录，然后点击 Package Scene。该目录现在应包含一个新的 ZIP 文件，你随后可以把它随错误报告一起发送。

选项：

  • Include Blend File（包含 Blend 文件）：这会把 .blend 文件包含在 ZIP 包中

  • Include Debug Log（包含调试日志）： 如果存在日志 .debug.py 文件，则将其包含在 ZIP 包中。

  • Use RenderMan Textures（使用 RenderMan 纹理）： 这会将场景中的所有纹理文件路径替换为等效的 RenderMan .tex 纹理。
