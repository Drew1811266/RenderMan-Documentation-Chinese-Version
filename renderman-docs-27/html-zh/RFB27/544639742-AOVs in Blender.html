AOVs in Blender

Source: https://rmanwiki-27.pixar.com/space/RFB27/544639742/AOVs+in+Blender
Downloaded: 2026-06-15T03:49:12+00:00

Arbitrary Output Variables（任意输出变量，AOV）是渲染器生成的辅助图像。可以同时生成任意数量的 AOV，每个 AOV 都可以写入不同文件、使用不同的显示驱动器，或采用不同的像素过滤器设置。在某些情况下，可能会使用特殊的像素过滤模式，避免以不合理的方式混合来自不同采样的值；这些模式通常会选取单个采样来代表整个像素。

大体而言，RIS 中的 AOV 主要分为以下类别：内置、积分器（全局）、自定义以及 光路表达式。

如果用户关注通常称为“渲染通道（render passes）”、用于合成 Beauty 图像的内容，请查看关于 光路表达式（LPE）。

AOV 管理器

默认情况下，RenderMan for Blender 会尝试转换 Blender 自带的默认输出。

如果点击 View Layer Properties（视图层属性） ，这个 UI 对资深用户来说应该大体熟悉。



不过，也可以点击“Switch to RenderMan Displays”切换为使用 RenderMan for Blender 的 AOV 管理器；点击该按钮后，UI 会显示为如下形式：



该按钮还会尝试根据先前 UI 中选定的输出创建等效的 AOV。
AOV 列表下的每个项目都是一个包含若干通道的单独输出。



从列表中选择一个项目后，会显示该 AOV 的文件格式选项以及相关通道。

这里可以看到 dspy_2 会输出到 OpenEXR。
打开 OpenEXR Settings（OpenEXR 设置）区段后，会显示 OpenEXR 显示驱动器的选项，例如压缩和类型（float 或 half float）。

Camera 选项允许你使用不同于主场景摄像机的相机来渲染该 AOV。Denoise 选项会让 RenderMan 在外部/批量渲染期间对该 AOV 执行降噪。 



要向 AOV 添加通道，请使用 Add Channel（添加通道）下拉菜单。



可以点击 Delete Channel（删除通道）从 AOV 中移除通道。


选择一个通道后，还会出现一组选项，用于修改该通道的行为。界面也会显示类型和来源。

也可以从 Add Channel 下拉菜单中点击 Custom（自定义），创建自己的自定义通道。
然后需要为该通道指定名称、类型和来源（Light Path Expression，光路表达式）。
