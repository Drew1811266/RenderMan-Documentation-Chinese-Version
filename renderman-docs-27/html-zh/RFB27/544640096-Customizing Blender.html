自定义 Blender

Source: https://rmanwiki-27.pixar.com/space/RFB27/544640096/Customizing+Blender
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan for Blender（RfB）的许多方面都可以通过 JSON（.json 文件）自定义，例如默认值和标签。

环境变量

可以使用 rfb_envvars.json 文件设置环境变量。该文件会在启动时从用户配置目录自动读取（参见： Blender 文档）。注意，此方法不能用于设置任何标准 Blender 环境变量，包括 OCIO。

文件格式示例：

{
    "$schema": "./schemas/rfbEnvVarsSchema.json",
    "environment": {
        "RFB_LOG_LEVEL": {
            "value": "INFO"
        },
        "RFB_SITE_PATH": {
            "value": "/path/to/rfb_config_files"
        }
    }

高级：如果希望 RfB 忽略 rfb_envvars.json，请将环境变量 RFB_IGNORE_ENVVARS_JSON 设置为 1

配置路径

为了查找配置 JSON 文件，RfB 会按以下顺序检查环境变量：

  • RFB_SITE_PATH

  • RFB_SHOW_PATH

  • RFB_USER_PATH

每个环境变量也可以包含多个以冒号分隔的路径（在 Windows 上，这些路径以分号分隔）。

也可以在 RenderMan for Blender 首选项的 Advanced（高级）区段下指定配置路径（需要重启）。

rfb.json

这个 JSON 文件用于配置某些设置，例如站点的 Tractor 引擎配置、目录映射，或添加额外的用户标记。下面是一个示例：

{
    "$schema": "./schemas/rfbSchema.json",
    "tractor_cfg": {
        "engine": "tractor-engine",
        "port": 80,
        "user": ""
    },    
    "dirmaps": {
        "seahome_to_z": {
            "from": "/seahome/",
            "to": "Z:/",
            "zone": "UNC"
        }
    },
    "woffs": [
        "R90007"
    ], 
    "user tokens": ["PROD", "SHOT"]
}

配置面板属性

某些 RenderMan 面板中的属性可以配置为使用不同的默认值。下面是一个示例。

{
    "name": "rman_properties_scene",
    "params": [
        {
            "name": "motion_blur",
            "label": "Motion Blur (Blurry!)",
            "type": "int",
            "default": 1,
            "help": "TURNS ON MOTION BLUR!"
        },
        {
            "panel": "RENDER_PT_renderman_custom_options",
            "name": "user_foobar",
            "label": "FooBar",
            "riopt": "user:foobar",
            "type": "int",
            "default": 0,
            "help": "This is a test user option"
        }        
    ]
}

"name" 应该填写要覆盖的配置名称（此例为场景属性）。"params" 应跟在 "name" 之后。示例中的第一个属性是已有属性，我们覆盖了 "label"（标签）、"default"（默认值）和 "description"（描述）。如果要添加新属性，还需要指定该属性应绘制在哪个面板中；此例将它添加到场景的自定义选项面板。这个示例还展示了如何添加 RenderMan 用户选项。注意，在这种情况下，JSON 文件本身的名称并不重要；只要第一个 "name" 与已知配置匹配即可生效。应用上面的覆盖后，结果应如下所示：

要获取可配置面板列表，可以在 Blender 的 Python 控制台中运行以下代码：

import RenderManForBlender.rfb_api as rfbapi
rfbapi.GetConfigurablePanels()

着色器默认值

还可以覆盖任意 RenderMan 着色器的默认值（BxDF、图案节点等）。下面是覆盖 PxrEnvDayLight 参数的示例：

{
    "name": "PxrEnvDayLight.args",
    "params": [
        {
            "name": "sunDirection",
            "default": [0.0, 0.0, 1.0] 
        }
    ]
}

在这个例子中，我们希望更改 sunDirection 的默认值，使其默认指向 Z 轴正方向。"name" 应该是要覆盖的 Args 文件名称（也可以是 OSL 着色器的名称），后面跟随要配置的参数。注意，在这种情况下，JSON 文件本身的名称并不重要；只要第一个 "name" 与已知着色器 Args 文件匹配即可生效。

显示通道

最后，还可以添加可在 AOV 中使用的自定义显示通道。注意，这个 JSON 文件的名称 确实 重要；它必须命名为： rman_dspychan_definitions.json。下面是添加名为 MyAOV 的新通道的示例。

{
    "$schema": "./schemas/rfbDspyChansSchema.json",
    "channels": {
        "MyAOV": {
            "description": "This a custom AOV",
            "channelType": "color",
            "channelSource": "MyAOV",
            "group": "Pixar"
        }        
    }
}

自定义着色器

还可以使用环境变量 RMAN_SHADERPATH 添加自己的自定义着色器（适用于 OSL 着色器），并使用 RMAN_RIXPLUGINPATH 添加 C++ 着色器。对于后者，还需要配套的 Args 文件。 

其他示例
