PxrProjectionLayer

Source: https://rmanwiki-27.pixar.com/space/REN27/542224380/PxrProjectionLayer
Downloaded: 2026-06-15T03:49:12+00:00

PxrTexture 的专用版本，用于投射纹理；设计上与 PxrProjectionStack 配合使用。

它提供额外功能，用于优化纹理制作，并一次应用多个纹理。


输入参数

Filename

纹理的文件名。必须提供此参数。

此参数不应留空，否则会改为投射 missing color 。

First Channel Offset

用于文件名查找的首个通道偏移量。

Filter

选择纹理查找期间可使用的不同重建滤波器。PxrProjectionLayer 可用的滤波器为：

ValueFilter
0Closest
1Smart cubic
2Linear

Blur

指定从纹理文件读取的图像需要模糊的程度。


Mip Interpolate

选择是否在多分辨率纹理的相邻分辨率之间插值，从而在各级别之间获得更平滑的过渡。


Missing Color

如果打开纹理时出错，则使用此颜色。


Missing Alpha

如果打开纹理时出错，则使用此 Alpha 值。


Linearize

对纹理应用反向 sRGB 变换。如果你在 sRGB 空间中绘制纹理（大多数绘制软件的默认设置），但在线性数据空间中查看数据，纹理会显得发白。此选项会对纹理应用 sRGB 变换，使其再次呈现视觉线性的效果。


PreMultiply

如果希望最终颜色乘以 Alpha，请启用此项。


Mask

可以使用黑白遮罩来避免在不可见点上读取纹理。PxrProjectionLayer 提供这样的输出（resultMask），用于避免读取被遮挡或位于投射窗口之外的纹素。


Manifold

提供应用纹理的范围。在这里接入 PxrProjector，以向纹理函数提供映射坐标。默认值为 s,t。

通道 

通道部分允许投射最多 16 个额外纹理，这些纹理可以传递给 PxrProjectionStack 节点进行合成，并输出到单独的 AOV 或输出。

Premultiply Channels

将通道颜色乘以 resultA。当这些通道尚未由主纹理的 Alpha 和遮罩输入预乘时，这很有用。

Channels Filenames

最多 16 个额外颜色纹理的文件名。

Channels Linearize

对纹理应用 sRGB 线性化。

Channels Missing Color

无法读取纹理时使用的颜色。

Channels Input Black

调整纹理的黑场。类似于 Photoshop 的“色阶”。

Channels Input Gamma

调整纹理的 Gamma。类似于 Photoshop 的“色阶”。

Channels input White

调整纹理的白场。类似于 Photoshop 的“色阶”。

Channels Output Black

设置纹理的最大黑色值。类似于 Photoshop 的“色阶”。

Channels Output White

设置纹理的最小白色值。类似于 Photoshop 的“色阶”。



高级纹理设置

MipBias

偏移多级纹理选择：渲染时选择更高分辨率（负整数）或更低分辨率（正整数）的层级。

MaxResolution

将多级纹理选择限制到首选最大级别。根据你的选择，这可能会阻止最高分辨率被渲染。

OptimizeIndirect

在间接命中上将纹理滤波器设置为 Box（低成本），而不是使用其他可见效果所选的滤波器。


输出参数

resultRGB

经过过滤的颜色结果。请注意，所有结果都会从 startChannel 偏移量开始查找。

resultR

R 通道结果

resultG

G 通道结果

resultB

B 通道结果

resultA

Alpha 通道，会考虑通过遮罩输入提供的遮罩。

outChannelsRGB

RGB 通道数据会打包到单个结构输出中。PxrProjectionStack 会从该结构的 numChannels 成员获取可用层数。
