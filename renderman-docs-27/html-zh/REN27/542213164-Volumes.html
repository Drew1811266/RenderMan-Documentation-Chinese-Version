渲染雾、烟、云、浑浊水体甚至玻璃等效果很复杂，因为光不再只是与材质表面交互；它还会在 参与介质中发生衰减和散射。 体积渲染 是处理复杂光传输所必需的。

处理 均匀 体积效果时（参与介质在体积内部不发生变化，即清澈玻璃或清水），可以使用像 PxrVolume 这样的专用体积着色器，或使用像 PxrSurface 这样支持某些体积效果的表面着色器，并将其应用到任意类型的闭合几何体上，例如 细分网格。不过，如果效果是 非均匀 的（例如雾或云），通常就需要一种特殊几何类型，以提供会在三维域中变化的值。RenderMan 提供了体积图元（RiVolume）来满足这一需求。

体积图元本质上就是一个与坐标轴对齐的矩形盒，它可以在盒内的三维坐标处返回任意变量的值。这不同于参数曲面，参数曲面通常只能响应由二维坐标参数化的表面查询。对于烟雾等参与介质的体积渲染，最常用的变量是体积密度。对于火焰等更复杂的情况，燃料、温度或速度等数据场也可能是与体积图元关联的变量的一部分。

体积描述

RenderMan 通常使用两种体积图元描述方式。

  • 最简单的方式是一个在三维中细分为规则体素栅格的盒，并在栅格的每个角点为任意变量提供值。在盒内查找某个变量时，会对最近的栅格点执行三线性或三次插值。这种表示虽然简单，但对于高密度体积可能会生成笨重难用的 RIB 文件。

    RenderMan for Maya 使用这种方法来渲染 Maya 流体。在下面所示的 Maya 用户界面中，盒的边界可在左侧查看器中直接操作。在属性编辑器中，resolution（分辨率）描述盒的细分数量，这最终会影响体积模拟和渲染的精度；在此例中，盒被划分为 10x10x10 个体素。与体积关联的变量在 Contents Method（内容方法）下拉菜单中描述；这里启用了变量“Density”和“Velocity”，而“Temperature”和“Fuel”处于禁用状态。




  • 如果体积已经通过外部资源描述，例如磁盘上的 OpenVDB 文件，则体积图元支持任意 C++ 体积场插件 ，这些插件可以直接与外部资源对接，以计算盒范围内任意变量的值。RenderMan 提供了一个名为 impl_openvdb 的插件，可以直接使用 VDB 文件。在这种情况下，体积描述只涉及盒的范围、OpenVDB 文件，以及 OpenVDB 文件中对应有用图元变量的网格。

体积着色

由于体积的几何描述相对简单，设置体积的着色参数对整体外观至关重要。对于雾、烟和云等体积效果，RenderMan 提供了专用着色器 PxrVolume。完成体积模拟后，典型的渲染工作流包括：按上文设置几何描述，决定几何描述中的哪些变量需要直接映射到 PxrVolume 的输入，或使用中间 Pattern 节点重新映射，最后调整 PxrVolume 中的设置以控制整体体积外观。请注意，体积中的散射由所选积分器中的 Max Path Length（最大路径长度）参数控制，而不是由单独的漫反射和镜面追踪深度控制。

“dice” “minlength”属性应设置为 -1，以根据体素数据为最小长度应取的值提供提示。未来这可能成为默认行为。
