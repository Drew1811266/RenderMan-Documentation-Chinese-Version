Installing on macOS

Downloaded: 2026-06-15T03:49:12+00:00

最低系统要求

此版本支持以下操作系统。RenderMan 也许可以在不满足这些要求的环境中运行，但可能不稳定，且不属于支持范围：

  • macOS 15.x Sequoia

Pixar 的 RenderMan 软件以下列形式交付： 

  • RenderMan-Installer-version-osxflavor.dmg - RenderMan 渲染工具包。

该 flavor 字符串包含常见发行版，例如 Sequoia，以及 clang/icc 版本和微处理器指令集，例如 i686。把这些部分组合起来，例如会得到一个名为 RenderMan-Installer-27.0.0_1234567-osxSequoia_clang16.x86_64.dmg。

还必须满足以下附加要求：

  • 所有系统：Intel CPU 能够运行完整的 AVX 指令集。

基于 GUI 的安装

要安装软件，只需双击已下载的相应 DMG 文件。


接下来我们将逐步说明这个过程。首先，你必须使用论坛账户登录，并提供已购买许可证的序列号。

接下来，如果你是在浮动许可证服务器上 首次 安装，可能会看到以下内容：

这表示安装器即将把你的许可证服务器绑定到当前运行安装器的主机。

如果这台机器 不是 你希望运行许可证服务器的机器，请按 Cancel（取消） 按钮，并在你希望指定为许可证服务器的机器上重新运行安装器。

安装期间，你会看到 RenderMan 24 许可证服务器，而不是 RenderMan 25 许可证服务器。这是预期现象。


如果你决定将这台机器作为许可证服务器，或者你正在安装节点锁定许可证，安装器随后会联系 Pixar 并获取你的许可证文件。默认情况下，它会把许可证文件放在：

  • Windows: C:\Program Files\Pixar\pixar.license

  • macOS: /Applications/Pixar/pixar.license

  • Linux: /opt/pixar/pixar.license

接下来，你会看到希望安装的软件包列表。 

安装器还会尝试查找可安装的相应插件。不过，如果它找不到 Maya 安装，或者判断你已经安装了该软件的最新版本，它会显示一条警告，提示出现了异常情况。

现在你可以在这里决定要安装哪些软件和软件包。例如，如果你不希望安装 Katana 插件，可以取消选中它们。如果你 确实 希望为浮动许可证安装许可证服务器，可以选择 "Show All" 按钮，并同时选择许可证服务器软件包。 安装许可证服务器 会详细说明安装完成后必须在各个平台上执行哪些步骤，以确保浮动许可证服务器正常工作。

选定所需的软件包后，点击 Next（下一步） 按钮即可进入下载和安装阶段。

此阶段完成后，会出现 Success!（成功！） 屏幕。此时，你可以退出安装器。

安装器完成后，浮动许可证服务 尚未安装 或 尚未启动。

macOS 许可证服务器 会介绍安装和启动浮动许可证服务的所有细节。


下载

除了 RenderMan、许可证服务器和 DCC 插件之外，安装器也可以下载其他类型的文件。其中包括 RenderMan 示例。这些文件会放在运行 RenderManInstaller 的用户的下载目录中。

由于 macOS 的新安全功能，我们不再为应用程序提供 macOS pkg 文件的直接下载。在该平台上，下载只能通过 RenderMan 安装器完成。

Python

如果你计划使用 Python 绑定，需要从 www.python.org下载 Python 安装器。RenderMan Pro Server 27 绑定兼容 3.7、3.9、3.10 和 3.11。
