Installing for Large Sites

Source: https://rmanwiki-27.pixar.com/space/REN27/542212671/Installing+for+Large+Sites
Downloaded: 2026-06-15T03:49:12+00:00

集成在单个 GUI 程序中的 RenderMan Installer（RenderMan 安装程序），并不适合在中大型数量的机器上安装 RenderMan。RenderMan Installer 会下载并安装各个组件安装器，但这些单独的组件安装器也可以单独从 论坛网站 获取（需要论坛登录）。

获取许可证文件

如果不使用 RenderMan Installer，有两种获取许可证文件的方法：

  1. 运行 LicenseApp 工具并点击 "Get License"（获取许可证）按钮。它会提示你输入论坛用户名、密码和序列号，并下载许可证文件。 

  2. 前往 论坛网页 并输入你的论坛用户名、密码、序列号、hostname（主机名）和 hostid。hostid 可以通过使用 LicenseApp 工具确定，也可以通过使用特殊标志运行许可证服务器来确定：

    ./PixarLicenseServer -hostid

如果使用节点锁定许可证，则每台机器都需要单独的许可证文件。如果使用浮动许可证，只需要一个文件，可按需复制。

许可证文件位置

许可证文件名为 pixar.license，通常放在发行版目录旁边。例如，如果在每个操作系统上使用默认安装位置，许可证文件应为：

  • Linux

    /opt/pixar/pixar.license

  • Windows

    C:\Program Files\Pixar\pixar.license

  • macOS

    /Applications/Pixar/pixar.license

浮动许可证注意事项

请参阅 安装许可证服务器 以获取有关如何设置浮动许可证服务，以及如何启动、停止和配置这些服务的更详细信息。

在浮动许可证安装中，网络中需要有一台机器安装并运行 PixarLicenseServer。每台需要使用许可证的工作站都必须持续保持到该机器的网络连接。可以通过几种方式将每台工作站指向许可证服务器机器。请选择最适合你环境的方法：

  1. 每台工作站都可以像上文所述，在发行版旁边的默认位置简单地放置一份 pixar.license 文件。该许可证文件包含足够的信息，使软件（prman 或 RenderMan for Maya 等）能够确定这是网络许可证，以及许可证服务器的地址。如果不希望将 pixar.license 文件安装在默认位置，可以将环境变量 PIXAR_LICENSE_FILE 设置为许可证文件的完整路径名。

    PIXAR_LICENSE_FILE=/this/is/a/nonstandard/pixar.license

  2. 可以设置工作站，使环境变量 PIXAR_LICENSE_FILE 包含端口号和主机名值，从而无需在客户端工作站上放置任何文件。

    PIXAR_LICENSE_FILE=9010@license-server 

  3.  可以将 rendermn.ini 文件中的 /licenseserver 值设置为该位置

    /licenseserver 9010@license-server

如果没有对应的 DNS 服务器条目，许可证服务器地址可能需要采用 hostname.company.com 这样的形式，或者类似 10.0.0.2 的地址，甚至是 localhost（本机）。

LicenseApp 工具了解所有这些查找许可证服务器的方法，因此它适合从任意工作站测试和监控许可证状态。它也有助于诊断诸如 PIXAR_LICENSE_FILE 已设置但指向不可读文件，或无法通过网络访问许可证服务器等情况。
