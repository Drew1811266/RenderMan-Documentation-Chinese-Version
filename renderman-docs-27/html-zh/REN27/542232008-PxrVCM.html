PxrVCM

Source: https://rmanwiki-27.pixar.com/space/REN27/542232008/PxrVCM
Downloaded: 2026-06-15T03:49:12+00:00

使用 PxrVCM 积分器渲染出的菲涅耳透镜，并为带纹理区域光将 Trace Light Paths 设为 on。

此积分器目前在 RenderMan XPU 中不可用。

PxrVCM 是 RenderMan 中的核心最终质量积分器。它将双向路径追踪与渐进式光子映射（也称为顶点合并）结合起来。这些技术各自都能比纯正向路径追踪算法更高效地捕获某一范围的光传输路径：

  • 对于室内场景，或具有显著间接照明的场景，双向路径追踪可能比正向路径追踪收敛更快。尤其是当正向路径追踪不容易找到通往主导光源的间接路径时，双向路径追踪会很有帮助，特别是在主导光源距离照亮场景只差“一次反弹”时。它还可能整体加快间接照明，因为双向路径追踪可以在光路径与视线路径之间建立多次连接，并复用着色器评估。（正向路径追踪器必须在每次反弹处运行着色器，因此无法复用着色器评估。）请注意，如果图案评估很昂贵，这种收益会更明显；没有图案输入的 BxDF 不会从中受益。

  • 对于镜面-漫反射-镜面照明（焦散的反射或折射），渐进式光子映射收敛更快，尤其是在处理相对较小的光源时。

这些采样技术会与直接照明（将视线顶点直接连接到光源）一起，通过多重重要性采样进行组合；这种组合方式使综合算法在大量镜头中都比单独使用任一技术更稳健，并且收敛更快。

PxrVCM 积分器在默认运行模式下会启用所有这些技术；但在需要时，也可以通过 connectPaths 和 mergePaths 参数禁用连接和/或合并，使其作为纯双向路径追踪器、纯单向路径追踪器运行，甚至只使用纯渐进式光子映射。


上方图像展示了一个由壁灯中小型光源照亮的盒子。由于壁灯遮挡了大量直接光，此场景中的大部分照明都是间接照明。等时长渲染对比：左图只使用正向路径追踪（mergePaths=0，connectPaths=0），每像素 2048 次采样。右图使用双向路径追踪（connectPaths=1，mergePaths=0），每像素 600 次采样。


左侧骷髅图像只使用正向路径追踪（connectPaths=0，mergePaths=0）渲染，每像素 512 次采样。由于高度镜面的金属骷髅带来了难以积分的照明，焦散噪声很强，并且存在大量 fireflies（亮点噪声）。右侧图像使用完整的顶点连接与合并算法（connectPaths=1，mergePaths=1）渲染，同样为每像素 512 次采样。fireflies 几乎不存在，焦散也被解析得好得多。



上方两张图展示了 PxrVCM 渲染出的一些有趣焦散。左图中，一个玻璃透镜被哑光平面切开，右侧还有一个垂直的哑光平面。来自小型光源的光被透镜折射，在哑光表面上形成拉长的焦散。右图中，透镜被替换为玻璃球。

对于直接照明，PxrVCM 会查看 numLightSamples 和 numBxdfSamples 参数值，以确定灯光与 Bxdf 直接照明采样数量。为了获得最低噪声，numLightSamples 和 numBxdfSamples 应设置为相同值。更深光线深度和/或更低吞吐量下的直接照明采样削减默认启用，所用方案可通过 reduceDirectSamples 参数控制。

PxrVCM 会强制在体积中使用多重散射；尤其是双向路径追踪非常适合体积中的多重散射。不过，PxrVCM 目前不支持体积光子；因此，体积焦散等效果可能收敛较慢。

需要注意，此积分器与 PxrPathTracer 之间有几项重要差异：

  • PxrVCM 不显式支持像 PxrPathTracer 那样全局关闭焦散光路的能力。不过，默认情况下，场景的渲染结果可能与使用 PxrPathTracer 时相同。这是因为灯光会使用 Trace Light Paths 参数控制这些光线的追踪，而该参数默认关闭。你应该有选择地开启此功能，以生成视觉上所需的效果，而不是全部开启；全部开启可能成本较高，也更难进行艺术指导。（LPE 可用于在 PxrVCM 中移除焦散路径，但计算仍然会执行。） 

  • 与 PxrPathTracer 相比，PxrVCM 对 numLightSamples 与 numBxdfSamples 取不同值的容忍度更低。这两个参数（几乎）应始终设置为相同值，否则可能出现不必要的噪声。

  • 由于双向路径追踪的性质，PxrVCM 不暴露任何控制间接光线采样数量的选项；所有顶点最多只生成一条间接光线。

  • 渲染裁剪窗口得到的质量可能不同于完整帧渲染。这是因为光路径按像素生成，并存储在光子图中。渲染较小区域时，保存的光子会相应减少。在理论上可以生成全部光路径，但这样对性能的影响会开始削弱渲染最终图像小区域所带来的收益。

Parameters（参数）

参数

说明

connectPaths

控制是否使用“顶点连接”。双向路径追踪相对于正向路径追踪的优势，在于它能够通过在视线路径与光路径之间反复建立连接，沿完整的组合路径重复复用间接贡献，从而让具有显著间接照明的室内场景更快收敛。将此参数设为 0（关闭顶点连接）或 1（开启顶点连接）。默认情况下，顶点连接已启用。

没有顶点合并、但启用路径连接时，VCM 只作为双向路径追踪器运行。 

mergePaths

控制是否使用“顶点合并”。顶点合并会启用渐进式光子映射的一种变体，它在镜面-漫反射-镜面（焦散）照明中比路径追踪收敛更快。通常建议启用顶点合并，但这会为光子带来额外的时间和内存开销；在镜面-漫反射-镜面传输很少的场景中，它可能并非必要。将此参数设为 0（关闭顶点合并）或 1（开启顶点合并）。默认情况下，顶点合并已启用。请注意，光子生成要求 Hider 的 incremental 标志处于开启状态；当前迭代中生成的光子会用作下一次迭代的光子图。

启用顶点合并并禁用顶点连接时，VCM 会实现双向渐进式光子映射的一种变体；通常不推荐此模式，因为它不会在视线顶点与光源之间建立直接连接，因此直接照明通常比其他所有模式收敛更慢。

同时禁用顶点合并与顶点连接时，VCM 只作为正向的单向路径追踪器运行（不会生成光路径）。

mergeRadius

指定顶点合并中使用的半径。它以屏幕空间像素为单位测量（不是世界空间）。增大此半径会把光子的影响模糊到更宽区域，这可能有助于降低焦散噪声。不过，增大半径也会减慢合并速度，并增加渐进式光子映射的初始偏差，需要更多迭代才能得到无偏结果。默认值 5.0 会以一定初始偏差为代价显著降低噪声，而该偏差会随采样增加而降低。使用较低采样数时，如果这些偏差伪影不可接受，降低 mergeRadius 可以减少这些伪影，但代价是噪声增加。

timeRadius

指定光子可被合并时允许的最大时间差，以快门区间的比例来测量。默认情况下，timeRadius 为 1.0：视线顶点可以与任意时刻发射的光子合并。这可能导致运动物体产生的焦散结果不准确（即在时间上被拖抹）。将 timeRadius 设为较小值，会在时间维度上收紧可合并光子的搜索半径，从而为运动模糊焦散带来更好结果；代价是静态对象需要更多光子才能解析焦散。

maxIndirectBounces

控制组合后的光路径与视线路径的最大长度，包括连接光线。例如，值为 4 时允许最多 4 次全局照明反弹。此参数值为 0 时只允许直接照明，这意味着不会生成光路径（因为视线路径顶点始终会直接连接到光源）。此参数的默认值为 10。

numLightSamples

控制视线路径上每个顶点用于直接照明的灯光采样数量。默认值为 1。

numBxdfSamples

控制视线路径上每个顶点用于直接照明的 Bxdf 采样数量。默认值为 1。

rouletteDepth

控制积分器从哪个路径长度开始执行俄罗斯轮盘，以降低整体路径长度。默认值为 4。请注意，俄罗斯轮盘会分别应用于视线路径和光路径。

rouletteThreshold

控制路径吞吐量阈值；低于该阈值时开始执行俄罗斯轮盘。默认值为 0.2。提高阈值会缩短路径长度，从而整体提升速度，但代价是噪声更高。

clampDepth

如果指定了 clampLuminance 参数值，则 clampDepth 控制从哪个光线深度开始基于逐光线亮度进行钳制。例如，将此参数设为 2，并同时为 clampLuminance 指定值 4，将确保在所有间接照明中，每条光线贡献的亮度不超过 4，同时不会影响或钳制直接照明。默认值为 2。

clampLuminance

默认情况下，PxrVCM 积分器会把任何技术在任何顶点上计算出的亮度钳制到最多 10.0。不过，可以通过为 clampLuminance 参数指定不同的值来改变此行为。为 clampLuminance 参数指定相对较低的值（例如 2 到 20 之间）可以大幅加快收敛。在某些情况下，间接照明光路可能会因为钳制而明显变暗；这在特定情况下可能是可以接受的折衷。将此参数设为非常大的数字（例如 1e30）会有效禁用所有钳制。默认值为 10.0。

photonGuiding

设置光子发射期间使用光子引导的概率。值为 0.0 表示不使用光子引导。0.0 到 1.0 之间的值会组合光子引导与标准光子发射。最后，值为 1.0 表示所有光子都被引导，这可能生成有偏图像。默认值为 0.0。

photonGuidingBBoxMin

photonGuidingBBoxMax

这两个值可用于显式指定光子应朝向其发射的包围盒（以世界空间坐标表示）。如果未指定此包围盒，系统会自动计算一个稍微宽松的包围盒，覆盖场景中直接可见的部分。photonGuidingBBoxMin 的默认值为 (1e30, 1e30, 1e30)，photonGuidingBBoxMax 的默认值为 (-1e30, -1e30, -1e30)。
