Installing on Linux

Source: https://rmanwiki-27.pixar.com/space/REN27/542212252/Installing+on+Linux
Downloaded: 2026-06-15T03:49:12+00:00

最低系统要求

此版本支持以下操作系统。RenderMan 也许可以在不满足这些要求的环境中运行，但可能不稳定，且不属于支持范围：

  • 兼容任何支持 gcc 11.2 和 glibc 2.34 或更高版本的 Linux 发行版（例如 RHEL9.x 及更新版本）

RenderMan 的设计目标是在大量 CPU 核心之间高效扩展。因此，尽管以上只是最低要求，配备多个处理器的系统会获得最佳性能。

Pixar 的 RenderMan 软件以下列形式交付： 

  • RenderMan-Installer-version-linuxflavor.rpm - RenderMan 渲染工具包。

该  flavor  字符串包含常见发行版，例如 RHEL9，以及  gcc/icc  版本和微处理器指令集。把这些部分组合起来，例如会得到一个名为 RenderMan-Installer-27.0.0_2376203 的 RHEL 软件包-linuxRHEL9_gcc11icx232.x86_64.rpm 。

请注意，在 Linux 上，用户必须手动设置环境变量，才能从终端窗口使用可执行文件。此过程会在下方的命令行安装部分说明。

你还必须满足顶层 安装和授权 页面列出的要求。

基于 GUI 的安装

尽管 RPM 软件包通常通过命令行安装，但某些 Linux 发行版提供基于 GUI 的软件包管理器，例如  gnomerpm 。某些发行版还允许在目录浏览器中双击 RPM 文件，这会使用默认参数调用 RPM。你可以在自己的 Linux 系统上尝试这样做。如果安装没有自行开始（因为系统缺少双击 .rpm 所需的组件），则可以按下面的步骤开始安装过程。

下面的流程适用于 27.0 正式版，而不适用于 Beta。27.0 Beta 不包含 GUI 安装器。作为替代，你可以分别下载每个 RPM，并使用类似下面的命令安装（这里只展示 RenderManProServer 的命令，其他组件也可以用类似方式安装）。如果你的系统权限需要，请在命令前加上 sudo ：

rpm -ivh RenderManProServer-27.0b1_2376203-linuxRHEL9_gcc11icx232.x86_64.rpm

要使用 GUI 安装器安装 RenderMan（仅限 27.0 正式版），请运行：

rpm -ivh RenderMan-Installer-27.0_2376203-linuxRHEL9_gcc11icx232.x86_64.rpm

由于 EL9 中的用户安全机制发生变化，如果 RenderMan 安装器应用在 sudo 下无法启动，可能需要先运行以下命令行：

$ xhost local:root

RenderMan 的 RtxSVG.so 插件要求安装 compat-expat1 软件包。如果你计划使用 SVG 纹理，请先安装 compat-expat1。

如果你已经解包 .rpm，则可以运行以下命令来启动基于 GUI 的安装器：

sudo /opt/pixar/RenderMan-Installer/bin/RenderManInstaller

安装器会连接 RenderMan Pixar 服务器，并下载你已授权的组件。在安装过程中，你会看到默认安装位置是 /opt/pixar；它将成为上面列出的所有软件组件的根目录。

接下来我们将逐步说明这个过程。

首先，你必须使用论坛账户登录，并提供已购买许可证的序列号。

接下来，如果你是在浮动许可证服务器上 首次 安装，可能会看到以下内容：

这表示安装器即将把你的许可证服务器绑定到当前运行安装器的主机。

如果这台机器  不是  你希望运行许可证服务器的机器，请按  Cancel（取消） 按钮，并在你希望指定为许可证服务器的机器上重新运行安装器。

如果你决定将这台机器作为许可证服务器，或者你正在安装节点锁定许可证，安装器随后会联系 Pixar 并获取你的许可证文件。默认情况下，它会把许可证文件放在：

  • Windows: C:\Program Files\Pixar\pixar.license

  • macOS: /Applications/Pixar/pixar.license

  • Linux: /opt/pixar/pixar.license



接下来，你会看到希望安装的软件包列表。 

安装器还会尝试查找可安装的相应插件。不过，如果它找不到 Maya 安装，或者判断你已经安装了该软件的最新版本，它会显示一条警告，提示出现了异常情况。

现在你可以在这里决定要安装哪些软件和软件包。例如，如果你不希望安装 Katana 插件，可以取消选中它们。如果你 确实 希望为浮动许可证安装许可证服务器，可以选择 “Show All” 按钮，并同时选择许可证服务器软件包。  安装许可证服务器 会详细说明安装完成后必须在各个平台上执行哪些步骤，以确保浮动许可证服务器正常工作。

选定所需的软件包后，点击 Next（下一步） 按钮即可进入下载和安装阶段。

此阶段完成后，会出现 Success!（成功！） 屏幕。此时，你可以退出安装器。

安装器完成后，浮动许可证服务 尚未安装 或 尚未启动。

Linux 许可证服务器 会介绍安装和启动浮动许可证服务的所有细节。

下载

除了 RenderMan、许可证服务器和 DCC 插件之外，安装器也可以下载其他类型的文件。其中包括 RenderMan 示例。这些文件会放在运行 RenderManInstaller 的用户的下载目录中。

Python

如果你计划使用 Python 绑定，需要从 www.python.org下载 Python 安装器。RenderMan Pro Server 27 绑定兼容 3.7、 3.9、3.10 和 3.11。

帮助

如需进一步帮助，请查看 RenderMan 支持论坛。请注意，需要注册。
