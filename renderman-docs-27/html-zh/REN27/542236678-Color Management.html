Color Management

Source: https://rmanwiki-27.pixar.com/space/REN27/542236678/Color+Management
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan 在所有 bridge 产品中都对行业标准 ACES 色彩管理系统以及其他颜色空间提供稳健支持，例如流行的 Filmic Blender，也包括交互式 RenderMan Image Tool。

OCIO 配置

RenderMan 随附多个 OCIO 配置：

LINEAR-sRGB
最初的 Sony Pictures Imageworks 配置已经使用了很长时间。

ACES-1.2
用于电影制作和计算机图形的视觉特效标准 Academy Colour Encoding System（学院色彩编码系统）。

FILMIC BLENDER
一种流行的基于 BT.709 的配置，可提供 high dynamic range（高动态范围）以及胶片底片中常见的其他效果。由 Troy Sobotka 开发。

配置选择

  • $OCIO：使用系统环境变量定义的颜色空间。通常为 sRGB。

  • Off：Scene Linear 被定义为整个场景的颜色空间。

  • ACES1.2：使用随 RenderMan 24 提供的 ACES 1.2 色彩编码系统。

  • Filmic Blender：使用流行的 BT.709 颜色空间。

  • Custom：允许指定自定义颜色空间文件。

请注意，在 OCIO 配置之间切换时，需要关闭并重新打开 IT，新的颜色空间才能在 IT 中正确显示。

受色彩管理的项目

  • UI 色彩值

    • 所有 DCC 应用都会提供受色彩管理的拾色器，并返回 rendering colorspace（渲染颜色空间）中的颜色。RenderMan 只是按原样使用这些颜色。

  • 灯光

    • Color Temperature 在 24.0 之前按 Rec.709 计算，现在则受色彩管理。

    • PxrEnvDayLight 也受色彩管理，并依赖 srgb_linear role（角色）。

  • 纹理

    • 纹理必须先转换到 rendering color space（渲染颜色空间） 后才能使用。这项任务可由 Texture Manager 完成。

    • 也可以使用 PxrColorSpace pattern 将纹理颜色从一个颜色空间转换到另一个颜色空间，但会带来性能成本。

  • Patterns（图案）

    • PxrBlackBody 受色彩管理。

    • PxrHairColor 受色彩管理。

    • PxrColorSpace 允许美术人员从一个颜色空间转换到 rendering space（渲染空间） 。
