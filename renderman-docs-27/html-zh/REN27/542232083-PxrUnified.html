PxrUnified

Downloaded: 2026-06-15T03:49:12+00:00

图片来自 Incredibles 2

PxrUnified 最初作为实验性积分器包含在 RenderMan 中。最终，此积分器会成为功能完整、可用于可靠生产渲染的积分器。此积分器在 RenderMan XPU 中不可用。

请注意，PxrUnified 目前 不支持

  • Holdout workflow（遮挡保留工作流）

  • Bidirectional mode（双向模式）中的 LPE 输出

  • Shadow Collectors（阴影收集器）

  • Exclude Shadow Subsets（排除阴影子集）


PxrUnified 是 Pixar 自家电影和动画短片所依赖的积分器。它既实现正向路径追踪算法，也实现双向选项，并使用 Manifold Next Event Estimation（流形下一事件估计）技术来更快处理焦散，在这里该技术称为 Manifold Walk（流形行走）。它还提供两种 Indirect Guiding（间接引导） 选项，通过从场景中照明更充分或更重要的区域进行采样来改善间接照明。其中一种间接引导方法基于 Müller 等人的 Practical Path Guiding for Efficient Light-Transport Simulation（用于高效光传输模拟的实用路径引导）。这种方法开销相对较高，但对某些场景可能有益，尤其是具有高度镜面间接交互的场景。第二种方法称为 risPathGuiding，开销低得多，对漫反射和光泽场景更有效。请参见 Path Guiding（路径引导）。

请注意，Max Path Length 是用于控制全局 trace depth（追踪深度）的单一控件；PxrUnified 会忽略单独的镜面和漫反射 trace depth 设置。这意味着更简单的控件更直观，但某些场景可能需要更长渲染时间，并且外观可能不同于用户已根据场景优化反弹次数的其他积分器。将此控件设为 0 表示只启用直接照明和摄影机可见效果。提高该设置会允许光在场景对象中传播得更深，并从对象上继续反弹。

对于直接照明，numLightSamples 和 numBxdfSamples 参数决定 light（灯光）与 Bxdf（材质）直接照明采样的数量，生成的直接照明贡献会使用多重重要性采样进行组合。

对于间接照明，numIndirectSamples 参数控制是否发射光线来采样间接路径。目前只支持 0 和 1 两个值。

SubSurface Over-sampling（次表面过采样）用于在渲染复杂照明以及启用次表面着色的对象时帮助降低噪声。

有关这些高级特性的更多技术信息，可以参考以下研究论文：

  • 论文作者：Thomas Müller、Markus Gross 和 Jan Novák。  Practical Path Guiding for Efficient Light-Transport Simulation（用于高效光传输模拟的实用路径引导） 。《Computer Graphics Forum》（EGSR 2017 论文集），第 36 卷，第 4 期

  • 论文作者：Johannes Hanika、Marc Droske 和 Luca Fascione。 Manifold Next Event Estimation（流形下一事件估计）。 Eurographics Symposium on Rendering 2015，第 34 卷，第 4 期





Basic Parameters（基本参数）

Parameter

说明

traceLightPaths

此参数启用双向路径追踪模式。PxrUnified 具有 4 种积分模式，具体取决于 light path（光路径）的采样方式。还有更多选项可用，并且某些选项组合会因选择不同而无效。其选择方式类似下面提到的 PxrVCM。随着此积分器最终定型，相关设置会补充更多细节。

  • Off（关闭）

    • 单向路径追踪，类似 PxrPathTracer

  • On（开启）

    • 双向路径追踪，类似 PxrVCM 中 connect on、merge off 的模式。Indirect Guiding（间接引导）会被禁用。

  • Photons Only（仅光子）

    • 渐进式光子映射，类似 PxrVCM 中 connect off、merge on 的模式。Indirect Guiding（间接引导）会被禁用。随后会显示 Photon Controls（光子控制）。

  • UPS/VCM

    • 双向路径追踪结合渐进式光子映射，类似 PxrVCM 中 connect on、merge on 的模式。

maxIndirectBounces

控制最大光线深度的绝对上限。例如，此参数值为 0 时只允许直接照明；值为 4 时允许最多 4 次全局照明反弹。默认值为 8。

useTraceDepth

启用时，积分器会遵循场景中分配给对象的 trace depth（追踪深度）；这通常用作渲染优化，将各类反弹数量限制到视觉上必要的程度。请参见 Trace Depth。

maxRayDistance

此参数控制一条以 scene units（场景单位）计的 间接 光线在被终止前可传播的距离。如果你的场景尺度使得相距很远的对象在视觉上不会发生间接交互，可以使用此参数优化场景。请注意，穿过窗户和其他对象的光线也是 indirect rays（间接光线），可能会过早终止并从玻璃折射中消失。（设想一个从客厅窗户看远处山脉的渲染，请确保这些光线能传播得足够远。）

catchAllLights

在最后一次命中处，对灯光进行无阴影评估；默认值为 off（0.0）。如果发现折射变暗，将此滑块设为 1.0 会加入灯光采样的完整贡献。开启此项可能会让场景变亮。

emissionMultiplier

乘以材质和灯光上设置的所有 emissive（自发光）和 "glow"（辉光）属性。

accumOpacity

控制路径追踪器是否沿路径跟踪累积不透明度。默认值为 off。如果计划渲染折射对象，并且关心用于合成的 alpha 值，可能需要开启此项，例如汽车车窗。

specularCurvatureFilter

Filter（过滤）镜面交互的粗糙度；设为 0.0 表示不进行过滤，即 "ground truth"（基准真值）。默认值旨在降低噪声。

numLightSamples 

控制每个摄影机命中点用于直接照明的灯光采样数量。默认值为 1。较小数值可能提升交互式和渐进式更新的性能。较大数值可能看起来收敛更好，但更新会更慢。

numBxdfSamples

控制每个摄影机命中点用于直接照明的 Bxdf 采样数量。默认值为 1。较小数值可能提升交互式和渐进式更新的性能。较大数值可能看起来收敛更好，但更新会更慢。

numIndirectSamples

作为开关使用：off（0，或无间接照明）与 on（1）。

sssOversampling

控制每个摄影机命中点要生成的 subsurface rays（次表面光线）数量。默认值为 8。

allowMultilobeIndirect

允许在毛发渲染期间复用采样，以加快收敛。这可能会改变生成的 LPE。

rouletteDepth

控制从哪个光线深度开始执行 Russian Roulette（俄罗斯轮盘）。在此深度之后，Russian Roulette 可能会终止光线，尤其是当它们对结果贡献很小时。默认值为 1。

rouletteThreshold

控制 path throughput（路径吞吐量）阈值；低于该阈值时会执行 Russian Roulette（俄罗斯轮盘）。它控制光线贡献在何种程度上小到可以终止该光线。默认值为 0.2。

directClamp

设置数值被钳制的幅度。对于直接照明，默认值较高，以保留场景照明的动态范围。降低该值会减少 fireflies（亮点噪声）等噪声，但代价是动态范围降低。默认值为 1000000.0。

indirectClamp

设置数值被钳制的幅度。对于间接照明，默认值较低，这可以避免已经吸收部分能量的次级效果产生噪声。降低该值会减少噪声，但也会使间接效果变暗。默认值为 8.0。

causticClamp

设置数值被钳制的幅度。对于焦散，默认值较高，以保留光线从对象表面反射或穿过对象聚焦后产生的光图案。降低该值会减少 fireflies（亮点噪声）等噪声，但代价是动态范围降低。默认值为 1000000.0。


Manifold Walk（流形行走）（当前不适用于 BDPT 模式）

Parameter

说明

manifoldWalk

启用或禁用 manifold walk（流形行走）计算。请注意，场景还需要额外设置；这些协同工作的设置可在 Lights、PxrSurface 和积分器中找到。

maxIterations

这是 walk（行走）在放弃并终止前可执行的最大迭代次数。

maxInterfaces

这是 walk（行走）在通往灯光途中预计会遇到的最大透射界面数量。数值过低可能会阻止它离开界面链并找到光源。

walkThreshold

这是用于控制 manifold walk（流形行走）收敛的阈值。较高值可能产生噪声。

enableVolumeCaustics

此参数允许在介质中（如果存在介质）产生体积焦散。包围该介质的对象必须属于特定 trace set（追踪集合）：mw_volume_include_subset。这是一项昂贵技术。

RIS Indirect Guiding（间接引导）/ RIS 间接引导

Parameter

说明

risPathGuiding

设为 1 时，会启用间接路径引导。在渲染进行期间，此技术会学习场景中的间接 radiance（辐射亮度）分布，并使用多重重要性采样将这些信息与 Bxdf 采样结合，以缩短收敛时间。它对光泽和漫反射表面尤其有效，也适用于间接光占主导的区域。Path Guiding（路径引导）






PPG Indirect Guiding（间接引导）/ PPG 间接引导

Parameter

说明

indirectTrainingSamples

这是用于“学习”场景光照分布的采样数量。此数值不应超过 hider 设置的 minSample，也不应超过 maxSamples 的 10% 到 20%。64 应作为最小值（这意味着 minSamples 至少应为 64；如果设为 0，则为 maxSamples 的平方根，可能过低）。

indirectSpatialBlurRadius

此参数会模糊 spatial tree（空间树）的边界；它只影响采样，不是对最终采样结果进行模糊的参数。如果你看到由网格结构导致的不连续，可以使用此参数。

indirectDirectionalBlurRadius

此参数会模糊 spatial tree（空间树）中给出的方向；它只影响采样，不是对最终采样结果进行模糊的参数。如果你看到由网格结构导致的不连续，可以使用此参数。

indirectOversampling

Over sample（过采样）lookup tree（查询树），以获得更好的结果。通常这只是非常细微的效果。


Photons（光子）（Enabled：当 Trace Light Paths 设为 Photons 或 UPS/VCM 模式时启用）

Parameter

说明

photonEstimationRadius

用于光子查找优化的搜索半径，以 scene units（场景单位）计。通常默认值即可，除非你想在收敛过程中获得不同的预览效果。

photonEstimationNumber

在上述搜索半径内用于估计的 photons（光子）数量。

photonVisibilityRod

用于 photons（光子）的 targeting shape（定向形状），可开启或关闭。

photonVisibilityRodDirectProb

控制 photons（光子）向下方指定 Rod box 的定向程度。接近 1.0 的值会开始使图像产生偏差。

photonVisibilityRodDirectMin

光子定向区域的最小边界。

photonVisibilityRodDirectMax

光子定向区域的最大边界。

photonAdaptive

基于论文： "Adaptive Progressive Photon Mapping"（“自适应渐进式光子映射”）



Diagnostic（诊断）

参数

说明

suppressNaNs

此参数会在单个采样被 splat（泼溅写入）到图像之前，让返回 "not a number"（非数字）的采样失效。执行此检查会有开销，并且默认关闭。

enableShadingTimers

（developer option，开发者选项）启用计时器，用于按 profileGroup user attribute（用户属性）对 shader 组的着色成本进行收集。

enableSampleTimers

（developer option，开发者选项）启用用于跟踪像素采样迭代成本的计时器。
