PxrDisplace

Source: https://rmanwiki-27.pixar.com/space/REN27/542221096/PxrDisplace
Downloaded: 2026-06-15T03:49:12+00:00

这是一个置换插件，会在当前空间（通常是对象空间）中置换表面点 P。它同时支持标量和矢量输入。在执行实际置换之前，它会先把所有已连接的标量和矢量输入相加。 

所有 与置换变换相关的 函数都位于 PxrDispTransform pattern 中。 

所有 置换分层函数 都位于 PxrDispScalarLayer 和 PxrDispVectorLayer pattern 中。 

这种模块化设计没有把所有函数都放进一个大型置换插件，而是将变换与分层分离开来，为支持不同的生产级置换工作流提供更多灵活性和效率。

需要设置置换边界才能看到置换。出于性能原因，默认并不总是设置它。先使用较小的值；如果发生裁切，再缓慢增大该值。

Displacement Space（置换空间）

我们提供两种用于指定置换的渲染空间：世界空间和对象空间。这些设置在 PxrDispTransform 节点中指定，但在这里解释。

默认情况下，RenderMan 会在对象空间中执行置换。这通常适合雕刻模型。但对某些对象来说，这可能导致不连续。例如，一组一起置换的对象，比如由一组平面组成的环境部件。在对象空间中，它们可能会像下图那样在面片之间产生缝隙或不连续。

要修正这个特定场景，可以选择世界空间，使用相同的八个面片获得下方结果。这会解决海面平面、悬崖和岩石等问题，在这些情况下置换会跨多个形状共享。



Displacement Types（置换类型）

置换有两种类型：标量和矢量。两类置换的来源都可以是程序化的，也可以来自纹理。

  • 标量置换是一种浮点置换。它只有一个 float 值。因此，它会让 P 向上或向下置换。

  • 矢量置换会在 XYZ 三个维度上置换 P。

下方示例展示了使用同一组置换值时，标量置换（使用 float，例如 resultF、resultR 等）与矢量置换（使用 vector，例如 resultRGB、resultXYZ）会产生不同结果。

Input Parameters（输入参数）

Enabled

如果勾选，则启用置换。这对调试很有用，因为有时需要关闭置换。

Gain

置换的 Gain。如果为零，则不会产生置换。较高数值会产生更大的置换。注意，如果它超过设置的 Displacement Bounds，可能会造成问题，并会在渲染时打印警告。

Scalar Displacement

标量置换输入。这是一个浮点置换。可以将它连接到 PxrTexture pattern 节点或任何 float pattern 节点。 

Vector Displacement

矢量置换输入。矢量置换数据是一个三通道数据文件，即 XYZ。可以将它连接到 PxrTexture pattern 节点，或任何输出 vector 或 color 的 pattern 节点。

Model Displacement

Model Displacement 是模型的矢量置换。它会加到上面的 Vector Displacement 上。

建模师使用 Mudbox 或 ZBrush 雕刻模型并不少见。随后着色艺术家会通过程序化创建矢量置换来做外观开发，以添加更多细节。因此，同时提供 Model Displacement 和 Vector Displacement 可支持这种灵活工作流。


性能提示：

如果需要从渲染相机（位于最终帧的正确位置）获得更高质量，可以使用 Micropolygon Length 并缓慢降低该数值来减小多边形尺寸（减小最终多边形尺寸以获得更细的细节）。

始终使用能够产生正确且完整置换的最小包围盒设置。
