简介

细分曲面是在 RenderMan 中为复杂平滑几何体建模的最常见方式，而且理由充分：它们能保证模型的平滑性，同时对建模师施加的拓扑约束很少。

理解细分曲面的一种常见方式，是想象一个多边形网格（即 控制网格）在每个顶点处经过局部平均（或平滑），从而创建出一组外观比原始网格更平滑的新顶点。将这种平均算子应用到整个网格一次，就是一个 细分步骤。如果能够执行无限多个细分步骤，最终会收敛到一个 极限曲面，它会根据所用细分类型具备一组有保证的平滑性属性。 

与大多数其他几何类型一样，细分网格由它的控制网格描述。与 NURBS 不同，控制网格并不局限于矩形形状，因此在这一点上，控制网格与多边形曲面非常相似。但多边形曲面需要大量点才能显得平滑，而细分网格的极限曲面则保证是平滑的。这意味着无论表面如何动画、观察距离多近，都不会出现多边形伪影或分面。

与许多其他渲染器不同，RenderMan 的细分曲面实现不会对多边形网格应用固定次数的细分步骤。相反，它会自适应地镶嵌细分为微多边形，其大小 默认按屏幕上的像素计算。此外，即使由于微多边形长度设置导致微多边形较大，它们的顶点也会直接从极限曲面计算，以获得最高精度。

与多边形的比较

为资产建模时，一个常见问题是：多边形资产应该按原样渲染，还是转换为细分曲面？如果资产本质上并不平滑（也就是具有硬轮廓的简单几何体），那么多边形通常就足够了。如果资产目标是平滑表面，取舍就会更复杂：

  • 表示一个平滑表面可能需要的多边形数量，可能明显高于等价的细分网格，并且需要更多内存。在这种情况下，细分网格显然是更好的表示。

  • 不过，如果渲染器不需要执行任何细分，多边形通常会便宜得多。为了能够执行细分，渲染器需要在细分曲面上保留额外内存。如果资产一开始就建模过密，极限曲面可能与控制网格差别不大，为执行细分所需的额外内存就浪费了。在这种情况下，多边形表示可能已经足够（也就是可以认为它“足够平滑”）。请注意，渲染器会尽力缓解细分网格建模过密的情况：它会在前置的“预镶嵌细分”步骤中，将底层表示转换为本质上类似多边形的表示，并通过丢弃细分数据来节省内存。

  • 细分网格能够执行水密切分，以闭合置换可能造成的孔洞或裂缝。在 RenderMan 中，多边形无法做到这一点，因为它们没有保留足够的数据。

  • 细分网格有一些拓扑约束；特别是，面不能带孔，也不能表示非流形曲面（即一条边可能关联到两个以上面的曲面，可以想象两个立方体共享同一条边）。在实践中，这通常不是严重问题。

特性

细分方案

RenderMan 的细分网格实现包含以下细分方案：

  • Catmull-Clark 细分曲面 是行业标准。它是一种基于四边形的细分方案，最适合主要由四边形组成的控制网格。任何非四边形几何体都会在第一次细分步骤中立即转换为四边形。它通常适用于大多数情况，并且在纹理过滤方面没有特殊顾虑。

  • Loop 细分曲面 （XPU 不支持）是一种基于三角形的方案，针对完全由三角形组成的控制网格进行了优化：它们往往比 Catmull-Clark 方案需要更少内存。不过，由于三角形的性质，导数可能在面与面之间不连续，这可能导致纹理伪影，需要额外工作才能解决。

  • Bilinear 细分曲面 只会将高层级的四边面渲染为双线性面片，并将所有 n 边面细分为 n 个 blp。与其他 SubdivisionMesh 一样，它支持所有分层编辑，保证水密，并且可以与 PTex 配合使用。

锐利折痕和角点

并不是所有几何体在各处都平滑。需要保留锐利特征时，可以把边或顶点标记为 折痕 或 角点。这意味着在该特征附近不会应用实际的细分，生成的极限曲面会保持与控制网格一致。

半锐利折痕和角点

用多边形近似平滑表面时，通常会对锐利边应用圆角操作，把它转换为圆滑边。遗憾的是，圆角往往会创建更多多边形。把多边形网格转换为细分曲面，可以在一定程度上缓解这个问题：此时表面的平滑性得到保证；但是圆滑边的大小仍由该边相邻面的大小决定，可能仍然需要向模型中引入新的面来控制边缘曲线的尺寸。

为缓解这个问题，RenderMan 的细分曲面实现允许将边和顶点标记为 半锐利 折痕和角点。将一条边标记为半锐利折痕，意味着该边附近生成的极限曲面会处于两种结果之间：一种是它被标记为完全锐利折痕时的位置（即控制网格），另一种是它作为完全平滑表面通常会达到的位置。下面的示例显示了由立方体控制外壳生成的细分网格，并对其应用了不同的折痕强度。

水密切分

使用细分网格而不是多边形网格的一个好处是，执行细分所需的拓扑数据也有助于保证表面保持水密，无论置换中发生什么情况都是如此。特别是，如果共享边两侧紧邻的面计算出不同的置换结果，在多边形网格上，这些面彼此拉开时很可能会出现裂缝。虽然这看起来显然应该避免，但在实践中，尤其是使用来源于 PTex 或基于 UDIM 工作流的复杂纹理资产时，做到这一点并不容易。如果使用细分网格而不是多边形网格，并启用水密切分，渲染器就可以确保面边界处不会出现裂缝。

水密切分可能明显慢于常规切分，并会延长首个像素时间（TTFP）。

边界插值

为了提供尽可能平滑的极限曲面，细分曲面的平均行为默认会使极限曲面从控制网格的边界处“偏离”。处理非闭合网格时，这可能并不理想，尤其是这些网格需要与其他相邻网格无缝连接时。为方便起见，RenderMan 提供了 边界插值规则 来覆盖这种行为，使表面能够一直延伸到控制网格的边界。不同规则允许平滑或锐利的 角点 （即恰好关联两条边的顶点）。下面的示例显示了应用到一个简单 9 面控制网格上的边界插值规则。

边界插值的概念也可以扩展到处理表面数据时应用的插值规则，其中最典型的是纹理坐标（在此语境中，RenderMan 使用术语 facevarying 边界插值）。处理不连续 UV 区域之间的接缝时，渲染器可以使用几种不同规则，在远离接缝处的细分结果平滑性与精确插值接缝的需求之间进行取舍。请注意，无论选择哪种规则，UV 纹理贴图编辑器也必须遵守相同规则，否则会产生不良纹理伪影。下图展示了应用到一个 4x4 四边形网格上的不同规则，该网格被分成三个 UV 区域。

“interpolateboundary”标签控制插值边界面边的插值方式。该标签有一个可选整数参数和零个浮点参数。如果未指定整数参数，则假定其值为 1。值 0 表示不应发生任何边界插值行为（这是未指定该标签时的默认行为）。值 1 表示所有边界边链都是锐利折痕，并且恰好关联两条边的边界顶点是锐利角点。值 2 表示所有边界边链都是锐利折痕；边界顶点不受影响。

例外

“RenderMan”默认值为 0，不过桥接产品会将其覆盖为 1。这个 RenderMan 默认值未来可能会改变。

由于 Katana 遵循 RenderMan 默认值，它是唯一指定 0 的桥接产品。
