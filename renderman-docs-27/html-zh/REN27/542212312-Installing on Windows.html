Installing on Windows

Source: https://rmanwiki-27.pixar.com/space/REN27/542212312/Installing+on+Windows
Downloaded: 2026-06-15T03:49:12+00:00

最低系统要求

此版本支持以下操作系统。RenderMan 也许可以在不满足这些要求的环境中运行，但可能不稳定，且不属于支持范围：

  • Microsoft Windows 10 或 11 64 位

RenderMan 的设计目标是在大量 CPU 核心之间高效扩展。因此，尽管以上只是最低要求，配备多个处理器的系统会获得最佳性能。

Pixar 的 RenderMan 软件以下列形式交付： 

  • RenderMan-Installer-version-windowsflavor_vc143icx232.x86_64.msi - RenderMan 渲染工具包。

该 flavor 字符串包含常见发行版，例如 windows10，以及 vc/icc 版本和微处理器指令集，例如 i686。把这些部分组合起来，例如会得到一个名为 RenderMan-Installer-27.0_1234567-windows11_vc143icx232.x86_64.msi

还必须满足以下附加要求：

  • 所有系统：CPU 能够运行完整的 AVX 指令集。

基于 GUI 的安装

要安装软件，只需双击相应的 MSI 文件，并逐步确认各个对话框。默认值在大多数情况下都适用。


安装过程中，你首先可以选择根目录。默认情况下，此目录为 C:\ProgramFiles\Pixar\；它将成为上面列出的所有软件组件的根目录。安装器还会在下载目录中生成一个日志文件，其名称格式为： rmaninstaller.XXXX.txt。此文件可在之后出现安装问题时引用。

安装器会连接 RenderMan Pixar Server，并下载你已授权的组件。在安装过程中，你会看到默认安装位置是 /opt/pixar；它将成为上面列出的所有软件组件的根目录。


接下来我们将逐步说明这个过程。首先，你必须使用论坛账户登录，并提供已购买许可证的序列号。

接下来，如果你是在浮动许可证服务器上 首次 安装，可能会看到以下内容：

 

这表示安装器即将把你的许可证服务器绑定到当前运行安装器的主机。

如果这台机器 不是 你希望运行许可证服务器的机器，请按 Cancel 按钮，并在你希望指定为许可证服务器的机器上重新运行安装器。

如果你决定将这台机器作为许可证服务器，或者你正在安装节点锁定许可证，安装器随后会联系 Pixar 并获取你的许可证文件。默认情况下，它会把许可证文件放在：

  • Windows: C:\Program Files\Pixar\pixar.license

  • macOS: /Applications/Pixar/pixar.license

  • Linux: /opt/pixar/pixar.license

接下来，你会看到希望安装的软件包列表。 

安装器还会尝试查找可安装的相应插件。不过，如果它找不到 Maya 安装，或者判断你已经安装了该软件的最新版本，它会显示一条警告，提示出现了异常情况。

现在你可以在这里决定要安装哪些软件和软件包。例如，如果你不希望安装 Katana 插件，可以取消选中它们。如果你 确实 希望为浮动许可证安装许可证服务器，可以选择 "Show All" 按钮，并同时选择许可证服务器软件包。 安装许可证服务器 会详细说明安装完成后必须在各个平台上执行哪些步骤，以确保浮动许可证服务器正常工作。

选定所需的软件包后，点击 Next 按钮即可进入下载和安装阶段。

此阶段完成后，会出现 Success! 屏幕。此时，你可以退出安装器。

安装器完成后，浮动许可证服务应已启动。

Windows 许可证服务器 会介绍在浮动许可证服务未启动时，如何配置并启动这些服务的所有细节。


下载

除了 RenderMan、License Server 和 DCC 插件之外，安装器也可以下载其他类型的文件。其中包括 RenderMan Examples。这些文件会放在运行 RenderManInstaller 的用户的 Download 目录中。

Python

如果你计划使用 Python 绑定，需要从 www.python.org下载 Python 安装器。RenderMan Pro Server 27 绑定兼容 3.7、3.9、3.10 和 3.11。
