macOS License Server

如果你使用 RenderMan Installer 安装了 License Server，那么所需的 pixar.license 文件已经位于：

/Applications/Pixar/pixar.license

如果你是通过下载并运行软件包安装器进行安装，那么安装完成后，请运行 Pixar LicenseApp；它通常位于这里：

/Applications/Pixar/PixarLicense-LA-23.0./LicenseApp/bin/LicenseApp

这会帮助你下载并安装许可证服务器的许可证文件。当 LicenseApp 发现没有许可证文件时，界面会如下所示：

 

点击 "Get License" 按钮，为许可证服务器下载新的许可证文件。 它会把此许可证文件放在：

 

/Applicaitons/Pixar/pixar.license

如果你确实有许可证，但许可证服务器尚未启动，RenderMan 将报告许可证错误：

R50005 {SEVERE}  License error - code 111:  connect Connection refused
R50005 {CONTINUED} license source: 9010@mylicserver

如果随后运行 LicenseApp，它同样会报告无法联系许可证服务器。

LicenseApp 是一个有用工具，可指示许可证服务器当前状态；它也可以在客户端机器上运行，并会远程查询服务器状态。

在 macOS 上设置许可证服务

在启动 License Server 之前，必须先在浮动许可证绑定到的 macOS 机器上把它安装为服务。为此，请在 shell 窗口中执行以下命令：

sudo /Applications/Pixar/PixarLicense-LA-23.0/macOS_installService.sh 23 /Applications/Pixar

启动 License Server

注意：你必须使用上面列出的命令设置许可证服务，以确保该服务可以轻松启动和停止。

 

要启动许可证服务，请在 shell 窗口中执行以下命令：

sudo launchctl load /Library/LaunchDaemons/com.pixar.renderman.LicenseServer.plist

当 License Server 正常运行时，LicenseApp 程序会显示许可证功能和数量信息：

停止 License Server

如果需要停止许可证服务，以便让它重置并读取新的许可证文件，请在 shell 窗口中执行以下命令：

sudo launchctl unload /Library/LaunchDaemons/com.pixar.renderman.LicenseServer.plist

当 License Server 不再运行时，LicenseApp 程序会显示该信息：

RenderMan 和其他插件将不再运行。

重启 License Server

每次更新许可证文件后，都需要重启许可证服务，以加载新的许可证内容。可以在 shell 中输入以下命令来重启许可证：

sudo launchctl unload /Library/LaunchDaemons/com.pixar.renderman.LicenseServer.plist
sudo launchctl load /Library/LaunchDaemons/com.pixar.renderman.LicenseServer.plist

默认情况下，许可证服务器安装在 /Applications/Pixar/PixarLicense-LA-<version>，并包含以下组件和脚本：

    • PixarLicenseServer - 许可证服务器程序

    • LicenseApp/bin/LicenseApp - 许可证状态应用程序

    • queryPRMan.py - 用于查询许可证服务器的示例 Python 脚本

    • macOS_installService.sh - 安装器用于设置许可证服务器系统服务的脚本
