PxrLayerSurface

Source: https://rmanwiki-27.pixar.com/space/REN27/542216714/PxrLayerSurface
Downloaded: 2026-06-15T03:49:12+00:00

PxrLayerSurface 是一种简化材质，依赖于对 PxrLayer 与 PxrLayerMixer进行分层。这是你在应用程序中进行分层的推荐节点。你会注意到该节点与 PxrSurface 不同：许多你熟悉的参数看起来都不见了。

PxrLayerSurface 既作为分配给场景中对象的材质，也作为中心节点，用于修改所有分层材质的全局功能。由于并非所有参数都能在分层节点上覆盖，该节点包含适用于所有连接层的全局参数。你可能记得在 PxrSurface 中，参数会根据性质以粗体或斜体标识。PxrLayerSurface 只包含那些曾以斜体显示的参数。

例如：目前对 Beckmann 和 GGX 这样的不同反射模型进行分层并不合理，因此所选模型会由所有层共享。

参数从下方开始。



Input Material

连接到一个层 pattern 节点，用于对 Bxdf 参数进行分层。这让用户能够通过分层效果创建更复杂的外观。例如灰尘、划痕，甚至标签。

下方参数不能被覆盖，并将被所有连接的层共同遵循。



 Diffuse Parameters

Diffuse Parameters

Double Sided

如果开启，会照亮此 Diffuse 波瓣的表面两侧，也就是说，也会照亮法线背向摄像机的表面（二维对象）。这样背面会被着色而不是黑色。默认关闭。


Exponent

Diffuse exponent 控制漫反射衰减。它是一个幂指数。数值越高，衰减越锐利。例如，在月球表面，可以设置较高的 Exponent 来产生更锐利的衰减。

Diffuse exponent 仅适用于 Lambertian Diffuse，也就是 Roughness 为 0.0 时。将 Diffuse exponent 与 Oren-Nayar Roughness 组合没有意义，因此当 Roughness > 0.0 时会被忽略。

 Specular Parameters

Specular, Rough Specular, and Clear Coat Parameters

Specular 参数控制镜面反射。你可以在这里定义对象有多闪亮或多具反射性：它是塑料、抛光大理石桌面，还是一面镜子？下面的 Rough Specular 和 Clear Coat 波瓣具有相同的设置和效果。


Specular Fresnel Mode

在 Artistic 模式中，Specular Fresnel 响应由 Face Color、Edge Color 和 Fresnel Exponent 控制。

在 Physical 模式中，Specular Fresnel 响应由 Refractive Index、Extinction Coefficient 和 Edge Color 控制。


Specular Model

选择要使用的 Specular 模型：Beckmann 或 GGX。Beckmann 适合完美镜面和类似铬的材质。GGX 可能因其“尾部”更受青睐，也就是高光从光源中心反射向外柔和衰减的方式。左侧为 Beckmann，右侧为 Roughness 0.25 的 GGX。

Fresnel Exponent (Artistic Mode)

Specular Fresnel 曲线指数。较低数值会减弱 Face Color 的效果，同时增强 Edge Color的效果。较高数值则相反。如果正面和边缘颜色相同，则没有可见效果。下方使用 红色 Face Color 和 绿色 Edge Color，并在较小 Roughness 下将 Fresnel Exponent 从 0.1 增加到 1.5，最后到 5.0。


Shading Tangent

控制 Anisotropy 方向。仅当连接到 Pattern 时有效。这对制作拉丝金属很有用。下方三个示例使用纹理以及 Anisotropy -10

Double Sided

如果开启，会照亮此 Specular 波瓣的表面两侧，也就是说，它也会照亮法线背向摄像机的表面。默认关闭以提升性能。


 Specular Clearcoat Globals

Specular Energy Compensation

对 Diffuse 和 Subsurface 照明波瓣应用 Fresnel 能量补偿。值 1.0 会尝试根据 Specular 和 Rough Specular 的照明响应将它们变暗，从而完全平衡这些结果。

Specular 和 Rough Specular 的 Roughness 也会被考虑在内。随着 Specular 的正面或边缘颜色接近 1.0，该效果会淡出，因此金属可以添加一个 Diffuse 基准颜色。可查看 Clearcoat Energy Compensation 的可视化示例。


Clearcoat Energy Compensation

对 Clearcoat 本身以外的所有波瓣应用 Fresnel 能量补偿。值 1.0 会尝试根据 Clearcoat 照明响应将它们变暗，从而完全平衡这些结果。

Clearcoat Roughness 也会被考虑在内。随着 Clearcoat 的正面或边缘颜色接近 1.0，该效果会淡出，因此金属可以添加一个 Diffuse 基准颜色。左侧为 0.0（默认），右侧为 1.0。注意发生的变暗（能量守恒变化）。

 Iridescence Parameters

Iridescence

Iridescence 是一种依赖视角的光散射，会导致颜色偏移。这与肥皂泡、孔雀羽毛或闪亮甲虫上的色彩旋涡是同一种效果。“Holographic” 或变色漆也会使用这种效果。

Iridescence Mode

选择要使用的 Iridescence 模式： Artistic 或 Physical。

在 Artistic 模式中，我们只设置 2 种颜色。根据 Iridescence 缩放因子，你会看到 N 条“彩虹”。默认的红色和蓝色适合获得最大色彩跨度，但你可以通过更改这些默认值来减少渲染出的颜色数量。除非另有说明或正在演示 Artistic 参数，示例都使用 Physical 模式。

在 Physical 模式中，我们传入薄膜厚度，单位为纳米。当物理厚度接近可见光谱时，会产生 Iridescence 效果。你可以从大约 800nm 开始，并增加该值来观察效果。该选项很好，因为它以灵活性为代价减少了需要调整的参数。除非另有说明或正在演示 Artistic 参数，示例都使用 Physical 模式。下方是 Artistic （左）和 Physical（右）模式。

Fresnel Exponent

控制正面与边缘之间的衰减。类似于 Fresnel Exponent （Specular Lobes 中的参数）。通常保持默认值为 5.0

Falloff Speed

这仅适用于 Artistic 模式 。

从 Primary Color 到 Secondary Color 的衰减速度。数值越大，衰减越慢。下方使用 Artistic Mode 的默认值，但我们将 Falloff Speed 从 0.1 改为 0.5，再改为 1.0

Falloff Scale

这仅适用于 Artistic 模式 。

此项设置 Iridescence “彩虹”颜色重复的次数。下方从 0.5 到 1.0，最后到 3.0。注意较高值会开始重复彩虹效果。这对模拟油膜图案很有用，例如水面油膜或肥皂泡。

Flip Hue Direction

这仅适用于 Artistic 模式 。

在 primary 和 secondary colors 之间翻转色轮方向。默认情况下，色轮方向为逆时针。左侧为 off，右侧为 on。

Double Sided

如果开启，会照亮此 Iridescence 波瓣的表面两侧。这对没有厚度建模的薄开口表面（如羽毛和叶片）很有用。


 Fuzz Parameters

Fuzz Parameters

此参数会引入一点逆向反射，并帮助模拟织物、Fuzz 和细粉末。


Cone Angle

Fuzz 粗糙度（对应 Marschner R cone angle）。请注意，这里使用了 Marschner 毛发模型中的内容，有助于模拟类似毛发/皮毛的响应。数值越高，正对角度处的效果越强。下方为 8、16 和 32。


Double Sided

如果开启，会照亮此 Fuzz 波瓣的表面两侧，也就是说，它也会照亮法线背向摄像机的表面。


 Subsurface Scattering Parameters

Subsurface Scattering Parameters

Subsurface Model

选择次表面散射模型：Jensen Dipole、d'Eon Better Dipole、Burley Normalized、Multiple Mean Free Paths，以及新的 Path Traced 选项。参数会根据所选模型填充，并且只对该模型有效。如果更改所用模型，你的连接可能会因参数无效而丢失。你可以在主 PxrSurface 页面了解这些控制项和模型的详细信息。

Burley Normalized 可在保留细节的同时产生准确效果。

Jensen and d'Eon Dipoles 非常适合软糖这类高度半透明对象，以及扩展的艺术指导。

Multiple Mean Free Paths 非常适合通过纹理以直观方式产生色彩渗透。虽然不一定物理正确，但它对纹理颜色的散射很适合艺术指导，同时能避免颜色偏移。

Exponential Path Traced 对准确且详细的次表面散射非常出色。

Non-Exponential Path Traced 与上述 Path Traced 解法类似，但允许光线散射得更远。


Double Sided

如果开启，该 Subsurface 波瓣会照亮表面的两侧，也就是说，它也会照亮法线背向摄像机的表面。如果未开启，则不会在背面渲染。


Trace Control:

Consider Backside

控制 Subsurface 是否考虑另一侧的表面。这针对命中侧，而不是照明侧（照明侧由 subsurfaceDoubleSided 控制）：

  • “Off” - 完全忽略另一侧的表面。这有助于让对象看起来比实际更厚。

  • “On” - 正常模式，扩散会发生在正面与其后的第一个表面之间。


Continuation Ray Mode


当用户需要散射在不同对象之间产生贡献，或在内部带有空洞/空间的对象内部产生贡献时，此选项很有用，例如嘴巴与面部属于同一网格的模型。

控制 Continuation Ray Mode：

“Off” - 只从对象中追踪出去（默认）。

“Last Hit” - 忽略内部几何体，并跳到最后一个表面。

“All Hits” - 当光线离开对象时，在所有命中处进行散射（收集光照）。这可以带来额外亮度，代价是增加噪声。

下方左侧使用“All Hits”，可以清楚看到嵌入球体中的“SUBSURFACE”文字。右侧设置为“Off”，这会通过截断该衰减导致嵌入几何体渲染不正确。

Max Continuation Hits

在 All Hits 模式中要测试的最大命中数。仅当 Continuation Ray Mode 等于 All Hits 时有效。


Follow Topology

控制在 Subsurface 计算中考虑法线的强度。这也可能影响通过凹凸贴图创建的可见细节。


Trace Subset

指定在被光线间接击中时用于包含/排除的追踪子集。
 

 Single Scatter Parameters

Single Scattering Parameters

单次散射是一种简单且开销较低的散射效果。下方有一个 圆盘灯被放置在模型内部 （所有这些示例都如此），我们先在没有漫反射颜色的情况下渲染，然后再使用默认中灰漫反射颜色渲染。


Double Sided

如果开启，该单次散射波瓣会照亮表面的两侧，也就是说，它也会照亮法线背向摄像机的表面。


Trace Control Parameters（操作方式与说明中展示的 Subsurface Controls 类似）：

Consider Backside

控制单次散射是否考虑另一侧的表面。这针对命中侧，而不是照明侧（照明侧由 singlescatterDoubleSided 控制）：

  • “Off” - 完全忽略另一侧的表面。这有助于让对象看起来比实际更厚。

  • “On” - 正常模式，扩散会发生在正面与其后的第一个表面之间。


Continuation Ray Mode

控制 Continuation Ray Mode：

“Off” - 只从对象中追踪出去（默认）。

“Last Hit” - 忽略内部几何体，并跳到最后一个表面。

“All Hits” - 当光线离开对象时，在所有命中处进行散射（收集光照）。这可以带来额外亮度，代价是增加噪声。


Max Continuation Hits

在 All Hits 模式中要测试的最大命中数。仅当 Continuation Ray Mode 等于 All Hits 时有效。


Direct Gain Mode

控制 Continuation Ray Mode：

“First Hit” - 只追踪到下一个表面（这与 considerBackside 绑定）。

“Last Hit” - 忽略内部几何体，并跳到最后一个表面。

“All Hits” - 当光线离开对象时，在所有命中处进行散射（收集光照）。这可以带来额外亮度，代价是增加噪声。


Trace Subset

指定在被光线间接击中时用于包含/排除的追踪子集。


Scattering Globals

Irradiance Tint

在由 Subsurface 或单次散射进行散射之前应用到照明上的 Tint。


Irradiance Roughness

应用于 Subsurface 或单次散射的 Diffuse 粗糙度。值为 0 表示经典 Lambertian 着色模型。非零值会提高 Oren-Nayar 着色模型的微表面粗糙度。更大的值会产生更粗糙的 Diffuse，并可能显得略暗。


Unit Length

Subsurface 和单次散射的 Unit Length。它是 Mean Free Path Distance 的乘数。Mean Free Path Distance 通常以毫米为单位。如果场景以其他尺度建模，应相应设置 Unit Length。默认值 0.1 适合以厘米建模的场景（Maya 默认）和以毫米测量的 Mean Free Path Distance。较小数值会让对象更接近漫反射（半透明程度更低），较大数值会增强散射效果，使其更半透明。


 Glass Parameters

Glass Parameters

Thin

如果开启，会根据 Refractive Index 在反射和折射之间正确分配能量，但不会在 Refraction 中弯折光线（用单层玻璃模拟双层玻璃，或模拟中空玻璃装饰物）。下方的 内部 参数不适用于该模式。


Interior Parameters

Single Scatter Albedo

Single Scatter Albedo 颜色会在玻璃表面内部创建雾。这对于含有淤泥或脏污（均匀）水的容器很有用。 Extinction 不应为黑色或 0。纹理化需要 3D Manifold 才能工作。


Overlapping Volumes

用于启用或禁用表面内的 Overlapping Volumes。如果关闭，则只有内部体积可见。如果开启，内部体积会与表面内部的其他体积混合。


Multiscatter

启用时，该参数会在内部体积中启用间接反弹。


Extinction

Extinction 颜色。它应用于上方的 Single Scatter Albedo。要让 Single Scatter Albedo 生效，它不应为黑色。该参数选择最受 Extinction 影响的颜色或颜色组合。选择纯红色表示红色会被消光，但绿色和蓝色组合不会。


Single Directionality

控制散射的方向性。

0：各向同性

1：前向

-1：后向


 Global Properties Parameters

Properties Parameters

本节为整个材质提供全局控制。例如，如果只在这里提供 Bump，它会应用到上面的所有波瓣。Presence 也是一个全局参数，它会遮罩掉或“镂空”所有波瓣，就像遮罩值为 0.0 的位置对象不存在一样。

Presence

在这里连接遮罩，可把镂空图案应用到对象上。这对于创建叶片和其他细薄、复杂形状等镂空效果很有用。当从相机直接看到时，它也可把灰度值用作不透明度，以得到半不透明结果。

  • 对于非 Thin 玻璃折射，以及颜色通过内部衰减的“thick”风格对象，此设置会被忽略。

Presence Cached

指定 Presence 是否缓存。启用缓存时，Presence 将按 per micropolygon 求值，而不是 per shading point ；关闭缓存时则按 per shading point 求值。 

  • 由于 Presence 计算可能很昂贵，如果存在纹理等较重的着色网络，缓存有利于提高渲染速度，不过会增加内存。

  • 由于缓存本质上存储在 tessellation 层级，如果需要更细的渲染细节，建议使用未缓存的 Presence，因为它会按 per shading point 求值。

  • 内存和速度之间存在取舍。如果 Presence 非常简单，例如常量值，那么缓存和未缓存可能几乎一样快，并能节省一些内存。请注意，如果 Presence 为常量 1.0（默认值），PxrSurface 会优化所有这些处理。

  • 当 Subsurface 或单次散射开启时，为避免伪影（与 volume tracking 相关），PxrSurface 会在内部强制使用未缓存的 Presence（忽略 Presence Cached 的参数设置）。 

  • 出于类似原因，对于非 Thin 玻璃折射，PxrSurface 会忽略 Presence，无论它是否缓存。

Shadow Mode

阴影不透明度计算模式：

  • Shader and shadow color

  • Shadow color only

Shadow Color

指定 Shadow Color。该参数可用于为艺术指导目的伪造阴影颜色。

Utility Pattern

指定一个用于输出的 Pattern 节点。例如：可以把 PxrMatteID 连接到此参数来输出 ID AOV。也可以输出另一种 Pattern，例如纹理。
