txmake

Source: https://rmanwiki-27.pixar.com/space/REN27/542236544/txmake
Downloaded: 2026-06-15T03:49:12+00:00

txmake 会读取名为 foo.format 的图像文件（或多个图像文件），并创建一个名为 foo.tex的 RenderMan 纹理文件。 foo.tex 原有内容（如果有）会被静默覆盖。输入文件支持以下格式：BMP、DPX、GIF、HDR、JPEG、MayaIFF、OpenEXR、PIC、PNG、PPM、Pixar DeepData、Pixar Texture、SGI、TGA、TIFF、XPM、Z-File。

txmake 基于 IceMan 图像合成引擎编写，并增加了对以下功能的支持：可处理大型输入图像，所有图像计算都以浮点执行，可选择多个通道，并且在计算 MIPmap 或 RIPmap 的每个后续层级时都可以应用滤镜。

纹理

包裹模式

-mode <m> 将 <m> 设置选择应用到 S 和 T
-smode <m> 将 <m> 设置选择应用到 S
-tmode <m> 将 <m> 设置选择应用到 T

<m> 控制纹理坐标位于第一个 ST 图块之外时返回的纹理值。

  • black 适用于基于图块的工作流，例如 UDIM。

  • clamp 适用于非周期性纹理访问。

  • periodic 适用于周期性纹理访问。

默认值为 'black'。

图像尺寸调整

-resize <r> 应用尺寸调整操作，将尺寸调整到最接近的 2 的幂-尺寸。

对于简单纹理或环境纹理，纹理文件的分辨率由图像文件的高度、宽度以及尺寸调整选项指定的操作决定。如果图像的宽度或高度正好是 2 的幂，该维度会保持不变。任何不是 2 的幂的维度都会根据尺寸调整操作处理。如果图像尺寸已经是 2 的幂，则尺寸不会改变。除所用文件类型和执行转换的机器资源之外，转换没有尺寸限制。

请注意，许多文件格式可能有独立于 txmake 的存储类型和尺寸限制。TIFF 和 EXR 是最常见的类型。

  • 转换为 .tx 的 TIFF 文件可能比其他格式性能更好，但最大尺寸限制为 4GB。

  • OpenEXR 文件可以更大，也可能更慢；但在使用以下设置转换为 .tx 时，它们的限制最少： -format openexr 


<r> 尺寸调整操作可以是以下选项之一：

  • none 不执行尺寸调整。纹理贴图大小将使用下一个更高的 2 的幂。纹理文件中未被图像覆盖的区域会设为 black。

  • up 将图像向上调整到下一个更高的 2 的幂。纹理访问会校正这次尺寸调整。

  • down 将图像向下调整到下一个更低的 2 的幂。纹理访问会校正这次尺寸调整。

  • round 将图像调整到最接近的 2 的幂。纹理访问会校正这次尺寸调整。

  • up- 将图像向上调整到下一个更高的 2 的幂。生成图像上的纹理坐标范围为 0 到 1。

  • down- 将图像向下调整到下一个更低的 2 的幂。生成图像上的纹理坐标范围为 0 到 1。

  • round- 将图像调整到最接近的 2 的幂。生成图像上的纹理坐标范围为 0 到 1。

图像来源：NASA

默认值为 'up'。

调整尺寸滤镜

 -filter <f> 在调整尺寸时应用以下滤镜选项之一。

<f> 是用于调整图像尺寸的滤镜。以下选项可用于调整图像尺寸。锐化滤镜通常更适合放大。

  • point

  • box

  • triangle

  • sinc

  • gaussian

  • gaussian-soft

  • catmull-rom

  • mitchell

  • cubic

  • lanczos

  • bessel

  • blackman-harris

默认值为 'catmull-rom'。

图像模糊控制（-blur <f> / -sblur <f> / -tblur <f>）

-blur <f> 将模糊应用到 S 和 T 两个方向
-sblur <f> 只将模糊应用到 S
-tblur <f> 只将模糊应用到 T

<f> 大于 1.0 的值会模糊图像。

默认值为 1.0。

Alpha 边缘像素扩展（Bleed Alpha）

-bleedalpha

围绕 alpha 遮罩径向扩张像素值，适用于 纹理烘焙。这个边缘扩展区域有助于避免 ST 接缝周围的采样伪影。

mipmap 映射

mipmap 映射会根据屏幕上纹理的可见大小和角度，从金字塔层级中自动选择合适的纹理分辨率层级，从而改善纹理走样并提升渲染性能。这样可以减少处理时间，代价是损失一些不易察觉的细节。

mipmap 模式

-pattern <p>

<p> 可以是以下选项之一： 

  • single 纹理文件中只存储一张纹理图像。这实际上会禁用 mipmap 映射，并且对大多数应用来说会导致非常差的纹理性能。

  • diagonal 通过在 S 和 T 两个方向上连续将纹理图像降低到一半分辨率，生成更低分辨率的纹理。这样得到的纹理文件约比原始纹理图像大三分之一。

  • all 纹理文件中存储完整的一组纹理图像（即 ripmap）。这会让纹理文件约为原始纹理图像的四倍大小。

默认值为 'diagonal'。

mipmap 来源 

-mipfrom <s/n>

<s/n> 可以是以下选项之一，其中使用 n 整数。

  • previous 从前一个层级创建每个 mipmap 层级。

  • source 从源图像创建所有 mipmap。

  • <n> 指定显式的 mipmap 层级。

默认值为 'source'。

用户 mipmap

-usermipmap <picnames>

<picnames> 列表中的文件对应金字塔中的各个层级（列表中的第一个名称 = 金字塔的最高分辨率层级）。请注意，生成这些“用户 mipmap”有几条规则：

  • 第一张图像决定要写入纹理的变换矩阵。

  • 第一张图像决定纹理尺寸。

  • 该列表中的所有图像必须具有相同数量的通道。

  • 该列表中的图像必须至少与对应的金字塔层级一样大（也就是说，4x4 图像不能放入 8x8 图像）。

  • 该列表中的图像必须全部具有相同的相对尺寸（例如 4x2 图像和 4x4 图像不兼容，但 4x4 与 8x8 兼容，4x2 与 8x4 也兼容）。

  • 除非格式为 tiff，否则该列表中的任何图像都不能大于第一张图像。

  • 除非选择 "single"，否则该列表中的最后一张图像会被向上过滤到金字塔的其余部分。

mipmap 滤镜

-mipfilter <f>

<f> 是用于创建 mipmap 的像素滤镜。请从以下选项中选择：

  • point

  • box

  • triangle

  • sinc

  • gaussian

  • gaussian-soft

  • catmull-rom

  • mitchell

  • cubic

  • lanczos

  • bessel

  • blackman-harris

默认值为 'box'。

环境贴图设置

经纬度贴图

-envlatl

创建经纬度环境贴图。

漫反射卷积

 -diffconv <brdf,res,mip>

txmake 可以生成旁带数据，用于蒙特卡洛积分的控制变量法。旁带是漫反射 BRDF 与环境贴图的卷积。仅支持经纬度贴图。支持 OpenEXR 和 TIFF 文件。

  • brdf 是 {Lambert, KajiyaKay} 之一。此项为必需。

  • xres 是 mipmap 金字塔中计算卷积时所用的图像宽度。此项可选，默认值为 256。

  • "mip" 是可选的字面值。如果添加，卷积图像会生成 mipmap；否则不会生成（其他 mip 层级将包含 black）。

立方体环境贴图

-envcube

从 6 张图像创建立方体环境贴图，例如：

txmake -format openexr red.png yellow.png green.png cyan.png blue.png magenta.png cubemap.exr

环境立方体贴图会使用特定格式的布局创建：tiff、pixar、openexr。

视场角设置 

-fov <f>

<f> 是以度为单位的立方体贴图视场角。

Ptex 立方体贴图

-penvcube

从 6 张图像创建 Ptex 立方体环境贴图。

球谐带数

-shbands <n>

嵌入 <n> 个球谐带。

格式

图像文件格式

-format <f>

<f> 对应底层纹理图像格式（对于生成的大型 >4GB 图像，必须使用 openexr）：

  • tiff

  • pixar

  • openexr

默认的 tiff 格式是在 rendermn.ini 中作为配置选项指定的。

数据压缩方式

 -compression <c>

对于 Pixar 格式，<c> 为：

  • none

  • lossless

  • lossy

对于 OpenEXR 格式，<c> 为：

  • none

  • rle

  • zip

  • piz

  • pxr24

  • b44

  • b44a

TIFF 格式始终使用 LZW 压缩。Pixar 格式默认使用 'lossless' 压缩，而 OpenEXR 默认使用 'zip'。

输出数据类型

-byte / -short / -half / -float

控制文件格式的数据类型。

  • byte 输出无符号 8-bit 数据

  • short 输出有符号 16-bit 数据

  • half 输出半精度浮点数据（OpenEXR/penv）

  • float 输出浮点数据

通道

-ch <#,#:#>

默认情况下，纹理文件中通道的数量和顺序直接取决于图像文件中的通道集合。RGBA 图像会生成四通道纹理，RGB 图像会生成三通道纹理，R 图像会生成单通道纹理。-ch 选项可用于选择并重新排序图像文件的通道。通道编号用于选择图像文件的通道。通道索引从 0 开始。对于 RGB，`-ch 1` 只会选择图像文件的绿色通道，并将其放入纹理文件的通道 0。可以使用数字列表选择多个通道，例如 `-ch 0,1,2` 和 `-ch 0:2` 会选择图像文件的 RGB 通道，而 `-ch 2,1,0` 会选择相同通道，但顺序为蓝、红、绿。

显示窗口设置

-extraargs displaywindow

displaywindow 仅用于 exr 输入。


凹凸转粗糙度

凹凸转粗糙度要求所有凹凸贴图或法线贴图都转换为 *.b2r 纹理。

b2r 纹理包含 5 个通道，并且始终以 OpenEXR 格式生成，因为它能为半精度浮点数据提供最佳压缩率。

-bumprough <f> <i> <n> <iu> <iv> <i>

  • <f>：粗糙度缩放系数

    • 默认值：2.0（极少需要更改）

  • <i>：反转凹凸信号（0|1）

    • 默认值：0

  • <n>：解释为法线贴图（0|1）

    • 默认值：0

  • <iu>：在 U 方向反转贴图（0|1）

    • 默认值：0

  • <iv>：在 V 方向反转贴图（0|1）

    • 默认值：0

  • <iv>：重新拟合斜率分布，以获得更好的结果。

    • 默认值：1。 

    • 这一步耗时较长，但从远处观看时能更好地保留细节。

典型的 txmake 凹凸贴图调用如下：

txmake -resize round- -mode periodic -bumprough 2 0 0 0 0 1 bump.tif bump.b2r

法线贴图的调用如下：

txmake -resize round- -mode periodic -bumprough 2 0 1 0 0 1 normalmap.tif normalmap.b2r

命令行选项

较新文件检查 

-newer

如果源文件比目标纹理更旧，则不执行任何操作。

目录映射设置

-dirmap <s> -dirmapend

对输入和输出文件应用目录映射。可以指定多个目录映射。<s> 的语法应类似于 RiOption "searchpath" "dirmap"，例如：

["NFS" "z:/" "/mnt/"] ["NFS" "Z:/" "/mnt/"]

线程

-t:X

使用 X 个线程。默认使用所有线程。

详细输出

-verbose

在 txmake 执行时打印更新。

版本

-version

打印版本并退出。

帮助

-help

打印用法并退出。
