txmake

Source: https://rmanwiki-27.pixar.com/space/REN27/542236544/txmake
Downloaded: 2026-06-15T03:49:12+00:00

txmake 会读取名为 foo.format 的图像文件（或多个图像文件），并创建一个 RenderMan texture file，名称为 foo.tex的 RenderMan texture file。 foo.tex 原有内容（如果有）会被静默覆盖。输入文件支持以下格式：BMP、DPX、GIF、HDR、JPEG、MayaIFF、OpenEXR、PIC、PNG、PPM、Pixar DeepData、Pixar Texture、SGI、TGA、TIFF、XPM、Z-File。

txmake 基于 IceMan image compositing engine 编写，并增加了对以下功能的支持：可处理大型输入图像，所有图像计算都以 floating point 执行，可选择多个 channels，并且在计算 MIPmap 或 RIPmap 的每个后续层级时都可以应用 filters。

纹理（Texture）

纹理 Wrap Mode（包裹模式）

-mode <m> 将 <m> 设置选择应用到 S 和 T
-smode <m> 将 <m> 设置选择应用到 S
-tmode <m> 将 <m> 设置选择应用到 T

<m> 控制 texture coordinates 位于第一个 ST tile 之外时返回的 texture values。

  • black 适用于基于 tile 的工作流，例如 UDIM。

  • clamp 适用于非周期性的 texture access。

  • periodic 适用于周期性的 texture access。

默认值为 'black'。

图像尺寸调整（Image Resize）

-resize <r> 应用 resize 操作，将尺寸调整到最接近的 2 的幂-尺寸。

对于简单 texture 或 environment texture，texture file 的分辨率由图像文件的高度、宽度以及 resize option 指定的操作决定。如果图像的宽度或高度正好是 2 的幂，该维度会保持不变。任何不是 2 的幂的维度都会根据 resize operation 调整。如果图像尺寸已经是 2 的幂，则尺寸不会改变。除所用文件类型和执行转换的机器资源之外，转换没有尺寸限制。

请注意，许多文件格式可能有独立于 txmake 的存储类型和尺寸限制。TIFF 和 EXR 是最常见的类型。

  • 转换为 .tx 的 TIFF 文件可能比其他格式性能更好，但最大尺寸限制为 4GB。

  • OpenEXR 文件可以更大，也可能更慢；但在使用以下设置转换为 .tx 时，它们的限制最少： -format openexr 


<r> resize operation 可以是以下选项之一：

  • none 不执行 resize。texture map size 将使用下一个更高的 2 的幂。texture file 中未被图像覆盖的区域会设为 black。

  • up 将图像向上调整到下一个更高的 2 的幂。texture access 会校正这次 resize。

  • down 将图像向下调整到下一个更低的 2 的幂。texture access 会校正这次 resize。

  • round 将图像调整到最接近的 2 的幂。texture access 会校正这次 resize。

  • up- 将图像向上调整到下一个更高的 2 的幂。生成图像上的 texture coordinates 范围为 0 到 1。

  • down- 将图像向下调整到下一个更低的 2 的幂。生成图像上的 texture coordinates 范围为 0 到 1。

  • round- 将图像调整到最接近的 2 的幂。生成图像上的 texture coordinates 范围为 0 到 1。

图像来源：NASA

默认值为 'up'。

调整尺寸滤镜（Resize Filter）

 -filter <f> 在 resized 时应用以下 filter choices 之一。

<f> 是用于调整图像尺寸的 Filter。以下选项可用于 resize 图像。sharpening filters 通常更适合放大。

  • point

  • box

  • triangle

  • sinc

  • gaussian

  • gaussian-soft

  • catmull-rom

  • mitchell

  • cubic

  • lanczos

  • bessel

  • blackman-harris

默认值为 'catmull-rom'。

图像模糊控制（Image Blur：-blur <f> / -sblur <f> / -tblur <f>）

-blur <f> 将 blur 应用到 S 和 T 两个方向
-sblur <f> 只将 blur 应用到 S
-tblue <f> 只将 blur 应用到 T

<f> 大于 1.0 的值会模糊图像。

默认值为 1.0。

Alpha 边缘像素扩展（Bleed Alpha）

-bleedalpha

围绕 alpha mask 径向扩张 pixel values，适用于 texture baking。这个 gutter 有助于避免 ST seams 周围的 sampling artifacts。

mipmapping

mipmapping 通过根据屏幕上 texture 的可见大小和角度，从 pyramid 中自动选择合适的 texture resolution level，来改善 texture aliasing 和渲染性能。这样可以减少处理时间，代价是损失一些不易察觉的细节。

mipmap 模式（Mipmap Pattern）

-pattern <p>

<p> 可以是以下选项之一： 

  • single texture file 中只存储一张 texture image。这实际上会禁用 mipmapping，并且对大多数应用来说会导致非常差的 texturing performance。

  • diagonal 通过在 S 和 T 两个方向上连续将 texture image 降低到一半分辨率，生成更低分辨率的 textures。这样得到的 texture file 约比原始 texture image 大三分之一。

  • all texture file 中存储完整的一组 texture images（即 ripmap）。这会让 texture file 约为原始 texture image 的四倍大小。

默认值为 'diagonal'。

mipmap 来源（Mipmap Source） 

-mipfrom <s/n>

<s/n> 可以是以下选项之一，其中使用 n 整数。

  • previous 从前一个层级创建每个 mipmap level。

  • source 从 source image 创建所有 mipmaps。

  • <n> 指定显式的 mipmap level。

默认值为 'source'。

用户 mipmap（User Mipmaps）

-usermipmap <picnames>

<picnames> 中的文件对应 pyramid 中的各个层级（picnames 中的第一个名称 = pyramid 的最高分辨率层级）。请注意，生成这些 "user mipmaps" 有几条规则：

  • 第一张图像决定要写入 texture 的 transformation matrix。

  • 第一张图像决定 texture 的 dimensions。

  • picnames 中列出的所有图像必须具有相同数量的 channels。

  • picnames 中列出的图像必须至少与对应的 pyramid level 一样大（也就是说，4x4 图像不能放入 8x8 图像）。

  • picnames 中的图像必须全部具有相同的相对尺寸（例如 4x2 图像和 4x4 图像不兼容，但 4x4 与 8x8 兼容，4x2 与 8x4 也兼容）。

  • 除非格式为 tiff，否则 picnames 中列出的任何图像都不能大于第一张图像。

  • 除非选择 "singl"，否则 picnames 中列出的最后一张图像会被向上过滤到 pyramid 的其余部分。

mipmap 滤镜（Mipmap Filter）

-mipfilter <f>

<f> 是用于创建 mipmap 的 pixel filter。请从以下选项中选择：

  • point

  • box

  • triangle

  • sinc

  • gaussian

  • gaussian-soft

  • catmull-rom

  • mitchell

  • cubic

  • lanczos

  • bessel

  • blackman-harris

默认值为 'box'。

环境贴图设置（Environment Map）

经纬度贴图（Lat-Long Map）

-envlatl

创建经纬度环境贴图（lat-long environment map）。

漫反射卷积（Diffuse Convolution）

 -diffconv <brdf,res,mip>

txmake 可以生成 side-band data，用于 Monte Carlo integration 的 control variates method。side-bands 是 diffuse BRDF 与 environment map 的 convolution。仅支持 lat-long maps。支持 OpenEXR 和 TIFF 文件。

  • brdf 是 {Lambert, KajiyaKay} 之一。此项为必需。

  • xres 是 mip-map pyramid 中计算 convolution 时所用的图像宽度。此项可选，默认值为 256。

  • "mip" 是可选的 literal。如果添加，convolution image 会生成 mip-map；否则不会生成（其他 mip levels 将包含 black）。

立方体环境贴图（Cube Map）

-envcube

从 6 张图像创建 cube environment map，例如：

txmake -format openexr red.png yellow.png green.png cyan.png blue.png magenta.png cubemap.exr

Environment cube maps 会使用特定格式的 layout 创建：tiff、pixar、openexr。

视场角设置（Filed of View） 

-fov <f>

<f> 是以度为单位的 cube map field-of-view。

Ptex 立方体贴图（Ptex Cube Map）

-penvcube

从 6 张图像创建 ptex cube environment map。

球谐带数（Spherical Harmonic Bands）

-shbands <n>

嵌入 <n> 个 spherical harmonic bands。

格式（Format）

图像文件格式（Image Format）

-format <f>

<f> 对应底层 texture image format（对于生成的大型 >4GB 图像，必须使用 openexr）：

  • tiff

  • pixar

  • openexr

默认 tiff 是在 rendermn.ini 中作为 configuration option 指定的。

数据压缩方式（Data Compression）

 -compression <c>

对于 Pixar format，<c> 为：

  • none

  • lossless

  • lossy

对于 OpenEXR format，<c> 为：

  • none

  • rle

  • zip

  • piz

  • pxr24

  • b44

  • b44a

TIFF format 始终使用 LZW compression。Pixar format 默认使用 'lossless' compression，而 OpenEXR 默认使用 'zip'。

输出数据类型（Data Type）

-byte / -short / -half / -float

控制文件格式的数据类型（file format datatype）。

  • byte 输出无符号 8-bit 数据（unsigned 8-bit data）

  • short 输出有符号 16-bit 数据（signed 16-bit data）

  • half 输出 half-float data（OpenEXR/penv）

  • float 输出 float data

通道（Channel）

-ch <#,#:#>

默认情况下，texture file 中 channels 的数量和顺序直接取决于 image file 中的 channel 集合。RGBA picture 会生成四通道 texture，RGB picture 会生成三通道 texture，R picture 会生成单通道 texture。-ch option 可用于选择并重新排序 image file 的 channels。channel #'s 用于选择 image file 的 channels。Channels indexing 从 0 开始。对于 RGB，`-ch 1` 只会选择 image file 的 green channel，并将其放入 texture file 的 channel 0。可以使用数字列表选择多个 channels，例如 `-ch 0,1,2` 和 `-ch 0:2` 会选择 image file 的 RGB channels，而 `-ch 2,1,0` 会选择相同 channels，但顺序为 blue、red、green。

显示窗口设置（Display Window）

-extraargs displaywindow

displaywindow 仅用于 exr input。


Bump To Roughness（凹凸转 Roughness）

Bump to roughness 要求所有 bump maps 或 normal maps 都转换为 *.b2r texture。

b2r textures 包含 5 个 channels，并且始终以 OpenEXR format 生成，因为它能为 half-float data 提供最佳 compression ratios。

-bumprough <f> <i> <n> <iu> <iv> <i>

  • <f>：roughness scaling factor

    • 默认值：2.0（极少需要更改）

  • <i>：invert bump signal（0|1）

    • 默认值：0

  • <n>：interpret as normal map（0|1）

    • 默认值：0

  • <iu>：在 U 方向 invert map（0|1）

    • 默认值：0

  • <iv>：在 V 方向 invert map（0|1）

    • 默认值：0

  • <iv>：重新拟合 slope distribution，以获得更好的结果。

    • 默认值：1。 

    • 这一步耗时较长，但从远处观看时能更好地保留 details。

典型的 txmake bump map 调用如下：

txmake -resize round- -mode periodic -bumprough 2 0 0 0 0 1 bump.tif bump.b2r

normal map 的调用如下：

txmake -resize round- -mode periodic -bumprough 2 0 1 0 0 1 normalmap.tif normalmap.b2r

命令行选项（Command Line）

较新文件检查（Newer） 

-newer

如果 source 比 target texture 更旧，则不执行任何操作。

目录映射设置（Directory Map）

-dirmap <s> -dirmapend

对输入和输出文件应用 dirmapping。可以指定多个 dirmaps。<s> 的语法应类似于 RiOption "searchpath" "dirmap"，例如：

["NFS" "z:/" "/mnt/"] ["NFS" "Z:/" "/mnt/"]

线程（Threads）

-t:X

使用 X 个 threads。默认使用所有 threads。

详细输出（Verbose）

-verbose

在 txmake 执行时打印 updates。

版本（Version）

-version

打印 version 并退出。

帮助（Help）

-help

打印 usage 并退出。
