Stylized Control

Source: https://rmanwiki-27.pixar.com/space/REN27/542221999/Stylized+Control
Downloaded: 2026-06-15T03:49:12+00:00

Stylized Control pattern 会与 Display Filters 交互，让你控制自己的 Stylized Looks（风格化外观）。

将它们接入你的 surface，即可按逐材质方式自定义。 

Input Parameters（输入参数）

Random Type

id 使用随机 color float。integer 使用整数值。两者都用于在相邻 shape nodes 上生成变化，以供最终 edge detection 使用。

Cam Depth Type

2 种模式：camera P 的长度，或 camera P 的 Z 分量绝对值

Lines Page

Section Spread Scale

如果相邻对象上的 random id sections colors 过于接近，可将此值调到 1 以上，让随机颜色分布得更远，从而在对象边界上得到更一致的 edge detection。

Albedo Color

透传 color，可将输出 RGB 连接到 PxrSurface Diffuse color

Line Albedo Color

当选择 Line Color From Pattern 选项时，PxrStylizedLines 生成的线条颜色

Line Albedo HSV

Line Albedo color/map 的 Hue、Saturation、Value 色彩校正

Line Thickness

如果在 PxrStylizedLines 中选择 Line Thickness From Pattern，则此 Line Thickness 会用于附加了该节点的所有对象。

Line Thickness Remap Switch

OFF：仅使用 float Line Thickness 值

ON：将上游 Pattern（fractal、texture 等）连接到 Line Thickness，然后使用下面的 remap_min 和 remap_max 在两者之间做线性 step。例如：如果有一个在 0..1 之间变化的 fractal 接入 Line Thickness，remap_min=0、remap_max=10，则 fractal 为 1（white）时最终 line thickness 为 10，fractal 为 0（black）时为 0

Line Thickness Remap Min

如果在 PxrStylizedLines 中选择 Line Thickness From Pattern，并且上游节点已连接到 Line Thickness，则使用此 min 值作为上游 0 值的 remap 目标。示例：如果 PxrFractal 接在 Line Thickness 上游，PxrFractal pattern 中的 0（black）值会 remap 到此 min

Line Thickness Remap Max

如果在 PxrStylizedLines 中选择 Line Thickness From Pattern，并且上游节点已连接到 Line Thickness，则使用此 min 值作为上游 0 值的 remap 目标。示例：如果 PxrFractal 接在 Line Thickness 上游，PxrFractal pattern 中的 1（white）值会 remap 到此 max

Line Mask

通过接入上游节点 texture 或 pattern，遮罩掉最终 edge detected lines 中不需要的区域。

Hatching Mask

通过接入上游节点 texture 或 pattern，遮罩掉最终 hatching 中不需要的区域。值存储在 NPRlineMask AOV 的 GREEN 通道中

Sections Map Switch

使用下面 sections \map 中的 map 来定义图像中的变化，而不是 random id

Sections Map

可选 color map，用于在 random id per object 未检测到线条的位置添加线条。例如：如果希望在角色下巴处触发 edge detection，而 random sections id 没有差异，则可接入一张在目标区域绘制了 2 种不同颜色的上游 texture map，最终 sections map 会传递给 edge detection algorithm。默认黑色（0,0,0）不会触发变化。

Sections Map Black Mask Switch

在自定义 sections Map 中为黑色的位置，composite over 现有 sections random color

Sections HSV

以 HSV 为 Sections Map 着色。第 1 个输入 = Hue，第 2 个输入 = Saturation，第 3 个输入 = Value

Hatching Projections Page

Hatching Mask

通过接入上游节点 texture 或 pattern，遮罩掉最终 hatching 中不需要的区域。值存储在 NPRlineMask AOV 的 GREEN 通道中。

P Triplanar

P：object space P；__Pref：如果作为 primvar 存在；Input P Triplanar：连接到此节点 inputPtriplanar inputs 的上游 PxrManifold3d。

Input P Triplanar

如果在 PxrStylizedHatching 中使用 Blended Triplanar Projections，可选择在这里将上游 point P XYZ 接入 RGB。例如：对于角色的变形动画，可在这里连接通过 PxrProjector 或 PxrManifold3d 提供的自定义 __Pref。

N Triplanar

N：默认 global N；__Nref：如果作为 primvar 存在；Input N Triplanar：用户连接到此节点 inputNtriplanar inputs 的任何上游 XYZ for N。

Input N Triplanar

如果在 PxrStylizedHatching 中使用 Blended Triplanar Projections，可选择在这里将上游 normal N XYZ 接入 RGB。示例：对于角色的变形动画，可在这里连接通过 PxrProjector 或 PxrManifold3d 提供的自定义 __Nref。

Input Texture Coordinates

如果在 PxrStylizedHatching 中使用 Texture Coordinates，可选择在这里将上游 point P XYZ 接入 RGB。例如，如果需要自定义 camera projection，可在这里接入上游 PxrProjector。

Triplanar Scale X

Scale：缩放 P 的 X 分量，用于 triplanar projections，以改变该平面上的 texture frequency

Triplanar Scale Y

Scale：缩放 P 的 Y 分量，用于 triplanar projections，以改变该平面上的 texture frequency

Triplanar Scale Z

Scale：缩放 P 的 Z 分量，用于 triplanar projections，以改变该平面上的 texture frequency


将 Teapot 上的 Triplanar Scale XYZ 从 1 改为 3
