PxrDistantLight

Source: https://rmanwiki-27.pixar.com/space/REN27/542227947/PxrDistantLight
Downloaded: 2026-06-15T03:49:12+00:00

此无限远灯光模拟太阳或月亮等明亮的远距离光源。此灯光会照亮整个场景，并且默认将灯光视为光线平行（可在下方设置中更改）。

只会应用旋转（方向）。此灯光的平移和缩放会被忽略。

你会看到对“非物理”控制的提及。这些控制用于帮助艺术家通过忽略我们通常模拟的某些物理定律，制作艺术指导图像。

Parameters（参数）


Basic


Intensity

线性缩放此灯光的贡献。灯光强度使用基于物理的控制来计算。默认灯光从较高的 50000.0 值开始。


Exposure

指定灯光的 Exposure，以 2 的幂表示。Exposure 增加 1 会使光源发出的能量翻倍。值 0 在光源处产生 1 的强度，-1 产生 0.5。现实世界照明能量很高，典型曝光值较低；若要获得等效的 Intensity，你可能需要输入非常大的数字。对于熟悉摄影测量的艺术家来说，这也更顺手。


Angle Extent

光源的张角，单位为度（Sun=0.53）。值越高，阴影边缘越柔和（光线变得不那么平行）；值越低，阴影越锐利。较大的值会通过增加灯光角度来提高阴影柔和度。注意，增大此参数也会增加场景中的光量，你可能需要大幅降低灯光 Intensity/Exposure 来补偿，或者可以使用 Area Normalize 来防止照明发生变化。


Color

发出灯光的颜色。


Enable Temperature

开启或关闭色温。下面使用以下温度：6500K（默认）、4000K，最后是 8500K。


Color Temperature

允许用户选择灯光的色温，单位为 Kelvin。不同于基础灯光颜色，这让用户可以根据标准温度测量轻松选择可信的灯光颜色。此控制的作用类似灯光前方的滤镜或色片（一层带色调的透明表面）。如果灯光为白色，你会得到所选色温。如果灯光已有颜色，色温会使它更冷（高于 6500K）或更暖（低于 6500K）。


该效果在完全饱和的单色灯光上不太明显。默认值为 6500K，在大多数显示器上应非常接近白色（sRGB 和 Rec 709 使用的 D65 标准照明体）。


Primary Visibility

默认情况下，灯光不会直接对相机可见。开启此项会使其可见，并且可使用发光标记 'O' 写入 LPE。此灯光在无限距离处显示为一个小圆。


Refine

Emission Focus

用于塑造发射的离轴余弦幂指数。它控制灯光的光照扩散范围。数值越高，灯光越会向中心聚焦，从而缩窄光照扩散范围。对于 PxrDistantLight 而言，提高该参数时，照明看起来会变暗。若要让阴影显得更大/更柔和，应使用 Angle Extent。


Emission Focus Tint

离轴灯光颜色色调。它会从灯光的离轴角方向朝中心方向，为衰减区域中的发射着色。默认情况下，色调颜色为黑色（无）。对于 PxrDistantLight 而言，这会为灯光颜色着色。更高的 Emission Focus 设置会增加所选颜色的饱和度。


Specular Amount

发出的镜面反射光数量。这是一个非物理控制。你可以使用一个 Specular Amount 为 0.0 的灯光，作为场景对象的纯漫反射光来源，并避免添加高光。此场景有细微反射，但你可以注意到灯光从最右侧铬球的反射中消失。


Diffuse Amount

发出的漫反射光数量。这是一个非物理控制。你可以使用一个 Diffuse Amount 为 0.0 的灯光，作为场景对象的纯高光来源。 



Shadows

如果灯光的 Trace Light Paths 在 Advanced 中启用，则所有阴影参数都会被忽略。下面的选项在脱离默认状态使用时都是非物理控制。如果 Trace Light Paths 启用，则会强制执行其物理属性。



Enable Shadows

启用光线追踪阴影。下图分别为开启（默认）和关闭。


Shadow Color

发出光所投射阴影的颜色。

.0 0.0 0.0

Shadow Distance

从被着色点位置开始计算的最大阴影距离。-1.0 表示未设置，此时使用被着色点与灯光上点之间的距离。你可以使用此控制人为缩短阴影投射距离。它也可能通过不计算该距离之外的阴影来提高渲染速度。这里用它移除了由灯光效果投射到后墙上的阴影。


Shadow Falloff

从灯光处开始发生阴影衰减的距离。-1.0 会关闭阴影衰减。它与 Shadow Max Distance 一起使用，为被 Shadow Max Distance 参数削弱或截断的阴影创建伪淡出。下面是两个参数一起使用且衰减增加的示例。注意，此参数可能影响复杂灯光设置中的性能。


Shadow Falloff Gamma

衰减区域中阴影强度的 gamma。这需要使用 Shadow Max Distance 和 Shadow Falloff。


Trace Subset

要纳入追踪阴影求交的几何体集合。如果未指定，则所有几何体都会纳入追踪阴影求交。


Don't Trace Subset

要忽略追踪阴影求交的几何体集合。如果未指定，则所有几何体都会用于追踪阴影求交。



Advanced


Normalize

 

启用 Normalize 时，随着你改变光源的 angleExtent，灯光对场景的贡献量不会改变。这让你更容易调整高光大小，而不改变场景照明。

 

Trace Light Paths

启用此灯光的光照和光子追踪。此值会强制使用基于物理的灯光，并且副作用是禁用上方的 Shadows 控制。用户在使用 PxrVCM 或 PxrUnified 积分器时，可以利用此功能选择哪些灯光发射光子。


Thin Shadow

为此灯光启用薄阴影并禁用折射焦散。如果 Trace Light Paths 已启用，此参数将被忽略。这是一个非物理控制，它会为透射对象创建“伪造的”彩色阴影，而不需要为焦散生成光子。下面从 On（默认）变为 Off（不透明阴影）。请注意，选择 Off 会失去彩色阴影和内部彩色反射，但会换来一些渲染速度提升。


Visible in Refraction

开启此项后，可让灯光通过玻璃和窗户等透射效果可见。


Manifold Walk

与 PxrUnified Integrator 配合使用，并启用 Enable Manifold Walk 以创建低成本焦散。注意：你必须禁用 Thin Shadows


Manifold Walk Exclude Group

与 PxrUnified Integrator 配合使用，并启用 Enable Manifold Walk, 此追踪集合会在流形探索期间被忽略，以避免阻挡流形探索光线或为其投射阴影


Light Samples

指定覆盖此光源要采用的灯光采样数量。如果设置为非零值，它会覆盖积分器执行的采样。如果此灯光产生难以解决的噪声并需要更多样本，你可能会用到它。


Light Group

指定用于灯光组 LPE 的灯光组名称。这对于生成逐灯光 AOV，供后续在合成中调整很有用。


Importance Multiplier

用户无需设置显式的 Light Samples，而可以通过更改此值来改变该灯光在内部被分配的样本数量。RenderMan 会在渲染时为场景中的所有灯光创建一组样本；更改此值会在各灯光之间重新平衡样本。注意，提高此值会使 更多 样本从此灯光中被选择，同时减少场景中其他灯光的样本。低于默认值则会 减少 此灯光的样本，并把更多样本分配给场景中的其他灯光。
