Cryptomatte 由 Psyop Studios 开发，用于生产管线。 当前版本为 Cryptomatte 1.2.0

 

采样滤镜的输出允许美术人员（基本上）自动生成 ID，供 Nuke 等合成软件包在选择和遮罩操作中用于指定对象或材质选择。你可以在 Katana 和 Maya 的 Cryptomatte 输出页面找到示例文件。下面展示的是一个包含茶壶的简单场景中 Cryptomatte 输出的对象结果和材质结果。

Parameters（参数）

Filename（文件名）

要写入的 EXR 图像文件名；默认值为 “cryptomatte.exr”。目前它需要写入与 beauty 或 denoise EXR 输出分离的单独文件。

Manifest（清单）

合法取值为 “None”、“Header”（默认）和 “Sidecar”。 Cryptomatte 格式允许使用一个可选的 JSON 格式清单，将名称映射到 ID。它可以作为字符串 metadata 字段包含在 EXR 头部，也可以存储在 sidecar 文件中，也可以省略。

Layer（层）

要使用哪个属性来识别对象并将对象分组。可设为 “Name”（默认）以使用对象的直接名称；设为 “Path” 以使用对象的完整 outliner 路径；设为 “Material” 以按着色器网络对对象分组；或设为 “Attribute” 以使用自定义用户字符串属性。这里的选择也会决定存储在 EXR 头部中的 metadata 字符串名称，以及 EXR 图像自身通道的名称。

“Attribute” 层目前在 XPU 中不受支持。

Attribute（属性）

如果 Layer 选择了 “Attribute”，此参数允许你指定要使用的自定义字符串 user attribute 名称。 

此参数目前在 XPU 中不受支持。

Levels

要包含多少个 ID/覆盖率对；默认值为 6。

Accuracy（精度）

计算但不存入文件的额外 ID/覆盖率对；默认值为 4。如果某个像素中贡献对象的数量没有超过 Levels 与 Accuracy 的总和，那么该像素计算得到的遮罩结果将是精确的。否则，遮罩结果将是近似的；增大此值可能会为像素中较不显著的对象生成更好的估计（到一定程度为止），但代价是更多内存。
