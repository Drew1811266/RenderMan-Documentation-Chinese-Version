PxrBump

Downloaded: 2026-06-15T03:49:12+00:00

PxrBump 用于为已置换的瓦片提供 secondary bump（次级凹凸）。



Bump mapping（凹凸贴图）是一种成本较低的方法，可在不建模细节、也不使用 displacement（置换）改变对象形状的情况下为对象添加细节。这是一种 shading “trick”（着色技巧），由提供的 pattern 产生细节错觉。

这会从 bump map（凹凸贴图，即 scalar displacement map（标量置换贴图），因此只使用第一个通道进行置换）生成 bumped normal（凹凸法线）。凹凸会应用在 object（对象） space（空间）中。 object（对象） space（空间）与 current（当前） space（空间）之间的缩放变化会由该节点自动应用。

Texture atlas format（纹理图集格式）文件可以按 UDIM (Mari) 格式或 Mudbox 格式读取。此插件也支持 Rtx procedural texture plugins（Rtx 程序化纹理插件）。


Input Parameters（输入参数）

Scale（缩放）

应用到凹凸的 object space（对象空间）Scale（缩放）。

Disabled（禁用）

关闭凹凸效果，便于调试。改用 geometric normal（几何法线）。

Input Bump（输入凹凸）

如果 Input Bump（输入凹凸） 已连接（并且 Filename（文件名） 未指定），pattern 会执行连接到 Input Bump（输入凹凸） 的上游 pattern network 三次：一次在 shading point（着色点），一次在 P+du 以及 P+dv 处。然后对结果做叉乘，以获得 bump normal（凹凸法线）。

注意，如果你使用固定 texture（纹理）和 UV，强烈建议使用下面的 Filename（文件名） 参数，因为它会快三倍！

如果 Filename（文件名） 已指定， Input Bump（输入凹凸） 会被忽略。

Filename（文件名）

纹理的 filename（文件名）。如果未提供， Input Bump（输入凹凸） 会被改用。

如果 Filename（文件名） 已指定时，pattern 没有连接上游 graph。它会改为从纹理文件中读取 bump value（凹凸值）。输入文件可以是 color（彩色）或 monochrome（单色）文件；如果文件是 RGB，则只会读取单个通道（由 First Channel Offset（第一通道偏移） 参数指定）。bump pattern（凹凸图案）会在过滤纹理的同时从纹理取得 derivatives（导数），这与使用 Input Bump（输入凹凸）相比具有性能优势。

PxrBump pattern 目前不允许修改纹理的 manifold（流形）；请使用下面的控制项。

Bump Manifold（凹凸流形）

此项接受来自 PxrBumpManifold2D


Advanced Texture Settings（高级纹理设置）

MIPBias

为渲染选择的 MIP selection（MIP 选择）添加偏移，可选择更高分辨率（负整数）或更低分辨率（正整数）。

MaxResolution

将 MIP selection（MIP 选择）限制到首选的最高级别。根据你的选择，这可能会阻止最高分辨率被渲染。

 

Advanced（高级）

Input Normal（输入法线）

要添加到 bump normal（凹凸法线）上的额外法线。

Reverse Normal（反转法线）

反转结果法线。

Adjust Amount（调整量）

当 geometric normals（几何法线）背向摄像机时，用于调整法线的量。当我们在法线中看到裂缝时，这很有用。你可以在 此处找到示例。

Surface Normal Mix（表面法线混合）

将结果法线与原始 surface normals（表面法线）混合的量。趋近 1.0 时会混入更多表面法线。你可以在 此处找到示例。

Output Parameters（输出参数）

resultN

bumped normal（凹凸法线）。
