为任务选择合适的工具

一般来说， 细分曲面 有很多优势，但有时确实需要使用多边形网格。

细分曲面

  • 优点

    • 在任意分辨率下都能保持完全平滑。

    • 可以通过折痕和角点实现锐利或半锐利特征。

    • 内存占用可以非常轻量。

    • 置换裂缝更少（水密）。

  • 缺点

    • 需要不同的建模技术/技能。

    • 需要标记锐利边和顶点。

    • 存在一些拓扑约束（四边形或三角形，不能有非流形几何体）。

多边形网格

  • 优点

    • 非常适合 3D 扫描网格这类重型几何体。

    • 最适合会被破碎或模拟的几何体。

    • 适合只有锐利边且不需要置换的模型。

    • 如果不进行置换，就不会生成微多边形。

  • 缺点

    • 重型模型会占用大量内存，无论它们在图像中看起来多小。

    • 在低分辨率的平滑对象上可能会看到阴影终止线。

    • 可能会看到置换裂缝。

NURBS

  • 优点

    • 在任意分辨率下都能保持完全平滑。

    • 内存占用轻量。

  • 缺点

    • 只能使用四边形建模。

    • XPU 不支持。

    • 如今已不太常用。

推荐阅读：

  • 几何体

  • 细分曲面

  • 多边形

细分建模

  • 始终尽可能建模出 最简单的控制笼 。

  • 始终以平滑方式查看模型，或使用 PxrVisualizer。

    • 甚至可以在实时 PxrVisualizer 渲染中进行建模！

  • 使用折痕来制作锐利或半锐利边，而无需添加额外的边循环。

细分折痕演示

我们选择了一个二十面体的所有边，并将它们的折痕值从 0.0 动画到 5.0。

所有边都可以具有不同的折痕值，并且仍能在最终曲面上保持平滑过渡。

  • 角点对顶点的作用类似于折痕。

细分角点演示

在这段动画中，一个三角形面的 3 条边具有 5.0 的折痕值。

我们将该三角形顶点的角点值从 0 提高到 5.0。

请注意收紧区域周围非常平滑的过渡。

  • 细分曲面有 3 种类型。

    • Catmull-Clark 是最灵活也最常用的一种。

    • Loop 是基于三角形的细分曲面。

      • 没有折痕或角点。

      • 适用于布料模拟。

    • Bilinear 类似于在所有边上具有无限硬折痕的 Catmull-Clark。

镶嵌细分

如果你来自需要手动设置细分级别的其他渲染器，可以忘掉那套做法： 曲面的镶嵌细分是自动的， 在 RenderMan 中（我们称之为 切分）。

主要控制项是 微多边形长度。

  • 默认情况下，它以像素表示；micropolygonlength 为 1（默认值）意味着每个微多边形大约为 1 像素宽。

  • 如果不希望镶嵌细分动态变化，可以将它设置在对象空间或世界空间中，或者定义一个切分相机。

除非看到伪影，否则建议禁用 “raster-oriented dicing”。

细分与多边形切分演示

我们比较了一个带折痕和置换的 Catmull-Clark 细分曲面与一个等效的多边形网格。

增加微多边形长度等同于让对象在图像中的尺寸变小。

  • 当对象很大时，它会被精细镶嵌细分为数百万个微多边形，从而显示细致的置换细节。

  • 当对象变得小于一个像素时，只会保留基础网格。

微多边形长度 有点像一种“自动细节层级”；当对象在屏幕上小到不需要完整细节表示时，它可以节省内存。

请注意， 细分曲面会退化为少得多的面，从而节省内存并加快光线追踪。

这就是所谓的 数据放大：紧凑的几何表示可以按需扩增，以创建更多细节。

模型细节

通常，更好的做法是依靠置换和凹凸贴图来添加细节。

  • 置换可以添加较大的几何特征，这些特征会影响对象的轮廓和投影。

    • 置换会影响首个像素时间（微多边形生成），但不会拖慢着色。

  • 凹凸贴图或法线贴图会添加小尺度细节。

    • 凹凸贴图会在着色期间增加少量成本（每次光线命中时都会重新计算）。

    • 请注意，如果入射光线过于“模糊”，我们的凹凸贴图节点不会计算 bump；这意味着没有 bump 也不会被注意到。

如果你在 ZBrush 中建模了一个高度详细的对象，应考虑：

  • 重新拓扑以创建轻量模型

  • 从高分辨率网格中提取置换贴图。

我们的置换节点具有一个 “model displacement” 输入，用于连接提取出的置换贴图。 

内存使用与性能

多边形网格

  • 如果没有置换，则 首个像素时间 （TTFP）最短。

  • 进行置换时，网格会被切分并生成微多边形。

  • 未置换的重型网格会使用相同数量的 RAM，无论它是全屏显示还是只有 2 像素高。

细分曲面

  • TTFP 略长，但切分是多线程的，且效率很高。

  • 在顶点数量相同的情况下，会比网格多使用一点内存。

  • 稀疏建模的细分曲面在只有 2 像素高时，只会包含基础网格面。

    • 这种自适应性会降低内存使用并加快光线追踪。
