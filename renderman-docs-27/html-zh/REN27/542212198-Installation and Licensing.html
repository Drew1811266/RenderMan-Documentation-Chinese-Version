系统要求

硬件

RenderMan RIS 需要能够运行 AVX 指令集或更高版本指令集的 CPU。建议 RIS 至少配备 8GB RAM。 

RenderMan XPU 需要能够运行 AVX 指令集的 CPU。RenderMan XPU 支持 Linux 和 Windows 系统。有关支持的显卡、VRAM 要求以及驱动要求的更多信息，请参阅 XPU 技术规格。

若要在 “it” 中使用 NVIDIA Optix AI Denoiser，需要支持 Optix 7 的硬件，即 NVIDIA “Pascal” 架构或更新架构。

支持的操作系统

Linux：EL9（RHEL 9.x、AlmaLinux 9.x、Rocky Linux 9.x）

Windows：Windows 10 和 11

macOS：Sequoia（15.x）

在运行 RenderMan 之前，请访问你所用桥接产品的安装页面，确保已完成正确设置。请注意，Houdini 和 Katana 等产品可能需要手动配置环境文件。

桥接产品

Katana：7.0、7.5、8.0、8.5 和 9.0

Maya：2024、2025、2026

Blender：3.6 到 5.0。

Houdini：20.0、20.5、21.0

RenderMan for Houdini（RfH）仅支持以下生产版 Houdini 版本：

  • RenderMan for Houdini 27.2 支持：20.0.896、20.5.684、21.0.512、21.0.559、21.0.596

其他重要信息：

  • 如果 Python 版本不匹配，RenderMan for Houdini 将无法运行。请务必为 Houdini 和 RfH 安装相同的 Python 版本。 

  • Houdini Apprentice 版本无法使用 RenderMan 进行渲染；请查看 SideFX 网站，了解允许 Third Party Rendering 的选项。

构建插件和工具

RenderMan 插件：自定义 RenderMan 插件需要使用与 RenderMan 所支持的 VFX Reference Platform RenderMan 所支持的版本中指定的编译器来构建。请参阅 RenderMan 发行说明 了解本版本所支持的 VFX Reference Platform 版本。

在 Linux 上，RtxSVG.so 插件要求安装 compat-expat1 软件包。


使用浮动许可证安装 RenderMan

Pixar 的 RenderMan 最好通过集成安装程序安装。使用你的论坛登录名和密码，以及购买时获得的序列号，安装程序可以自动下载许可证，并同时安装许可证服务器软件和 RenderMan 软件。

你的网络中只需要一台机器安装 Pixar License Server，该机器也可以是你安装应用程序和插件的同一台机器。不过，运行许可证服务器的机器不需要安装 RenderMan 应用程序。 首先 在你想指定为许可证服务器的机器上运行安装程序，然后再在其他工作站上运行安装程序。

使用节点锁定许可证安装 RenderMan

Pixar 的 RenderMan 以单一安装程序交付。节点锁定许可证的安装步骤与浮动许可证相同，但无需安装或运行浮动许可证服务器。

使用你的论坛登录名，安装程序可以自动下载许可证，并安装你的 RenderMan Software 节点锁定授权副本。


大型站点安装


RenderMan 安装过程是自动化的，默认安装到系统驱动器。安装程序还会自动设置其他系统变量，以支持：

  • 由 Autodesk Maya 等 DCC 应用程序发现插件

  • 使用 RenderMan 可执行文件进行命令行渲染

如果选择非标准安装位置，你可能需要自行定制这些环境变量才能完成安装。


虽然也许可以在不同操作系统，甚至在受支持操作系统的其他版本上安装和使用 RenderMan， 但并不推荐这样做。这可能不稳定，也会带来许多麻烦；因此，为了所有相关人员的正常工作体验，这种用法不受支持。
