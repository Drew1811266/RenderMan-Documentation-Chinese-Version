Stylized Hatching

Source: https://rmanwiki-27.pixar.com/space/REN27/542221833/Stylized+Hatching
Downloaded: 2026-06-15T03:49:12+00:00

Stylized Hatching（风格化排线）主要发生在一个 DisplayFilter（显示滤镜）节点中，并可通过 PxrStylizedControl 获取逐对象的可选信息。

它可以单独用于简单、酷炫的风格，也可以与其他 Hatching（排线）节点或 Toon & Lines（卡通与线条）节点组合（串联）。将 Hatching 与其他 Stylized 节点串联时，理想顺序是先 Hatching，再把 Lines 叠在上面。你可以组合多个 Hatching，并使用节点上的 Mask/Input Mask 功能，根据不同输入信号（光源、摄像机范围，等等）把它们遮罩在一起。

Result（结果）

  • Disable（禁用） - 关闭此节点；如果前面已连接另一个 DisplayFilter，则显示该串联链之前的 beauty（主图像），否则在没有其他 DisplayFilter 节点连接时显示常规 beauty 渲染。

  • Hatching On Color - 忽略 base color 的 albedo color，改为对所有对象使用给定颜色。

    • Hatch On Color - 对所有对象使用此颜色。
    • Hatch On Background - 使用背景颜色或纹理作为对象的 albedo color。如果有运动，纹理会游移。

    • Add Spec - 将 specular contribution 添加到排线

  • Hatching Comp - 自动合成，将排线添加到上一个 beauty（主图像）
  • Hatching Blend - 灰度显示，仅显示排线纹理混合（如果显示为红色，说明排线纹理需要是 3 色 RGB，而不是单通道 float）
  • Signal Luminance - 用于将颜色转换为灰度的正确 RGB 权重；目前未在任何模式中使用，只作为可视化器
  • Signal Energy - 此节点创建并用于混合排线的内部白色着色器通道

  • Triplanar Test Colors - 使用纯色预览三平面混合

Input Page

Signal

用于驱动排线 Signal（信号）的 color AOV 名称。通常是 diffuse；也可以在这里放入任何 light AOV。此 AOV 必须在 Maya/Katana/Houdini 中设置，否则 DisplayFilter 会退出。

Mode

在底层，会基于 Signal AOV 创建一个白色着色器近似来驱动 Hatching。
mono - 单通道 
average - Signal AOV 的 RGB 分量平均值
Div Albedo AOV - 用 NPRalbedo AOV 或 albedo AOV 中的纯 albedo，去除 Signal AOV 光照结果中的 albedo 分量
Div NPRalbedo AOV - 用 NPRalbedo AOV 或 albedo AOV 中的纯 albedo，去除 Signal AOV 光照结果中的 albedo 分量。必须在 PxrStylizedControl Pattern 节点中设置 Albedo Color。

Hatching Projection Page

Frequency

数值越高，频率越高。对于 Screen 从 1 开始。对于 Triplanar 从 0.005 开始。

Projection

Screen - 使用 “Screen” 或 “NDC” 坐标来投影 Hatching 纹理。动画和摄像机移动时纹理会游移。非常适合静帧或不运动的几何体。

Triplanar - 从 NPRPtriplanar 与 NPRNtriplanar AOV 创建三平面投影（由 PxrStylizedControl 写出）。

NPRtexture Coords AOV - 使用 NPRtextureCoords AOV 的 RED 与 GREEN 通道中的 XY 坐标（在 PxrStylizedControl 中设置）。这便于用户设置任何自定义坐标系统，例如摄像机投影。

Flip

使用 Triplanar Projection（三平面投影）时，在 “t” 方向翻转纹理

Triplanar P

NPRPtriplanar - 使用 NPRPtriplanar AOV（由 PxrStylizedControl 设置）。默认情况下这是对象空间中的 P。
P - 使用 factory P AOV
使用 NPRPtriplanar AOV（由 PxrStylizedControl 设置）。默认情况下这是对象空间中的 P。 使用 __Pref AOV。必须在 DCC（Maya/Houdini/Katana）中作为 “rest position” 设置，并创建该 AOV。
aov - 要用于 P 的 triplanar（三平面）混合的自定义 color AOV 名称。如果设置为 “NPRPtriplanar”：需要将 PxrManifold3d 插入 PxrStylizedControl 节点，以提供自定义坐标系统。也可以是 factory “P”、“__Pref”、“Pworld”，前提是这些 AOV 已设置。

Triplanar P String

条件可见性：当 Triplanar P 设为 aov时，P 的 triplanar（三平面）混合使用的 color AOV 名称。如果设置为 “NPRPtriplanar”：需要将 PxrManifold3d 插入 PxrStylizedControl 节点，以提供自定义坐标系统。也可以是 factory “P”、“__Pref”、“Pworld”，前提是这些 AOV 已设置。

Triplanar N

NPRNtriplanar - 使用 NPRNtriplanar AOV（由 PxrStylizedControl 设置）
Nn - 使用 factory surface normal N AOV
__Nref - 使用 __Nref AOV。必须在 DCC（Maya/Houdini/Katana）中作为 “rest position” 设置，并创建该 AOV。
aov - 要用于 N 的 triplanar（三平面）混合的自定义 color AOV 名称。如果设置为 “NPRNtriplanar”：需要将 PxrManifold3d 插入 PxrStylizedControl 节点，以提供自定义坐标系统。也可以是 factory “N”、“__Nref”，前提是这些 AOV 已设置。

Triplanar N String

条件可见性：当 Triplanar N 设为 aov时，N 的 triplanar（三平面）混合使用的 color AOV 名称。如果设置为 “NPRNtriplanar”：需要将 PxrManifold3d 插入 PxrStylizedControl 节点，以提供自定义坐标系统。也可以是 factory “N”、“__Nref”，前提是这些 AOV 已设置。


在上方视频中，左侧列 N 显示由于对象变形，纹理在 triplanar Normal mask（三平面法线遮罩）中游移。右侧列 __Nref 显示变形几何体表面上的原始 N，因此即使面朝向不同方向，纹理仍保持一致。排线纹理只会在对象进入或离开阴影时发生变化，因为排线基于光照。

Triplanar Blending

值为 0 表示没有混合，逐渐到 1 表示完全混合。

Mode

1 value - 所有排线纹理使用 1 个频率。使用 Frequency 上方参数。
8 values - 8 个不同频率，分别用于每个单独的排线纹理，以获得自定义结果
Camdist Range - 使用摄像机距离驱动排线纹理的频率
Light Signal Range - 使用 Input/Signal 驱动排线的频率

Hatching Freq Tex 1 - Tex 8

条件可见性：当 Mode = 8 values时，纹理 1-8 的排线频率。每个纹理都可以有自己的频率。对于 Screen 从 1 开始。对于 Triplanar 从 0.005 开始。

Hatch Tex Color Mix

从预定义集合中选择一组协调的排线纹理。也可以指定你自己的自定义纹理集目录名（会展开为 dirname/dirname-01.tex 等）。或者分别选择 8 个单独纹理。

Hatching Color Page

Color From

选择哪个 AOV 决定排线的着色

Color

排线颜色，当 Color From 为 从预定义集合中选择一组协调的排线纹理。也可以指定你自己的自定义纹理集目录名（会展开为 dirname/dirname-01.tex 等）。或者分别选择 8 个单独纹理。

Hatch On Color

用于在纹理背后承载排线的底色 Color。仅当 Result（结果） 设为 Hatch On Color。

Hatching Canvas Page

Enable

开启 Canvas mode（画布模式）的开关

Canvas Color

Alpha 外部的背景或 “paper”（纸张）颜色

Canvas Texture

提供背景 “paper”（纸张）纹理。如果留空，则自动使用 Canvas Color。

Hatching Textures Page

Type

Standard - 从多个随附预设中选择排线
Custom - 提供你自己的包含 8 个纹理的目录（必须命名为 pxrStylized-customname-01.tex）。示例请见： C:\Program Files\Pixar\RenderManProServer-25.0\lib\textures\StylizedLooks\
Individual - 提供你自己的独立纹理，使用 .tex 格式

Standard（标准）-

从多个预设中选择排线

Custom（自定义）-

自动用平面颜色替换纹理，以获得卡通赛璐璐外观。仍使用所有混合重映射设置来分布平面颜色。忽略纹理。

Progressive

如果纹理最初不是这样创建的，则把纹理彼此叠加合成（每个纹理会逐步添加到前一个纹理上）。在使用 8-val Individual（单独）-

Invert

反转排线纹理查找的渐进顺序。对于 diffuse 信号，通常保持关闭（默认）。

Blending Page

Color Blend Signal

用 RED、ORANGE、YELLOW、GREEN、BLUE、INDIGO、VIOLET、TURQUOISE 替换 tex1-tex8，用于预览混合

Blending Mode

在 Mix 与 Weighted 混合之间选择。Mix 会在所有纹理之间自动混合。Weighted 使用自定义加权函数，并带有 Blending Hold 和 Blending Overlap

First Texture is White

在最亮的纹理之前， tex1，添加纯白色

Last Texture is Black

在最暗的纹理之后， tex8，添加纯黑色

Hold

大于 0 的值会让每个纹理更加突出

Overlap

Bell Curve 加权函数的 Overlap。0.125 是最小值，会混合 2 个纹理；更高的值会混合更多纹理，最多 8 个。

Output Page

AOV

此特定 Hatching 节点使用的 color AOV 名称。默认写入 GREEN 通道，位于 NPRhatchOut；用于合成时的额外 AOV 支持。

Channel

写入灰度排线输出的通道

Mask Page

Light Mask Switch

使用上面 signal_aov_string 中定义的 light AOV，作为先前串联的 Stylized shaders 和/或基于物理的照明的遮罩

Alpha Hatch Switch

使用 Light Mask 时，排线的黑色区域对先前串联的层或基于物理的层透明

NPR Mix

0：禁用此 Hatching Node；如果已串联，则显示前一个节点，否则在没有前一个 Stylized Looks 节点时显示原始 beauty（主图像）。1：显示此节点的完整 Hatching。

Compositing Page（合成页面）

Mix

0：禁用此 Hatching Node；如果已串联，则显示前一个节点，否则在没有前一个 Stylized Looks 节点时显示原始 beauty（主图像）。1：显示此节点的完整 Hatching。

Gamma

应用到最终排线的 Gamma；较小数值会提高对比度。可尝试 0.5。
