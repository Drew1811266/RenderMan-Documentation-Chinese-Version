欢迎使用 RenderMan 27.0

RenderMan 27 是十多年来最重要的一次发布，带来了突破性的性能、交互性和创作控制能力。凭借 XPU 的最终帧能力、深度合成工作流以及扩展的着色功能，Pixar 继续重新定义面向高端动画长片和 VFX 的渲染技术。以下是本次发布的一些亮点：

XPU 改进

  • 最终帧渲染 - XPU 架构现在支持最终帧渲染，使美术人员能够同时利用 CPU 和 GPU 资源，高效生成高质量图像。这一根本性转变提升了渲染的交互性和可扩展性，使 XPU 成为未来 RenderMan 开发的核心。

  • 交互式降噪 - 交互式 denoiser（降噪器）显著缩短了渲染会话中的迭代时间。对于处理复杂灯光和着色设置的美术人员来说，此功能尤其有价值：它能在保持最终图像保真度的同时提供快速反馈。

  • 检查点 - checkpointing（检查点）的引入使美术人员能够将部分帧作为检查点输出。此功能减少了重新渲染所耗费的时间，并提高了制作效率。

  • 合成 - 在现有 LPE（Light Path Expressions，光路表达式）支持的基础上，XPU 现在包含视觉特效所需的更多合成能力，例如 mattes（遮罩）、holdouts（遮挡/挖空）以及更丰富的 AOV 选项；Cryptomatte 将在后续小版本中到来。这显著改进了合成管线，并为图像组装提供精细控制。

  • 深度工作流 - XPU 现在支持 deep data（深度数据）工作流，可对复杂体积和透明元素进行精确合成。OpenEXR Deep IDs 已完整实现，提供更顺畅的数据提取和合成灵活性。

  • Stylized Looks - Stylized Looks™ 工具集不仅已在 XPU 中完整实现，而且大幅扩展，可实现一系列非照片级真实感效果，例如绘画风格、toon shading（卡通着色）和素描式渲染。这些能力扩展了美术人员的创作可能性，使他们能够突破传统照片级真实感的审美边界。

  • OSL 显示与采样过滤器 - Open Shading Language（OSL）显示过滤器和采样过滤器已集成到 XPU 中，提供更强的着色自定义能力、更好的艺术控制，以及更高效的制作工作流。

  • 网格灯 - XPU 现在提供完整的 mesh light（网格灯）支持，可使用物理灯光类型，并大幅提升灯光效果在几何形态上的灵活性。

  • MaterialX Lama - MaterialX Lama 作为 Early Access（抢先体验）在 XPU 中提供，通过 Industrial Light & Magic 的着色工具集强化美术工作流。它将在 27 发布周期内获得完整支持。

  • 更多 Lookdev 功能 - 嵌套实例化、OSL 改进（trace、getattribute）、内部体积聚合，以及玻璃和单次散射增强，使 XPU 与 RIS 更加接近，扩展了外观开发能力，并确保不同渲染架构之间的一致性。

  • 多 GPU 支持 - 借助多 GPU 支持，XPU 现在可以在 CPU 旁同时利用多个 GPU，从而优化硬件利用率并调动所有渲染资源。

集成与管线

  • Solaris - 扩展的 Solaris 支持、UI 改进，以及更深入的管线集成。

  • DCC 支持 - 与最新数字内容创作工具完全兼容，确保对领先行业标准提供稳健支持。

  • Apple Silicon - RenderMan RIS 将在后续小版本中于 macOS 上原生支持 ARM 处理器。

其他功能

  • 符合 VFX Reference Platform 2024 - RenderMan 27 与 VFX Reference Platform 2024 保持一致，确保与行业标准制作管线的兼容性、稳定性和集成能力。

RIS 更新 

  • 支持与维护 - 虽然 RIS 在 RenderMan 27 中仍可使用，但未来版本将会弃用；同时鼓励工作室在新项目中采用 XPU。

技术增强

桥接产品

  • Texture Manager 现在默认使用 OpenImageIO 进行 mipmapped 纹理转换，从而大幅缩短处理时间。这一选择以行业标准 OpenEXR 格式取代了 Pixar 纹理格式。Pixar 的纹理转换工具 txmake 会继续随附，以保持兼容性。为了让纹理在 RenderMan 中最有效地工作，你需要在 maketx 中使用特定选项。详情请参阅 从 txmake 切换到 OpenImageIO。

光线追踪改进

  • 相对像素方差和最小采样数 现在已在 XPU 中按对象实现，可提供更准确的自适应采样。

  • 针对 GPU 光线追踪精度的修复，确保在极端平行光线追踪条件下正确处理几何体。

几何与着色增强

  • 嵌套实例化现在已获得完整支持，可在实例之间启用材质继承和属性传播。

  • 次表面散射已得到改进，噪声更低，并且对具有混合 lobe 的表面提供更好的重要性采样，从而确保更快收敛。 

  • 玻璃和体积单次散射改进，提供更准确的折射、阴影和消光效果。

  • 新的 OSL trace 能力，包括与 PxrDirt 和其他程序化着色技术更好的交互。

  • 毛发和绒毛渲染改进，包括 PxrChiangHair 的 glow color 和 gain 控制。由于支持 Min Width，极细毛发的收敛性得到了显著改善。

  • 细分网格的法线改进，尤其是在半锐折痕附近。

体积与照明

  • 内部体积聚合现在已获得完整支持，使 XPU 可高效渲染浑浊液体或半透明晶体等高度复杂的材质。

  • XPU 中的 Light filters（灯光过滤器） 解决了特定 falloff 和颜色 spline 配置的问题。

  • PxrPathTracer 增强，包括对 clampDepth 和 clampLuminance 参数的支持。

图像输出与 AOV

  • XPU 中的 Deep OpenEXR 支持，包括自动生成 OpenEXR 3.0 风格的 ID 清单。

  • 扩展的 deep data（深度数据）控制，允许美术人员直接配置深度阴影误差参数。

  • Matte object（遮罩对象）支持和 holdout 工作流，增强多通道渲染中的合成与集成。

  • 在包含许多 AOV 和/或大量使用 Ptex 的场景中， 交互性能已大幅提升，尤其是在旋转相机视图时。

降噪与性能优化

  • denoiser 现在会输出基本计时统计，并且可更灵活地配置以适应现代管线。

  • XPU 的内存分配改进，可动态调整纹理缓存大小以高效使用资源。

重要说明

  • 使用小写 <udim> 的路径会丢失纹理；请将路径更新为大写 <UDIM>，以符合 USD 合规性。Maya 和 Blender 会自动升级路径。RfH 现在提供一个菜单脚本，可手动调用以执行此转换。

  • 由于 MaterialX Lama 在 XPU 中仍处于 Early Access（抢先体验），在 27.0 初始版本中，相比 RIS 预计会出现一些明显的外观差异。请参阅 XPU 功能与限制 以了解预期差异的详细说明。欢迎你在 Early Access 期间提供反馈，我们会持续改进。

深入技术发布说明

XPU

General（常规）

  • XPU 的最低驱动要求已经变更。驱动必须支持 CUDA 12.8.1。

  • XPU 现在支持 XPUGPU 和 XPU 模式下的多 GPU。正在渲染的场景必须能装入每个 GPU 的内存，并且这些 GPU 应当相同。RMAN-15205

  • 为 XPU 中的异步 exitAt 和 checkpoint interval（检查点间隔）添加了支持。RMAN-22620

  • 修复了 XPU 中的一个 bug：小幅编辑过去可能需要很长时间处理（每次相机移动都超过 1 秒）。PRMAN-3079

  • 显示更新速度已大幅提升，尤其是在包含许多 AOV 的场景进行交互式渲染时。RMAN-21859

  • XPU 现在会遵循 RMAN_DUMP_FILELOG 环境变量。

光线追踪

  • XPU 自适应采样已基于 Pixar Animation Studios 的技术得到改进，可更有效地配合机器学习 denoiser（降噪器）并缩短渲染时间。这可能会使未降噪图像比以前包含更多 fireflies（亮点噪声），但 ML denoiser 很擅长移除这类伪影。此外，对于大多数像素样本为黑色的特别困难采样场景，自适应采样也已改进。PRMAN-3340、PRMAN-3004

  • Attribute "shade" "relativepixelvariance" 和 "minsamples" 现在已在 XPU 中实现。RMAN-22492

  • "relativevariance" 自适应度量现在已在 XPU 中实现。与 "variance" 自适应度量一样，如果没有显式指定 variance channel（方差通道），系统会自动创建一个或多个方差通道（取决于 "adaptall" 参数）。RMAN-22556

  • 修复了 GPU 上的一个问题：当光线与 Z 轴完全平行时，过去可能会漏掉几何体。

  • 基于 "mse" AOV 通道的自适应采样现在已在 XPU 中实现。

  • Hider 选项 "extrememotiondof" 现在已在 XPU 中实现，与 RIS 类似。当其值为 0（默认值）时，会降低景深噪声。当其值为 1 时，会使用理论上更正确但噪声更高的景深采样。RMAN-23974

图像输出和显示

  • XPU 现在完整支持 display filters（显示过滤器）。通过 OSL 可使用等价于 Read、ReadRegion 和 Write 的 API：

    • Read 等价项： texture(AOVName, u, v)；

    • ReadRegion 等价项： texture(AOVName, x, y, "subimage", 1)；

    • Write 等价项： Ci += debug(AOVName) * rgb；

  • XPU 部分支持 sample filters（采样过滤器）。Read 和 Write 受支持，而来自 sample filters 的 ReadRegion、primvar 访问和 trace() 尚不受支持。 

  • XPU 完整支持连接到 sample filters 和 display filters 的 OSL patterns。RMAN-19911

  • XPU 现在支持深度输出。RMAN-22453

    • 新增了 Option "deep" "int maxsamples"，用于控制每个子像素位置保留的最大深度样本数。将此值从默认值 1 提高，并同时修改 Option "limits" "deepshadowerror"，即可直接控制 deep output（深度输出）在内存消耗、磁盘空间和深度精度之间的权衡。目前此选项仅适用于 XPU。RMAN-21297

    • RenderMan XPU 现在完整支持 OpenEXR Deep IDs，包括自动生成 OpenEXR 3.0 风格 ID 清单的能力，其 metadata 类型为 "CompressedIDManifest"，用于 model ids。若要使用此功能，必须向渲染器提供以下内容：

      • 必须将名为 "int id" 和 "int id2" 的两个 display channels 传递给 deep display driver；

      • 几何实例必须具有 Attribute "identifier" "string name"、"int id" 和 "int id2"

      如果满足所有这些条件，display driver 会自动创建一个压缩 ID 清单，其中包含 "model.id0" 和 "model.id1" 通道，并使用 hashScheme "custom"（因为假定 DCC 已经处理了从 id0/id1 表示的 64 位值到名称的哈希方案）。"id" 和 "id2" 数据通道将被重命名为
      "model.id0" 和 "model.id1"，并且类型为 UINT。更多信息请参阅：
      https://openexr.com/en/latest/DeepIDsSpecification.html

      deep utility（深度工具）dsview 现在会识别这些 deep ID manifests，并且在打印或表示 pixel function（像素函数）时，自动显示深度样本的模型名称
      在 pixel window（像素窗口）中。

      新增了 deepidextract 工具，可用于只选择 model name（模型名称）与字符串匹配的深度数据。它类似于 OpenEXR 文档中的 "deepidselect" 示例
      ，该示例来自 OpenEXR 文档；但它完整支持用于匹配字符串的正则表达式。

  • XPU 现在支持输出 OpenEXR 图像的完整 metadata，包括 data 和 display windows（数据窗口和显示窗口）。RMAN-21507

  • XPU 现在支持 cpuTime builtin AOV（内建 AOV）。PRMAN-3023

  • XPU 现在支持 PxrSphereCamera。PRMAN-2942

  • XPU 现在支持 matte geometry（遮罩几何体）。RMAN-21289

  • XPU 现在支持 holdout 工作流（__illumholdout light groups 除外）。RMAN-21290

  • XPU 现在支持 shadows 和 invshadows lpe prefix。RMAN-22350

  • PxrAovLights 现在在 XPU 中受支持。RMAN-19950

  • 修复了在没有显式 "z" 或 "Z" Display 时，XPU 中 DisplayChannel "id" "filter" "zmin" 的功能。

  • 修复了使用 PxrDisplayFilterCombiner 或 PxrSampleFilterCombiner 时出现的虚假错误。

  • XPU 现在支持对 AOV 的动态修改。因此，可以在交互式环境中启用和禁用交互式 denoiser（降噪器）。RMAN-19414
