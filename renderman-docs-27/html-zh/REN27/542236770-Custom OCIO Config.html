使用自己的 OCIO 配置

OCIO Roles（OCIO 角色）

RenderMan 依赖几个 OpenColorIO 角色，需要在你的 config.ocio 文件中定义。

  • rendering： MANDATORY（必需）

    • 这是渲染器执行所有计算时所在的颜色空间。 

  • data： STANDARD（标准）

    • 这类输入不需要转换，因为它提供的是数值数据（法线贴图、凹凸贴图、粗糙度贴图等）。 

  • srgb_linear： MANDATORY（必需）

    • 线性 sRGB/Rec.709 图像，例如 HDRI 环境贴图。 

  • srgb_texture： 

    • 非线性 sRGB 图像，例如 JPEG、PNG 等。

      • 这个角色已添加到我们的 basic linear 配置和 Filmic-Blender 配置中。

      • 在 ACES-1.2 中，它已经是以下项的别名： Utility - sRGB - Texture。

实际使用中，这 4 个角色会满足大多数人有效使用色彩管理的需求。不过，仍然可以通过 JSON 配置文件添加更多角色。

JSON Configuration file（JSON 配置文件）

许多配置包含大量颜色空间，其中很多面向与 3D 渲染无关的特殊情况。对 3D 艺术家真正重要的内容可以归结为少数几个控件。

为了提供集中配置点，RenderMan 需要一个 JSON 配置文件来提供：

  • UI 控件中可用的 Roles/ColorSpaces 列表。

  • 将插入到纹理名称中、用于消除歧义的 aliases（别名/缩写形式）列表。

  • 可显示在 UI 控件中的 help strings（帮助字符串）列表。

  • 一组供 Texture Manager 分配初始 txmake 设置的规则。

File name（文件名）

JSON 文件的名称基于包含 config.ocio 文件的目录：

$RMANTREE/lib/ocio/basic/config.ocio  →  rman_color_config_basic.json
$RMANTREE/lib/ocio/ACES-1.2/config.ocio  →  rman_color_config_ACES-1.2.json

File location（文件位置）

启动时，位于 $RMANTREE/etc 中的文件总是会被读取。

如果定义了 $RMAN_COLOR_CONFIG_DIR environment variable（环境变量），则所有匹配 rman_color_config_*.json pattern（模式）、位于该目录中的文件都会被加载。这个机制可用于覆盖 stock configurations（内置配置），或为自定义 OpenColorIO 配置添加支持。

Rules（规则）

规则允许 Texture Manager 为新引用的图像设置合理的转换默认值。如果设置得当，它可以完全自动化纹理转换。

规则由节点类型定义：pattern、light、lightfilter 和 imageplane。这样可以专门化转换，而无需依赖一套复杂规则。

args

这是一个 txmake flags（txmake 标志）及其值的 dictionary（字典），用来定义可由规则修改的 baseline（基线）。

带有 null 值为 null 的规则会被禁用。

texture_type

"regular" 用于通用纹理，"envlatl" 用于环境光纹理。

texture_format

纹理的文件格式："pixar"、"tiff"、"openexr"

resize

使用给定模式将图像 resize（调整尺寸）到 2 的幂："none"、"up"、"up-"、"down"、"down-"、"round"、"round-"。

texture_filter

用于 resize（调整尺寸）的滤镜："point"、"box"、"triangle"、"sinc"、"gaussian"、"gaussian-soft"、"catmull-rom"、"mitchell"、"cubic"、"lanczos"、"bessel"、"blackman-harris"

smode

s 方向上的重复模式："black"、"clamp"、"periodic"

tmode

t 方向上的重复模式："black"、"clamp"、"periodic"

data_type

"float"（float 32）、"half"（float 16）、"byte"（int 8）、"short"（int 16）

compression

用于纹理瓦片的压缩，可节省磁盘空间。它依赖纹理格式；请参阅 txmake help。

compression_level

支持该参数的压缩器使用的压缩级别；请参阅 txmake help。

ociocolorspace

目标颜色空间就是纹理颜色空间。

ocioconvert

源颜色空间就是图像颜色空间。

ociodither

输出到非浮点纹理格式时，可选地对像素值执行抖动。不推荐。

bumprough

这个参数是一个 dict，包含 bump to roughness conversion（凹凸转粗糙度转换）的值："factor"、"normalmap"、"invert"、"invertU"、"invertV"、"refit"

mipfilter

用于创建 MIP levels（MIP 层级）的 resizing filter（尺寸调整滤镜）。与 texture_filter 相同。

rules

规则是应当求值为 True 或 False 的 Python 字符串。

所有规则都按文件中的顺序求值 来修改 args dict。如果想在某条规则为 True 时跳过后续规则，可以在规则中添加一个 "break": true entry（条目），以便在规则为 True 时跳过后续规则。

它们可以包含以下 substitution tokens（替换标记）：

node_type

PxrTexture、PxrDomeLight 等

classification

bxdf、pattern、light、lightfilter 等

img_name

不含扩展名的图像名称

img_ext

图像的扩展名

img_atlas

如果是 texture atlas（纹理图集），则为 True

img_type

'int' 或 'float'

img_depth

8、16 或 32

img_nchan

图像中包含的通道数量

ocioconfig

OCIO 配置的完整路径

ocioconfig_name

OCIO 配置的名称：'basic'、'ACES-1.2' 等

要让 ACES 颜色空间正确应用到灯光，你的 rendering role（rendering 角色）必须命名为（或 alias 为）"ACES - ACEScg" 或 "ACEScg"。此外，RenderMan 依赖 OCIO 配置中定义的 luma coefficients（亮度系数）。这些系数在 OCIO v2 中已弃用，并且在默认 ACES 配置中可能不正确。
