使用自己的 OCIO config

OCIO Roles（OCIO 角色）

RenderMan 依赖几个 OpenColorIO roles，需要在你的 config.ocio 文件中定义。

  • rendering： MANDATORY（必需）

    • 这是 renderer 执行所有计算时所在的 space。 

  • data： STANDARD（标准）

    • 这类 input 不需要转换，因为它提供的是 numerical data（normal map、bump map、roughness map 等）。 

  • srgb_linear： MANDATORY（必需）

    • 线性 sRGB/Rec.709 图像，例如 HDRI environment map。 

  • srgb_texture： 

    • 非线性 sRGB 图像，例如 JPEG、PNG 等。

      • 这个 role 已添加到我们的 basic linear configuration 和 Filmic-Blender configuration 中。

      • 在 ACES-1.2 中，它已经是以下项的 alias： Utility - sRGB - Texture。

实际使用中，这 4 个 roles 会满足大多数人有效使用 colour-management（色彩管理）的需求。不过，仍然可以通过 JSON configuration file 添加更多 role。

JSON Configuration file（JSON 配置文件）

许多 configurations 包含大量 color spaces，其中很多是面向与 3D rendering 无关的特殊情况。对 3D artists 真正重要的内容可以归结为少数几个 controls。

为了提供集中配置点，RenderMan 需要一个 JSON configuration file 来提供：

  • UI controls 中可用的 Roles/ColorSpaces 列表。

  • 将插入到 texture names 中、用于消除歧义的 aliases（shortened forms）列表。

  • 可显示在 UI controls 中的 help strings 列表。

  • 一组供 Texture Manager 分配初始 txmake settings 的 rules。

配置 JSON 文件名（File name）

JSON file 的名称基于包含 config.ocio file 的目录：

$RMANTREE/lib/ocio/basic/config.ocio  →  rman_color_config_basic.json
$RMANTREE/lib/ocio/ACES-1.2/config.ocio  →  rman_color_config_ACES-1.2.json

配置文件位置（File location）

启动时，位于 $RMANTREE/etc 中的文件总是会被读取。

如果定义了 $RMAN_COLOR_CONFIG_DIR environment variable，则所有匹配 rman_color_config_*.json pattern、位于该目录中的文件都会被加载。这个机制可用于覆盖 stock configurations，或为 custom OpenColorIO configuration 添加支持。

规则（Rules）

Rules 允许 Texture Manager 为新引用的 images 设置合理的 conversion defaults。如果设置得当，它可以完全自动化 texture conversion。

Rules 由 node types 定义：pattern、light、lightfilter 和 imageplane。这样可以专门化 conversion，而无需依赖一套复杂 rules。

args

这是一个 txmake flags 及其 values 的 dictionary，用来定义可由 rules 修改的 baseline。

带有 null 值的 rules 会被禁用。

texture_type

"regular" 用于通用 textures，"envlatl" 用于 environment light textures。

texture_format

texture 的 file format："pixar"、"tiff"、"openexr"

resize

使用给定 mode 将图像 resize 到 2 的幂："none"、"up"、"up-"、"down"、"down-"、"round"、"round-"。

texture_filter

用于 resize 的 Filter："point"、"box"、"triangle"、"sinc"、"gaussian"、"gaussian-soft"、"catmull-rom"、"mitchell"、"cubic"、"lanczos"、"bessel"、"blackman-harris"

smode

s 方向上的 repeat mode："black"、"clamp"、"periodic"

tmode

t 方向上的 repeat mode："black"、"clamp"、"periodic"

data_type

"float"（float 32）、"half"（float 16）、"byte"（int 8）、"short"（int 16）

compression

用于 texture tiles 的 Compression，可节省磁盘空间。它依赖 texture format；请参阅 txmake help。

compression_level

支持该参数的 compressors 使用的 Compression level；请参阅 txmake help。

ociocolorspace

destination colorspace 是目标颜色空间，也就是 texture 的 colorspace。

ocioconvert

source colorspace 是源颜色空间，也就是 image 的 colorspace。

ociodither

输出到 non-float texture format 时，可选地对 pixel values 执行 dither。不推荐。

bumprough

这个参数是一个 dict，包含 bump to roughness conversion values："factor"、"normalmap"、"invert"、"invertU"、"invertV"、"refit"

mipfilter

用于创建 MIP levels 的 resizing filter。与 texture_filter 相同。

rules

Rules 是应当 evaluate 为 True 或 False 的 Python strings。

所有 rules 都按文件中的顺序进行 evaluate 来修改 args dict。如果想在 rule 为 True 时跳过后续 rules，可以在 rule 中添加一个 "break": true entry，以便在 rule 为 True 时跳过后续 rules。

它们可以包含以下 substitution tokens：

node_type

PxrTexture、PxrDomeLight 等

classification

bxdf、pattern、light、lightfilter 等

img_name

不含 extension 的图像名称（image name）

img_ext

image 的 extension（图像扩展名）

img_atlas

如果是 texture atlas，则为 True

img_type

'int' 或 'float'

img_depth

8、16 或 32

img_nchan

image 中包含的 channels 数量

ocioconfig

OCIO config 的 full path

ocioconfig_name

OCIO config 的 name：'basic'、'ACES-1.2' 等

要让 ACES colorspace 正确应用到 lights，你的 rendering role 必须命名为（或 alias 为）"ACES - ACEScg" 或 "ACEScg"。此外，RenderMan 依赖 OCIO configs 中定义的 luma coefficients。这些系数在 OCIO v2 中已 deprecated，并且在 default ACES config 中可能不正确。
