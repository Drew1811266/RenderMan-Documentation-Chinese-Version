渲染已完成场景涉及许多环节。从处理质量到为后续合成选择合适的输出，本节会覆盖所有这些主题。

Quality（质量）

为了理解噪声去除和抗锯齿的工作方式，我们为用户说明 Sampling（采样） 和 Filtering（滤波）。 这也是大部分调校可能发生的位置，因此理解质量与性能之间的平衡非常重要。

Trace Depth

控制 trace depth（追踪深度）可以改变场景的外观和性能。不同积分器也会以不同方式处理这些设置。RenderMan 让这些参数的调校更容易完成，从而在资源约束内实现所需外观。

Interactive Rendering and Checkpointing（交互式渲染与检查点）

渐进式 /Interactive（交互式） Rendering（渲染）让调校场景材质和灯光变得轻松。 Checkpointing（检查点） 和 Recovery（恢复）会把这个渐进式功能扩展到最终渲染中，让你可以在渲染逐步细化时创建保存点。你可以用它更快地审批未完成帧，并从上次停止的位置继续，而不会浪费宝贵的 CPU 周期！

Diagnostics（诊断）

RenderMan diagnostics（诊断） 可帮助你在易读的显示中可视化场景性能。有时你可能不清楚场景在哪些地方消耗了最多时间或资源，但我们的诊断输出会帮助你定位甚至最隐蔽的性能信息。

Holdouts and Outputs（遮挡保留与输出）

Holdout（遮挡保留） 工作流对于把 CG 对象整合到实拍底板中至关重要。在某些应用中，这只需点击一个按钮即可完成！为了改善合成整合，甚至改善完整渲染的镜头，RenderMan 会使用强大的 outputs（输出）， 以终极灵活性让合成中的调整更轻松、更强大。

Baking（烘焙）

在渲染中复用某些数据可以简化资产管理，同时提升渲染效率。 Baking（烘焙） 提供了烘焙到 2D 输出或点云输出的方式。

Nested Dielectrics

渲染彼此交互的表面时，尤其是透明/折射表面时，理解 Intersect Priority（相交优先级）设置 非常重要。Intersect Priority（相交优先级）允许用户只需添加属性即可获得正确的折射，而无需通过繁琐建模来解决该问题。
