OpenVDB Implicit Field Plugin

Source: https://rmanwiki-27.pixar.com/space/REN27/542213271/OpenVDB+Implicit+Field+Plugin
Downloaded: 2026-06-15T03:49:12+00:00

本文档面向直接使用 RenderMan 接口或 RIB 文件的开发者。


OpenVDB 隐式场插件 impl_openvdb.so 支持使用 OpenVDB 文件，可以作为 隐式曲面 ，也可以作为 体积使用。需要两个字符串参数（第三个参数是可选的，如下所述）。第一个参数是 OpenVDB 文件名。第二个参数是将作为场函数来源的网格名称。该网格必须为 float 类型，并且可以是 level set（水平集）（getGridClass() == openvdb::GRID_LEVEL_SET）或 fog volume（雾体积）（openvdb::GRID_FOG_VOLUME）。如果该网格是 level set（水平集），它会被转换为 fog volume（雾体积）：内部体素的场函数值为 1，外部体素的值为 0，窄带体素的值从 0 到 1 线性变化。如果场函数网格的类别没有在 OpenVDB 文件中指定，或者既不是 level set（水平集）也不是 fog volume（雾体积），可以通过在网格名称后追加“:levelset”或“:fogvolume”来提供覆盖。

该插件支持将 OpenVDB 数据渲染为曲面（使用 RiBlobby）和体积（RiVolume）。下面是将 OpenVDB dragon 数据集渲染为曲面的语法示例。

Blobby 1 [1004  0 0 0 2 1] [0]
    ["${RMANTREE}/lib/plugins/impl_openvdb.so/impl_openvdb.so" "dragon.vdb" "ls_dragon"]

下面是将 OpenVDB dragon 数据集渲染为体积的语法示例。

Volume "blobbydso:${RMANTREE}/lib/plugins/impl_openvdb.so" [-99 103.4 -41.6 49.6 -57.6 77.3]
    [0 0 0] "constant string[2] blobbydso:stringargs" ["dragon.vdb" "ls_dragon"]

可以通过在可选的第三个字符串参数中提供包含速度数据的网格名称来支持变形运动模糊。该网格必须是 openvdb::Vec3f类型，并且必须是 fog volume（雾体积）。（如果 OpenVDB 文件中速度数据的网格类别被设置为其他类型，可以通过在网格名称后追加“:fogvolume”来覆盖。）下面是渲染一个 OpenVDB 数据集的语法示例，该数据集包含一个体积，其场函数位于名为“density”的网格中，速度位于名为“velocity”的网格中。

Volume "blobbydso:${RMANTREE}/lib/plugins/impl_openvdb.so" [-30 30 -30 30 -30 30]
    [0 0 0] "constant string[3] blobbydso:stringargs" ["sphere.vdb" "density"
    "velocity"] "constant float[2] time" [0 1] "varying vector[1] dPdtime" []

支持图元变量，并会将它们绑定到 OpenVDB 文件中同名的网格。RenderMan 中 color、vector、point 和 normal 类型的图元变量会绑定到类型为 openvdb::Vec3fGrid 的网格，而 float 变量会绑定到类型为 openvdb::FloatGrid的网格。在下面的示例中，我们使用三个网格（density、fuel 和 glow）来驱动场函数和两个附加 primvars；同时也显示了 vdb_print file.vdb 的结果。

Volume "blobbydso:${RMANTREE}/lib/plugins/impl_openvdb.so" [-100 100 -100 100 -100 100]
    [0 0 0] "constant string[2] blobbydso:stringargs" ["file.vdb" "density"]
     "varying float fuel" [] "varying color glow" []

fuel         float -100 -100 -100  100 100 100  200x200x200   1.8MVox 17.1MB
density      float -100 -100 -100  100 100 100  200x200x200   1.8MVox 17.1MB
glow         vec3s -100 -100 -100  100 100 100  200x200x200   1.8MVox 51.2MB

从 RenderMan 24.2 开始，该插件以 JSON 字符串形式接受额外参数。JSON 字符串作为可选的第四个字符串参数提供。如果不想指定速度网格，但希望提供 JSON 参数，应将速度网格参数留为空字符串。JSON 字符串应是一个顶层 JSON 字典，并包含以下一个或多个键：

  • filterWidth
  • velocityScale
  • densityMult
  • densityRolloff

这些参数在下文中说明。下面是如何将 JSON 参数传给 DSO 的示例：

Volume "blobbydso:${RMANTREE}/lib/plugins/impl_openvdb.so" [-99 103.4 -41.6 49.6 -57.6 77.3] [0 0 0]
    "constant string[2] blobbydso:stringargs" [
        "dragon.vdb" "ls_dragon" ""
        "{\"filterWidth\": 0.5, \"velocityScale\": 1.0, \"densityMult\": 1.0, \"densityRolloff\": 0.0}"
    ]

默认情况下，该插件会对体素数据执行未过滤查找。它也可以选择对 OpenVDB 文件中的体素数据执行完全过滤查找，以避免走样。要使用 OpenVDB 插件的 mipmapping 功能，必须使用 vdbmake 工具处理 OpenVDB 文件，并且必须设置 filterWidth （默认值为 0，会禁用 mipmapping）。

如果通过设置 filterWidth 为 1 来启用过滤，则用于 OpenVDB 体素的过滤器 width（宽度）会优先抗锯齿而不是细节（DSO 会选择 PRMan 微体素的最大维度）；如果 PRMan 微体素由于较高的相对 shadingrate 而非常各向异性，结果可能会变模糊。将 filterWidth 设置为小于 1 的值可以减轻模糊，但代价是可能出现走样。

默认情况下，速度数据会由插件直接使用，不进行任何进一步变换。如果速度向量的幅度不合适， velocityScale 参数可以在 JSON 字符串参数中设置。如果设置了该参数， velocityScale 会作为速度网格中数值的乘数。

densityMult 和 densityRolloff 参数可用于执行简单的密度操作。 densityMult 会缩放密度网格中的值，而 densityRolloff 参数会应用对数衰减（值越高，衰减越强）。默认情况下， densityMult 设置为 1（不缩放），而 densityRolloff 设置为 0（无衰减）。使用这些参数比在着色器中操作密度值更高效。

在 RenderMan 24.2 之前的版本中， filterWidth 和 velocityScale 会作为可选 float 参数传给 DSO。第一个 float 参数是 filterWidth ，第二个是 velocityScale。请注意，如果将 velocityScale 作为 float 参数传入，那么还必须传入 filterWidth。还要注意，在 RenderMan 24.2 中，以 float 参数形式传递这些参数已弃用。下面示例展示了如何将 filterWidth 和 velocityScale 作为 float args 传入：

Blobby 1 [1004 0 2 0 3 1] [1 0.25]
    ["${RMANTREE}/lib/plugins/impl_openvdb.so" "water.vdb" "surface" "vel:fogvolume"]
