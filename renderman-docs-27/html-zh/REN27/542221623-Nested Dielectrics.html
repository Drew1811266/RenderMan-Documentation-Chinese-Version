XPU 尚不支持相交优先级。

渲染折射对象时，需要特别注意界面（对象彼此接触的位置）。设置不佳可能显示空气间隙、颜色错误等伪影，或者在使用焦散时得到错误结果。当你渲染玻璃中的液体、香槟或冰块中的气泡，或复杂流体模拟时，就需要考虑这种工作流程。

通常，用户会通过创建布尔操作来建模这类对象，使这些表面不会真正彼此接触；它们要么存在其一，要么存在另一方，不会同时存在。这样设置可能很复杂，而当尝试用流动水体这样的模拟来实现这种效果时，几乎不可能。因此在 RenderMan 中，可以向形状添加一个名为 相交优先级。 它是一个简单数值，用来指定优先级；数值越高，优先级越高。

可以在这里找到该主题的论文：

  • Charles M Schmidt、Brian Budge。 Simple Nested Dielectrics in Ray Traced Images, 2002


我们会缓存材质中设置的 IOR，用它确定光线弯折；你可以用属性覆盖此值，也可以让材质自行设置。


下面是几种不同解决方案的示例。

从左到右：使用相交优先级和重叠；液体与玻璃重叠；玻璃与液体相接；液体略小于玻璃

为了更好地理解液体放置方式，下面用玻璃横截面展示了几种可能选项。

  • 重合 - 这种方式中，表面完全相接。注意同一位置上的着色点彼此争抢时产生的噪点伪影。

  • 收缩 - 这个示例将玻璃内部的液体略微缩小，但会产生空气间隙。

  • 相交/重叠 - 这是避免问题的最佳放置方式，但需要向渲染器提供更好的提示，说明它们具有不同优先级。


RenderMan 的相交优先级和 IOR 假定模型按体积方式构建（没有二维对象），并使用朝外法线。

在上面的示例中，重叠的液体和玻璃最接近正确方法：使用重叠，同时将 相交优先级 添加到玻璃和液体形状上。注意液体颜色的变化，以及它捕捉光线和折射的效果好了多少。玻璃的 相交优先级 为 1，而液体为 2。如果加入冰块，则冰块为 3。



在某些情况下，你可能会在几何体重叠处看到一条很细的暗线。要解决这个问题，可以创建一个 Trace Bias 属性并添加到玻璃形状上。默认值为 0.0001，但可以用非常小的增量提高，直到伪影消失。请注意， Trace Auto Bias 的属性必须添加到对象并关闭，手动追踪偏差设置才会生效，除非 全局设置 中的 Trace Auto Bias 已经关闭。
