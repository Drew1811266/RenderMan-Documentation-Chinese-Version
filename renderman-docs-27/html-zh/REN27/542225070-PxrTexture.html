PxrTexture

Source: https://rmanwiki-27.pixar.com/space/REN27/542225070/PxrTexture
Downloaded: 2026-06-15T03:49:12+00:00

PxrTexture 是用于加载纹理的主节点；与所有纹理样式节点一样，此节点接受一个流形，该流形描述用于应用纹理的 2D 或 3D 域。

如果没有连接流形，默认行为是应用到几何体上定义的 s,t 域。

纹理图集格式文件可以读取 UDIM (Mari)、ZBrush 或 Mudbox 格式。此插件还支持 Rtx 程序化纹理插件。

输入参数

Filename

纹理的文件名。必须提供此参数。

要使用图像序列，通常会通过在你偏好的表示方式中嵌入帧号，使用字符串替换构建纹理贴图引用。下面是一个示例：

（Maya）$PROJPATH/sourceimages/flapping.<f4>.tex

（Katana 示例）'/images/flapping.%04d.tex' % frame

（Houdini）在文件名中使用 $F、$F3 或 Houdini expressions，`padzero(3, $F)`

最好避免在文件路径中使用特殊字符、变音符号和空格，因为这些内容可能无法被正确解析，导致纹理渲染失败。

除了我们支持的图集替换字符串，你还可以在文件名中引用常量字符串 primvar，以便在不同几何片段之间复用纹理；此时 primvar 会指定文件名约定。

  • <primstr:primvarname> - 将替换为具名常量字符串基元变量的值，例如资产名称。例如，在一个带有 "const string model" ["apple"] 的苹果上，"/assets/<primstr:model>/diffuse.tex" 会扩展为 "/assets/apple/diffuse.tex"

First Channel Offset

偏移要查找的第一个通道。

Atlas Style

如果指定了图集样式，则 filename 必须包含 <UDIM>，它会根据表面参数化和 atlasStyle 自动替换。

第一个 UV tile 的 Mari UDIM 示例为 Diffuse.1001.tex 其中 "1001" 会替换纹理文件名中的 <UDIM>。

例如： 

如果你的纹理使用 Mari UDIM 样式，且文件名为： Diffuse.1001.tex，则需要使用 Diffuse.<UDIM>.tex 才能让纹理加载到其正确的 UDIM 中。  

注意：<UDIM> 可以使用 UPPER，也可以使用小写 <udim>



用户在所有图集纹理工作流中都应避免使用负纹理坐标

Invert T

反转纹理查找的 t 参数。

Filter

选择纹理查找期间可使用的不同重建滤镜。PxrTexture 可用的滤镜为：

  • 0: Nearest

  • 1: Box

  • 2: Bilinear

  • 3: Bspline

  • 6: Gaussian

  • 7: Lagrangian

Blur

指定对从纹理文件取回的图像进行模糊的程度。

Mip Interpolate

选择是否在多分辨率纹理的相邻分辨率之间插值，从而在层级之间获得更平滑的过渡。

Missing Color

如果打开纹理时出错，使用此颜色。

.0 0.0 1.0

Missing Alpha

如果打开纹理时出错，使用此 alpha。

Linearize

对纹理应用反向 sRGB 变换。如果你在 sRGB 空间中绘制纹理（大多数绘画软件的默认值），但在数据线性空间中查看数据，纹理会显得发白。此选项会对纹理应用 sRGB 变换，使其再次呈现视觉线性。

Manifold

提供应用纹理的域。默认值为 s,t。



调整输出

Color Scale

纹理中色值的倍增值，可用于调整亮度或操作单个颜色通道

Color Offset

对结果应用偏移，从而移动结果颜色

Saturation

提高（大于 1）或降低（小于 1）纹理结果的饱和度

Alpha Scale

alpha 通道的倍增值；当 alpha 用于驱动参数时很有用

Alpha Offset

alpha 通道的偏移；当 alpha 用于驱动参数时很有用

高级设置

MipBias

mip 选择偏差会选择用于渲染的更高分辨率（负整数）或更低分辨率（正整数）层级。

 
MaxResolution

将 mip 选择钳制到首选最大层级。这可能会根据你的选择阻止最高分辨率被渲染。

 
OptimizeIndirect

在间接命中上将纹理滤镜设置为 Box（低开销），而不是用于其他可见效果的所选滤镜。

输出参数

resultRGB

过滤后的颜色结果。请注意，所有结果都会从 startChannel 偏移量开始查找。

resultR

R 通道结果

resultG

G 通道结果

resultB

B 通道结果

resultA

如果存在 alpha 且连接了 resultRGB，则返回 RGB 之后的通道。如果未连接 RGB，则返回第一个通道。
