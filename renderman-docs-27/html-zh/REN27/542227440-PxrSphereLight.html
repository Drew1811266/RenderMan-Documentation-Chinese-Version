PxrSphereLight

Source: https://rmanwiki-27.pixar.com/space/REN27/542227440/PxrSphereLight
Downloaded: 2026-06-15T03:49:12+00:00

此面光源模拟点光源和球形光源，例如灯泡、头灯等。

注意，此灯光的缩放始终是统一缩放。此灯光是单面的，也就是说，球体内部没有照明。如果需要内部照明，请改用 PxrDomeLight 。


你会看到对“非物理”控制的提及。这些控制用于帮助艺术家通过忽略我们通常模拟的某些物理定律，制作艺术指导图像。



Parameters（参数）


Basic

Intensity

线性缩放此灯光的贡献。灯光强度使用物理衰减计算。下面的 Intensity 参数在每次渲染中增加 1.0。
 


Exposure

指定面光源的 Exposure（曝光），以 2 的幂表示。Exposure（曝光）增加 1 会使光源发出的能量翻倍。值 0 在光源处产生 1 的强度，-1 产生 0.5。现实世界照明能量很高，典型曝光值较低；如果要获得等效的  Intensity ，你可能必须输入非常大的数字。对于熟悉摄影测量的艺术家来说，这也更顺手。下面显示 1.0、然后 2.0、最后 3.0 的值。Intensity 被重置为 1.0，所以由于灯光的物理衰减，结果会比上方示例开始时更暗。




Color

发出灯光的颜色。



Enable Temperature

开启或关闭色温。下面我们使用同一个 Luxo/Pixar HDRI，并使用以下温度：6500K（默认）、4000K，最后是 8500K。




Color Temperature

允许用户选择灯光的色温，单位为开尔文。不同于基础灯光颜色，这让用户可以根据标准温度测量轻松选择可信的灯光颜色。此控制的作用类似灯光前方的滤镜或灯光色片（一层带色调的透明表面）。如果灯光为白色，你会得到所选色温。如果灯光已有颜色，色温会使它更冷（高于 6500K）或更暖（低于 6500K）。


该效果在完全饱和的单色灯光上不太明显。默认值为 6500K，在大多数显示器上应非常接近白色（sRGB 和 Rec 709 使用的 D65 标准照明体）。


Primary Visibility

默认情况下，灯光不会直接对摄影机可见。开启此项会使其可见，并且可使用发射 token 'O' 写入 LPE。


Refine


请注意，在用户界面中，PxrSphereLight 包含一个方向标记，指向发射方向和锥形可见性方向。你可能需要旋转这盏球形灯光，让灯光指向正确方向。



Emission Focus

离轴余弦幂指数，用于塑造发射。这会控制灯光的 扩散范围 。数值越高，灯光越会开始向中心聚焦，从而缩窄光照扩散范围。下面的值分别为 0.0（Off）、10.0，最后是 50.0。



Emission Focus Tint

离轴灯光颜色色调。它会从灯光的离轴角方向朝中心方向，为衰减区域中的发射着色。默认情况下，色调颜色为黑色（无）。除了着色之外，大于白色的值还会提高衰减区域的强度。下面的 Emission Focus 设为 10.0，并使用以下颜色：白色、红色和蓝色。注意，使用白色时，发射聚焦看起来几乎像是关闭的。



Specular Amount

发出的镜面反射光数量。这是一个非物理控制。 你可以使用一个  Specular Amount  为 0.0 的灯光，作为场景对象的纯漫反射光来源，并避免添加高光。 下面我们从默认值（1.0）变为 0.5，然后变为 0.0（无贡献）。此场景有细微反射，但你可以注意到在最右侧铬球的反射中，灯光消失了。



Diffuse Amount

发出的漫反射光数量。这是一个非物理控制。 你可以使用一个  Diffuse Amount  为 0.0 的灯光，作为场景对象的纯高光来源。 下面我们从默认值（1.0）变为 0.5，然后变为 0.0（无贡献）。



Intensity Near Dist

被照亮点与灯光之间的近距离，在该距离内样本不会继续变亮。当灯光靠近表面时，这可能有助于避免热点和采样问题。下面分别为 0.0、2.5，最后是 5.0。这些值取决于你的场景尺度/距离，因此这里只是示例。



Specular Near Dist

带有镜面着色的被照亮点与灯光之间的近距离，在该距离内样本不会继续变亮。当灯光靠近镜面表面时，这可能有助于避免热点和采样问题。 注意，此值不会应用于从摄影机看到的灯光。 下面，右侧球体的距离分别为 0.0、5.0，最后是 10.0。这些值取决于你的场景尺度/距离，因此这里只是示例。

                         


Diffuse Near Dist

带有漫反射着色的被照亮点与灯光之间的近距离，在该距离内样本不会继续变亮。当灯光靠近漫反射表面时，这可能有助于避免热点和采样问题。 注意，此值不会应用于从摄影机看到的灯光。 下面，左侧球体的距离分别为 0.0、5.0，最后是 10.0。这些值取决于你的场景尺度/距离，因此这里只是示例。

                         


Cone Angle

灯光上虚拟遮扉的角度，用于把它变成聚光灯。这不会影响 IES 光型文件（如果你正在使用 IES Profile）。下面我们从 90（默认）变为 45，最后是 20。



Cone Softness

控制 Cone Angle的截断柔和度。常规范围为 0（硬截断）到 1（平滑过渡），但也可以设为高于 1，以获得额外渐变。上方的 Cone Angle 示例中，你会注意到边缘看起来很硬；此参数可以改善该结果，使其更悦目。下面我们从 0.0（默认，关闭）变为 0.25，然后是非常柔和的 0.75。


Light Profile


IES Profile

IES 灯光配置文件的名称。下面使用三个不同的 IES 光型文件和一些暗调氛围照明。



Profile Scale

重新缩放 IES 光型文件的角度分布。小于 0 的值表示光型文件覆盖的角度范围较小，并会有效增大投射光型文件的大小。默认值为 0，表示完整覆盖（无缩放）。

Profile Normalization

设置 IES 光型文件，防止其振幅改变灯光的能量。

Shadows

如果灯光的  Trace Light Paths  在 Advanced（高级）中启用，则所有阴影参数都会被忽略。下面的选项在脱离默认状态使用时都是非物理控制。如果  Trace Light Paths  启用，则会强制执行其物理属性。



Enable Shadows

启用光线追踪阴影。下图分别为开启（默认）和关闭。




Shadow Color

发出光投射的阴影颜色。下面分别为红色、黄色和蓝绿色。



.0 0.0 0.0

Shadow Max Distance

从被着色点位置开始计算的最大阴影距离。-1.0 表示未设置，此时使用被着色点与灯光上点之间的距离。你可以使用此控制人为减少投射阴影的距离。它也可能通过不计算该距离之外的阴影来提高渲染速度。这里用它移除了该灯光闪光灯泡效果在后墙上的阴影。




Shadow Falloff

从灯光处开始发生阴影衰减的距离。-1.0 会关闭阴影衰减。它与  Shadow Max Distance  一起使用，为被  Shadow Max Distance  参数削弱或截断的阴影创建伪淡出。下面是两个参数一起使用且衰减增加的示例。从左到右分别为 Off、0.25，然后是 1.0。注意，此参数可能影响复杂灯光设置中的性能。




Shadow Falloff Gamma

衰减区域中阴影强度的伽马。这需要使用  Shadow Max Distance  和  Shadow Falloff 。默认值为 1.0，这里首先显示默认值，然后是 0.5，最后是 0.25。



Trace Subset

要纳入追踪阴影求交的几何体集合。如果未指定，则所有几何体都会纳入追踪阴影求交。


Don't Trace Subset

要忽略追踪阴影求交的几何体集合。如果未指定，则所有几何体都用于追踪阴影求交。



Advanced


Normalize

启用 Normalize 后，当你缩放光源时，贡献到场景中的灯光量不会改变。这会让你更容易在不改变场景照明的情况下调整高光大小。


Trace Light Paths

启用从此灯光开始的光线和光子追踪。该值会强制使用基于物理的灯光，并且作为副作用禁用上面的  Shadows  控制。使用 PxrVCM 或 PxrUnified 积分器时，用户可利用此功能有选择地决定哪些灯光发射光子。


Thin Shadow

为此灯光启用薄阴影并禁用折射焦散。如果  Trace Light Paths  已启用，则会忽略此参数。这是一个非物理控制，可为透射对象创建“伪”彩色阴影，而无需为焦散生成光子。下面我们从 On（默认）切换到 Off（不透明阴影）。注意，选择 Off 时会丢失彩色阴影和内部彩色反射，但会换来一些渲染速度。


 

Visible in Refraction

开启此项可使灯光通过玻璃和窗户等透射效果可见。


Manifold Walk

与 PxrUnified Integrator 一起使用，并启用 Enable Manifold Walk 以创建低成本焦散。注意：你必须禁用 Thin Shadows


Manifold Walk Exclude Group

与 PxrUnified Integrator 一起使用，并启用 Enable Manifold Walk， 在流形探索期间会忽略此追踪集，以避免停止或遮挡流形探索光线


Light Samples

指定此光源要采用的灯光采样数量覆盖值。如果设为非零值，它会覆盖积分器执行的采样。除非你遇到来自该灯光且无法解决的噪声，否则建议保持默认值。


Light Group

指定用于灯光组 LPE 的 Light Group 名称。这对于生成逐灯光 AOV 以便之后在合成中调整很有用。


Importance Multiplier

用户不必显式设置 Light Samples，而可以通过更改此值来改变内部分配给该灯光的采样数量。RenderMan 会在渲染时为场景中的所有灯光创建一组采样，改变此值会在这些灯光之间重新平衡采样。注意，增大此值会使 更多 采样从此灯光中被选择，同时减少场景中其他灯光的采样。低于默认值则会 减少 该灯光的采样，同时为场景中的其他灯光提供更多采样。
