RenderMan 支持完整范围的几何图元，包括 多边形 ,  NURBS 、细分曲面 , 曲线 、体积 以及 隐式曲面 。 对象实例化 也受到完整支持。

选择图元

多边形 简单且单个开销较低，但详细模型和曲面可能需要大量多边形才能达到可接受的质量，因此很快会消耗大量内存（即使 RenderMan 的实现非常高效）。不过，尤其是在渲染建筑或城市等硬表面对象集合时，它们的效率仍然很难被超越。

NURBS 是真正的曲面，效率高，并且能产生非常平滑的结果。不过，它们要求受限的纯四边形拓扑，建模时可能较为苛刻。

细分曲面 通常能以更简单的方式生成平滑曲面，几乎没有建模限制，并且非常适合动画用途。许多多边形网格可以任意转换为细分曲面，并立即获得平滑外观的优势。不过请注意，细分曲面天生具有更高的内存成本；如果几何体细节过密，密集的细分曲面可能不如等价的多边形网格高效。

曲线 是一类专用图元，尤其擅长渲染头发、毛皮或草叶这类细长几何体。

隐式曲面 是另一类专用图元，最常用于处理由第三方软件包生成的流体模拟。它们也可以用于直接渲染黏性流体。

体积 用于渲染具有非均匀密度的参与介质，例如雾、烟或云。

镶嵌细分

在 RenderMan 中，一些图元可能会在运行时进行镶嵌细分。这可能发生在 NURBS 和细分曲面上（为了提供平滑表面），也可能发生在与位移着色器关联的图元上（为了提供可进行位移的详细网格）。

请注意， 镶嵌细分不会影响着色发生的频率 （因为着色发生在每次光线命中时）。尤其是，较粗的镶嵌细分不应影响任何纹理过滤，也不应产生模糊的图案。

但是， 镶嵌细分会影响位移计算的频率 （因为位移发生在镶嵌细分后网格的每个顶点上）。

Micropolygonlength

图元会被镶嵌细分为 微多边形。镶嵌细分程度由 dice micropolygonlength 控制。

Attribute "dice" "constant float micropolygonlength" [1]

这是镶嵌细分试图生成的平均 微多边形边长 。 值越大，微多边形越大。

  • 更大 micropolygonlength 意味着 更粗 的镶嵌细分；对象可能显示出分面或锐利边缘，但渲染成本应更低

  • 更小 micropolygonlength 意味着 更细 的镶嵌细分；对象应更平滑，并以更高渲染成本更好地贴合你的理想形状

默认情况下，micropolygonlength 值以屏幕上的像素为单位表示（即微多边形投影到屏幕上后的像素长度）。这可以通过使用 dice attribute strategy进行修改。

Attribute "dice" "string strategy" ["planarprojection"]

该 micropolygonlength 属性是已弃用 shading rate的后继。shading rate 以 面积表示，而 micropolygonlength（顾名思义）是 长度。 


这意味着，为了获得类似的镶嵌细分级别，使用 shading rate 值 X 就相当于使用 micropolygonlength 值 sqrt(X)：

  • shading rate = 0.25, micropolygonlength = 0.5

  • shading rate = 1, micropolygonlength = 1

  • shading rate = 4, micropolygonlength = 2

如果需要匹配以前的镶嵌细分级别，这是一个有用的换算。

选择 micropolygonlength 值

大多数情况下， micropolygonlength 值为 1（默认值）应能很好地工作。如果表面平滑且没有位移，在看到任何差异之前，你也许可以使用更大的值（2 或 4）。

有时，如果几何体非常详细和/或发生了位移，可能需要使用更小的 micropolygonlength值。0.5 常用于更精细的细节。不过，请注意，更小的微多边形会导致更大的内存消耗和更慢的渲染时间。这种增长可能非常快，因为每当你把 Micropolygon Length (ML) 减半时，微多边形数量就会变为四倍。

  • ML 1 = 每像素 1 个微多边形（默认）

  • ML 0.5 = 每像素 4 个微多边形（适合非常精细的位移）

  • ML 0.25 = 每像素 16 个微多边形（可能微多边形过多且收益递减）

  • 0.125 = 每像素 64 个微多边形

  • 0.0625 = 每像素 256 个微多边形

可以通过查看 Prototype Attributes 或 RenderMan 提供的 "PrimVars" 了解更多几何设置。
