Presence and Opacity

Source: https://rmanwiki-27.pixar.com/space/REN27/542221558/Presence+and+Opacity
Downloaded: 2026-06-15T03:49:12+00:00

在 RenderMan 中，可以使用材质上的 Presence Parameter 控制对象的不透明度。它通常设计为 0 或 1 的 “cutout” 效果；介于两者之间的值可用于半透明对象或幽灵般效果。


如果使用 0 到 1 之间的值（或带有中灰值的 pattern），默认情况下会看到噪点，因为这里使用随机采样技术。不过，integrators 允许通过在 Stochastic Opacity Depth 参数中指定深度来获得更平滑的效果；每次相交都会计作一个深度限制。下文会对此进行说明。

如果渲染厚玻璃，presence 会被忽略，因为需要知道光线何时进入和离开 volume。此处使用 cutout 可能会破坏这一条件。

最简单的例子是树叶。与其实际建模一片叶子，让艺术家费力并增加场景中的几何复杂度，通常会用一个平面表示叶子，并使用一张贴图提供形状。然后将这张贴图连接到 Presence Parameter。

这是使用该参数的传统方式。但你也可能希望在渲染中使用部分 Presence 值，让某些对象呈现半不透明效果。默认情况下，我们使用随机方法确定 presence。这意味着当数值不是 0 或 1 时，会在不同 presence 水平上随机采样。这可能产生噪点，并需要更多采样才能清理干净或收敛。下面有一组重叠平面，Presence value 设为 0.15。请注意， 较低的 presence 值会比更高的值显示更多噪点。

一些 production integrators 支持用名为 “Stochastic Opacity Depth” 的参数解决此问题。该值默认为 0，表示 “off”。不过，每增加一级，它都会告诉渲染器在指定深度合成半不透明颜色，而不是继续随机采样；超过该深度后不再这样处理。下面是 depth 1、2 和 3 的示例。

从性能角度考虑，你可能会注意到，具有较高不透明度的重叠对象会产生更少噪点。通常只需把 Stochastic Opacity Depth 设到相对于场景其余部分噪点可接受的位置，而不必逐个计算重叠表面数量并把该数量直接用作设置值。

Cached versus Uncached（缓存与非缓存）

为了避免一些昂贵计算，我们会将 presence 值与几何体的微多边形一起缓存。通常这没有问题，因为默认 dicing 方案会创建像素大小或更小的微多边形。不过，你可能会注意到，细分不佳的对象会出现边缘毛边或错误数值。

请注意这里叶片边缘是半透明且不正确的。解决这个问题有两种方法：

  • 如果使用 cached，降低 micropolygon length 以创建更小的微多边形，从而获得更多细节

  • 可以从 cached 切换到 non-cached，改为针对每个 shading sample 求值；这可能代价较高，但作为解决方案是“最简单”的
