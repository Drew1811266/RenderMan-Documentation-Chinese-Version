Stylized Lines

Source: https://rmanwiki-27.pixar.com/space/REN27/542221920/Stylized+Lines
Downloaded: 2026-06-15T03:49:12+00:00

PxrStylizedLines filter（滤镜）允许你为 Stylized（风格化）渲染添加线稿。它可以单独用于技术制图风格，也可以与 toon（卡通）和 hatching（排线）结合，创造更有创意的外观。你甚至可以叠加多个 PxrStylizedLines filter，创建颜色、破碎和线宽变化，以模拟各种传统线稿技法。 


Result（结果）

使用这些设置预览特定 AOV，帮助开发创意线稿。

  • Disable（禁用） - 关闭此节点；如果前面已连接另一个 DisplayFilter，则显示该串联链之前的 beauty（主图像），否则在没有其他 DisplayFilter 节点连接时显示常规 beauty 渲染

  • Final Lines Comp（最终线条合成） - 查看 Final Lines（最终线条）输出，它会根据 Compositing Page（合成页面）中选择的模式合成到上一层之上

  • Final Lines（最终线条） - 仅查看线条输出
  • Visualizer（可视化器）- 此分隔符下方的菜单项表示主要用于调试和可视化的模式
  • Final Lines Invert（最终线条反相） - 线条的反相可视化
  • Final Lines Alpha（最终线条 Alpha） - 由线条生成的 Alpha 遮罩通道。

  • Line Thickness from PxrStylizedControl（来自 PxrStylizedControl 的线条粗细）- 可视化从插入 Line Thickness Input（线条粗细输入）的任何 pattern（图案）或纹理生成的线条粗细，该输入位于你的 PxrStylizedControl 节点。

  • Line Thickness from Lighting（来自光照的线条粗细）- 可视化由场景光照生成的 Line Thickness（线条粗细）

  • Signal Luminance - 用于将颜色转换为灰度的正确 RGB 权重；目前未在任何模式中使用，只作为可视化器
  • Signal Energy - 此节点创建并用于混合 hatching（排线）的内部 white shader pass（白色着色器通道）


Pro Tip（专业提示）:
在交互式渲染期间，尝试更改 Result（结果） 下拉菜单，以切换不同模式



Line Detection Page（线条检测页面）

Type（类型）

线条检测类型。根据选择，会选取不同数据 AOV 来运行线条检测：

Outline - 基于对象 Alpha 生成线条
Sections - 基于随机 ID 生成线条
LineNZ - 基于表面法线的 X、Y 分量与 Z 深度的组合生成线条，适合单位尺度
Curvature - 基于表面法线变化生成线条
Camdist - 基于摄像机距离变化生成线条
Custom AOV - 使用下面定义的自定义 AOV

Data AOV（数据 AOV）

用于进行线条检测的颜色 AOV 名称。 

通常这是 diffuse（漫反射），但也可以在这里放入 light AOV（灯光 AOV），根据灯光发射决定照片写实与 Stylized（风格化）之间的过渡。此 AOV 必须在 Maya/Katana/Houdini 中设置，否则 DisplayFilter 会退出。

Write To Type AOV（写入 Type AOV）

用线条检测的结果覆盖上面 Type（类型）中选择的 Data AOV（数据 AOV）。通常在计算出线条检测之后，我们不再需要原始数据 AOV；因此出于可视化和合成目的，会用线条检测的最终结果写回覆盖此 AOV。

Weight（权重）

在线条检测加权过程中获得变化

Activation（激活）

在线条检测加权过程中获得变化。可在 Spline（样条，用户自定义控制点）或 Sigmoid 之间选择。

Line Thickness Page（线条粗细页面）

Line Thickness（线条粗细）

线条的粗细，以像素计量


Line Thickness Remap Page（线条粗细重映射页面）

by PxrStylizedControl（按 PxrStylizedControl）

Multiply（相乘）：将 Line Thickness（线条粗细）乘以 PxrStylizedControl 上逐对象的 Line Thickness 值

PxrStylizedControl Scale（PxrStylizedControl 缩放） 

全局缩放 Line Thickness（线条粗细）。如果你已逐对象设置 Line Thickness，且它们的值各不相同，当你提高渲染分辨率时，可使用此全局缩放因子增加或减少所有线条的 Line Thickness，因为它们基于像素宽度。

by Z Depth（按 Z 深度）

Multiply Line Thickness by Z Depth（按 Z 深度乘以线条粗细）：将线条粗细乘以 Z Depth（Z 深度）

Cam Dist Z Min

重映射摄像机距离：Z 上的 Linear step Min（线性阶跃最小值，来自 lineNZ 的蓝色通道）

Cam Dist Z Max

重映射摄像机距离：Z 上的 Linear step Max（线性阶跃最大值，来自 lineNZ 的蓝色通道）

Z Min Thickness Scale

在 Cam Dist Z Min 处，按此数值缩放线条粗细

Z Max Thickness Scale

在 Cam Dist Z Max 处，按此数值缩放线条粗细

by Lighting（按光照）

Multiply Line Thickness by Lighting signal（按光照信号乘以线条粗细）：将线条粗细乘以 Lighting signal（光照信号；可在 Visualizer 下拉菜单中查看 Line Thickness From Lighting 以查看该信号）

Low Mult（低值乘数）

Scale（缩放）：在阴影中按此值乘以 Line Thickness（线条粗细）

High Mult（高值乘数）

Scale（缩放）：在受光区域按此值乘以 Line Thickness（线条粗细）

Invert

反转上面的 min/max（最小/最大值）：使阴影中的线条更粗

Gamma

对光照响应应用 gamma 曲线，以驱动粗细


Input Page（输入页面）

Signal（信号）

用于 Line Color（线条颜色）以及由光照驱动的 Line Thickness（线条粗细）的 light Signal color AOV（光照信号颜色 AOV）名称。通常为 diffuse（漫反射）；也可以在这里放入任何 light AOV（灯光 AOV）。此 AOV 必须在 Maya/Katana/Houdini 中设置，否则 DisplayFilter 会退出。

Mode

在底层，会基于 Signal AOV 创建一个白色着色器近似来驱动 Hatching（排线）。默认使用 Signal AOV 的 RGB 分量平均值，但也可以使用 NPRalbedo AOV 或 albedo AOV 中的纯 albedo（反照率），去除 Signal AOV 光照结果中的 albedo 分量。如果选择 NPRalbedo AOV，则必须在 Stylized Controller Pattern 节点中设置 Albedo Color。


Dilation Sorting Page（膨胀排序页面）

Enable

开启后：会将参与生成线条的信号（albedo color（反照率颜色）、thickness（粗细）、Z 深度等）膨胀到几何体外，因为线条检测会越出几何体。如果 Dilate Sort 关闭，几何体外不正确的数据会产生不理想的双线。它还会进行排序，为更靠近摄像机的对象分配更高优先级。

Radius

用于膨胀信号的 Radius（半径）大小，以像素计。

Depth Scale

Scale Z depth（缩放 Z 深度）：根据场景尺度缩放 Z 深度以进行排序

Weight

与 Line Thickness 边缘检测相同的 Weight 类型，但用于 Dilate Sort 半径

Use Alpha（使用 Alpha）

如果对象有 Alpha 通道：相机深度上的零 Alpha 会不正确。请改为切换为 ON，根据 Alpha 在对象的相机深度与 Dilate_Sort_Max_depth 之间混合

Max Depth

Alpha 外部要考虑的 Max Depth（最大深度）


Line Color Page（线条颜色页面）

From（来源）

提供多个不同选项来确定 Line Color（线条颜色）：

Color Swatch（颜色色样）- 从下方 Line Color 中为此节点的所有线条选择颜色
PxrStylizedControl - 从 PxrStylizedControl pattern（图案）节点获取 Line Color（线条颜色）
NPRalbedo AOV - 从 NPRalbedo AOV 获取 Line Color（线条颜色），该 AOV 由 PxrStylizedControl pattern（图案）节点设置
albedo AOV - 从 RenderMan 默认 albedo AOV

Line Color（线条颜色）

用于选择颜色的 Color swatch（颜色色样）

Light Tint（光照着色）

根据从亮部到阴影的光照为线条着色：

HSV Light Dark（HSV 亮部/暗部）- 在光照的 Light（亮部）与 Dark（暗部/阴影）部分选择 Hue/Saturation/Value（色相/饱和度/明度）tint（色调）
Line Color Shadow（线条阴影颜色）- 选择阴影区域中的 Line Color（线条颜色）


Line Distort Page（线条扭曲页面）

RenderMan v25 新增！

Enable

开启 Line Distort（线条扭曲）功能

Offset

使线条偏离原始位置的扭曲量

Remap（重映射）

Remap line distort signal（重映射线条扭曲信号）：将 line distort signal（线条扭曲信号，通常是插入到 PxrStylizedControl Distort/Distort U & Distort V input 的 PxrFractal）在 -1 到 1 之间重映射（假设原始范围为 0 到 1）

Demo videos（演示视频）

Mask Page（遮罩页面）

PxrStylizedControl Mask（PxrStylizedControl 遮罩）

使用 PxrStylizedControl pattern（图案）节点中配置的 mask（遮罩）

Input Mask（输入遮罩）

使用上方 Input/Signal 数据作为先前串联的 Stylized shaders（风格化着色器）和/或基于物理的照明的遮罩

Final Line Gamma（最终线条 Gamma）

线条检测发生后，对 Final Lines（最终线条）应用 gamma。使用 Curvature Line Type（曲率线条类型）时，可用此参数控制 Curvature falloff（曲率衰减），以微调线稿

Lines Inside Alpha（Alpha 内部线条）

乘以 Alpha，使线条检测得到的线条仅位于内部，不越出几何体

Compositing Page（合成页面）

Mode（模式）

Over - 使用 Lines（线条）的 Alpha，合成到上一层之上
Plus - 使用 Lines（线条）的 Alpha，与上一层相加
Multiply - 使用 Lines（线条）的 Alpha，与上一层相乘

Linework Tips（线稿提示）

  • 为了让线条更扎实，可尝试将 Weight Type（权重类型）改为 Hermite 或 Double Sigmoid

  • 将纹理或 pattern（图案）插入 PxrStylizedControl 节点的 Line Thickness input（线条粗细输入），以破碎并改变线宽，为线条增加创意变化。

  • 尝试通过叠加多个 PxrStylizedLines filter（滤镜）来探索线稿；每个 filter 都有自己独特的宽度、颜色和粗细。
