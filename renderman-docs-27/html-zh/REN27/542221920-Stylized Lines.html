Stylized Lines

Source: https://rmanwiki-27.pixar.com/space/REN27/542221920/Stylized+Lines
Downloaded: 2026-06-15T03:49:12+00:00

PxrStylizedLines filter 允许你为 Stylized 渲染添加线稿。它可以单独用于技术制图外观，也可以与 toon 和 hatching 结合，创造更有创意的外观。你甚至可以把多个 PxrStylizedLines filter 相互叠加，创建颜色、破碎和 width 变化，以模拟各种传统线稿技法。 


Result（结果）

使用这些设置预览特定 AOV，帮助开发创意线稿。

  • Disable（禁用） - 关闭此节点；如果前面已连接另一个 DisplayFilter，则显示该串联链之前的 beauty，否则在没有其他 DisplayFilter 节点连接时显示常规 beauty 渲染

  • Final Lines Comp - 查看 Final Lines 输出，它会根据 Compositing Page 中所选模式合成到上一层之上

  • Final Lines - 仅查看线条输出
  • Visualizer - 此分隔符下方的菜单项表示主要用于调试和可视化的模式
  • Final Lines Invert - 线条的反相可视化
  • Final Lines Alpha - 由线条生成的 alpha 遮罩通道。

  • Line Thickness from PxrStylizedControl - 可视化插入到 Line Thickness Input 的任何 pattern 或纹理所生成的线条粗细，该输入属于你的 PxrStylizedControl 节点。

  • Line Thickness from Lighting - 可视化由场景光照生成的 Line Thickness

  • Signal Luminance - 用于将颜色转换为灰度的正确 RGB 权重；目前未在任何模式中使用，只作为 visualizer
  • Signal Energy - 此节点创建并用于混合 hatching 的内部 white shader pass


Pro Tip:
在交互式渲染期间，尝试更改 Result（结果） 下拉菜单，以切换不同模式



Line Detection Page

Type

线条检测类型。根据选择，会选取不同 data AOV 来运行线条检测：

Outline - 基于对象 alpha 生成线条
Sections - 基于 random id 生成线条
LineNZ - 基于表面法线的 X、Y 分量与 Z depth 的组合生成线条，适合单位尺度
Curvature - 基于表面法线变化生成线条
Camdist - 基于摄像机距离变化生成线条
Custom AOV - 使用下面定义的 custom AOV

Data AOV

用于进行线条检测的 color AOV 名称。 

通常这是 diffuse，但也可以在这里放入 light AOV，根据灯光发射决定照片写实与 Stylized 之间的过渡。此 AOV 必须在 Maya/Katana/Houdini 中设置，否则 DisplayFilter 会退出。

Write To Type AOV

用线条检测的结果覆盖上面 Type 中选择的 Data AOV。通常在计算出线条检测之后，我们不再需要原始 data AOV；因此出于可视化和合成目的，会用线条检测的最终结果写回覆盖此 AOV。

Line Thickness Page

Line Thickness

线条的厚度，以 pixels 计量


Line Thickness Remap Page

by PxrStylizedControl

Multiply：将 Line Thickness 乘以 PxrStylizedControl 上逐对象的 Line Thickness 值

PxrStylizedControl Scale 

全局缩放 Line Thickness。如果你已逐对象设置 Line Thickness，且它们的值各不相同，当你提高渲染分辨率时，可使用此全局 scale 增加或减少所有线条的 Line Thickness，因为它们基于 pixel width。

by Z Depth

Multiply Line Thickness by Z Depth：将 Line Thickness 乘以 Z Depth

Cam Dist Z Min

Remap 摄像机距离：Z 上的 Linear step Min（来自 lineNZ 的蓝色通道）

Cam Dist Z Max

Remap 摄像机距离：Z 上的 Linear step Max（来自 lineNZ 的蓝色通道）

Z Min Thickness Scale

在 Cam Dist Z Min 处，按此数值缩放 Line Thickness

Z Max Thickness Scale

在 Cam Dist Z Max 处，按此数值缩放 Line Thickness

by Lighting

Multiply Line Thickness by Lighting signal：将 Line Thickness 乘以 Lighting signal（在 Visualizer 下拉菜单中查看 Line Thickness From Lighting 以查看该 signal）

Low Mult

Scale：在阴影中按此值缩放（multiply）Line Thickness

High Mult

Scale：在受光区域按此值缩放（multiply）Line Thickness

Invert

反转上面的 min/max：使阴影中的线条更粗

Gamma

对光照响应应用 gamma curve，以驱动 thickness


Input Page

Signal

用于 Line Color 和由光照驱动的 Line Thickness 的 light Signal color AOV 名称。通常为 diffuse；也可以在这里放入任何 light AOV。此 AOV 必须在 Maya/Katana/Houdini 中设置，否则 DisplayFilter 会退出。

Mode

在底层，会基于 Signal AOV 创建一个白色 shader 近似来驱动 Lines。默认使用 Signal AOV 的 RGB 分量平均值，但也可以用 NPRalbedo AOV 或 albedo AOV 中的纯 albedo，去除 Signal AOV 光照结果中的 albedo 分量。如果选择 NPRalbedo AOV，则必须在 Stylized Controller Pattern 节点中设置 Albedo Color。


Dilation Sorting Page

Enable

开启后：会将参与生成线条的信号（albedo color、thickness、z-depth 等）膨胀到几何体外，因为线条检测会越出几何体。如果 Dilate Sort 关闭，几何体外不正确的数据会产生不理想的双线。它还会排序，为更靠近摄像机的对象分配更高优先级。

Radius

用于膨胀信号的 Radius 大小，以 pixels 计。

Depth Scale

Scale Z depth：根据场景尺度缩放 Z depth 以进行排序

Weight

与 Line thickness edge detection 相同的 Weight 类型，但用于 Dilate Sort radius

Use Alpha

如果对象有 alpha channel：camera depth 上的零 alpha 会不正确。请改为切换为 ON，根据 alpha 在对象的 camera depth 与 Dilate_Sort_Max_depth 之间混合

Max Depth

alpha 外部要考虑的 Max Depth


Line Color Page

From

提供多个不同选项来确定 Line Color：

Color Swatch - 从下方 Line Color 中为此节点的所有线条选择颜色
PxrStylizedControl - 从 PxrStylizedControl pattern 节点获取 Line Color
NPRalbedo AOV - 从 NPRalbedo AOV 获取 Line Color，该 AOV 由 PxrStylizedControl pattern 节点设置
albedo AOV - 从 factory RenderMan albedo AOV

Line Color

用于选择颜色的 Color swatch

Light Tint

根据从亮部到 shadow 的光照为线条着色：

HSV Light Dark - 在光照的 Light 与 Dark（shadow）部分选择 Hue/Saturation/Value tint
Line Color Shadow - 选择 shadow areas 中的 Line Color


Line Distort Page

New in RenderMan v25!

Enable

开启 Line Distort 功能

Offset

使线条偏离原始位置的 distortion 量

Remap

Remap line distort signal：将 line distort signal（通常是插入到 PxrStylizedControl Distort/Distort U & Distort V input 的 PxrFractal）在 -1 到 1 之间重映射（假设原始范围为 0 到 1）

Mask Page

PxrStylizedControl Mask

使用 PxrStylizedControl pattern 节点中配置的 mask

Input Mask

使用上方 Input/Signal 数据作为先前串联的 Stylized shaders 和/或基于物理的照明的遮罩

Final Line Gamma

线条检测发生后，对 Final Lines 应用 gamma。使用 Curvature Line Type 时，可用此参数控制 Curvature falloff，以微调线稿

Lines Inside Alpha

乘以 alpha，使线条检测得到的线条仅位于内部，不越出 geo

Compositing Page（合成页面）

Mode

Over - 使用 Lines 的 alpha，合成到上一层之上
Plus - 使用 Lines 的 alpha，与上一层相加
Multiply - 使用 Lines 的 alpha，与上一层相乘

Linework Tips

  • 为了让线条更扎实，可尝试将 Weight Type 改为 Hermite 或 Double Sigmoid

  • 将纹理或 pattern 插入 PxrStylizedControl 节点的 Line Thickness input，以破碎并改变 width（线宽），为线条增加创意变化。

  • 尝试通过叠加多个 PxrStylizedLines filter 来探索线稿；每个 filter 都有自己独特的宽度、颜色和厚度。
