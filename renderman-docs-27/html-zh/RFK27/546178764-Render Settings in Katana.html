Render Settings in Katana

Source: https://rmanwiki-27.pixar.com/space/RFK27/546178764/Render+Settings+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

在 RenderMan for Katana 中控制渲染的设置，大致可分为两类：渲染输出和渲染输入。Render “output” 指摄影机、分辨率和裁剪窗口等内容；这些属性主要位于 RenderSettings 节点中。Render “input” 指 RenderMan 选项和属性，以及 RfK 插件配置。访问这些内容主要通过 PrmanGlobalStatements 和 PrmanObjectStatements 节点。

RenderSettings

RenderSettings 节点是 Katana 原生节点，Katana 在线文档中有详细说明。最常需要注意的属性包括：

Attribute（属性）

默认值

说明

cameraName

/root/world/cam/camera

用于渲染的摄影机。

renderer

prman

要使用的渲染器。

resolution

512x512

输出图像的尺寸。

RenderSettings 节点中的常用属性

PrmanGlobalStatements 和 PrmanObjectStatements

RenderMan 的强大能力和灵活性意味着用户可以通过多种设置获得大量控制。这些设置都通过 PrmanIntegratorSettings、PrmanGlobalStatements 和 PrmanObjectStatements 在 RfK 中公开。PrmanGlobalStatements 中的属性用于控制 RenderMan 的全局选项。  

PrmanIntegratorSettings 设置 integrator（积分器），用于 RenderMan 渲染及其关联控制。生产中通常选择 PxrPathTracer integrator（积分器）。

PrmanGlobalStatements 用于按位置设置 Attributes（属性）。由于这两个节点中经常使用的设置数量都不多，因此默认情况下二者都会隐藏不太常用的属性。 Parameter Layout 设置用于根据常见使用场景展开或折叠属性集合：  Simple、Advanced， 或 all。 不过，即使是 Simple 布局也会公开许多设置。下面列出最有用的设置：

Attribute（属性）

默认值

说明

pixelVariance

0.005

像素值相对于真实像素值的可接受估计方差上限。

maxSamples

0 (64)

采样数的显式上限。提高该值可获得更高质量，但渲染更慢。

renderDisplay

Monitor

Preview 和 Live Rendering 的目标帧缓冲。设为 'it' 可使用 RenderMan Image Tool。

PrmanGlobalStatements 中的常用 Options（选项）


PrmanObjectStatements 连接到场景图中的特定位置（例如对象、层级或 CEL 分组）。该节点常用于控制 visibility 属性和 trace depth。下面说明 PrmanObjectStatements 中的一些常用设置：

Attribute（属性）

默认值

说明

visibility.camera

Yes

控制后续图元对摄影机的可见性。

visibility.transmission

Yes

控制图元对透射（阴影）射线的可见性。

trace.maxdiffusedepth

1

限制与关联图元相关的间接照明漫反射反弹次数。

trace.maxspeculardepth

2

限制与关联图元相关的间接照明镜面反弹次数。

PrmanObjectStatements 中的常用 Options（选项）


作为额外辅助，这些节点各自都有设置预设，会根据渲染类型调整 RenderMan 控制：  Default、Draft、Intermediate， 或 High Quality。  这些预设之间的差异很小但很重要。它们最适合用于快速切换和参考值。注意，对设置所做的更改不会“粘附”到该预设；如果切换到其他预设，这些更改会丢失。如果需要设置符合特定需求的预设集合，请参阅自定义预设页面。 

错误处理

渲染期间遇到的错误会根据属性 "prmanGlobalStatements.errorHandler" 处理：

Error Handler

行为

ignore

忽略所有错误，且不生成诊断消息

print

每次发生错误时都为该错误生成诊断消息

渲染系统会尝试忽略错误信息并继续渲染

printonce

仅在错误首次发生时生成诊断消息

abort

第一个错误会导致生成诊断消息，并且渲染系统会立即终止

abortall

对 Katana 生成的错误以及 prman 错误都中止渲染

RenderMan 或 Katana 的错误处理可以通过使用 错误处理器覆盖属性函数来覆盖。这些属性函数提供了按位置或按特定错误更改错误处理行为的机会。attribute function 预期返回一个包含整数 "abort" 属性的 attribute，用于指示是否应终止渲染。

Error handler override functions（错误处理器覆盖函数）应添加到全局属性 "prmanGlobalStatements.plugin.katanaErrorHandlerAttrFncs" 和 "prmanGlobalStatements.plugin.rendermanErrorHandlerAttrFncs"。

日志记录

当前日志级别默认为 "INFO"，由 KATANA_HOME 中的日志配置文件决定。要更改日志级别，需要复制配置文件、修改级别并设置环境变量（使用 Linux）：

mkdir $KATANA_RESOURCES/Logging

cp $KATANA_HOME/bin/log.conf $KATANA_RESOURCES/Logging/log.conf

编辑 log.conf，并将 "rootLogger" 改为所需级别："ERROR"、"WARN"、"INFO"、"DEBUG"


Windows 中的工作流类似：创建 $KATANA_RESOURCES/Logging 文件夹，复制 log.conf 文件，并将 "rootLogger" 值改为列出的选项。
