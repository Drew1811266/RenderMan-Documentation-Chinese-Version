Light Arrays

Source: https://rmanwiki-27.pixar.com/space/RFK27/546178401/Light+Arrays
Downloaded: 2026-06-15T03:49:12+00:00

Light arrays（灯光数组）与 instance arrays 非常相似。主要区别是你不需要 instance source location。Light arrays 可以用 OpScript 创建。  参见 lightArray.katana 文件，其中有一个示例展示如何从 point cloud（点云）创建 light arrays。 

如果你从 GafferThree 中的一个 light 创建 light array，需要确保 location type 的更改是在 ImplicitResolvers 运行之后 才发生。例如，如果你通过 OpScript 更改 location type，请将 executionMode 设为 "deferred"。这样 light array 就能遵循 GafferThree light 的 mute 状态。

创建 light array 的步骤如下：

  • 在 GafferThree 中创建一个 light location。这个 light 上已设置的任何 light filters（灯光滤镜）和 light linking（灯光链接），在该 light 转换为 light array 后都会被保留。

  • 将 light 的类型设为 "light array"。

  • 设置 geomety.instanceMatrix，用它表示 light 的每个 instance 的 transform。light array 的 local transform 会应用到整个 light array，而不是应用到各个单独的 lights。

以下是可以在 light array location 上设置的属性。只有 geometry.instanceMatrix 是必需的：

geometry.instanceMatrix

对于 instance array 中的每个 instance，instanceMatrix 中都有 16 个值，用来表示该 instance 的 transformation matrix。注意，这个 transform 是相对于 instance array 自身的 transform。

geometry.instanceMaterialOverride

可以用此属性覆盖 light 的 attributes。对于每个被覆盖的 attribute，每个 instance 都需要一个元素。例如，要覆盖 intensity，就设置 geometry.instanceMaterialOverride.prmanLightParams.intensity attribute。注意，设置此 attribute 后，你就不再是在实例化该 light，因为 RenderMan light parameters 不能按 instance 变化。
