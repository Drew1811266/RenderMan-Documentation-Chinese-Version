安装路径变量

你需要设置几个环境变量，才能让 RfK 与 PRMan 和 Katana 一起正常运行。RfK 会通过 $RMANTREE 环境变量查找 RenderMan。只要 $RMANTREE 设置正确，RfK 就会找到相应的 PRMan 库和命令。Katana 查找 RfK 的方式与查找任何其他插件相同，即通过 $KATANA_RESOURCES 环境变量。这些环境变量的路径取决于你是否将 RfK/PRMan 安装在默认位置。  

环境变量

默认安装路径

RMANTREE

/opt/pixar/RenderManProServer-27.0

KATANA_RESOURCES

/opt/pixar/RenderManForKatana-27.0/plugins/katana8.0

RenderMan 的默认着色器搜索路径与 rendermn.ini 文件中的默认路径一致。如果需要额外搜索路径，则需要设置 RMAN_SHADERPATH 和/或 RMAN_RIXPLUGINPATH 环境变量以包含你的特定路径。这些环境变量会替换默认路径，而不是追加到默认路径，因此你还需要同时包含默认路径。

输出路径变量

如有必要，可以使用以下环境变量将渲染日志输出定向到其他位置： RFK_REDIRECT_OUTPUT

着色器发现

启动期间，RenderMan for Katana 会自动加载所有“可发现”的着色器。如果着色器位于搜索路径中，并且（对于非 OSL 着色器）有关联的 Args 文件，则该着色器就是可发现的。在指定的环境变量搜索路径中搜索着色器时，会使用标准着色器搜索路径机制：

着色器类型

环境变量

默认路径

OSL

RMAN_SHADERPATH

$RMANTREE/lib/shaders

插件 (C++)

RMAN_RIXPLUGINPATH

$RMANTREE/lib/plugins

如果同时设置了环境变量和 PrmanGlobalStatements 中的搜索路径属性，最终搜索路径将是这两个字符串的并集，并且环境变量优先。

递归

RfK 会自动沿着着色器路径递归查找子目录中的着色器。默认情况下，它会向下递归 3 层子目录。可使用 shaderPathRecursionLimit RfK 配置文件中的设置 config.xml。

Args 文件

Args 文件需要放在 Args 目录中。可参考 RMANTREE/lib/plugins 中的设置示例。大致如下：

$RMAN_RIXPLUGINPATH/pattern_remap.so

$RMAN_RIXPLUGINPATH/Args/pattern_remap.args

可以在开发者文档中了解更多 Args 文件信息 下载。
