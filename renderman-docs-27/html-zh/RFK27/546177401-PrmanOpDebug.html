PrmanOpDebug

Source: https://rmanwiki-27.pixar.com/space/RFK27/546177401/PrmanOpDebug
Downloaded: 2026-06-15T03:49:12+00:00

Katana 使用 Op 来构建和修改场景的属性状态。有些 Op 会在交互式 Katana 会话中产生可见效果，但有些会在后台运行。如果你编写自己的 terminal op，或者正在调试某个属性为什么没有进入渲染，那么在你的 Op 运行位置检查属性状态会非常有用。PrmanOpDebug 节点可让你检查完整的 Op 集合，包括 Interactive Render Filters (IRFs)、Implicit Resolvers 和 Terminal Ops。

如何使用该节点

  1. 将该节点插入到 Node Graph（节点图）的末尾。如果有 Render node（渲染节点），则放在它正前方

  2. 该节点会自动填充当前的 IRF、Implicit Resolver 和 Terminal Op 状态。

  3. 展开 Parameters Tab（参数选项卡）中的每个组，即可查看完整 Op 列表

  4. 要查看某个 Op 的效果，可以右键单击该 Op 名称并选择 "View at Location"

  1. 可以在 Katana 的 Attributes Tab（属性选项卡）中查看属性状态。

  2. 请注意，此节点创建后，不会拾取场景 Op 和 IRF 状态的任何变化。每种 Op 类型在其对应的 user 组下都有一个刷新按钮。
