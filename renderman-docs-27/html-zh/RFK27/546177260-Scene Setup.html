Scene Setup

如果你想跳过所有说明，可以直接查看一个简单示例场景，它是 Katana "hello_pony" 演示场景的 RenderMan 版本（hello_pony_prman.katana 位于 $KATANA_RESOURCES/Examples）。下面会更详细地说明如何用一个球体和一个环境光来构建类似场景的每个阶段。

Create Camera and Geometry（创建摄影机和几何体）

可以用很多方式把几何体带入 Katana 场景；但作为入门，只需使用 PrimitiveCreate 节点向场景添加一个球体。然后使用标准 CameraCreate 节点创建一个 Katana 摄影机：

默认情况下，摄影机和球体都会创建在原点。根据需要重新定位摄影机和/或球体，让球体位于摄影机的视场角中。使用 Viewer 选项卡可以快速通过摄影机查看并重新框选球体，也可以在 Viewer 中使用操纵器移动对象。

Add Lights（添加灯光）

RenderMan 不能使用 Katana 内置灯光或其他渲染器的灯光；你需要使用内置  RenderMan lights（RenderMan 灯光）。这些随 RfK 提供的灯光现在可以通过 GafferThree 节点中的 Add（添加）  上下文菜单使用。创建一个 GafferThree 节点，然后按 E 以便在 Parameters 选项卡中将其设为可编辑。在 Parameters 选项卡中，右键单击 Gaffer object 表并选择  Add（添加）。你会看到一个上下文菜单，其中除了简单的 "light" 选项外，还有几种 RenderMan（"Pxr"）灯光类型。创建一个 PxrEnvDayLight：

 

RenderMan 灯光上几个常调整的控制项是 exposure（曝光） 和 visibility（可见性）。灯光具有物理衰减；如果场景中的灯光看起来偏暗，你可能需要调高 exposure（曝光）。exposure 属性的工作方式类似胶片摄影机的曝光：增加 1 会让光源发出的能量翻倍。

默认情况下，灯光对摄影机不可见；因此如果你想在背景中看到日光颜色，需要让 PxrEnvDaylight 对摄影机可见。此设置包含在 Object（对象） RenderMan 灯光设置中：

它并不令人兴奋，但这是确认所有组件都已就位的好测试（至少目前如此！）。如果渲染期间出现任何错误，请检查你的安装和搜索路径。如果安装了其他渲染器，需要确保 RenderSettings 已设置为 renderer=prman。

右键单击节点时，也可以从上下文菜单中选择 Live Render（实时渲染） 这样你就可以在修改场景时查看变化，而无需重新启动渲染。

Add Materials（添加材质）

让我们稍微美化一下这个球体。所有新的 Pixar Surface Materials（Pixar 表面材质） 都可以在 RfK 中的 Material 节点和 PrmanShadingNode 的着色器列表里使用。对于这个简单场景，我们只创建一个 Material 节点，然后选择 Add Shader->bxdf：

从 prmanBxdfShaders 列表中选择 PxrSurface：

要将 Material 与其几何体连接起来，需要创建并连接一个 MaterialAssign 节点，并将 materialAssign 属性设置为材质位置。最简单的方法是按住 Shift 并用鼠标中键单击拖动，将 Material 节点直接拖到 MaterialAssign 节点上：

如果启用 MaterialAssign 节点进行编辑（将鼠标悬停在该节点上并按 E），也可以从 Material 节点按住 Shift 并用鼠标中键单击拖动到 Parameter 选项卡中的 materialAssign 框中。用类似方式把 primitive 添加到 Material CEL 位置，即可完成材质分配：

 

此时渲染仍会得到一个漫反射灰色球体，所以我们通过调整 Color（颜色）、 Fresnel（菲涅耳） 和 Roughness（粗糙度） 来让表面更有光泽，这些参数位于 Primary Specular（主镜面反射）中。  于是我们得到了 Sunrise on a Sphere（球体上的日出）：

Patterns（图案）

PxrSurface 的属性可以连接到 RenderMan 丰富 pattern（图案） 节点集的输出。可以使用 PrmanShadingNode 而不是 Material 节点来设置 PxrSurface bxdf，然后将该节点连入 NetworkMaterial 节点，从而添加纹理、凹凸贴图和程序化颜色等内容。这是一个 shading network（着色网络）的起点，你可以按自己的创意继续扩展。

我们给场景添加一个带有基于 fractal（分形）着色的地面平面。使用另一个 PrimitiveCreate 添加平面。然后创建一个 NetworkMaterial 节点，并从 bxdf 中选择 Add Terminal（添加终端） 菜单。创建一个 PrmanShadingNode，并将 NodeType 设置为 PxrSurface。添加新的 MaterialAssign 节点，并像给球体分配材质那样将平面连接到 NetworkMaterial。稍作整理并协调颜色后，节点图现在已经设置好一个灯光、一个摄影机和两个几何体，每个几何体都分配了 PxrSurface 材质：

我们通过一系列 PrmanShadingNodes 添加 fractal shading（分形着色）；每个节点都带有一个 Pattern，将输出馈送到下一个节点，最后一个节点把结果馈送到 Color（颜色） 输入，也就是 PxrSurface 节点的输入：

Fine Tuning（精细调整）

现在你已经有了一个可渲染场景，可以开始通过调整着色器和渲染器本身来精细调整渲染。大多数着色器调整都可以在 Live Rendering（实时渲染）期间完成。另一方面，渲染配置必须在渲染开始前设置（例如分辨率、采样、integrator（积分器））。RfK 有多个位置可用于配置渲染器属性。下面讨论最常用的设置；关于这些设置以及其余设置的更多细节，可以在 Render Settings（渲染设置）页面中找到。

General Render Settings（常规渲染设置）

如果 RenderMan 是你的 Katana 会话中唯一安装的渲染器，那么渲染时会默认使用它。如果还安装了其他渲染器，则需要创建一个 RenderSettings 节点，并将 Renderer 属性设置为 "prman"：

RenderSettings 节点还用于控制以下内容：

  • image resolution（图像分辨率）

  • camera（摄影机）

  • crop window（裁剪窗口）

  • shutter open/close（快门开启/关闭）。

RenderMan Controls（RenderMan 控制项）

RenderMan-specific rendering controls（RenderMan 专用渲染控制项），例如 Options 和 Attributes，可在 PrmanGlobalStatements 节点中使用。其他 RfK 配置选项也会在这里公开。例如，如果你希望渲染结果发送到 RenderMan Image Tool ('it') 而不是 Monitor 窗口，可以使用 renderDisplay 属性在 PrmanGlobalStatements 中进行控制。
