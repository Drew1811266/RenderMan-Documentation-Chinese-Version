PxrVary Example（PxrVary 示例）

动画或视觉特效的着色中最常见需求之一，是为许多对象高效改变着色器属性。这可以用于群集模拟，也可以用于森林中的树叶。这种变化可以让场景更真实且更具视觉吸引力。然而，手动修改几十到几百个着色器往往成本过高。这时 PxrVary 就派上用场！

附带一个 Katana 示例文件，展示在使用该 Pattern 时如何使用 OpScripts 生成源变量。请阅读下方关于场景设置逻辑的说明。

  • 注意，Identifier ID 不需要 OpScripts 来定义 ID 也能工作。我们会自动生成 identifier ID 和 identifier Name，因此它们无需 OpScript 也能工作。


  • 对于 Attribute selections（属性选择），请确保在 PxrVary 节点中加上 "user:" 前缀。在本例中使用 user:rmanFloatAttr


  • 你可以在 Katana documentation中找到更多关于 Primvars 的有用信息。


这是一个用于在球体上创建 ID 的示例 OpScript。

Interface.SetAttr('prmanStatements.attributes.identifier.id', IntAttribute(1))

Interface.SetAttr('prmanStatements.attributes.user.rmanFloatAttr', FloatAttribute(3))

gb = GroupBuilder()
gb:set("value", FloatAttribute(1))
gb:set("scope", StringAttribute("primitive"))
gb:set("inputType", StringAttribute("float"))
Interface.SetAttr("geometry.arbitrary.FloatPrimvar", gb:build())


OpScript 被分配在 primitive create 正下方。每个 OpScript 略有不同，以确保变化。

这里可以看到有四个不同输出，展示使用不同 PxrVary 源（Name、ID、PrimVar、Attribute）得到的不同结果。渲染每个输出都会生成不同颜色的球体。下载场景进行实验！
