创建网格灯

RenderMan 中的网格灯是使用 PxrMeshLight 创建的。任何一段几何体都可以用作网格灯（也称为“几何区域光”）。下面介绍在 RfK 中创建网格灯的两个主要工作流。 

方法 1：PxrMeshLight GafferThree Package

使用这种方法创建网格灯很简单。首先，在 Scenegraph 中选择要转换为网格灯的几何体。然后在 GafferThree 中添加一个 PxrMeshLight。

Mesh Light Geometry 参数也可以引用带有 Scene Graph 位置的节点。如果几何体的名称或位置发生变化，Node Graph 引用会自动更新该参数。

方法 2：geometry.areaLightGeometrySource 属性

网格灯几何体需要使用 RfK 在处理灯光时会查找的特定属性附加到灯光。该属性名为 "geometry.areaLightGeometrySource"。此属性是灯光上的字符串属性，其值应为几何体位置。为方便起见，RfK 发行版中的 PrmanMeshLightCreate 宏演示了这种设置。

方法 3：指定 PxrMeshLight 材质

你可以为对象指定 PxrMeshLight 材质来创建网格灯。不过，这样将无法利用灯光链接和灯光过滤器选项。

实例化网格灯

如果你使用上面描述的方法 1 或方法 2 创建了网格灯，场景中会有两个位置与同一个网格灯关联。那么应该实例化哪个位置？始终应当实例化几何体位置。

如果使用层级实例化或实例数组，请确保网格灯位于实例源位置之下，这样 RfK 才会保留与灯光位置关联的材质属性和灯光过滤器。

网格灯也可以用于灯光数组。在这种情况下，RfK 使用来自灯光数组位置的材质，而不是几何体位置上的材质。

其他说明

几何体可以在有或没有 Bxdf 的情况下渲染。如果没有指定其他 Bxdf，RfK 会为网格灯指定 PxrBlack（不适用于方法 3）。如果几何体指定了 Bxdf，则会根据该着色器的特性渲染该几何体。

建议提高网格灯上的 micropolygonlength，以改善性能。此属性可在 PxrMeshLight GafferThree package 的 Object Tab 中找到。

如果要使用 PxrMeshLight 的 Texture Color 参数，可以使用 PrmanShadingNodes 和 NetworkMaterial 节点创建着色网络。然后，在创建该灯光的 GafferThree 下游，使用 MaterialAssign 节点将 NetworkMaterial 指定给灯光位置。
