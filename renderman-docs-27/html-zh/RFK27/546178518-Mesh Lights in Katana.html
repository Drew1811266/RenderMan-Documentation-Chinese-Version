创建 Mesh Lights（网格灯）

RenderMan 中的 Mesh Lights 是使用 PxrMeshLight 创建的。任何一段 geometry 都可以用作 mesh light（也称为 "geometric area light"）。下面介绍在 RfK 中创建 mesh lights 的两个主要工作流。 

方法 1：PxrMeshLight GafferThree Package

使用这种方法创建 mesh lights 很简单。首先，在 Scenegraph 中选择要转换为 mesh light 的 geometry。然后在 GafferThree 中添加一个 PxrMeshLight。

Mesh Light Geometry 参数也可以引用带有 Scene Graph location 的节点。如果 geometry 的名称或 location 发生变化，Node Graph 引用会自动更新该参数。

方法 2：geometry.areaLightGeometrySource Attribute

Mesh light geometry 需要使用 RfK 在处理 lights 时会查找的特定 attribute 附加到 light。该 attribute 名为 "geometry.areaLightGeometrySource"。此 attribute 是 light 上的 string attribute，其值应为 geometry location。为方便起见，RfK 发行版中的 PrmanMeshLightCreate macro 演示了这种设置。

方法 3：指定 PxrMeshLight Material

你可以为对象指定 PxrMeshLight material 来创建 mesh light。不过，这样将无法利用 light linking 和 light filter 选项。

实例化 Mesh Lights

如果你使用上面描述的方法 1 或方法 2 创建了 mesh light，场景中会有两个 location 与同一个 mesh light 关联。那么应该 instance 哪个 location？始终应当 instance geometry location。

如果使用 hierachical instancing 或 instance arrays，请确保 mesh light 位于 instance source location 之下，这样 RfK 才会保留与 light location 关联的 material properties 和 light filters。

Mesh lights 也可以用于 light arrays。在这种情况下，RfK 使用来自 light array location 的 material，而不是 geometry location 上的 material。

其他说明

geometry 可以在有或没有 Bxdf 的情况下渲染。如果没有指定其他 Bxdf，RfK 会为 mesh lights 指定 PxrBlack（不适用于方法 3）。如果 geometry 指定了 Bxdf，则会根据该 shader 的特性渲染该 geometry。

建议提高 mesh light 上的 micropolygonlength，以改善性能。此 attribute 可在 PxrMeshLight GafferThree package 的 Object Tab 中找到。

如果要使用 PxrMeshLight 的 Texture Color 参数，可以使用 PrmanShadingNodes 和 NetworkMaterial 节点创建 shading network。然后，在创建该 light 的 GafferThree 下游，使用 MaterialAssign 节点将 NetworkMaterial 指定给 light location。
