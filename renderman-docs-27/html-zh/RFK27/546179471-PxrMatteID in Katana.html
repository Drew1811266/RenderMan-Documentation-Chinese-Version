PxrMatteID in Katana

Source: https://rmanwiki-27.pixar.com/space/RFK27/546179471/PxrMatteID+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

分配 Matte ID 给已渲染对象分配 Matte ID，并将它们作为 AOV 的一部分输出，对后期合成很有用，因为合成师可以用它隔离对象或材质。

最基本的做法可以分几步完成。在本教程中，我们会完成设置 Matte ID 的流程，用于在合成中分离具有不同材质的对象。本教程使用的场景可在 RfK Examples 目录中找到：matteid.katana。

下面是一个包含四种不同材质的简单示例。每个 MatteID 输出可以包含三个遮罩，因为颜色输出有三个通道：红、绿、蓝。由于此场景中有四种材质，因此需要使用两个 MatteID 输出。

可以在这里找到示例文件以便跟随操作： matteid.zip

第 1 部分：设置 MatteID 属性

首先，我们会为球体分配绿色 MatteID。这通过在球体上设置用户属性完成。以下是可用于设置此属性的 OpScript：

MatteID OpScript
gb = GroupBuilder()

gb:set("value", FloatAttribute({0.0, 1.0, 0.0}, 3))

gb:set("type", StringAttribute("color"))

Interface.SetAttr("prmanStatements.attributes.user.MatteID0", gb:build())

在立方体和圆锥体上创建相同的用户属性，但将值设置为剩余两个颜色通道：蓝色和红色。对于地面平面，需要设置下一个可用的 MatteID：MatteID1：

MatteID OpScript
Interface.SetAttr("prmanStatements.attributes.user.MatteID1", gb:build())

第 2 部分：用两种方式关联 ID

  1. 要将对象或着色器关联到 MatteIDx，需要向该对象添加一个特殊属性：
    在每个 location 上设置 prmanStatements.attributes.user.MatteIDx 属性，用于在 MatteIDx AOV 中为该对象分配颜色。
    每个 MatteIDx AOV 可以包含 3 组对象，分别对应每个 RGB 通道。在此示例中有 4 个对象，因此前三个放在 MatteID0 AOV 上，最后的平面放在 MatteID1 AOV 上。

  2. 如果你觉得通过材质 location 分配 MatteID 用户属性比在对象上分配更方便，可以使用 material.underlayAttrs 属性。
    在每个 location 上设置 material.underlayAttrs.prmanStatements.attributes.user.MatteIDx 属性，用于在 MatteIDx AOV 中为该对象分配颜色。

第 5 部分：使用 MatteID Texture 分离材质

如果场景中的每个对象都有不同材质，或者只想隔离场景中的每个对象，我们可以在第 3 部分停止。但由于球体有两种材质，并且我们要隔离不同材质，因此需要使用 PxrMatteID 的 Matte Texture 参数。Matte Texture 参数允许将颜色或 pattern 分配为每个遮罩的输出。目标是得到一个遮罩 AOV，其中球体的紫色部分为绿色，金色部分为蓝色，以匹配立方体的遮罩。

如果还记得，我们在第 2 部分中没有对 PxrMatteID 节点做任何更改，它保持默认状态。我们会保留这个节点不变，但将它从分配给球体的 PxrSurface 上断开。这样做是为了只影响一个对象，而不是全部三个。

创建新的 PxrMatteID，并将它连接到球体的 PxrSurface。在这种情况下，球体有一个唯一材质，以及一个与立方体相同的材质，因此仍可以使用 MatteID0 和 Matte Texture 0。将 PxrChecker pattern 连接到 PxrMatteID 的 Matte Texture 0 参数。将棋盘格颜色设置为蓝色和绿色。

好了，这算是起作用了，但球体中本应为蓝色的部分变成了黑色。这是因为 MatteID 颜色会与连接到 PxrMatteID 的 pattern 相乘。为了得到正确的蓝/绿棋盘格输出，需要更改之前在球体上设置的 MatteID0 用户属性。将该值设置为白色，然后再次渲染。

任务完成！现在可以添加额外的 PxrMatteID，并把更多 pattern 链接起来，在渲染中获得不同结果。你的合成师会感谢你。
