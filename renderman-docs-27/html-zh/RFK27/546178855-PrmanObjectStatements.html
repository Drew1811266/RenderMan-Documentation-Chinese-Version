PrmanObjectStatements

Source: https://rmanwiki-27.pixar.com/space/RFK27/546178855/PrmanObjectStatements
Downloaded: 2026-06-15T03:49:12+00:00

PrmanObjectStatements 会分配到一个特定位置或一组位置，并在转换为 prman 时设置一个 AttributeBegin/AttributeEnd 块的状态。此节点可用于为对象分配细分近似以渲染平滑表面、为置换渲染应用置换边界，以及更改可见性标志和成员关系。


在 RenderMan 中，可以通过不同方式选择对象及其实例的渲染方式。这些设置在该节点中方便地分为：

部分

说明

attributes

可随实例变化的属性（例如 lighting subset 或 camera visibility)

primAttributes

被视为 "prototype"（原型）属性的按图元属性（在下表中称为 "Primitive Variables"）

primvars

属于图元定义的常量按图元属性。这些属性不能按实例改变。

traversal

用于场景遍历的 RfK 插件控制。


这些设置列在下面的表格中。注意，这些表格目前并不与 PrmanObjectStatements UI 中参数的呈现方式完全对齐。

Traversal

当 RfK 遍历 Katana 场景图时，它会并行处理位置，并将它们传递给 RenderMan 进行渲染。默认情况下，从 Katana 取得的每个新的子位置都会生成一个新线程进行处理。这个默认行为不太可能为任意给定场景提供最佳性能；不过 PrmanObjectStatements 提供了用于调优特定管线（例如你所在工作室的标准 Op Chain）或多种场景类型（例如许多实例、许多纹理、许多图元，或三者都有）中的效率和并行级别的设置。

Force Serial

第一个工具是用于控制场景中并行深度的属性： forceSerial。

例如，考虑一个场景：

/root/world/geo/pine_tree
/root/world/geo/elm_tree
/root/world/geo/oak_tree

其中每个树位置都包含分支、树叶、根等。

理想情况下，应在足够高的层级并行遍历，避免让每片树叶或每个分支都在不同线程中处理。在 RfK 中，我们通过在希望停止并行处理的位置上将 "prmanStatements.traversal.forceSerial" 设置为 True。当遍历期间遇到该属性时，RfK 会在该位置切换为串行，并且所有子位置都会在单个线程中运行。

在上面的示例中，将 "/root/world/geo/*_tree" 上的 "forceSerial" 设为 true，意味着每棵树会在不同线程上处理，但每棵树的所有内容会在其各自线程中串行处理。你需要评估自己的场景设置，判断在哪个层级强制串行处理才合理。

每个线程都有自己的 Geolib 运行时，其中包含已求值位置的逐线程缓存。因此，任何 Op 所需的所有位置都必须按线程求值。继续上面的示例：如果有三个线程分别查看 "/root/world/geo" 下的不同位置，你会看到 "/root" 被求值三次，"/root/world" 被求值三次，"/root/world/geo" 也被求值三次。如果以串行运行，或在 "/root" 处将并行遍历强制为一个线程，那么只有一个 Geolib 运行时会对该位置求值，因此你只会看到该 Op 在 "/root" 处打印一次。

Evict Here

我们用于提高并行遍历效率的第二个工具是 evictHere 设置。 

默认情况下，当一个 Katana scenegraph 位置被处理并转换为 Rix scenegraph 时，该位置的同级位置会从线程的运行时内存中逐出。若已知这些同级位置最好保留在缓存中供后续访问，则将此标志设为 false 可防止在该位置进行同级位置逐出。

Flatten Instance Source

最后一个工具是 flattenInstanceSource 设置。

此设置并不会改变 RfK 对 Katana 场景的并行遍历行为，而是在构建实例时改善 RenderMan 的遍历。默认情况下，RenderMan 会将一个实例源扁平化为组中每个几何图元对应的独立对象原型。随着实例数量增加，或实例源内部图元数量增加，这可能变得昂贵。将此选项设为 Yes 会告诉 RenderMan 将这个实例源位置保持为单个对象原型。

此设置会覆盖 prmanGlobalStatements.plugin.flattenInstanceSources 的全局设置；该全局设置可以在所有实例源位置上启用此设置。
