PrmanGlobalStatements

Downloaded: 2026-06-15T03:49:12+00:00

PrmanGlobalStatements 节点是 RenderMan 专用节点，包含用于配置 RenderMan 全局 Options（选项）的 attributes。

PrmanGlobalStatements attribute 与其对应的 RenderMan option 通常是一一映射关系。此外，还有若干 RfK 专用 plugin control options，下面会详细列出。


  • Common（通用）
  • Baking（烘焙）
  • Shading（着色）
  • Dicing（细分）
  • Display（显示）
  • Statistics（统计）
  • Configuration（配置）
  • RIB
  • OSL
  • Volumes（体积）

Common（通用）

Name（名称）

Type（类型）

Default（默认值）

Description（说明）

hider:adaptall

int

0

启用所有 AOVs 的 adaptive sampling（自适应采样）。启用后，在判断 pixel 是否收敛时会考虑所有 channels（beauty pass + AOVs）。默认值（off）会忽略 AOVs，只考虑 beauty pass 的 pixel convergence。因此，如果某些 AOVs 从未达到指定 variance threshold，关闭 "adaptall" 可能会加快渲染。Value range（取值范围）：true/false。

hider:adaptivemetric

string

variance

指定用于驱动 adaptive sampling 的 metric。"contrast" 测量一个 pixel 中 samples 之间的 contrast。"variance" 测量一个 pixel 中 samples 的 variance。"halfbuffer" 将 samples 分成两类，并测量两个 sample classes 各自平均值之间的差异。"contrast-v22" 使用 RenderMan 22 及更早版本的 relative contrast computation。"variance-v22" 使用 RenderMan 22 及更早版本的 relative variance computation。Supported values（支持的值）：contrast、variance、halfbuffer、contrast-v22、variance-v22。

hider:bluenoise

int

1

用于控制 samples 是否应具有视觉上令人满意的 blue noise distribution 的设置。Default（默认值）为 1（on）。如果相邻 pixels 中的 samples 需要完全 decorrelated，则设为 0（off）。Value range（取值范围）：true/false。

hider:darkfalloff

float

0.025

在 adaptive sampler 中降低 low luminance samples 的优先级。使用此参数可避免对暗部区域过度采样，同时保持整体 image quality。较高值会剔除更多 samples，加快 render time，但会增加暗部区域 noise。Value range（取值范围）：0 到 1。

hider:decidither

int

0

interactive rendering 允许的最大 pixel decimation rate。在快速编辑重型 scenes 且此设置非零时，renderer 会尝试用类似 dissolve 的效果逐步更新整个屏幕，以减少 tearing。将其设为 zero 可禁用此行为，并强制它完整更新能处理到的 buckets。Value range（取值范围）：0 到 6。

hider:exposurebracket

float[2]

-1.0 1.0

post-production 中用于帮助指导 adaptive sampling 的预期 exposure Bracket [min max]。

hider:extrememotiondof

int

0

启用改进的 sampling accuracy。在某些同时使用大量 motion blur 和 depth of field 的情况下，sampling artifacts 可能导致不理想结果。此 flag 会使用较慢但更准确的 sampling method 来缓解 artifacts。由于这种 sampling method 会降低性能，因此只应在必要时使用此 flag。Value range（取值范围）：true/false。

hider:geomShadowTermBias

int

1

此设置控制何时计算并向 Poly Meshes 应用 bias，以避免 geometric shadow terminator artifacts；这些 artifacts 会出现在 shading points 从 light 看相对于 shading normal 可见、但相对于 geometric normal 不可见的情况下。默认值为 1（on）。如果不希望将此 bias 应用于 scene 中的 Poly Meshes，请设为 0（off）。Value range（取值范围）：true/false。

hider:incremental

int

0

启用结果的 progressive display（渐进显示），而不是 bucketed results。Value range（取值范围）：true/false。

hider:jitter

int

1

此 option 切换 camera samples 中的 jitter（variation）。默认情况下，此 hider option 已启用（设为 1），意味着 camera samples 会随机放置在 pixel location 中。关闭时，camera rays 会放置在 pixel 中心。关闭后，surface edges 会显得 aliased 和 jagged。Value range（取值范围）：true/false。

hider:maxsamples

int

0

samples 数量的显式限制。如果 maxsamples 等于其默认值，则最大 samples 数将设为 64。注意 minsamples 默认等于 maxsamples 设置的平方根。Value range（取值范围）：0 或更大。

hider:minextrasamples

int

-1

这是一个 sample 未通过 adaptive sampler 的 convergence test 后，pixel 应继续发射的最小 extra samples 数。默认匹配 minsamples。

hider:minsamples

int

-1

指定 adaptive sampling 的最小 samples 数。默认值是 maxsamples 的平方根。

hider:pixelfiltermode

string

importance

指定如何执行 camera samples 的 pixel filtering。"Importance"（默认值）会在 screen space 中围绕 pixel center warp samples，并且不在 pixels 之间共享 sample results。某些 postprocessing algorithms（例如 denoisers）需要 "Importance"，要获得 blue-noise samples 的视觉收益也需要它；生成 variance AOVs 也需要它。"Weighted" 是较旧的方法，会混合相邻 pixels 的 sample results。Supported values（支持的值）：weighted、importance。

hider:samplemotion

int

1

此 option 切换 hider 中 motion blurred micropolygon 的 sampling，因此只有在 scene 使用 motion blur 时才有意义。默认情况下，此 hider option 已启用（设为 1），意味着 micropolygons 会在 hider 中被 blur 并采样。禁用时（设为 0），micropolygons 在采样前不再 blur；不过仍会为所有 objects 计算 dPdtime variable。此 variable 可以作为 arbitrary output variable（aov）输出到单独 image，并用于作为 postprocess 执行 motion blur。Value range（取值范围）：true/false。

hider:sampleoffset

int

0

这允许并行渲染多张 images（使用不同的 sampleoffset values），然后将它们合并。使用 non-adaptive sampling 时：假设用 sampleoffsets 0、256、512 和 768 分别渲染四张 images，每张 256 samples。如果合并这四张 images，将得到与渲染一张 1024 samples image 完全相同的结果。使用 adaptive sampling 时：假设再次渲染四张 images，每张的 "maxsamples" 为 256，sampleoffsets 为 0、256、512 和 768。假设由于 adaptive sampling，某个 pixel 在四张 images 中各只获得 64 pixel samples。那么合并后的 image 使用 sample numbers 0-63、256-319、512-575 和 768-831 渲染。由于 samples 的 stratification，这不如渲染一张具有 256 个 consecutive samples 的 image 那么好。不过，它仍优于只用 64 samples 渲染单张 image。

hider:samplestride

int

1

这有助于并行渲染多张 images（使用相同的 aamplestride 但不同的 sampleoffset），并在 interactive rendering 期间逐步合并它们。Default value（默认值）为 1。使用 non-adaptive sampling 时：假设用 samplestride 4 和 sampleoffsets 0、1、2、3 分别渲染四张 images，每张 256 samples。如果合并这四张 images，将得到与渲染一张 1024 samples image 完全相同的结果，并且在 incremental/interactive rendering 期间可以最好地利用 stratified progressive sample sequences。如果能在 interactive rendering 期间用 samples 0-3、4-7、8-12 等 "assemble" images，就可以获得更快渲染的收益，而不必一开始就跳到高 samples。使用 adaptive sampling 时：假设再次渲染四张 images，每张的 "maxsamples" 为 256，samplestride 为 4，sampleoffsets 为 0、1、2、3。这次使用 adaptive sampling（PixelVariance > 0）。由于四张 images 中每张都比合并 image 更 noisy，adaptive sampling 在判定 image error 可接受之前会使用比单张 image 渲染更多的 samples。因此可能需要调整 PixelVariance 设置。

lighting:minimumestimate

1e-6

light selection 的 minimum estimate。estimate 低于此值的 local lights 会被忽略。增大此值可能有助于加快 direct lighting，但会牺牲 accuracy。

lighting:selectionlearningscheme

1

使用 machine learning algorithm 更准确地对 lights 进行 importance sample，并考虑 occlusion 和 light filters。Value range（取值范围）：true/false。

trace:bvhcompression

float

0

构建 ray accelerator 时，向系统提供倾向 memory utilization 还是 speed 的提示。Valid values 位于 [0, 1] 范围内，值越高，memory representation 越 compact。随着不同 strategies 被激活，此控制在 compression rate 上大体近似线性，但存在一些 discontinuities。获得的结果很可能依赖 scene。经验法则是，除非需要降低 memory utilization 以渲染特定 scene 或 sequence，否则将此控制保持为 0。调优时，先将控制设为 1，查看通过 compression 可节省多少 memory 以及生成什么 time overhead，然后将数值调回较低值（如果可能），以达到 speed 和 memory usage 之间的理想平衡。Value range（取值范围）：0 到 1。

checkpoint:asfinal

int

0

启用后，在写入 render 的 final image 时会保留 extra channels 和 checkpoint tag。Final image 会与任何其他 checkpoint 相同，而不是瘦身后的 image。这意味着当 image 达到设置的 quality 并完成后，用户以后始终可以重新启动它。Value range（取值范围）：true/false。

checkpoint:command

string

empty（空）

checkpoint 后要调用的 system command。如果启用了 system calls，则在写入 checkpoint 后会调用指定 command。这是同步执行的；运行期间 rendering threads 会静止，并且在 process 返回前不会再次恢复，从而避免 command 耗时导致的潜在 race conditions。token %i 会替换为当前 increment，并以 5 位 zero-padded。token %e 会替换为 elapsed time（秒），并以 6 位 zero-padded。token %r 会替换为此次 checkpoint files 更新的 reason（要么完全 `finished`，要么因 exitat option 提前 `exiting`，要么是普通 `checkpoint`）。Literal % 字符可以用 %% 插入。

checkpoint:exitat

string

empty（空）

render 在写入 final checkpoint 并终止前应运行多久。Value 也可以是 "10m" 或 "0.5h" 这样的字符串。设为 0 或 empty 可禁用。

checkpoint:interval

string

empty（空）

每 N 个 progressive steps 将 framebuffers Write（写入）到磁盘。如果 value 为正，其单位会解释为 seconds。如果 value 为负，其单位会解释为 increments。Value 也可以是 "10m" 或 "0.5h" 这样的字符串。将 option 设为 0 或 empty 会禁用 checkpoints。

checkpoint:keepfiles

int

0

如果设为 true，prman 会为每个 checkpoint 写入一个新的编号 image。注意：这会禁用 checkpoint file 的 final write，这意味着 render 无法使用 'recover' option 恢复。Value range（取值范围）：true/false。
