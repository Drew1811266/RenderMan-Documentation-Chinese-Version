PrmanGlobalStatements 节点是 RenderMan 专用节点，包含用于配置 RenderMan 全局选项的属性。

PrmanGlobalStatements 属性与其对应的 RenderMan 选项通常是一一映射关系。此外，还有若干 RfK 专用插件控制选项，下面会详细列出。

类型

说明

hider:adaptall

int

0

启用所有 AOVs 的自适应采样。启用后，在判断像素是否收敛时会考虑所有主图像通道 + AOVs。默认值（off）会忽略 AOVs，只考虑主图像通道的像素收敛。因此，如果某些 AOVs 从未达到指定方差阈值，关闭 "adaptall" 可能会加快渲染。取值范围：true/false。

hider:adaptivemetric

string

variance

指定用于驱动自适应采样的度量。"contrast" 测量一个像素中样本之间的对比度。"variance" 测量一个像素中样本的方差。"halfbuffer" 将样本分成两类，并测量两个样本类别各自平均值之间的差异。"contrast-v22" 使用 RenderMan 22 及更早版本的相对对比度计算。"variance-v22" 使用 RenderMan 22 及更早版本的相对方差计算。支持值：contrast、variance、halfbuffer、contrast-v22、variance-v22。

hider:bluenoise

int

1

用于控制样本是否应具有视觉上令人满意的蓝噪声分布的设置。默认值为 1（on）。如果相邻像素中的样本需要完全去相关，则设为 0（off）。取值范围：true/false。

hider:darkfalloff

float

0.025

在自适应采样器中降低低亮度样本的优先级。使用此参数可避免对暗部区域过度采样，同时保持整体图像质量。较高值会剔除更多样本，加快渲染时间，但会增加暗部区域噪声。取值范围：0 到 1。

hider:decidither

int

0

交互式渲染允许的最大像素抽取率。在快速编辑重型场景且此设置非零时，渲染器会尝试用类似溶解的效果逐步更新整个屏幕，以减少画面撕裂。将其设为 0 可禁用此行为，并强制它完整更新能处理到的分块。取值范围：0 到 6。

hider:exposurebracket

float[2]

-1.0 1.0

后期制作中用于帮助指导自适应采样的预期曝光范围 [min max]。

hider:extrememotiondof

int

0

启用改进的采样精度。在某些同时使用大量运动模糊和景深的情况下，采样伪影可能导致不理想结果。此标志会使用较慢但更准确的采样方法来缓解伪影。由于这种采样方法会降低性能，因此只应在必要时使用此标志。取值范围：true/false。

hider:geomShadowTermBias

int

1

此设置控制何时计算并向 多边形网格 应用偏置，以避免几何阴影终止伪影；这些伪影会出现在着色点从灯光方向看相对于着色法线可见、但相对于几何法线不可见的情况下。默认值为 1（on）。如果不希望将此偏置应用于场景中的 多边形网格，请设为 0（off）。取值范围：true/false。

hider:incremental

int

0

启用结果的渐进显示（渐进显示），而不是分块结果。取值范围：true/false。

hider:jitter

int

1

此选项切换相机采样中的抖动（变化）。默认情况下，此 hider 选项已启用（设为 1），意味着相机采样会随机放置在像素位置中。关闭时，相机光线会放置在像素中心。关闭后，表面边缘会出现锯齿和毛边。取值范围：true/false。

hider:maxsamples

int

0

样本数量的显式限制。如果 maxsamples 等于其默认值，则最大样本数将设为 64。注意 minsamples 默认等于 maxsamples 设置的平方根。取值范围：0 或更大。

hider:minextrasamples

int

-1

这是一个样本未通过自适应采样器的收敛测试后，像素应继续发射的最小额外样本数。默认匹配 minsamples。

hider:minsamples

int

-1

指定自适应采样的最小样本数。默认值是 maxsamples 的平方根。

hider:pixelfiltermode

string

importance

指定如何执行相机采样的像素过滤。"Importance"（默认值）会在 屏幕空间中围绕像素中心对样本进行扭曲，并且不在像素之间共享样本结果。某些后处理算法（例如降噪器）需要 "Importance"，要获得 蓝噪声样本的视觉收益也需要它；生成方差 AOVs 也需要它。"Weighted" 是较旧的方法，会混合相邻像素的样本结果。支持值：weighted、importance。

hider:samplemotion

int

1

此选项切换 hider 中带运动模糊的微多边形 的采样，因此只有在场景使用运动模糊时才有意义。默认情况下，此 hider 选项已启用（设为 1），意味着 微多边形会在 hider 中被模糊并采样。禁用时（设为 0），微多边形在采样前不再模糊；不过仍会为所有对象计算 dPdtime 变量。此变量可以作为任意输出变量（AOV）输出到单独图像，并用于在后期处理中执行运动模糊。取值范围：true/false。

hider:sampleoffset

int

0

这允许并行渲染多张使用不同的 sampleoffset 值，然后将它们合并。使用非自适应采样时：假设用 sampleoffsets 0、256、512 和 768 分别渲染四张图像，每张 256 样本。如果合并这四张图像，将得到与渲染一张 1024 样本图像完全相同的结果。使用自适应采样时：假设再次渲染四张图像，每张的 "maxsamples" 为 256，sampleoffsets 为 0、256、512 和 768。假设由于自适应采样，某个像素在四张图像中各只获得 64 像素样本。那么合并后的图像使用样本编号 0-63、256-319、512-575 和 768-831 渲染。由于样本的分层，这不如渲染一张具有 256 个 连续样本的图像那么好。不过，它仍优于只用 64 样本渲染单张图像。

hider:samplestride

int

1

这有助于并行渲染多张使用相同的 samplestride 但不同的 sampleoffset，并在交互式渲染期间逐步合并它们。默认值为 1。使用非自适应采样时：假设用 samplestride 4 和 sampleoffsets 0、1、2、3 分别渲染四张图像，每张 256 样本。如果合并这四张图像，将得到与渲染一张 1024 样本图像完全相同的结果，并且在增量/交互式渲染期间可以最好地利用分层渐进式样本序列。如果能在交互式渲染期间用样本 0-3、4-7、8-12 等 "assemble" 图像，就可以获得更快渲染的收益，而不必一开始就跳到高样本。使用自适应采样时：假设再次渲染四张图像，每张的 "maxsamples" 为 256，samplestride 为 4，sampleoffsets 为 0、1、2、3。这次使用自适应采样（PixelVariance > 0）。由于四张图像中每张都比合并图像噪声更大，自适应采样在判定图像误差可接受之前会使用比单张图像渲染更多的样本。因此可能需要调整 PixelVariance 设置。

trace:bvhcompression

float

0

构建光线加速结构时，向系统提供倾向内存利用率还是速度的提示。有效值位于 [0, 1] 范围内，值越高，内存表示越紧凑。随着不同策略被激活，此控制在压缩率上大体近似线性，但存在一些不连续性。获得的结果很可能依赖场景。经验法则是，除非需要降低内存利用率以渲染特定场景或序列，否则将此控制保持为 0。调优时，先将控制设为 1，查看通过压缩可节省多少内存以及生成什么时间开销，然后将数值调回较低值（如果可能），以达到速度和内存使用量之间的理想平衡。取值范围：0 到 1。

int

0

checkpoint:command

string

空

检查点后要调用的系统命令。如果启用了系统调用，则在写入检查点后会调用指定命令。这是同步执行的；运行期间渲染线程会静止，并且在进程返回前不会再次恢复，从而避免命令耗时导致的潜在竞态条件。占位符 %i 会替换为当前增量，并以 5 位 0 填充。占位符 %e 会替换为秒，并以 6 位 0 填充。占位符 %r 会替换为此次检查点文件更新的状态：完全 `finished`、因 exitat 选项提前 `exiting`，或普通 `checkpoint`。字面量 % 字符可以用 %% 插入。

空

string

每 N 个渐进步数将帧缓冲写入磁盘。如果值为正，其单位会解释为 秒。如果值为负，其单位会解释为 增量。取值也可以是 "10m" 或 "0.5h" 这样的字符串。将选项设为 0 或空会禁用 检查点。

int

1

默认情况下，RenderMan 会将一个实例源扁平化为组中每个几何体对应的独立对象原型。将此选项设为 'false' 会把每个实例源位置保持为单个对象原型。对于包含大量实例的场景，这可能带来很大的性能提升。
