Stats in Katana

Source: https://rmanwiki-27.pixar.com/space/RFK27/637632529/Stats+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan 诊断信息在 Katana 中可以通过 live stats（实时统计）和离线 JSON reports 使用。也可以通过 environment variable（环境变量）或专用 OpScript 访问更高级的配置。

Live Stats（实时统计）

PrmanLiveStats Tab 会显示 active Katana render 中指标数据的 live overview。

它会实时显示多个关键指标，包括进程内存、经过的 wall clock time 和渲染进度。窗口会在渲染开始和停止时自动连接和断开连接。当渲染完成或取消后，其数据会保留在 live stats view 中，直到下一个渲染开始或加载新的 scene。

请参阅 Live Stats Overview（Live Stats 概览），了解此窗口内容的详细信息。

JSON Report（JSON 报告）

要生成渲染结束时的 diagnostic report，请将 PrmanGlobalStatements JSON Report Filename attribute 设置为所需的输出文件名（prmanGlobalStatements.options.statistics.jsonFilename）。

请参阅 Diagnostic Report（诊断报告）文档，了解此 report 的配置和解读的详细信息。

Custom Configuration（自定义配置）

要更细致地控制 diagnostic output，可以指定 RfK 使用一个 configuration file（配置文件），并通过 environment variable（环境变量） $RMAN_STATS_CONFIG_PATH来设置。此外，RfK 支持通过 specialized attributes 自定义 configuration file 的 search path 和/或 file name：

prmanGlobalStatements.stats.configPath（default： “$RMANTREE/etc")

prmanGlobalStatements.stats.configFile（default： "stats.ini")

如果设置了这些 attributes，它们会优先于其他 options。“stats” attributes 不会暴露在 PrmanGlobalStatements 中；必须通过 AttributeSet 或 OpScript 设置。下面的 OpScript 会把这两个 attributes 作为 user args 暴露出来，并使用上面列出的 defaults。将其复制并粘贴到你的 Katana scene 中，并按需修改 user attributes configPath 和 configFile name。

RfK 将按以下优先级搜索 configuration file：

  1. The prmanGlobalStatements.stats options 的优先级高于所有其他 options（见下方 OpScript）。

  2. $RMAN_STATS_CONFIG_PATH/stats.ini 如果设置了该 environment variable。

  3. $RMANTREE/etc/stats.ini

 PrmanStatsConfiguration OpScript

      <port name="i0" source="MaterialAssign.out" type="in"/>
      <port name="out" type="out"/>
      <group_parameter name="PrmanStatsConfiguration">
        <string_parameter name="CEL" value="((/root))"/>
        <string_parameter name="location" value="/root/world/location"/>
        <group_parameter name="script">
          <string_parameter name="lua" value=" -- Default: &apos;stats.ini&apos;&#0010;local configFile = Interface.GetOpArg(&apos;user.configFile&apos;):getValue()&#0010;Interface.SetAttr(&apos;prmanGlobalStatements.stats.configFile&apos;, StringAttribute(configFile))&#0010;&#0010;-- Default: &apos;.:${RMANTREE}/etc&apos;&#0010;-- This will override RMAN_STATS_CONFIG_PATH environment variable, if set.&#0010;local configPath = Interface.GetOpArg(&apos;user.configPath&apos;):getValue()&#0010;Interface.SetAttr(&apos;prmanGlobalStatements.stats.configPath&apos;, StringAttribute(configPath))&#0010;"/>
        </group_parameter>
        <string_parameter name="executionMode" value="immediate"/>
        <string_parameter name="applyWhere" value="at locations matching CEL"/>
        <string_parameter name="applyWhen" value="during op resolve"/>
        <string_parameter name="modifierNameMode" value="node name"/>
        <string_parameter name="modifierName" value="modifier"/>
        <string_parameter name="resolveIds" value=""/>

        <string_parameter name="disableAt" value=""/>

Troubleshooting（故障排查）

通常，stats system 会在后台安静地完成大部分工作。不过，当情况没有按预期运行时，系统内置了一些机制，可用于获取 diagnostics system 自身的 diagnostics。

首先要明白，你不能禁用 stats collection（收集）。此功能嵌入渲染器，并保持 “always on”。不过，你可以控制 diagnostic data 如何 delivered（交付） 和 presented（呈现）。

用于配置 stats system 的 stats.ini 文件包含一些 flags，可在 config file 加载和后续处理期间提高 verbosity。如果没有明确使用 custom（自定义）stats.ini 文件，则默认配置将来自 $RMANTREE/etc/stats.ini。

在 config file 中，将 stats debug logLevel 设置为 5：

# Stats processing log level.
# Range is 0 (none) to 5 (debug output). Default is 3 (warnings)
logLevel 5

你应该会看到 RZ0000 输出到 Render Log 的消息增加，这会帮助你验证它正在读取预期的 stats.ini 文件。它还会告诉你 WebSocket stats server 是否已为 live stats 启用。

RZ0000 {INFO}   : SessionConfig - Loaded stats config from file: '/path/to/stats.ini'.
RZ9000 rfk_statsserver_<renderboot_PID>
RZ0000 {INFO} : Session - Created stats session: 'Stats Session Name'.
RZ0000 {DEBUG} : Session - WebSocket stats server not enabled
RZ0000 {DEBUG} : Session - Loading listeners​

如果提高 log level verbosity 后仍未看到这些日志消息，你很可能正在读取另一个 stats.ini configuration file。请检查 RMAN_STATS_CONFIG_PATH environment variable（环境变量），或来自 prmanGlobalStatements.stats attributes 的 path/filename values。

请参阅通用 Troubleshooting（故障排查） 文档，以获取诊断 stats system 问题的进一步帮助。
