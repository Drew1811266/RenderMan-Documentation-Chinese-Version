XGen in Katana

Source: https://rmanwiki-27.pixar.com/space/RFK27/546179303/XGen+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

许多使用 Maya 的工作室会用 XGen 创建毛发与毛皮。RenderMan for Maya 支持渲染 XGen 毛发。许多人不知道的是，也可以通过 Katana 渲染 XGen 毛发。

作为 RenderMan for Maya 的一部分，我们提供 RenderMan Procedural，可用于在 Katana 内渲染 XGen 毛发。本页说明如何通过 Katana 渲染 XGen 的细节，并提供让 Katana 渲染 XGen 的分步说明。我还包含了用于生成本页末尾图像的文件。下面是我们将使用的示例渲染。

Maya 内准备

在此示例中，我根据纹理中的值为每根毛发分配一组逐毛发颜色。为了让 RenderMan 实现这一点，需要从 XGen 输出 scalpS 和 scalpT 基元变量。在着色网络中，读取纹理时会引用这些 primvar。下面的 Katana 小节会详细说明如何操作。以下屏幕截图展示了 XGen 属性的 Preview/Output 选项卡为启用此功能应呈现的状态：

重要：更新 XGen Preview

无论你打算如何为毛发着色，都必须点击 XGen 窗口左上角的眼睛图标，执行 Update the XGen Preview。这会让 XGen 处理 RenderMan 在渲染期间通过程序化过程调用它时所需的信息。

Katana 中的流程

由于我们提供了一个与 XGen 对接的 RenderMan 程序化过程，可在渲染时动态创建曲线并传给 RenderMan，因此从 Katana 渲染 XGen 的工作主要是在场景中添加 RenderMan 程序化过程并配置其所需参数。

你需要执行以下步骤：

  • 通过 Node Graph 中的 PrimitiveCreate 节点添加一个“渲染器过程节点”
  • 设置程序化过程的参数
  • 创建材质并将其指定给渲染器过程节点

添加“渲染器过程节点”

第一步是通过 Node Graph 中的 PrimitiveCreate 节点向场景图添加两个“渲染器过程节点”位置。我包含了一张屏幕截图，展示你应如何设置参数。第一个向 XGen 指定 groom，第二个向 XGen 指定与之关联的几何体。在此示例中，groom 是从平面生成的，因此我相应命名了场景图位置。

设置程序化过程参数

为正确设置程序化过程，需要传入几个参数以让 XGen 渲染。我们分几步完成。每个程序化过程使用两个 AttributeSet 节点（总共四个），并使用一个 OpScript 节点设置 XGen 在场景中需要的其他参数。

第一组 AttributeSet 节点会告诉渲染器过程节点共享库的位置，其中包含知道如何调用 XGen 以动态创建曲线或几何体的代码。必须为 groom 程序化过程创建一个，也为平面程序化过程创建一个。下面两张屏幕截图展示 AttributeSet 节点应有的样子。注意，该路径引用 RenderMan for Maya， 不是 RenderMan for Katana。这是因为 XGen RenderMan 程序化过程是随 RenderMan for Maya 发行包提供的。

设置渲染器过程节点的路径后，下一步是向程序化过程传递参数，使其知道要查看哪些文件以及如何构建曲线。你要使用另外两个 AttributeSet 节点来完成，一个用于 groom 程序化过程，一个用于平面程序化过程。下面包含两张屏幕截图。

因为上方图像中的参数被截断，我也在下面列出它们。

groom procedural： -frame 1 -file /data/work/samples/rfm/XGenWithTexture__MyCollection.xgen -palette MyCollection -geom /data/work/samples/rfm/XGenWithTexture__MyCollection.abc -patch groom -description MyDescription -fps 24

plane procedural： -frame 1 -file /data/work/samples/rfm/XGenWithTexture__MyCollection.xgen -palette MyCollection -geom /data/work/samples/rfm/XGenWithTexture__MyCollection.abc -patch pPlane1 -description MyDescription -fps 24

这里传入的参数对正确渲染场景至关重要。你必须：

  • 确保帧号正确 - 在生产 Katana 设置中，通常会用表达式驱动它
  • 确保 .xgen 和 .abc 文件的路径正确。若没有这些文件，请确认已在 Maya 内的 XGen 窗口中点击 "Update the XGen Preview" 按钮
  • -palette 参数需要引用你的 XGen collection
  • -description 参数需要引用包含 groom 的 XGen description
  • 平面程序化过程的 -patch 必须匹配 Maya 中生成 XGen groom 的节点名称

Autodesk 的 XGen Technical Documentation 中的 RenderMan DSO page 记录了程序化过程支持的所有参数。

添加所有 AttributeSet 节点后，还需要继续操作。必须添加一个 OpScript 节点，用于设置 XGen 执行工作所需的几个额外参数。下面是该节点的屏幕截图。

节点中的代码如下：

gb = GroupBuilder()
gb:set("value", StringAttribute({"false,8.85,12.05,10.85"}, 1))
gb:set("type", StringAttribute("string"))
Interface.SetAttr("prmanStatements.attributes.user.irRenderCam", gb:build())

gb:set("value", StringAttribute({"54.4322"}, 1))
gb:set("type", StringAttribute("string"))
Interface.SetAttr("prmanStatements.attributes.user.irRenderCamFOV", gb:build())

gb:set("value", StringAttribute({"0.745476,2.77556e-17,-0.666532,0,-0.469152,0.710328,-0.524718,0,0.473457,0.70387,0.529533,0,8.85,12.05,10.85,1"}, 1))
gb:set("type", StringAttribute("string"))
Interface.SetAttr("prmanStatements.attributes.user.irRenderCamXform", gb:build())

gb:set("value", StringAttribute({"1"}, 1))
gb:set("type", StringAttribute("string"))
Interface.SetAttr("prmanStatements.attributes.user.irRenderCamRatio", gb:build())

这些参数已由 Autodesk 在其 XGen Technical Documentation 中的 RenderMan DSO page中完整记录。 

需要特别注意以下几点：

  • 你告诉 XGen 的视野 必须 与 Katana 中使用的相机视野匹配
  • irRenderCam 属性 必须 与 Katana 中使用的相机位置匹配
  • irRenderCamXForm 必须 与 Katana 中使用的相机矩阵匹配。。

在生产场景中，你通常会使用脚本或其他程序化方法来确保所有这些参数与相机匹配，因为手动完成很容易出错。
