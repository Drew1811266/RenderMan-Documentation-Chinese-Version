关于 Analytic Lights（解析灯光）

RenderMan for Katana（RfK）最有价值的功能之一，是能够创建并管理 Lights（灯光）。下面各节将简要概述如何利用这些能力。

使用 RenderMan LightPackages（灯光包）

RfK 提供 PRMan 专用的 light packages，用于在 GafferThree 中创建每一种 light。这些 light packages 会尽量减少创建 light 所需的步骤。 Shortcuts（快捷方式） 让你只需一次按键即可添加一个 light。每个 light package 还包含一组 light-specific attributes（特定于灯光的属性），可在 GafferThree 的 Object 选项卡中使用，让你无需额外添加 PrmanObjectStatements 节点，也能逐灯控制某些属性。

PxrMeshLight package 包含以下属性：

analytic light packages 只包含 Camera Visibility，该项默认设为 'Off'，因为 renderer 会忽略 analytic lights 的其他 visibility attributes。

创建 Mesh Lights（网格灯）

在 GafferThree 中有一个专门用于创建 mesh lights（网格灯）的 light package。要创建 mesh light，请在 Scene Graph 中选择所需几何体，然后在 GafferThree 中添加 "PxrMeshLight"。你可以在下面的链接中找到关于 mesh light 设置的详细说明：

使用 Light Filters（灯光滤镜）

Light filters（灯光滤镜）是控制 light 外观的强大方式。在 Katana 2.0 或 2.1 中，可以使用 PrmanLightFilterCreate 节点将 light filters 添加到场景中。Katana 2.2 及更高版本提供 GafferThree 对 light filters 的支持。你可以在下面的链接中找到关于 light filter 设置的详细说明：

  • Katana 中的 Light Filters

使用 Portal Lights

PxrPortalLight 是一种经过优化的 environment light（环境光）。由于 Portals 是 lights，而不是 light filters，因此它们需要特殊工作流。GafferThree 对 portal lights 的支持将在未来版本的 Katana 中提供。你可以在下面的链接中找到关于 portal light 设置的详细说明：

  • Katana 中的 Portal Lights

Light Sampling（灯光采样）

Light Sampling 能力可通过 integrator 中的 sample balancing 获得，通常使用 numLightSamples 选项。也可以使用 Light Parameter 列表中的 Importance multiplier，从 sample pool 中为该灯光分配更多 samples。
