Using Displacement

Downloaded: 2026-06-15T03:49:12+00:00

使用 PxrDisplace 创建额外几何细节是一种常见工作流，通常会结合 Mudbox 或 ZBrush 等软件导出贴图。我们推荐使用矢量置换，因为这对分层最合理；不过我们会用同一个节点渲染标量贴图和矢量贴图。

需要确保带有置换的对象具有非零置换边界。出于性能原因，该值默认为零，因此对于需要置换的对象，必须始终增大此值。

  • 创建 PrmanObjectStatements 节点，并将其分配给你的对象（使用 CEL 参数）

  • 在 PrmanObjectStatements 节点中找到 "primAttributes.displacementbound" 下的 displacement bound（置换边界）

  • 将 "Sphere" 设置为大于零的值：它要足够大，能够包住置换，但也要足够小，避免让渲染器认为曲面延伸超过预期的最终边界。如果边界太小，会看到裁切或缺失片段；如果太大，可能会过早对对象进行置换。之后我们会自动收紧这些边界，使其更高效。因此宁可偏大，也不要偏小。值为 0（零）会禁用该对象的置换。

RenderMan 会为过大或过小的边界打印警告，最小因子为 10，以帮助你调整

下面是一个为对象添加置换的典型网络：

  • 创建一个 Network Material

    • 在该节点中使用 Add Terminal，为 Bxdf 或曲面着色创建一个端口

    • 使用同一菜单为 Displacement（置换）创建第二个端口

  • 一个 PrmanShadingNode 用作 surfaceShader，通常类型为 PxrSurface

  • 第二个 PrmanShadingNode 的类型为 PxrDisplace

  • 将 Pattern 输出（标量置换为 float，矢量置换为 RGB）连接到 displacementShader

渲染以查看结果。确保在 Merge 节点之后有一个 MaterialAssign，并且它具有指向对象的正确 CEL path（CEL 路径），同时网络材质位置设置为 NetworkMaterial
