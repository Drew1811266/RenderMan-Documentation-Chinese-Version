Preset Browser in Katana

Downloaded: 2026-06-15T03:49:12+00:00

Introduction（简介）

  • 可以使用 menu arrow button 打开 Preset Browser，并在 RenderMan 下找到它。

  • Presets 包含 light rigs、environment lights 和 materials 等类别。

  • 可以按 name、usage category 或 metadata 搜索 assets。

Saving and Using（保存和使用）

  • 保存/导出 material 很简单！

    • 必须选择 BxDF network material，该 material 来自 Scene Graph（export 会识别已连接的 nodes）注意：factory library 不能修改；你可以复制一份或创建新的 library

  • 在 Preset Browser 中选择所需类别（Material、Light 等）

  • 选择 Save icon（preview image 下方闪亮的 material ball + 图标）

  • 为它提供有用的 name，并按 pipeline 需要添加 metadata

  • RenderMan 会生成 preview swatch render，并自动填充到 Preset Browser 中

  • 保存 light rigs 的方式相同，只是使用 material save preset icon 旁边带 + 符号的小 lightbulb icon。你可以导出 Gaffer node 或单独的 lights。

  • Importing/Using a saved asset（导入/使用已保存的 asset）

    • 只需 right-click 并为选中的 object 选择 "Import and Assign"

    • 根据 object 需要进行调整！

Environment Maps（环境贴图）

  • 可以使用 Preset Browser 中 large preview swatch 下方的 image icon+ 保存 Environment Map

    • 在 Preset Browser 中选择 Environment category

    • 从 file browser window 选择所需的 OpenEXR 或 .HDR file

    • 可以在 dialog 中添加/更改 metadata，或更改 name。

    • 然后我们会将 image 转换为兼容的 .tex format。这可能需要一些时间，取决于 original image size。

    • 随后会执行 preview render，并自动填充到 browser category 中。

  • 使用已保存的 map 很简单。

    • right-click 该 asset 并 import。

    • 更换 map 就像 double-click 另一个 environment map asset 一样简单！

Asset Dependencies（Asset 依赖项）

当 asset 保存到 disk 时，它的所有 dependencies 也会保存到 disk。 

包括：

  • Textures（纹理）：以 tex format 存储，支持 UDIM。

  • HDRI environment maps（HDRI 环境贴图）：以 tex format 存储。我们使用特定的 flags 组合，以尽可能高效地压缩 texture。

  • OSL shaders（OSL 着色器）：以 oso format 存储以保护你的 IP；如果想共享 source code，也可以手动将 osl file 添加到 asset directory。

这些 dependencies 会增加 preset 在 disk 上的大小；如果要通过 internet 分发它们，需要考虑这一点。

Cross-Application Materials（跨应用 Materials）

RenderManAssets 设计为可以轻松在一个 application（如 Katana）和另一个 application（Maya、Houdini、Blender 等）之间来回传递。

Compatibility Issues（兼容性问题）

然而，如果 material 包含 non-native RenderMan nodes，仍可能遇到 compatibility issues。例如，RenderMan for Maya 会自动转换若干 native Maya nodes，而这些 nodes 在 Katana 中不可用，例如 file node 或 Maya 的 Cloud pattern。为缓解这些问题，每个 RenderManAsset 都会维护 used nodes 列表，并用它评估 preset 与当前 host application 的 compatibility；如果不兼容，会显示 warning。

system 会检查：

  • asset 是否包含 host-specific nodes

  • asset 的 renderer version 是否高于或等于 current version

  • 如果 asset 包含 host-specific nodes，请确保 current host version 高于或等于 asset 的 host version。

Best Practice（最佳实践）

如果想确保 cross-application compatibility，请避免使用 non-Pixar BxDFs 和 Patterns。

Library Configuration（Library 配置）

RenderMan 随附 standard library，可在以下位置找到： $RMANTREE/lib/RenderManAssetLibrary

建议将 standard library 复制到你选择的位置，并保留 default library 不变作为 backup。新的副本应 readable and writable。如果 asset browser 指向 non-writable location，则不允许创建 new assets。

Environment variable（环境变量）

asset library 的 location 可以通过 RMAN_ASSET_LIBRARY environment variable 设置。

可以用不同 methods 设置：

Linux / OS X
RMAN_ASSET_LIBRARY = $RMANTREE/lib/RenderManAssetLibrary
Windows
RMAN_ASSET_LIBRARY = %RMANTREE%/lib/RenderManAssetLibrary

System commands（系统命令）

Linux

在所有 Posix platforms（Linux & OSX）上，可以在用于启动 Katana 的 shell 中定义 environment variable，也可以将其追加到 shell configuration file（.cshrc 用于 csh/tcsh， .profile 或 .bashrc 用于 bash）。

csh / tcsh
setenv RMAN_ASSET_LIBRARY $RMANTREE/lib/RenderManAssetLibrary
bash
export RMAN_ASSET_LIBRARY=$RMANTREE/lib/RenderManAssetLibrary
OSX

  • 如果从 shell 运行 Katana，可以像 Linux 一样 setup environment variables。

Windows

  • Right-click My Computer 并选择 Properties（或在 control panel 中 double-click System），然后点击 Advanced tab，再点击 Environment Variables。
