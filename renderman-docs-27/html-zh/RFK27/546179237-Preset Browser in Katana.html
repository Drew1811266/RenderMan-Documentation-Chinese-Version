Preset Browser in Katana

Source: https://rmanwiki-27.pixar.com/space/RFK27/546179237/Preset+Browser+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

简介

  • 可以使用菜单箭头按钮打开 Preset Browser（预设浏览器），并在 RenderMan 下找到它。

  • Presets（预设）包含灯光套件、环境灯光和材质等类别。

  • 可以按名称、用途类别或元数据搜索资产。

保存和使用

  • 保存/导出材质很简单！

    • 必须选择 BxDF 网络材质，该材质来自 Scene Graph（导出会识别已连接的节点）注意：出厂库不能修改；你可以复制一份或创建新的库

  • 在 Preset Browser（预设浏览器）中选择所需类别（Material、Light 等）

  • 选择保存图标（预览图像下方闪亮的材质球 + 图标）

  • 为它提供有用的 name，并按 pipeline 需要添加 metadata

  • RenderMan 会生成预览色样渲染，并自动填充到 Preset Browser（预设浏览器）中

  • 保存灯光套件的方式相同，只是使用材质保存预设图标旁边带 + 符号的小灯泡图标。你可以导出 Gaffer 节点或单独的灯光。

  • 导入/使用已保存资产

    • 只需右键单击并为选中的对象选择 "Import and Assign"

    • 根据对象需要进行调整！

环境贴图

  • 可以使用 Preset Browser（预设浏览器）中大预览色样下方的图像加号图标保存环境贴图

    • 在 Preset Browser（预设浏览器）中选择环境类别

    • 从文件浏览器窗口选择所需的 OpenEXR 或 .HDR 文件

    • 可以在对话框中添加/更改元数据，或更改名称。

    • 然后会将图像转换为兼容的 .tex 格式。这可能需要一些时间，取决于原始图像尺寸。

    • 随后会执行预览渲染，并自动填充到浏览器类别中。

  • 使用已保存的贴图很简单。

    • 右键单击该资产并导入。

    • 更换贴图就像双击另一个环境贴图资产一样简单！

资产依赖项

当资产保存到磁盘时，它的所有依赖项也会保存到磁盘。 

包括：

  • 纹理：以 tex 格式存储，支持 UDIM。

  • HDRI 环境贴图：以 tex 格式存储。我们会使用一组特定标志尽可能高效地压缩纹理。

  • OSL 着色器：以 oso 格式存储以保护你的 IP；如果想共享源代码，也可以手动将 osl 文件添加到资产目录。

这些依赖项会增加预设在磁盘上的大小；如果要通过互联网分发它们，需要考虑这一点。

跨应用材质

RenderManAssets 设计为可以轻松在一个应用程序（如 Katana）和另一个应用程序（Maya、Houdini、Blender 等）之间来回传递。

兼容性问题

然而，如果材质包含非原生 RenderMan 节点，仍可能遇到兼容性问题。例如，RenderMan for Maya 会自动转换若干 Maya 原生节点，而这些节点在 Katana 中不可用，例如文件节点或 Maya 的 Cloud pattern。为缓解这些问题，每个 RenderManAsset 都会维护已用节点列表，并用它评估预设与当前宿主应用程序的兼容性；如果不兼容，会显示警告。

系统会检查：

  • 资产是否包含宿主专用节点

  • 资产的渲染器版本是否高于或等于当前版本

  • 如果资产包含宿主专用节点，请确保当前宿主版本高于或等于资产的宿主版本。

最佳实践

如果想确保跨应用兼容性，请避免使用非 Pixar 的 BxDFs 和 Patterns。

库配置

RenderMan 随附标准库，可在以下位置找到： $RMANTREE/lib/RenderManAssetLibrary

建议将标准库复制到你选择的位置，并保留默认库不变作为备份。新的副本应可读可写。如果资产浏览器指向不可写位置，则不允许创建新资产。

环境变量

资产库的位置可以通过 RMAN_ASSET_LIBRARY 环境变量设置。

可以用不同 methods 设置：

Linux / OS X
RMAN_ASSET_LIBRARY = $RMANTREE/lib/RenderManAssetLibrary
Windows
RMAN_ASSET_LIBRARY = %RMANTREE%/lib/RenderManAssetLibrary

系统命令

Linux

在所有 POSIX 平台（Linux 和 OSX）上，可以在用于启动 Katana 的 shell 中定义环境变量，也可以将其追加到 shell 配置文件（.cshrc 用于 csh/tcsh， .profile 或 .bashrc 用于 bash）。

csh / tcsh
setenv RMAN_ASSET_LIBRARY $RMANTREE/lib/RenderManAssetLibrary
bash
export RMAN_ASSET_LIBRARY=$RMANTREE/lib/RenderManAssetLibrary
OSX

  • 如果从 shell 运行 Katana，可以像 Linux 一样设置环境变量。

Windows

  • 右键单击 My Computer（我的电脑）并选择 Properties（属性）（或在 control panel（控制面板）中双击 System（系统）），然后点击 Advanced（高级）选项卡，再点击 Environment Variables（环境变量）。
