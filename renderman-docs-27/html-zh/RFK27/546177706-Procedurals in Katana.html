Katana 中的过程程序

Source: https://rmanwiki-27.pixar.com/space/RFK27/546177706/Procedurals+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan 过程程序使你能够在渲染期间动态创建几何体，然后传递给 RenderMan。通过在最后一刻把几何体传入 RenderMan，可以节省在 Katana、Maya 或其他应用程序中生成和显示几何体所需的时间和内存。

过程程序最常用于毛发/头发渲染、群集渲染、动态生长的植被，或程序化创建的环境。它们几乎可以用于任何内容。

虽然过程程序有很多优点，但其中一个缺点也来自它的优势。由于几何体只在 RenderMan 内部生成，Katana 对该几何体一无所知，也无法显示它。如果你用于创建过程程序的应用程序有某种静态表示，例如 Alembic 文件，我们建议使用 Katana Viewer 代理功能，这样你在 Viewer 中看到的就不只是包围盒。有关如何使用 Viewer 代理，请参阅 Katana Help。

在 Node Graph 中设置过程程序

要让 Katana 正确地将过程程序传递给 RenderMan，主要需要执行三个步骤。

  • 设置一个类型为 "renderer procedural" 的 Scene Graph 位置。

  • 设置包含 RenderMan 将调用的过程程序代码的共享库路径。

  • 设置过程程序执行工作所需的任何额外参数。

下面详细说明每个步骤。

步骤 1： 设置一个类型为 "renderer procedural" 的 Scene Graph 位置

在 Scene Graph 中，每个位置的 "type" 控制该位置的 Attributes 如何传递给 RenderMan。为了让 RenderMan 在运行时调用过程程序，必须添加一个类型为 "renderer procedural" 的 Scene Graph 位置。

最简单的方法是使用 PrimitiveCreate 节点，并将 type 设置为 "renderer procedural"。同时指定名称。在下面的示例中，我将其设置为 /root/world/geo/procedurals/myprocedural

步骤 2： 设置包含 RenderMan 将调用的过程程序代码的共享库路径

创建过程程序几何体的应用程序开发者会提供 RenderMan 过程程序库。请查阅该应用程序的文档，或咨询你的流程团队以了解在哪里可以找到此路径。下面我指定了一个假想库的路径。使用 AttributeSet 节点来完成此操作。要使用的属性名称是：rendererProcedural.procedural

步骤 3： 设置过程程序执行工作所需的任何额外参数

每个过程程序都有一组唯一参数，必须传递给它，才能让过程程序为正在渲染的帧生成正确的几何体。这些参数很可能类似于你会传给命令行工具的内容。

下面我使用 AttributeSet 节点指定了一些假想属性，供假想的 Procedural.so 正常执行。要使用的属性名称是：rendererProcedural.args.extraArgs

超越基础

下面的信息只提供将 RenderMan 过程程序插入 Katana 场景的最基本方法。你或流程团队还可以采用其他更高级的选项。对于下面每种方法，实际效果可能因情况而异。有关每种方法的更多详细信息，请查看 Katana 中 Help 菜单提供的 Katana 文档。

  • 创建并使用 Args 文件用于渲染器过程程序，从而更轻松地指定过程程序执行工作所需的参数。

  • 创建一个包装过程程序的自定义节点 - 这对用户最友好，但需要开发者。
