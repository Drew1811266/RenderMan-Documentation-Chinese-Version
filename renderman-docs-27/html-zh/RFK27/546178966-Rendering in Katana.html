渲染完成后的 scene 涉及许多环节。从处理 quality 到选择适合后续 compositing 的 outputs，本节会覆盖所有这些主题。

Quality（质量）

为了理解 noise removal 和 anti-aliasing 的工作方式，我们为用户说明 Sampling（采样） 以及 Filtering（过滤）。这里可能是你大部分 tuning 工作发生的地方，因此理解 quality 与 performance 之间的平衡很重要。

Trace Depth（追踪深度）

控制 trace depth 可以改变 scene 的外观和 performance。不同 integrators 对这些 settings 的处理方式也不同。RenderMan 让这些参数的 tuning 变得容易，从而在 resource constraints 内实现所需 look。

Interactive Rendering and Checkpointing（交互式渲染与检查点）

Progressive/Interactive（交互式） Rendering 让调整 scene materials 和 lights 变得轻松。 Checkpointing（检查点） 和 Recovery（恢复）会将这个 progressive feature 扩展到 final renders，使你可以在 render 逐步 refine 的过程中创建 save points。你可以用它更快地审批 incomplete frames，并从中断处继续，而不会浪费宝贵的 CPU cycles。

Diagnostics（诊断）

RenderMan diagnostics（诊断） 可通过易读的 display 帮助你 visualize scene performance。有时可能不清楚 scene 把最多 time 或 resources 花在哪里，但我们的 diagnostic output 会帮助你定位即使最隐蔽的 performance information。

Holdouts and Outputs（Holdouts 与输出）

Holdout（遮罩保留） workflow 对于将 CG objects 集成到 live action plates 中至关重要。在某些 applications 中，这就像点击一个 button 一样简单。为了改进 compositing integration，甚至改进 fully rendered shots，RenderMan 会使用强大的 outputs（输出） 让 compositing 中的 adjustments 更容易、更强大，并具备最高灵活性。

Baking（烘焙）

在 rendering 中重用某些 data 可以简化 asset management，并提升 rendering efficiency。 Baking（烘焙） 提供了 bake 到 2D output 或 point cloud output 的方法。

Nested Dielectrics（嵌套电介质）

渲染彼此交互的 surfaces 时，尤其是 transparent/refractive surfaces，理解 Intersect Priority（相交优先级） settings（设置）很重要。Intersect priority 允许 users 只需添加 attributes，就能获得正确的 refractions，而不必通过繁琐 modeling 来解决问题。
