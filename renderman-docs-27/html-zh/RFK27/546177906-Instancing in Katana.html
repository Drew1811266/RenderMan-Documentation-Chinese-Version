Katana 中的实例化

Source: https://rmanwiki-27.pixar.com/space/RFK27/546177906/Instancing+in+Katana
Downloaded: 2026-06-15T03:49:12+00:00

如果场景中有在多个位置重复使用的几何体，实例化可以让你在做少量额外设置后减少渲染的内存占用。例如，如果场景由多个建筑组成，每个建筑旋转略有不同，就可以使用实例化只将一个副本加载到内存中。

RenderMan for Katana 支持三种实例化方法：叶级实例化、层级实例化和实例数组。第一种叶级方法允许通过属性对匹配的几何体片段进行实例化。第二种和第三种是更明确的方式：先设置对象原型，然后显式实例化它。 teapots_instancingExample.katana 文件包含一个示例，展示如何使用这些方法渲染上方图像。

性能说明

实例化可以节省内存，但可能影响性能，例如首像素时间（渲染开始发送像素的速度），因为 Katana 可能会处理许多节点。能够实例化层级的选项在首像素时间上会更高效，但你可能会失去对某些希望覆盖的属性的控制；这是需要权衡的地方。

在 PrmanGlobalStatements 中可以指定 prmanGlobalStatments.plugin.flattenInstanceSources 或者可以在 PrmanObjectStatements 中使用 prmanStatements.traversal.flattenInstanceSource 这些选项会告诉 Katana 将实例保留为组，而不是将其展平；这样会缩短首像素时间，因为不会为可能大量涌入的实例执行展平。

可以在主 实例化 页面上阅读更多关于嵌套与展平的内容。

 本质上，场景图展平时间对应于实例数量乘以实例源下方的位置数量。实例源越复杂、实例越多，展平成本就越高。

叶级实例化

你可能会发现场景占用了大量内存，并认为由于场景有很多重复几何体，也许可以利用实例化。叶级实例化方法可通过 instance.ID 属性高效地将已有几何体位置转换为彼此的实例。RenderMan for Katana 会把它遇到的第一个具有某个 instance.ID 的位置作为几何体原型，随后所有位置都会成为实例。所有具有匹配 instance.ID 的位置必须完全相同，否则根据 RenderMan for Katana 最先遇到哪个位置，可能出现未定义行为。下面的屏幕截图展示如何使用 AttributeSet 节点设置该 instance.ID 属性。

顾名思义，叶级实例化只适用于没有子级的场景图位置。RenderMan for Katana 不会继续遍历带有 instance.ID 属性的位置，因此它不能用于实例化几何体组。如果需要实例化层级，必须使用层级实例化或实例数组。接下来的两个小节会介绍它们。

层级实例化

在这种方法中，实例源/对象原型通过其场景图位置指定，然后在对应的实例位置中引用。如果你希望实例化一个位置层级，这种方法很适合。 

使用这种方法创建实例的步骤如下。

  • 将要用作对象原型的场景图位置的 "type" 更改为 "instance source"。请注意，实例源的场景图位置必须是 group（组）。如果将几何图元的类型改为 "instance source"，RfK 将没有关于图元类型的任何信息，实例将无法渲染。

  • 为实例创建位置。可以使用 LocationCreate 节点或 OpScript 完成。如果希望实例源可见，也必须为它创建一个新的实例（相比之下，instance.ID 方法会自动使实例源可见）。

  • 将每个实例位置的 geometry.instanceSource 属性设置为实例源的场景图位置

  • 使用 Transform3D 节点或 OpScript 在实例位置上设置变换。

创建实例后，如果需要，可以使用将材质应用到场景图位置的标准 Katana 机制来覆盖材质。注意，在实例源位置上分配的任何材质都会被忽略。这是为了允许在实例位置上进行覆盖。不过，只能覆盖实例源位置根部的材质。实例源的子位置上的材质覆盖需要使用用户属性完成。 

实例数组

与层级实例化一样，此方法直接把位置指派为实例源。标记为实例源的位置可以由实例数组位置引用。与层级实例化相比，此方法有一些优缺点。主要优点是只需创建一个场景图位置来表示所有实例。这可以节省 Katana 场景图处理时间。此外，可以在单个实例数组中引用多个实例源。主要缺点是实例数组无法按每个实例覆盖所有属性，只支持用户属性覆盖。在决定实例化方法之前，确定需求很重要。

设置实例源位置的步骤与层级实例化相同。只需将表示对象原型的 group 类型设置为 "instance source"。然后需要创建一个 "instance array" 类型的位置，并在其上构建实例数组属性。

实例数组位置上需要以下属性：

string geometry.instanceSource

此实例数组中引用的实例源位置数组。

int geometry.instanceIndex

instanceIndex 属性为实例数组中的每个实例都有一个元素。每个元素都会映射到 geometry.instanceSource 属性中的一个索引。此映射决定每个实例使用哪个实例源。

double 或 float geometry.instanceMatrix

对于实例数组中的每个实例，instanceMatrix 中都有 16 个值表示该实例的变换矩阵。注意，此变换是相对于实例数组自身的变换。

此外，还有一些可用的可选属性：

int geometry.instanceSkipIndex

要从实例数组中省略的索引列表。用于剔除某些实例。

group geometry.arbitrary

虽然格式化为 constant PrimVars，但在渲染器中这些属性会变成 用户属性。value 和 indexValue 属性中应为实例数组中的每个实例都提供一个元素。

请参见这个 instanceArray.zip 示例（也可在 RfK 的 Examples 目录中找到），其中展示如何从点云设置实例数组。

在实例级别覆盖材质

着色覆盖可以通过多种方式实现。在示例文件中采用了一种简单方法：为每个实例分配一个全新的 Bxdf。RenderMan for Katana 还可以用更复杂的方式改变每个实例上的着色。其中一种方法是为每个实例创建一个唯一的 "user" 属性。在分配给对象原型的材质内，可以创建一个 PxrAttribute 模式来读取用户属性，然后使用其输出驱动 Bxdf 上的某个参数。

虽然可以按每个实例改变 Bxdf、变换、灯光链接和属性，但有些内容不能变化。PrmanObjectStatements 中 "primvars" 和 "primAttributes" 组下的 PrimVars 和参数不能按每个实例变化。置换也不能跨实例变化，因为它会改变渲染器中几何体原型的表示。有关实例化的更深入讨论，请阅读 RenderMan 文档中的 实例化 页面。它提供了很好的背景信息，以及可以跨实例变化内容的完整列表。

实例化灯光

可以使用上述方法或使用 灯光数组来实例化灯光。与实例化几何体相比，实例化灯光有更多限制。灯光着色器参数不能按每个实例变化。同样，PxrPortalLight 和 PxrEnvDay 灯光不能被实例化。还要注意，如果要使用灯光过滤器实例化灯光，不要使用叶级实例化方法，因为这种方法不允许实例化层级。
