The ROP Node

Source: https://rmanwiki-27.pixar.com/space/RFH27/544646437/The+ROP+Node
Downloaded: 2026-06-15T03:49:12+00:00

在 RenderMan 中渲染是通过 ROP 节点在 Houdini 中完成的。如果选择工具架按钮，我们会创建并连接常用的渲染选项，包括使用 PxrPathTracer Integrator（积分器）。

渲染有很多选项，其中一些较为冷门，或只适用于特殊情况。我们支持在 Houdini 中使用多个 ROP 节点，因此你可以按任务定制每一个节点，无论是 IPR 设置、用于渲染通道的不同显示驱动，还是渲染过程中希望看到的实用调整。请务必使用你想查看其结果的那个 ROP 节点上的渲染控制。举例来说，你可以创建一个面向快速 IPR 的 ROP 节点，再创建另一个用于最终生产渲染的 ROP 节点。


本文档旨在帮助你尽可能简单地理解这些差异。

关于为合成提供 AOVs 和 LPE 内容的信息，请参见 AOVs 页面。

关于 Driver 选项卡以及渲染到设备的信息可参见 此处。


标准控制

按钮

  • Render（渲染）： 启动到 RenderMan 的交互式渲染

  • Stop（停止）： 取消或结束交互式渲染 (也可以从 /wiki/spaces/REN20/pages/21561593)

  • Render in Background（后台渲染）： 在保持 Houdini UI 可用的同时进行渲染

  • View RIB（查看 RIB）： 导出 RIB 文件（默认位置为你的 .hip 场景文件所在位置）。默认使用 Binary（二进制）以节省空间。你可以在 RIB 选项卡中更改此设置

  • Bake（烘焙）： 运行一次烘焙渲染，用于 烘焙 pattern 网络

  • Controls...（控制...）： 用于渲染的附加选项，例如质量覆盖和帧设置

常用选择

  • Valid Frame Range（有效帧范围）： 选择要渲染的帧；对交互式渲染无效

    • Start/End/Inc（起始/结束/增量）： 帧范围和增量，接受浮点值

  • Render With Take（使用 Take 渲染）： 要使用的版本

  • Camera（摄影机）： 指向用于渲染的所选摄影机的路径。必须指定一个摄影机。如果使用我们的工具架按钮，我们会填充该字段

  • Override Camera Resolution（覆盖摄影机分辨率）： 将渲染分辨率从摄影机设置中指定的分辨率改为此处的设置


渲染

  • Integrator（积分器）： 要使用哪个 积分器。如果此项为空或未设置，则使用 PxrDefault Integrator（积分器）

  • Enable Depth of Field（启用景深）： 允许使用 Depth of Field（景深）设置；如果你在调整场景时想暂时禁用景深，这是一个简单的开关

  • Allow Motion Blur（允许运动模糊）： 为场景实体开启运动模糊。请注意，体积的运动模糊由 BxDF, PxrVolume 单独控制

Sampling Tab（采样选项卡）

另请参见 Renderer Options（渲染器选项） 了解更多细节

  • Incremental（增量）： 渲染帧级遍次，而不是基于 bucket/tile 的渲染。对于大多数渲染选择都更推荐使用它，包括批处理和 带检查点的 渲染。

  • Adaptive Metric（自适应度量）： 选择 采样方法 供自适应采样器使用

    • Exposure Bracket (variance)（曝光包围，variance）：  最小曝光包围用于确定最大强度；超过该强度后，任何强度方差、对比或差异在视觉上都不再重要。另外，最大曝光包围用于确定在假设合成结果位于此范围内时，为得到最大导数需要沿 gamma 校正曲线向下走多远。

    • Dark Falloff (contrast)（暗部衰减，contrast）： 如果图像中的某个区域暗于此阈值，则停止采样，因为它将不可见或难以被感知

  • Pixel Variance（像素方差）： 这是渲染的主要质量控制。值越低，质量越高。典型生产设置为 0.01 到 0.005（极高质量）。根据场景复杂度，0.015 通常是较好的中等质量设置。

  • Interactive Refinement（交互式细化）： 在交互式渲染中，渲染器被允许以多激进的方式对一帧进行粗略采样。数值越高，在收敛前越激进、越模糊。

  • Sampling（采样）

    • Minimum Samples（最小采样数）： 每个像素所需的最低采样数。值为 0 时，会取 Maximum Samples 设置的最接近平方根。为了捕捉小型图像特征，例如小粒子或细发丝，可能需要更高的数值。

    • Extra Minimum Samples（额外最小采样数）： 如果某个像素未通过方差测试，此值会在再次测试方差前强制增加指定数量的采样。为 0 时，默认使用与 Minimum Samples 相同的数量。

    • Maximum Samples（最大采样数）： 在停止对一个像素采样之前，我们会采用的最多采样数。渲染景深、运动模糊或其他噪声较大的效果时，可能需要更高的数值。

    • Adapt All（适应全部）： 将所有 AOVs 都与方差阈值进行测试。这意味着 AOVs 更可能没有噪声，但会显著增加渲染时间，因为某些 AOVs 可能触发更多采样，却对图像质量提升不大。

  • Pixel Filter Mode（像素滤波模式）

    • Weighted（加权）： 采样会一起滤波。仅在 RIS 中支持。

    • Importance（重要性）： 像素彼此独立。这是降噪应用所必需的，包括 Optix 和 RenderMan 的交互式机器学习降噪器。

  • Sample Offset（采样偏移）： 允许并行渲染多张图像（使用不同的 sampleoffset 值），然后再将它们合成

  • Extreme Motion/DOF（极端运动/景深）： 当景深或运动模糊在图像中极端或占主导时，改变采样技术。这更准确，但会带来一些额外开销。

  • Default Ray Depth（默认射线深度）： 默认 深度 按散射类型（漫反射或镜面反射）全局应用到场景中的所有对象。对象属性可以覆盖此值，但不能覆盖积分器的 Max Depth（最大深度）。

  • Filter（滤波器）： 父图像滤波器的类型和窗口大小；默认的 Gaussian 2.0 2.0 通常适用于绝大多数情况。


着色



另请参见 Renderer Options（渲染器选项） 了解更多细节

  • Trace（追踪）

    • World Origin（世界原点）： 允许用户选择场景原点；这对非常大的场景很有用，因为精度问题可能导致渲染变慢，或在光线追踪中产生伪影。

      • Camera（摄影机）： 默认使用渲染摄影机的位置作为世界原点 (0, 0, 0)

      • World（世界）： 原始默认值，使用你的场景网格作为场景原点

      • World Offset（世界偏移）： 用户提供的坐标，用作渲染期间场景的新原点

    • World Offset（世界偏移）： 如果上面选择了 World Offset，则此处为新的场景原点的用户定义位置

  • Light Learning Scheme（灯光学习方案）： 此选项允许渲染器利用灯光属性、位置和遮挡信息，告知渲染器如何在场景中平衡采样以改善收敛。复杂场景比简单场景或灯光很少的场景受益更多。

灯光学习可能会带来显著的内存成本。


Render（渲染）

另请参见 Renderer Options（渲染器选项） 了解更多细节

  • Checkpoint Render（检查点渲染）： 以批处理方式渲染场景，同时逐步留下检查点文件，供稍后查看或重新启动

  • Search Paths（搜索路径）： 这些路径指向 RenderMan 应查找 自定义 插件、图像目录以及更多内容的位置

    • Shader Path（着色器路径）： 用于查找着色器插件的路径

    • Texture Path（纹理路径）： 用于搜索纹理文件的路径

    • Display Path（Display 路径）： 用于查找显示插件的路径

    • Archive Path（归档路径）： 用于查找并加载延迟读取归档/RIB 的路径

    • Procedural Path（程序化路径）： 用于查找程序化插件的路径（例如 Maya 的 Xgen）

    • Rix Plugin Path（Rix 插件路径）： 指向其他 RenderMan 插件的路径

    • Directory Map（目录映射）： 路径重映射

  • Limits（限制）： 这些设置定义渲染器某些功能的内存限制。在现代机器上，你可能有足够内存把大多数实体保存在 RAM 中；但对于渲染的某些部分，可能可以有效减少大型场景渲染所用的内存（几何体必须驻留内存）。默认情况下，我们会尝试把这些限制设为合理值。更多信息请查看 Configuration（配置）


Dicing（切分）

另请参见 Renderer Options（渲染器选项） 了解更多细节

  • Minimum Width（最小宽度）： 用于渲染头发、毛发和曲线的最小像素宽度。少量增加可能改善细曲线的抗锯齿。

  • Offscreen Multiplier（屏幕外倍率）： 屏幕外对象的切分速率倍率；这些对象可能在反射中形状较差，或投射出明显来自切分不足对象的阴影。


Statistics（统计）

另请参见 Renderer Options（渲染器选项） 了解更多细节

  • Debug（调试）： 允许输出着色调试消息

  • Render Statistics（渲染统计）： 渲染完成时输出渲染统计信息


Advanced（高级）


另请参见 Renderer Options（渲染器选项） 了解更多细节

  • LPE： 本节定义输出到 LPE 的波瓣，其中包括来自 PxrSurface 和 PxrMarschnerHair 的波瓣。自定义材质也可以在此处定义作为用户波瓣输出的波瓣。请注意，每组 Diffuse、Specular 和 User 的 1 号位置都是空的，用作透传项。

  • Bucket Order（bucket 顺序）： 渲染桶顺序。支持的值：horizontal、vertical、zigzag-x、zigzag-y、spacefill、random、spiral、circle

  • Order Origin（顺序原点）： 使用 Spiral 或 Circle 选项时，指定第一个被渲染的桶/瓦片（bucket/tile）

  • Frame Aspect（帧宽高比）： 渲染输出图像的宽高比

  • Variable Substitution（变量替换）： 仅用于 RIB 文件解析
