Checkpoints and Recovery

Source: https://rmanwiki-27.pixar.com/space/RFH27/544645601/Checkpoints+and+Recovery
Downloaded: 2026-06-15T03:49:12+00:00

仅适用于 Rendering to Disk（渲染到磁盘）。Checkpoints（检查点）和 recovery（恢复）会被 interactive rendering（交互式渲染）忽略。

Recovery（恢复）

Recovery（恢复）允许从先前暂停或未完成就停止的 render（渲染）继续/恢复。若要启用它，请勾选 Recover Render（恢复渲染）（位于 RenderMan RIS ROP 的 Advanced（高级） 选项卡中）。 

Checkpoints（检查点）

render recovery（渲染恢复）通常与 checkpointing（检查点）一起使用。若要启用它，请勾选 Recover Render（恢复渲染）（位于 RenderMan RIS ROP 的 Advanced（高级） > Render（渲染） 选项卡中）。

  • Interval（间隔）- 检查点保存之间的时间

  • Exit At（退出时间）- 在此时间退出渲染，不再继续渲染

  • As Final（作为最终结果）- 即使渲染完成，也保存继续该渲染所需的数据。这会导致文件更大

  • Keep Files（保留文件）- Name（命名）并保留所有过去的 checkpoints（检查点），而不是只保留最近的一个

  • Command（命令）- RIB 文件位置的命令（如果生成了 RIB 文件）

Incremental Renders（增量渲染）

未启用 checkpointing（检查点）时，incremental renders（增量渲染）无法恢复。启用 incremental（增量）但关闭 checkpoints（检查点）时，buckets（桶）直到渲染结束才会写出。见下表：

Recovery（恢复）和 'keepfiles'

如果想使用 recover（恢复）功能，请不要为 checkpointing（检查点）启用 keepfiles 选项。

Option "checkpoint" "uniform int keepfiles" [0]（checkpoint keepfiles 选项）

keepfiles 选项会让 prman 为调试目的保存一系列 checkpoints（检查点），为每个 checkpoint image（检查点图像）赋予带有 increment（增量编号）和 elapsed render time（已用渲染时间）标记的唯一文件名。但是尝试恢复文件时，prman 不会扫描目录中的 checkpointed files（检查点文件），也不会尝试选择其中一个；它只会按原始 render output name（渲染输出名称）查找文件。
