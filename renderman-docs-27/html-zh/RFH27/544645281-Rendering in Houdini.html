Houdini 暴露了用于 render frames（渲染帧）和 sequences（序列）的所有实用功能。下方列出了涵盖的主题，并附有你将看到的功能说明。

  • Output（输出）
  • Options（选项）

渲染已完成的 scene（场景）包含许多部分。从处理 quality（质量）到为后续 compositing（合成）选择合适的 outputs（输出），本节会涵盖所有这些主题。

Quality（质量）

为了理解 noise removal（噪声移除）和 anti-aliasing（抗锯齿）的工作方式，我们为用户介绍 Sampling（采样） 以及 Filtering（滤波）。这是你可能进行大部分 tuning（调校）的地方，理解 quality 与 performance（性能）之间的平衡非常重要。

Trace Depth（追踪深度）

控制 trace depth（追踪深度）可以改变 scene（场景）的 look（外观）和 performance（性能）。不同 integrators（积分器）对这些设置的处理方式也不同。RenderMan 让这些参数的调校更容易，使你能在 resource constraints（资源约束）内实现所需外观。

Interactive Rendering（交互式渲染）与 Checkpointing（检查点）

Progressive（渐进式）/Interactive（交互式） Rendering（渲染）可以让 scene materials（场景材质）和 lights（灯光）的调校变得轻松。 Checkpointing（检查点） 与 Recovery（恢复）会把这种 progressive（渐进式）功能扩展到 final renders（最终渲染），让你能在 renders 逐步 refine（细化）时创建 save points（保存点）。你可以用它更快批准未完成 frames，并从离开的地方继续，而不会浪费宝贵的 CPU cycles！

Diagnostics（诊断）

RenderMan 的 diagnostics（诊断） 可以通过易读的 display（显示）帮助你可视化 scene performance（场景性能）。有时并不清楚 scene 在哪里花费了最多 time（时间）或 resources（资源），但我们的 diagnostic output（诊断输出）会帮助你定位甚至非常隐蔽的 performance information（性能信息）。

Holdouts and Outputs（遮挡保留与输出）

Holdout（遮挡保留） workflow（工作流）对于将 CG objects（CG 对象）集成到 live action plates（实拍底板）中至关重要。在某些应用中，这只需要点击一个按钮！为了改进 compositing integration（合成集成），甚至改进完整渲染的 shots（镜头），RenderMan 会使用强大的 outputs（输出），以便在 compositing（合成）中更轻松且更灵活地进行调整。

Baking（烘焙）

在 rendering（渲染）中复用某些 data（数据）可以简化 asset management（资产管理），同时提升 rendering efficiency（渲染效率）。 Baking（烘焙） 提供了 bake（烘焙）到 2D 或 point cloud output（点云输出）的方法。

Nested Dielectrics（嵌套电介质）

渲染彼此相互作用的 surfaces（曲面）时，尤其是 transparent/refractive（透明/折射）曲面，理解 Intersect Priority（相交优先级） settings（设置）很重要。Intersect priority 允许用户只需添加 attributes（属性）即可获得正确 refractions（折射），无需通过繁琐建模来解决问题。
