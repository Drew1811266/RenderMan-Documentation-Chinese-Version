Tractor Spool Panel

Source: https://rmanwiki-27.pixar.com/space/RFH27/544642287/Tractor+Spool+Panel
Downloaded: 2026-06-15T03:49:12+00:00

Tractor 会将任务分发到由执行服务器组成的渲染农场。它会管理来自多个用户的大量并发作业队列，同时执行依赖关系和调度策略。

在 Solaris 中，点击任意 UI pane（UI 面板）中的 + 图标即可打开 Tractor Spool panel（Tractor 提交面板）。

Spool to（提交到）

选择将作业提交到 LocalQueue 还是 Tractor 渲染农场

Render Node（渲染节点）

选择要提交的 USD Render ROP 节点

Render Threads（渲染线程）

用于渲染的线程数。0 表示使用所有可用线程。

Engine Host（Engine 主机）

运行 Tractor Engine 的机器主机名。

Engine Port（Engine 端口）

Tractor Engine 监听的端口。默认值为 80

User（用户）

提交作业的用户名。

Paused（暂停）

选中此选项时，作业会以暂停状态提交。

Delete Temp Files（删除临时文件）

删除批量渲染所需的所有临时文件

Priority（优先级）

作业的优先级。

Service（服务）

此作业的服务键。

Env Keys（环境键）

可以指定多个键，之间应以空格分隔。

LocalQueue 和 Tractor 开箱即可识别以下环境键：
- prman-*（例如 prman-24.0）
- rmantree=*（例如 rmantree=$RMANTREE）

After（之后）

将作业处理的开始时间延迟到指定时间
格式：MONTH/DAY HOUR:MINUTES

例如：11/24 13:45

Crews（团队）

Crews 列表。
请参见 Tractor 文档中的 'Crews'。
Tier（层级）

该作业所属的派发层级。
请参见 Tractor 文档中的 Scheduling Modes。

Projects（项目）

该作业所属的项目。
请参见 Tractor 文档中的 'Limits Configuration'。

Comments（注释）

关于该作业的附加注释

Metadata（元数据）

要添加到作业上的元数据。

When Done Command（成功完成时命令）

作业无错误完成时运行的命令。

When Always Command（始终运行命令）

无论作业完成时是否有错误都会运行的命令。
