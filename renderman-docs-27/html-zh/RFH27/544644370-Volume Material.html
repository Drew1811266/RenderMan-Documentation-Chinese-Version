PxrVolume 是一种材质，可用于渲染复杂度差异极大的体积效果，包括：

  • Constant fog（恒定雾） （一种恒定的 均匀 体积，具有 单次散射);

  • Smoke（烟雾） （一种 非均匀 体积，单次散射即可满足需求）；

  • Fire（火） （一种具有 发光 属性的非均匀体积）；

  • Clouds（云） （一种通常需要 多重散射 的非均匀体积）

PxrVolume 旨在作为一种基础材质，输入少量体积属性即可使用。复杂体积属性的建模应由上游节点处理，或由模拟工具烘焙到 OpenVDB 等体积文件格式中。

PxrVolume 用户应了解均匀体积与非均匀体积的区别：均匀体积具有恒定的密度和颜色，而非均匀体积的密度和/或颜色会变化。非均匀体积的渲染通常比均匀体积慢得多，也更容易产生噪点。

PxrVolume 是一个 专用的 体积着色器，经过优化，专门用于处理 "pure"（纯粹）体积。它无法处理两种不同介质边界处的折射或反射等任何表面效果，因此不能使用 PxrVolume 来渲染浑浊玻璃或水面这类对象。如果你需要既能处理表面效果、又能处理有限体积效果的 shader（着色器），请查看 PxrSurface。


Basic Workflows（基础工作流）

Constant fog（恒定雾）

恒定的雾效（即涉及均匀体积的效果）只使用 PxrVolume 参数即可轻松建模，不需要任何输入连接。这会提供最高效的渲染。

对于任何包围相机或包裹光源的雾效，PxrVolume 必须附加到 RiVolume 对象上。否则，PxrVolume 可以附加到任何封闭几何体上，用于创建形状区域等简单效果。

如果只需要深度衰减， Diffuse Color 可以设为黑色：光不会被体积散射，只会被吸收，并且 Density 是唯一重要的参数。否则，diffuse color 需要设为雾的颜色。


Smoke（烟雾）

对于 Smoke 等非均匀介质，典型做法是将烟雾属性烘焙到 OpenVDB file 等文件中。OpenVDB file 会包含少量属性，包括密度、velocity（如果使用 motion blur）以及可能的 albedo。OpenVDB 应附加到 RiVolume 并使用 impl_openvdb plugin 读取。随后会将 OpenVDB file 中密度属性的名称直接用于 PxrVolume 的 Density Float PrimVar 参数。


Smoke 通常具有很低的 albedo，因此 Diffuse Color 参数会使用较暗的颜色。（如果颜色已烘焙到 OpenVDB file 中，可使用 PxrPrimVar node 从中提取 color 属性。）由于这种低 albedo，体积中多次散射后的光量通常很低，因此 Multiple Scattering 可以保持关闭，并且 Samples 可以提高到 1 以上，以加快渲染。

由于 Smoke 通常是各向同性的， Anisotropy 设置可以保持默认。 Equiangular weight 如果预计光源位于体积附近或体积内部，可保持原值；如果预计没有任何光源靠近体积，可设为 0.0。


Clouds（云）

Clouds 是非常难渲染的体积，因为水滴的聚合行为及其与光的交互是复杂现象。与 Smoke 不同，从云到达眼睛的光通常已经在云内部反弹了数十甚至数百次。除此之外，散射行为（光在云内部反弹的方式）也非常复杂，因为它既涉及前向散射，也涉及后向散射，并且两者都有多个峰值。

对云本身建模的典型方法，是将云的密度烘焙到 OpenVDB file 等文件中，然后将 OpenVDB file 中密度属性的名称直接用于 PxrVolume 的 Density Float PrimVar 参数。Clouds 通常会反射所有光，因此一个简单的 Diffuse Color （1, 1, 1）设置就足够。为了捕捉云本身复杂的光照行为， Multiple Scattering 是必需的，并且还需要设置 Primary and Secondary Anisotropy。

下方 Anisotropy 设置体现了对真实云复杂散射行为的物理近似。由于使用了 multiple scattering，sample 参数不会被使用，可设为 1。最后，如果云内部没有光源， Equiangular weight 可以设为 0.0，因为 equiangular sampling 不会带来收益。（另一方面，在暴风天气中，如果可能在云内部放置光源来模拟闪电，equiangular sampling 可能会有帮助。）

注意，体积中的散射由所选 integrator 中的 Max Path Length 参数以及 diffuse trace depth 控制。更多信息可在下方名为 Multiple Scattering。

Advanced Workflows（高级工作流）

Crepuscular Beams（曙暮光束）

Crepuscular beams（有时称为“God rays（神光）”）只是体积中光散射的结果。当这种光被分离成光束时（例如穿过森林冠层的几束阳光），体积中会散射光的部分与其余部分之间的对比会形成醒目的效果。


为了高效渲染 Crepuscular beams，需要尽量最大化 直接照明 - 即最大化体积中可直接追踪到光源的部分。如果体积必须经过另一个表面或材质，以间接方式追踪到光源，光束的收敛可能会非常慢，并产生不可接受的噪点。在上图中，右侧的彩色玻璃窗被建模为 “thin glass（薄玻璃）”：它不是依赖二次材质反弹来经玻璃折射，而是让从体积朝光源追踪的 shadow rays 不受阻挡（不折射），只依赖彩色玻璃材质的 shadow tinting（阴影着色）行为。“thin” 参数在 PxrSurface 中（位于 glass 参数下）可实现这一优化。

Performance Suggestions（性能建议）

Density 是体积最关键的属性。它同时衡量体积会吸收多少光、散射多少光。一次渲染过程中，体积积分可能需要数十亿次密度求值，因此必须确保渲染器能尽快评估密度。这也是将复杂信号烘焙到 OpenVDB 等体积文件格式中会很有用的原因，因为渲染器可以直接从数据中查找值。为了获得最佳性能，我们强烈建议尽可能使用 PrimVar density parameters ，而不是使用输入连接，即使该输入连接只是一个单独的 PxrPrimVar node。

对于使用 RiVolume primitive 的非均匀体积，在渲染开始时，PxrVolume 会自动计算体积内部各处的密度，并在渲染过程中利用这些信息大幅加速密度采样。此计算的精度（以及变形运动模糊的精度）由 dice attribute micropolygonlength 控制。将 micropolygonlength 设为较低数值会得到非常精确的密度估计，但可能导致初始计算耗时很长（表现为第一次渲染迭代很慢），也可能导致渲染器使用大量内存。将该值设得过高会使密度估计不够准确，可能漏掉体积中局部密度很高的孤立区域。对于 2K 渲染，除非变形运动模糊需要更低设置，否则建议使用 5 或更高的设置。如果相机穿过体积或靠近体积，可能适合将 dicing projection 从 planar 改为 world 或 spherical。这样可能会损失相机附近的一些细节，但也可能避免在镜头附近生成大量数据。

Emissive volumes（发光体积） 可能收敛非常慢，尤其当它们是场景中唯一的光源时，可能需要数百甚至数千个 camera samples 才能渲染出无噪图像。在体积被支持为可用于直接照明的完整光源之前，我们建议非常谨慎地使用发光体积。

Multiple Scattering 是一种非常昂贵的效果，因为它意味着光会在体积中多反弹许多次，同时也因为 PxrVolume 本身无法采集更多样本（ Samples 参数会被忽略），只能受 Integrator 设置支配。此外，multiple scattering 可能会减慢体积后方对象的收敛。因此，我们建议尽可能避免 multiple scattering。对于低 albedo 介质，multiple scattering 对最终渲染的贡献很小。对于 Clouds 等介质，其光传输涉及数十或数百次光反弹，遗憾的是 multiple scattering 可能无法避免。

使用 单次散射时，提高体积中采集的 Samples 数量来改善收敛，通常比增加 camera samples 数量更具成本效益。

Equiangular sampling（等角采样） 可改善由体积内部或附近光源照明的体积的收敛；但如果光源很远，反而会拖慢体积渲染。equiangular sampling 的默认值（0.5）是一种折衷，在大多数场景中表现尚可。如果确定体积附近没有光源，应将 equiangular weight 设为 0.0 以加快渲染。在 21.5 及更高版本中，multi-scatter volumes 会禁用 Equiangular sampling。

在 samples 较低时，Equiangular sampling 可能导致 volume alpha 出现负值。这是该参数工作方式产生的副作用。有两种解决方式：将 Equiangular Weight 设为 0.0，或允许采集更多 samples，使结果正确平均为正数。


Anisotropic 体积由于所涉及采样的本质，往往比各向同性体积收敛更慢。
 

Parameters（参数）


Diffuse Color

体积的颜色。默认值为白色（1,1,1）。在此参数上建立连接会创建非均匀体积。

如果熟悉散射、吸收和消光系数，请注意这里的 diffuse color 是 scattering albedo（散射反照率），即散射系数除以消光系数。

Emit Color

体积的发光颜色。这可用于建模磷光雾或火焰等效果。默认值为黑色（0,0,0），即体积不会发光。下列图像展示了在场景中没有其他光源时，将 emit color 设为（1, 1, 1）和（5, 2.5, 2.5）的效果。

Light Source

选择此选项时，体积的发光属性会作为光源采样。通过改进采样，这可能会使场景变亮。注意，emit color 不能为黑色，否则此功能不会生效。对于单次散射体积，即使 Max Path Length 为 1，它也等价于关闭 light source 选项的 multiscatter volume。下方为关闭与开启的对比。

属于 聚合体积 不能作为光源处理。在某些 DCCs 中，将体积放入 aggregate（聚合）意味着选择 light source（光源）选项不会产生效果，并且需要将该体积从 aggregate 中移除。

如果体积光源还需要配合高级照明功能，例如 light filters（灯光滤镜），则无法使用 light source 选项。在这种情况下，需要创建体积副本，并附加 PxrMeshLight shader。有关此工作流设置的更多细节，请参阅下方 Using Meshlight 部分。

Multiple Scattering

使用 聚合体积，因为聚合体积始终执行多重散射。

此参数用于提示体积是否应在体积内部计算间接照明（也称为 多重散射，因为光会在体积中散射 不止一次）。如果多重散射参数设为 0，并且积分器遵循此提示，PxrVolume 将只执行 单次散射：体积内部的点只会由光源直接照明。如果设为 1，体积内部的点也会受到间接照明。下方第一张图将 multiple scattering 设为 0，即单次散射体积。中间图开启 multiple scattering，并使用 2 次光反弹。右侧图开启 multiple scattering，并使用 4 次光反弹。

对于具有高各向异性的极高密度体积，光在到达眼睛前往往会在体积内部散射许多次，multiple scattering 是获得正确外观的唯一方式。它也常常是正确渲染体积焦散等特定效果的唯一方式。另一方面，multiple scattering 可能非常昂贵，这也是该参数默认为 0（关闭）的原因。此外，在渲染均匀体积时，关闭 multiple scattering 可能会让直接照明的 表面 在体积后方时噪点更少。

注意，由于某些 integrators 的算法性质，尤其是 PxrVCM 等双向路径追踪器，它们只能实现体积的 multiple scattering，因此会忽略这一提示。

Velocity and Motion Blur（速度与运动模糊）

与 RenderMan 中所有其他几何体类型不同， deformation motion-blur（变形运动模糊）体积主要通过 Bxdf 控制启用。使用 PxrVolume 启用变形运动模糊很直接：为 velocity 参数指定一个 vector-valued 值。在底层，渲染器会自动使用 velocity vector 为所有 varying inputs 生成随时间变化的数据。这个预处理步骤发生在渲染开始时，并会增加首个像素时间；不过，为了大幅提高渲染模糊体积的效率，这是必要的。

渲染器支持使用 "Eulerian" 方法处理 deformation motion-blur（变形运动模糊）体积，该方法在 23.5 中加入（现在也是唯一支持的方法）。此方法预处理时间短、内存开销很低、bias 较低（假设 micropolygon length 设为合理值），并且相当准确。这项技术的缺点是 velocity 值 必须非零，并且必须在运动路径的所有位置提供，即使体积在当前烘焙 shutter time 的该路径上没有“presence”也一样。这是因为 Eulerian 技术假定体积随时间变化的属性（例如随时间变化的密度）可以通过使用当前时间的 velocity 值进行回溯，并在回溯位置查找密度来近似。下列来自 Houdini 的图像展示了 Eulerian motion blur 的边界速度要求。第 1 张图显示两个带密度的球体，以及代表 velocity field bounds 的线框盒。绿色线条表示体积预期运动。如果 velocity field 没有在整个运动范围内完整写出，Eulerian motion blur 会像右侧渲染图像所示那样裁切 velocity。注意，在某些高度混沌的流体场景中，即使不满足此要求，渲染图像仍可能看起来可以接受（裁切可能并不明显）。

velocity vector 值应相对于整个 shutter range。如果使用以每秒单位测量的 velocity data，而 shutter 未设置为整整一秒的时间长度，可能需要按每秒帧数（即 1/24.0）缩放数据，才能得到正确图像。我们提供了 Velocity Multiplier 控制项，帮助在数据测量方式不正确时进行转换，也允许对 motion blur 做艺术化调整。或者，也可以直接在几何体上设置 Attribute "volume" "fps" 来完成同样的事；在这种情况下，该属性的值应直接是每秒帧数（24），而不是倒数。

下列图像使用 Eulerian motion blur 渲染，并结合了一个包含 velocity grid 的 OpenVDB file。左图在没有 blur 的情况下渲染，右图则通过连接到 PxrPrimVar node 从文件读取 velocity grid 来提供 velocity。velocity 被夸大了 10 倍。注意，这个 OpenVDB file（来自 openvdb.org）不满足在所有运动区域都具有 velocity data 的边界要求；这一点可从体积轮廓边缘在夸大的运动下仍没有显著变化看出。不过，没有明显令人反感的伪影，并且远离边界的体积区域传达了所需的 velocity detail。


 Density Parameters（密度参数）

Extinction Distance

这是均匀体积变为不透明时的距离（以 scene units 计）。如果此值非零，它会覆盖 density connections，并保持均匀体积。这非常适合创建填满场景的 fog/mist。下方我们先将参数设为较低值，然后逐渐增大，直到场景中最后一盏灯清晰可见。


Density Float PrimVar

来自几何体的 primvar 会覆盖体积的密度 float 值，默认未设置。设置此项会覆盖 densityFloat 输入，并且比简单使用 PxrPrimVar 图案连接更高效。

对于使用烘焙体积表示（例如 OpenVDB file）的工作流，我们强烈建议使用此参数设置，而不是使用连接到 Density Float 参数的 noise pattern。

Density Float

体积密度直接控制光如何被体积衰减，即直接影响体积如何投射 shadows（阴影）。如果 density color 从默认值改变，density float 参数会被 density color 覆盖。除非需要 colored shadows（彩色阴影），否则应优先设置此参数，而不是 density color 参数，因为这样体积渲染会更高效。

在此参数上建立 input connection 会创建非均匀体积。

密度也称为消光系数（吸收系数与散射系数之和）。

下方从左到右：密度为 0.1、0.25 和 2.0。

Density Color PrimVar

来自几何体的 primvar 会覆盖体积的密度颜色，默认未设置。设置此项会覆盖 densityColor 输入，并且比简单使用 PxrPrimVar 图案连接更高效。

对于使用烘焙体积表示（例如 OpenVDB file）的工作流，我们强烈建议使用此参数设置，而不是使用连接到 Density Color 参数的输入。

Density Color

体积密度直接控制光如何被体积衰减，即直接影响体积如何投射 shadows（阴影）。如果 density color 从默认值改变，density color 参数会覆盖 density float。如果不需要来自体积的 colored shadows（彩色阴影），则应设置 density float 参数，并保持 density color 参数不变；这样体积渲染会更高效。

在此参数上建立 input connection 会创建非均匀体积。

密度也称为消光系数（吸收系数与散射系数之和）。

下方从左到右：density color 为（1.0, 0.25, 0.25）、（2.0, 0.25, 0.25）和（5.0, 0.25, 0.25）。注意，由于体积密度在红色通道中更高，更多红光会被散射掉，留下青色贡献。

Max Density

max density 参数只由非均匀体积使用，并控制用于采样体积的步长。为了获得正确的无偏渲染，max density 必须高于体积内部遇到的任何 density。较高的 max density 值可能导致渲染变慢，因为需要更多步来采样体积。将 max density 设得过小会加快渲染，但也会导致不正确的（有偏）渲染，因为体积中的高密度区域会采样不足。

从 20.7 版开始，仅对于 RiVolumes，PxrVolume 还会自动计算体积各子区域的 density bounds，并利用这些信息沿光线改变步长。这意味着只有在体积中密度更高的部分才会采用更多步。因此，对于非常非均匀的体积，设置较高的 max density 值不会再对整个体积产生负面影响；在实践中，通常可以直接将其设为很高的值，而不再担心该参数。

max density 的默认值为 1.0。

 Anisotropy Parameters（各向异性参数）

Primary Anisotropy

控制体积散射光的方向。anisotropy 参数范围为 -1 到 1，默认值为 0。值为 0 表示体积是各向同性的：光以相等概率向所有方向散射。大于 0 的正值表示体积为前向散射：入射光更可能沿同一方向散射（远离入射光方向）。小于 0 的 anisotropy 值表示体积为后向散射：入射光更可能沿反方向散射（朝向入射光方向返回）。

许多体积（例如 Smoke）通常是各向同性的，但其他体积可能具有很强的各向异性。例如，肉和皮肤等有机材料往往具有很强的前向散射特性。注意，一旦涉及 multiple scattering，高各向异性的影响会减弱，因为这类材质在多次光反弹后往往会收敛到各向同性行为。

Secondary Anisotropy and Lobe Blend Factor（二级各向异性与波瓣混合因子）

某些体积材质的散射特性无法用单个 anisotropy lobe 捕捉。例如，Clouds 的散射行为非常复杂：它们大多具有很强的前向散射，但仍有相当大的后向散射分量，这很大程度上造成了暗云上可见的 “silver lining（银边亮缘）”。为了渲染这类材质，可以在体积上启用第二个 anisotropy lobe，并配合一个 blend factor，用于决定如何在两个独立 lobe 之间混合。blend factor 为 0（默认）表示只考虑 primary anisotropy。blend factor 为 1 表示只使用 secondary anisotropy，而 blend factor 为 0.5 表示两个 lobe 被同等考虑。

从左到右：对于 primary anisotropy = 0.0、secondary anisotropy = 0.8 的单次散射体积，lobe blend factor 分别设为 0、0.5 和 1.0。注意，第一张和第三张图本质上使用单个 anisotropic lobe 渲染。

从左到右：对于 primary anisotropy = 0.0、secondary anisotropy = 0.8 的 multiple scattering 体积，lobe blend factor 分别设为 0、0.5 和 1.0。注意，第一张和第三张图本质上使用单个 anisotropic lobe 渲染。

 Sampling Parameters（采样参数）

这些 sampling parameters 对 聚合体积。

Equiangular Weight

设置使用 equiangular sampling 而不是 density sampling 的概率。Equiangular sampling 可改善靠近光源的体积的收敛，而处理密集或密度高度变化的体积时，density sampling 可能是更好的技术。默认值 0.5 表示两种技术会同等使用，并通过 multiple importance sampling 组合。如果体积较密且远离光源，降低 equiangular weight 可能会改善收敛。此设置不适用于 multi-scatter volumes。

min/max Samples

此设置控制单次散射体积的 sample quality。更多 volume samples 会以渲染时间为代价改善体积收敛。不过，在包含其他对象的场景中，这种取舍是：体积会采样更多，但不会增加图像其他区域的成本，并可能改善结果。注意，体积内部的收敛速率也会变化，这意味着某些区域用少量 volume samples 就能合理收敛，而其他区域可能需要更多帮助。

  • minSamples 强制设置采样下限，仅在 Single Scattering 下有效

  • maxSamples 是 Volume 内每条 ray 的最大限制

 Multiscatter Optimization

Multiscatter Optimization (Valid for PxrPathTracer)（多重散射优化，适用于 PxrPathTracer）

On/Off，默认值为 off。

Extinction Multiplier

此参数在 Multiscatter 体积渲染期间使用。该值必须大于 0.0，默认值为 1.0，即不做修改。低于 1 的值会降低 extinction coefficients，高于 1.0 的值会提高它们。

Contribution Multiplier

此 multiplier 在 Multiscatter 体积渲染期间使用。此参数会乘到 multi scatter events 的直接照明贡献上。值必须大于 0.0，默认值为 1.0，即不做修改。低于 1.0 的值会降低直接照明贡献，高于 1.0 的值会提高它们（这可能让体积显得更柔和、更半透明）。

 Using Meshlight（使用 Meshlight）

上面的 “Light Source” 参数在使用具有发光属性的体积时适用于很多场景。但有时需要控制来自体积的光的影响范围。这需要使用 light filter（灯光滤镜）。PxrVolume material 无法使用 light filter，因此可以将 PxrMeshlight 将材质与用于着色的 PxrVolume 一起指定给体积对象。

这样做会照亮场景，同时允许通过 mesh light controls 连接并使用 light filter。但此工作流有一个注意事项，需要在设置时额外考虑。

将 mesh light 应用于体积时，可能会注意到照明增强和/或额外噪点。这是因为体积发光与 mesh light 本身产生了双重贡献。以下是当前用于减少此问题的步骤。

如果不需要 light filter，则此工作流是不必要的，直接使用 “Light Source” 参数即可。

  1. 复制原始体积

    1. 在副本上指定一个重复的 PxrVolume，并让它正确连接到原始 patterns；这些 patterns 可能用于驱动 emission、density 等

    2. DCC SG Node/Material Builder: PxrMeshlight -> 体积

    3. 从 indirect visibility（间接可见性）中隐藏此体积；我们只需要光照贡献。

  2. 在原始体积中断开 emission；它现在会由副本体积的 mesh light source 照亮。这可防止双重光照贡献

  3. 按需将 light filter 应用于带有 meshlight 的副本体积
