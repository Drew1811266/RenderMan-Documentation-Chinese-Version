Texture Manager

Source: https://rmanwiki-27.pixar.com/space/RFH27/544644341/Texture+Manager
Downloaded: 2026-06-15T03:49:12+00:00

Texture Manager（纹理管理器）的职责很简单：将图像转换为 RenderMan 的纹理格式，可使用 OpenImageIO 或 txmake。它既处理单张图像，也处理纹理图集（UDIM、Mudbox、ZBrush）。

你可以在 Houdini 中选择 New Tab，然后在列表底部选择 RenderMan Texture Manager；该列表属于 New Pane Type（新窗格类型）。

用户界面（User Interface）

打开新场景时，Texture Manager 会找到所有作为纹理使用的图像，并将它们加入转换队列。转换设置会基于一组规则，这些规则定义在 JSON 文件 中（见下文）。使用 Settings（设置）窗格修改当前所选图像的设置，然后点击 Apply（应用）按钮，用新设置重新转换图像。

只需按下 Parse Scene（解析场景） 即可在需要时转换新纹理。 Pick Images（选择图像） 和 Pick Directories（选择目录） 允许你预先转换图像，而不必了解如何使用 OpenImageIO 或 txmake。

Stop / Start the conversion queue（停止/启动转换队列）

默认情况下，转换队列始终运行，但有时你可能希望停止转换过程。

可以点击进度条右侧的图标来停止和重新启动转换队列。队列停止时，正在运行的 txmake 进程会立即被终止，但已有纹理会保持不变。重新启动后，转换会从上次停止的位置继续。

Conversion information（转换信息）

纹理管理器窗口底部有磁盘用量和进度指示器。

列表中的每个文件也都有一个状态图标。

交互式与批量渲染

  • 在交互式渲染期间，纹理管理器会把新纹理加入队列，并在转换完成前返回一个临时纹理。转换完成后，它会向渲染器发送更新，你的纹理就会显示出来。

  • 当你启动 Batch（批量） 或 Preview（预览） 渲染时，需要先完成所有纹理转换。

Settings（设置）

Backend（后端）： OpenImageIO 会使用 OpenImageIO 库进行纹理转换，并启动 rmanoiiotool 进程。 Legacy（旧版） 会使用 RenderMan 的 txmake 进程。此更改会在重新启动 Houdini 后生效。

Fallback Path（备用路径）：默认情况下，纹理管理器会将纹理保存在源图像旁边。如果由于任何原因无法保存到该目录，就会使用备用纹理路径。

Ignore Texture（忽略纹理）：列出不需要由纹理管理器转换的扩展名。

Workers（工作进程）：同时启动的工作进程数量。默认值为 2，但增加更多进程通常有助于加快队列速度。

Always Fallback（始终使用备用路径）：始终将转换后的纹理保存到备用纹理目录。

Keep Extensions（保留扩展名）： 保留原始输入图像的扩展名。

Show Advanced（显示高级选项）：在用户界面中显示更多高级选项。

Texture Manager Configuration（纹理管理器配置）

纹理管理器会根据以下内容，使用一组规则选择最佳的 OpenImageIO/txmake 参数：

  • 使用该纹理的节点类型： pattern, light, lightfilter 或 imageplane。

  • 该节点类型的一组基础参数

  • 在考虑文件名、位深等因素时，用于修改基础参数的一条或多条规则。

Custom（自定义）设置可以通过 OCIO 配置定义。 

默认规则是为效率而构建的；除非有特定的流水线需求，否则你不应需要修改它们。
