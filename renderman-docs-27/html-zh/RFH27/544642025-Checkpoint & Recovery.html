Checkpoint & Recovery

Source: https://rmanwiki-27.pixar.com/space/RFH27/544642025/Checkpoint+%26+Recovery
Downloaded: 2026-06-15T03:49:12+00:00

PRMan 具备恢复中断渲染的能力。具体方式取决于它运行的 mode（模式）。输出到 TIFF 和 OpenEXR 图像的 Non-incremental renders（非增量渲染）始终可以恢复。

输出到 OpenEXR 图像的 incremental renders（增量渲染）也可以恢复，但前提是使用了 checkpointing（检查点）选项。

Checkpointing（检查点）可以在 rendermanrendersettings node 的 Rendering → Render → Checkpoint Render 下找到

Checkpoint Exit at
此方法通过限制总渲染时间来工作（使用与 periodic checkpoints（周期检查点）相同的记法指定）。除非达到时间限制，否则渲染会正常进行；达到限制时，它会完成当前 increment（增量），写出 checkpoint image（检查点图像），然后停止渲染该帧。如果有多帧需要渲染，此时会直接继续下一帧；每帧都可以有自己的时间限制。

可以使用此 parameter（参数）设置 renderer（渲染器）退出的 hour / minutes / seconds（小时/分钟/秒）时间。

下面是所需格式示例

  • 60s - 表示 60 秒

  • 60m - 表示 60 分钟

  • 60h - 表示 60 小时

  • 60d - 表示 60 天

Checkpoint Interval（检查点间隔）

这是 periodic checkpoint（周期检查点）。可以用 increment 数量（即 image passes（图像遍历次数））或经过的 wall clock time（墙钟时间）作为间隔来指定。例如，可以要求 renderer 每隔 100i 写出 checkpoint，这意味着每 100 个 increments 更新一次磁盘上图像，分别记录第 100、200、300 个 increment 等时的渲染状态。




两种生成 checkpoints 的方法可以一起使用。例如，可以请求每 100 个 increments 写出一个 checkpoint，直到经过 15 分钟。此时，已写出的任何 periodic checkpoints 都会在退出时被最终 checkpoint 直接替换。

请注意，checkpointing 是为 batch rendering 到磁盘图像而设计的。渲染到实时 framebuffer（如 "it"）时，渲染进行过程中已经会即时更新。在内置 display drivers 中，目前只有 TIFF 和 OpenEXR display drivers（显示驱动） 支持 checkpointing（检查点）。RenderMan LOP 上的 "Output Type" 必须设为 OpenEXR 或 TIFF，而不是 "Raster"，这样才会使用 RenderMan display drivers（显示驱动）。即使 Husk 可以在 Raster 模式中输出 OpenEXR 或 TIFF 图像，这种情况下也不会使用这些显示驱动，因此检查点不会生效。你可能会注意到 RenderMan 生成的图像文件包含标记为 ~w 的通道；这是为从该文件恢复渲染而存储的数据。



Recovery（恢复）

只要 checkpoint frames（检查点帧）已存在，中断渲染的 Recovery（恢复）就是自动的。

RenderMan 随后会照常加载 scene（场景），但不会从头开始并覆盖现有图像，而是检查这些图像来确定中断位置。如果成功，它会从接近中断的位置继续。如果图像已经完成、缺失，或由于某种原因与当前 scene 或彼此不匹配，则会静默地从头开始。

不必在开始渲染的同一台 render machine（渲染机器）上恢复渲染。只要 RenderMan process（RenderMan 进程）仍能找到 scene file（场景文件）指定的图像，并且这些图像处于一致且可恢复的状态，就可以恢复渲染。

Recovery 也可以与上述 checkpointing 选项配合使用。在 time limit（时间限制）的情况下，这基本上相当于原始时间限制的延伸。incremental renders 可以用这种方式拆分为任意数量的 time slices（时间片）。不过请注意，每次 recovery 都需要重新加载 scene 及其所有 assets，因此应谨慎使用。此外，此功能仅支持 OpenEXR 文件。
