如果用户希望利用 Houdini SOPs networks 原生的 VEX/VDB 流程，推荐的着色工作流是使用 KaBoom Box HDA 的各项功能；但如果确实想在 Houdini 之外渲染体积，也可以先用 HDA 处理 VDB 序列，再把它们导入另一个 DCC，并使用 PxrVolume presets，而不是在 Houdini 中继续使用 HDA。

对于使用 viewport（视口）进行 look-development，这也可能是更理想的工作流。将 shading calculation data（着色计算数据）缓存到 VDB 中，也可以为高分辨率或数据量很大的体积节省渲染时间。

默认情况下，为了在 Houdini 中完整使用所有功能，这些选项会保持关闭。

如果要导出到其他软件，并使用 RenderMan Preset Manager 中的 volume shaders 进行渲染，请将所有这些选项勾选为 ON（开启） 以移除 Houdini 使用的额外 attributes，并为模拟生成的 volumes 做好 OpenVDB sequence export（OpenVDB 序列导出）准备。

勾选 Export Prepped Sim Fields 参数时，会跳过生成 diffuse 和 emission color fields，因为这些字段会改由其他 DCCs 中的 preset shaders 生成。

也可以选择使用你从 KaBoom Box HDA 生成的 volume fields：保持此选项未勾选，然后在其他第三方软件中应用的 shader 里，在 Smoke、Scatter 和/或 Fire 选项卡中勾选 “Use Binding Field Only”，并在 Bindings 选项卡中输入这些字段的 string name（字符串名称）。
