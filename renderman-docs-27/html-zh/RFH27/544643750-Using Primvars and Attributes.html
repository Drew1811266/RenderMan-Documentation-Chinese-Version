Using Primvars and Attributes

Downloaded: 2026-06-15T03:49:12+00:00

图元变量（Primitive Variables，通常称为 "primvars"）是附加到几何体上的数据。当着色器需要引用来自几何体的自定义数据时，这很有用。不需要把 Houdini primvars 重新映射为 RenderMan primvars。所有几何属性都会作为 primvars 传递给 RenderMan，并会根据属性类型采用特定的变化率。 

下面示例展示 Houdini 的 point、vertex、uniform、detail 四种属性类型之间的差异，以及它们如何转换为 RenderMan primvars。无论类型为何，这些属性都会成为 RenderMan primvars，因此可通过 PxrPrimvar VOP 读取，用于在着色网络中引用相应数据。 

Houdini 属性也可用于控制着色、切分、置换或几何体专属设置等渲染方面，方法是创建 RenderMan 能识别的 图元变量（primitive variables）。可以通过 SOP 中的 Houdini detail（细节）属性设置，也可以向 OBJ 添加 RenderMan 备用参数。如果参数和属性都已设置，detail 属性会覆盖 OBJ 参数。

 

Instance Attributes（实例属性） 是附加到实例上的数据。它们用于在不同实例之间变化属性。与 primvars 类似，实例属性也可以通过 SOP 属性或 OBJ 参数添加。
