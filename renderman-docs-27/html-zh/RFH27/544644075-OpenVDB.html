OpenVDB

Source: https://rmanwiki-27.pixar.com/space/RFH27/544644075/OpenVDB
Downloaded: 2026-06-15T03:49:12+00:00

OpenVDB 是一种用于体积的开源分层数据结构。它已经成为应用程序之间交换体积数据的标准。有关 OpenVDB 的更多信息，请参阅 OpenVDB FAQ。

从 OpenVDB 文件渲染

如果你已经有 .vdb 文件，RenderMan 会使用 blobbydso:impl_openvdb procedural（程序过程）读取该文件，而不是把 .vdb 文件转换为 volume（体积）。  

只需创建一个 Geometry（几何体） SOP。进入该节点内部，并设置 File SOP 的 Geometry File 参数，使其指向你的 .vdb 文件。

这样就可以了！

RenderMan for Houdini 会自动检测 bounding box（边界框）和所有 VDB 网格名称。

将 PxrVolume 或你的 volume Bxdf（体积 Bxdf）指派给 Geometry 节点。别忘了在 PxrVolume 中正确设置 density primvar（密度 primitive variable）名称（它应匹配 .vdb 文件中的 density grid（密度网格）名称）。

如果已导出，你可以通过节点 info（信息）按钮查看可用的 primvars（图元变量）；示例包括 density、emission、temperature、velocity...

从 Houdini 体积渲染

要渲染 Houdini volume network（体积网络）或 non-VDB file（非 VDB 文件），请将其连接到 Convert VDB SOP。将 “Convert To” 改为 VDB。 渲染时，RenderMan for Houdini 会自动把 .vdb 文件写出到 Houdini 临时目录。这样，我们就能利用高效的 VDB 格式，而不会发出可能变得非常庞大的 RiVolume 数据。
