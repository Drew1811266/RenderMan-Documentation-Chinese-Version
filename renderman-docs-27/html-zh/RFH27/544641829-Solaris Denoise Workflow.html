Solaris Denoise Workflow

Source: https://rmanwiki-27.pixar.com/space/RFH27/544641829/Solaris+Denoise+Workflow
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan 要求启用特定 AOV，denoiser（降噪器）才能在 batch renders（批渲染）中正确工作。

这些 AOV 可以通过两种方式应用。

可以使用 rendermanrendervars node，也可以通过 rendermanrendersettings node。两种都是有效 workflow（工作流）。

需要激活 Denoiser AOV，最简单的方法是点击右上角的蓝色 R（下方图片中以红色高亮）。

这会启用 Denoiser 所需的全部 AOV。

激活所需的 denoising AOV 后，可以使用以下两种设置之一：

Setup 1 - ROP Fetch Node（设置 1 - ROP Fetch 节点）

  1. 在 OUT context（OUT 上下文）中创建一个 ROP fetch node（节点）

  2. 将 Source parameter（参数）设置为所需的 LOP usdrender_rop node（节点），该节点位于 Solaris/Stage context（上下文）中。

  3. 将一个 ROP denoise node（节点）连接到 ROP fetch node input（节点输入端）。

  4. 在 ROP denoise node（节点）上设置 mode type（模式类型）、output location（输出位置）、asymmetry（非对称）、flow 和 debug 选项；准备渲染后，按 Denoise 按钮。

    Houdini 将渲染 LOP usdrender_rop node（节点），使用 denoise node 上的帧范围。所有帧渲染完成后，denoise process（降噪过程）将开始。

Setup 2 - ROP USD Render Node（设置 2 - ROP USD Render 节点）

  1. 在 OUT context（OUT 上下文）中创建一个 ROP usdrender node（节点），并将 render delegate（渲染委托）改为 RenderMan RIS（Solaris 中的 denoising 目前不支持 XPU rendering（XPU 渲染））

  2. 将 LOP path parameter（参数）设置为 Solaris/Stage context（上下文）中包含某个 RenderSettings primitive 的目标节点 (最可能的默认情况是 /stage/rendermanrendersettings)

  3. 将 Render Settings（渲染设置） parameter（参数）设置为场景图树中所需的 RenderSettings primitive 所在的位置（最可能的默认情况是 /Render/rendersettings）

  4. 将一个 ROP denoise node（节点）连接到 ROP usdrender node input（节点输入端）

  5. 在 ROP denoise node（节点）上设置所需参数，然后点击 denoise 按钮，开始将帧渲染到磁盘并执行 denoise（降噪）。

Houdini 将渲染 ROP usdrender node（节点），使用 denoise node 上的帧范围。所有帧渲染完成后，denoise process（降噪过程）将开始。

Note（注意）：目前还不支持在 TOPs / PDG 中使用 Solaris 的 denoising workflow（降噪工作流）。
