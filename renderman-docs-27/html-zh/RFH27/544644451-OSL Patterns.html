创建 PxrOSL VOP 节点（注意：操作符名称为 pxrosl:2，以避免与 Houdini 随附节点冲突）。

  • oso 的 File Path（文件路径）。参数和输入会从这个 oso 加载。

  • osl 源文件使用 Source File Path（源文件路径）。它会被编译到 File Path 指定的位置。

  • 在代码块中编写 osl。 

设置 RMAN_SHADERPATH 环境变量，以从路径中加载 oso 文件。它们会根据保存位置的目录层级组织到 TAB 菜单中。 

设置环境变量 RFH_ARGS2HDA=1 后，启动时会根据在 RMAN_SHADERPATH 和/或 RMAN_RIXPLUGINPATH 中找到的 .args 或 .oso 文件创建 Houdini 数字资产，用于把这些着色器描述转成可用资产。

  • 在 $RMAN_SHADERPATH 中找到的 .oso 着色器会自动转换为 otls；转换结果会存储在 $HOUDINI_USER_PREF_DIR/otls/rfh_osllib.hda 中，作为 Houdini 后续可加载的库文件。
