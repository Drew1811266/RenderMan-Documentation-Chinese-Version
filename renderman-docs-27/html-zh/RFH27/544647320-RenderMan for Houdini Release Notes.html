欢迎使用 RenderMan for Houdini

RenderMan for Houdini（RfH）继续为最新的 RenderMan 提供完整支持。请参见 RenderMan 发行说明 了解更多细节。

当前版本支持：

  • Houdini 21.0

  • Houdini 20.5

  • Houdini 20.0

请查看下面的发行说明，了解所有新功能和已知问题。请参见 RenderMan for Houdini 安装 页面，获取 RfH 支持的具体 production builds（生产构建）列表。

使用 RenderMan 渲染无法在 Houdini Apprentice editions（Houdini Apprentice 版本）中运行。请参见 SideFX 网站，了解允许第三方渲染的选项

已知限制

Solaris

  • 由于 SideFx 的限制，在某些 shading network（着色网络）场景中，visualizer node（可视化节点）无法正确单独显示网络。

MaterialX Lama

  • 对复杂的 MaterialX Lama shading networks（着色网络）执行 solo（单独显示）。

XPU

  • CUDA device（CUDA 设备）会保存在场景中，而不是作为偏好设置保存。

General（常规）

  • 目前使用 ACES 渲染到 IT 时，需要将环境变量 IT_OCIOV1 设置为你的 ACES config.ocio。

  • 不支持使用 Takes 渲染。 

  • 无法使用 Mantra lights（Mantra 灯光）或 VOPs 进行渲染

  • 绕过 shader nodes（着色器节点）

  • Auto camera creation（IPR 自动相机创建）

  • RenderMan 裁剪平面

  • Display edits（显示编辑）无法从 image tool（图像工具）切换到 mplay。

  • 跨帧降噪 PDG 支持。

  • 对已删除节点执行 undo（撤销），或折叠为 subnet（子网），可能无法按预期工作。

  • 重新连接 subnet indirect inputs（子网间接输入）
