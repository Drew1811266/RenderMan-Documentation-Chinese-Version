AOV Setup and Viewing

Source: https://rmanwiki-27.pixar.com/space/RFH27/544645977/AOV+Setup+and+Viewing
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan for Houdini 提供了一套强大的 AOV（任意输出变量） 创建与 light transport output（光传输输出）系统，称为 Light Path Expressions（LPE，光路表达式）。为简化使用，许多最常用的 AOV 都以 presets（预设）形式提供在 ROP node（ROP 节点）的 Images（图像）下。

Display Driver（显示驱动）

Display driver（显示驱动） 用于标识渲染图像的 file output（文件输出）和类型。你也可以在下文说明的 "Extra Image Planes" 下指定要添加到文件中的额外 AOV。默认情况下，我们提供一个默认指向 OpenEXR 文件的单一 driver（驱动）。你可以使用 Displays（显示输出） 参数添加更多 display；该参数位于 Images Tab（Images 选项卡）顶部

  • Display（显示）- 要渲染的文件路径和名称，可使用适当的 tokens（标记），例如在 .exr 文件中用 $F4 表示 4 位帧号填充
  • Create Intermediate Directories（创建中间目录）- 允许 Houdini 根据你在上面使用的 tokens（标记）创建正确的目录结构
  • Device（设备）- 渲染目标；在 Houdini 内部，我们默认使用 "it" 作为 display device（显示设备），但也可以指定受支持的文件格式以及 Houdini 自身的 display（显示）
  • Denoise（降噪）- 对此 driver（驱动）的 channels（通道）进行降噪，可以作为 single frame（单帧）或 sequence/multi-frame（序列/多帧）处理
Output（输出）
  • Pixel Filter（像素滤波器） - 用于将 samples（采样）组合成最终 pixel color（像素颜色）的 filter（滤波器）；请注意，Denoise 选项会强制使用 Importance Sample Filtering（重要性采样滤波）
  • Relative Pixel Variance（相对像素方差）- 这是最终帧 Quality（质量）设置上的 multiplier（乘数）。降低该 multiplier 可能会让此 display driver（显示驱动）作为质量的主要 driver（驱动）被采样；如果高于 1，则实际上可能被忽略
  • Filter Width（滤波宽度）- 增大该值可能会在将更多 samples（采样）合并过滤为一个 pixel（像素）之前纳入更多采样，从而使结果图像更柔和。请注意，每个 per-filter（按滤波器）提供的值通常是最低默认值
  • Remap（重映射）- Remap（重映射） 图像，以便在渲染中处理 highlights（高光）和其他 color values（颜色值）
Extra Image Planes（额外图像平面）

这里包含额外常用 outputs（输出）的 presets（预设），并提供如下所述的自定义选项。


Built-in AOV（内置 AOV）

可以选择以下任一项，而不是手动指定 variable（变量）：

Default AOVs（默认 AOV）是 beauty（美图）和 alpha channels（Alpha 通道）。

Standard AOVs（标准 AOV）是用于 compositing operations（合成操作）的 data AOV（数据 AOV），这些操作需要 normals（法线）、depth（深度）等 data driven pass（数据驱动通道）。

Diagnostic（诊断）会提供关于渲染的数据，并将其存储在 channel（通道）中供之后使用

Lighting passes（灯光通道）是合成中进行视觉调整时常用的典型 passes（通道）。我们还提供 per-lobe LPE（按 lobe 的 LPE），用于从 Pxr materials（Pxr 材质）中提取特定 lighting information（照明信息）。请注意，这些结果是 additive（相加）的，并且物理正确。Compositing operations（合成操作）涉及加法和减法，以保留渲染期间产生的结果。乘法和除法可能引入 artifacts（瑕疵），不建议在 pixel level（像素级）的后期操作中使用。

Shadow AOVs（阴影 AOV）要求计算 Beauty render（美图渲染），用于将 shadows（阴影）合成到 filmed plates/footage（实拍底板/素材）上

Mattes（遮罩）可以帮助你在后期选择并隔离对象或材质；现在我们推荐使用 PxrCryptomatte。

Integrator AOVs（积分器 AOV，原文 Inegrator）是内建于所用 integrator（积分器）的 data AOV（数据 AOV）（请验证所选积分器是否支持）。__depth 很有用，并且经过 anti-aliased（抗锯齿）处理。再次提醒，请查看以下页面上的示例： 示例页面 页面。

Custom AOV（自定义 AOV）

你可以在这里指定自己的 AOV 和 Light Path Expressions（LPE，光路表达式）。这对于创建并存储 per-light（按灯光）、per-object（按对象）或你希望捕获的特定 light paths（光线路径）的 LPE 很有用。

要指定 AOV，应向 RenderMan 提供以下信息：

Name（名称） - 指定 AOV 的 name（名称）和 type（类型），例如 varying normal Nn。这些内置选择只需选中即可；我们会向 renderer（渲染器）提供正确名称。

Source（源） - 为 LPE 设置 source string（源字符串，即 expression 表达式）。如果使用 built-in LPE（内置 LPE；见下图）， 可以将此字段留空；否则必须提供适当的 LPE expression（LPE 表达式），例如 type（类型）： color（颜色） 以及表达式： lpe:C<RD>[<L.>O]（LPE 表达式）。


Filter（滤波器） - 对于 color AOV（颜色 AOV），推荐的 filter（滤波器）是 Gaussian；通常应与 Beauty 或 Ci Display Channel 匹配。Data AOV（数据 AOV）应不进行滤波。

Relative Pixel Variance（相对像素方差） - 这是最终帧 Quality（质量）设置上的 multiplier（乘数）。降低该 multiplier 可能会让此 display channel（显示通道，原文 chennel）作为主要质量度量被采样；如果高于 1，则实际上可能被忽略

Filter Width（滤波宽度） - 如上所述，使用 color AOV（颜色 AOV）时，Gaussian 的 filter width（滤波宽度）为 2 2 会匹配默认渲染设置。

Statistics（统计信息） - 只有在为 AOV Denoising（降噪）并手动配置时，才会使用此字段。
