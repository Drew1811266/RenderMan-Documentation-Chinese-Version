AOV Setup and Viewing

Source: https://rmanwiki-27.pixar.com/space/RFH27/544645977/AOV+Setup+and+Viewing
Downloaded: 2026-06-15T03:49:12+00:00

RenderMan for Houdini 提供了一套强大的 AOV（任意输出变量） 创建和光传输输出系统，称为 Light Path Expressions（LPE，光路表达式）。为简化使用，许多最常用的 AOV 都以预设形式提供在 ROP 节点的 Images 选项卡下。

Display Driver（显示驱动）

Display driver（显示驱动） 用于标识渲染图像的文件输出和类型。你也可以在下文说明的 "Extra Image Planes（额外图像平面）" 下指定要加入文件的额外 AOV。默认情况下，系统会提供一个指向 OpenEXR 文件的显示驱动。你可以使用 Displays（显示输出） 参数添加更多显示输出；该参数位于 Images 选项卡顶部

  • Display（显示）- 要渲染的文件路径和名称，可使用适当的替换标记（token），例如在 .exr 文件中用 $F4 表示 4 位帧号填充
  • Create Intermediate Directories（创建中间目录）- 允许 Houdini 根据上面使用的替换标记创建正确的目录结构
  • Device（设备）- 渲染目标；在 Houdini 中，我们默认使用 "it" 作为显示设备，但也可以指定受支持的文件格式以及 Houdini 自身的显示输出
  • Denoise（降噪）- 对此显示驱动的通道进行降噪，可按单帧或序列/多帧处理
Output（输出）
  • Pixel Filter（像素滤波器） - 用于将采样合成为最终像素颜色的滤波器；请注意，Denoise 选项会强制使用 Importance Sample Filtering（重要性采样滤波）
  • Relative Pixel Variance（相对像素方差）- 这是最终帧 Quality（质量）设置的乘数。降低该乘数可能会使此显示驱动作为质量的主要驱动被采样；如果高于 1，则实际上可能被忽略
  • Filter Width（滤波宽度）- 增大该值可能会在将更多采样合并滤波为一个像素之前纳入更多采样，从而使结果图像更柔和。请注意，每个滤波器提供的值通常是最低默认值
  • Remap（重映射）- Remap（重映射） 图像，以便在渲染过程中处理高光和其他颜色值
Extra Image Planes（额外图像平面）

这里包含额外常用输出的预设，并提供如下所述的自定义选项。


Built-in AOV（内置 AOV）

可以选择以下任一项，而不是手动指定变量：

Default AOVs（默认 AOV）包含 beauty（美图）和 Alpha 通道。

Standard AOVs（标准 AOV）是用于合成操作的数据 AOV，这类操作需要法线、深度等数据驱动通道。

Diagnostic（诊断）会提供关于渲染的数据，并将其存储在通道中供之后使用

Lighting passes（灯光通道）是合成中进行视觉调整时常用的典型通道。我们还提供 per-lobe LPE（按波瓣的 LPE），用于从 Pxr 材质中提取特定照明信息。请注意，这些结果是相加且物理正确的。合成操作会使用加法和减法来保留渲染期间产生的结果。乘法和除法可能引入瑕疵，不建议在像素级的后期操作中使用。

Shadow AOVs（阴影 AOV）需要计算 Beauty 渲染，用于将阴影合成到实拍底板/素材上

Mattes（遮罩）可以帮助你在后期选择并隔离对象或材质；现在我们推荐使用 PxrCryptomatte。

Integrator AOVs（积分器 AOV，英文原页拼写为 Inegrator）是内建于所用积分器的数据 AOV（请验证所选积分器是否支持）。__depth 很有用，并且已进行抗锯齿处理。请查看以下页面上的示例： 示例页面。

Custom AOV（自定义 AOV）

你可以在这里指定自己的 AOV 和 Light Path Expressions（LPE，光路表达式）。这对于创建并存储逐灯光、逐对象或特定光路的 LPE 很有用。

要指定 AOV，应向 RenderMan 提供以下信息：

Name（名称） - 指定 AOV 的名称和类型，例如 varying normal Nn。这些内置选项只需选择即可；系统会向渲染器提供正确名称。

Source（源） - 为 LPE 设置源字符串（表达式）。如果使用 内置 LPE（见下图）， 可以将此字段留空；否则必须提供适当的 LPE 表达式，例如类型： color（颜色） 以及表达式： lpe:C<RD>[<L.>O]。


Filter（滤波器） - 对于颜色 AOV，推荐使用 Gaussian 滤波器；通常应与 Beauty 或 Ci Display Channel 匹配。数据 AOV 应不进行滤波。

Relative Pixel Variance（相对像素方差） - 这是最终帧 Quality（质量）设置的乘数。降低该乘数可能会让此显示通道作为主要质量度量被采样；如果高于 1，则实际上可能被忽略

Filter Width（滤波宽度） - 如上所述，使用颜色 AOV 时，Gaussian 的滤波宽度为 2 2 会匹配默认渲染设置。

Statistics（统计信息） - 只有在为 AOV Denoising（降噪）并手动配置时，才会使用此字段。
