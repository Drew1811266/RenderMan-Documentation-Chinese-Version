Preset Browser

Source: https://rmanwiki-27.pixar.com/space/RFH27/544642975/Preset+Browser
Downloaded: 2026-06-15T03:49:12+00:00

  • 可以通过在 Houdini 中添加 tab/panel（标签页/面板）来打开 Preset Browser：Houdini > New Pane Tab Type > RenderMan Preset Browser

  • Presets（预设）包含 light rigs（灯光组）、environment lights（环境灯光）和 materials（材质）等类别。

  • 可以按名称、usage category（用途类别）或 metadata（元数据）搜索 assets（资产）。
  • 可以选择两种 preset image（预设图像）尺寸。

Materials（材质）

  • 保存/导出 material（材质）很简单！

    • 选择 PxrMaterialBuilder Node（节点）（导出会识别已连接的 nodes）。请确保 Material Builder 内部有一个（且只有一个）BxDF，否则无法判断要保存/渲染的内容。
    • 适用于 PxrSurface / PxrLayerSurface 和 Lama materials（Lama 材质）
    • 在 Preset Browser 中选择所需类别
    • 选择 Save icon（保存图标）（预览图像下方的 material ball+ 图标）


      为方便起见，也可以拖放 PxrMaterialBuilder 来保存 asset（资产）。选择该 node（节点），按住鼠标左键，将 node 拖到 Preset Browser 标签页上，然后松开鼠标按钮。

    • 为它指定一个有用的名称，并添加你的 pipeline（流程）所需的 metadata（元数据）
    • RenderMan 会生成 preview swatch render（预览色块渲染），并自动填充到 Preset Browser 中
  • Importing/Using a saved asset（导入/使用已保存资产）
    • 只需右键单击，并对选中的对象选择 "Import and Assign"。
    • 根据对象需要进行任何调整！

Light Rigs（灯光组）

  • 保存/导出 light rig（灯光组）和保存 materials（材质）一样简单！
    • 请确认已选择 LighRigs 类别。
    • 选择要加入 light rig asset（灯光组资产）的 light nodes（灯光节点）。
    • 点击 Save icon（保存图标）（预览图像下方的灯泡图标）

    • 为它指定一个有用的名称，并添加你的 pipeline（流程）所需的 metadata（元数据）

      与 materials（材质）一样，也可以将 light nodes 拖放到 Preset Browser 标签页上来创建 light rigs。

  • Importing/Using a saved light rig（导入/使用已保存灯光组）

    • 只需右键单击并选择 "Import"，即可将 light rig 导入到场景中
    • Preset Browser 会将 light rig 中的所有 lights（灯光）导入到一个 subnet node（子网节点）下。

Environment Maps（环境贴图）

  • 可以使用 Preset Browser 中大型 preview swatch（预览色块）下方的 image icon+（图像加号图标）保存 Environment Map。
    • 在 Preset Browser 中选择 Environment 类别
    • 从 file browser window（文件浏览器窗口）中选择所需的 OpenEXR 或 .HDR 文件
    • 可以在 dialog（对话框）中添加/修改 metadata（元数据）或更改名称。
    • 然后会将图像转换为兼容的 .tex 格式。这可能会因原始图像大小而花费一些时间。
    • 随后会执行 preview render（预览渲染），并自动填充到浏览器类别中。
  • 使用已保存的 map（贴图）很简单。
    • 选择一个 PxrDomeLight object node（对象节点）。
    • 右键单击 asset（资产）并导入。

    • 更换 map（贴图）只需双击另一个 environment map asset（环境贴图资产），同样简单！

      如果导入 map（贴图）时未选择 PxrDomeLight object node，Preset Browser 会自动为你创建一个。


Asset Dependencies（资产依赖项）

将 asset（资产）保存到磁盘时，其所有 dependencies（依赖项）也会保存到磁盘。 

这些包括：

  • Textures（纹理） ：以 tex 格式存储，支持 UDIM。
  • HDRI environment maps（HDRI 环境贴图） ：以 tex 格式存储。我们使用特定的 flags（标志）组合，以尽可能高效地压缩 texture（纹理）。
  • OSL shaders（OSL 着色器） ：以 oso 格式存储，以保护你的 IP（知识产权）；如果想共享源代码，也可以手动将 osl 文件添加到 asset directory（资产目录）中。

这些 dependencies（依赖项）会增加 preset（预设）在磁盘上的大小；如果要通过互联网分发它们，应记住这一点。


Cross-Application Materials（跨应用材质）

RenderManAssets 被设计为可以轻松地从一个应用程序（如 Houdini）传输到另一个应用程序（Katana、Maya、Blender 等），反向传输也一样。

Compatibility Issues（兼容性问题）

不过，如果 material（材质）包含原生 Houdini nodes，仍可能遇到 compatibility issues（兼容性问题）。为减轻这些问题，每个 RenderManAsset 都会维护已使用 nodes 的列表，并用它评估 preset 与当前 host application（宿主应用程序）的兼容性；如果不兼容，就显示警告。

系统会检查：

  • asset（资产）是否包含 host-specific nodes（特定宿主节点）
  • asset 的 renderer version（渲染器版本）是否高于或等于当前版本
  • 如果 asset 包含 host-specific nodes，请确保当前 host version（宿主版本）高于或等于 asset 的 host version。

Best Practice（最佳实践）

如果想确保 cross-application compatibility（跨应用兼容性），请避免使用非 Pixar 的 BxDFs 和 Patterns。

Library Configuration（库配置）

RenderMan 随附一个 standard library（标准库），位置是： $RMANTREE/lib/RenderManAssetLibrary


建议将 standard library（标准库）复制到你选择的位置，并保留默认库不变作为备份。新的副本应可读写。如果 asset browser（资产浏览器）指向不可写位置，它不会允许你创建新 assets。


Environment variable（环境变量）

asset library（资产库）的位置可以通过 RMAN_ASSET_LIBRARY environment variable（环境变量）设置。
