Preset Browser

Source: https://rmanwiki-27.pixar.com/space/RFH27/544642975/Preset+Browser
Downloaded: 2026-06-15T03:49:12+00:00

  • 可以通过在 Houdini 中添加标签页/面板来打开 Preset Browser：Houdini > New Pane Tab Type > RenderMan Preset Browser

  • 预设包含灯光组、环境灯光和材质等类别。

  • 可以按名称、用途类别或元数据搜索资产。
  • 可以选择两种预设图像尺寸。

Materials（材质）

  • 保存/导出材质很简单！

    • 选择 PxrMaterialBuilder 节点（导出会识别已连接的节点）。请确保 Material Builder 内部有一个（且只有一个）BxDF，否则无法判断要保存/渲染的内容。
    • 适用于 PxrSurface / PxrLayerSurface 和 Lama 材质
    • 在 Preset Browser 中选择所需类别
    • 选择 Save 图标（预览图像下方的 material ball+ 图标）


      为方便起见，也可以拖放 PxrMaterialBuilder 来保存资产。选择该节点，按住鼠标左键，将节点拖到 Preset Browser 标签页上，然后松开鼠标按钮。

    • 为它指定一个有用的名称，并添加你的流程所需的元数据
    • RenderMan 会生成预览色块渲染，并自动填充到 Preset Browser 中
  • 导入/使用已保存资产
    • 只需右键单击，并对选中的对象选择 "Import and Assign"。
    • 根据对象需要进行任何调整！

Light Rigs（灯光组）

  • 保存/导出灯光组和保存材质一样简单！
    • 请确认已选择 LighRigs 类别。
    • 选择要加入灯光组资产的灯光节点。
    • 点击 Save 图标（预览图像下方的灯泡图标）

    • 为它指定一个有用的名称，并添加你的流程所需的元数据

      与材质一样，也可以将灯光节点拖放到 Preset Browser 标签页上来创建灯光组。

  • 导入/使用已保存灯光组

    • 只需右键单击并选择 "Import"，即可将灯光组导入到场景中
    • Preset Browser 会将灯光组中的所有灯光导入到一个 subnet 节点下。

Environment Maps（环境贴图）

  • 可以使用 Preset Browser 中大型预览色块下方的 image icon+（图像加号图标）保存 Environment Map。
    • 在 Preset Browser 中选择 Environment 类别
    • 从文件浏览器窗口中选择所需的 OpenEXR 或 .HDR 文件
    • 可以在对话框中添加/修改元数据或更改名称。
    • 然后会将图像转换为兼容的 .tex 格式。这可能会因原始图像大小而花费一些时间。
    • 随后会执行预览渲染，并自动填充到浏览器类别中。
  • 使用已保存的贴图很简单。
    • 选择一个 PxrDomeLight 对象节点。
    • 右键单击资产并导入。

    • 更换贴图只需双击另一个环境贴图资产，同样简单！

      如果导入贴图时未选择 PxrDomeLight 对象节点，Preset Browser 会自动为你创建一个。


Asset Dependencies（资产依赖项）

将资产保存到磁盘时，其所有依赖项也会保存到磁盘。 

这些包括：

  • Textures（纹理） ：以 tex 格式存储，支持 UDIM。
  • HDRI environment maps（HDRI 环境贴图）：以 tex 格式存储。我们使用特定的 flags 组合，以尽可能高效地压缩纹理。
  • OSL shaders（OSL 着色器）：以 oso 格式存储，以保护你的 IP；如果想共享源代码，也可以手动将 osl 文件添加到资产目录中。

这些依赖项会增加预设在磁盘上的大小；如果要通过互联网分发它们，应记住这一点。


Cross-Application Materials（跨应用材质）

RenderManAssets 被设计为可以轻松地从一个应用程序（如 Houdini）传输到另一个应用程序（Katana、Maya、Blender 等），反向传输也一样。

Compatibility Issues（兼容性问题）

不过，如果材质包含原生 Houdini 节点，仍可能遇到兼容性问题。为减轻这些问题，每个 RenderManAsset 都会维护已使用节点的列表，并用它评估预设与当前宿主应用程序的兼容性；如果不兼容，就显示警告。

系统会检查：

  • 资产是否包含特定宿主节点
  • 资产的渲染器版本是否高于或等于当前版本
  • 如果资产包含特定宿主节点，请确保当前宿主版本高于或等于资产的宿主版本。

Best Practice（最佳实践）

如果想确保跨应用兼容性，请避免使用非 Pixar 的 BxDFs 和 Patterns。

Library Configuration（库配置）

RenderMan 随附一个标准库，位置是： $RMANTREE/lib/RenderManAssetLibrary


建议将标准库复制到你选择的位置，并保留默认库不变作为备份。新的副本应可读写。如果资产浏览器指向不可写位置，它不会允许你创建新资产。


Environment variable（环境变量）

资产库的位置可以通过 RMAN_ASSET_LIBRARY 环境变量设置。
