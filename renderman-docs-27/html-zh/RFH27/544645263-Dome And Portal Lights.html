穹顶灯与门户灯

Source: https://rmanwiki-27.pixar.com/space/RFH27/544645263/Dome+And+Portal+Lights
Downloaded: 2026-06-15T03:49:12+00:00

虽然门户灯是矩形形状，但它不能与 PxrRectLight（矩形灯） 互换，也不能由它替代，因为我们是使用  PxrDomeLight（穹顶灯） 从三维环境获得照明。

PxrPortalLight 是 单面。也就是说，门户背面没有照明。如果需要照亮背面，可以再创建一个朝向相反方向的门户。

可以像对其他灯光一样向门户灯添加灯光滤镜。实际上，PxrBarnLightFilter 用于这种灯光类型时效果很好。

更多信息可参阅技术论文：

  • Benedikt Bitterli、Jan Novák, Wojciech Jarosz。 Portal-Masked Environment Map Sampling（门户遮罩环境贴图采样）。Eurographics Symposium on Rendering（EGSR），2015


门户灯（Portal Lights）不能与 PxrEnvDayLight 系统一起工作。请使用带有适当 HDRI 或 HDRI 序列的 PxrDomeLight。

与同一父级穹顶灯和同一门户名称关联的所有门户灯  必须具有相同旋转。如果需要更改门户灯的旋转，则需要使用新的门户名称。不过，不同的平移和缩放可以共享同一个门户名称。



“为什么要使用门户灯？”

由于对于渲染器来说，解析很少或没有直接照明的室内场景很困难，门户灯有助于把渲染器的注意力引导到外部光源。如下方经典 Classroom 场景所示，RenderMan 可以很好地利用 PxrPortalLight。两张图像都以相同的每像素 32 个采样渲染。但可以清楚看到，右侧使用门户灯的结果比左侧未使用时有效得多。

参数

基础


Intensity Multiplier（强度倍增）

相对于父级穹顶灯强度的强度调整。该值会与父级穹顶灯的强度和功率相乘。


Color Tint（颜色染色）

此参数会对穹顶纹理中的颜色进行染色。下方每个门户都分配了不同颜色，形成漂亮的彩虹效果。

此参数会对穹顶纹理中的颜色进行染色。下方每个门户都分配了不同颜色，形成漂亮的彩虹效果。


Map Gamma（贴图 Gamma）

对纹理进行伽马校正。每个通道（RGB）都可以独立调整。如果该贴图伽马保持默认值不变，则会继承父级穹顶灯的贴图伽马。


Map Saturation（贴图饱和度）

调整纹理的饱和度。如果该贴图饱和度保持默认值不变，则会继承父级穹顶灯的贴图饱和度。

Enable Temperature（启用色温）

打开或关闭色温。


Color Temperature（色温）

允许用户以开尔文选择灯光色温。与基本灯光颜色不同，这让用户可以基于标准温度测量轻松选择合理的灯光颜色。此控件会像位于灯光前方的滤镜或色片（带色透明表面）一样工作。如果灯光为白色，将得到所选色温；如果灯光已带颜色，色温会使其变得更冷（高于 6500K）或更暖（低于 6500K）。


在完全饱和的单色光上，该效果不会那么明显。默认值为 6500K，在大多数显示器上应非常接近白色（sRGB 和 Rec 709 使用的 D65 光源）。


Primary Visibility（主可见性）

默认情况下，灯光不会直接对摄影机可见。启用后会使其可见，并可使用发射标记 'O' 写入 LPE。


细化


Specular Amount（镜面量）

发射的镜面光数量。这是非物理控件。  可以使用一个 Specular Amount（镜面量）  为 0.0 的灯光，使其只作为场景对象的漫射光源，并避免添加高光。请注意，场景仍会收集房间中其他对象产生的间接反射。


Diffuse Amount（漫射量）

发射的漫射光数量。这是非物理控件。  可以使用一个 Diffuse Amount（漫射量）  为 0.0 的灯光，使其只作为场景对象的高光源。


阴影

如果灯光的  Trace Light Paths（追踪光路）  在 Advanced 中启用，则所有阴影参数都会被忽略。以下选项在偏离默认状态使用时都是非物理控件。如果  Trace Light Paths（追踪光路）  启用，则会强制使用它们的物理性质。



Enable Shadows（启用阴影）

启用光线追踪阴影。下方显示开启（默认）和关闭。


Shadow Color（阴影颜色）

由发射灯光投射的阴影颜色。下方展示来自不同门户灯的红、绿、蓝。

由发射灯光投射的阴影颜色。下方展示来自不同门户灯的红、绿、蓝。

.0 0.0 0.0

Shadow Max Distance（阴影最大距离）

从被着色点的位置开始计算的最大阴影距离。-1.0 表示未设置，将使用被着色点与灯光上点之间的距离。可以使用此控件人为缩短阴影投射的距离。由于不会计算该距离之外的阴影，这也可能提高渲染速度。请注意椅子和桌腿的阴影在一定距离后被截断，以及教室前方墙面上的阴影。


Shadow Falloff（阴影衰减）

从灯光开始计算、阴影衰减开始的距离。-1.0 会关闭阴影衰减。它会与  Shadow Max Distance（阴影最大距离）  一起使用，为阴影创建人为淡出；这些阴影会被  Shadow Max Distance（阴影最大距离）  参数缩短或截断。下方示例同时使用了两个参数，并增大了衰减。请注意，此参数可能影响复杂照明设置的性能。


Shadow Falloff Gamma（阴影衰减 Gamma）

衰减区域中阴影强度的伽马。此项需要使用  Shadow Max Distance（阴影最大距离）  和  Shadow Falloff（阴影衰减） 。


Trace Subset（追踪子集）

用于追踪阴影相交计算的几何体集合。如果未指定，则所有几何体都会被纳入追踪阴影相交计算。


Don't Trace Subset（不追踪子集）

用于追踪阴影相交计算时忽略的几何体集合。如果未指定，则所有几何体都会用于追踪阴影相交计算。


高级


Trace Light Paths（追踪光路）

启用从此灯光发出的光线追踪和光子追踪。该值会强制使用基于物理的灯光，并作为副作用禁用上面的  Shadows（阴影）  控件。使用 PxrVCM 或 PxrUnified 积分器时，用户可以通过此功能选择性决定哪些灯光发射光子。


Thin Shadow（薄阴影）

为此灯光启用薄阴影并禁用折射焦散。如果  Trace Light Paths（追踪光路）  已启用，则会忽略此参数。这是一个非物理控件，会为透射对象创建“假”彩色阴影，而无需为焦散生成光子。下方从开启（默认）切换到关闭（不透明阴影）。请注意，选择关闭后会丢失彩色阴影和内部彩色反射，但会获得一些渲染速度。


 

Visible in Refraction（折射中可见）

启用后，使灯光能通过玻璃和窗户等透射效果可见。


Manifold Walk（流形行走）

与 PxrUnified Integrator 一起使用，并启用 Enable Manifold Walk（启用 Manifold Walk（流形行走）） 来创建低成本焦散。注意：必须禁用 Thin Shadows


Manifold Walk Exclude Group（流形行走排除组；Manifold Walk（流形行走））

与 PxrUnified Integrator 一起使用，并启用 Enable Manifold Walk（启用 Manifold Walk（流形行走））， 此追踪集会在流形探索期间被忽略，以避免停止或遮挡流形探索光线。


Light Samples（灯光采样数）

指定此光源要采集的灯光采样数覆盖值。如果设置为非零值，它将覆盖积分器执行的采样。如果此灯光存在难以解决的噪声并需要更多采样，可考虑使用此项。


Light Group（灯光组）

指定用于灯光组 LPE 的灯光组名称。这对于生成逐灯光 AOV 以便后续在合成中调整很有用。


Importance Multiplier（重要性倍增）

相比设置显式 Light Samples，用户可以通过更改此值来改变灯光内部分配到的采样数量。RenderMan 会在渲染时为场景中的所有灯光创建一组采样，更改此值会在灯光之间重新平衡采样。请注意，增大该值会导致 更多 采样从此灯光中被选择，同时减少场景中其他灯光的采样。低于默认值会 减少 采样，同时为场景中的其他灯光提供更多采样。


应用程序接口


RenderMan for Maya 或 Katana 等桥接应用提供了便捷界面，用于创建门户灯并将其链接到父级穹顶灯。它还会为旋转与父级穹顶灯不同的门户灯自动分配唯一的门户名称。默认情况下，门户灯会继承父级穹顶灯的所有参数。如果在门户灯中修改了某个参数，则会改用本地覆盖的参数。有两个例外： 强度和颜色会与父级穹顶灯的强度和颜色相乘。

桥接产品会把所有修改过的参数作为 PxrPortalLight 写入 RIB。请注意，PxrDomeLight 不会被写出。
