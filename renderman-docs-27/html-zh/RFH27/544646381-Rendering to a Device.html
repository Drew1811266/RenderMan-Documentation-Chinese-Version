渲染到设备

你可以通过 RenderMan 的 Image Tool（图像工具） 或 Render View（渲染视图）以交互方式预览你的渲染。

有关渲染设置的更多信息，请参见 ROP Node（ROP 节点） 文档

使用 Image Tool

将 Display Device（显示设备） 设置为 "it"（默认值）

使用 Render View

转到 Render View（渲染视图）。选择你的输出，例如 /out/ris1。然后点击 Render（渲染） 按钮。  

你也可以渲染到磁盘/RIB

可渲染 RIB 导出

在 RenderMan RIS ROP 的 Disk File 设置，位于 Driver（驱动） 选项卡中。 

勾选该参数旁的复选框，当你选择 Render to Disk（渲染到磁盘）时，它会输出 RIB 文件。

若要更快导出 RIB，请按 ROP 节点上或 RenderMan Menu 中的 Dump RIB 按钮。这些工具会自动切换 RIB 渲染复选框。

  • Write RIB（写出 RIB）- 渲染时导出 RIB 文件

  • Binary（二进制）- 默认情况下我们会写出紧凑文件；但如果要手动编辑或调试，请取消勾选此项，以输出更大的 ASCII 文件

  • As Archive（作为归档）- 此输出会包含场景实体，但不包含渲染设置或显示输出等全局数据。该文件本身不可渲染，但稍后可以在场景中引用

  • Disk File（磁盘文件）- RIB 文件的输出位置，默认使用你的场景文件（.hip）所在位置

  • VDB Path（VDB 路径）- VDB 数据的导出位置

  • Create Intermediate Directories（创建中间目录）- 自动创建适当的文件结构

  • Initialize Simulation OPs（初始化模拟 OP）- 渲染时，所有 POP 和 DOP 都会重置为初始状态

  • 默认显示 SOHO 视口菜单

RIB 归档创建

要创建用于稍后渲染的归档，可按下面步骤操作。归档适合在渲染时延迟读取场景实体，避免把所有内容都加载进桥接产品，从而在这些内容直到渲染时才需要显示或处理的情况下消耗内存和资源。归档不能单独渲染，因为它们不包含适当的渲染选项，它们用于复杂度较高的场景装配。

创建 RenderMan ROP 节点。

这里我们在 Objects Tab 中，将候选对象设置为场景内的 pyro_import 对象。

在 Driver Tab（用于 RIB 导出）中，我们选择 "Write RIB" 和 "As Archive"，并提供适当的路径。

请注意，我们使用 $F4 标记，这样每一帧都会得到一个 RIB 文件，其中 4 表示 4 位数字填充。
