有没有好奇过，当你按下 render 按钮时，RenderMan hood 之下到底发生了什么？

嗯……这里是对这个过程的简化说明。

  1. Read：将 scene description 读入 renderer（如果已经读入，则更新它）

  2. 对 geometry 进行 tessellation（并可选 displacement）；构建 ray tracing data structures

  3. 生成并发射 camera rays

  4. 对 ray hit points 进行 shading：评估 shading networks、materials，并计算 direct illumination

  5. 生成 indirect illumination 所需的 rays；trace 这些 rays

  6. 回到第 4 步。重复。


第 1 步：场景转换（Step 1 Scene translation）
Step 2 Path tracing（路径追踪）
第 3 步：显示数据（Step 3. Display the data）
第 4 步：降噪处理（Step 4. Denoising）

场景转换阶段（Scene Translation）

对 geometry 进行 tessellation / 生成 micropolygons

把 data 放入 RAM（system 或/和 vram）

TIP：将 geometry 保持为 polygons 更便宜，但 RenderMan 更偏好 subdivision

事实：你从不需要选择 subdiv levels；RenderMan 会为你完成全部处理，并始终保持平滑

Path tracing（路径追踪）

从 camera 发射光线 rays

命中场景中的 surfaces，

发现 shaders 并进行 render

TIP：Procedural shading networks 总是比 textures 更昂贵。如果你确实需要 procedural networks，可以将它们 bake 到 disc。

显示数据阶段（Display the data）

显示 render（IT 或 viewport），或保存到 disc（2D image 或 Deep data）

最终降噪阶段（Denoising）

清理 path-traced variance，并生成干净结果
