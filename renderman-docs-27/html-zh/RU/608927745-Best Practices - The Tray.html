通过工具节点传递贴图，以辅助外观开发。

绘制纹理贴图时，可能会发生绘制过程反馈有限的情况。这意味着你也许已经有一张纹理可以尝试，但还不能确定它是否正确实现了想要的效果。它可能太暗或太亮；你的粗糙度贴图可能过于粗糙，也可能不够粗糙。无穷无尽的变化可能迫使你在选定的绘制程序中来回调整，而这会变得令人沮丧。

在 Maya 中使用 IPR 模式，你可以让纹理通过一些工具节点，并直接在渲染中进行数值调整。 

这些 Pattern 网络 可以在渲染期间求值，也就是每次处理纹理时求值；或者你也可以 烘焙 结果，从而提升复杂资产的整体效率。（在获得客户最终批准之前，你可能希望避免烘焙，因为否则可能会不断往返渲染农场。）有些 Pattern 的求值开销高于其他 Pattern，并且可能开始影响场景渲染时间。烘焙可以改善这一点，因为它只会作为纹理求值一次并缓存。关于效率主题，你可以阅读 Improving Performance这篇文章了解更多。

虽然 Ptex 可以节省为资产准备纹理的时间，但其性能不如在定义了 UV 的对象上使用标准纹理。在规划项目时，应权衡这种成本收益。如果有许多对象需要制作纹理，Ptex 总体上可能会为艺术家节省时间。否则，使用纹理贴图和典型的 UDIM（或 “tiled”）UV 布局，可能会获得更好的渲染效率和更简单的流程。

下面，Specular Roughness 纹理使用一个 PxrColorCorrect 节点进行了修改，并在 Maya 的 IPR 中调整。Pattern 的 Color Correct 部分使用了不同 Gamma 值。由于该纹理是灰阶（标量）贴图，因此 RGB 通道都有相同的值。这里只操作 Red Channel，然后连接到 Primary Specular Roughness 参数。或者，你也可以把 Color Correct Pattern 连接到类似 PxrToFloat 这样的转换 Pattern，但这会给节点图增加一次额外求值，而在这个示例中并不必要。
