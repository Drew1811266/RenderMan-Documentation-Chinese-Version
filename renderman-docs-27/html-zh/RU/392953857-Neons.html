Neons（霓虹灯）

Downloaded: 2026-06-15T03:49:12+00:00

谁会不喜欢有趣的霓虹招牌呢？！

在本课中，我们将看看如何在 Solaris 中使用 Houdini 21 新增的 Geometry Lights 来渲染霓虹招牌。

在其他 DCC 中使用 RenderMan XPU 渲染网格灯很直接。通常只需选择想要转换成网格灯的几何体即可。

在 Solaris 中，将几何体转换为网格灯需要一点设置；但完成后，我们就能相当细致地控制网格灯的外观以及发光方式。

概括来说，在 Solaris 中，我们需要为想要转换成网格灯的几何体指定一个带有 Glow 参数的材质。

USD 资产设置

因此，在 Solaris 中，我们有一个 Neon R 霓虹招牌，它是使用 component builder 创建成 USD 资产的。

我们有一个带纹理的材质，并将其应用到招牌、霓虹支架、霓虹孔洞和立杆上。

然后还有四个其他着色器，每个都是启用了 Glow 的 PxrSurface，并且各自有轻微颜色变化：其中三个是发光的橙色材质，一个是发光的白色材质。

所有材质指定完成后，USD 资产被写出到磁盘。

Solaris Geometry Lights（几何灯）

转到主要的 Solaris 设置，第一步是读入 Neon Sign 的 USD 资产。

接下来就是网格灯的关键步骤，这要归功于 Solaris 的 Geometry Light node。

如下图所示，我们有四个 geometrylight 节点，每个都会从我们定义的 prim 创建一个网格灯。

把它拆解到一根霓虹灯管上，让我们更仔细地看看设置。

在 Primitives 参数中，我们会告诉 Solaris/RenderMan 希望将哪些 prim 转换为网格灯。我们还需要把 API Schema 设置为 MeshLightAPI

Material Sync Mode 有几种不同模式：

Material Glow Tints Light

这允许我们使用指定 PxrSurface 的 Glow 参数，作为初始网格灯强度和颜色。

Independant

这让我们可以独立于 Glow 调整强度、颜色和其他值；这里演示的是让每根霓虹灯管发出与原始发光颜色不同的颜色。

No Material Response.

实际上，这会让几何体对相机不可见，但我们仍然可以从 prim 发光并控制它的灯光值。我们仍然保留那些夸张的颜色，但看不到网格灯 prim。
