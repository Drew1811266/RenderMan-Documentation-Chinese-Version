RenderMan 提供多个节点，可用于随机化纹理着色、文件选择以及随机方向/放置。

我们要看的两个节点是 PxrVary 节点和 PxrRandomTextureManifold 节点。

在这个示例中，我们有七个喷漆罐；目前它们看起来相当乏味。

本课的目标是为每个罐子创建随机颜色的盖子。

我们还设计了七种不同标签，希望把它们随机应用到罐身标签上。

喷漆罐彩色盖

我们先处理盖子，使用一个极其实用的随机化节点：PxrVary。

该节点接收基础颜色（或彩色输入），并允许色相、饱和度、gamma 以及许多其他参数按实例获得随机值。

先把 PxrVary 连接到 PxrSurface 的 Diffuse color，并将 Input Color 设置为明亮的橙色。

这会把所有盖子都变成同一种橙色，而这不是我们想要的。

我们希望每个盖子都有随机颜色。

调整以下值后，会得到下面的图像。

0.1

这样看起来“还可以”，但我们其实不想让 RenderMan 决定选择哪些随机颜色。我们想自己选择它们。

我们希望对 PxrVary 随机选择颜色时使用的调色板进行艺术指导。

实现方式是使用 PxrRamp 节点，在其中设置 ramp 颜色。我们还需要把 Ramp Type 设置为 Random Object Color。

注意！这项技术只有在你把 Ramp Type 设置为 Random Object Color 时才会生效。

理解这是一个随机节点很有帮助，因此有时很难艺术指导哪个盖子获得哪种随机颜色。有时，如你所见，PxrVary 会随机给两个盖子分配相同颜色。

另外，ramp 中颜色的宽度会决定随机分配和概率，因此要调出你想要的准确随机外观可能有点棘手。

这里的关键词就是随机！

如果你有 100 个罐子，而且并不在意哪个盖子是哪种颜色，那么 PxrVary 就是解决方案；但如果你需要特定罐子拥有特定颜色的盖子，那么为每个罐子定义 PrimVar 值（然后在 PxrVary 的 PrimVar attribute 中指定 PrimVar name）才是办法，不过那是另一课的内容。

我们想要随机颜色的盖子……也确实得到了随机颜色的盖子。

接下来处理标签。

喷漆罐标签

这里有几张纹理，我们想再次随机地分配给每个标签，不太在意哪张配哪个标签。我们要的就是随机！

要实现这一点，我们需要两个节点。

第一个是 PxrMultiTexture 节点，我们会把所有贴图加载到每个 filename 参数中。

然后，需要一个 PxrRandomTextureManifold 节点，通过把 PxrMultiTexture 文件分配到喷漆罐上，来控制它们的随机性。

像上面的节点网络那样连接它们，你就会得到类似这样的结果！

注意：你需要把 Number of Textures 参数设置为要使用的文件列表中的纹理数量。

PxrRandomTextureManifold 还有各种可调参数，可以添加更多随机化，例如随机纹理方向、翻转、偏移等等，所以纹理随机化的数量……几乎没有尽头！

更多用途

这两个节点并不只是用于随机选择颜色或漫反射纹理。

例如，PxrRandomTextureManifold 也可以用于随机分配和变换粗糙度/凹凸贴图、叠加随机纹理、变换随机边缘破碎遮罩等。

PxrVary 可以为粗糙度贴图添加随机颜色/饱和度变化，让每个实例拥有不同的粗糙度值。同样的技术也可用于设置随机凹凸和镜面反射值，以及控制清漆涂层上的随机厚度。

这两个节点的可能性没有尽头。

如果你想在所有实例上实现完整的着色随机化，这两个节点是不错的起点！
