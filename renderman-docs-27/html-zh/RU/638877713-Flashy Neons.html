接续上一课，现在让我们在 Solaris 中用一些很酷的 VEX code 让 neon sign 活起来！

在本课中，我们将修改来自 Glow Neons 课程的四个 Glow shaders，使其能够用 Primvar 控制 PxrSurface Glow Intensity。完成后，我们就能使用一些巧妙的 VEX code 来控制 neons 的 animation。

USD Asset Setup（USD 资产设置）

我们想要的是动态控制 neon tubes 的“chasing” animation。

首先需要编辑 Glow shaders。在 Solaris 中，可以通过创建一个 editmaterialproperties node 来完成。我们可以把 glow shader 加到这个 node 中，并进一步编辑它。

在第一个 Glow Shader 内，创建一个 PrimVar node，将 Variable type 保持为 Float，但一定要把 Float value 设置为 0.1。这会给我们一个初始 intensity value。

在这个示例中，我们的 PrimVar 名为 Outer_INT

然后把 Primvar 的 resultR 连接到 PxrSurface 的 glowgain。

我们需要对其他每个 Glow shaders 重复这些步骤，并确保每个 PrimVar 都有唯一名称。

Inner_INT

完成后，需要重新 publish USD asset，然后把它 reload 到我们的 main network 中。

Time to Animate!（开始制作动画！）

手动 animate 这些 neons 会花不少时间，但既然我们在 Solaris 中有 VEX 的力量，为什么还要手动做呢？

Start：先创建一个 AttributeWrangle node，并把下面的 code 粘贴到 VEXpression box 中。

int off = chi("off");
int on = chi("on");
int transition = chi("transition");
int A = off;
int B = off + transition;
int C = off + transition + on;
int D = off + 2 * transition + on;

float val = 0;
int frame = int(@Frame - $FSTART + chi("frameOffset") * @elemnum) % (D + 1);

if(frame < A)
    val = 0;
else if (frame >= A && frame < B)
    val = 1 - (B - frame) / float(B - A);
else if(frame >= B && frame < C)
    val = 1;
else if (frame >= C && frame < D)
    val = (D - frame) / float(D - C);
val *= ch("intensity");

@inputs:defaultFloat = val;
//@inputs:intensity = val;

编译完成后，你会看到下面的 interface！

最后一件事是指定我们希望在哪些 Primitives 上运行 code。

使用一个方便的 wildcard 作为 

/NeonR/materials/**/**_INT

会遍历所有 materials，并找到名称中带有 _INT 的 nodes。这样我们就能轻松快速地拾取 Glow Shaders 中的所有 primvars。

下面是 parameters 的快速说明

OFF

每个 neon 保持 OFF 的 frames 数

ON

neons 保持 ON 的 frames 数

Transition

向上/向下渐变（ramp up / ramp down）所需的 frames 数

FrameOffset

更改这个 value 可调整所有 neons 之间的 offset time

Intensity

neons 应达到的最大 intensity

就是这样……可以自由调整 parameters 来创建你自己的 chasing times，也可以把这段 code 用到所有 neons 上。

只要确保你有一个 PrimVar 连接到 你的 surfaces 里的 glowgain 参数， 就可以开始了。
