接续上一课，现在让我们在 Solaris 中用一些精彩的 VEX 代码让霓虹灯招牌动起来！

在本课中，我们将修改来自 霓虹灯 课程的四个 Glow 着色器，使其能够通过 Primvar 控制 PxrSurface 的 Glow Intensity。完成后，我们就能用一些巧妙的 VEX 代码来控制霓虹灯动画。

USD 资产设置

我们的目标是动态控制霓虹灯管的“追逐式”动画。

首先需要编辑 Glow 着色器。在 Solaris 中，可以通过创建一个 editmaterialproperties 节点来完成。我们可以把 Glow 着色器添加到这个节点中，并进一步编辑它。

在第一个 Glow 着色器内，创建一个 PrimVar 节点，将 Variable type 保持为 Float，但务必把 Float value 设置为 0.1。这会给我们一个初始强度值。

在这个示例中，我们的 PrimVar 名为 Outer_INT

然后把 Primvar 的 resultR 连接到 PxrSurface 的 glowgain。

我们需要对其他每个 Glow 着色器重复这些步骤，并确保每个 PrimVar 都有唯一名称。

开始制作动画！

手动为这些霓虹灯制作动画会花不少时间，但既然我们在 Solaris 中可以使用 VEX，为什么还要手动做呢？

先创建一个 AttributeWrangle 节点，并把下面的代码粘贴到 VEXpression 框中。

int off = chi("off");
int on = chi("on");
int transition = chi("transition");
int A = off;
int B = off + transition;
int C = off + transition + on;
int D = off + 2 * transition + on;

float val = 0;
int frame = int(@Frame - $FSTART + chi("frameOffset") * @elemnum) % (D + 1);

if(frame < A)
    val = 0;
else if (frame >= A && frame < B)
    val = 1 - (B - frame) / float(B - A);
else if(frame >= B && frame < C)
    val = 1;
else if (frame >= C && frame < D)
    val = (D - frame) / float(D - C);
val *= ch("intensity");

@inputs:defaultFloat = val;
//@inputs:intensity = val;

编译完成后，你会看到下面的界面！

最后一件事是指定我们希望在哪些图元（Primitives）上运行代码。

使用一个方便的通配符 

/NeonR/materials/**/**_INT

会遍历所有材质，并找到名称中带有 _INT 的节点。这样我们就能轻松快速地拾取 Glow 着色器中的所有 Primvar。

下面是参数的快速说明

OFF

每个霓虹灯保持 OFF 的帧数

ON

霓虹灯保持 ON 的帧数

Transition

渐入和渐出所需的帧数

FrameOffset

更改这个值可调整所有霓虹灯之间的时间偏移

Intensity

霓虹灯应达到的最大强度

就是这样……可以自由调整参数来创建你自己的追逐时间，也可以把这段代码用到所有霓虹灯上。

只要确保有一个 PrimVar 连接到 表面上的 glowgain 参数， 就可以开始了。
