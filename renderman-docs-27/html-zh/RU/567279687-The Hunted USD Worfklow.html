先从一个解释开始，以防你还不太确定 USD 是什么...

“Universal Scene Description (USD) 是 Pixar 的 open source file format，用于描述 CG scenes，以便在多个 applications 和 artists 之间实现 concurrent workflows。”

USD 的主要好处之一，是通过 USD Layering 实现 collaboration，它允许不同 departments 的多个 artists 同时处理同一个 asset。

上图展示了 TS4 中 Antiques Mall 的一个 scene。它说明了 USD layering 的力量：animator 可以处理 character animation，同时 layout artist 在 shelves 上布置 dressing，modeller 也同时完成 chair 的 modelling。lighter 则在为 desk lamp asset 添加 lights。

所有这些都能同时发生，一个 department 不需要依赖或等待另一个 department 完成，才能完成自己的 task。

带着这个认识，你就可以开始理解，为什么无论对大型 studios 还是 individual artists，USD workflow 都能增强 creativity 和 collaboration。

由于我很早就决定要在 Solaris 中制作这件作品，我已经知道，针对 assets、camera、set dressing 和 lighting 使用 USD workflow，会带来 USD 所提供的许多好处。

使用 Solaris 中的 componentbuilder node in Solaris（Solaris 中的节点），我能够为 scene 中的每个 main props 创建单独的 USD assets。

下面是我创建的这些 assets 列表

Road（道路）

Hills（山丘）

UFO hunting van（UFO 猎手面包车）

Train tracks（铁路轨道及轨枕场景元素）

Railroad sign（铁路标志牌道具元素）

Grass（草）

Trees（树木）

Stones & Rocks（石头与岩石）

Mail boxes（路边邮箱道具元素）

PowerBox

以 railroad powerbox（铁路道旁电源箱这个 USD 资产）为例，

我首先确定现实世界中这类 unit 的 physical size。得到这些 dimensions 后，我就能创建一个匹配这些 dimensions 的 box，并发布一个名为 USD asset 的资产： PowerBox.usd

然后，我可以使用这个 power box 的 initial version 开始 layout，并放心地知道自己之后可以回到这个 prop，完成 modelling 和 texturing，再重新发布 PowerBox.usd file，并在 main layout USD layer 中重新加载它，它就会立即切换版本。

在处理 assets 时拥有这种能力，可以让你快速进入 initial layout stage，而不必担心之后的 refinements（除非你对 asset 本身做了相当剧烈的改动！！）

当然，这个解释只是对 USD 的简化说明；加入其他 USD features，例如 modelling 和 shading variants，会为使用 USD 工作提供更多额外好处。
