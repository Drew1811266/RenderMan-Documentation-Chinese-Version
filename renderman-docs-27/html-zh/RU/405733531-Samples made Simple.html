本页的目标是为你在 RenderMan XPU 中设置 Min 和 Max samples 提供一个直接明了的概览和起点；但在开始之前，先把一些术语说明清楚...

Sample（采样）

一条 ray 会从 camera 发出，并在 scene 中反弹。它在 scene 中找到的能量会被存储到一个 pixel 中。

低 samples 会带来大量 noise，但图像生成很快。

高 samples 会得到更干净的图像，但 render 会更慢。

RenderMan 一定有办法在过低和过高之间协商出良好平衡，对吧？没错！这种质量协商称为 Adaptive Sampling（自适应采样），并通过 Pixel Variance 来衡量...

Pixel Variance（像素方差）

这是对 sampling 导致的 noise 的度量。

pixel variance 越高，图像看起来越 noisy。

pixel variance 越低，图像看起来越干净。

我们会向 renderer 指定一个 variance threshold（方差阈值） 用来告诉它，我们愿意在完成图像中接受多少 noise。

pixel variance threshold 越低，最终 render 越干净，但完成渲染所需时间也越长。一般来说，每当你把 variance threshold 减半，render time 就会增加到四倍。

How to choose a threshold?（如何选择 threshold？）

所以...这里的大问题是... “我的 pixel variance value 应该设成多少？”

答案是，没有任何单一数值可以在你的 scene 中每次都适用；但把 min 和 max 设为较宽范围，并依靠 pixel variance 来指导选择，是一个很好的起点。

理想情况下，你应该像上图那样做一些测试，看看什么设置适合你。

好消息是，由于 RenderMan Denoiser 非常擅长在保留 hair 和 surface imperfections 中细微细节的同时去除 noise，它已经让 Pixar 能够把大多数 production shots 的 pixel variance 提高到 0.18，这反过来意味着我们不仅能以前所未有的速度 render shots，也能让 artists 更快地 iterate 和 refine shots。

它也让 Pixar 能够渲染过去不可能完成的 shots。

既然知道 RenderMan denoiser 能为我们提供支持，我们就知道把 pixel variance 降到 0.01 太低了，因此一个不错的起点是 0.1

再做一次 render，运行 denoiser，然后看看效果。

然后再启动几次 render，每次提高 pixel variance，直到达到 rendering sweet spot：快速 render times，以及 denoiser 能够清理的可接受 noise 量。

当你把繁重工作交给 denoiser 后，会惊讶于 pixel variance 可以推到多高。

这个示例演示了如何在单张 image 上设置 threshold。

如果你正在制作 animation，别忘了渲染一段 frame sequence，以确保 motion blur 和 depth of field 足够干净，并避免 low samples 带来的典型问题，例如星星、灰尘或 particles 这类微小对象丢失。

Visualizing your samples（可视化你的 samples）。

你可能还会问自己另一个问题... “我怎么知道 render 中所有 samples 都用在了哪里？”

RenderMan 有一个非常有用的 AOV，名为 Sample Count，可以把它添加到 render 中，并在 IT 中检查其 values，帮助你判断 renderer 正在哪里计算 samples。
