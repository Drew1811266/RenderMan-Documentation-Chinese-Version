为任务选择正确工具

一般来说， 细分曲面 有许多优势；在 Pixar，绝大多数建模都使用这种方法创建。但有时候你确实需要 多边形网格。本页会帮助你判断哪种 建模技术 最适合你想要的外观或效果。

细分曲面

  • 优点（Pros）

    • 在任何 分辨率 下都能保持完全平滑。

    • 可以通过 折痕和角点 获得 尖锐或半尖锐特征。

    • 它们在内存中可以非常轻量。

    • 位移裂缝更少，拓扑更封闭（水密）

  • 缺点（Cons）

    • 需要不同的建模技术/技能

    • 需要标记 尖锐边和顶点

    • 存在一些 拓扑约束（四边形或三角形，不能有非流形几何体）

多边形网格

  • 优点（Pros）

    • 非常适合 3D 扫描网格这类复杂几何体

    • 最适合会被 破碎/模拟的几何体。

    • 适合只有尖锐边且不需要位移的模型

    • 如果没有位移，就不会生成微多边形。

  • 缺点（Cons）

    • 复杂模型会使用大量内存，即使它们在图像中很小。

    • 在低分辨率的平滑对象上，阴影终止线可能可见

    • 可能出现位移裂缝

细分建模提示

  • 始终建模尽可能简单的 control cage。

  • 始终使用平滑显示查看你的模型，或使用 PxrVisualizer来渲染它。

    • 你甚至可以在实时 PxrVisualizer 渲染中进行建模！

  • 使用折痕创建尖锐或半尖锐边，而不必添加额外边环。

SubDiv Types Demo（细分类型演示）

细分曲面 有三种类型。

  1. Catmull-Clark 是 细分方案 中最灵活、最流行的一种。

  2. Loop 是基于三角形的细分曲面。没有折痕或角点。适用于布料模拟

  3. Bilinear 类似于所有边都带有无限硬折痕 的 Catmull-Clark

细分镶嵌（Tesselation）控制

如果你来自另一个必须设置 细分级别的渲染器，请把这件事忘掉： 曲面细分是自动处理的 在 RenderMan 中（我们称它为 细分).

主要控制项是 Micropolygon Length。

  • 默认情况下，它以像素表示；Micropolygon Length 为 1（默认值）意味着每个微多边形大约 1 像素宽。

  • 如果不希望细分动态变化，可以在 对象空间或世界空间 中设置它，或定义 细分 camera。

除非你看到一些 伪影，否则我们建议禁用“面向栅格的细分”。

SubDiv 与多边形细分演示

我们比较了一个带折痕、带位移的 Catmull-Clark 细分曲面 与等效的 多边形网格。

提高 Micropolygon Length 等同于让 对象 在 图像 中的尺寸变小。

  • 当对象很大时，它会被精细细分成数百万个微多边形，从而显示细致的位移细节。

  • 当对象变得小于一个像素时，只剩下基础网格。

Micropolygon Length 有点像“自动细节级别”。当对象在屏幕上小到不需要完整细节表示时，它会节省内存。

注意 细分曲面会降到更少的面，从而节省内存并加快光线追踪。

这就是我们所说的 Data Amplification：一种紧凑的 geometric representation，可在需要时增强以创建更多 细节。

模型细节

通常，更好的做法是依靠位移和凹凸映射来添加细节。

  • 位移允许你添加较大的几何特征，这些特征会影响对象轮廓和阴影效果。

    • 位移会影响首像素时间（微多边形生成），但不会拖慢着色。

  • 凹凸或法线贴图会添加小尺度细节。

    • 凹凸映射会在着色期间增加少量成本（每次光线命中都会重新计算）。

    • 注意，如果 入射光线过于“模糊”，我们的 凹凸映射节点不会计算凹凸，因此没有凹凸也不会被注意到。

如果你在 ZBrush 中建模了一个高度细节化的对象，应考虑：

  • 重新拓扑以创建轻量模型

  • 从 高分辨率网格 中提取 位移贴图。

我们的 位移节点 有一个“Model 位移” 输入，可连接提取出的位移贴图。 

 

内存使用与性能

多边形网格

  • 最短的 time to first 像素（TTFP），前提是没有位移。

  • 当使用位移时，网格会被切分，并生成微多边形。

  • 未使用位移的复杂网格会使用相同数量的 RAM，无论它们是全屏还是只有 2 像素高。

细分曲面

  • TTFP 稍长，但细分是多线程且高效的。

  • 在相同顶点数量下，比网格多使用一点内存。

  • 稀疏建模的细分曲面在 2 像素高时只包含基础网格面。

    • 这种自适应能力会降低内存使用量，并加快光线追踪。
