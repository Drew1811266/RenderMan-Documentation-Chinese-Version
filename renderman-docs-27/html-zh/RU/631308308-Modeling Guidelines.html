为任务选择正确工具

一般来说， subdivision surfaces 有许多优势；在 Pixar，绝大多数 modeling 都使用这种方法创建。但有时候你确实需要 polygonal mesh。本页会帮助你判断哪种 modelling technique 最适合你想要的 look 或 effect。

细分曲面类型（Subdivision surfaces）

  • 优点（Pros）

    • 在任何 resolution 下都能保持完全平滑。

    • 可以通过 creases 和 corners 获得 sharp 或 semi-sharp features。

    • 它们在 memory 中可以非常轻量。

    • displacement cracks 更少，拓扑更封闭（watertight）

  • 缺点（Cons）

    • 需要不同的 modeling techniques/skills

    • 需要标记 sharp edges 和 vertices

    • 存在一些 topological constraints（quads 或 tris，不能有 non-manifold geometry）

Polygon meshes（多边形网格）

  • 优点（Pros）

    • 非常适合 3D scanned mesh 这类 heavy geometry

    • 最适合会被 fractured / simmed 的 geometry。

    • 适合只有 sharp edges 且不需要 displacement 的 models

    • 如果没有 displaced，就不会生成 micropolygons。

  • 缺点（Cons）

    • heavy models 会使用大量 memory，即使它们在 image 中很小。

    • 在低 resolution 的 smoothed objects 上，shadow terminator 可能可见

    • 可能出现 displacement cracks 裂缝

Subdivision Modeling Tips（细分建模提示）

  • 始终建模尽可能简单的 control cage。

  • 始终使用 smoothing 查看你的 model，或使用 PxrVisualizer来 render 它。

    • 你甚至可以在实时 PxrVisualizer render 中进行 model！

  • 使用 creases 创建 sharp 或 semi-sharp edges，而不必添加额外 edge loops。

SubDiv Types Demo（细分类型演示）

subdivision surfaces 有三种类型。

  1. Catmull-Clark 是 subdivision schemes 中最灵活、最流行的一种。

  2. Loop 是基于 triangle 的 subdivision surface。没有 creases 或 corners。适用于 cloth simulations

  3. Bilinear 类似于所有 edges 都带有无限 hard creases 的 Catmull-Clark

细分镶嵌（Tesselation）控制

如果你来自另一个必须设置 subdivision levels 的 renderer，请把这件事忘掉： surface tesselation 是自动处理的 在 RenderMan 中（我们称它为 dicing).

主要控制项是 micropolygon length。

  • 默认情况下，它以 pixels 表示；micropolygon length 为 1（默认值）意味着每个 micropolygon 大约 1 pixel 宽。

  • 如果不希望 tesselation 动态变化，可以在 object 或 world space 中设置它，或定义 dicing camera。

除非你看到一些 artefacts，否则我们建议禁用“raster-oriented dicing”。

SubDiv vs Poligon Dicing Demo（SubDiv 与 Poligon Dicing 演示）

我们比较了一个带 crease、带 displacement 的 Catmull-Clark subdivision surface 与等效的 polygon mesh。

提高 micropolygon length 等同于让 object 在 image 中的尺寸变小。

  • 当 object 很大时，它会被精细 tessellated 成数百万个 micropolygons，从而显示细致的 displacement detail。

  • 当 object 变得小于一个 pixel 时，只剩下 base mesh。

Micropolygon Length 有点像“automatic level of detail”。当 object 在屏幕上小到不需要完整细节表示时，它会节省 memory。

注意 subdiv 会降到更少的 faces，从而节省 memory 并加快 raytracing。

这就是我们所说的 Data Amplification：一种紧凑的 geometric representation，可在需要时增强以创建更多 detail。

Model detailing（模型细节）

通常，更好的做法是依靠 displacement 和 bump mapping 来添加 details。

  • Displacement 允许你添加较大的 geometric features，这些特征会影响 object 的 outline 和 shadowing。

    • Displacement 会影响 time to first pixel（micropolygon generation），但不会拖慢 shading。

  • Bump 或 normal maps 会添加小尺度 details。

    • Bump mapping 会在 shading 期间增加少量 cost（每次 ray hit 都重新计算）。

    • 注意，如果 incoming ray 过于“blurry”，我们的 bump-mapping nodes 不会计算 bump，因此没有 bump 也不会被注意到。

如果你在 ZBrush 中建模了一个高度 detailed object，应考虑：

  • retopologizing 以创建轻量 model

  • 从 high-resolution mesh 中提取 displacement map 贴图。

我们的 displacement nodes 有一个“model displacement” input，可连接提取出的 displacement map。 

 

Memory usage and performance（内存使用与性能）

Polygon meshes（多边形网格）

  • 最短的 time to first pixel（TTFP），前提是没有 displacement。

  • 当 displaced 时，mesh 会被 diced，并生成 micropolygons。

  • 未 displacement 的 heavy meshes 会使用相同数量的 RAM，无论它们是 full-screen 还是只有 2 pixels 高。

细分曲面类型（Subdivision surfaces）

  • TTFP 稍长，但 dicing 是 multi-threaded 且高效的。

  • 在相同 vertices 数量下，比 meshes 多使用一点 memory。

  • 稀疏 modeled 的 subdiv 在 2 pixels 高时只包含 base mesh faces。

    • 这种 adaptivity 会降低 memory usage，并加快 raytracing。
