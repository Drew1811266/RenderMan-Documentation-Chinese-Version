匹配拍摄素材并进行渲染

在 Maya 中开始渲染流程之前，我首先需要把对象的旋转匹配到真实转台拍摄素材。使用 Nuke 中的运动跟踪，我应用了带通滤镜来隔离平台运动，并手动跟踪关键点以识别旋转轴，不过数据仅限于二维空间。

把数据导入 Maya 后，我手动调整比例和方向，使实体转台与数字场景对齐。通过把二维跟踪点投射到三维平面上，我能够准确同步 CG 中对象的旋转与拍摄素材。

将 CG 对象与拍摄素材对齐后，我为最终渲染准备场景。为了捕获角色在地面上的阴影而不影响 alpha，我使用了遮挡平面。这让渲染结果更容易整合到背景底板中。

渲染之前，我运行了多帧测试帧，通过与参考素材比较清晰度，确保景深（DOF）和运动模糊行为正确。最终渲染使用 RenderMan RIS 完成。

为了提高合成灵活性，我使用 Light Path Expressions（LPE）导出了一组 AOV，包括直接漫反射、间接漫反射、直接镜面反射和间接镜面反射。我还使用 Cryptomatte 在合成期间快速而准确地创建遮罩。

为了降低采样噪点，我启用了 RenderMan Machine Learning Denoiser；它在保持较低渲染时间的同时，明显提升了输出清晰度。
