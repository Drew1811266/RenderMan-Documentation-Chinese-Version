Rendering and Compositing（渲染与合成）

现在一切都组合起来了，是时候设置 Katana rendering 了，包括收集所有内容、设置不同的 PrmanGlobalStatments、RenderSettings、AOVs，并为不同 passes streamline scene。

为了在 compositing 中获得最大 control，我决定把 scene 拆成三个不同 passes：Character（Dragon）、Gold（all treasure）、Environment（columns、cave ceiling 等），以及 AO。

在 import stage，我主要在 Katana 中使用 usdIN；它现在是随 pixarUSD installation 自动随 Katana 安装的 native node。对于一些 static assets，我使用 alembicIN。

  • openUSD - 所有带 instances 的内容（例如 gold treasure、gold treasure sim、environment 的 pillars）都使用 USD 导入。选择 USD 的原因是它在 disk 上 weight 很低，并且在 primvars 方面很 versatile。

  • Alembic - 对于 static nature 的 elements（例如 cave ceiling、gold treasure 的 bottom layer），它们被导出为 ABC。
