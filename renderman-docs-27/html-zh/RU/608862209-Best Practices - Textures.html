纹理贴图基础。

我们先把桌子看作一个基础材质：它带有木质颜色和有光泽的饰面，并使用纹理。

在 Pixar，我们不使用 Metallic（金属度）工作流程。这些纹理课程旨在用 PxrSurface 教你 Pixar 的工作方式。

虽然它像抛光木材一样有轻微反射，但并不过于闪亮。参考图非常关键，它们能确保你实现艺术指导或你自己目标中规定的正确外观。

一开始这看起来很简单，但通过一些快速技巧和提示以及后续示例来拆解时，会发现有很多事情需要考虑。

标准纹理命名、建议和使用思路：

  • 纹理应根据其预期效果和分配对象来恰当命名。

    例如：<object>_<effect>.jpg，或者 tableTop_specular.jpg。

    • 不要在名称中使用空格和特殊字符，例如：theCar&boat.jpg 或 “the Car and Boat.jpg”。

    • 文件路径中也不要使用空格和特殊字符。这些纹理可能会加载失败。

    • 纹理序列应该为帧号使用清晰的命名约定。mySequence.0001.jpg 是正确写法，而不是 mySequence0001.jpg 或 mySequence.jpg.0001。

  • 大多数格式都可以使用，例如 .png、.exr、.tif 和 .jpg，但要用于渲染，它们必须转换为 .tex 格式。
    在所有符合 RenderMan 规范的 DCC 中，当你启动渲染时会发生这种转换，并且结果会缓存在磁盘上，直到它在场景中被修改或更新后才需要重新使用。包含大量纹理的场景，首次渲染时转换可能需要一段时间。
    Solaris 是这里的例外。你需要使用 Texture Manager 手动转换它们。

  • 有时会出现可互换的命名对；保持一致性对流水线以及艺术家准确工作都很重要。

    • Diffuse 和 Albedo

    • Roughness 和 Glossiness（请注意，RenderMan 使用标准的 “Roughness” 约定，其中 0 = 镜面，1 = 粗糙或接近漫反射）
      Roughness 也适用于 Diffuse 参数，因此艺术家有时会用 Glossiness 作为名称替代。

  • 纹理可以是颜色（RGB）或标量（灰度），颜色通道可以连接到匹配的参数来驱动它们。例如，如果有用，一张图像的 Red 通道可以用来驱动一个标量（浮点滑块）。

  • 法线贴图通常因其细节表现而优先于凹凸贴图（标量或灰度）。

  • ACESCG 是强烈推荐的。 作为数据使用的纹理，例如凹凸、遮罩和位移， 不应 进行线性化处理。

  • 位移贴图和其他数据纹理应使用尽可能高的精度，以避免纹理量化造成伪影。通常它们会从建模应用导出为浮点 EXR 或 Tiff。关于法线和标量位移，请参见 工作流程。

如果 Maya File Node 具备你需要的功能，或者你喜欢 Maya 工作流程在纹理序列等方面的简单性，那么没有特别的理由不能用它来做纹理制作。Maya File Node 和 PxrTexture 节点都会在视口中显示纹理。不过，只有 Maya File Node 会在视口中显示 UDIM/平铺纹理，并对视口应用校正后的伽马。
