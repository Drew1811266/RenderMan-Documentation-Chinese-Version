一开始这看起来很简单，但通过一些快速 tips 和 hints 以及后续示例来拆解时，会发现有很多事情需要考虑。

标准 texture naming、建议和使用思路：

  • Textures 应根据其预期效果和分配对象来恰当命名。

    例如：<object>_<effect>.jpg，或者 tableTop_specular.jpg。

    • 不要在名称中使用空格和特殊字符，例如：theCar&boat.jpg 或 “the Car and Boat.jpg”。

    • 文件路径中也不要使用空格和特殊字符。这些 textures 可能会加载失败。

    • Texture sequences 应该为帧号使用清晰的命名约定。mySequence.0001.jpg 是正确写法，而不是 mySequence0001.jpg 或 mySequence.jpg.0001。

  • 大多数格式都可以使用，例如 .png、exr、.tif 和 .jpg，但要用于渲染，它们必须转换为 .tex format。
    在所有符合 RenderMan 规范的 DCCs 中，当你启动渲染时会发生这种转换，并且结果会缓存在磁盘上，直到它在场景中被修改或更新后才需要重新使用。包含大量 textures 的场景，首次渲染时转换可能需要一段时间。
    Solaris 是这里的例外。你需要使用 Texture Manager 手动转换它们。

  • 有时会出现可互换的命名对；保持一致性对 pipeline 以及艺术家准确工作都很重要。

    • Diffuse 和 Albedo

    • Roughness 和 Glossiness（请注意，RenderMan 使用标准的 “Roughness” 约定，其中 0 = mirrored，1 = rough 或接近 diffuse）
      Roughness 也适用于 Diffuse 参数，因此艺术家有时会用 Glossiness 作为名称替代。

  • Textures 可以是 color（RGB）或 scalar（greyscale），颜色通道可以连接到匹配的参数来驱动它们。例如，如果有用，一张图像的 Red channel 可以用来驱动一个 scalar（float slider）。

  • Normal maps 通常因其细节表现而优先于 bump maps（scalar 或 grayscale）。

  • ACESCG 是强烈推荐的。 作为数据使用的 textures，例如 bump、masks 和 displacement， 不应 被 linearized（线性化处理）。

  • Displacement maps 和其他 data textures 应使用尽可能高的精度，以避免 texture quantization 造成 artifacts。通常它们会从建模应用导出为 floating point EXR 或 Tiff。关于 normal 和 scalar displacement，请参见 workflows。

如果 Maya File Node 具备你需要的功能，或者你喜欢 Maya workflow 在 texture sequences 等方面的简单性，那么没有特别的理由不能用它来做 texturing。Maya file node 和 PxrTexture node 都会在 viewport 中显示 textures。不过，只有 Maya file node 会在 viewport 中显示 UDIM/tiled textures，并对 viewport 应用校正后的 gamma。
