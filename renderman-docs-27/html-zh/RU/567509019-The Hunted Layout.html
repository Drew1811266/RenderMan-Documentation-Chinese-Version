一旦我把每个道具的初始代理版本都做成 USD 文件，就可以开始处理场景布局。

HuntedLayout.mov

Houdini 和 Solaris 非常适合处理单个置景资产，也适合散布大量实例。

我实现了这两种工作流：用于草地、岩石、普通植物和背景树木的散布与实例化技术。

我通过 layout 节点手动放置主道具，例如配电箱、铁路标志和邮箱。我也手动放置了草地，以及前景树木和岩石，以确保场景构图符合我的目标。当你不太在意散布的创意构图时，散布非常适合实例化对象。不过很多时候，你仍需要依靠手动放置或移动已散布对象，来适配镜头和故事。

借助 R27 中 XPU 的最新改进，我能够在 Solaris 中使用简单的天空穹顶运行交互式视口，从而获得快速且交互式的布局会话。再加上交互式降噪器的能力，你就拥有了一套能轻松处理布局工作的交互式工具包。

stagemanager.mov

额外提示：做布局时，如果资产已经应用了许多纹理，可以使用 RenderMan 中的 Occlusion Integrator。这会提供非常快速的遮蔽 IPR 会话，而不必把所有着色器覆盖成简单的灰色材质。

感谢 RenderMan 的 Dylan Sisson 提供这个提示！
