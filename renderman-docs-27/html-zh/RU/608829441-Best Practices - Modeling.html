建模

创建资产时，必须理解哪些对象属于特定类别，并据此为它们建模相应的细节层级：

主角级资产
例如主要角色，以及镜头或故事中的视觉焦点。

布景资产
这些资产可能会根据它们在场景中的位置拥有不同细节级别。近景物体可能需要更多细节。

道具
这类对象通常会尽量保持简单。出色的纹理处理能让低细节资产呈现出相当不错的效果，这一点可能会让你惊讶。

RenderMan 被设计为在可能的情况下渲染细分曲面，尤其适用于动画角色和主角级对象。这可以减少潜在撕裂，并改善近距离观看的角色和有机物体外观。在使用置换时，这通常也是很好的选择。

多边形在置换后容易分离，因此应转换为细分曲面。

为细分曲面建模时，请记住以下事项，以获得想要的形状，同时不破坏性能：

  • 使用默认 Catmull-Clark 方案转换为细分网格时，为避免过度切分，请只使用四边形。这意味着所有面都应有四条边。避免三角形和 N 边面。

  • 在基础网格中只建模必需的最低限度细节，使细分后的表面形状正确即可，不要过度添加。

  • 尽量减少折痕边；它们可能代价较高。

  • 利用更高的 micropolygonlength 设置。它会提供自动细节层级控制，数值越高越粗糙，但仍可能足以获得良好效果。合理的制作质量可以尝试 1.5 到 3 之间的设置。（注意，如果它过于粗糙，尤其是在边缘附近会产生着色伪影，此时应缓慢降低该值。任何小于 1 的数值都是 非常高 的质量。）

多边形对任何带有硬法线的对象也很有用，例如许多无机物体的分面表面：建筑、道具、布景件等。近距离渲染时，你可以软化这些法线，或添加更多建模细节让它看起来更好，但要注意，如果处理不足，着色过程中可能出现轮廓伪影。

尽管有这些注意事项，背景和道具对象使用多边形仍能很好地渲染，并且性能相当不错。这些对象通常不会变形，而且在场景中数量很多。

把这些对象都做成细分曲面，可能会让内存占用膨胀，而视觉收益很小。

在多边形上优先使用凹凸贴图而不是置换；如果确实需要置换，则将它们转换为细分曲面。

布局 UV 时，避免重叠 UV，并注意带有翻转边的 UV 壳，因为这可能在置换时产生伪影，或在渲染期间触发警告。

我们的纹理缓存机制可以在场景中填充高质量纹理细节和外观，但整场景中用于光线追踪的几何体必须驻留在内存中。只要谨慎选择建模方式，你就能在场景中放入相当多的内容，并一次渲染大量纹理，规模可达许多 GB。
