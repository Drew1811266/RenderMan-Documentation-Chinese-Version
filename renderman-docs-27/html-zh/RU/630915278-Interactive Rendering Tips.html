一份用于优化 RenderMan XPU 交互式渲染体验的实用指南

Interactive Denoiser（交互式降噪器）

RenderMan 出色的批处理降噪器也可以在交互式视口模式中运行，因此开启它后，当你处理着色器、灯光、相机和最终细化时，它会对视口进行降噪。

对于较小的场景，例如道具外观开发，你可能希望把 Interactive Denoiser Min Samples 保持在 4；对于几何体、实例和着色器更多的大型场景，可以把它提高到 10 或 15。

这会告诉 RenderMan 在降噪器开始发挥作用之前要渲染多少样本。

调整 Interval 也能带来一些交互式降噪加速。

这完全取决于你的场景及其复杂度。

Image Resolution（图像分辨率）

降低图像分辨率会提升交互式性能，尤其是在更大、更复杂的场景中。

Scale Resolution（缩放分辨率）

缩放 IPR 分辨率可以显著提升交互响应速度。例如，在 RfB 中有选项可把分辨率缩放改为 75%、50%、33%、25% 和 10%

Solaris 也有类似选项。

实际效果是将分辨率降低，再让 DCC 把图像重新缩放回视口尺寸，为艺术家提供更“块状”的渲染；这通常已经足够在布局、灯光和场景调度中做创作决策。

Pruning（裁剪）

裁剪（或隐藏）镜头画幅或相机视锥之外的几何体，有助于加快 RenderMan 的首像素时间并提升交互性。你需要记住，相机外的对象可能仍会影响镜头的整体外观，例如投射阴影；但如果这不是问题，裁剪这些对象就是良好实践。

按距离裁剪对象也同样有效。

Progressive Pixels（渐进像素）

将 RenderMan Progressive Pixels 设置为 2 - 4，会提高视口交互速度。

Time to First Pixel（首像素时间）

当你第一次启动 XPUGPU 或 Hybrid 交互式渲染时，NVidia 驱动需要进行一些初始的繁重工作，把 RenderMan 编译成适合你的显卡的 CUDA 代码。

第二次重新启动 IPR 会话时，这项工作已经被驱动缓存，因此你的首像素时间会快得多。

Camera Distance（相机距离）

如果对象应用了 SubDivision 且靠近相机，XPU 的启动时间会比这些对象距离更远时更长。较低的 MicroPolygon Length 也会影响 XPU 的初始启动时间。

关闭 Subdivision 与 Displacement

关闭 Subdivision 和 Displacement 会加快 XPU 的首像素时间；在很多情况下，即使不启用这两个功能，你仍然可以做出有效的艺术判断。

提高 MicroPolygon Length

网格会在渲染开始时相对于相机进行切分（dicing）。提高 MicroPolygon Length 会减少渲染器启动前需要完成的曲面细分（tessellation）工作量，从而加快首像素时间。这也会带来更低的内存消耗。

Material Override（材质覆盖）

在处理布局或相机场景调度时，你并不总是需要看到所有纹理和复杂的着色网络，因此可以给场景几何体应用一个简单的材质覆盖，或使用 PxrOcclusion 积分器。

完成你的布局工作，然后回到 PxrPathTracer！

只使用几何体的 DisplayColor 来渲染覆盖材质，也是一种有用技术。

Render Regions（局部渲染区域）

永远不要低估裁剪窗口（crop window）或感兴趣渲染区域（render region of interest）的力量，它们可以加快场景中特定区域的细化。
