Using Motion Blur

Source: https://rmanwiki-27.pixar.com/space/RFH27/544644938/Using+Motion+Blur
Downloaded: 2026-06-15T03:49:12+00:00

See Houdin's Motion Blur page for more information.

To turn on motion blur, enable the RenderMan ROP's Properties | Motion Blur: Allow Motion Blur.

Motion Blur Type: This option toggles the sampling of motion-blurred micropolygon in the hider, and is thus only meaningful if motion blur is being used in your scene. By default, this hider option is enabled (set to 1), meaning that micropolygons will be blurred and sampled in the hider. When disabled (set to 0), micropolygons will no longer be blurred before sampling; however, the dPdtime variable will still be computed for all objects. This variable can be output into a separate image using an arbitrary output variable (aov) and used to perform motion blur as a postprocess.

Xform Time Samples: Number of transformation blur motion samples for this object. When set to 0, the global parameter is used. Dial this up to output more motion segments for objects in order to create a smooth blur/motion path in the render. This would help curving motions stay curved as opposed to linear from one point to another, with enough samples

Geo Time Samples:  Number of deformation blur motion samples for this object. When set to 0, the global parameter is used. Dial this up to output more samples for objects that deform in order to create a smooth motion path in the render. This would help curving motions stay curved as opposed to linear from one point to another, with enough samples

Shutter Offset: An offset to be added to the global motion blur times.

If you need to render motion blur for simulated objects (breaking, falling, bouncing, particles, etc), you need to enable Geometry Velocity Blur on the object node. This provides the correct data to render motion blur and deformation blur in RenderMan for these objects. This is because changing topology requires special consideration for motion blur to work.
