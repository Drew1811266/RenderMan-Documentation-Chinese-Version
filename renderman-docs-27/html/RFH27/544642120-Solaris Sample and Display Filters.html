General information about display and sample filters in RenderMan: Sample and Display Filters

To add a sample of display filter simply add one of the dedicated LOP node to create the corresponding USD filter primitive.

The previous ways (either via the dedicated tab on the Render Settings LOP or via the PxrSampleFilter and PxrDisplayFilter LOPs) to add a filter, while still supported, are deprecated.

Then assign it to the render settings primitive:

  • Houdini 20.5 and below: use a PxrAssign LOP (or either the PxrAssignSampleFilter or PxrAssignDisplayFilter alias) to assign a filter primitive to a render settings primitive. The render settings primitive must exist before assignment.

  • Houdini 21.0 and above: while the PxrAssign LOP workflow is still valid, it is also possible to assign directly a filter on the Render Settings LOP.

If the assignment was properly done, in the Scene Graph, on the Render Settings primitive, the filter primitive should be listed on either ri:sampleFilters or ri:displaytFilters attributes in H21 and above or, on either outputs:ri:sampleFilters or outputs:ri:displayFilters attributes for H20.5 and below.
