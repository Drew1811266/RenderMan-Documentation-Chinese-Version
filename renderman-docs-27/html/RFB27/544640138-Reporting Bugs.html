Reproducing Issues/Bugs

When reporting issues or bugs, more often than not, we will require a method to reproduce the issue. This usually means we will need to see your files/assets.

In RfB, there is a convenient Package Scene button that can be used to package your files together ready to be sent to RenderMan Support.

Debug Logging

For crashes, RfB can log your render session to a debug python file. To start a log, turn on Debug Logging in the render properties tab as seen in the image below.

This option has to be turned on before rendering has started

The log will be placed in the same directory as the current blend file (with a .debug.py file extension). Please ensure that the folder the current blend file is in is writable.

The logging file can take up a lot of disk space, especially for heavier scenes. You’ll want to make sure the option is turned off after you’ve finished generating the log file.

Package Scene

To package your scene, in the viewport press the 'n' key. This should bring up the side menu.

Next, click on the Renderman tab.

Under Utilities, click the Package Scene button. This will bring up a file browser.

Select a new or an existing empty directory, and then click Package Scene. The directory should now contain a new ZIP file, which you can then send in with your bug report.

Options:

  • Include Blend File: this includes the .blend file with the ZIP package

  • Include Debug Log: include the logging .debug.py file with the ZIP package, if it exists.

  • Use RenderMan Textures: this replaces all of the texture file paths with the equivalent RenderMan .tex textures in the scene.
