Neon Modelling

Source: https://rmanwiki-27.pixar.com/space/RU/592216069/Neon+Modelling
Downloaded: 2026-06-15T03:49:12+00:00

My aim with this project was to create a fully procedural Neon Sign maker, so that regardless of the letters or design I wanted to create, Houdini would generate the geometry procedurally for me.

I began by creating the vector designs in Illustrator, and once I was happy with the design, I exported a single SVG file.

This SVG file was imported into Houdini, and then each part of each letter was blasted out into separate pieces for the main letter panels, the inner and outer extrusion, and then one for each of the tubes.

I created a procedural system to find the first and last point on each neon tube curve using a grouprange node.

I also used a carve node to adjust the placement of these points along the curve, allowing me to change the break points in the neon sign.

I was able to use a copytopoints node to instance the neon hole geometry to the first and last points on my neon curves.

The neon tube holders, again, used a procedural approach using the carve and copytopoints nodes to place them along the neon tube curves.

The bolts were hand-placed using a node from Sidefx Labs called Labs Pick and Place. This allowed me to add a touch of irregularity to the sign.

To add a final element of handcrafted imperfection to the signs, I used the sculpt node to introduce subtle bends and dents to the panels.

The UVs and Texel density were also procedural, using Labs Auto UV and texeldensity nodes

Once I had one letter done, it was a rapid and easy process to create all the other letters and arrow. Note: The neon tubes themselves were imported separately into Solaris. I wanted to keep them separated from the sign itself for the texturing and shading workflow.
