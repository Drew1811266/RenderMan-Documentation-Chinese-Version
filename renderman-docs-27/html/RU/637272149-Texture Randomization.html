Texture Randomization

Downloaded: 2026-06-15T03:49:12+00:00

RenderMan features several nodes that can provide random texture colouring, file selection, and random orientation/placement.

The two nodes we’re going to take a look at are the PxrVary node and the PxrRandomTileManifold node.

In this example, we have seven spray cans, and at the moment, they look pretty dull.

Our goal in this lesson is to create lids for each can in random colors.

We also have seven different labels we’ve designed and want to apply them to the cans’ labels randomly.

Spray Can Colored Lids

Let’s tackle the lids first, using an incredibly useful randomization node called PxrVary.

This node takes a base color (or colored input) and allows for hue, saturation, gamma, and many other parameters per instance to have random values.

Let’s start by connecting a PxrVary into the Diffuse color of our PxrSurface and set the Input Color to a bright orange.

This will turn all the lids to the same orange color, which is not what we want.

We want each lid to have a random color.

Adjusting the following values yields the image below.

This looks ‘ok’, but we don’t really want to let RenderMan decide on which random colors to choose. We want to choose them ourselves.

We wish to art-direct the color palette from which the PxrVary will randomly select colors.

The way to achieve this is to use a PxrRamp node, from where we set our ramp colors. We also need to set the Ramp Type to Random Object Color.

Gotcha! This technique will only work if you set the Ramp Type to Random Object Color.

It’s helpful to understand that this is a random node, so at times it can be tricky to art-direct which lid gets which random color. Sometimes, as you can see, the PxrVary will randomly assign two lids the same color.

Also, the width of the colors in the ramp determines random assignment and probability, so it can be a little tricky to dial in the exact random look you’re going for.

The keyword here is Random!

If you had 100 cans and didn’t really care which lid had which color, then the PxrVary is the solution for you, but if you needed particular cans to feature specific colored lids, then defining a PrimVar value to each can (and then specifying the PrimVar name in the PrimVar attribute on the PxrVary) is the way, but that’s for another lesson.

We asked for random-colored lids … and we got random-colored lids.

Let’s move on to working on the labels next.

Spray Can Labels

Here we have several textures we want to assign to each label, again randomly — we don’t really mind which goes with which label. We want it random!

There are two nodes we need to achieve this.

The first is the PxrMultiTexture node, which we’ll load in all the maps into each filename parameter.

Then, we need a PxrRandomTextureManifold node to control the randomness of the PxrMultiTexture files by assigning them to our cans.

Note: You need to set the Number of Textures parameter to the number of textures in the file list you want to use.

The PxrRandomTextureManifold has a variety of other parameters that can be tweaked to add more randomization, such as random texture orientation, flipping, offset, and many more, so the amount of texture randomization is … endless!

Further uses

Both of these nodes aren’t just for randomly choosing colors or diffuse textures.

For instance, the PxrRandomTextureManifold can also be used to randomly assign and transform roughness/bump maps, layer random textures, transform random edge breakup masks, etc.

The PxrVary can be used to add random color/saturation variation to roughness maps, so each instance has different roughness values. The same technique can be used to set random bump and specular values, as well as to control random thickness on a clear coat.

The possibilities of both these nodes are endless.

If you want complete shading randomization across all your instances, these two nodes are a good place to start!
